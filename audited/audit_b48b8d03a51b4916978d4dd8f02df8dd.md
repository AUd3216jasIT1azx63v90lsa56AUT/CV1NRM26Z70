### Title
Authorized party can revoke EcrecoverAuthorizer's Midnight authorization, permanently blocking victim's signature-based revocation path - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
When a victim pre-authorizes `EcrecoverAuthorizer` in Midnight and then uses it to grant full Midnight control to address X via a signed `Authorization`, X can immediately call `midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim)` directly. Because X is now a valid authorized party for victim in Midnight, this call succeeds and strips `EcrecoverAuthorizer` of its Midnight delegation. Any subsequent attempt by victim to use `EcrecoverAuthorizer` to revoke X's authorization will revert with `Unauthorized()`, because Midnight's `setIsAuthorized` guard re-checks `isAuthorized[victim][ecrecoverAuthorizer]` at call time.

### Finding Description

**Exact code path:**

`Midnight.setIsAuthorized` enforces: [1](#0-0) 

```solidity
function setIsAuthorized(address authorized, bool newIsAuthorized, address onBehalf) external {
    require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
    isAuthorized[onBehalf][authorized] = newIsAuthorized;
    ...
}
```

`EcrecoverAuthorizer.setIsAuthorized` calls into Midnight with `msg.sender = address(ecrecoverAuthorizer)`: [2](#0-1) 

The authorization check in `EcrecoverAuthorizer` reads `isAuthorized` at signature-submission time: [3](#0-2) 

**Exploit flow:**

**Preconditions:**
- `midnight.isAuthorized(victim, address(ecrecoverAuthorizer)) == true` (victim has pre-authorized EcrecoverAuthorizer as required for it to act on their behalf)
- Victim has signed `Authorization{authorizer=victim, authorized=X, isAuthorized=true, nonce=0, deadline=T}`

**Step 1 — Anyone submits victim's signature to EcrecoverAuthorizer:**
- `EcrecoverAuthorizer.setIsAuthorized(auth, sig)` passes all checks: deadline valid, nonce 0 matches, `signer == authorization.authorizer` (victim)
- Calls `midnight.setIsAuthorized(X, true, victim)` — passes because `isAuthorized[victim][ecrecoverAuthorizer] == true`
- State after: `isAuthorized[victim][X] = true`

**Step 2 — X immediately calls Midnight directly:**
- `midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim)` with `msg.sender = X`
- Guard: `require(victim == X || isAuthorized[victim][X])` → `isAuthorized[victim][X] == true` ✓
- State after: `isAuthorized[victim][ecrecoverAuthorizer] = false`

**Step 3 — Victim attempts to revoke X via EcrecoverAuthorizer:**
- Victim signs `Authorization{authorizer=victim, authorized=X, isAuthorized=false, nonce=1}`
- `EcrecoverAuthorizer.setIsAuthorized` passes its own checks (nonce 1, valid sig)
- Calls `midnight.setIsAuthorized(X, false, victim)` with `msg.sender = address(ecrecoverAuthorizer)`
- Guard: `require(victim == ecrecoverAuthorizer || isAuthorized[victim][ecrecoverAuthorizer])` → `false || false` → **REVERTS `Unauthorized()`**

**Why no existing check stops it:** There is no constraint in `Midnight.setIsAuthorized` preventing a delegated caller (X) from modifying the `isAuthorized` entry for the very contract (EcrecoverAuthorizer) that granted X its delegation. The guard is a pure point-in-time check with no chain-of-trust awareness. [4](#0-3) 

### Impact Explanation
Victim permanently loses the ability to use `EcrecoverAuthorizer` to revoke X's Midnight authorization. X retains full Midnight control over victim's account (withdraw, repay, withdrawCollateral, setConsumed, further authorization changes). Victim's only recourse is to submit an on-chain `midnight.setIsAuthorized(X, false, victim)` transaction directly — but if victim is a smart contract wallet or hardware wallet that cannot call Midnight directly, or if X front-runs any such attempt by re-authorizing itself, the victim may have no recovery path.

### Likelihood Explanation
**Preconditions:** Victim must have pre-authorized `EcrecoverAuthorizer` in Midnight (the standard setup shown in `SetIsAuthorizedWithSigTest.sol` line 56) and must have signed an `Authorization` granting `isAuthorized=true` to X. Both are the intended normal usage of the periphery contract. [5](#0-4) 

**Feasibility:** Steps 1 and 2 can be executed in the same block or even the same `multicall`. The signed authorization is public once submitted to a mempool or relayer. Any party who sees the signature (including X themselves) can front-run or immediately follow up with step 2. Repeatability is unconditional once preconditions are met.

### Recommendation
Remove the requirement for `EcrecoverAuthorizer` to be pre-authorized in Midnight's `isAuthorized` mapping. Instead, Midnight should expose a dedicated `setIsAuthorizedWithSig` entry point that accepts the EIP-712 authorization struct and signature directly, verifying the signer internally without routing through the `isAuthorized` delegation map. This eliminates the attack surface entirely: `EcrecoverAuthorizer` would call `midnight.setIsAuthorizedWithSig(authorization, signature)` and Midnight would perform the `ecrecover` check itself, requiring no prior on-chain authorization of `EcrecoverAuthorizer`.

Alternatively, if the current architecture is preserved, `Midnight.setIsAuthorized` should prevent a delegated caller (`msg.sender != onBehalf`) from revoking the authorization of any address that was the source of `msg.sender`'s own delegation — but this requires tracking the authorization chain and is significantly more complex.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract EcrecoverAuthorizerRevocationBlockTest is BaseTest {
    function testXCanBlockVictimRevocation() public {
        address victim = makeAddr("victim");
        address X      = makeAddr("X");

        // Step 0: victim pre-authorizes EcrecoverAuthorizer in Midnight (standard setup)
        vm.prank(victim);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
        assertTrue(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));

        // Step 1: victim signs Authorization{authorizer=victim, authorized=X, isAuthorized=true, nonce=0}
        Authorization memory auth = Authorization({
            authorizer: victim,
            authorized: X,
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep  = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest     = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[victim], digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Anyone submits the signature — X is now authorized by victim in Midnight
        ecrecoverAuthorizer.setIsAuthorized(auth, sig);
        assertTrue(midnight.isAuthorized(victim, X));

        // Step 2: X immediately revokes EcrecoverAuthorizer's Midnight authorization
        vm.prank(X);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim);
        assertFalse(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));

        // Step 3: victim tries to revoke X via EcrecoverAuthorizer — MUST REVERT
        Authorization memory revokeAuth = Authorization({
            authorizer: victim,
            authorized: X,
            isAuthorized: false,
            nonce: 1,
            deadline: block.timestamp + 1 days
        });
        bytes32 rStructHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, revokeAuth));
        bytes32 rDigest     = keccak256(bytes.concat("\x19\x01", domainSep, rStructHash));
        (uint8 rv, bytes32 rr, bytes32 rs) = vm.sign(privateKey[victim], rDigest);
        Signature memory revokeSig = Signature({v: rv, r: rr, s: rs});

        vm.expectRevert(); // Midnight.Unauthorized
        ecrecoverAuthorizer.setIsAuthorized(revokeAuth, revokeSig);

        // Assert: X still has full Midnight control over victim
        assertTrue(midnight.isAuthorized(victim, X));
        // Assert: victim cannot use EcrecoverAuthorizer to revoke X
        assertFalse(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));
    }
}
```

**Expected assertions:** `vm.expectRevert()` fires on the revocation attempt; final `assertTrue(midnight.isAuthorized(victim, X))` confirms X retains control; final `assertFalse` confirms EcrecoverAuthorizer is permanently de-authorized for victim.
