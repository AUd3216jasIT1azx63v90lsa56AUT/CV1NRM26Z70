### Title
`isHealthy` reverts on arithmetic overflow in `mulDivDown` when `collateral * price > type(uint256).max` — (`src/Midnight.sol`)

### Summary

`isHealthy` accumulates `maxDebt` by calling `mulDivDown(collateral[i], price, ORACLE_PRICE_SCALE)` for each active collateral. The `mulDivDown` implementation performs a plain Solidity 0.8+ multiplication `x * y`, which reverts on overflow rather than wrapping. When a borrower holds a collateral amount near `type(uint128).max` and the oracle returns a price large enough that `collateral * price > type(uint256).max`, `isHealthy` reverts unconditionally, causing any caller that depends on it — including `take()` for a seller with debt and `withdrawCollateral()` — to revert as well.

### Finding Description

**Root cause — `mulDivDown` is not overflow-safe:** [1](#0-0) 

```solidity
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;   // plain multiplication — reverts on uint256 overflow in Solidity ≥0.8
}
```

There is no `unchecked` block and no 512-bit intermediate. Solidity 0.8 overflow protection turns any `x * y > type(uint256).max` into a hard revert.

**Call site in `isHealthy`:** [2](#0-1) 

```solidity
while (_collateralBitmap != 0) {
    uint256 i = UtilsLib.msb(_collateralBitmap);
    CollateralParams memory collateralParam = market.collateralParams[i];
    uint256 price = IOracle(collateralParam.oracle).price();
    maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)  // ← overflows here
                .mulDivDown(collateralParam.lltv, WAD);
    _collateralBitmap = _collateralBitmap.clearBit(i);
}
```

- `_position.collateral[i]` is `uint128` (max `2^128 − 1 ≈ 3.4 × 10^38`).
- `price` is an unchecked `uint256` returned by the oracle.
- `ORACLE_PRICE_SCALE = 1e36`.

Overflow condition: `collateral[i] * price > 2^256 − 1`.

With `collateral[i] = type(uint128).max`, overflow occurs when `price > 2^256 / 2^128 = 2^128 ≈ 3.4 × 10^38`. Expressed in oracle-scale units, this means the collateral is priced at more than `3.4 × 10^38 / 1e36 = 340` loan-asset units per collateral unit — a realistic ratio for many token pairs (e.g., 1 high-value token vs. a low-denomination loan token).

**Attacker-controlled inputs:**

1. **Borrower** supplies collateral up to `type(uint128).max` (enforced by `toUint128` cast, so this is the protocol-defined maximum).
2. **Oracle price** is an external `uint256` with no upper-bound validation anywhere in the protocol.

No admin key or malicious oracle is required: a legitimate oracle for a high-value/low-denomination token pair can return a price in this range. The market creator (listed as an unprivileged role) can also permissionlessly deploy a market with any oracle.

**Why existing checks do not stop it:**

- `toUint128` only prevents collateral from exceeding `type(uint128).max`; it does not prevent the product `collateral * price` from exceeding `type(uint256).max`.
- There is no oracle price cap or sanity check anywhere in the protocol.
- `isHealthy` has no `try/catch`; a revert inside it propagates to every caller. [3](#0-2) 

### Impact Explanation

Any code path that calls `isHealthy` on a position with the overflow-triggering collateral/price combination will revert:

- **`take()` on a seller with debt** — the seller health check calls `isHealthy`; the taker's transaction reverts, freezing the order book for that seller.
- **`withdrawCollateral()`** — calls `isHealthy` to verify the borrower remains healthy after withdrawal; reverts, permanently locking the borrower's collateral.

Funds are frozen without any admin escape hatch.

### Likelihood Explanation

**Preconditions:**
1. A market exists where the oracle can return a price `> 2^128` (i.e., collateral worth > 340× the loan asset per unit). This is realistic for token pairs with large denomination differences.
2. A borrower supplies collateral near `type(uint128).max`. For tokens with few decimals (e.g., 0–6 decimals), this is a large but finite and depositable amount.

Both conditions can be satisfied without any privileged action. The overflow is deterministic and repeatable once the state is reached. A fuzz campaign over `(collateral, price)` pairs will find it quickly.

### Recommendation

Replace the plain-multiplication `mulDivDown` with a 512-bit overflow-safe implementation (e.g., Solady's `FullMath.mulDiv` or OpenZeppelin's `Math.mulDiv`):

```solidity
// Replace:
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;
}

// With a 512-bit safe version that does not revert on intermediate overflow:
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return Math.mulDiv(x, y, d); // OpenZeppelin, or equivalent assembly-based implementation
}
```

This ensures `isHealthy` never reverts due to arithmetic for any `uint128` collateral and any `uint256` oracle price.

```solidity
import "forge-std/Test.sol";

contract IsHealthyOverflowTest is Test {
    Midnight midnight;

    function setUp() public {
        midnight = new Midnight(/* constructor args */);

    function testFuzz_isHealthy_noOverflowRevert(
        uint128 collateralAmount,

        // Bound to realistic but large values
        collateralAmount = uint128(bound(collateralAmount, 1, type(uint128).max));
        // Price range that can cause overflow: price > type(uint256).max / collateralAmount
        oraclePrice = bound(oraclePrice, type(uint256).max / uint256(collateralAmount), type(uint256).max);

        MockOracle oracle = new MockOracle(oraclePrice);
        CollateralParams[] memory params = new CollateralParams[](1);
        params[0] = CollateralParams({
            token: address(0x1),
            lltv: 0.8e18,
            maxLif: 1.1e18,
            oracle: address(oracle)
        });
        Market memory market = Market({
            loanToken: address(0x2),

            maturity: block.timestamp + 365 days,
            rcfThreshold: 0,
            enterGate: address(0),
            liquidatorGate: address(0)
        });

        // Set up borrower position with large collateral and nonzero debt
        address borrower = address(0xBEEF);
        // ... supply collateral and create debt via take() ...

        bytes32 id = midnight.toId(market);

        // Assert: isHealthy must not revert
        // Expected: returns true or false, never reverts
        bool healthy = midnight.isHealthy(market, id, borrower); // ← reverts with arithmetic overflow
        // Assertion: the line above should not revert
    }
}
```

**Expected assertion failure:** `isHealthy` reverts with `Panic(0x11)` (arithmetic overflow) instead of returning `true` or `false`, violating the invariant that healthy-position checks must not revert for valid protocol state.
