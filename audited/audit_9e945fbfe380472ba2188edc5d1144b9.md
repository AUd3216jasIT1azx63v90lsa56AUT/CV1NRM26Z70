### Title
Codeless `loanToken` Passes `touchMarket()` Validation, Permanently DoS-ing All Loan-Token Operations on the Market - (File: src/libraries/SafeTransferLib.sol / src/Midnight.sol)

### Summary
`touchMarket()` validates collateral parameters, maturity, LLTV, and maxLif, but performs **no check** that `market.loanToken` has deployed code. `SafeTransferLib.safeTransferFrom` and `safeTransfer` both guard with `require(token.code.length > 0, NoCode())`. Any unprivileged actor can therefore create a market whose `loanToken` is a codeless address (e.g., `address(1)`, an undeployed EOA, or a precompile), causing every subsequent call that touches the loan token to revert permanently, making the market permanently inoperable.

### Finding Description

**Root cause — missing code-existence check in `touchMarket()`:**

`touchMarket()` validates only collateral-side parameters and maturity: [1](#0-0) 

There is no `require(market.loanToken.code.length > 0)` anywhere in this function. A market with `loanToken = address(1)` passes every check and is permanently stored (`_marketState.tickSpacing = DEFAULT_TICK_SPACING`).

**Trigger — `SafeTransferLib` NoCode guard:**

Both transfer helpers enforce code existence: [2](#0-1) [3](#0-2) 

**Affected call sites in `take()`:**

After all position/accounting state mutations (lines 408–418), `take()` calls: [4](#0-3) 

Both calls revert with `NoCode()`. Because the revert propagates, the in-transaction state mutations are rolled back, but the market itself (created by `touchMarket` at line 347) remains permanently registered with `tickSpacing > 0`.

**Other permanently broken entry-points on the same market:**

- `withdraw()` → `SafeTransferLib.safeTransfer(market.loanToken, ...)` at line 499 [5](#0-4) 
- `repay()` → calls `safeTransferFrom` on `loanToken` [6](#0-5) 
- `claimContinuousFee()` → `safeTransfer(market.loanToken, ...)` at line 324 [7](#0-6) 
- `liquidate()` → transfers loan token for repayment

**What still works:** `supplyCollateral` and `withdrawCollateral` transfer only the collateral token, so collateral is not frozen. However, the market is permanently inoperable for its primary purpose (lending, borrowing, repayment, withdrawal of credit).

**Exploit flow:**
1. Attacker calls `touchMarket(market)` with `market.loanToken = address(1)` and valid collateral params. Succeeds; market is created.
2. Any user calls `take(offer, ...)` on this market. `touchMarket` is a no-op (market already exists). State mutations execute. `safeTransferFrom(address(1), ...)` reverts with `NoCode()`. Transaction reverts.
3. Same revert occurs for every future `take`, `withdraw`, `repay`, `liquidate` call on this market — permanently.

### Impact Explanation
The market is permanently registered but permanently inoperable for all loan-token flows. No credit can be issued, no debt repaid, no credit withdrawn, no liquidation executed. This is a complete, irreversible DoS on the market. Collateral supplied to this market can still be withdrawn (collateral token path is unaffected), so collateral funds are not frozen, but the market itself is a dead end.

### Likelihood Explanation
Preconditions are trivially satisfiable by any unprivileged caller: supply a valid collateral array, a valid maturity, and set `loanToken` to any address with no code (precompile `address(1)`–`address(9)`, any undeployed address). No special role, signature, or capital is required. The attack is repeatable for any market ID not yet created. It is a one-shot, zero-cost griefing action.

### Recommendation
Add a code-existence check for `market.loanToken` inside `touchMarket()` before the market is registered:

```solidity
require(market.loanToken.code.length > 0, LoanTokenHasNoCode());
```

This mirrors the pattern already used in `SafeTransferLib` and closes the gap at the point of market creation rather than at every transfer site.

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;
            uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];
            _marketState.settlementFeeCbp3 = _defaultSettlementFeeCbp[3];
            _marketState.settlementFeeCbp4 = _defaultSettlementFeeCbp[4];
            _marketState.settlementFeeCbp5 = _defaultSettlementFeeCbp[5];
            _marketState.settlementFeeCbp6 = _defaultSettlementFeeCbp[6];
            _marketState.continuousFee = defaultContinuousFee[market.loanToken];
            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);
        }
        return id;
    }
