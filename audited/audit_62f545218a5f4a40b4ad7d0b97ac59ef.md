### Title
Dust Buy Offer with `offerPrice < WAD` Bypasses `maxAssets` Cap and Grants Free Credit — (`src/Midnight.sol`)

### Summary
When a buy offer is created with `offer.maxAssets > 0` and a tick where `offerPrice < WAD`, taking it with `units = 1` causes `buyerAssets = mulDivDown(1, buyerPrice, WAD) = 0`. Because the `maxAssets` consumed counter increments by `buyerAssets` (zero), the cap check always passes — even on a fully-consumed offer — and the take can be repeated indefinitely. Each successful take increases the maker's credit and the taker's debt by 1 unit with zero token transfer, corrupting the asset-accounting invariant. This is reachable through `SetterRatifier.setIsRootRatified` as the ratification path.

### Finding Description
**Exact code path** (`src/Midnight.sol`, `take()`):

```
Line 363: buyerAssets = units.mulDivDown(buyerPrice, WAD)
           = 1 * buyerPrice / WAD
           = 0   (because buyerPrice < WAD when offerPrice < WAD)

Line 364: sellerAssets = units.mulDivDown(sellerPrice, WAD) = 0

Line 367-369 (maxAssets branch, offer.buy == true):
    newConsumed = consumed[maker][group] += buyerAssets   // += 0
    require(newConsumed <= offer.maxAssets)               // 0 <= 1 → passes
                                                          // even if already consumed == maxAssets

Lines 408-414: buyerPos.credit += 1, sellerPos.debt += 1  (units = 1)

Lines 455-456:
    safeTransferFrom(loanToken, payer, address(this), buyerAssets - sellerAssets) // transfer 0
    safeTransferFrom(loanToken, payer, receiver, sellerAssets)                    // transfer 0
```

**Attacker-controlled inputs:**
- `offer.buy = true`, `offer.maxAssets = 1`, `offer.tick` set to any tick where `tickToPrice(tick) < WAD` (any tick below `MAX_TICK`)
- `units = 1`

**Ratification path:** Operator calls `SetterRatifier.setIsRootRatified(maker, root, true)` (permitted because `isAuthorized[maker][operator] == true`). The root encodes the dust offer. `Midnight.take()` calls `isRatified()`, which passes the Merkle proof check and returns `CALLBACK_SUCCESS`.

**Why existing checks fail:**
- `require(newConsumed <= offer.maxAssets)` — passes because `consumed += 0`; the counter never advances regardless of how many times the offer is taken
- `require(UtilsLib.atMostOneNonZero(offer.maxAssets, offer.maxUnits))` — passes (only `maxAssets` is set)
- No check enforces `units == 0 || buyerAssets > 0`

### Impact Explanation
The maker's `credit` increases by `units` per call with zero loan-token inflow to the contract. The taker's `debt` increases by `units` with zero loan-token outflow. Because `consumed` never advances, the offer can be replayed an unbounded number of times. The maker can accumulate arbitrary credit redeemable via `withdraw()`, draining loan tokens that were deposited by legitimate lenders. The contract's balance invariant — that every unit of credit is backed by a corresponding asset deposit — is broken.

### Likelihood Explanation
Preconditions are minimal: any maker can create a buy offer at any tick below `MAX_TICK` with `maxAssets = 1`. An authorized operator (or the maker directly) calls `setIsRootRatified`. Any taker can then call `take(offer, ..., 1, ...)` repeatedly. No privileged role, no oracle manipulation, and no special token behavior is required. The attack is repeatable in a single transaction via `multicall`.

### Recommendation
Add a guard in `take()` that rejects non-zero `units` when the resulting asset amounts are both zero:

```solidity
require(units == 0 || buyerAssets > 0 || sellerAssets > 0, ZeroAssetTake());
```

Alternatively, require that `buyerAssets > 0` whenever `units > 0` and `offer.maxAssets > 0` (since the cap is asset-denominated, a zero-asset fill is semantically meaningless and bypasses the cap).

### Proof of Concept
The existing test `testBugBuyMaxAssetsBypass` in `test/TakeTest.sol` (lines 857–889) is a complete Foundry unit test that already proves the bug. To extend it as a replay PoC:

```solidity
function testDustOfferUnboundedReplay() public {
    collateralize(market, borrower, 1000);
    lenderOffer.maxUnits = 0;
    lenderOffer.maxAssets = 1;
    lenderOffer.tick = MAX_TICK - 16; // offerPrice < WAD → buyerAssets = 0

    uint256 creditBefore = midnight.creditOf(id, lender);

    // Replay 100 times — consumed never advances
    for (uint256 i; i < 100; i++) {
        take(1, borrower, lenderOffer);
    }

    // Assertions
    assertEq(midnight.consumed(lender, lenderOffer.group), 0);       // cap never consumed
    assertEq(midnight.creditOf(id, lender), creditBefore + 100);     // 100 free credit units
    assertEq(midnight.debtOf(id, borrower), 100);                    // 100 debt, 0 assets received
    assertEq(loanToken.balanceOf(address(midnight)), 0);             // no backing assets
}
```

Expected: all assertions pass, demonstrating unbounded offer replay and free credit accumulation with no asset transfer.
