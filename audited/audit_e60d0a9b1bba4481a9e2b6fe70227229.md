Audit Report

## Title
Signature Malleability in `setIsAuthorized` Enables Front-Running to Consume Victim's Nonce - (File: `src/periphery/EcrecoverAuthorizer.sol`)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` passes raw `(v, r, s)` to `ecrecover` with no check that `s` is in the lower half of the secp256k1 curve order. Because the function is fully permissionless (`msg.sender` is never validated), an attacker can observe a pending `setIsAuthorized` transaction, compute the malleable signature form, and front-run the victim to consume their nonce — causing the victim's transaction to revert with `InvalidNonce()`.

## Finding Description
**Root cause:** No low-`s` guard exists anywhere in `setIsAuthorized` or the `Signature` struct definition.

`src/periphery/EcrecoverAuthorizer.sol` line 31:
```solidity
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
```
There is no `require(uint256(signature.s) <= 0x7FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF5D576E7357A4501DDFE92F46681B20A0)` guard.

`src/periphery/interfaces/IEcrecoverAuthorizer.sol` lines 5–9 define `Signature` as a plain `(uint8 v, bytes32 r, bytes32 s)` struct with no constraints.

**Permissionless entry:** `setIsAuthorized` never checks `msg.sender` against `authorization.authorizer` (the only `msg.sender` reference in the file is in the emitted event at line 39). Any caller can submit any valid signed authorization.

**Exploit flow:**
1. Victim signs `Authorization{authorizer, authorized, isAuthorized, nonce=N, deadline}` producing `(v, r, s)` and broadcasts `setIsAuthorized(auth, {v, r, s})`.
2. Attacker observes the mempool, computes the malleable form: `v' = 55 - v` (flips between 27↔28), `s' = secp256k1_order - s`.
3. Attacker front-runs with `setIsAuthorized(auth, {v', r, s'})`.
4. Attacker's tx: line 26 passes (`nonce == N`), nonce incremented to N+1; `ecrecover(digest, v', r, s')` returns the same `authorizer` address; authorization is written to Midnight.
5. Victim's tx: line 26 fails (`nonce[authorizer]` is now N+1 ≠ N) → reverts with `InvalidNonce()`.

**Existing checks and why they fail:** The `InvalidNonce()` check at line 26 and `InvalidSignature()` check at line 32 both pass for the malleable form — the nonce is valid at the time of the attacker's front-run, and `ecrecover` returns a non-zero address identical to the original signer.

## Impact Explanation
The victim's `setIsAuthorized` transaction is griefed: it reverts with `InvalidNonce()` and the victim wastes gas. Because the `authorization` struct is fixed by the victim's signature, the attacker cannot alter the authorization's content — the same authorization is applied. The concrete harms are: (1) victim's tx reverts and gas is lost; (2) the victim loses control over the timing and submission of their own signed message. Note: the claim that subsequent pre-signed nonces (N+1, N+2…) become invalid is incorrect — after the attacker consumes nonce N, the on-chain nonce advances to N+1, so the victim's pre-signed nonce N+1 message remains valid.

## Likelihood Explanation
Preconditions are minimal: the attacker only needs to monitor the public mempool and perform one modular subtraction (`s' = n - s`) and one subtraction (`v' = 55 - v`). No special role, capital, or privileged access is required. The attack is repeatable for every `setIsAuthorized` submission and costs only the gas of a single transaction. On any chain without private mempools, this is trivially executable.

## Recommendation
Add a low-`s` normalization check at the top of `setIsAuthorized`:

```solidity
uint256 constant SECP256K1_ORDER_HALF =
    0x7FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF5D576E7357A4501DDFE92F46681B20A0;

require(uint256(signature.s) <= SECP256K1_ORDER_HALF, InvalidSignature());
```

This eliminates the malleable form by rejecting any signature with `s` in the upper half of the curve order, matching the behavior of OpenZeppelin's `ECDSA.recover` and EIP-2 canonical signature enforcement.

## Proof of Concept
```solidity
// Foundry fork test sketch
function testMalleabilityFrontRun() public {
    // 1. Victim signs authorization with nonce 0
    Authorization memory auth = Authorization({
        authorizer: victim, authorized: someAddr,
        isAuthorized: true, nonce: 0, deadline: type(uint256).max
    });
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(victimKey, digest(auth));

    // 2. Attacker computes malleable form
    uint256 N = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;
    Signature memory malSig = Signature({v: 55 - v, r: r, s: bytes32(N - uint256(s))});

    // 3. Attacker front-runs
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(auth, malSig); // succeeds, nonce → 1

    // 4. Victim's original tx reverts
    vm.prank(victim);
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(auth, Signature({v: v, r: r, s: s}));
}
```
