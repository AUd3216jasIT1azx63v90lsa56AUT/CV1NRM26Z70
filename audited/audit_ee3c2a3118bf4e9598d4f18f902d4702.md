### Title
Fee-on-Transfer Collateral Token Causes Permanent Accounting/Balance Divergence and DoS on `withdrawCollateral` - (File: src/Midnight.sol)

### Summary

`supplyCollateral` records the caller-supplied `assets` value directly into `position[id][onBehalf].collateral[collateralIndex]` before calling `safeTransferFrom`, but with a fee-on-transfer collateral token the contract only receives `assets * (1 - fee_rate)` tokens. `withdrawCollateral` later calls `safeTransfer(collateralToken, receiver, assets)` using the inflated accounting value, which exceeds the actual contract balance and causes a permanent revert. The fee-shortfall amount is permanently frozen in accounting but is unrecoverable.

### Finding Description

**Exact code path:**

`supplyCollateral` at [1](#0-0)  writes the full caller-supplied `assets` into storage at line 533, then pulls tokens at line 545:

```
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets); // line 533
...
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets); // line 545
```

With a fee-on-transfer token, `safeTransferFrom` succeeds but only `assets * (1 - fee_rate)` tokens arrive. The accounting diverges from the actual balance by `assets * fee_rate`.

`withdrawCollateral` at [2](#0-1)  decrements accounting by the full `assets` and then calls:

```
SafeTransferLib.safeTransfer(collateralToken, receiver, assets); // line 572
```

`SafeTransferLib.safeTransfer` at [3](#0-2)  propagates the token's revert directly if the contract balance is insufficient.

**Why no existing check stops it:**

`touchMarket` at [4](#0-3)  validates only maturity, collateral count, sort order, allowed LLTV tiers, and valid `maxLif`. There is no check that the collateral token is not a fee-on-transfer token. Any unprivileged actor (market creator is explicitly listed as unprivileged) can create a market with a fee-on-transfer collateral token.

The `isHealthy` check at line 568 is irrelevant for a borrower with no debt (it returns `true` without calling the oracle per the comment at line 548), so it does not block the path to the failing `safeTransfer`.

**Exploit flow:**

1. Deploy `FeeCollateral` ERC20 with a 0.5% transfer fee.
2. Create a Midnight market with `FeeCollateral` as the collateral token (any unprivileged actor can do this via `touchMarket`).
3. Borrower calls `supplyCollateral(market, 0, 1000, borrower)`:
   - Accounting: `position[id][borrower].collateral[0] = 1000`
   - Actual tokens received by Midnight: `995`
4. Borrower calls `withdrawCollateral(market, 0, 1000, borrower, borrower)`:
   - Accounting update succeeds: `position[id][borrower].collateral[0] = 0`
   - `isHealthy` passes (no debt)
   - `safeTransfer(FeeCollateral, borrower, 1000)` → **REVERTS** (contract holds only 995)
5. The 5-token shortfall is permanently frozen: the borrower can withdraw at most 995, but the accounting records 1000. After withdrawing 995, the remaining 5 in accounting can never be recovered (contract holds 0 tokens for this collateral).

### Impact Explanation

The borrower's collateral accounting permanently exceeds the actual token balance attributable to that market. Calling `withdrawCollateral` with the full recorded amount always reverts. The fee-shortfall portion (`assets * fee_rate`) is permanently frozen in `position.collateral[i]` with no recovery path: the contract holds zero tokens for that collateral after a partial withdrawal, yet the accounting still shows a nonzero balance. This violates the core invariant that contract balances cover collateral.

### Likelihood Explanation

Any unprivileged market creator can deploy a market with a fee-on-transfer collateral token. The precondition is simply that such a token is used as collateral — no privileged action, no oracle manipulation, no user mistake required. The effect is deterministic and repeatable on every `supplyCollateral` call. Fee-on-transfer tokens (e.g., tokens with deflationary mechanics or protocol fees) exist on mainnet.

### Recommendation

In `supplyCollateral`, measure the actual received amount using a balance-delta pattern instead of trusting the caller-supplied `assets`:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

This ensures `position.collateral[i]` always reflects the actual tokens held, eliminating the divergence.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {IMidnight, Market, CollateralParams} from "src/interfaces/IMidnight.sol";

// Fee-on-transfer token: deducts 0.5% on every transfer/transferFrom
contract FeeCollateral {
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;
    uint256 constant FEE_BPS = 50; // 0.5%

    function mint(address to, uint256 amount) external { balanceOf[to] += amount; }
    function approve(address spender, uint256 amount) external returns (bool) {
        allowance[msg.sender][spender] = amount; return true;
    }
    function transfer(address to, uint256 amount) external returns (bool) {
        uint256 fee = amount * FEE_BPS / 10000;
        balanceOf[msg.sender] -= amount;
        balanceOf[to] += amount - fee;
        return true;
    }
    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        allowance[from][msg.sender] -= amount;
        uint256 fee = amount * FEE_BPS / 10000;
        balanceOf[from] -= amount;
        balanceOf[to] += amount - fee;
        return true;
    }
}

contract FeeOnTransferCollateralDoSTest is Test {
    Midnight midnight;
    FeeCollateral feeToken;
    address borrower = address(0xB0B);

    function setUp() public {
        midnight = new Midnight(address(this), address(this), address(this), address(this));
        feeToken = new FeeCollateral();
    }

    function testFeeOnTransferCollateralDoS() public {
        // Build market with fee-on-transfer collateral
        CollateralParams[] memory cp = new CollateralParams[](1);
        cp[0] = CollateralParams({
            token: address(feeToken),
            lltv: 0.77e18,
            maxLif: maxLif(0.77e18, 0.25e18),
            oracle: address(new DummyOracle()),
        });
        Market memory market;
        market.loanToken = address(new DummyLoanToken());
        market.maturity = block.timestamp + 100;
        market.collateralParams = cp;
        midnight.touchMarket(market);

        // Fund borrower and approve
        feeToken.mint(borrower, 1000);
        vm.prank(borrower);
        feeToken.approve(address(midnight), 1000);

        // supplyCollateral: accounting records 1000, contract receives 995
        vm.prank(borrower);
        midnight.supplyCollateral(market, 0, 1000, borrower);

        bytes32 id = midnight.toId(market);
        assertEq(midnight.collateral(id, borrower, 0), 1000, "accounting shows 1000");
        assertEq(feeToken.balanceOf(address(midnight)), 995, "contract holds only 995");

        // withdrawCollateral(1000) must revert: contract cannot send 1000 when it holds 995
        vm.prank(borrower);
        vm.expectRevert(); // safeTransfer reverts — insufficient balance
        midnight.withdrawCollateral(market, 0, 1000, borrower, borrower);

        // Partial withdrawal of 995 succeeds, but 5 tokens remain frozen in accounting
        vm.prank(borrower);
        midnight.withdrawCollateral(market, 0, 995, borrower, borrower);
        assertEq(midnight.collateral(id, borrower, 0), 5, "5 tokens frozen in accounting");
        assertEq(feeToken.balanceOf(address(midnight)), 0, "contract holds 0 tokens");

        // The remaining 5 in accounting can never be withdrawn
        vm.prank(borrower);
        vm.expectRevert(); // safeTransfer(token, borrower, 5) reverts — contract has 0
        midnight.withdrawCollateral(market, 0, 5, borrower, borrower);
    }
}
```

**Expected assertions:**
- `midnight.collateral(id, borrower, 0) == 1000` after supply (accounting inflated)
- `feeToken.balanceOf(address(midnight)) == 995` after supply (actual balance)
- `withdrawCollateral(market, 0, 1000, ...)` reverts
- After partial withdrawal of 995: `midnight.collateral(id, borrower, 0) == 5` and `feeToken.balanceOf(address(midnight)) == 0`
- `withdrawCollateral(market, 0, 5, ...)` reverts — permanent freeze confirmed

    function safeTransfer(address token, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transfer, (to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferReturnedFalse());
    }
