### Title
Missing collateral capacity guard in `repaidUnits` path causes arithmetic underflow in `liquidate` - (File: `src/Midnight.sol`)

### Summary
When `repaidUnits > 0` is passed to `liquidate`, `seizedAssets` is computed at line 652 as `repaidUnits.mulDivDown(lif, WAD).mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice)` with no subsequent check that the result does not exceed `_position.collateral[collateralIndex]`. At very low oracle prices, this value can far exceed available collateral, causing an arithmetic underflow or `CastOverflow` at line 670. The RCF check does not prevent this: when the liquidated collateral price is near zero, the threshold condition `(collateral_capacity - maxRepaid) < rcfThreshold` reduces to `0 < rcfThreshold`, which is trivially satisfied for any positive `rcfThreshold`, allowing arbitrarily large `repaidUnits` through.

### Finding Description
**Code path:** [1](#0-0) 

```solidity

    repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
} else {
    seizedAssets = repaidUnits.mulDivDown(lif, WAD).mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice);
}
``` [2](#0-1) 

The RCF check that follows:
```solidity
require(
    repaidUnits <= maxRepaid
        || _position.collateral[collateralIndex].mulDivDown(liquidatedCollatPrice, ORACLE_PRICE_SCALE)
            .mulDivDown(WAD, lif).zeroFloorSub(maxRepaid) < market.rcfThreshold,
    RecoveryCloseFactorConditionsViolated()
);
```
is designed to limit over-liquidation relative to debt recovery, **not** to enforce that `seizedAssets <= collateral[collateralIndex]`. When `liquidatedCollatPrice` is very low, `collateral_capacity = collateral * price / ORACLE_PRICE_SCALE * WAD / lif ≈ 0`, so `zeroFloorSub(maxRepaid) = 0`, and `0 < rcfThreshold` is always true for any positive `rcfThreshold`. The check passes unconditionally, allowing `repaidUnits` up to `_position.debt`.

Then at line 670:
```solidity
uint128 newCollateral = _position.collateral[collateralIndex] - UtilsLib.toUint128(seizedAssets);
```

`UtilsLib.toUint128` reverts with `CastOverflow` if `seizedAssets > type(uint128).max`, or the subtraction reverts with an arithmetic underflow if `seizedAssets > collateral[collateralIndex]`.

**Concrete exploit — single-collateral with bad debt (non-post-maturity):**

- `collateral = 1`, `price = 1`, `ORACLE_PRICE_SCALE = 1e36`, `maxLif = 1.1e18`, `WAD = 1e18`
- Bad debt loop: `badDebt = debt.zeroFloorSub(collateral.mulDivUp(1, 1e36).mulDivUp(WAD, maxLif))` → `_position.debt` reduced to `ceil(1/1.1e36) = 1`
- RCF check: `maxRepaid = 1 / 0.12 ≈ 8.33`; `repaidUnits = 1 ≤ 8.33` → passes
- `seizedAssets = 1 * 1.1e18 / 1e18 * 1e36 / 1 = 1.1e36`
- `1 - 1.1e36` → **arithmetic underflow revert**

**Concrete exploit — multi-collateral, RCF bypass (non-post-maturity, `rcfThreshold > 0`):**

- Collateral A (liquidated): `collateral_A = 100`, `price_A = 1`, `lltv = 0.8e18`, `maxLif = 1.1e18`
- Collateral B: `collateral_B = 1000`, `price_B = ORACLE_PRICE_SCALE`, `lltv = 0.8e18`
- `debt = 900`, `maxDebt = 800` → unhealthy, no bad debt (B covers all)
- `maxRepaid = 100 / 0.12 ≈ 833`; `repaidUnits = 900 > 833` → first condition fails
- `collateral_capacity = 100 * 1 / 1e36 * WAD / 1.1e18 ≈ 0`; `0 < rcfThreshold` → second condition passes
- `seizedAssets = 900 * 1.1e18 / 1e18 * 1e36 / 1 = 9.9e38 > type(uint128).max` → **`CastOverflow` revert**

The test suite confirms awareness of this gap: `testLiquidateUnitsInput` bounds `liquidationOraclePrice >= fullRepaymentPrice(units)` to avoid it, and `testLiquidateWithBadDebtRepaidInput` explicitly caps `repaid` to `maxRepaidFromCollat = collateral * price / ORACLE_PRICE_SCALE * WAD / lif` — a guard that does not exist in the contract. ### Impact Explanation
A liquidator calling `liquidate` with `repaidUnits > 0` on a position whose liquidated collateral oracle returns a very low price will receive an arithmetic underflow or `CastOverflow` revert instead of a successful liquidation or a descriptive error. This violates the invariant that valid liquidations (passing a `repaidUnits` within the borrower's debt) must not revert unexpectedly. In the multi-collateral case, the RCF bypass makes this reachable even without bad debt, for any market with `rcfThreshold > 0`.

### Likelihood Explanation
Requires: (1) a market with a collateral whose oracle price has dropped to near zero, (2) a liquidator using the `repaidUnits` input path with a value larger than `collateral * price / ORACLE_PRICE_SCALE * WAD / lif`. In the multi-collateral case, any positive `rcfThreshold` (the default in all test markets) enables the RCF bypass. Oracle price crashes are a normal liquidation trigger, making this a realistic scenario. The liquidator can work around it by switching to the `seizedAssets` input, but the unexpected revert is a protocol-level invariant violation.

### Recommendation
After computing `seizedAssets` at line 652, add an explicit cap or revert before the subtraction:

```solidity
require(seizedAssets <= _position.collateral[collateralIndex], InsufficientCollateral());
```

Or equivalently, cap `seizedAssets` at `_position.collateral[collateralIndex]` and recompute `repaidUnits` accordingly. This mirrors the test-side guard already used in `testLiquidateWithBadDebtRepaidInput` (`maxRepaidFromCollat`) and the `fullRepaymentPrice` helper.

### Proof of Concept
```solidity
function testFuzz_repaidUnitsUnderflow(uint256 repaidUnits, uint256 price) public {
    // Single-collateral market, price near zero
    price = bound(price, 1, 1e10); // very low price
    uint256 units = 1e18;
    collateralize(market, borrower, units);
    setupMarket(market, units);
    Oracle(market.collateralParams[0].oracle).setPrice(price);

    uint256 C = midnight.collateral(id, borrower, 0);
    uint256 debtAfterBadDebt = units - _badDebt();
    repaidUnits = bound(repaidUnits, 1, debtAfterBadDebt);

    // Assert: seizedAssets must not exceed C
    uint256 lif = market.collateralParams[0].maxLif;
    uint256 seizedAssets = repaidUnits * lif / WAD * ORACLE_PRICE_SCALE / price;
    // This assertion FAILS: seizedAssets >> C when price is near zero
    assertLe(seizedAssets, C, "seizedAssets must not exceed collateral");

    // The actual call reverts with arithmetic underflow or CastOverflow
    vm.expectRevert(); // arithmetic underflow or CastOverflow
    midnight.liquidate(market, 0, 0, repaidUnits, borrower, false, address(this), address(0), "");
}
```

Expected assertion: `seizedAssets <= collateral[collateralIndex]` for all `repaidUnits` in `[0, debtAfterBadDebt]` and all valid oracle prices. The fuzz test will find counterexamples at low prices, reproducing the underflow.
