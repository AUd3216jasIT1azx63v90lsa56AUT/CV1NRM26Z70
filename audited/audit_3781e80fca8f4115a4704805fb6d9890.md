### Title
Frontrunnable Factory Initializer Grants Attacker Full Factory Ownership — (`File: src/ConfidencePoolFactory.sol`)

### Summary
`ConfidencePoolFactory.initialize()` is `external` with no access control beyond the `initializer` modifier. In the standard UUPS deployment pattern, the proxy is deployed first and `initialize()` is called in a separate transaction. An attacker monitoring the mempool can frontrun that call, supply their own parameters, and become the factory owner — gaining control over `defaultOutcomeModerator`, `poolImplementation`, the stake-token allowlist, and UUPS upgrade authority.

### Finding Description
`ConfidencePoolFactory.initialize()` is declared `external initializer`:

```solidity
function initialize(address safeHarborRegistry_, address poolImplementation_, address defaultOutcomeModerator_)
    external
    initializer
{
    ...
    __Ownable_init(msg.sender);   // msg.sender becomes owner
    ...
    poolImplementation = poolImplementation_;
    defaultOutcomeModerator = defaultOutcomeModerator_;
}
``` [1](#0-0) 

The `initializer` modifier only prevents a second call; it imposes no restriction on *who* makes the first call. The implementation contract correctly calls `_disableInitializers()` in its constructor, but the proxy contract's storage is uninitialized until `initialize()` is called on it. If the deployer sends the proxy deployment and the `initialize()` call as two separate transactions (the common pattern when using deployment scripts or multisigs), an attacker can observe the uninitialized proxy in the mempool and submit `initialize()` with attacker-controlled arguments at a higher gas price. [2](#0-1) 

### Impact Explanation
A successful frontrun gives the attacker full factory ownership. As owner the attacker can:

1. **`setPoolImplementation(maliciousImpl)`** — all pools created afterward use the malicious implementation, which can contain arbitrary backdoors to drain staker funds.
2. **`setDefaultOutcomeModerator(attacker)`** — every new pool's `outcomeModerator` is set to the attacker at clone time and is immutable per-pool thereafter. The moderator controls `flagOutcome`, which determines whether stakers recover principal or it is swept to `recoveryAddress`.
3. **`_authorizeUpgrade`** — the attacker can upgrade the factory proxy itself to an arbitrary implementation.
4. **`setStakeTokenAllowed` / `pause`** — attacker can block legitimate pool creation or allow malicious tokens. [3](#0-2) 

The `outcomeModerator` is set once at `initialize()` time and has no per-pool setter, so pools created under the attacker-controlled factory are permanently compromised. [4](#0-3) 

### Likelihood Explanation
Any actor monitoring the mempool can execute this attack. The window exists whenever the proxy deployment and the `initialize()` call are not bundled into a single atomic transaction (e.g., separate deployment script steps, a multisig proposal, or a manual call). This is a well-known attack class for UUPS proxies and requires no special privileges or capital.

### Recommendation
1. **Atomic initialization**: deploy the proxy using `new ERC1967Proxy(implementation, abi.encodeCall(ConfidencePoolFactory.initialize, (...)))` so the proxy is initialized in its own constructor, leaving no uninitialized window.
2. **Alternatively**, add an `onlyOwner`-equivalent guard by storing the intended initializer address in the implementation's immutable storage and checking `msg.sender` inside `initialize()`.

### Proof of Concept
```
1. Deployer broadcasts: deploy ERC1967Proxy(factoryImpl, "") — no init data.
2. Attacker sees the uninitialized proxy in the mempool.
3. Attacker calls proxy.initialize(
       safeHarborRegistry_  = <legitimate registry>,
       poolImplementation_  = <malicious pool impl>,
       defaultOutcomeModerator_ = attacker
   ) with higher gas.
4. Attacker's tx mines first; attacker is now factory owner.
5. Deployer's subsequent initialize() call reverts (already initialized).
6. All pools created via the factory use the malicious implementation
   and have the attacker as their immutable moderator.
```

    function initialize(address safeHarborRegistry_, address poolImplementation_, address defaultOutcomeModerator_)
        external
        initializer
    {
        if (safeHarborRegistry_ == address(0)) revert ZeroAddress();
        if (poolImplementation_ == address(0)) revert ZeroAddress();
        if (defaultOutcomeModerator_ == address(0)) revert ZeroAddress();

        __Ownable_init(msg.sender);
        __Pausable_init();

        safeHarborRegistry = IBattleChainSafeHarborRegistry(safeHarborRegistry_);
        poolImplementation = poolImplementation_;
        defaultOutcomeModerator = defaultOutcomeModerator_;
    }
