### Title
`targetUnits=0` allows collateral slot poisoning on victim's position without any borrow — (`File: src/periphery/MidnightBundles.sol`)

### Summary
`supplyCollateralAndSellWithUnitsTarget` executes the collateral supply loop unconditionally before the take loop. When `targetUnits=0`, the take loop's entry condition `filledUnits < targetUnits` (i.e., `0 < 0`) is immediately false, so no takes execute, yet all `collateralSupplies` entries have already been applied to the victim's position. The post-loop check `require(filledUnits == targetUnits)` trivially passes as `0 == 0`, completing the call with collateral supplied but zero debt incurred.

### Finding Description

**Code path:**

1. **Line 127** — authorization check passes: attacker (`msg.sender`) is authorized by victim (`taker`) on Midnight.
2. **Lines 134–140** — collateral supply loop runs unconditionally for every entry in `collateralSupplies`. `pullToken(token, msg.sender, ...)` pulls from the attacker's own balance; `IMidnight(MIDNIGHT).supplyCollateral(market, ..., taker)` credits the collateral to the victim's position. The bundler is authorized to call this because the victim has authorized the bundler on Midnight (standard bundler setup). Each new collateral index with `assets > 0` sets a bit in `victim.collateralBitmap` and increments the active-collateral count.
3. **Line 144** — take loop: `for (... && filledUnits < targetUnits)` → `0 < 0` is false; loop body never entered.
4. **Line 163** — `require(0 == 0)` passes.
5. **Line 166** — `require(0 - 0 >= 0)` passes (attacker sets `minSellerAssets = 0`).
6. **Line 168** — `safeTransfer(loanToken, receiver, 0)` — zero transfer, no revert on standard ERC20.

**Root cause:** No guard prevents `targetUnits=0` when `collateralSupplies` is non-empty. The collateral supply and take loops are sequenced such that collateral is committed before the take loop runs, and the take loop is skippable at zero cost.

**Attacker-controlled inputs:**
- `targetUnits = 0`
- `minSellerAssets = 0`
- `collateralSupplies` = up to 16 entries covering distinct collateral indices not yet active on victim's position
- `takes` = any single valid-looking entry (needed to avoid `takes[0]` OOB revert at line 129; the loop body never executes it)
- `taker` = victim address

**Preconditions:**
- Victim has authorized the bundler on Midnight (required for bundler to call `supplyCollateral` on victim's behalf — standard bundler usage)
- Victim has authorized the attacker on Midnight (passes line 127 check)
- Attacker holds the collateral tokens and has approved the bundler to pull them

### Impact Explanation

The attacker can activate up to `MAX_COLLATERALS_PER_BORROWER` (16) collateral slots on the victim's position in a single call. If the victim already has `k` active slots, the attacker fills the remaining `16 - k` slots with dust amounts. Any subsequent `supplyCollateral` call by the victim for a new collateral type reverts with `TooManyActivatedCollaterals`. This is a targeted, repeatable DoS: even if the victim withdraws the dust to clear slots, the attacker can re-poison them. The attacker bears the cost of the dust collateral tokens but can use arbitrarily small amounts.

### Likelihood Explanation

**Preconditions are realistic:** Any user who delegates to a keeper or uses a smart-account pattern will have authorized both the bundler and a third-party operator on Midnight. The attacker only needs to be that authorized operator. The collateral cost is dust (1 wei per slot). The attack is repeatable every time the victim clears a slot.

### Recommendation

Add a guard at the start of the function (or before the collateral supply loop) that rejects a non-empty `collateralSupplies` when `targetUnits == 0`:

```solidity
require(targetUnits > 0 || collateralSupplies.length == 0, InvalidInput());
```

Alternatively, enforce `targetUnits > 0` unconditionally, since a zero-borrow sell bundle is semantically meaningless.

    function supplyCollateralAndSellWithUnitsTarget(
        uint256 targetUnits,
        uint256 minSellerAssets,
        address taker,
        address receiver,
        CollateralSupply[] memory collateralSupplies,
        Take[] memory takes,
        uint256 referralFeePct,
        address referralFeeRecipient
    ) external {
        require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
        require(referralFeePct < WAD, PctExceeded());
        address loanToken = takes[0].offer.market.loanToken;
        // touchMarket to have the correct settlement fees.
        bytes32 id = IMidnight(MIDNIGHT).touchMarket(takes[0].offer.market);

        Market memory market = takes[0].offer.market;
        for (uint256 i; i < collateralSupplies.length; i++) {
            address token = market.collateralParams[collateralSupplies[i].collateralIndex].token;
            pullToken(token, msg.sender, collateralSupplies[i].assets, collateralSupplies[i].permit);
            forceApproveMax(token, MIDNIGHT);
            IMidnight(MIDNIGHT)
                .supplyCollateral(market, collateralSupplies[i].collateralIndex, collateralSupplies[i].assets, taker);
        }

        uint256 filledUnits;
        uint256 filledSellerAssets;
        for (uint256 i; i < takes.length && filledUnits < targetUnits; i++) {
            require(takes[i].offer.buy, InconsistentSide());
            require(IMidnight(MIDNIGHT).toId(takes[i].offer.market) == id, InconsistentMarket());
            uint256 unitsToTake = min(
                targetUnits - filledUnits,
                takes[i].units,
                ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
            );
            try IMidnight(MIDNIGHT)
                .take(
                    takes[i].offer, takes[i].ratifierData, unitsToTake, taker, address(this), address(0), ""
                ) returns (
                uint256, uint256 resSellerAssets
            ) {
                filledUnits += unitsToTake;
                filledSellerAssets += resSellerAssets;
            } catch {}
        }

        require(filledUnits == targetUnits, OutOfOffers());

        uint256 referralFeeAssets = filledSellerAssets.mulDivDown(referralFeePct, WAD);
        require(filledSellerAssets - referralFeeAssets >= minSellerAssets, SellerAssetsTooLow());
        if (referralFeeAssets > 0) SafeTransferLib.safeTransfer(loanToken, referralFeeRecipient, referralFeeAssets);
        SafeTransferLib.safeTransfer(loanToken, receiver, filledSellerAssets - referralFeeAssets);
    }
