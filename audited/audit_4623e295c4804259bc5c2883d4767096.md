Audit Report

## Title
Front-running `buyWithAssetsTargetAndWithdrawCollateral` via partial offer fill causes DoS through exact equality check - (File: src/periphery/MidnightBundles.sol)

## Summary
`buyWithAssetsTargetAndWithdrawCollateral` and `supplyCollateralAndSellWithAssetsTarget` compute `unitsToTake` using a live read of `consumed[maker][group]` via `ConsumableUnitsLib.consumableUnits`. Any unprivileged taker can front-run the victim's bundle by partially filling the same offer, reducing the available capacity so the bundle fills fewer assets than `targetFilledBuyerAssets`/`targetFilledSellerAssets`, causing the post-loop exact equality check to revert. The attacker receives a legitimate fill on each front-run, making the attack economically neutral or profitable.

## Finding Description

**Root cause — live state read combined with exact equality check and no slippage tolerance.**

`ConsumableUnitsLib.consumableUnits` reads the live `consumed[maker][group]` value at call time: [1](#0-0) 

For a sell offer (`buy == false`) with `maxAssets > 0`, it returns `sellerAssetsToUnits(midnight, id, offer, offer.maxAssets - consumed)`. Any increase in `consumed` before this call reduces the returned value.

In `Midnight.take`, every successful take of an offer with `maxAssets > 0` increments the shared `consumed[offer.maker][offer.group]` mapping: [2](#0-1) 

This mapping is global — any taker of the same offer group increments it.

Inside `buyWithAssetsTargetAndWithdrawCollateral`, `unitsToTake` is bounded by `consumableUnits`: [3](#0-2) 

After the loop, an exact equality is enforced: [4](#0-3) 

The same pattern exists in `supplyCollateralAndSellWithAssetsTarget`: [5](#0-4) 

**Exploit flow:**

1. Maker posts a sell offer with `maxAssets = M`, `maxUnits = 0`.
2. Victim constructs a bundle with `targetBuyerAssets = X`, `takes = [offer]`, requiring `N = buyerAssetsToUnits(X)` units of capacity.
3. Victim broadcasts the bundle transaction.
4. Attacker front-runs with a direct `Midnight.take(offer, ..., Y_units, ...)` where `Y_units >= 1`. This increments `consumed[maker][group]` by `sellerAssets(Y_units)`.
5. Victim's bundle executes. `consumableUnits` now returns `sellerAssetsToUnits(M - consumed_after_attack)`, which is less than `N`.
6. `unitsToTake = min(N, takes[0].units, N - delta) = N - delta` for some `delta > 0`.
7. The take succeeds but fills fewer buyer assets: `filledBuyerAssets = X - epsilon < targetFilledBuyerAssets`.
8. `require(filledBuyerAssets == targetFilledBuyerAssets, OutOfOffers())` reverts.

**Why existing checks do not stop it:**

The `try/catch` around the inner `take` call only catches reverts from `take`. Here, `take` succeeds — it fills `N - delta` units without reverting. The revert originates from the post-loop equality check in the bundler, not from `take` itself: [6](#0-5) 

There is no slippage tolerance or minimum-fill parameter on the assets-target variants. `consumableUnits` has no snapshot or reservation mechanism. The attacker requires no special authorization — `Midnight.take` is permissionless for any taker: [7](#0-6) 

## Impact Explanation

Any victim using `buyWithAssetsTargetAndWithdrawCollateral` or `supplyCollateralAndSellWithAssetsTarget` targeting an offer with `maxAssets > 0` can be griefed indefinitely. The attacker front-runs each retry with a minimal partial fill (even 1 unit), causing every bundle attempt to revert with `OutOfOffers`. The victim cannot complete their intended trade through the bundler as long as the attacker continues to front-run. This constitutes a targeted, sustained denial of service against a specific user's use of the periphery contract.

## Likelihood Explanation

**Preconditions:**
- Victim uses `buyWithAssetsTargetAndWithdrawCollateral` or `supplyCollateralAndSellWithAssetsTarget` targeting an offer with `maxAssets > 0`.
- Attacker is any address that can call `Midnight.take` on the same offer (no special role required).
- Attacker can observe the mempool (standard on all non-private-mempool chains).

**Feasibility:** High. The attacker submits a `take` with 1 unit before the victim's bundle lands. The attack is repeatable on every retry. The attacker's only cost is gas; they receive a legitimate fill in return, making the attack economically neutral or profitable.

## Recommendation

Replace the exact equality check with a `>=` check and add a refund of any unfilled `targetFilledBuyerAssets - filledBuyerAssets` back to `msg.sender`. Introduce a `minFilledBuyerAssets` parameter (slippage tolerance) so the victim can specify an acceptable lower bound rather than requiring an exact fill. The same fix applies symmetrically to `supplyCollateralAndSellWithAssetsTarget` with a `minFilledSellerAssets` parameter. This mirrors the design of `buyWithUnitsTargetAndWithdrawCollateral`, which already refunds unused tokens: [8](#0-7) 

## Proof of Concept

**Minimal fork test plan:**

1. Deploy Midnight and MidnightBundles on a local fork.
2. Maker creates a sell offer with `maxAssets = 1000`, `maxUnits = 0`, at a known tick.
3. Victim calls `buyWithAssetsTargetAndWithdrawCollateral` with `targetBuyerAssets` corresponding to exactly 100 units of capacity.
4. Before the victim's transaction lands, attacker calls `Midnight.take(offer, ..., 1, ...)` directly, incrementing `consumed[maker][group]` by `sellerAssets(1)`.
5. Victim's bundle transaction executes. `consumableUnits` returns capacity for 99 units. `unitsToTake = 99`. `filledBuyerAssets` corresponds to 99 units, which is less than `targetFilledBuyerAssets`.
6. Observe revert with `OutOfOffers()` at line 224 of `MidnightBundles.sol`.
7. Repeat steps 4–6 on every retry to confirm indefinite DoS. [9](#0-8)

    function buyWithAssetsTargetAndWithdrawCollateral(
        uint256 targetBuyerAssets,
        uint256 minUnits,
        address taker,
        TokenPermit memory loanTokenPermit,
        Take[] memory takes,
        CollateralWithdrawal[] memory collateralWithdrawals,
        address collateralReceiver,
        uint256 referralFeePct,
        address referralFeeRecipient
    ) external {
        require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
        require(referralFeePct < WAD, PctExceeded());
        address loanToken = takes[0].offer.market.loanToken;
        // touchMarket to have the correct settlement fees.
        bytes32 id = IMidnight(MIDNIGHT).touchMarket(takes[0].offer.market);

        pullToken(loanToken, msg.sender, targetBuyerAssets, loanTokenPermit);
        forceApproveMax(loanToken, MIDNIGHT);

        uint256 referralFeeAssets = targetBuyerAssets.mulDivDown(referralFeePct, WAD);
        uint256 targetFilledBuyerAssets = targetBuyerAssets - referralFeeAssets;

        uint256 filledUnits;
        uint256 filledBuyerAssets;
        for (uint256 i; i < takes.length && filledBuyerAssets < targetFilledBuyerAssets; i++) {
            require(!takes[i].offer.buy, InconsistentSide());
            require(IMidnight(MIDNIGHT).toId(takes[i].offer.market) == id, InconsistentMarket());
            uint256 unitsToTake = min(
                TakeAmountsLib.buyerAssetsToUnits(
                    MIDNIGHT, id, takes[i].offer, targetFilledBuyerAssets - filledBuyerAssets
                ),
                takes[i].units,
                ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
            );
            try IMidnight(MIDNIGHT)
                .take(takes[i].offer, takes[i].ratifierData, unitsToTake, taker, address(0), address(0), "") returns (
                uint256 resBuyerAssets, uint256
            ) {
                filledUnits += unitsToTake;
                filledBuyerAssets += resBuyerAssets;
            } catch {}
        }

        require(filledBuyerAssets == targetFilledBuyerAssets, OutOfOffers());
        require(filledUnits >= minUnits, UnitsTooLow());

        Market memory market = takes[0].offer.market;
        for (uint256 i; i < collateralWithdrawals.length; i++) {
            IMidnight(MIDNIGHT)
                .withdrawCollateral(
                    market,
                    collateralWithdrawals[i].collateralIndex,
                    collateralWithdrawals[i].assets,
                    taker,
                    collateralReceiver
                );
        }

        if (referralFeeAssets > 0) SafeTransferLib.safeTransfer(loanToken, referralFeeRecipient, referralFeeAssets);
    }
