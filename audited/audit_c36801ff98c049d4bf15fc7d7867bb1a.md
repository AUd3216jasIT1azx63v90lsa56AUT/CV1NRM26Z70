### Title
Malicious-ratifier gas griefing via unbounded `takes[]` loop and silent `try/catch` in `MidnightBundles` - (`src/periphery/MidnightBundles.sol`)

### Summary
Every bundler function in `MidnightBundles` iterates over a caller-supplied, unbounded `takes[]` array and wraps each `Midnight.take()` call in a bare `try {} catch {}`. Because `Midnight.take()` forwards all available gas to the maker-controlled `IRatifier.isRatified()` call with no gas cap, a malicious maker can deploy a ratifier that burns ≈63/64 of the forwarded gas then reverts. A victim taker who includes even two such offers in their `takes[]` array exhausts their entire gas budget with zero fills.

### Finding Description

**Code path:**

`MidnightBundles.buyWithUnitsTargetAndWithdrawCollateral` (and the three sibling functions) at lines 71–86:

```solidity
for (uint256 i; i < takes.length && filledUnits < targetUnits; i++) {
    ...
    try IMidnight(MIDNIGHT)
        .take(takes[i].offer, takes[i].ratifierData, unitsToTake, taker, ...) returns (...) {
        filledUnits += unitsToTake;
        filledBuyerAssets += resBuyerAssets;
    } catch {}   // ← silent, no gas check
}
```

Inside `Midnight.take()` at line 356:

```solidity
require(IRatifier(offer.ratifier).isRatified(offer, ratifierData) == CALLBACK_SUCCESS, RatifierFail());
```

This is a plain external call with **no gas stipend**. The EVM forwards 63/64 of the caller's remaining gas to `isRatified()`.

**Root cause:** Three missing controls acting together:
1. No gas limit on the `isRatified()` call in `Midnight.take()`.
2. `try/catch {}` in the bundler silently swallows any revert, including one that consumed 63/64 of the gas.
3. No cap on `takes.length`.

**Attacker inputs (all permissionless):**

1. Attacker deploys `MaliciousRatifier` whose `isRatified()` spins in a loop consuming ≈63/64 of forwarded gas, then reverts.
2. Attacker calls `midnight.setIsAuthorized(address(maliciousRatifier), true, attacker)` — satisfies the `isAuthorized[offer.maker][offer.ratifier]` check at line 355.
3. Attacker creates `N` offers with `offer.ratifier = address(maliciousRatifier)` and distributes them off-chain (the protocol is explicitly off-chain routed).
4. A victim taker's routing software includes these offers in `takes[]` and calls any bundler function.

**Gas cascade per iteration (EIP-150 63/64 rule):**

- Bundler forwards ≈63/64 of its gas to `take()`.
- `take()` forwards ≈63/64 of that to `isRatified()`.
- `isRatified()` burns ≈all of it, then reverts.
- Bundler retains ≈(1/64)² ≈ 0.024% of original gas after iteration 1.
- After iteration 2: ≈(1/64)⁴ ≈ 6×10⁻⁸ % — effectively zero.

The bundler then OOGs on the next external call (`ConsumableUnitsLib.consumableUnits()` or the next `try`), consuming the victim's full gas budget. The final `require(filledUnits == targetUnits, OutOfOffers())` is never reached; the transaction reverts OOG.

**Existing protections reviewed and found insufficient:**

- `isAuthorized[offer.maker][offer.ratifier]` (line 355): The attacker self-authorizes their own malicious ratifier — no barrier.
- `ConsumableUnitsLib.consumableUnits()` (called before `try`): Pure view call, does not touch the ratifier — no barrier.
- `catch {}` (lines 85, 160, 221, 300): Designed to handle asynchrony, but also catches gas-burning reverts — no barrier.
- No `takes.length` cap anywhere in `MidnightBundles`.

### Impact Explanation
Any user of the bundler who includes even two offers from a malicious maker in their `takes[]` array will have their entire transaction gas consumed with zero fills. The victim pays full block-gas-limit cost and receives nothing. The attack is repeatable against any taker using off-chain routing that does not inspect ratifier bytecode.

### Likelihood Explanation
**Preconditions:** Attacker deploys one contract and calls `setIsAuthorized` once — both permissionless. Offers are distributed off-chain by protocol design (`offers_are_not_protocol_broadcasted: true`, `offers_can_be_distributed_offchain: true`). Routing software that aggregates offers from public orderbooks is the expected usage pattern. The attacker can post offers with competitive prices to maximize inclusion probability. The attack is repeatable at zero marginal cost per victim (the same two offers can be reused against any number of takers).

### Recommendation
Apply a fixed gas stipend to the `isRatified()` call in `Midnight.take()`:

```solidity
require(
    IRatifier(offer.ratifier).isRatified{gas: 100_000}(offer, ratifierData) == CALLBACK_SUCCESS,
    RatifierFail()
);
```

Additionally, add a `takes.length` cap in `MidnightBundles` (e.g., `require(takes.length <= 256)`). The gas cap is the primary fix; the length cap is defense-in-depth.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity 0.8.34;

import {IRatifier} from "src/interfaces/IRatifier.sol";
import {Offer} from "src/interfaces/IMidnight.sol";

/// Ratifier that burns ~63/64 of forwarded gas then reverts.
contract GasGriefRatifier is IRatifier {
    function isRatified(Offer memory, bytes memory) external view returns (bytes32) {
        uint256 start = gasleft();
        // Burn gas until ~1/64 remains, then revert.
        while (gasleft() > start / 64) { /* spin */ }
        revert("griefed");
    }
}

contract GasGriefTest is MidnightBundlesTest {
    function testGasGriefViaMaliciousRatifier() public {
        GasGriefRatifier badRatifier = new GasGriefRatifier();

        // Attacker (maker = lender) authorizes malicious ratifier.
        vm.prank(lender);
        midnight.setIsAuthorized(address(badRatifier), true, lender);

        // Build 2 malicious offers.
        Offer memory badOffer = offers[0];
        badOffer.ratifier = address(badRatifier);

        Take[] memory takes = new Take[](2);
        takes[0] = Take({offer: badOffer, units: 1, ratifierData: hex""});
        takes[1] = Take({offer: badOffer, units: 1, ratifierData: hex""});

        uint256 gasBefore = gasleft();
        // Victim (taker = lender) calls bundler with N=2 malicious offers.
        // Expect OOG or OutOfOffers with near-full gas consumed.
        try midnightBundles.buyWithUnitsTargetAndWithdrawCollateral{gas: 3_000_000}(
            2, type(uint256).max, lender, _noPermit(), takes,
            new CollateralWithdrawal[](0), address(0), 0, address(0)
        ) {} catch {}
        uint256 gasUsed = gasBefore - gasleft();

        // Assert: nearly all 3M gas consumed, zero fills.
        assertGt(gasUsed, 2_900_000, "gas griefed");
        assertEq(midnight.debtOf(id, lender), 0, "no fills");
    }
}
```
