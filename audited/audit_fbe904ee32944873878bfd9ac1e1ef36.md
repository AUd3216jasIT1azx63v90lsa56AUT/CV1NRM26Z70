### Title
`sellerAssetsToUnits` reverts on `sellerPrice = 0` while `Midnight.take()` succeeds, causing `supplyCollateralAndSellWithAssetsTarget` to revert for valid buy offers - (`File: src/periphery/TakeAmountsLib.sol`)

### Summary

When a buy offer is placed at tick `T` where `tickToPrice(T) == settlementFee(id, ttm)`, `sellerPrice` computes to exactly `0`. `sellerAssetsToUnits` then calls `mulDivUp(targetSellerAssets, WAD, 0)`, which triggers an arithmetic underflow at `d - 1` (Solidity 0.8.x checked arithmetic), reverting the call. `Midnight.take()` does not revert in this case — it computes `sellerAssets = units.mulDivDown(0, WAD) = 0` and proceeds normally. This divergence causes `MidnightBundles.supplyCollateralAndSellWithAssetsTarget` to revert entirely when any offer in the `takes` array hits this condition, since `sellerAssetsToUnits` is called outside the `try/catch` block.

### Finding Description

**Code path in `TakeAmountsLib.sellerAssetsToUnits`** (`src/periphery/TakeAmountsLib.sol`, lines 41–46):

```solidity
uint256 offerPrice = TickLib.tickToPrice(offer.tick);
uint256 settlementFee =
    IMidnight(midnight).settlementFee(id, UtilsLib.zeroFloorSub(offer.market.maturity, block.timestamp));
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
return
    offer.buy ? targetSellerAssets.mulDivUp(WAD, sellerPrice) : targetSellerAssets.mulDivDown(WAD, sellerPrice);
```

When `offer.buy = true` and `offerPrice == settlementFee`, `sellerPrice = 0`. The call then reaches:

```solidity
// UtilsLib.mulDivUp:
return (x * y + (d - 1)) / d;  // d = 0 → (d - 1) = uint256(0) - 1 → underflow → REVERT
``` [1](#0-0) [2](#0-1) 

**Contrast with `Midnight.take()`** (`src/Midnight.sol`, lines 361–364):

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice;  // = 0, no underflow
uint256 buyerPrice = sellerPrice + _settlementFee;                            // = settlementFee
uint256 buyerAssets = offer.buy ? units.mulDivDown(buyerPrice, WAD) : ...;   // fine
uint256 sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : ...; // = 0, fine
```

`mulDivDown(units, 0, WAD) = (units * 0) / WAD = 0` — no revert. `Midnight.take()` succeeds and yields `sellerAssets = 0`. [3](#0-2) 

**Call site in `MidnightBundles.supplyCollateralAndSellWithAssetsTarget`** (`src/periphery/MidnightBundles.sol`, lines 285–300):

```solidity
uint256 unitsToTake = min(
    TakeAmountsLib.sellerAssetsToUnits(          // ← NOT inside try/catch; reverts here
        MIDNIGHT, id, takes[i].offer, targetFilledSellerAssets - filledSellerAssets
    ),
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
);
try IMidnight(MIDNIGHT).take(...) returns (...) { ... } catch {}  // ← try/catch only wraps take()
```

The `sellerAssetsToUnits` call is outside the `try/catch`. The bundler's own NatSpec at line 246 confirms: *"Reverts if TakeAmountsLib or ConsumableUnitsLib reverts."* So the entire transaction reverts. [4](#0-3) [5](#0-4) 

**Root cause**: The NatSpec on `sellerAssetsToUnits` documents *"Reverts if offerPrice < settlementFee in case of a buy offer (midnight reverts too)."* This is correct for the strict-less-than case (Solidity 0.8.x underflow on `offerPrice - settlementFee`). But the equal case (`offerPrice == settlementFee`) is not handled: `Midnight.take()` accepts `sellerPrice = 0` and returns `sellerAssets = 0`, while `sellerAssetsToUnits` reverts via `mulDivUp`'s `d - 1` underflow. [6](#0-5) 

**Attacker-controlled inputs**: Any unprivileged user (market creator, lender, borrower) can post a buy offer (`offer.buy = true`) at any tick. Both `tickToPrice` and `settlementFee` are multiples of `PRICE_ROUNDING_STEP = CBP = 1e12`. [7](#0-6) [8](#0-7) 

Since `tickToPrice` is monotonically increasing from tick 0 to `MAX_TICK = 5820`, and the maximum settlement fee at 360 days is `0.005e18`, there exist low-tick values whose price (a multiple of 1e12) equals a valid settlement fee (also a multiple of 1e12). An attacker can enumerate ticks offline and post an offer at the matching tick. [9](#0-8) 

### Impact Explanation

Any call to `supplyCollateralAndSellWithAssetsTarget` that includes a buy offer at tick `T` where `tickToPrice(T) == settlementFee(id, ttm)` reverts entirely. Because `sellerAssetsToUnits` is called before the `try/catch`-wrapped `Midnight.take()`, the revert propagates out of the bundler. The taker cannot complete the bundled sell-with-assets-target operation through this offer. If the attacker's offer is the only or first offer in the `takes` array, the entire bundler call fails, preventing the taker from borrowing against their collateral via the bundler.

### Likelihood Explanation

**Preconditions**:
1. A buy offer exists at tick `T` where `tickToPrice(T) == settlementFee(id, ttm)` — achievable by any unprivileged user who enumerates tick prices offline.
2. A taker routes through this offer via `supplyCollateralAndSellWithAssetsTarget` — possible if the offer appears in an off-chain order book and is included in the `takes` array.
3. `targetSellerAssets > 0` — always true inside the loop due to the `filledSellerAssets < targetFilledSellerAssets` loop guard.

The attack is repeatable: the attacker can maintain the offer indefinitely (it is never consumed since the bundler reverts before `take()` is called). The settlement fee is bounded and discrete, making tick alignment feasible.

### Recommendation

In `sellerAssetsToUnits`, explicitly guard against `sellerPrice == 0` for buy offers, mirroring the behavior of `Midnight.take()` (which returns `sellerAssets = 0` for any units). When `sellerPrice == 0` and `targetSellerAssets > 0`, no finite number of units can yield a positive `sellerAssets`, so the function should return `type(uint256).max` (signaling "more units than possible") rather than reverting:

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
if (offer.buy) {
    if (sellerPrice == 0) return type(uint256).max;
    return targetSellerAssets.mulDivUp(WAD, sellerPrice);
} else {
    return targetSellerAssets.mulDivDown(WAD, sellerPrice);
}
```

This makes `unitsToTake = min(type(uint256).max, takes[i].units, consumableUnits(...))`, so the bundler proceeds to call `Midnight.take()` with the capped units, receives `sellerAssets = 0`, makes no progress toward `targetFilledSellerAssets`, and eventually fails with `OutOfOffers()` — a graceful failure rather than a mid-loop revert.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {TickLib} from "src/libraries/TickLib.sol";
import {TakeAmountsLib} from "src/periphery/TakeAmountsLib.sol";

contract SellerPriceZeroTest is Test {
    // Setup: deploy Midnight, create market, set settlement fee to F = tickToPrice(T) for some tick T.
    // Step 1: find tick T such that tickToPrice(T) == F (enumerate offline).
    // Step 2: feeSetter calls setMarketSettlementFee(id, ttmIndex, F).
    // Step 3: attacker (unprivileged) creates buy offer at tick T.
    // Step 4: taker calls supplyCollateralAndSellWithAssetsTarget with that offer.

    function testSellerAssetsToUnitsRevertsWhenSellerPriceZero() public {
        // Find a tick T where tickToPrice(T) == some valid settlement fee F.
        // e.g., tickToPrice(low_tick) == 1e12 (1 CBP).
        uint256 T = /* computed offline */ 0; // adjust to match
        uint256 F = TickLib.tickToPrice(T);
        assertGt(F, 0);

        // Set market settlement fee to F (requires feeSetter, done in setup).
        // midnight.setMarketSettlementFee(id, ttmIndex, F);

        // Construct buy offer at tick T.
        Offer memory offer;
        offer.buy = true;
        offer.tick = T;
        offer.market = market;

        // Assert sellerAssetsToUnits reverts.
        vm.expectRevert(); // arithmetic underflow in mulDivUp(d-1) when d=0
        TakeAmountsLib.sellerAssetsToUnits(address(midnight), id, offer, 1e18);

        // Assert Midnight.take() does NOT revert (sellerAssets = 0).
        // (requires valid offer setup: ratifier, maker, taker, collateral)
        (uint256 buyerAssets, uint256 sellerAssets) = midnight.take(offer, "", 1, taker, address(this), address(0), "");
        assertEq(sellerAssets, 0, "sellerAssets must be 0 when sellerPrice=0");

        // Assert bundler reverts entirely.
        vm.expectRevert();
        midnightBundles.supplyCollateralAndSellWithAssetsTarget(
            1e18, type(uint256).max, taker, receiver,
            new CollateralSupply[](0), takes, 0, address(0)
        );
    }
}
```

**Expected assertions**:
- `sellerAssetsToUnits` reverts (arithmetic underflow).
- `Midnight.take()` succeeds with `sellerAssets == 0`.
- `supplyCollateralAndSellWithAssetsTarget` reverts (propagated from `sellerAssetsToUnits`).
