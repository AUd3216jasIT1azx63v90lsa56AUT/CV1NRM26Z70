Audit Report

## Title
Authorized Operator Can Front-Run and Permanently Invalidate Authorizer's Pending Signed Authorization — (`src/periphery/EcrecoverAuthorizer.sol`)

## Summary

`EcrecoverAuthorizer.setIsAuthorized` unconditionally increments `nonce[authorization.authorizer]` at line 26 before the signer is verified at lines 33–36. Because the signer check accepts any address for which `IMidnight.isAuthorized(authorization.authorizer, signer)` returns `true`, any previously-authorized operator can construct an `Authorization` struct naming the victim as `authorizer`, sign it with their own key, and submit it with higher gas to consume the victim's current nonce. The victim's pending signed message is permanently invalidated since nonces are strictly sequential, and the attack is indefinitely repeatable.

## Finding Description

**Root cause — ordering of nonce increment vs. signer check:** [1](#0-0) 

The nonce is consumed unconditionally at line 26. Signer identity is not verified until lines 33–36: [2](#0-1) 

The second branch of the `||` on line 34 accepts any `signer` for which `isAuthorized[authorization.authorizer][signer] == true` — i.e., any operator the authorizer has previously authorized via `Midnight.setIsAuthorized`.

**Attacker-controlled inputs:**

The `Authorization` struct contains `authorizer` as a plain field: [3](#0-2) 

Nothing in the code binds `authorization.authorizer` to `msg.sender` or to the recovered signer. The attacker freely sets `authorizer = victim` and signs the struct with their own key.

**Exploit flow:**

1. Victim signs `Authorization{authorizer=victim, authorized=X, nonce=N, deadline=T}` and broadcasts.
2. Attacker (who holds `isAuthorized[victim][attacker] == true`) observes the pending tx, reads `N = nonce[victim]`.
3. Attacker constructs `Authorization{authorizer=victim, authorized=Y, nonce=N, deadline=T2}`, signs with attacker's key, submits with higher gas.
4. Attacker's tx: nonce check `N == N` passes → `nonce[victim]` becomes `N+1`; `ecrecover` returns attacker's address; `isAuthorized[victim][attacker] == true` → `Unauthorized()` check passes; `isAuthorized[victim][Y]` is set.
5. Victim's tx: nonce check `N == N+1` fails → `InvalidNonce()` revert.
6. Victim must re-sign with nonce `N+1`; attacker repeats indefinitely.

**Why existing checks fail:**

The `Unauthorized()` guard is designed to prevent strangers from acting on behalf of the authorizer, but it explicitly permits any previously-authorized operator. There is no requirement that the signer must be the authorizer themselves when consuming the authorizer's nonce. The nonce increment at line 26 is unconditional and precedes all identity verification. [4](#0-3) 

## Impact Explanation

Any authorized operator can permanently invalidate any pending signed authorization message from the authorizer by front-running with a higher-gas transaction that consumes the same nonce. The attacker can repeat this for every new nonce the victim signs, creating an indefinite DoS on the `EcrecoverAuthorizer` signed-message delegation path. If the victim needs to establish or revoke an authorization via a signed message (e.g., for a time-sensitive gate-restricted operation with a deadline), they are locked out of that path. The victim retains the ability to call `Midnight.setIsAuthorized` directly as `msg.sender`, but the signed-message delegation path is fully griefable. [5](#0-4) 

## Likelihood Explanation

**Preconditions:** `isAuthorized[victim][attacker] == true` — the attacker must have been previously authorized by the victim. This is a normal protocol state (e.g., victim authorized a keeper, relayer, or another user). The protocol documentation explicitly notes that "authorized accounts can authorize other accounts on behalf of the user," making this a common and expected state. [6](#0-5) 

**Feasibility:** Mempool front-running is standard on non-private mempools. The attacker only needs to observe the victim's pending tx and submit with higher gas. **Repeatability:** The attack is repeatable for every nonce the victim signs, making it a persistent DoS with no natural termination. The only escape for the victim is to revoke the attacker's authorization via a direct `Midnight.setIsAuthorized` call (not via `EcrecoverAuthorizer`), then re-sign.

## Recommendation

Move the nonce increment to after signer verification, or — preferably — require that the signer is strictly the `authorization.authorizer` (i.e., remove the `isAuthorized` branch from the signer check in `EcrecoverAuthorizer`). The signed-message path is specifically designed for the authorizer to delegate via their own signature; allowing third-party operators to sign on behalf of the authorizer conflates two distinct trust models and creates the nonce-griefing surface. Alternatively, key the nonce on `(authorizer, signer)` rather than `authorizer` alone, so each signer has an independent nonce that cannot interfere with the authorizer's own nonce. [7](#0-6)

// Minimal Foundry test sketch
function test_frontRunNonce() public {
    // Setup: victim authorizes attacker via direct call
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim prepares signed Authorization with nonce=0
    uint256 victimNonce = ecrecoverAuthorizer.nonce(victim);
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: someAddress,
        isAuthorized: true,
        nonce: victimNonce,   // == 0
        deadline: block.timestamp + 1 hours
    });
    Signature memory victimSig = signAuthorization(victimPrivKey, victimAuth);

    // Attacker front-runs: constructs Authorization with authorizer=victim, nonce=0, signs with attacker key
    Authorization memory attackerAuth = Authorization({
        authorizer: victim,
        authorized: attackerAddress,
        isAuthorized: true,
        nonce: victimNonce,   // same nonce
        deadline: block.timestamp + 2 hours
    });
    Signature memory attackerSig = signAuthorization(attackerPrivKey, attackerAuth);

    // Attacker submits first (higher gas in real scenario)
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(victim), 1); // nonce consumed

    // Victim's tx now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
