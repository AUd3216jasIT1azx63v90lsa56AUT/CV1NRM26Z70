### Title
Fee-on-Transfer Collateral Token Inflates `collateral[i]` Causing Undercollateralized Borrowing - (`src/interfaces/IERC20.sol` / `src/Midnight.sol`)

### Summary

`supplyCollateral` records the caller-supplied `assets` parameter directly into `_position.collateral[collateralIndex]` before executing the token transfer, with no balance-delta check. When the collateral token charges a fee on transfer, fewer tokens arrive at the contract than are recorded. `isHealthy` then computes `maxDebt` from the inflated stored value, making the position appear healthier than it is and allowing undercollateralized borrowing.

### Finding Description

**Code path and root cause:**

In `supplyCollateral` (`src/Midnight.sol` lines 532–545), the accounting update happens before the transfer and uses the raw input `assets`:

```solidity
// line 533 — records input, not actual received amount
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);
// ...
// line 545 — transfer happens after; fee-on-transfer tokens deliver assets - fee
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
``` [1](#0-0) 

`SafeTransferLib.safeTransferFrom` only verifies the call succeeded and returned `true`; it performs no balance-before/balance-after check: [2](#0-1) 

`isHealthy` then uses the stored (inflated) `_position.collateral[i]` to compute `maxDebt`:

```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [3](#0-2) 

`IERC20` has no mechanism to detect or account for transfer fees: [4](#0-3) 

**Market creation is fully permissionless.** `touchMarket` validates only LLTV tiers, maxLif, collateral sort order, and count — there is no check that the collateral token is not fee-on-transfer: [5](#0-4) 

**Exploit flow:**
1. Attacker deploys a `FeeOnTransferERC20` (1% fee) and creates a market with it as collateral via `touchMarket` — permissionless.
2. Attacker calls `supplyCollateral(market, 0, 100e18, attacker)`:
   - `_position.collateral[0]` is written as `100e18` (line 533).
   - Only `99e18` tokens arrive at the contract (line 545, fee deducted).
3. Attacker takes a lender's buy offer, borrowing up to `100e18 * price * LLTV / WAD` units.
4. `isHealthy` computes `maxDebt` using `100e18` — overstated by `1e18 * price * LLTV / WAD`.
5. The actual backing is only `99e18` tokens; the position is undercollateralized from inception.

### Impact Explanation

The borrower's recorded collateral exceeds actual contract balance. The position is undercollateralized from the moment of borrowing. If the oracle price drops even slightly, or at maturity, the shortfall cannot be covered by seizing collateral, creating bad debt that is socialized among lenders. The invariant `contract balances cover collateral` is broken: `collateral[i] > actual token balance increase`.

### Likelihood Explanation

Market creation is permissionless and any ERC20 can be used as collateral. Fee-on-transfer tokens (e.g., USDT with fee enabled, STA, PAXG, or any custom token) are a known token class. The attacker needs only to deploy or use an existing fee-on-transfer token, create a market, and call `supplyCollateral`. The attack is repeatable on every `supplyCollateral` call with such a token, and scales linearly with the fee rate and supplied amount.

### Recommendation

Replace the input-based accounting with a balance-delta pattern in `supplyCollateral`:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

This ensures `collateral[i]` always reflects actual tokens held, regardless of token transfer mechanics. The same pattern should be applied to any other function that pulls tokens and records the amount (e.g., `repay` for the loan token).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";

contract FeeOnTransferERC20 {
    // 1% fee on every transferFrom; returns true
    mapping(address => mapping(address => uint256)) public allowance;
    mapping(address => uint256) public balanceOf;
    // ... standard ERC20 + fee logic
    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        uint256 fee = amount / 100;
        balanceOf[from] -= amount;
        balanceOf[to] += amount - fee; // fee burned/retained
        return true;
    }
}

contract FeeOnTransferCollateralTest is Test {
    Midnight midnight;
    FeeOnTransferERC20 feeToken;
    Market market;

    function setUp() public {
        midnight = new Midnight(...);
        feeToken = new FeeOnTransferERC20();
        // configure market with feeToken as collateral, valid LLTV, oracle at price=1e36
    }

    function testFeeOnTransferInflatesCollateral() public {
        uint256 assets = 100e18;
        // mint and approve
        feeToken.mint(address(this), assets);
        feeToken.approve(address(midnight), assets);

        uint256 balBefore = feeToken.balanceOf(address(midnight));
        midnight.supplyCollateral(market, 0, assets, address(this));
        uint256 balAfter = feeToken.balanceOf(address(midnight));

        uint256 actualReceived = balAfter - balBefore;
        uint256 recorded = midnight.collateral(marketId, address(this), 0);

        // KEY ASSERTIONS:
        assertEq(actualReceived, 99e18, "only 99e18 arrived");
        assertEq(recorded, 100e18, "but 100e18 was recorded");
        assertGt(recorded, actualReceived, "collateral accounting overstated");

        // Borrow at LLTV of 100e18 — position is undercollateralized
        // take lender offer for recorded * lltv units
        // assert isHealthy() returns true despite actual backing being only 99e18
        assertTrue(midnight.isHealthy(market, marketId, address(this)));
    }
}
```

**Expected assertions:** `recorded (100e18) > actualReceived (99e18)` and `isHealthy` returns `true` despite the position being undercollateralized by `1e18 * price * LLTV`.
