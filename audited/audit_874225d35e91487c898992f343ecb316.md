Audit Report

## Title
Midnight-Authorized Agent Can Exhaust Victim's EcrecoverAuthorizer Nonces to Invalidate Pre-Signed Authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that the victim has authorized on Midnight to sign and submit `Authorization` structs naming the victim as `authorizer`. Because the nonce is keyed on `authorization.authorizer` and the `Unauthorized` check accepts any Midnight-authorized signer, an attacker holding a Midnight authorization from the victim can self-sign structs with `authorizer = victim` and submit them in a loop, permanently consuming `nonce[victim]`. Every pre-signed authorization the victim distributed to other agents becomes permanently invalid with `InvalidNonce`.

## Finding Description

**Exact code path — `EcrecoverAuthorizer.setIsAuthorized` (src/periphery/EcrecoverAuthorizer.sol:24-48):**

```solidity
// Line 26 — nonce keyed on authorization.authorizer, incremented on success
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// Lines 31-36 — signer check: authorizer OR any Midnight-authorized delegate
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
require(signer != address(0), InvalidSignature());
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

**Root cause:** The second branch of the `Unauthorized` check — `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` — allows any address the victim has authorized on Midnight to act as the signer for an `Authorization` struct that names the victim as `authorizer`. The attacker's own ECDSA signature over a struct with `authorizer = victim` satisfies this check, and the nonce increment on line 26 fires against `nonce[victim]` and persists (the transaction succeeds end-to-end).

**Exploit flow:**
1. Precondition: `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` — victim previously authorized attacker on Midnight (e.g., as a trading bot or router).
2. Victim pre-signs N authorizations with nonces 0…N-1 and distributes them to N agents for use in `take`, `repay`, `withdraw`, `liquidate`, or `claimFee` flows.
3. Attacker constructs `Authorization(authorizer=victim, authorized=attacker, isAuthorized=true, nonce=0, deadline=far_future)`, signs it with their own private key, and calls `EcrecoverAuthorizer.setIsAuthorized`.
4. Line 26 passes (`0 == nonce[victim]`), increments `nonce[victim]` to 1. The transaction succeeds.
5. Line 34 passes because `isAuthorized(victim, attacker) == true`.
6. Attacker repeats with nonce=1, 2, …, N-1.
7. All N pre-signed authorizations from the victim now revert with `InvalidNonce` when any agent tries to submit them.

**Why existing checks fail:**
- The `InvalidNonce` check (line 26) enforces sequential ordering only; it does not require that the authorizer themselves signed this particular nonce consumption.
- The `Unauthorized` check (lines 33-36) does not restrict who may consume the authorizer's nonces — it only requires the signer to be either the authorizer or a Midnight-authorized delegate. A Midnight-authorized delegate can therefore sign arbitrary `Authorization` structs naming the authorizer as `authorizer` and burn nonces at will.
- There is no `msg.sender == authorization.authorizer` guard or requirement that the signature originate from the authorizer's own key when consuming nonces.

## Impact Explanation
Any agent holding a Midnight authorization from the victim can unilaterally burn the victim's sequential nonces in `EcrecoverAuthorizer`. This permanently invalidates any batch of pre-signed `Authorization` structs the victim distributed off-chain. Every protocol action that relies on those pre-signed authorizations — `take`, `repay`, `withdraw`, `liquidate`, `claimFee` — reverts with `InvalidNonce` when submitted by the intended agents, causing a sustained and unrecoverable DoS on all those core market actions for the victim. The victim cannot "un-burn" consumed nonces; they must re-sign and redistribute all authorizations, which the attacker can immediately burn again as long as the Midnight authorization persists.

## Likelihood Explanation
**Precondition:** The attacker must hold `isAuthorized[victim][attacker] == true` on Midnight. This is a normal operational state — users routinely authorize bots, routers, or agents to act on their behalf. No tokens, collateral, or special role beyond this existing authorization is required.

**Feasibility:** The attack requires only repeated calls to `EcrecoverAuthorizer.setIsAuthorized` with self-signed structs. Each call costs one ECDSA signature (computed off-chain) and one transaction. On any L2 or low-fee chain, the cost per nonce burned is negligible. The attacker can front-run or race any legitimate submission.

**Repeatability:** The attack is indefinitely repeatable as long as the attacker remains authorized on Midnight. Even if the victim revokes the attacker's Midnight authorization, any nonces already burned are permanently consumed.

## Recommendation
Restrict nonce consumption to the authorizer's own key only. Remove the second branch from the `Unauthorized` check in `EcrecoverAuthorizer.setIsAuthorized`, requiring `signer == authorization.authorizer` unconditionally:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This preserves the permissionless submission model (anyone can relay a pre-signed authorization) while ensuring only the authorizer's own private key can consume their nonces. Delegation of the ability to modify `isAuthorized` on Midnight can still be achieved by the authorizer signing and distributing their own `Authorization` structs.

## Proof of Concept

**Minimal Foundry test (extend `EcrecoverAuthorizerTest` in `test/SetIsAuthorizedWithSigTest.sol`):**

```solidity
function testNonceBurningByAuthorizedAgent() public {
    // Setup: borrower authorizes ecrecoverAuthorizer and attacker on Midnight
    vm.prank(borrower);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
    vm.prank(borrower);
    midnight.setIsAuthorized(attacker, true, borrower);

    // Borrower pre-signs authorization for lender at nonce 0
    Authorization memory victimAuth = makeAuthorization(borrower, lender, true);
    Signature memory victimSig = signAuthorization(victimAuth, borrower);

    // Attacker constructs and self-signs a struct naming borrower as authorizer
    Authorization memory attackAuth = Authorization({
        authorizer: borrower,
        authorized: attacker,
        isAuthorized: true,
        nonce: ecrecoverAuthorizer.nonce(borrower), // = 0
        deadline: block.timestamp + 1 days
    });
    Signature memory attackSig = signAuthorization(attackAuth, attacker);

    // Attacker burns nonce 0
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);
    assertEq(ecrecoverAuthorizer.nonce(borrower), 1); // nonce burned

    // Victim's pre-signed authorization now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}
```

**Expected result:** The attacker's call succeeds and increments `nonce[borrower]` to 1. The victim's pre-signed authorization reverts with `InvalidNonce`, confirming the DoS.
