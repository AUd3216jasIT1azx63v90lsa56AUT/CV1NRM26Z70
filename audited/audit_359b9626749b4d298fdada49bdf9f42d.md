### Title
Authorized Delegate Can Escalate Privileges via EcrecoverAuthorizer to Drain Victim's Excess Collateral - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is already authorized by the `authorization.authorizer` in Midnight, not just the authorizer themselves. This allows an attacker who holds an existing authorization from the victim to sign a new `Authorization` struct naming a second address (`attacker2`) as authorized over the victim's account, bypassing the invariant that only the account owner (or their explicitly intended delegate) may modify their own authorization state. The result is that `attacker2` gains full operator rights over the victim's account and can drain all excess collateral.

### Finding Description
**Root cause — `EcrecoverAuthorizer.setIsAuthorized`, line 33–36:**

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

The second branch `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` permits any address that the victim has previously authorized in Midnight to act as the signer for a new `Authorization` struct whose `authorizer` field is the victim. This is a transitive privilege escalation: a limited delegate (e.g., a trading bot, ratifier, or callback contract) can manufacture a signature that grants an arbitrary third address (`attacker2`) full operator rights over the victim.

**Exact reachable path:**

Preconditions:
- `isAuthorized[victim][EcrecoverAuthorizer] == true` (victim opted into off-chain sig flow)
- `isAuthorized[victim][attacker] == true` (victim previously authorized attacker for any purpose)
- Victim has collateral deposited and active debt

Step 1 — Attacker constructs and signs off-chain:
```
Authorization {
  authorizer: victim,
  authorized: attacker2,
  isAuthorized: true,
  nonce: ecrecoverAuthorizer.nonce(victim),
  deadline: block.timestamp + 1
}
```
Signature is produced by `attacker`'s private key.

Step 2 — Anyone calls `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`:
- Deadline check passes.
- Nonce check passes and increments `nonce[victim]`.
- `ecrecover` returns `attacker`.
- Check at line 34: `attacker == victim` → false; `isAuthorized[victim][attacker]` → **true** → passes. [2](#0-1) 

Step 3 — `EcrecoverAuthorizer` calls `Midnight.setIsAuthorized(attacker2, true, victim)`:
- Check at line 732: `victim == EcrecoverAuthorizer` → false; `isAuthorized[victim][EcrecoverAuthorizer]` → **true** → passes.
- `isAuthorized[victim][attacker2] = true` is written. [3](#0-2) 

Step 4 — `attacker2` calls `Midnight.withdrawCollateral(market, idx, assets, victim, attacker2)`:
- Authorization check at line 556 passes because `isAuthorized[victim][attacker2] == true`.
- Collateral is reduced; `isHealthy` is called after the reduction. [4](#0-3) 

**Why existing checks are insufficient:**
The `Midnight.setIsAuthorized` guard correctly checks that `msg.sender` (the `EcrecoverAuthorizer` contract) is authorized by the victim — this is satisfied by precondition 1. The `EcrecoverAuthorizer` guard is the broken link: it was intended to verify the victim's own signature but instead accepts any signature from an existing delegate.

The Certora `EcrecoverAuthorizer.spec` only verifies nonce monotonicity and revert conditions; it does not assert that the signer must equal `authorization.authorizer`. [5](#0-4) 

### Impact Explanation
`attacker2` gains full operator rights over the victim's Midnight account. Via `withdrawCollateral`, `attacker2` can drain all excess collateral (any collateral above the minimum required to keep the position healthy at the LLTV boundary). The `isHealthy` check at line 568 prevents the position from being pushed into undercollateralization through `withdrawCollateral` alone, so the position is left maximally leveraged at the LLTV boundary rather than technically undercollateralized. However, `attacker2` also inherits the right to call `setConsumed` (canceling victim's offers), `setIsAuthorized` (granting further addresses operator rights), and `repay` — constituting full account takeover for all authorized operations. [6](#0-5) 

### Likelihood Explanation
Preconditions are realistic and common: any user who uses `EcrecoverAuthorizer` for off-chain authorization flows must authorize it in Midnight, satisfying precondition 1. Precondition 2 is satisfied by any other address the victim has ever authorized (ratifier, callback, bot, keeper). Both authorizations are normal protocol usage. The attack requires no special privileges, no oracle manipulation, and no token owner action. It is repeatable as long as the victim's nonce is known (it is public storage).

### Recommendation
Remove the delegate branch from the signer check in `EcrecoverAuthorizer.setIsAuthorized`. Only the `authorization.authorizer` themselves should be permitted to sign an `Authorization` struct that modifies their own authorization state:

```solidity
require(signer == authorization.authorizer, Unauthorized());
``` [1](#0-0) 

If delegated signing is intentionally desired, it must be scoped to a separate, explicitly opt-in mechanism that does not allow the delegate to expand their own or others' permissions.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);

/// EcrecoverAuthorizer increments nonce on success and does not change other nonces.
rule effects(env e, EcrecoverAuthorizer.Authorization authorization, EcrecoverAuthorizer.Signature signature, address other) {
    require other != authorization.authorizer;
    uint256 nonceBefore = nonce(authorization.authorizer);
    uint256 otherNonceBefore = nonce(other);

    setIsAuthorized(e, authorization, signature);

    assert nonce(authorization.authorizer) == nonceBefore + 1;
    assert nonce(other) == otherNonceBefore;
}
