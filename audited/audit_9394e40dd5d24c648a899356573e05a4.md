### Title
`sellerAssetsToUnits` division-by-zero when `tickToPrice(tick) == settlementFee` blocks `supplyCollateralAndSellWithAssetsTarget` - (File: src/periphery/TakeAmountsLib.sol)

### Summary
`TakeAmountsLib.sellerAssetsToUnits` computes `sellerPrice = offerPrice - settlementFee` for buy offers and then divides by it. When `offerPrice == settlementFee`, `sellerPrice` is exactly zero, causing an unconditional division-by-zero revert. The core `Midnight.take()` handles this case without reverting (seller simply receives zero assets), so the periphery and core diverge. Because `supplyCollateralAndSellWithAssetsTarget` calls `sellerAssetsToUnits` outside its `try/catch` block, any revert there propagates to the caller, permanently DoS-ing that entry-point for any taker whose `takes[]` array includes such an offer.

### Finding Description

**Root cause — `TakeAmountsLib.sellerAssetsToUnits` (lines 44–46):**

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
return
    offer.buy ? targetSellerAssets.mulDivUp(WAD, sellerPrice) : ...;
```

When `offer.buy == true` and `offerPrice == settlementFee`, `sellerPrice = 0`. The subsequent `mulDivUp(targetSellerAssets, WAD, 0)` triggers a division-by-zero revert in Solidity 0.8+. [1](#0-0) 

**Core `take()` does NOT revert in the same scenario (lines 361–364):**

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice; // = 0
uint256 sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : ...; // = 0, no revert
```

`mulDivDown(units, 0, WAD)` multiplies first (result 0) then divides by `WAD` — no division-by-zero. `take()` succeeds and the seller receives zero assets. [2](#0-1) 

**`supplyCollateralAndSellWithAssetsTarget` calls `sellerAssetsToUnits` outside the `try/catch` (lines 285–291):**

```solidity
uint256 unitsToTake = min(
    TakeAmountsLib.sellerAssetsToUnits(          // ← NOT inside try/catch
        MIDNIGHT, id, takes[i].offer, targetFilledSellerAssets - filledSellerAssets
    ),
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
);
try IMidnight(MIDNIGHT).take(...) ...            // ← only take() is guarded
```

A revert from `sellerAssetsToUnits` is not caught and bubbles up, reverting the entire bundle call. [3](#0-2) 

**Attacker-controlled inputs:** The attacker is an unprivileged maker who places a buy offer at tick `T` where `tickToPrice(T) == settlementFee(id, ttm)`. No privileged access is required; the attacker only needs to observe the current settlement fee and find (or wait for) a matching accessible tick.

**Existing protections are insufficient:** The NatSpec comment on `sellerAssetsToUnits` documents revert only for `offerPrice < settlementFee` (which causes an underflow revert in Solidity 0.8+). The `==` case is undocumented and unguarded. [4](#0-3) 

### Impact Explanation
Any call to `supplyCollateralAndSellWithAssetsTarget` whose `takes[]` array contains a buy offer at the settlement-fee price point reverts entirely. The attacker can grief any taker who includes that offer, permanently blocking the periphery sell path for that offer. Because the offer itself is valid and `take()` would accept it, the DoS is invisible to the taker until the bundle reverts.

### Likelihood Explanation
The settlement fee is a discrete value set by governance; tick prices are also discrete. The condition `tickToPrice(T) == settlementFee` is achievable whenever the fee is set to a value that coincides with any accessible tick price. An attacker can monitor the fee, identify the matching tick, and post a buy offer there at negligible cost (no capital at risk for a buy offer with `maxAssets = 0` or a small amount). The attack is repeatable: a new offer can be posted after each cancellation.

### Recommendation
Add an explicit guard in `sellerAssetsToUnits` for the `sellerPrice == 0` case, mirroring the `buyerPrice <= WAD` guard in `buyerAssetsToUnits`:

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
require(sellerPrice > 0, SellerPriceIsZero());   // add this
return offer.buy
    ? targetSellerAssets.mulDivUp(WAD, sellerPrice)
    : targetSellerAssets.mulDivDown(WAD, sellerPrice);
```

Update the NatSpec to document that the function reverts when `offerPrice <= settlementFee` (not just `<`). Alternatively, wrap the `sellerAssetsToUnits` call in `supplyCollateralAndSellWithAssetsTarget` inside a `try/catch` or a conditional skip, consistent with how `take()` itself is guarded.

### Proof of Concept

```solidity
function testSellerAssetsToUnitsZeroSellerPriceReverts() public {
    // 1. Set settlement fee to match a specific tick price
    uint256 targetTick = 100; // some accessible tick
    uint256 fee = TickLib.tickToPrice(targetTick);
    midnight.setDefaultSettlementFee(address(loanToken), 1, fee);
    midnight.touchMarket(market);

    // Confirm sellerPrice == 0
    uint256 sfee = midnight.settlementFee(id, market.maturity - block.timestamp);
    assertEq(TickLib.tickToPrice(targetTick), sfee, "precondition: price == fee");

    // 2. Attacker posts buy offer at that tick
    Offer memory attackOffer = lenderOffer;
    attackOffer.tick = targetTick;
    attackOffer.buy = true;
    attackOffer.maxUnits = type(uint256).max;

    // 3. sellerAssetsToUnits reverts with division-by-zero
    vm.expectRevert();
    TakeAmountsLib.sellerAssetsToUnits(address(midnight), id, attackOffer, 1e18);

    // 4. take() with units=1 succeeds (sellerAssets == 0)
    collateralize(market, borrower, 1);
    (uint256 buyerAssets, uint256 sellerAssets) = take(1, borrower, attackOffer);
    assertEq(sellerAssets, 0, "take succeeds, seller gets 0");
    assertGt(buyerAssets, 0, "buyer pays fee");

    // 5. supplyCollateralAndSellWithAssetsTarget reverts
    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: attackOffer, units: type(uint256).max, ratifierData: hex""});
    collateralize(market, borrower, 1000);
    vm.prank(borrower);
    vm.expectRevert(); // division-by-zero from sellerAssetsToUnits
    midnightBundles.supplyCollateralAndSellWithAssetsTarget(
        1e18, type(uint256).max, borrower, borrower,
        new CollateralSupply[](0), takes, 0, address(0)
    );
}
```

Expected assertions: step 3 reverts (division-by-zero), step 4 succeeds with `sellerAssets == 0`, step 5 reverts — confirming the invariant violation.
