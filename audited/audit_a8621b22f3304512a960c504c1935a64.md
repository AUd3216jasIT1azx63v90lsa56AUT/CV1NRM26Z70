### Title
Unbounded `_poolsByAgreement` array enables sponsor to OOG `getPoolsByAgreement` in constrained-gas contexts - (File: src/ConfidencePoolFactory.sol)

### Summary
`createPool` imposes no per-agreement pool count limit, so a pool sponsor (agreement owner) can push an unbounded number of entries into `_poolsByAgreement[agreement]`. `getPoolsByAgreement` returns the entire array with no pagination and no gas guard, so once the array is large enough, any caller that invokes it with a fixed gas budget (e.g. a callback forwarding 100 000 gas) will OOG, permanently DoS-ing that integration for the targeted agreement.

### Finding Description
`_poolsByAgreement` is a `mapping(address => address[])` that grows by one entry on every successful `createPool` call. [1](#0-0) 

`createPool` enforces: a valid agreement, an allowlisted token, `expiry ≥ now + 30 days`, and `msg.sender == IAgreement(agreement).owner()`. There is no cap on how many times the same sponsor may call it for the same agreement. [2](#0-1) 

`getPoolsByAgreement` copies the entire storage array into memory and returns it: [3](#0-2) 

Reading N cold storage slots costs ≈ 2 100 gas each (EIP-2929). With a 100 000 gas budget, the function OOGs at roughly N ≈ 45–50 entries. A sponsor who creates that many pools (each a cheap clone deployment with `minStake = 0`) permanently breaks any downstream contract that calls `getPoolsByAgreement` with a fixed gas limit for that agreement. The factory comment acknowledges "duplicates are allowed and curated off-chain," confirming no on-chain bound exists. [4](#0-3) 

`docs/DESIGN.md` contains no mention of this trade-off and does not accept unbounded array growth as intentional behavior.

### Impact Explanation
Any protocol integration (router, aggregator, on-chain registry consumer) that calls `getPoolsByAgreement` inside a callback or with a capped gas stipend is permanently DoS'd for the targeted agreement once the array exceeds the threshold. The view function itself is not callable in constrained contexts, and there is no fallback (`poolCountByAgreement` exists but does not help a caller that needs the addresses). Core pool functionality (staking, claiming, resolution) is unaffected, so the impact is scoped to integrations relying on this view.

### Likelihood Explanation
The actor is the pool sponsor — the agreement owner — who is explicitly not a trusted actor in the protocol's adversarial model. Prerequisites are: owning a valid agreement, having access to an allowlisted token, and paying gas for ≈ 50 clone deployments (each ~300 000–500 000 gas). This is a one-time, low-cost griefing action that is permanent for the targeted agreement. Any agreement owner with a grievance against an integrating protocol can execute this without any victim mistake or trusted-role compromise.

### Recommendation
Add a per-agreement pool count cap in `createPool`, e.g.:

```solidity
uint256 private constant _MAX_POOLS_PER_AGREEMENT = 20;

// inside createPool, before the push:
if (_poolsByAgreement[agreement].length >= _MAX_POOLS_PER_AGREEMENT)
    revert TooManyPoolsForAgreement();
```

Alternatively, add offset/limit parameters to `getPoolsByAgreement` so callers can paginate, and keep `poolCountByAgreement` as the length oracle. Both mitigations together are the most robust fix.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.26;

import "forge-std/Test.sol";
import {ConfidencePoolFactory} from "src/ConfidencePoolFactory.sol";
// ... standard test imports / mock setup

contract PoolConsumer {
    IConfidencePoolFactory factory;
    constructor(address f) { factory = IConfidencePoolFactory(f); }

    /// Called with a capped gas stipend to simulate a callback
    function process(address agreement) external returns (address[] memory) {
        return factory.getPoolsByAgreement{gas: 100_000}(agreement);
    }
}

contract GetPoolsByAgreementOOGTest is Test {
    // Standard fixture: deploy factory, mock registry returning isAgreementValid=true,
    // mock agreement returning owner=sponsor, allowlist a mock ERC20.

    function test_sponsorOOGsGetPoolsByAgreement() public {
        address sponsor = makeAddr("sponsor");
        address agreement = address(mockAgreement); // owner() == sponsor

        // Sponsor creates N pools for the same agreement
        uint256 N = 50;
        vm.startPrank(sponsor);
        for (uint256 i = 0; i < N; i++) {
            factory.createPool(
                agreement,
                address(mockToken),
                block.timestamp + 31 days,
                0,                    // minStake = 0, no funds required
                makeAddr("recovery"),
                new address[](0)
            );
        }
        vm.stopPrank();

        assertEq(factory.poolCountByAgreement(agreement), N);

        // Deploy consumer and call with capped gas
        PoolConsumer consumer = new PoolConsumer(address(factory));
        // Expect OOG revert when called with 100 000 gas
        vm.expectRevert();
        consumer.process(agreement); // internally forwards 100_000 gas
    }
}
```

**Decisive assertion:** `consumer.process(agreement)` reverts (OOG) after N ≈ 45–50 pools. The threshold can be found by bisecting N in a fuzz test. `poolCountByAgreement` (which reads only the array length — one SLOAD) succeeds at the same gas limit, confirming the root cause is the full-array copy in `getPoolsByAgreement`, not the call overhead.

    function createPool(
        address agreement,
        address stakeToken,
        uint256 expiry,
        uint256 minStake,
        address recoveryAddress,
        address[] calldata accounts
    ) external whenNotPaused returns (address pool) {
        if (agreement == address(0) || stakeToken == address(0)) revert ZeroAddress();
        if (recoveryAddress == address(0)) revert ZeroAddress();
        if (!allowedStakeToken[stakeToken]) revert StakeTokenNotAllowed();
        if (expiry < block.timestamp + _MIN_EXPIRY_LEAD) revert ExpiryTooSoon();
        // aderyn-fp-next-line(reentrancy-state-change)
        if (!safeHarborRegistry.isAgreementValid(agreement)) revert InvalidAgreement();
        // aderyn-fp-next-line(reentrancy-state-change)
        if (IAgreement(agreement).owner() != msg.sender) revert UnauthorizedCreator();

        // Salt incorporates the per-agreement index so an agreement can back many pools while
        // keeping deterministic, collision-free clone addresses.
        bytes32 salt = keccak256(abi.encode(agreement, _poolsByAgreement[agreement].length));
        pool = Clones.cloneDeterministic(poolImplementation, salt);

        // aderyn-fp-next-line(reentrancy-state-change)
        IConfidencePool(pool)
            .initialize(
                agreement,
                stakeToken,
                address(safeHarborRegistry),
                defaultOutcomeModerator,
                expiry,
                minStake,
                recoveryAddress,
                msg.sender,
                accounts
            );

        _poolsByAgreement[agreement].push(pool);
