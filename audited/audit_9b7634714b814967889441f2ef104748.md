### Title
Missing `loanToken` validation in `touchMarket` creates permanently non-functional market - (File: src/Midnight.sol)

### Summary
`touchMarket` validates maturity, collateral list ordering, LLTV tiers, and maxLif values, but performs no check on `market.loanToken`. An unprivileged caller can invoke `touchMarket` directly with `loanToken == address(0)`, permanently writing market state (`tickSpacing = DEFAULT_TICK_SPACING`). Every subsequent `take()` call on that market reaches `SafeTransferLib.safeTransferFrom(address(0), ...)`, which unconditionally reverts with `NoCode()` because `address(0).code.length == 0`.

### Finding Description
**Root cause — `touchMarket`, `src/Midnight.sol` lines 755–791:**

`touchMarket` is `public` and callable by anyone. Its validation block checks:
- `market.maturity <= block.timestamp + 100 * 365 days` [1](#0-0) 
- `collateralParams.length > 0` and `<= MAX_COLLATERALS` [2](#0-1) 
- Each collateral token is strictly greater than the previous (sorted, no duplicates, no `address(0)`) [3](#0-2) 
- `isLltvAllowed(lltv)` and valid `maxLif` [4](#0-3) 

There is **no check on `market.loanToken`**. After passing all checks, `tickSpacing` is written to storage unconditionally: [5](#0-4) 

**Exploit flow:**

1. Attacker calls `midnight.touchMarket(market)` with `market.loanToken = address(0)` and otherwise valid parameters (valid maturity, one valid collateral with allowed LLTV/maxLif). The call succeeds; `marketState[id].tickSpacing = DEFAULT_TICK_SPACING` is written permanently.

2. Any subsequent call to `take()` on this market reaches line 347: `bytes32 id = touchMarket(offer.market)` — which now returns the existing ID without re-entering the creation branch. [6](#0-5) 

3. `take()` proceeds through all pre-transfer logic and reaches: [7](#0-6) 
   ```solidity
   SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
   SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
   ```

4. `SafeTransferLib.safeTransferFrom` immediately reverts: [8](#0-7) 
   ```solidity
   function safeTransferFrom(address token, ...) internal {
       require(token.code.length > 0, NoCode()); // address(0).code.length == 0 → always reverts
   ```

The same `NoCode()` revert applies to `safeTransfer` calls in `withdraw`, `repay`, and `claimContinuousFee` for the same market. [9](#0-8) 

**Why existing checks do not stop it:** The `SetterRatifier.isRatified` check at line 356 only verifies Merkle proof membership and that the maker ratified the root — it does not inspect `market.loanToken`. [10](#0-9)  No other guard in `take()` inspects `loanToken` before the transfer.

### Impact Explanation
Once `touchMarket` is called with `loanToken == address(0)`, the market state is permanently written (`tickSpacing > 0`). Every `take()`, `withdraw()`, `repay()`, and `claimContinuousFee()` call on this market will revert at the `SafeTransferLib` `NoCode()` guard. The market is created but permanently non-functional; no trade can ever settle, and no credit or debt position can be exited through normal protocol flows.

### Likelihood Explanation
`touchMarket` is `public` with no access control. Any address can call it with `loanToken = address(0)` and a single valid collateral entry. The precondition (valid collateral params, valid maturity) is trivially satisfiable. The attack is a single transaction, costs only gas, and is irreversible once the market state is written.

### Recommendation
Add a non-zero check on `loanToken` inside the market-creation branch of `touchMarket`:

```solidity
require(market.loanToken != address(0), InvalidLoanToken());
```

This mirrors the implicit protection already applied to collateral tokens via the strict-greater-than ordering check (`collateralToken > previousCollateralToken` where `previousCollateralToken` starts at `address(0)`). [11](#0-10) 

### Proof of Concept
```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {Midnight, Market, CollateralParams, Offer} from "src/Midnight.sol";

contract ZeroLoanTokenTest is Test {
    Midnight midnight;
    address attacker = makeAddr("attacker");
    address maker    = makeAddr("maker");

    function setUp() public {
        midnight = new Midnight(/* constructor args */);
    }

    function testZeroLoanTokenMarketCreatedAndTakeReverts() public {
        // Step 1: build a market with loanToken == address(0)
        CollateralParams[] memory cp = new CollateralParams[](1);
        cp[0] = CollateralParams({
            token:   address(uint160(1)),   // > address(0), passes sort check
            lltv:    0.77e18,
            maxLif:  midnight.maxLif(0.77e18, midnight.LIQUIDATION_CURSOR_LOW()),
            oracle:  address(0)
        });
        Market memory m = Market({
            loanToken:      address(0),     // ← attacker-controlled zero address
            maturity:       block.timestamp + 1 days,
            collateralParams: cp,
            rcfThreshold:   0,
            enterGate:      address(0),
            liquidatorGate: address(0)
        });

        // Step 2: attacker creates the market — must succeed
        vm.prank(attacker);
        bytes32 id = midnight.touchMarket(m);
        assertGt(midnight.tickSpacing(id), 0, "market created");

        // Step 3: any take() on this market must revert with NoCode()
        Offer memory offer = /* minimal valid offer for market m, maker-signed */;
        vm.prank(maker);
        vm.expectRevert(SafeTransferLib.NoCode.selector);
        midnight.take(offer, ratifierData, 1, maker, address(0), address(0), "");
    }
}
```

**Expected assertions:**
- `midnight.tickSpacing(id) > 0` — market is permanently created
- `take()` reverts with `SafeTransferLib.NoCode` — all trading is permanently blocked
