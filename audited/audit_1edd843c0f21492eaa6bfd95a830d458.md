### Title
Taker-Supplied `takerCallback` Becomes Unchecked `payer` for Sell Offers, Enabling Token Drain from Victim Callback Contracts - (File: src/Midnight.sol)

### Summary
For sell offers (`offer.buy == false`) where the maker signs with `offer.callback == address(0)`, the `payer` for the `safeTransferFrom` calls is derived entirely from the attacker-supplied `takerCallback` parameter. `hashOffer` in `src/ratifiers/libraries/HashLib.sol` commits `offer.callback` and `keccak256(offer.callbackData)` but never commits `takerCallback` or `takerCallbackData`, and no authorization check binds `takerCallback` to the taker. An attacker can therefore designate any Midnight-approved callback contract as the payer, pulling tokens from it without its consent.

### Finding Description

**`hashOffer` scope:**
`HashLib.hashOffer` encodes `offer.callback` and `keccak256(offer.callbackData)` into the EIP-712 leaf hash, but `takerCallback` and `takerCallbackData` are runtime parameters of `take()` and are never included in the signed hash. [1](#0-0) 

**Payer derivation in `take()`:**
```
Line 420: address buyerCallback = offer.buy ? offer.callback : takerCallback;
Line 422: address payer = buyerCallback != address(0) ? buyerCallback : (offer.buy ? buyer : msg.sender);
```
For `offer.buy == false` and `takerCallback != address(0)`:
- `buyerCallback = takerCallback` (attacker-supplied)
- `payer = takerCallback` (attacker-supplied) [2](#0-1) 

**Token pulls use `payer`:**
```solidity
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
``` [3](#0-2) 

**No authorization check on `takerCallback`:**
The only authorization check in `take()` is:
```solidity
require(taker == msg.sender || isAuthorized[taker][msg.sender], TakerUnauthorized());
```
There is no corresponding check that `takerCallback` is authorized by `taker` or equals `msg.sender`. [4](#0-3) 

**Exploit flow:**
1. Maker signs a valid sell offer with `offer.callback = address(0)`.
2. Attacker identifies `victimContract`: a Midnight-aware contract that (a) has approved the Midnight contract to spend its loan tokens, and (b) implements `IBuyCallback.onBuy` returning `CALLBACK_SUCCESS = keccak256("morpho.midnight.callbackSuccess")` without validating the `buyer` parameter.
3. Attacker calls `take(offer, ratifierData, units, taker, receiver, victimContract, takerCallbackData)`.
4. `buyerCallback = victimContract`, `payer = victimContract`.
5. Midnight calls `victimContract.onBuy(...)`. If the victim only checks `msg.sender == midnight` (a common pattern), it returns `CALLBACK_SUCCESS`.
6. Midnight executes `safeTransferFrom(loanToken, victimContract, ...)`, draining `buyerAssets` from `victimContract`.

**Why existing checks fail:**
- The ratifier only verifies the EIP-712 hash of the `Offer` struct, which does not include `takerCallback`.
- The `onBuy` callback guard only prevents the exploit against contracts that validate the `buyer` argument; contracts that only gate on `msg.sender == midnight` pass through.
- `CALLBACK_SUCCESS` is a non-trivial bytes32 value, but any contract implementing `IBuyCallback` correctly will return it. [5](#0-4) 

### Impact Explanation
Tokens are pulled via `safeTransferFrom` from `victimContract` — an address not committed in the maker's signed offer hash and not authorized by the taker — violating the invariant that "signatures, gates, ratifiers, callbacks, and approvals bind the right user/market/action/amount/deadline." The victim loses `buyerAssets` of the loan token with no recourse.

### Likelihood Explanation
**Preconditions:**
1. A valid sell offer exists (signed, not expired, not fully consumed).
2. A victim contract exists that: (a) has approved Midnight for the loan token, and (b) implements `IBuyCallback.onBuy` returning `CALLBACK_SUCCESS` without checking the `buyer` argument.

Condition (b) is realistic: aggregator routers, vault wrappers, and periphery contracts built on Midnight are natural targets. Any such contract that gates only on `msg.sender == midnight` is exploitable. The attack is permissionless, repeatable across any valid sell offer, and requires no special role.

### Recommendation
Add an authorization check on `takerCallback` mirroring the existing `taker` check:

```solidity
require(
    takerCallback == address(0) ||
    takerCallback == msg.sender ||
    isAuthorized[taker][takerCallback],
    TakerUnauthorized()
);
```

This ensures the taker explicitly authorizes the contract that will be used as payer, binding the payment source to the taker's intent without requiring it to be committed in the offer hash.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {ICallbacks} from "src/interfaces/ICallbacks.sol";
import {ConstantsLib} from "src/libraries/ConstantsLib.sol";

// Victim: approves Midnight, returns CALLBACK_SUCCESS unconditionally
contract VictimCallback is IBuyCallback {
    bytes32 constant CALLBACK_SUCCESS = keccak256("morpho.midnight.callbackSuccess");
    function onBuy(bytes32, Market memory, uint256, uint256, uint128, address, bytes memory)
        external pure returns (bytes32) { return CALLBACK_SUCCESS; }
}

contract PayerConfusionTest is Test {
    function test_takerCallbackPayerConfusion() public {
        // Setup: deploy Midnight, create market, fund victim, victim approves Midnight
        // Maker signs sell offer with offer.callback = address(0)
        // Attacker calls take(..., takerCallback = address(victimCallback))
        
        uint256 victimBalanceBefore = loanToken.balanceOf(address(victim));
        
        midnight.take(offer, ratifierData, units, attacker, attacker, address(victim), "");
        
        uint256 victimBalanceAfter = loanToken.balanceOf(address(victim));
        
        // Assert: victim's tokens were drained
        assertLt(victimBalanceAfter, victimBalanceBefore);
        // Assert: offer hash was identical regardless of takerCallback value
        assertEq(hashLib.hashOffer(offerWithCallback0), hashLib.hashOffer(offerWithCallback0));
        // Assert: attacker paid nothing
        assertEq(loanToken.balanceOf(attacker), attackerBalanceBefore);
    }
}
```

**Expected assertions:**
- `victimBalanceAfter < victimBalanceBefore` by `buyerAssets`
- `hashOffer(offer)` is identical whether `takerCallback` is `address(0)` or `address(victim)` (differential test)
- Attacker's token balance is unchanged
