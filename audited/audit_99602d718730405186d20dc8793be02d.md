### Title
Zero-oracle market creation permanently blocks liquidation of unhealthy positions - (File: src/Midnight.sol)

### Summary
`touchMarket` validates collateral token address, LLTV tier, and maxLif, but imposes no constraint on `collateralParams[i].oracle`. An unprivileged market creator can deploy a market with `oracle = address(0)` for any collateral slot. Once a borrower activates that collateral slot (setting the corresponding bitmap bit), every subsequent call to `liquidate` unconditionally calls `IOracle(address(0)).price()` inside the bitmap loop, which reverts in Solidity 0.8.34 because the zero address has no code and returns empty data that cannot be ABI-decoded as `uint256`. The position becomes permanently unliquidatable.

### Finding Description
**Code path:**

`touchMarket` (lines 754–791) iterates over `collateralParams` and enforces:
- `collateralToken > previousCollateralToken` (non-zero, sorted token)
- `isLltvAllowed(lltv)`
- `maxLif` in the two allowed tiers [1](#0-0) 

There is no `require(market.collateralParams[i].oracle != address(0))` check. The Certora invariant `createdMarketsHaveNonZeroCollaterals` only proves `token != 0`, not `oracle != 0`. [2](#0-1) 

In `liquidate`, the bitmap loop unconditionally calls `.price()` on whatever oracle address is stored in the caller-supplied `market.collateralParams[i]`: [3](#0-2) 

The market ID is derived from a full hash of the market struct including every `oracle` field: [4](#0-3) 

This means a liquidator cannot substitute a valid oracle — they must pass the exact market struct that was used at creation time, which has `oracle = address(0)`.

**Exploit flow:**
1. Attacker (market creator) calls `touchMarket` with `collateralParams[i].oracle = address(0)` — passes all validation.
2. Borrower supplies collateral at index `i` and takes a loan via `take`; bit `i` is set in `collateralBitmap`, `collateral[i] > 0`.
3. Position becomes unhealthy (price drop or post-maturity).
4. Any liquidator calls `liquidate` with the correct market struct.
5. The loop reaches bit `i`, executes `IOracle(address(0)).price()` (Solidity 0.8.34, line 610).
6. `address(0)` has no code; the CALL succeeds but returns 0 bytes; the ABI decoder for `uint256` reverts on empty returndata.
7. `liquidate` always reverts; the unhealthy position can never be liquidated.

The Certora `Reverts.spec` rule `oracleRevertCausesLiquidateRevert` explicitly proves this revert propagation — confirming the behavior, not preventing it: [5](#0-4) 

### Impact Explanation
Lenders' funds are frozen. The unhealthy position accumulates bad debt that can never be realized through `liquidate`. The core invariant "unhealthy positions remain liquidatable" is permanently violated for any market created with a zero oracle. Because market creation is permissionless, any actor can deploy such a market and attract borrowers (e.g., by offering favorable rates via `take`).

### Likelihood Explanation
Market creation is fully permissionless — no admin approval is required. The attacker only needs to call `touchMarket` with a crafted `CollateralParams` array. The condition is trivially reachable: one call creates the trap, and any borrower who interacts with the market is affected. The DoS is permanent and repeatable across any number of markets.

### Recommendation
Add a non-zero oracle check inside the `touchMarket` validation loop:

```solidity
require(market.collateralParams[i].oracle != address(0), ZeroOracle());
```

This should be placed alongside the existing token, LLTV, and maxLif checks at lines 762–772 of `src/Midnight.sol`. The corresponding Certora invariant in `CreatedMarkets.spec` should be extended to assert `market.collateralParams[i].oracle != 0` for all created markets. [1](#0-0)

```solidity
pragma solidity 0.8.34;

import {Test} from "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";
import {ERC20} from "test/erc20s/ERC20.sol";
import {Oracle} from "test/helpers/Oracle.sol";
import {BaseTest} from "test/BaseTest.sol";

contract ZeroOracleLiquidationDoSTest is BaseTest {
    function testZeroOraclePermanentlyBlocksLiquidation() public {
        // 1. Market creator builds a market with oracle = address(0) for collateral[0]
        CollateralParams[] memory params = new CollateralParams[](1);
        params[0] = CollateralParams({
            token: address(collateralToken1),
            lltv: 0.77e18,
            maxLif: maxLif(0.77e18, 0.25e18),
            oracle: address(0)          // <-- zero oracle, no validation blocks this
        });
        Market memory badMarket;
        badMarket.loanToken = address(loanToken);
        badMarket.maturity = block.timestamp + 100;
        badMarket.collateralParams = params;

        // 2. touchMarket succeeds — no oracle check
        midnight.touchMarket(badMarket);

        // 3. Borrower supplies collateral and takes a loan
        //    (setupMarket + collateralize helpers set up the position)
        setupMarket(badMarket, 1e18);
        collateralize(badMarket, borrower, 1e18);

        // 4. Make position unhealthy (warp past maturity)
        vm.warp(badMarket.maturity + 1);

        // 5. Liquidator attempts liquidation — must revert
        vm.expectRevert(); // ABI decode failure on address(0).price()
        midnight.liquidate(
            badMarket, 0, 0, 0, borrower, true, address(this), address(0), ""
        );

        // 6. Assert borrower still has debt — position is permanently unliquidatable
        bytes32 id = toId(badMarket);
        assertGt(midnight.debtOf(id, borrower), 0, "debt must remain: position unliquidatable");
    }
}
```

**Expected assertions:**
- `liquidate` reverts on every call (zero-address oracle returns empty data, ABI decode of `uint256` fails).
- `midnight.debtOf(id, borrower) > 0` after the revert — bad debt is never realized.
- Lender credit cannot be redeemed because `totalUnits` is never reduced via the bad-debt path.

```text
/// If any activated collateral oracle reverts on price, liquidate reverts.
rule oracleRevertCausesLiquidateRevert(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, uint256 revertingCollateralIndex, bool postMaturityMode) {

    bytes32 id = summaryToId(market);
    uint128 bitmap = collateralBitmap(id, borrower);
    require summaryGetBit(bitmap, revertingCollateralIndex), "revertingCollateralIndex is activated";

    liquidate@withrevert(e, market, collateralIndex, seizedAssets, repaidUnits, borrower, postMaturityMode, receiver, callback, data);

    assert lastReverted;
}
```
