### Title
Missing Oracle Address Validation in `touchMarket` Enables Permanent Fund Freeze via `isHealthy` Revert - (File: src/Midnight.sol)

### Summary
`touchMarket` validates `collateralToken`, `lltv`, and `maxLif` for each `CollateralParams` entry but performs no validation on the `oracle` field. Any unprivileged actor can create a market with `oracle = address(0)` or any non-contract address. Once a borrower has debt and an activated collateral bitmap bit pointing to such an oracle, every call path that invokes `isHealthy` — including `withdrawCollateral` and `liquidate` — permanently reverts, freezing the borrower's collateral with no recovery path.

### Finding Description

**Root cause — missing check in `touchMarket`:** [1](#0-0) 

The loop validates `collateralToken > previousCollateralToken`, `isLltvAllowed(lltv)`, and `maxLif`, but never checks `market.collateralParams[i].oracle != address(0)` or that the oracle address has code. The `oracle` field is stored verbatim.

**Trigger in `isHealthy`:** [2](#0-1) 

When `debt > 0`, the function iterates the collateral bitmap and calls `IOracle(collateralParam.oracle).price()`. For `oracle = address(0)` (the ECRECOVER precompile), the EVM call returns `success = true` with empty returndata. Solidity 0.8.x then attempts to ABI-decode the empty returndata as `uint256`, which reverts because fewer than 32 bytes were returned. This is a permanent, unconditional revert for any borrower with debt in this market.

**`withdrawCollateral` propagates the revert:** [3](#0-2) 

`require(isHealthy(market, id, onBehalf), UnhealthyBorrower())` — if `isHealthy` reverts (not returns false), the entire `withdrawCollateral` call reverts.

**`liquidate` has the same oracle call in its own loop:** [4](#0-3) 

The liquidation loop also calls `IOracle(_collateralParam.oracle).price()` for every activated collateral, so liquidation also permanently reverts, leaving the position unliquidatable.

**Exploit flow:**
1. Attacker (market creator) calls `touchMarket` with `CollateralParams { token: <valid_token>, lltv: <allowed>, maxLif: <valid>, oracle: address(0) }`.
2. Victim borrower calls `supplyCollateral` (activates the collateral bitmap bit at that index).
3. Victim calls `take` as seller, acquiring debt.
4. Victim calls `withdrawCollateral` → `isHealthy` → `IOracle(address(0)).price()` → ABI decode revert → `withdrawCollateral` reverts.
5. Any liquidator calls `liquidate` → same oracle loop → same revert.
6. Collateral is permanently frozen; debt persists with no resolution path.

The Certora `Reverts.spec` rules (`oracleRevertCausesWithdrawCollateralRevert`, `oracleRevertCausesLiquidateRevert`) confirm this propagation is real and unconditional, but those rules model a *temporarily* failing live oracle contract — not a permanently broken zero-address oracle baked in at market creation time with no admin override. [5](#0-4) 

### Impact Explanation
Borrowers who supply collateral and borrow in a market with `oracle = address(0)` have their collateral permanently frozen. `withdrawCollateral` always reverts (oracle revert propagates through `isHealthy`). `liquidate` also always reverts for the same reason. There is no admin function to change a market's `collateralParams` after creation, so the freeze is irrecoverable. This is a concrete fund-freeze impact.

### Likelihood Explanation
`touchMarket` is permissionless — any address can call it. Creating a market with `oracle = address(0)` requires no special privilege, no leaked key, and no governance action. The attacker only needs to front-run or independently create the market before victims interact with it. The condition is trivially reproducible and repeatable across any number of markets.

### Recommendation
Add a non-zero oracle address check inside the `touchMarket` validation loop:

```solidity
require(market.collateralParams[i].oracle != address(0), OracleIsZeroAddress());
```

Optionally also verify `market.collateralParams[i].oracle.code.length > 0` to reject non-contract oracles, consistent with how Morpho Blue validates oracle addresses at market creation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";
// ... standard test imports

contract ZeroOracleFreezeTest is Test {
    Midnight midnight;
    ERC20 loanToken;
    ERC20 collateralToken;

    function setUp() public {
        // deploy Midnight, tokens, fund actors
    }

    function testZeroOracleFreezesFunds() public {
        // 1. Create market with oracle = address(0)
        CollateralParams[] memory params = new CollateralParams[](1);
        params[0] = CollateralParams({
            token: address(collateralToken),
            lltv: 0.77e18,
            maxLif: maxLif(0.77e18, 0.25e18),
            oracle: address(0)          // <-- zero oracle
        });
        Market memory m;
        m.loanToken = address(loanToken);
        m.maturity = block.timestamp + 100;
        m.collateralParams = params;

        bytes32 id = midnight.touchMarket(m); // must succeed (no oracle check)

        // 2. Borrower supplies collateral and borrows
        deal(address(collateralToken), borrower, 1e18);
        vm.prank(borrower);
        midnight.supplyCollateral(m, 0, 1e18, borrower);
        // ... take() to create debt

        // 3. Assert withdrawCollateral reverts
        vm.prank(borrower);
        vm.expectRevert(); // ABI decode revert from address(0).price()
        midnight.withdrawCollateral(m, 0, 1, borrower, borrower);

        // 4. Assert liquidate also reverts
        vm.expectRevert();
        midnight.liquidate(m, 0, 0, 0, borrower, false, address(this), address(0), "");

        // 5. Assert collateral is still locked
        assertGt(midnight.collateral(id, borrower, 0), 0, "collateral frozen");
    }
}
```

Expected assertions: both `withdrawCollateral` and `liquidate` revert; borrower collateral balance remains non-zero with no recovery path.

/// If an activated collateral oracle reverts on price different than withdrawn collateral, withdrawCollateral reverts when the borrower has debt.
rule oracleRevertCausesWithdrawCollateralRevert(env e, Midnight.Market market, uint256 collateralIndex, uint256 assets, address onBehalf, address receiver, uint256 revertingCollateralIndex) {
    require singleRevertingOracle == market.collateralParams[revertingCollateralIndex].oracle, "oracle is reverting";
    require revertingCollateralIndex < 128, "clearBit produces a new bitmap whose summaryGetBit is unconstrained for indices >= 128";
    require revertingCollateralIndex != collateralIndex, "withdrawCollateral may clear the bit at collateralIndex before calling isHealthy";

    bytes32 id = summaryToId(market);
    uint128 bitmap = collateralBitmap(id, onBehalf);
    require summaryGetBit(bitmap, revertingCollateralIndex), "revertingCollateralIndex is activated";

    withdrawCollateral@withrevert(e, market, collateralIndex, assets, onBehalf, receiver);
    bool reverted = lastReverted;

    assert debtOf(id, onBehalf) > 0 => reverted;
}
