### Title
Zero-unit take on fully-consumed offer fires maker callback unconditionally, enabling repeated callback invocation — (`src/Midnight.sol`)

### Summary
`Midnight.take()` contains no guard preventing callback invocation when `units=0`. Because adding zero to `consumed[maker][group]` leaves it equal to `maxUnits`, the `require(newConsumed <= offer.maxUnits)` check passes on a fully-consumed offer, and both `buyerCallback`/`sellerCallback` branches fire unconditionally. An unprivileged taker can replay this call indefinitely, triggering the maker's callback each time with `units=0` and `assets=0`.

### Finding Description

**Exact code path — `src/Midnight.sol`:**

**Step 1 — Consumed check passes with `units=0` on an exhausted offer.** [1](#0-0) 

When `units=0`, `consumed[maker][group] += 0` leaves `newConsumed` equal to its prior value. If the offer was filled to exactly `maxUnits`, the check `require(newConsumed <= offer.maxUnits)` evaluates as `maxUnits <= maxUnits` — it passes. The Certora spec explicitly encodes this as expected behavior: [2](#0-1) 

The spec asserts only that `units == 0` if take does not revert on a fully-consumed offer; it does not assert that callbacks are suppressed.

**Step 2 — All position deltas are zero.**

With `units=0`, `buyerAssets=0`, `sellerAssets=0`, `buyerCreditIncrease=0`, `sellerCreditDecrease=0`, `sellerDebtIncrease=0`. No position state changes. [3](#0-2) 

**Step 3 — Callbacks fire unconditionally.** [4](#0-3) [5](#0-4) 

Neither branch has a `units > 0` guard. For a buy offer, `buyerCallback = offer.callback` (the maker's own callback contract); for a sell offer, `sellerCallback = offer.callback`. The callback is invoked with `units=0` and `assets=0`, but with the maker's original `offer.callbackData` intact.

**Step 4 — Repeatability.**

Because `consumed` is unchanged, the attacker can call `take(units=0)` an unbounded number of times against the same exhausted offer, triggering the maker's callback on every call.

**Attacker-controlled inputs:** `units=0`, valid (ratified) offer struct with `offer.callback != address(0)` and `consumed[maker][group] == maxUnits`.

**Why existing checks fail:**
- The `require(newConsumed <= offer.maxUnits)` check is the only consumed-cap guard; it passes when `units=0`.
- There is no `require(units > 0)` or `if (units == 0) return` early exit.
- The callback guard is purely `if (callback != address(0))` with no amount check.

### Impact Explanation

A maker's callback contract that acts on `callbackData` (e.g., supplies a fixed collateral amount, transfers tokens, or modifies external state) can be triggered repeatedly after the offer is fully consumed. Concretely, a `BorrowCallback`-style contract that calls `supplyCollateral` with an amount encoded in `callbackData` would drain its own token balance on each spurious invocation. More broadly, any callback that does not internally guard against `units=0` is exposed to unbounded replay by any unprivileged taker who holds the ratified offer.

### Likelihood Explanation

**Preconditions:**
1. A live, ratified offer with `offer.callback != address(0)` exists.
2. `consumed[maker][group] == maxUnits` (offer fully filled — a normal terminal state).
3. The callback contract performs a meaningful action when called with `units=0` (does not self-guard).

Conditions 1 and 2 are routine end-states for any callback-bearing offer. Condition 3 depends on callback design, but the protocol provides no protection and no documentation warning. The attack is permissionless, costs only gas, and is infinitely repeatable.

### Recommendation

Add an early return (or revert) when `units=0` before any callback is invoked. The cleanest fix is a guard immediately before the callback block:

```solidity
// In take(), before the callback section:
if (units == 0) return (0, 0);
```

Alternatively, wrap each callback branch with `if (units > 0 && callback != address(0))`. This preserves the existing no-op take behavior for ordering/ratifier purposes while ensuring callbacks are never fired on zero-unit fills.

### Proof of Concept

```solidity
// Foundry unit test
contract DrainCallback is ISellCallback {
    uint256 public callCount;
    address token;
    address recipient;

    constructor(address _token, address _recipient) {
        token = _token; recipient = _recipient;
    }

    function onSell(bytes32, Market memory, uint256, uint256, uint256, address, address, bytes memory data)
        external returns (bytes32)
    {
        callCount++;
        uint256 amount = abi.decode(data, (uint256));
        // transfers fixed amount regardless of units passed
        ERC20(token).transfer(recipient, amount);
        return CALLBACK_SUCCESS;
    }
}

function testZeroUnitCallbackReplay() public {
    // 1. Set up offer with DrainCallback, maxUnits = 100e18
    DrainCallback cb = new DrainCallback(address(loanToken), maker);
    deal(address(loanToken), address(cb), 1000e18); // funded callback

    lenderOffer.callback = address(cb);
    lenderOffer.callbackData = abi.encode(10e18); // 10 tokens per call
    lenderOffer.maxUnits = 100e18;

    // 2. Fill offer to maxUnits
    take(100e18, borrower, lenderOffer);
    assertEq(midnight.consumed(lenderOffer.maker, lenderOffer.group), 100e18);

    uint256 balanceBefore = ERC20(loanToken).balanceOf(address(cb));

    // 3. Attacker calls take(units=0) repeatedly on exhausted offer
    for (uint i = 0; i < 5; i++) {
        vm.prank(attacker);
        midnight.take(lenderOffer, hex"", 0, attacker, attacker, address(0), hex"");
    }

    // 4. Assert callback fired 5 extra times, draining callback contract
    assertEq(cb.callCount(), 6); // 1 real + 5 replays
    assertEq(ERC20(loanToken).balanceOf(address(cb)), balanceBefore - 5 * 10e18);
    // consumed unchanged
    assertEq(midnight.consumed(lenderOffer.maker, lenderOffer.group), 100e18);
}
```

**Expected assertions:** `callCount == 6`, callback balance drained by `5 × 10e18`, `consumed` unchanged at `maxUnits`.

/// A fully-consumed offer in units mode only allows no-op takes.
rule fullyConsumedOfferRevertsOnNonTrivialTake(env e, Midnight.Offer offer, bytes ratifierData, uint256 units, address taker, address receiver, address takerCallback, bytes takerCallbackData) {
    require offer.maxAssets == 0;

    uint256 consumedBefore = consumed(offer.maker, offer.group);

    require offer.maxUnits > 0 && consumedBefore >= offer.maxUnits, "assume the offer is fully consumed";

    take(e, offer, ratifierData, units, taker, receiver, takerCallback, takerCallbackData);

    // If take does not revert, its input has to be zero.
    assert units == 0;
}
