### Title
`mulDivDown` uint256 overflow in `isHealthy` and `liquidate` blocks health-dependent operations - (File: src/libraries/UtilsLib.sol)

### Summary
`UtilsLib.mulDivDown` is implemented as the bare expression `(x * y) / d` with no overflow guard. When `collateral * price` exceeds `type(uint256).max`, Solidity 0.8.x's checked arithmetic causes a panic revert. Because both `isHealthy` and `liquidate` call `mulDivDown(collateral, price, ORACLE_PRICE_SCALE)` directly, any position whose collateral amount times oracle price overflows uint256 permanently blocks withdrawals and liquidations.

### Finding Description

**Root cause — `UtilsLib.mulDivDown`:**

```solidity
// src/libraries/UtilsLib.sol L29-31
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;   // plain checked multiplication — overflows revert
}
```

**Call site 1 — `isHealthy` (src/Midnight.sol L954):**

```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
               .mulDivDown(collateralParam.lltv, WAD);
```

`_position.collateral[i]` is `uint128` (max `2^128 − 1`). `price` is an unconstrained `uint256` returned by the oracle. No bounds check on `price` exists anywhere in the protocol.

**Call site 2 — `liquidate` (src/Midnight.sol L613, L615):**

```solidity
maxDebt += _collateral.mulDivDown(price, ORACLE_PRICE_SCALE).mulDivDown(_collateralParam.lltv, WAD);
badDebt = badDebt.zeroFloorSub(
    _collateral.mulDivUp(price, ORACLE_PRICE_SCALE).mulDivUp(WAD, _collateralParam.maxLif)
);
```

`liquidate` performs the same unchecked multiplication independently of `isHealthy`.

**Overflow threshold:**

With `collateral = type(uint128).max = 2^128 − 1`, overflow occurs when:

```
price > (2^256 − 1) / (2^128 − 1) = 2^128 + 1
```

i.e., `price ≥ 2^128 + 2 ≈ 3.4 × 10^38`.

*(Note: the prompt's specific example of `price = 2^128` does NOT overflow — `(2^128−1)·2^128 = 2^256−2^128 < type(uint256).max`. The actual threshold is `price ≥ 2^128 + 2`.)*

**Realistic trigger — WBTC (8 dec) / USDC (6 dec):**

```
oracle price = 1000 × ORACLE_PRICE_SCALE = 1000 × 1e36 = 1e39
```

`1e39 > 2^128 + 2 ≈ 3.4e38`. With `collateral = type(uint128).max`:

```
(2^128 − 1) × 1e39 ≈ 3.4e77  >>  type(uint256).max ≈ 1.16e77  → OVERFLOW
```

**Exploit flow:**

1. Market creator deploys a market with a legitimate oracle that returns `price ≥ 2^128 + 2` (e.g., `1e39` for WBTC/USDC).
2. Borrower calls `supplyCollateral` with `assets` large enough that `collateral × price > type(uint256).max`. `supplyCollateral` stores collateral as `uint128` via `toUint128` and does not call the oracle — no revert here.
3. Borrower takes a loan (debt > 0), activating the oracle path in `isHealthy`.
4. Any subsequent call to `isHealthy`, `withdrawCollateral`, or `liquidate` hits `mulDivDown(collateral, price, ORACLE_PRICE_SCALE)`, which panics with `arithmeticError`.

**Existing protections — insufficient:**

- `toUint128` caps collateral at `type(uint128).max` but does not prevent the product from overflowing uint256.
- Market creation validates `lltv`, `maxLif`, collateral sorting, and count — but imposes no constraint on oracle return values.
- The Certora specs replace `mulDivDown` with ghost summaries that do not model overflow, so formal verification does not catch this path.

### Impact Explanation

Any borrower whose `collateral[i] × oracle_price > type(uint256).max` has their position permanently bricked:

- `withdrawCollateral` reverts (calls `isHealthy` at L568).
- `liquidate` reverts directly at L613/L615 before the liquidatability check.
- `isHealthy` itself reverts, making the position appear neither healthy nor unhealthy.

This violates the core invariants that healthy positions are not liquidatable and unhealthy positions remain liquidatable, and that collateral cannot be withdrawn outside health rules.

### Likelihood Explanation

**Preconditions:**

1. Oracle returns `price ≥ 2^128 + 2` — achievable with any token pair where the collateral unit is worth ≥ ~340 loan token units (e.g., WBTC/USDC, ETH/USDC with standard decimal configurations).
2. Borrower supplies collateral such that `collateral × price > type(uint256).max` — with `price = 1e39`, the threshold is `collateral > type(uint256).max / 1e39 ≈ 1.16e38`, which is below `type(uint128).max ≈ 3.4e38`.

Both conditions are reachable by unprivileged actors (market creator + borrower) using entirely legitimate inputs. The state is permanent once the position is created with debt.

### Recommendation

Replace the naive multiplication in `mulDivDown` and `mulDivUp` with a 512-bit intermediate representation (e.g., Solmate's `FullMath.mulDiv` or OpenZeppelin's `Math.mulDiv`) that avoids intermediate overflow:

```solidity
// src/libraries/UtilsLib.sol
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return Math.mulDiv(x, y, d); // OZ Math — uses 512-bit intermediate
}
```

Alternatively, add an explicit pre-check:

```solidity
require(y == 0 || x <= type(uint256).max / y, "overflow");
return (x * y) / d;
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";
import {ORACLE_PRICE_SCALE, WAD, maxLif, LIQUIDATION_CURSOR_LOW} from "src/libraries/ConstantsLib.sol";

contract OverflowOracle {
    // price = 1e39 > 2^128+2; realistic for WBTC/USDC
    function price() external pure returns (uint256) { return 1e39; }
}

contract MulDivOverflowTest is Test {
    Midnight midnight;
    OverflowOracle oracle;
    ERC20 loanToken;
    ERC20 collateralToken;
    Market market;
    bytes32 id;

    function setUp() public {
        midnight = new Midnight();
        oracle = new OverflowOracle();
        loanToken = new ERC20("Loan", "LOAN");
        collateralToken = new ERC20("Collateral", "COL");

        CollateralParams[] memory params = new CollateralParams[](1);
        uint256 lltv = 0.77e18;
        params[0] = CollateralParams({
            token: address(collateralToken),
            lltv: lltv,
            maxLif: maxLif(lltv, LIQUIDATION_CURSOR_LOW),
            oracle: address(oracle)
        });
        market.loanToken = address(loanToken);
        market.maturity = block.timestamp + 365 days;
        market.collateralParams = params;

        id = midnight.touchMarket(market);
    }

    function testIsHealthyRevertsOnOverflow() public {
        // Supply collateral large enough that collateral * price overflows uint256.
        // price = 1e39, overflow threshold: collateral > type(uint256).max / 1e39 ≈ 1.16e38
        // type(uint128).max ≈ 3.4e38 > 1.16e38 → overflow
        uint128 collateralAmount = type(uint128).max;

        deal(address(collateralToken), address(this), collateralAmount);
        collateralToken.approve(address(midnight), collateralAmount);
        midnight.supplyCollateral(market, 0, collateralAmount, address(this));

        // Give the borrower some debt so isHealthy queries the oracle.
        // (setup lender offer + take omitted for brevity; assume debt > 0)
        // Assert: isHealthy reverts with arithmetic overflow
        vm.expectRevert(stdError.arithmeticError);
        midnight.isHealthy(market, id, address(this));

        // Assert: withdrawCollateral also reverts (calls isHealthy internally)
        vm.expectRevert(stdError.arithmeticError);
        midnight.withdrawCollateral(market, 0, 1, address(this), address(this));

        // Assert: liquidate also reverts (calls mulDivDown directly at L613)
        vm.expectRevert(stdError.arithmeticError);
        midnight.liquidate(market, 0, 1, 0, address(this), false, address(this), address(0), "");
    }
}
```

**Expected assertions:**
- `isHealthy` reverts with `Panic(0x11)` (arithmetic overflow).
- `withdrawCollateral` reverts with `Panic(0x11)`.
- `liquidate` reverts with `Panic(0x11)`.
- Position is permanently unwithdrawable and unliquidatable.
