### Title
Front-running `setDefaultSettlementFee` via permissionless `touchMarket` snapshots stale lower fee into new market - (`src/Midnight.sol`)

### Summary
`touchMarket` is permissionless and snapshots `defaultSettlementFeeCbp[loanToken]` into the new market's `settlementFeeCbp0..6` at creation time with no access control or atomicity guarantee. An attacker who observes a pending `setDefaultSettlementFee` transaction in the mempool can front-run it by calling `touchMarket` first, causing the market to be permanently initialized with the old (lower) fee. All subsequent `take` calls in that market then pay a lower settlement fee than the feeSetter intended.

### Finding Description
**Code path:**

`touchMarket` (`src/Midnight.sol:755-791`) is a public, permissionless function. When `marketState[id].tickSpacing == 0` (market not yet created), it reads the current `defaultSettlementFeeCbp[market.loanToken]` array and copies it verbatim into the new market's state:

```solidity
uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
_marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
// ... through index 6
```

`setDefaultSettlementFee` (`src/Midnight.sol:277-285`) is feeSetter-only and updates `defaultSettlementFeeCbp[loanToken][index]` immediately with no timelock. It does **not** retroactively update any already-created market's `settlementFeeCbp*` fields.

`take` (`src/Midnight.sol:347, 360`) calls `touchMarket` internally and then reads `settlementFee(id, timeToMaturity)` which interpolates from the market's snapshotted `settlementFeeCbp*` values â€” not from `defaultSettlementFeeCbp`.

**Exploit flow:**

1. feeSetter broadcasts `setDefaultSettlementFee(loanToken, index, higherFee)` to the mempool.
2. Attacker observes the pending transaction and broadcasts `touchMarket(market)` (where `market.loanToken == loanToken`) with higher gas priority.
3. Attacker's `touchMarket` mines first: market is created with `settlementFeeCbp* = old_lower_fee`.
4. feeSetter's `setDefaultSettlementFee` mines: `defaultSettlementFeeCbp[loanToken][index]` is updated to `higherFee`, but the already-created market is unaffected.
5. All `take` calls in that market use `old_lower_fee` for settlement fee computation, paying less than the feeSetter intended.

**Why existing checks fail:**

- No access control on `touchMarket` â€” any address can create any valid market.
- No atomicity mechanism between fee updates and market creation.
- The Certora rule `newMarketSettlementFeesMatchDefault` only asserts fees match at the instant of creation; it does not prevent the race condition.
- The feeSetter can remediate via `setMarketSettlementFee`, but cannot prevent the initial exploit window.

### Impact Explanation
The created market permanently holds a lower settlement fee than the current protocol default. Every `take` in that market accrues less `claimableSettlementFee` to the protocol than intended. The state divergence between `defaultSettlementFeeCbp[loanToken][*]` and `marketState[id].settlementFeeCbp*` is concrete and measurable. The attacker (as taker or maker) benefits from reduced fee cost on all trades in that market until the feeSetter manually intervenes with `setMarketSettlementFee`.

### Likelihood Explanation
Preconditions: (1) feeSetter broadcasts a `setDefaultSettlementFee` transaction raising fees for a loanToken; (2) attacker has mempool visibility (standard MEV infrastructure). Both are realistic on any public EVM chain. The attack is repeatable: for every fee increase the feeSetter attempts, the attacker can front-run with a new market (different maturity or collateral params yields a different `id`). The attacker controls all inputs to `touchMarket` and needs no special role or capital beyond gas.

### Recommendation
The feeSetter should atomically combine the fee update and any desired market creation/update using `multicall`:

```solidity
bytes[] memory calls = new bytes[](2);
calls[0] = abi.encodeCall(IMidnight.setDefaultSettlementFee, (loanToken, index, newFee));
calls[1] = abi.encodeCall(IMidnight.setMarketSettlementFee, (id, index, newFee));
midnight.multicall(calls);
```

Alternatively, add a `require(msg.sender == feeSetter || marketState[id].tickSpacing > 0)` guard so that only the feeSetter can create a market that does not yet exist, or introduce a minimum fee floor that prevents the snapshotted fee from being meaningfully lower than the intended new default.

### Proof of Concept
```solidity
// Foundry unit test
function testFrontRunSetDefaultSettlementFee() public {
    address loanToken = address(someToken);
    uint256 oldFee = 1e12;   // 1 CBP
    uint256 newFee = 100e12; // 100 CBP

    // feeSetter sets initial default fee
    vm.prank(feeSetter);
    midnight.setDefaultSettlementFee(loanToken, 1, oldFee);

    // feeSetter queues a fee increase (pending in mempool)
    // Attacker front-runs: creates market BEFORE setDefaultSettlementFee mines
    Market memory market = buildValidMarket(loanToken);
    bytes32 id = midnight.touchMarket(market); // attacker call

    // feeSetter's tx mines after
    vm.prank(feeSetter);
    midnight.setDefaultSettlementFee(loanToken, 1, newFee);

    // Assert: market has old lower fee, default has new higher fee
    assertEq(midnight.defaultSettlementFeeCbp(loanToken, 1), uint16(newFee / 1e12));
    // Market still has old fee â€” divergence confirmed
    (, , , , , uint16 cbp1, , , , , , , ) = midnight.marketState(id);
    assertEq(cbp1, uint16(oldFee / 1e12));
    assertNotEq(cbp1, uint16(newFee / 1e12)); // invariant violated

    // All takes in this market pay oldFee, not newFee
    // settlementFee(id, 1 days) == oldFee, not newFee
    assertEq(midnight.settlementFee(id, 1 days), oldFee);
}
```

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;
            uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];
            _marketState.settlementFeeCbp3 = _defaultSettlementFeeCbp[3];
            _marketState.settlementFeeCbp4 = _defaultSettlementFeeCbp[4];
            _marketState.settlementFeeCbp5 = _defaultSettlementFeeCbp[5];
            _marketState.settlementFeeCbp6 = _defaultSettlementFeeCbp[6];
            _marketState.continuousFee = defaultContinuousFee[market.loanToken];
            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);
        }
        return id;
    }
