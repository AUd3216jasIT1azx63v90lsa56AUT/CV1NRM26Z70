### Title
`abi.decode` Silently Ignores Trailing Bytes in `ratifierData`, Accepting Malformed Input - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
`EcrecoverRatifier.isRatified` decodes `ratifierData` with `abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]))`. Solidity's ABI decoder does not revert when the input bytes contain trailing data beyond the encoded tuple; it silently stops reading after the expected types. A taker can therefore append arbitrary bytes to a valid `ratifierData` blob, pass it through `midnight.take`, and `isRatified` will return `CALLBACK_SUCCESS` as if the data were well-formed.

### Finding Description
In `EcrecoverRatifier.isRatified` (lines 35–36), `ratifierData` is decoded without any length validation:

```solidity
(Signature memory sig, bytes32 root, uint256 leafIndex, bytes32[] memory proof) =
    abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]));
``` [1](#0-0) 

Solidity's `abi.decode` reads exactly as many bytes as the declared types require and ignores any remainder. There is no check that `ratifierData.length` equals the expected encoded size. The only gate before this decode is `require(msg.sender == MIDNIGHT, NotMidnight())` (line 34), which is satisfied because `isRatified` is called by the Midnight contract during `take`. [2](#0-1) 

**Exploit flow:**
1. Taker constructs valid `ratifierData = abi.encode(sig, root, leafIndex, proof)`.
2. Taker appends arbitrary bytes: `ratifierData = bytes.concat(validData, hex"deadbeef...")`.
3. Taker calls `midnight.take(offer, ratifierData, ...)`.
4. Midnight calls `ecrecoverRatifier.isRatified(offer, ratifierData)`.
5. `abi.decode` decodes the valid prefix, ignores the trailing bytes, all subsequent checks pass, and `CALLBACK_SUCCESS` is returned.

No existing check prevents this. The `msg.sender == MIDNIGHT` guard is satisfied by the normal call path, and all cryptographic checks (Merkle proof, ecrecover, root-canceled) operate on the correctly decoded values, which are unaffected by the trailing bytes. [3](#0-2) 

### Impact Explanation
Malformed `ratifierData` (with arbitrary trailing bytes) is silently accepted and treated identically to well-formed data. No funds are at risk, no position state changes, and no invariant is broken. The impact is purely a behavior divergence: the contract's input validation is weaker than its interface implies, accepting data that is not a strict ABI encoding of `(Signature, bytes32, uint256, bytes32[])`.

### Likelihood Explanation
Trivially reachable by any taker on any call to `midnight.take` that uses `EcrecoverRatifier`. No special state is required; the taker only needs a valid `ratifierData` to begin with (which they must have anyway to fill the offer). Appending trailing bytes costs nothing and requires no privilege.

### Recommendation
Add an explicit length check before decoding. Compute the expected encoded length from the proof array length and assert equality:

```solidity
// After decoding, or before:
// Minimum: 3*32 (sig v padded, r, s) + 32 (root) + 32 (leafIndex) + 32 (proof offset) + 32 (proof length) + proof.length*32
// Simpler: re-encode and compare lengths, or check ratifierData.length == expected.
require(ratifierData.length == /* expected size */, InvalidRatifierData());
```

Alternatively, decode and then verify `ratifierData.length == abi.encode(sig, root, leafIndex, proof).length` to reject any trailing bytes.

### Proof of Concept
```solidity
function testTrailingBytesAccepted() public {
    Offer memory offer = makeOffer(lender);
    bytes32 _root = HashLib.hashOffer(offer);
    bytes memory validData = buildRatifierData(_root, lender);

    // Append arbitrary trailing bytes
    bytes memory malformedData = bytes.concat(validData, hex"deadbeefcafebabe");

    vm.prank(address(midnight));
    bytes32 result = ecrecoverRatifier.isRatified(offer, malformedData);

    // Assert: trailing bytes are silently ignored, call succeeds
    assertEq(result, CALLBACK_SUCCESS);
    // Assert: lengths differ (malformed input was accepted)
    assertGt(malformedData.length, validData.length);
}
```

Expected: test passes, confirming `CALLBACK_SUCCESS` is returned for malformed input with trailing bytes. [4](#0-3)
