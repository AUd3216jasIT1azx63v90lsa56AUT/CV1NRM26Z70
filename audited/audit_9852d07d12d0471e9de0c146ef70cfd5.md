### Title
Authorized operator can revoke victim's critical operator authorization via EcrecoverAuthorizer delegation - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is already authorized by the `authorization.authorizer` on Midnight, not only from the authorizer themselves. This means any authorized operator can sign and submit an `Authorization` struct that revokes another operator's authorization on behalf of the victim, without the victim's knowledge or consent. Near market maturity, this can be used to strip a victim's automated repay-operator of its authorization, blocking timely debt repayment and potentially triggering post-maturity liquidation.

### Finding Description
The authorization check in `EcrecoverAuthorizer.setIsAuthorized` at line 33–36 is:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The second branch `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` allows any address that the victim has previously authorized on Midnight to act as the signer for an `Authorization` struct where `authorizer = victim`. This means the signer does not need to be the victim; any of the victim's existing authorized operators qualifies.

**Exploit flow:**

1. **Preconditions:** `victim` has authorized both `operator` and `repayOperator` on Midnight (`isAuthorized[victim][operator] = true`, `isAuthorized[victim][repayOperator] = true`). `victim` has an active debt position with `market.maturity = block.timestamp + 1 hour`. `victim` relies on `repayOperator` (e.g., an automated keeper) to call `Midnight.repay` on their behalf.

2. **Attacker action:** `operator` constructs `Authorization{authorizer=victim, authorized=repayOperator, isAuthorized=false, nonce=N}` and signs it with their own private key.

3. **Trigger:** `operator` (or anyone) calls `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`. The deadline and nonce checks pass. `ecrecover` returns `operator`. The check at line 34 evaluates `IMidnight(MIDNIGHT).isAuthorized(victim, operator) == true`, so it passes. The call proceeds to `IMidnight(MIDNIGHT).setIsAuthorized(repayOperator, false, victim)`.

4. **Bad state:** `isAuthorized[victim][repayOperator]` is now `false`. Any subsequent call by `repayOperator` to `Midnight.repay(market, units, victim, ...)` will hit the `Unauthorized` check inside `repay` and revert.

The root cause is that the `isAuthorized` delegation branch in `EcrecoverAuthorizer` conflates "authorized to act on behalf of the user for protocol operations" with "authorized to modify the user's authorization registry." These are distinct privilege levels, but the contract treats them identically. [1](#0-0) [2](#0-1) 

### Impact Explanation
An authorized operator can permanently (until the victim re-authorizes) revoke any other operator's authorization on behalf of the victim. Timed near market maturity, this blocks the victim's automated repay-operator from calling `Midnight.repay`, leaving the debt unrepaid past maturity and exposing the victim to post-maturity liquidation of their collateral. The victim has no on-chain recourse within the same transaction; they must detect the revocation and re-authorize before the maturity boundary passes.

### Likelihood Explanation
**Preconditions:** The victim must have authorized at least two addresses on Midnight — one malicious/compromised operator and one critical repay-operator. This is a realistic setup for any user relying on keeper bots or automated position managers. The attack requires no special privileges beyond being an authorized operator, is executable in a single transaction, and is repeatable (the attacker can keep revoking re-authorizations as long as they remain authorized). The only mitigation available to the victim is to also revoke the attacker's authorization, but this is a race condition near maturity.

### Recommendation
Remove the delegated-signer branch from `EcrecoverAuthorizer.setIsAuthorized`. Authorization registry changes are a root-level privilege that should require the authorizer's own signature, not a delegated one. The fix is:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegated signing is intentionally desired for other use cases, introduce a separate, narrower permission (e.g., a dedicated `canManageAuthorizations` flag) rather than reusing the general `isAuthorized` mapping. [3](#0-2) 

### Proof of Concept
```solidity
// Foundry unit test
function testOperatorRevokesRepayOperator() public {
    // Setup: victim authorizes both operator and repayOperator
    vm.prank(victim);
    midnight.setIsAuthorized(operator, true, victim);
    vm.prank(victim);
    midnight.setIsAuthorized(repayOperator, true, victim);
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);

    // Setup: victim has active debt near maturity
    // (collateralize, take offer, skip to maturity - 1 hour)
    vm.warp(market.maturity - 1 hours);

    // Attacker (operator) signs revocation of repayOperator on behalf of victim
    Authorization memory auth = Authorization({
        authorizer: victim,
        authorized: repayOperator,
        isAuthorized: false,
        nonce: ecrecoverAuthorizer.nonce(victim),
        deadline: block.timestamp + 1
    });
    // operator signs — NOT victim
    Signature memory sig = signAuthorization(auth, operator /*uses operator's private key*/);

    // Anyone submits — passes because isAuthorized[victim][operator] == true
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);

    // Assert: repayOperator is now deauthorized
    assertEq(midnight.isAuthorized(victim, repayOperator), false);

    // Assert: repayOperator's repay call reverts Unauthorized
    deal(address(loanToken), repayOperator, debtAmount);
    vm.prank(repayOperator);
    vm.expectRevert(IMidnight.Unauthorized.selector);
    midnight.repay(market, debtAmount, victim, address(0), hex"");
}
```

Expected assertions: `isAuthorized[victim][repayOperator] == false` after the operator's submission, and `repay` called by `repayOperator` reverts with `Unauthorized`. [4](#0-3) [5](#0-4)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
