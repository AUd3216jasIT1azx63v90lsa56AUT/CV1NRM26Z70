### Title
Zero-Price Oracle Allows Liquidator to Seize Borrower Collateral for Zero Loan Token Repayment - (File: src/libraries/SafeTransferLib.sol)

### Summary

When the oracle for the liquidated collateral returns `0`, calling `liquidate` with `seizedAssets > 0` and `repaidUnits = 0` causes `repaidUnits` to be computed as `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif) = 0`. The protocol then transfers the full `seizedAssets` collateral to the liquidator while calling `safeTransferFrom(loanToken, payer, address(this), 0)`, which transfers nothing. The borrower's collateral is seized for free, violating the invariant that every non-zero collateral seizure must require proportional loan token repayment.

### Finding Description

**Exact code path** in `src/Midnight.sol`:

**Step 1 — Input validation passes.** [1](#0-0) 
`atMostOneNonZero(0, seizedAssets)` is `true` (repaidUnits=0, seizedAssets>0). `_position.debt > 0` is satisfied.

**Step 2 — Oracle returns 0; `liquidatedCollatPrice = 0`.** [2](#0-1) 
With `price = 0`: `maxDebt += collateral.mulDivDown(0, ORACLE_PRICE_SCALE).mulDivDown(lltv, WAD) = 0`.

**Step 3 — Liquidatability check passes.** [3](#0-2) 
`originalDebt > maxDebt` → `originalDebt > 0` → `true`. Position is liquidatable.

**Step 4 — All debt becomes bad debt.** [4](#0-3) 
`badDebt = originalDebt.zeroFloorSub(collateral.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, maxLif))`. Since `mulDivUp(x, 0, ORACLE_PRICE_SCALE) = (x*0 + (1e36-1))/1e36 = 0`, `badDebt = originalDebt`. `_position.debt` is zeroed at line 628.

**Step 5 — `seizedAssets > 0` branch entered; `repaidUnits` computed as 0.** [5](#0-4) 
`repaidUnits = seizedAssets.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif)`. First call: `(seizedAssets * 0 + (1e36-1)) / 1e36 = 0`. Second call: `(0 * WAD + (lif-1)) / lif = 0`. So `repaidUnits = 0`.

**Step 6 — Recovery close factor check passes.** [6](#0-5) 
`_position.debt` is now `0` (zeroed by bad debt). `maxRepaid = (0 - 0).mulDivUp(...) = 0`. `repaidUnits <= maxRepaid` → `0 <= 0` → passes.

**Step 7 — Collateral seized, zero loan tokens transferred.** [7](#0-6) 
`safeTransfer(collateralToken, receiver, seizedAssets)` sends real collateral to the liquidator. `safeTransferFrom(loanToken, payer, address(this), 0)` transfers nothing.

**`mulDivUp` implementation confirms zero result:** [8](#0-7) 

**The Certora `Reverts.spec` rule `oracleZeroCausesLiquidateWithRepaidRevert` only covers the `repaidUnits > 0` input path and explicitly does not cover `seizedAssets > 0`:** [9](#0-8) 

There is no guard that prevents `seizedAssets > 0` from proceeding when `liquidatedCollatPrice = 0`.

### Impact Explanation

An unprivileged liquidator seizes up to the borrower's full collateral balance at `collateralIndex` without paying any loan tokens. The lenders are simultaneously slashed via the bad-debt mechanism. The borrower suffers a direct, concrete loss of their collateral (which has real-world value despite the oracle returning 0) while receiving no debt relief beyond what the bad-debt mechanism already provides. This constitutes theft of borrower collateral and violates the core invariant that collateral cannot be seized outside health/liquidation rules with proper repayment.

### Likelihood Explanation

**Preconditions:**
1. The oracle for the liquidated collateral returns `0` (e.g., a stale/broken Chainlink feed, a custom oracle with a bug, or a price that rounds to 0 at the oracle's precision).
2. The borrower has a non-zero debt position with non-zero collateral at `collateralIndex`.
3. The attacker passes `seizedAssets` in `[1, _position.collateral[collateralIndex]]`.

The protocol's own test suite (`testFullBadDebtWithdrawCollateral`) and Certora specs explicitly model oracle-returns-0 as a valid, reachable scenario, not an impossible one. The attack is repeatable on any market whose oracle can return 0, is atomic (single transaction), and requires no special privileges.

### Recommendation

After computing `repaidUnits` from `seizedAssets` at line 650, add an explicit check:

```solidity
if (seizedAssets > 0) {
    repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
    require(repaidUnits > 0, ZeroPriceSeizure()); // new guard
}
```

This mirrors the existing Certora rule `oracleZeroCausesLiquidateWithRepaidRevert` (which already enforces revert for the `repaidUnits > 0` input path when price = 0) and closes the symmetric gap for the `seizedAssets > 0` input path. Alternatively, require `liquidatedCollatPrice > 0` before entering the non-zero seizure branch.

### Proof of Concept

```solidity
// Foundry fuzz test
function testFreeCollateralSeizureOnZeroOracle(uint128 seizedAssets) public {
    // Preconditions
    uint256 units = 100e18;
    seizedAssets = uint128(bound(seizedAssets, 1, type(uint128).max));

    collateralize(market, borrower, units);
    setupMarket(market, units);

    uint256 collateralBefore = midnight.collateral(id, borrower, 0);
    vm.assume(seizedAssets <= collateralBefore);

    // Oracle returns 0
    Oracle(market.collateralParams[0].oracle).setPrice(0);

    uint256 loanBalanceBefore = loanToken.balanceOf(address(midnight));

    // Liquidator calls with seizedAssets > 0, repaidUnits = 0
    (uint256 seized, uint256 repaid) =
        midnight.liquidate(market, 0, seizedAssets, 0, borrower, false, address(this), address(0), "");

    // Assertions
    assertEq(repaid, 0, "repaidUnits must be 0");
    assertGt(seized, 0, "collateral was seized");
    assertEq(loanToken.balanceOf(address(midnight)), loanBalanceBefore, "no loan tokens paid");
    assertEq(midnight.collateral(id, borrower, 0), collateralBefore - seized, "borrower lost collateral");
    // Liquidator received collateral for free — invariant violated
}
```

Expected: `repaid == 0`, `seized > 0`, `loanToken` balance of protocol unchanged, borrower collateral reduced. This directly demonstrates the invariant violation.

/// If liquidated collateral oracle returns 0 on price, liquidate with repaid input reverts.
rule oracleZeroCausesLiquidateWithRepaidRevert(env e, Midnight.Market market, uint256 collateralIndex, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, bool postMaturityMode) {
    require singleZeroOracle == market.collateralParams[collateralIndex].oracle, "oracle returns zero";
    require repaidUnits > 0, "using repaid units as input";

    liquidate@withrevert(e, market, collateralIndex, 0, repaidUnits, borrower, postMaturityMode, receiver, callback, data);

    assert lastReverted;
}
