### Title
`buyerAssetsToUnits` reverts with division-by-zero when `buyerPrice = 0` while `Midnight.take` succeeds - (File: src/periphery/TakeAmountsLib.sol)

### Summary
`TickLib.tickToPrice(0)` evaluates to exactly `0` due to the rounding step applied at the end of the formula. When a sell offer has `tick = 0` and `settlementFee = 0`, `buyerAssetsToUnits` computes `buyerPrice = 0`, passes the `require(buyerPrice <= WAD)` guard, and then panics with division-by-zero inside `targetBuyerAssets.mulDivDown(WAD, buyerPrice)`. `Midnight.take` with the same offer and `units = 1` succeeds without reverting, confirming the divergence.

### Finding Description

**`tickToPrice(0)` evaluates to 0:** [1](#0-0) 

With `tick = 0`:
- Exponent: `LN_ONE_PLUS_DELTA * (2910 - 0) ≈ 14.514e18`
- `wExp(14.514e18) ≈ 2.009e24`
- `1e36 / (1e18 + 2.009e24) ≈ 4.978e11`
- `divHalfDownUnchecked(4.978e11, 1e12) = (4.978e11 + 4.999e11) / 1e12 = 0` (integer division)
- Final: `0 * PRICE_ROUNDING_STEP = 0`

**`buyerAssetsToUnits` code path:** [2](#0-1) 

With `offer.buy = false`, `tick = 0`, `settlementFee = 0`:
1. `offerPrice = tickToPrice(0) = 0`
2. `sellerPrice = offerPrice = 0` (sell branch, no subtraction)
3. `buyerPrice = 0 + 0 = 0`
4. `require(0 <= WAD)` → **passes** (no zero-guard)
5. `targetBuyerAssets.mulDivDown(WAD, 0)` → **panic: division by zero**

**`Midnight.take` does not revert for the same offer with `units = 1`:** [3](#0-2) 

`buyerAssets = 1.mulDivUp(0, WAD) = 0`, `sellerAssets = 0` — no division, no revert.

**Bundler does NOT wrap `buyerAssetsToUnits` in try/catch:** [4](#0-3) 

The comment at line 175 explicitly states: *"Reverts if TakeAmountsLib or ConsumableUnitsLib reverts."* The try/catch only wraps the inner `IMidnight.take` call (lines 215–221), not the `buyerAssetsToUnits` call. A revert inside `buyerAssetsToUnits` propagates and aborts the entire `buyWithAssetsTargetAndWithdrawCollateral` transaction.

### Impact Explanation
Any lender calling `buyWithAssetsTargetAndWithdrawCollateral` with a `takes` array that includes a sell offer at `tick = 0` (and effective `settlementFee = 0`) will have their entire bundler transaction reverted. The lender's pre-pulled `targetBuyerAssets` tokens are returned (no permanent fund loss), but the bundler flow is completely DoS'd for that call. The lender cannot use the bundler to fill zero-price offers, even though `Midnight.take` accepts them.

### Likelihood Explanation
Preconditions: (1) a sell offer exists with `tick = 0` — a valid tick within `[0, MAX_TICK]`, always satisfying `0 % tickSpacing == 0`; (2) effective `settlementFee = 0` — possible when the protocol's default settlement fee is zero or the market-specific fee is zero. Both conditions are reachable without any privileged action. The borrower (maker of the sell offer) can create this offer permissionlessly. Any lender who then attempts to fill it via the bundler triggers the revert. Repeatability is unconditional given the preconditions.

### Recommendation
Add an explicit zero-guard for `buyerPrice` in `buyerAssetsToUnits` before the division:

```solidity
require(buyerPrice > 0, BuyerPriceIsZero());
```

This makes the revert explicit and meaningful rather than a panic, and aligns with the documented behavior that not all `targetBuyerAssets` values are reachable when `buyerPrice = 0`.

### Proof of Concept

```solidity
// Unit test: buyerAssetsToUnits reverts with tick=0, settlementFee=0
function testBuyerAssetsToUnitsZeroPriceReverts() public {
    // Ensure settlementFee = 0
    midnight.setDefaultSettlementFee(address(loanToken), 1, 0);
    midnight.touchMarket(market);
    midnight.setMarketTickSpacing(id, 1);

    offer.buy = false;
    offer.tick = 0; // tickToPrice(0) == 0

    // Assert buyerAssetsToUnits reverts (division by zero)
    vm.expectRevert(); // panic 0x12
    TakeAmountsLib.buyerAssetsToUnits(address(midnight), id, offer, 1);
}

// Contrast: Midnight.take with units=1 on the same offer does NOT revert
function testMidnightTakeZeroPriceDoesNotRevert() public {
    midnight.setDefaultSettlementFee(address(loanToken), 1, 0);
    midnight.touchMarket(market);
    midnight.setMarketTickSpacing(id, 1);

    offer.buy = false;
    offer.tick = 0;
    offer.maker = borrower;
    offer.receiverIfMakerIsSeller = borrower;

    collateralize(market, borrower, 1);
    (uint256 buyerAssets, uint256 sellerAssets) = take(1, lender, offer);

    // Both are 0, no revert
    assertEq(buyerAssets, 0);
    assertEq(sellerAssets, 0);
}
```

Expected assertions: first test reverts with arithmetic panic (division by zero); second test passes with `buyerAssets == 0` and `sellerAssets == 0`.
