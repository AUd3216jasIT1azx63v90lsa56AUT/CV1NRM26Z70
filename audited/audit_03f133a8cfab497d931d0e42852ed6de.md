### Title
Multi-collateral `mulDivUp` accumulation in `badDebt` loop suppresses bad debt realization for genuinely insolvent positions - (File: src/Midnight.sol)

### Summary
In `liquidate()`, `badDebt` is computed by iteratively subtracting each collateral's rounded-up value (`mulDivUp(price, ORACLE_PRICE_SCALE).mulDivUp(WAD, maxLif)`) from `originalDebt` via `zeroFloorSub`. With N activated collaterals, the composed double-`mulDivUp` introduces up to 2 units of upward rounding error per collateral. When the sum of these rounded-up values meets or exceeds `originalDebt`, `badDebt` floors to 0 even though the true collateral value at `maxLif` is strictly less than `originalDebt`, violating the invariant that bad debt must be realized whenever the position is genuinely insolvent.

### Finding Description

**Code path:** `src/Midnight.sol` lines 604–618.


**Root cause — rounding error bound per collateral:**

`mulDivUp(x, y, d)` is implemented as `(x*y + d - 1) / d`, so it equals `⌈x·y/d⌉ ≤ x·y/d + 1`.

For collateral `i`, the rounded-up value is:
```
v_i = mulDivUp(mulDivUp(c_i, p_i, ORACLE_PRICE_SCALE), WAD, maxLif_i)
```

Let `x_i = mulDivUp(c_i, p_i, ORACLE_PRICE_SCALE)`. Then:
- `x_i ≤ c_i·p_i / ORACLE_PRICE_SCALE + 1`
- `v_i ≤ x_i·WAD / maxLif_i + 1 ≤ (c_i·p_i / ORACLE_PRICE_SCALE + 1)·WAD/maxLif_i + 1`

Since `maxLif_i ≥ WAD` (proven in `certora/specs/ExactMath.spec`, rule `maxLifIsAtLeastWad`), `WAD/maxLif_i ≤ 1`, so:
```
v_i ≤ true_value_i + WAD/maxLif_i + 1 ≤ true_value_i + 2
```

**Accumulation across N collaterals:**

`MAX_COLLATERALS_PER_BORROWER = 16` (enforced by `certora/specs/CollateralBitmap.spec`, invariant `atMostMaxCollateralsBitsSet`). Total rounding surplus:
```
Σ v_i ≤ Σ true_value_i + 2·N  (N ≤ 16, so surplus ≤ 32 units)
```

If `originalDebt − Σ true_value_i < 2·N` (i.e., true bad debt is between 1 and 32 units), then `Σ v_i ≥ originalDebt`, and the iterative `zeroFloorSub` chain floors `badDebt` to 0.

**Concrete minimal example (2 collaterals, lltv = WAD):**
- `collateral_0 = collateral_1 = 1`, `price_0 = price_1 = ORACLE_PRICE_SCALE / 2`, `maxLif = WAD`
- True value per collateral: `1 · (ORACLE_PRICE_SCALE/2) / ORACLE_PRICE_SCALE · WAD/WAD = 0.5`
- `mulDivUp(1, ORACLE_PRICE_SCALE/2, ORACLE_PRICE_SCALE) = ⌈0.5⌉ = 1`; `mulDivUp(1, WAD, WAD) = 1`
- `Σ v_i = 2`, `originalDebt = 2`, true bad debt = 1
- `badDebt = 2.zeroFloorSub(1).zeroFloorSub(1) = 0` — bad debt suppressed

**No existing protection:** There is no minimum bad debt threshold, no rounding correction, and no guard that checks whether `Σ v_i` overshot `originalDebt`. The `zeroFloorSub` silently absorbs the overshoot.

**Attacker-controlled inputs:** Any unprivileged liquidator can call `liquidate(market, collateralIndex, 0, 0, borrower, false, receiver, address(0), "")` on a position engineered to sit in the 1–32 unit bad-debt window. The borrower (or a colluding party) can engineer this by choosing collateral amounts and oracle prices near the insolvency boundary across multiple collateral slots.

### Impact Explanation

Bad debt in the range `[1, 2·N]` units (up to 32 units with 16 collaterals) is never realized: `_position.debt` is not decremented, `_marketState.lossFactor` is not updated, and `_marketState.totalUnits` is not reduced. Lenders are not compensated for the actual loss. If the collateral prices remain at the same level (or the position is never fully liquidated), this bad debt is permanently unaccounted for, violating the protocol invariant that `bad_debt_realized_during_liquidation = true` (per `live_context.json` line 146).

### Likelihood Explanation

**Preconditions:**
1. A market with ≥ 2 activated collaterals per borrower (reachable by any borrower via `supplyCollateral`).
2. Oracle prices that place the true bad debt in the window `[1, 2·N]` units — achievable by choosing small collateral amounts or prices just below the bad-debt threshold.
3. Any unprivileged liquidator calls `liquidate(..., 0, 0, ...)`.

The scenario is repeatable: as long as prices stay in the window, every subsequent `liquidate` call with `seizedAssets = repaidUnits = 0` will also return `badDebt = 0`. The window is narrow (≤ 32 units) but deterministically reachable with small-denomination tokens or dust collateral positions.

### Recommendation

Replace the iterative `zeroFloorSub` accumulation with an exact comparison: compute `Σ v_i` as a `uint256` sum first, then set `badDebt = originalDebt.zeroFloorSub(sum)` in a single operation. This does not change the rounding direction for any individual collateral but prevents the per-collateral rounding surplus from compounding across the loop and overshooting `originalDebt`. Alternatively, add a post-loop guard: if `Σ v_i > originalDebt` but `Σ true_value_i < originalDebt` (checked with `mulDivDown`), force `badDebt = 1` to ensure at least 1 unit of bad debt is always realized for genuinely insolvent positions.

```text
    function zeroFloorSub(uint256 x, uint256 y) internal pure returns (uint256 z) {
        assembly {
            z := mul(gt(x, y), sub(x, y))
        }
    }

    /// @dev Returns (x * y) / d rounded down.
    function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
        return (x * y) / d;
    }

    /// @dev Returns (x * y) / d rounded up.
    function mulDivUp(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
        return (x * y + (d - 1)) / d;
    }
```
