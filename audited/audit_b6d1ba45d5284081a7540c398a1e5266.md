### Title
Fee Accrual Rounds to Zero for Small Elapsed Times, Violating Nonzero-Fee Invariant - (File: src/Midnight.sol)

### Summary
The fee accrual formula at `src/Midnight.sol` line 815 computes `fee = floor(postSlashPendingFee * elapsed / (maturity - lastAccrual))` using plain integer division via `mulDivDown`. When `postSlashPendingFee * elapsed < (maturity - lastAccrual)`, the result is exactly zero, accruing no fee despite nonzero time having elapsed and nonzero `pendingFee`. A lender can trigger `updatePosition` with a 1-second elapsed window on a long-maturity market to cause each call to accrue zero fee, violating the invariant that any nonzero elapsed time must produce a nonzero fee when `pendingFee > 0`.

### Finding Description
**Code path:** `updatePosition()` → `_updatePosition()` → `updatePositionView()` at `src/Midnight.sol` lines 797–819.

**Root cause:** `mulDivDown` is `(x * y) / d` (plain Solidity integer division, rounds toward zero). At line 815:
```solidity
uint128 fee = _lastAccrual < market.maturity
    ? uint128(postSlashPendingFee.mulDivDown(accrualEnd - _lastAccrual, market.maturity - _lastAccrual))
    : 0;
```
When `postSlashPendingFee * (accrualEnd - _lastAccrual) < (market.maturity - _lastAccrual)`, the division truncates to zero.

**Concrete preconditions:**
- `postSlashPendingFee = P` (any value ≥ 1)
- `elapsed = accrualEnd - _lastAccrual = 1 second`
- `market.maturity - _lastAccrual = T` where `T > P`

For a 1-year market (`T = 31,536,000 s`) and `P < 31,536,000 wei`, every single-second call accrues `fee = 0`.

**Exploit flow:**
1. Lender holds a position with `pendingFee = P` (e.g., 1 wei) on a market with maturity 1 year out.
2. Lender (or anyone) calls `updatePosition(market, lender)` 1 second after `lastAccrual`.
3. `updatePositionView` computes `fee = floor(1 * 1 / 31,536,000) = 0`.
4. `_updatePosition` writes `lastAccrual = block.timestamp` and adds `0` to `continuousFeeCredit`.
5. Lender's credit and `pendingFee` are unchanged; the elapsed second is consumed with no fee accrual.
6. Repeating this every second for the full year results in zero total fee accrued, despite `pendingFee > 0` throughout.

**Why existing checks fail:** There is no guard requiring `fee > 0` when `postSlashPendingFee > 0` and `elapsed > 0`. The Certora specs (`ContinuousFee.spec`) verify proportionality using the same `mulDivDown` formula and do not assert a nonzero-fee invariant. The existing fuzz test `testAccrualPreMaturity` computes `expectedFee = remaining.mulDivDown(elapsed, ttm)` and accepts zero as a valid expected value. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation
Each call to `updatePosition` with a sufficiently small elapsed window accrues zero fee. The maximum rounding loss per call is strictly less than 1 wei (the true rational fee is less than 1 wei in the attack window, so the integer result is 0). Over N calls, the cumulative fee leakage is at most N wei. For a 1-year market with `pendingFee < 31,536,000 wei`, the entire `pendingFee` can be avoided if `updatePosition` is called every second for the full year, leaking up to `pendingFee` wei total to the lender at the expense of `continuousFeeCredit`. The scoped impact of "up to 1 wei per second per lender" is arithmetically correct and bounded.

### Likelihood Explanation
**Preconditions:** Market maturity must be long enough that `pendingFee * 1 < (maturity - lastAccrual)`. This is satisfied for any market with more than `pendingFee` seconds remaining. The lender must trigger `updatePosition` (directly or via `withdraw(0)`) with a 1-second elapsed window. `updatePosition` is a public, permissionless function callable by anyone. No special role or privileged access is required.

**Feasibility:** Technically reachable with no attacker-controlled inputs beyond timing. However, the attack is economically unviable: each call to `updatePosition` costs at minimum ~20,000 gas. At 1 gwei gas price, each call costs ~20,000,000,000 wei, while the maximum savings per call is 1 wei. The gas cost exceeds the savings by ~10 orders of magnitude, making deliberate exploitation unprofitable. The invariant violation is real but the practical leakage is bounded to incidental rounding loss when `updatePosition` is called naturally with small elapsed windows.

### Recommendation
Replace the fee accrual with a round-up formula to ensure at least 1 wei is accrued whenever `postSlashPendingFee > 0` and `elapsed > 0` and `_lastAccrual < market.maturity`:

```solidity
uint128 fee = _lastAccrual < market.maturity
    ? uint128(postSlashPendingFee.mulDivUp(accrualEnd - _lastAccrual, market.maturity - _lastAccrual))
    : 0;
```

Using `mulDivUp` ensures `fee ≥ 1` whenever the numerator is nonzero, preserving the invariant. This is consistent with the protocol's existing pattern of rounding in favor of the protocol (e.g., `pendingFee` slash uses `mulDivUp` at line 810 to reduce `pendingFee` by at least the slash fraction). // Foundry unit test
function testFeeRoundsToZeroForSmallElapsed() public {
    // Setup: lender with pendingFee = 1 wei, 1-year market
    uint256 ttm = 365 days; // 31,536,000 seconds
    uint256 credit = 1e18;
    uint256 feeRate = 1; // minimal rate
    setupLender(credit, feeRate, ttm);

    uint256 pendingFeeStart = midnight.pendingFee(id, lender);
    vm.assume(pendingFeeStart > 0 && pendingFeeStart < ttm);
    // pendingFeeStart * 1 < ttm => fee rounds to 0

    // Advance 1 second and call updatePosition
    vm.warp(block.timestamp + 1);
    midnight.updatePosition(market, lender);

    // ASSERTION: fee should be > 0 but is 0 — invariant violated
    assertEq(midnight.continuousFeeCredit(id), 0, "fee accrued is zero despite elapsed time");
    assertEq(midnight.pendingFee(id, lender), pendingFeeStart, "pendingFee unchanged");

    // Repeat for the full year: total accrued fee remains 0
    for (uint256 i = 0; i < ttm - 1; i++) {
        vm.warp(block.timestamp + 1);
        midnight.updatePosition(market, lender);
    }
    // Expected: continuousFeeCredit == pendingFeeStart; Actual: 0
    assertGt(midnight.continuousFeeCredit(id), 0, "should have accrued nonzero fee over full year");
}

// Fuzz variant
function testFuzzFeeNonzeroWhenPendingFeeNonzeroAndElapsed(

    pendingFee = bound(pendingFee, 1, 1e6); // small pendingFee
    ttm = bound(ttm, 2, 365 days);

    setupLenderWithPendingFee(pendingFee, ttm);
    vm.warp(block.timestamp + elapsed);
    midnight.updatePosition(market, lender);

    // Invariant: fee > 0 whenever pendingFee > 0 and elapsed > 0
    // This assertion FAILS under the current implementation
    assertGt(midnight.continuousFeeCredit(id), 0, "invariant: nonzero elapsed => nonzero fee");
}
