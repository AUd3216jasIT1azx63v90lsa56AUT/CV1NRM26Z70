### Title
Unchecked oracle price overflow in `liquidate` permanently DoS-es liquidation - (File: src/Midnight.sol)

### Summary
`UtilsLib.mulDivDown` uses plain Solidity 0.8.x checked multiplication (`x * y`), which reverts on overflow. Because `touchMarket` imposes no upper bound on oracle return values, an attacker can deploy a malicious oracle returning `type(uint256).max`, create a permissionless market with it, and cause every call to `liquidate` to revert permanently via arithmetic overflow in the `maxDebt` loop.

### Finding Description

**Root cause — `mulDivDown` reverts on overflow:** [1](#0-0) 

`x * y` is plain checked Solidity 0.8.x arithmetic. If `_collateral * price > type(uint256).max`, the entire transaction reverts.

**Overflow site in `liquidate`:** [2](#0-1) 

For every activated collateral slot, `_collateral.mulDivDown(price, ORACLE_PRICE_SCALE)` is called. `_collateral` is a `uint128` (max ≈ 3.4 × 10³⁸). `ORACLE_PRICE_SCALE = 1e36`. Overflow occurs when `price > type(uint256).max / _collateral`. With `_collateral = type(uint128).max`, overflow triggers at `price > 2^128` (≈ 340 × `ORACLE_PRICE_SCALE`). A malicious oracle returning `type(uint256).max` triggers it with any non-zero collateral.

**No oracle price validation in `touchMarket`:** [3](#0-2) 

`touchMarket` is permissionless and validates only `lltv`, `maxLif`, token ordering, and maturity. The oracle address is accepted without any price-range check. Any address — including a contract returning `type(uint256).max` — is accepted.

**Formal verification treats bounded prices as an assumption, not an invariant:** [4](#0-3) 

The Certora spec explicitly requires `price * max_uint128 + ORACLE_PRICE_SCALE - 1 <= max_uint256` as a precondition (`require`), confirming the protocol has no on-chain enforcement of this bound.

**Exploit flow:**
1. Attacker deploys `MaliciousOracle` whose `price()` returns `type(uint256).max`.
2. Attacker calls `touchMarket` with `collateralParams[0].oracle = address(MaliciousOracle)` — succeeds, no price check.
3. Attacker (or any user) supplies collateral and borrows, creating a position with `_collateral > 0`.
4. Any call to `liquidate` enters the `while (_collateralBitmap != 0)` loop, executes `_collateral.mulDivDown(type(uint256).max, ORACLE_PRICE_SCALE)`, which computes `_collateral * type(uint256).max` — overflows uint256 — and reverts.
5. The same overflow hits `mulDivUp` on line 615 for the `badDebt` path.
6. Liquidation is permanently blocked for the entire market.

### Impact Explanation

Liquidation is permanently DoS-ed for any market whose oracle returns a price large enough to overflow `uint256` when multiplied by any non-zero collateral balance. Lenders who supplied credit to such a market cannot recover funds via liquidation. Unhealthy positions cannot be closed, violating the core invariant that unhealthy positions remain liquidatable.

### Likelihood Explanation

Market creation is fully permissionless — no governance approval, no oracle whitelist. The attacker needs only to deploy a two-line oracle contract and call `touchMarket`. The attack is repeatable for any number of markets. The only social-engineering requirement is that lenders supply credit to the malicious market, which is feasible if the oracle initially returns a normal price and later switches to `type(uint256).max`.

### Recommendation

Add an oracle price sanity check at the point of use. Before computing `maxDebt`, verify that `price <= type(uint256).max / _collateral` (or equivalently, cap `price` at `type(uint256).max / type(uint128).max`). Alternatively, use a `mulDivDown` variant implemented in assembly with explicit overflow detection (returning 0 or saturating) rather than reverting, so that an extreme price is treated as "infinitely healthy" rather than a revert. A complementary defence is to validate the oracle's current price during `touchMarket` or `supplyCollateral`.

import {Midnight} from "src/Midnight.sol";

contract MaxPriceOracle {
    function price() external pure returns (uint256) {
        return type(uint256).max;
    }
}

contract OverflowLiquidateDoSTest is Test {
    Midnight midnight;
    MaxPriceOracle oracle;
    ERC20 loanToken;
    ERC20 collateralToken;

    function setUp() public {
        midnight = new Midnight(address(this));
        oracle = new MaxPriceOracle();
        loanToken = new ERC20("Loan", "LOAN");
        collateralToken = new ERC20("Collateral", "COL");
    }

    function testLiquidateOverflowDoS() public {
        // 1. Build market with malicious oracle
        CollateralParams[] memory params = new CollateralParams[](1);
        params[0] = CollateralParams({
            token: address(collateralToken),
            lltv: 0.77e18,
            maxLif: ConstantsLib.maxLif(0.77e18, 0.25e18),
            oracle: address(oracle)          // returns type(uint256).max
        });
        Market memory market;
        market.loanToken = address(loanToken);
        market.maturity = block.timestamp + 365 days;
        market.collateralParams = params;
        market.rcfThreshold = type(uint256).max;

        // 2. touchMarket succeeds — no oracle price validation
        bytes32 id = midnight.touchMarket(market);

        // 3. Supply collateral (any non-zero amount)
        deal(address(collateralToken), address(this), 1e18);
        collateralToken.approve(address(midnight), 1e18);
        midnight.supplyCollateral(market, 0, 1e18, address(this));

        // 4. Inject debt directly (simulate borrow)
        // ... (use vm.store to write debt into position slot)

        // 5. liquidate reverts with arithmetic overflow
        vm.expectRevert(); // Solidity 0.8.x overflow panic
        midnight.liquidate(market, 0, 0, 0, address(this), false, address(this), address(0), "");

        // Assertion: liquidation is permanently blocked
        // Any non-zero collateral * type(uint256).max overflows in mulDivDown
    }
}

**Expected assertion:** `liquidate` reverts with a Solidity arithmetic overflow panic (`0x11`) whenever the oracle returns a price satisfying `collateral * price > type(uint256).max`, confirming permanent DoS of liquidation for the market.
