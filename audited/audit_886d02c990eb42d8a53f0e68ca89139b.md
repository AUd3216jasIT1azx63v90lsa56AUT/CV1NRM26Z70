### Title
Unbounded `tokens[]` array in `flashLoan` enables block-stuffing via linear gas growth - (File: src/Midnight.sol)

### Summary
`Midnight.flashLoan` accepts caller-supplied `tokens[]` and `assets[]` arrays with no length cap. The function iterates `tokens.length` twice, each iteration making an external ERC20 call via `SafeTransferLib`, and passes the full arrays to the callback. An unprivileged caller can supply an arbitrarily large array — using a single cheap ERC20 token duplicated N times with `assets[i] = 0` — to consume near-block-gas-limit gas in a single transaction, enabling block stuffing.

### Finding Description
**Code path:**

`flashLoan` at [1](#0-0)  performs:
1. A length-equality check only (`tokens.length == assets.length`) — no upper-bound cap. [2](#0-1) 
2. First loop: `safeTransfer(tokens[i], callback, assets[i])` for every `i`. [3](#0-2) 
3. Callback invocation passing the full arrays. [4](#0-3) 
4. Second loop: `safeTransferFrom(tokens[i], callback, address(this), assets[i])` for every `i`. [5](#0-4) 

`SafeTransferLib.safeTransfer` and `safeTransferFrom` unconditionally issue an external `call` regardless of `value`, with no zero-amount short-circuit: [6](#0-5) 

**Attacker inputs:**
- `tokens = [evilToken, evilToken, ..., evilToken]` — N duplicates of a single attacker-deployed ERC20 that accepts 0-amount transfers
- `assets = [0, 0, ..., 0]` — zero amounts; the contract needs no balance
- `callback` — attacker-controlled contract returning `CALLBACK_SUCCESS`

**Why checks fail:**
The only guard is `tokens.length == assets.length`. There is no `MAX_TOKENS` constant or any other length bound anywhere in `src/`. [2](#0-1) 

**Exploit flow:**
1. Attacker deploys a minimal ERC20 accepting 0-amount transfers and a callback returning `CALLBACK_SUCCESS`.
2. Attacker calls `midnight.flashLoan(tokens_N, zeros_N, callback, "")` with N ≈ block-gas-limit / per-iteration-gas.
3. Each of the 2N external calls costs ~2,100 gas (cold CALL) + ERC20 execution overhead.
4. At N = 1,000: ~2,000 external calls × ~3,000 gas each ≈ 6M gas, approaching or exceeding typical L2 block gas limits.
5. Transaction fills the block; legitimate transactions are excluded.

### Impact Explanation
An unprivileged caller can submit a single `flashLoan` transaction that consumes an amount of gas proportional to `tokens.length`, with no protocol-enforced ceiling. On L2 networks with low gas prices, the cost to the attacker is negligible while the block-stuffing effect is concrete: other users' transactions are delayed or excluded for the duration of the attack. The callback also receives and must process an unbounded array, amplifying the effect if the callback itself loops.

### Likelihood Explanation
- **Preconditions:** None beyond deploying a trivial ERC20 and callback contract. No funds, no position, no authorization required.
- **Feasibility:** Fully reachable on mainnet and all L2 deployments. Zero-amount transfers are universally accepted by standard ERC20s.
- **Repeatability:** The attack can be repeated every block at low cost on L2s.

### Recommendation
Add a maximum length check at the top of `flashLoan`:

```solidity
uint256 constant MAX_FLASH_TOKENS = 16; // or a similarly bounded constant

function flashLoan(...) external {
    require(tokens.length <= MAX_FLASH_TOKENS, TooManyTokens());
    require(tokens.length == assets.length, InconsistentInput());
    ...
}
```

This bounds gas cost per call to a known constant and eliminates the block-stuffing vector.

### Proof of Concept
```solidity
// Foundry fuzz test
contract FuzzFlashLoanGas is BaseTest {
    function testFuzz_flashLoanGasGrowsLinearly(uint16 n) public {
        n = uint16(bound(n, 1, 1000));

        // Deploy minimal ERC20 accepting 0-amount transfers
        MockERC20 tok = new MockERC20();
        GoodCallback cb = new GoodCallback(); // returns CALLBACK_SUCCESS, no-ops

        address[] memory tokens = new address[](n);
        uint256[] memory amounts = new uint256[](n);
        for (uint256 i; i < n; i++) {
            tokens[i] = address(tok);
            amounts[i] = 0;
        }

        uint256 gasBefore = gasleft();
        midnight.flashLoan(tokens, amounts, address(cb), "");
        uint256 gasUsed = gasBefore - gasleft();

        // Assert: gas grows linearly with n; document max n before block gas limit
        // Expected: gasUsed ≈ k * n for some constant k > 0
        // At n=1000 on a 30M-gas block, assert gasUsed > 5_000_000
        assertGt(gasUsed, uint256(n) * 2000);
    }
}
```

**Expected assertions:**
- `gasUsed` grows linearly with `n` (slope ≥ 2,000 gas/token from external calls alone)
- At `n ≈ 5,000–10,000`, `gasUsed` approaches or exceeds a 30M block gas limit
- No revert occurs for any `n` up to the block gas limit — confirming the absence of any protocol-level guard
