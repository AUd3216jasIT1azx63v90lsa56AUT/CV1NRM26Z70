### Title
Free Flash Loans Due to No Fee Charged in `flashLoan` — (File: src/Midnight.sol)

### Summary

The `flashLoan` function in `Midnight.sol` transfers tokens to a callback contract and requires back exactly the same amount with no fee or premium. Any unprivileged caller can borrow any token held by the protocol — including loan tokens from repayments and collateral tokens — for free within a single transaction. The protocol earns zero revenue from flash loan activity and provides a free capital source for arbitrageurs.

### Finding Description

The `flashLoan` function at lines 737–752 of `src/Midnight.sol` implements the following flow:

1. Transfer `assets[i]` of each `tokens[i]` to `callback`.
2. Call `IFlashLoanCallback(callback).onFlashLoan(...)`.
3. Pull back exactly `assets[i]` of each `tokens[i]` from `callback`.

```solidity
// src/Midnight.sol lines 737–752
function flashLoan(address[] calldata tokens, uint256[] calldata assets, address callback, bytes calldata data)
    external
{
    require(tokens.length == assets.length, InconsistentInput());
    emit EventsLib.FlashLoan(msg.sender, tokens, assets, callback);
    for (uint256 i = 0; i < tokens.length; i++) {
        SafeTransferLib.safeTransfer(tokens[i], callback, assets[i]);
    }
    require(
        IFlashLoanCallback(callback).onFlashLoan(msg.sender, tokens, assets, data) == CALLBACK_SUCCESS,
        WrongFlashLoanCallbackReturnValue()
    );
    for (uint256 i = 0; i < tokens.length; i++) {
        SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i]);
    }
}
```

There is no fee variable, no premium added to the repayment amount, and no accounting update. The repayment check is `safeTransferFrom(tokens[i], callback, address(this), assets[i])` — exactly `assets[i]`, not `assets[i] + fee`. [1](#0-0) 

The protocol holds two categories of tokens that are flash-loanable for free:
- **Loan tokens**: accumulated from repayments (`withdrawable`) and lender credit redemptions.
- **Collateral tokens**: deposited by borrowers via `supplyCollateral`.

There is no constant, configuration, or code path in `ConstantsLib.sol` or `Midnight.sol` that defines or applies a flash loan fee. [2](#0-1) 

### Impact Explanation

- **Revenue loss**: The protocol collects settlement fees and continuous fees on lending activity, but earns nothing from flash loan usage regardless of the borrowed amount or token.
- **Free capital for arbitrage**: Any actor can borrow the full balance of any token held by the contract — potentially millions in loan or collateral tokens — at zero cost, enabling risk-free arbitrage against external venues.
- **No direct fund theft** (tokens must be returned), but the protocol's liquidity is exploited as a free public good, subsidizing external actors at the expense of lenders who bear the opportunity cost.

**Impact: Medium**

### Likelihood Explanation

The `flashLoan` function is permissionless — no authorization, no role, no gate. Any EOA or contract can call it with any token address and any amount up to the contract's balance. The exploit requires only deploying a callback contract that implements `IFlashLoanCallback` and returns `CALLBACK_SUCCESS`. [3](#0-2) 

**Likelihood: High**

### Recommendation

Add a flash loan fee parameter (or a protocol-level constant) and require the callback to return `assets[i] + fee` rather than `assets[i]`. The fee should be credited to `claimableSettlementFee` or a dedicated flash loan fee accumulator claimable by the `feeClaimer`. For example:

```solidity
uint256 fee = assets[i].mulDivUp(FLASH_LOAN_FEE, WAD);
SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i] + fee);
claimableSettlementFee[tokens[i]] += fee;
```

Even a small fee (e.g., 5–9 bps, consistent with industry standards) would generate protocol revenue and deter purely extractive flash loan usage.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.13;

import {Test, console} from "forge-std/Test.sol";
import {Midnight} from "../src/Midnight.sol";
import {IERC20} from "forge-std/interfaces/IERC20.sol";

contract FreeFlashLoanCallback {
    bytes32 constant CALLBACK_SUCCESS = keccak256("morpho.midnight.callbackSuccess");

    function onFlashLoan(
        address,
        address[] memory tokens,
        uint256[] memory assets,
        bytes memory
    ) external returns (bytes32) {
        // Use the borrowed tokens for free here (arbitrage, etc.)
        // Then approve repayment of exactly assets[i] — no fee required
        for (uint256 i = 0; i < tokens.length; i++) {
            IERC20(tokens[i]).approve(msg.sender, assets[i]);
        }
        return CALLBACK_SUCCESS;
    }
}

contract FreeFlashLoanTest is Test {
    function test_free_flashloan() public {
        Midnight midnight = new Midnight();
        address loanToken = address(/* deploy mock ERC20 */);

        // Seed the Midnight contract with loan tokens (simulating repayments)
        deal(loanToken, address(midnight), 1_000_000e18);

        FreeFlashLoanCallback cb = new FreeFlashLoanCallback();

        address[] memory tokens = new address[](1);
        tokens[0] = loanToken;
        uint256[] memory assets = new uint256[](1);
        assets[0] = 1_000_000e18;

        uint256 balBefore = IERC20(loanToken).balanceOf(address(midnight));

        // Flash loan 1,000,000 tokens for free
        midnight.flashLoan(tokens, assets, address(cb), "");

        uint256 balAfter = IERC20(loanToken).balanceOf(address(midnight));

        // Protocol balance unchanged — zero fee collected
        assertEq(balAfter, balBefore);
        console.log("Fee paid:", 0); // always 0
    }
}
```

**Expected output**: The flash loan succeeds, the protocol's balance is unchanged, and zero fee is collected — confirming free flash loan access to the full protocol balance.
