### Title
Missing Lower-Bound Sanity Check on `market.maturity` Allows Creation of Immediately-Expired Markets — (File: src/Midnight.sol)

### Summary

`touchMarket()` enforces only an upper-bound on `market.maturity` (`MaturityTooFar`), but has no lower-bound check. Any caller can create a market whose maturity is already in the past. Such a market is immediately in post-maturity mode from block zero of its existence, making it permanently non-functional: no new lending or borrowing positions can ever be established in it.

### Finding Description

In `touchMarket()`, the only maturity validation is:

```solidity
require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
``` [1](#0-0) 

There is no corresponding `require(market.maturity > block.timestamp)` check. A market with `market.maturity = 0` (or any past timestamp) passes this guard and is written to storage with `tickSpacing = DEFAULT_TICK_SPACING`, marking it as a live market. [2](#0-1) 

Once such a market exists, every call to `take()` that would create new debt hits:

```solidity
require(block.timestamp <= offer.market.maturity || sellerDebtIncrease == 0, CannotIncreaseDebtPostMaturity());
``` [3](#0-2) 

Because `block.timestamp > market.maturity` is true from the very first block, `sellerDebtIncrease` must be 0 for every take. In a brand-new market no one holds credit, so `sellerCreditDecrease = 0` and `sellerDebtIncrease = units`. Every take that would create a real position reverts. The market's `withdrawable` counter stays at 0 forever, so no lender can ever withdraw. The market is permanently bricked.

`timeToMaturity` is computed via `zeroFloorSub`, so it collapses to 0:

```solidity
uint256 timeToMaturity = UtilsLib.zeroFloorSub(offer.market.maturity, block.timestamp);
``` [4](#0-3) 

This means the settlement fee is always evaluated at the 0-day breakpoint and `buyerPendingFeeIncrease` is always 0, further confirming the market is in a degenerate state from creation.

### Impact Explanation

- Any user (no privilege required) can permanently occupy a market ID with a non-functional market. Because the market ID is derived from all parameters including `maturity`, a legitimate market with the same collateral/loan token configuration but a correct future maturity will have a different ID and is unaffected.
- The created market is permanently in post-maturity mode: no lending, no borrowing, no position creation is ever possible.
- Gas is consumed for market creation and any subsequent failed interaction attempts.
- Integrators or front-ends that call `touchMarket` directly (it is `public`) with an accidentally wrong maturity will silently create a dead market with no recovery path — the market parameters are immutable once written. [5](#0-4) 

### Likelihood Explanation

- `touchMarket` is `public` and callable by anyone with no access control.
- The `Market` struct's `maturity` field is a plain `uint256` with no type-level constraint.
- A user or integrator who accidentally passes a Unix timestamp in seconds where milliseconds were expected, or transposes digits, will silently create a permanently dead market.
- A malicious actor can deliberately grief specific market parameter combinations by front-running legitimate market creation with a past-maturity version of the same parameters (different ID, so it does not block the legitimate market, but wastes gas and creates confusion).

### Recommendation

Add a lower-bound check inside the market-creation branch of `touchMarket()`, immediately alongside the existing upper-bound check:

```diff
require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
+require(market.maturity > block.timestamp, MaturityInThePast());
``` [1](#0-0) 

A new custom error `MaturityInThePast` should be added to `IMidnight.sol` alongside the existing `MaturityTooFar` error. [6](#0-5) 

### Proof of Concept

1. Deploy `Midnight` on a local fork.
2. Construct a valid `Market` struct with `maturity = block.timestamp - 1` (one second in the past). All other fields (loanToken, collateralParams, etc.) are valid.
3. Call `touchMarket(market)`. It succeeds — `marketState[id].tickSpacing` is now `DEFAULT_TICK_SPACING` (4), confirming the market is created.
4. Construct a valid `Offer` for this market (buy offer, maker ≠ taker, valid ratifier, valid tick, `start ≤ block.timestamp ≤ expiry`).
5. Call `take(offer, ..., units=1, ...)`. The call reverts with `CannotIncreaseDebtPostMaturity` because `block.timestamp > market.maturity` and `sellerDebtIncrease = 1 > 0`.
6. No position can ever be created. The market ID is permanently occupied by a non-functional market.

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;
            uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];
            _marketState.settlementFeeCbp3 = _defaultSettlementFeeCbp[3];
            _marketState.settlementFeeCbp4 = _defaultSettlementFeeCbp[4];
            _marketState.settlementFeeCbp5 = _defaultSettlementFeeCbp[5];
            _marketState.settlementFeeCbp6 = _defaultSettlementFeeCbp[6];
            _marketState.continuousFee = defaultContinuousFee[market.loanToken];
            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);
        }
        return id;
    }
