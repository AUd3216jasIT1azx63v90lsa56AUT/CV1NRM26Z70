### Title
Authorized agent can grief victim's pending pre-signed authorization by submitting a no-op re-authorization to consume the current nonce - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for, not only the authorizer themselves. Because every successful call unconditionally increments `nonce[authorizer]`, an attacker who is already an authorized Midnight agent of the victim can sign and submit a no-op re-authorization (e.g., `authorized=EcrecoverAuthorizer, isAuthorized=true` when already authorized) to burn the victim's current nonce and invalidate any pending pre-signed authorization the victim has broadcast off-chain.

### Finding Description

**Exact code path** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
...
Line 34: signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)
...
Line 47: IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

The authorization check on line 34 accepts two classes of signer: the authorizer themselves, **or** any address that the authorizer has previously authorized on Midnight. The nonce on line 26 is incremented unconditionally on every successful execution. There is no guard requiring the resulting `setIsAuthorized` call to actually change state.

**Attacker-controlled inputs:**
- `authorization.authorizer` = victim `V`
- `authorization.authorized` = `address(ecrecoverAuthorizer)` (already authorized → no-op)
- `authorization.isAuthorized` = `true`
- `authorization.nonce` = `N` (current `nonce[V]`)
- `authorization.deadline` = any future timestamp
- `signature` = attacker `A`'s own ECDSA signature over the above struct

**Exploit flow:**

1. Precondition: `midnight.isAuthorized(V, A) == true` (attacker is an authorized agent of victim).
2. Precondition: `midnight.isAuthorized(V, address(ecrecoverAuthorizer)) == true` (EcrecoverAuthorizer already authorized — no-op target).
3. Victim `V` has broadcast a pre-signed `Authorization` with `nonce=N` off-chain (e.g., to a relayer or mempool).
4. Attacker `A` constructs `Authorization(authorizer=V, authorized=ecrecoverAuthorizer, isAuthorized=true, nonce=N, deadline=future)` and signs it with their own key.
5. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
6. Line 25: deadline check passes.
7. Line 26: `N == nonce[V]` passes; `nonce[V]` incremented to `N+1`.
8. Lines 28–31: digest computed; `ecrecover` returns `A`.
9. Line 34: `IMidnight(MIDNIGHT).isAuthorized(V, A) == true` → passes.
10. Line 47: `midnight.setIsAuthorized(ecrecoverAuthorizer, true, V)` → no-op (already authorized).
11. Victim's pending pre-signed authorization (nonce `N`) now reverts with `InvalidNonce` when submitted.

**Why existing checks fail:**
- The `Unauthorized` check (line 34) is designed to allow agents to act on behalf of the authorizer, but it does not distinguish between actions that change state and no-op re-authorizations.
- There is no idempotency guard: `midnight.setIsAuthorized` on line 47 writes the same value unconditionally (`isAuthorized[V][ecrecoverAuthorizer] = true` again), and the nonce has already been consumed.
- The Certora spec `EcrecoverAuthorizer.spec` rule `effects` (line 19) only asserts that the nonce increments on success — it does not rule out the case where a Midnight-authorized agent (not the authorizer) triggers that increment.

### Impact Explanation
Any pending pre-signed `Authorization` the victim has created and shared off-chain (e.g., with a relayer, keeper, or counterparty) is permanently invalidated at the cost of a single transaction by any of the victim's existing Midnight-authorized agents. The victim must re-sign and re-broadcast. The attacker can repeat this indefinitely, making it impossible for the victim to use `EcrecoverAuthorizer` for off-chain authorization flows as long as the attacker remains an authorized agent.

### Likelihood Explanation
**Preconditions:**
1. Attacker must hold `midnight.isAuthorized(victim, attacker) == true`. This is a realistic state: any protocol, keeper, or counterparty the victim has previously authorized satisfies it.
2. Victim must have a pending pre-signed authorization in flight (mempool, relayer queue, or shared off-chain).
3. `EcrecoverAuthorizer` must already be authorized by the victim (so the re-authorization is a no-op with no collateral damage). If not, the attacker can still pick any other already-authorized address as `authorized`.

The attack is cheap (one transaction), repeatable, and requires no special privileges beyond being an existing authorized agent — a common operational state for any user interacting with protocols built on Midnight.

### Recommendation
Remove the agent delegation from `EcrecoverAuthorizer`. The purpose of `EcrecoverAuthorizer` is to let the authorizer sign off-chain; agents who already hold a Midnight authorization can call `midnight.setIsAuthorized` directly without a signature. Change line 33–36 to:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This eliminates the griefing vector entirely: only the authorizer's own key can consume their nonce, so no third party can burn a pending pre-signed authorization.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {EcrecoverAuthorizerTest, Authorization, Signature} from "./SetIsAuthorizedWithSigTest.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract NonceGriefingTest is EcrecoverAuthorizerTest {
    function testNonceGriefByAuthorizedAgent() public {
        // Setup: victim (borrower) authorizes EcrecoverAuthorizer on Midnight
        vm.prank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);

        // Setup: victim authorizes attacker (lender) as a Midnight agent
        vm.prank(borrower);
        midnight.setIsAuthorized(lender, true, borrower);

        // Victim pre-signs an authorization (nonce=0) to authorize otherLender
        Authorization memory victimAuth = makeAuthorization(borrower, otherLender, true);
        Signature memory victimSig = signAuthorization(victimAuth, borrower);
        // victimAuth.nonce == 0

        // Attacker (lender) constructs a no-op re-authorization signed with their own key
        Authorization memory attackAuth = Authorization({
            authorizer: borrower,
            authorized: address(ecrecoverAuthorizer), // already authorized → no-op
            isAuthorized: true,
            nonce: 0, // same nonce as victim's pending auth
            deadline: block.timestamp + 1 days
        });
        Signature memory attackSig = signAuthorization(attackAuth, lender); // signed by attacker

        // Attacker submits — succeeds because lender is authorized by borrower on Midnight
        ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);

        // Assert: nonce[borrower] is now 1
        assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

        // Assert: victim's pending pre-signed auth (nonce=0) now reverts with InvalidNonce
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

        // Assert: the no-op did not change Midnight state (EcrecoverAuthorizer still authorized)
        assertEq(midnight.isAuthorized(borrower, address(ecrecoverAuthorizer)), true);
    }
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(borrower) == 1` after attacker's call.
- Victim's pre-signed authorization reverts with `InvalidNonce`.

/// EcrecoverAuthorizer increments nonce on success and does not change other nonces.
rule effects(env e, EcrecoverAuthorizer.Authorization authorization, EcrecoverAuthorizer.Signature signature, address other) {
    require other != authorization.authorizer;
    uint256 nonceBefore = nonce(authorization.authorizer);
    uint256 otherNonceBefore = nonce(other);

    setIsAuthorized(e, authorization, signature);

    assert nonce(authorization.authorizer) == nonceBefore + 1;
    assert nonce(other) == otherNonceBefore;
}
