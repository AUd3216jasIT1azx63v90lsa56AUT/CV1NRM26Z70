Audit Report

## Title
Missing `units > 0` guard allows zero-cost callback invocation and transient liquidation lock manipulation - (File: src/Midnight.sol)

## Summary
`take()` contains no guard against `units == 0`. When called with `units=0` against a buy offer with a non-zero `offer.callback`, all accounting deltas are zero yet the function unconditionally sets the seller's transient liquidation lock and dispatches `IBuyCallback.onBuy(...)` with `buyerAssets=0`. Because `consumed` never advances on a zero-unit take, the same offer can be targeted in an unbounded loop at only gas cost.

## Finding Description

**Root cause**: `grep` for `require.*units` in `src/Midnight.sol` returns zero matches. No input validation on `units` exists anywhere in `take()`.

**Confirmed code path**:

- [1](#0-0)  `buyerAssets` and `sellerAssets` both resolve to `0` when `units=0`.
- [2](#0-1)  `consumed[offer.maker][offer.group] += 0`; `newConsumed` is unchanged, so `require(newConsumed <= offer.maxAssets)` passes trivially and is repeatable indefinitely.
- [3](#0-2)  All credit/debt deltas are zero; position state is unchanged.
- [4](#0-3)  `UtilsLib.tExchange(LIQUIDATION_LOCK_SLOT, id, seller, true)` executes **unconditionally**, regardless of `units`.
- [5](#0-4)  `IBuyCallback(buyerCallback).onBuy(id, market, 0, 0, 0, buyer, data)` is dispatched with no guard on `buyerAssets > 0`.
- [6](#0-5)  `safeTransferFrom(..., 0)` executes silently; standard ERC20 does not revert on zero-amount transfers.
- [7](#0-6)  Health check passes because `sellerDebtIncrease == 0`.

**Why existing checks fail**:
- `require(UtilsLib.atMostOneNonZero(offer.maxAssets, offer.maxUnits))` — checks mutual exclusivity only, not that `units > 0`. [8](#0-7) 
- `require(newConsumed <= offer.maxAssets)` — `newConsumed` is unchanged when `buyerAssets=0`, so this always passes and the same offer can be targeted indefinitely. [9](#0-8) 
- `require(offer.maker != taker)` — prevents self-take but any third party satisfies this. [10](#0-9) 

## Impact Explanation

1. **Callback invoked at zero economic cost**: `offer.callback.onBuy(id, market, 0, 0, 0, buyer, data)` executes. Any side effects inside the callback — state writes, event emissions, external protocol interactions — run for free. The callback must return `CALLBACK_SUCCESS` but is not required to transfer any tokens since `buyerAssets - sellerAssets = 0`.
2. **Liquidation lock observable during callback**: At the moment `onBuy` executes, `liquidationLocked(id, seller) == true` in transient storage. The callback can read this via `IMidnight.liquidationLocked` and branch on it or relay it to other contracts, violating the invariant that the lock is only meaningful during a real economic take.
3. **Unlimited griefing**: Since `consumed` never advances on a zero-unit take, the same offer can be targeted in an unbounded loop at only gas cost, enabling sustained griefing of any callback-bearing buy offer.

## Likelihood Explanation

**Preconditions**: A live buy offer with `offer.callback != address(0)` and `offer.maxAssets > 0` must exist — standard for lend-side offers using flash-lend callbacks. The taker must not equal `offer.maker` (trivially satisfied by any third party). No privileged access, oracle manipulation, or special token behavior required.

**Feasibility**: Fully reachable with a single external call. The fuzz test `testBuyBuyerCallback` in `test/TakeTest.sol` exists and the protocol does not reject `units=0` at the test level, confirming the path is live. [11](#0-10) 

## Recommendation

Add an input guard at the top of `take()`, immediately after the existing precondition checks:

```solidity
require(units > 0, ZeroUnits());
```

This single line eliminates all three impact vectors: zero-cost callback dispatch, transient lock manipulation, and unbounded griefing via repeated zero-unit calls.

## Proof of Concept

```solidity
// Attacker contract
function attack(IMidnight midnight, Offer memory buyOffer) external {
    // buyOffer.buy = true, buyOffer.callback != address(0)
    // Repeat indefinitely — consumed never advances
    for (uint i = 0; i < 1000; i++) {
        midnight.take(
            buyOffer,
            ratifierData,
            0,          // units = 0
            address(this),
            address(this),
            address(0),
            ""
        );
        // Each iteration: onBuy fires with buyerAssets=0, lock is set/unset
    }
}
```

The maker's `onBuy` callback fires 1000 times with zero token transfer. Any state-mutating logic inside the callback executes at attacker's gas cost only.
