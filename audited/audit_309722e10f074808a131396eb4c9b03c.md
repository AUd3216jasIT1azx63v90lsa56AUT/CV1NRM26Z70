### Title
Unprivileged `touchMarket` before `setDefaultSettlementFee` permanently snapshots zero settlement fees - (`src/Midnight.sol`)

### Summary
`touchMarket` is an unrestricted `public` function that snapshots `defaultSettlementFeeCbp[market.loanToken]` at creation time with no check that defaults have been initialized. If called before the `feeSetter` has invoked `setDefaultSettlementFee`, all seven `settlementFeeCbp` breakpoints are permanently written as zero. Subsequent markets for the same loan token created after the feeSetter sets non-zero defaults will have non-zero fees, producing a permanent per-market state inconsistency that only the `feeSetter` can manually remediate via `setMarketSettlementFee`.

### Finding Description
**Code path:**

`touchMarket` (`src/Midnight.sol:755`) is `public` with no caller restriction. The creation branch (`marketState[id].tickSpacing == 0`) reads the current `defaultSettlementFeeCbp[market.loanToken]` array and copies all seven values directly into `_marketState.settlementFeeCbp0`–`settlementFeeCbp6` (lines 777–784). There is no guard requiring that any of these values be non-zero, nor any check that `setDefaultSettlementFee` has ever been called for the loan token.

`setDefaultSettlementFee` (`src/Midnight.sol:277`) only writes to `defaultSettlementFeeCbp[loanToken]`; it does not retroactively update any already-created market's `MarketState`. `setMarketSettlementFee` (`src/Midnight.sol:258`) can correct a specific market's fees but is gated to `feeSetter`.

**Exploit flow:**
1. Midnight is deployed; `defaultSettlementFeeCbp[loanToken]` is `[0,0,0,0,0,0,0]` (Solidity default).
2. Attacker (any address) calls `touchMarket(market)` where `market.loanToken` is the target token.
3. `touchMarket` enters the creation branch, reads the all-zero array, and writes `settlementFeeCbp0`–`settlementFeeCbp6 = 0` into `marketState[id]`.
4. `feeSetter` later calls `setDefaultSettlementFee(loanToken, i, nonZeroFee)` for `i = 0..6`.
5. A second market (different `maturity` or collateral set) is created via `touchMarket`; it receives the now-non-zero defaults.
6. Market 1 permanently has zero settlement fees; Market 2 has non-zero fees. The `settlementFee` view function (`src/Midnight.sol:963`) returns 0 for all TTM values on Market 1.

**Why existing checks fail:** The only guards in `touchMarket` validate maturity range, collateral list structure, LLTV, and maxLif. None check the state of `defaultSettlementFeeCbp`. The Certora rule `newMarketSettlementFeesMatchDefault` verifies that market fees equal defaults at creation time — which is satisfied even when both are zero, so the formal spec does not catch this ordering issue.

### Impact Explanation
Market 1 collects zero settlement fees on every `take` call for its entire lifetime. The protocol earns no settlement revenue from that market. Meanwhile, identically-structured markets created after defaults are set charge the intended fees. This is a concrete, permanent, per-market state inconsistency: two markets with the same loan token have divergent `settlementFeeCbp` arrays solely due to creation ordering, not any privileged override.

### Likelihood Explanation
The precondition — `defaultSettlementFeeCbp[loanToken]` being all-zero — holds for every loan token until `feeSetter` explicitly initializes it. This is the natural state immediately after deployment and for any newly introduced loan token. An attacker can front-run the feeSetter's `setDefaultSettlementFee` transaction, or the condition can arise organically if a market creator acts before the feeSetter. The attack is a single permissionless call, requires no capital, and is repeatable for any loan token whose defaults have not yet been set.

### Recommendation
Add a check inside the market-creation branch of `touchMarket` that at least one `defaultSettlementFeeCbp[market.loanToken]` entry is non-zero, or introduce a dedicated boolean flag `defaultSettlementFeeInitialized[loanToken]` set by `setDefaultSettlementFee` and required by `touchMarket`. Alternatively, gate `touchMarket` so that market creation is only permitted after the feeSetter has initialized defaults for the loan token:

```solidity
require(
    defaultSettlementFeeCbp[market.loanToken][6] > 0,
    DefaultSettlementFeeNotSet()
);
```

This enforces the protocol-stated invariant ("A default settlement fee (per loan token) is set on new markets") at the code level rather than relying on operational ordering.

### Proof of Concept
```solidity
function testZeroFeeMarketFrontrun() public {
    // Precondition: feeSetter has NOT yet called setDefaultSettlementFee for loanToken
    // defaultSettlementFeeCbp[loanToken] == [0,0,0,0,0,0,0]

    // Step 1: attacker (unprivileged) creates market before defaults are set
    vm.prank(attacker);
    bytes32 id1 = midnight.touchMarket(market1); // market1 uses loanToken

    // Step 2: feeSetter sets non-zero defaults
    for (uint256 i = 0; i < 7; i++) {
        midnight.setDefaultSettlementFee(address(loanToken), i, maxSettlementFee(i));
    }

    // Step 3: create a second market (different maturity) for same loanToken
    bytes32 id2 = midnight.touchMarket(market2);

    // Assert: market1 has zero fees, market2 has non-zero fees
    uint16[7] memory fees1 = midnight.settlementFeeCbps(id1);
    uint16[7] memory fees2 = midnight.settlementFeeCbps(id2);
    for (uint256 i = 0; i < 7; i++) {
        assertEq(fees1[i], 0, "market1 fee should be zero");
        assertGt(fees2[i], 0, "market2 fee should be non-zero");
    }
    // Also assert settlementFee view returns 0 for market1 at any TTM
    assertEq(midnight.settlementFee(id1, 30 days), 0);
    assertGt(midnight.settlementFee(id2, 30 days), 0);
}

        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)

                    InvalidMaxLif()
