### Title
ECDSA Signature Malleability Accepted in `isRatified` — (`src/ratifiers/EcrecoverRatifier.sol`)

### Summary
`EcrecoverRatifier.isRatified` calls raw `ecrecover` with no check that `sig.s` is in the lower half of the secp256k1 curve order (`s ≤ secp256k1n/2`). For any valid canonical signature `(v, r, s)` over a digest, the malleable counterpart `(v ^ 1, r, secp256k1n − s)` recovers to the identical signer address and passes all existing checks, returning `CALLBACK_SUCCESS`. This means two structurally distinct `Signature` structs both ratify the same offer tree root, producing non-critical behavior divergence with no direct loss or freezing.

### Finding Description
**Code path:**

`EcrecoverRatifier.isRatified` (lines 33–46):

```
ecrecover(digest, sig.v, sig.r, sig.s)   // line 42 — no s-range check
require(_signer != address(0), ...)       // line 43
require(_signer == offer.maker || isAuthorized(...), ...) // line 44
``` [1](#0-0) 

**Root cause:** The `Signature` struct accepts any `(v, r, s)` triple. ECDSA over secp256k1 is symmetric: for every valid `(v, r, s)`, the tuple `(v ^ 1, r, secp256k1n − s)` is also a valid signature for the same digest and recovers to the same address. The contract performs no normalization or rejection of high-`s` values. [2](#0-1) 

**Attacker inputs:** Any taker who observes a valid `ratifierData` blob `(sig_canonical, root, leafIndex, proof)` can compute `sig_malleable = Signature({v: sig.v ^ 1, r: sig.r, s: secp256k1n - sig.s})` and submit `(sig_malleable, root, leafIndex, proof)` as `ratifierData`.

**Exploit flow:**
1. Maker signs root `R` with private key `k`, producing canonical `(v, r, s)`.
2. Taker calls `Midnight.take(offer, abi.encode(sig_canonical, R, leafIndex, proof), ...)` — succeeds.
3. Taker (or any observer) computes `sig_malleable = (v^1, r, secp256k1n − s)`.
4. Taker calls `isRatified` (via `Midnight.take`) with `sig_malleable` — also succeeds, returning `CALLBACK_SUCCESS`.

**Why checks fail:** The only post-`ecrecover` checks are `_signer != address(0)` and `_signer == offer.maker || isAuthorized(...)`. Both canonical and malleable signatures recover to the same non-zero address, so both pass. [3](#0-2) 

### Impact Explanation
Two structurally distinct `Signature` structs — `(v, r, s)` and `(v^1, r, secp256k1n−s)` — both pass `isRatified` for the same offer tree root and offer. This is non-critical behavior divergence: no funds are lost, no position is frozen, and no offer replay occurs (offer replay protection is root-based via `isRootCanceled`, not signature-based). The divergence is purely that the signature uniqueness invariant is violated at the ratifier level.

### Likelihood Explanation
Preconditions: a valid canonical signature for any root must exist (trivially satisfied whenever any offer is taken). The malleable counterpart is deterministically computable by any observer with no privileged access. The transformation is a single arithmetic operation (`secp256k1n − s`) and a bit-flip on `v`. Repeatable for every signature in the system.

### Recommendation
Add a low-`s` normalization check immediately before or after `ecrecover`:

```solidity
uint256 constant SECP256K1_N = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;
require(uint256(sig.s) <= SECP256K1_N / 2, MalleableSignature());
```

This is the same guard used by OpenZeppelin's `ECDSA.recover` and EIP-2 (enforced at the transaction level since Homestead). Alternatively, use OpenZeppelin's `ECDSA` library directly instead of raw `ecrecover`.

function testSignatureMalleability(uint256 privKey) public {

    address maker = vm.addr(privKey);

    Offer memory offer;
    offer.maker = maker;
    bytes32 root = HashLib.hashOffer(offer);

    // Build malleable counterpart
    uint256 SECP256K1_N = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;
    Signature memory malleable = Signature({
        v: canonical.v ^ 1,
        r: canonical.r,
        s: bytes32(SECP256K1_N - uint256(canonical.s))
    });

    vm.prank(maker);
    midnight.setIsAuthorized(address(ecrecoverRatifier), true, maker);

    // Both must return CALLBACK_SUCCESS
    vm.prank(address(midnight));
    bytes32 r1 = ecrecoverRatifier.isRatified(offer, abi.encode(canonical, root, 0, new bytes32[](0)));
    assertEq(r1, CALLBACK_SUCCESS);

    vm.prank(address(midnight));
    bytes32 r2 = ecrecoverRatifier.isRatified(offer, abi.encode(malleable, root, 0, new bytes32[](0)));
    assertEq(r2, CALLBACK_SUCCESS); // passes — malleability confirmed
}

    function isRatified(Offer memory offer, bytes memory ratifierData) external view returns (bytes32) {
        require(msg.sender == MIDNIGHT, NotMidnight());
        (Signature memory sig, bytes32 root, uint256 leafIndex, bytes32[] memory proof) =
            abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]));
        require(HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof), InvalidProof());
        require(!isRootCanceled[offer.maker][root], RootCanceled());
        bytes32 structHash = keccak256(abi.encode(HashLib.offerTreeTypeHash(proof.length), root));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, structHash));
        address _signer = ecrecover(digest, sig.v, sig.r, sig.s);
        require(_signer != address(0), InvalidSignature());
        require(_signer == offer.maker || IMidnight(MIDNIGHT).isAuthorized(offer.maker, _signer), Unauthorized());
        return CALLBACK_SUCCESS;
    }
