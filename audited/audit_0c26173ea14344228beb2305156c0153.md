**`mulDivUp` with zero numerator:** `mulDivUp(x, 0, d) = (x * 0 + (d-1)) / d = (d-1)/d = 0` (integer division). This is formally proven in `MulDiv.spec` rule `mulDivZero`.

### Title
Free Collateral Seizure via Zero Oracle Price in `seizedAssets` Path — (`File: src/Midnight.sol`)

### Summary

When the oracle for the liquidated collateral returns `0`, calling `liquidate` with `seizedAssets > 0` and `repaidUnits = 0` causes `repaidUnits` to be computed as `0` (because `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE) = 0`). The function then transfers real collateral tokens to the liquidator while pulling zero loan tokens from the payer, violating the invariant that collateral cannot be seized without repaying correct debt. The protocol's own LIVENESS documentation acknowledges oracle-returning-zero as a valid scenario and guards the `repaidUnits` input path (which reverts via division-by-zero), but leaves the `seizedAssets` input path completely unguarded.

### Finding Description

**Exact code path — `src/Midnight.sol`:**

**Oracle loop (lines 607–618):** All activated collateral oracles are called. If the oracle at `collateralIndex` returns `0`, `liquidatedCollatPrice` is set to `0`. [1](#0-0) 

**`NotLiquidatable` check (lines 620–624):** Uses `originalDebt > maxDebt`. With `liquidatedCollatPrice = 0`, this collateral contributes `0` to `maxDebt`, so the position is trivially liquidatable if any debt exists. [2](#0-1) 

**`badDebt` block (lines 626–641):** Executes when `badDebt > 0`. With oracle returning `0`, this collateral contributes `0` to the max-lif-adjusted sum, making `badDebt = originalDebt` (all debt is bad). Debt is reduced and `lossFactor` is updated. [3](#0-2) 

**`seizedAssets` computation (line 650):** The critical line:
```solidity
repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
```
With `liquidatedCollatPrice = 0`:
- `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE)` = `(seizedAssets * 0 + (ORACLE_PRICE_SCALE - 1)) / ORACLE_PRICE_SCALE` = `0`
- `mulDivUp(0, WAD, lif)` = `0`
- Therefore `repaidUnits = 0`

This is proven by the `mulDivZero` rule in `MulDiv.spec`. [4](#0-3) [5](#0-4) [6](#0-5) 

**RCF check (lines 662–667):** `repaidUnits(0) <= maxRepaid` is trivially true for any `maxRepaid >= 0`. The second branch of the `||` also evaluates to `0 < rcfThreshold` (since `liquidatedCollatPrice = 0` makes the left side `0`). The check never fires. [7](#0-6) 

**State mutation and transfers (lines 670–717):**
- `_position.collateral[collateralIndex] -= seizedAssets` — collateral is seized
- `_marketState.withdrawable += 0` — no loan tokens credited
- `_position.debt -= 0` — no debt repaid
- `safeTransfer(collateralToken, receiver, seizedAssets)` — attacker receives collateral
- `safeTransferFrom(loanToken, payer, address(this), 0)` — attacker pays nothing [8](#0-7) 

**The asymmetric protection gap:** The `repaidUnits > 0` input path reverts because `mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice=0)` is a division by zero. The `seizedAssets > 0` input path silently produces `repaidUnits = 0` instead of reverting. The Certora spec `Reverts.spec` proves the former but has no rule for the latter. [9](#0-8) 

The `LiquidationProfitability.spec` rule `liquidationLifSeizedAssets` asserts `seizedResult * price > (repaidResult - 1) * ORACLE_PRICE_SCALE()`. When `price = 0` and `repaidResult = 0`, this becomes `0 > -ORACLE_PRICE_SCALE()` in `mathint`, which is vacuously true — the rule does not catch the bug. [10](#0-9) 

The existing test `testFullBadDebtWithdrawCollateral` sets oracle to `0` but only calls `liquidate` with `seizedAssets = 0, repaidUnits = 0`, never exercising the `seizedAssets > 0` path. [11](#0-10) 

### Impact Explanation

An unprivileged liquidator seizes real collateral tokens (up to the full `_position.collateral[collateralIndex]` balance) while paying zero loan tokens. The protocol's `withdrawable` balance is not increased, so lenders receive no repayment for the seized collateral. This directly violates the core invariant: "collateral cannot be withdrawn or seized outside health/liquidation rules" — specifically, collateral is seized without any corresponding debt repayment or loan token transfer.

### Likelihood Explanation

**Preconditions:**
1. A market must exist where the oracle for at least one collateral can return `0`. The LIVENESS section of `Midnight.sol` explicitly documents this as a known possible scenario (not an impossible value), and the Certora spec models it with `forceOracleReturnZero` and `singleZeroOracle` ghosts.
2. The borrower must have debt and collateral at that index.
3. The position must be liquidatable (`originalDebt > maxDebt`), which is guaranteed when the oracle returns `0` (this collateral contributes `0` to `maxDebt`).

The attack is repeatable in any market where the oracle can reach `0` (e.g., a depeg event, a price feed failure, or a market created with a custom oracle). No special permissions are required — any address can call `liquidate`. [12](#0-11) 

### Recommendation

Add a guard before the `seizedAssets` computation to reject the call when the oracle price is zero and collateral is being seized:

```solidity
// In the `if (repaidUnits > 0 || seizedAssets > 0)` block, before line 650:
if (seizedAssets > 0) {
    require(liquidatedCollatPrice > 0, ZeroOraclePriceWithSeizedAssets());
    repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
}

rule liquidationLifSeizedAssets(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, address borrower, address receiver, address callback, bytes data, bool postMaturityMode) {
    uint256 maxLif = market.collateralParams[collateralIndex].maxLif;
    require maxLif >= WAD(), "see the rule maxLifIsAtLeastWad";

    bool maxLifReached = !postMaturityMode || e.block.timestamp >= require_uint256(market.maturity + TIME_TO_MAX_LIF());

    uint256 seizedResult;
    uint256 repaidResult;
    seizedResult, repaidResult = liquidate(e, market, collateralIndex, seizedAssets, 0, borrower, postMaturityMode, receiver, callback, data);

    mathint price = summaryPrice(market.collateralParams[collateralIndex].oracle);

    // lif >= WAD: liquidator receives collateral worth at least the repaid debt (up to 1 unit ceil rounding on repaidUnits) at the oracle price.
    assert seizedResult * price > (repaidResult - 1) * ORACLE_PRICE_SCALE();

    // lif == maxLif when in normal mode or when >= 15 min post-maturity: full liquidation incentive factor applies.
    assert maxLifReached => seizedResult * price * WAD() + ORACLE_PRICE_SCALE() * WAD() > (repaidResult - 1) * maxLif * ORACLE_PRICE_SCALE();
}
