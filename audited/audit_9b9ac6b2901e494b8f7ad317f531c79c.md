### Title
Missing oracle address validation in `touchMarket` allows permanent liquidation freeze - (`src/Midnight.sol`)

### Summary
`touchMarket` validates collateral token sorting, LLTV tiers, and maxLif values but performs no check on `collateralParams[i].oracle`. An unprivileged caller can create a market with `oracle = address(0)`, supply collateral, and borrow. Because `supplyCollateral` never calls the oracle, the position is established successfully. Every subsequent call that requires a health check (`isHealthy`, `withdrawCollateral` with debt, `liquidate`) will attempt `IOracle(address(0)).price()`, which reverts due to empty return-data decoding, permanently freezing the position and blocking liquidation.

### Finding Description
**Root cause — `touchMarket` (src/Midnight.sol lines 755–791):**

The validation loop iterates over `collateralParams` and enforces:
- `collateralToken > previousCollateralToken` (sorted, no duplicates)
- `isLltvAllowed(lltv)`
- `maxLif` is one of the two allowed values

There is no `require(market.collateralParams[i].oracle != address(0))` or any code-existence check on the oracle address. [1](#0-0) 

**Exploit flow:**

1. Attacker calls `touchMarket` with `collateralParams[0].oracle = address(0)`. All existing checks pass; the market is created.
2. Attacker calls `supplyCollateral` — oracle is never read here; only the token transfer and bitmap update occur. [2](#0-1) 
3. Attacker (as seller) takes a lender's buy offer via `take`, acquiring debt.
4. Any call to `isHealthy` with `debt > 0` enters the bitmap loop and executes `IOracle(address(0)).price()`. The EVM call to `address(0)` returns `(true, "")` (empty data); Solidity's ABI decoder requires 32 bytes for `uint256` and reverts. [3](#0-2) 
5. `liquidate` calls the same oracle loop unconditionally before the liquidatability check, so it also reverts. [4](#0-3) 
6. `withdrawCollateral` calls `isHealthy` after updating collateral state, so it reverts whenever the borrower has debt. [5](#0-4) 

**Why existing checks fail:**

The Certora `Reverts.spec` formally verifies that a *reverting* oracle causes `liquidate` to revert — this is treated as an accepted liveness property. [6](#0-5)  However, the spec assumes the oracle is a deployed contract that *can* revert; it does not model `address(0)` as a permanently broken oracle set at market creation. The `validMarket` test helper itself leaves `oracle` unset (defaulting to `address(0)`) and `touchMarket` succeeds, confirming no on-chain guard exists. [7](#0-6) 

### Impact Explanation
Once a borrower has debt in a market with `oracle = address(0)`:
- `isHealthy` always reverts → `withdrawCollateral` always reverts for that borrower.
- `liquidate` always reverts → the undercollateralized position can never be cleared.
- Lender funds are permanently locked; bad debt cannot be realized.
- The collateral bitmap bit is set and can never be cleared through normal protocol paths.

### Likelihood Explanation
No preconditions are required. `touchMarket` is permissionless and callable by any address. The attacker needs only to construct a `Market` struct with a valid token, LLTV, and maxLif, and set `oracle = address(0)`. A lender who does not inspect oracle addresses on-chain before placing offers is the victim. The attack is repeatable across any number of markets and is permanent once a position with debt exists.

### Recommendation
Add a non-zero oracle check inside the `touchMarket` validation loop:

```solidity
require(market.collateralParams[i].oracle != address(0), InvalidOracle());
```

Optionally, also verify that the oracle address has deployed code (`market.collateralParams[i].oracle.code.length > 0`) to prevent other zero-return or non-contract oracle addresses from being registered.

    vm.prank(borrower);
    collateralToken1.approve(address(midnight), 1e18);
    midnight.supplyCollateral(m, 0, 1e18, borrower);

    // 3. Establish debt via take (lender provides liquidity)
    // ... standard offer setup omitted for brevity ...

    // 4. Assert isHealthy reverts
    vm.expectRevert();
    midnight.isHealthy(m, id, borrower);

    // 5. Assert liquidate reverts
    deal(address(loanToken), address(this), 1e18);
    vm.expectRevert();
    midnight.liquidate(m, 0, 1e18, 0, borrower, false, address(this), address(0), "");

    // 6. Assert withdrawCollateral reverts (borrower has debt)
    vm.prank(borrower);
    vm.expectRevert();

Expected: steps 1–2 succeed; steps 4–6 all revert, confirming the position is permanently frozen.

    function validMarket(Market memory market) internal view returns (Market memory) {
        uint256 len =
            market.collateralParams.length > MAX_COLLATERALS ? MAX_COLLATERALS : market.collateralParams.length;
        vm.assume(len > 0);
        CollateralParams[] memory collateralParams = new CollateralParams[](len);
        for (uint256 i = 0; i < len; i++) {
            collateralParams[i].token =
                address(uint160(uint256(keccak256(abi.encode(market.collateralParams[i].token, i)))));
            uint256 lltv = allowedLltv(market.collateralParams[i].lltv);
            collateralParams[i].lltv = lltv;
            collateralParams[i].maxLif = maxLif(lltv, LIQUIDATION_CURSOR_LOW);
        }
        collateralParams = sortCollateralParams(collateralParams);
        market.collateralParams = collateralParams;
        market.maturity = bound(market.maturity, 0, vm.getBlockTimestamp() + 100 * 365 days);
        return market;
    }
