### Title
Post-Maturity Bad Debt + Collateral Seizure Causes `_position.debt` Underflow Revert - (File: src/Midnight.sol)

### Summary

In `Midnight.liquidate`, `_position.debt` is first reduced by `badDebt` at line 628 (computed using `maxLif`), then reduced again by `repaidUnits` at line 676 (computed using `lif`, which is strictly less than `maxLif` in early post-maturity mode). Because the `maxRepaid` guard is unconditionally skipped in post-maturity mode (line 655), no check prevents `repaidUnits` from exceeding the already-reduced `_position.debt`, causing a Solidity 0.8 checked-arithmetic revert that permanently freezes liquidation of any undercollateralized position entered in early post-maturity.

### Finding Description

**Code path:**

1. `originalDebt = _position.debt` (line 604); `badDebt` starts at `originalDebt` (line 605).
2. The collateral loop (lines 607–618) subtracts each collateral's coverage at `maxLif` using `mulDivUp`, leaving `badDebt = originalDebt − Σ⌈collateral[i]·price[i]/ORACLE_PRICE_SCALE · WAD/maxLif[i]⌉`.
3. Line 628 writes `_position.debt -= uint128(badDebt)`, so `_position.debt` now equals `Σ⌈collateral[i]·price[i]/ORACLE_PRICE_SCALE · WAD/maxLif[i]⌉`.
4. In post-maturity mode with `block.timestamp − market.maturity < TIME_TO_MAX_LIF`, line 646 sets `lif = WAD + (maxLif − WAD)·Δt/TIME_TO_MAX_LIF`, which is strictly less than `maxLif`.
5. When the liquidator passes `seizedAssets = _position.collateral[collateralIndex]` (all collateral of the seized asset), line 650 computes `repaidUnits = ⌈seizedAssets·price/ORACLE_PRICE_SCALE · WAD/lif⌉`. Because `lif < maxLif`, `WAD/lif > WAD/maxLif`, so `repaidUnits > ⌈seizedAssets·price/ORACLE_PRICE_SCALE · WAD/maxLif⌉`.
6. For a single-collateral position, `_position.debt` after step 3 equals exactly `⌈collateral·price/ORACLE_PRICE_SCALE · WAD/maxLif⌉`, which is strictly less than `repaidUnits`.
7. The `maxRepaid` guard (lines 659–667) is wrapped in `if (!postMaturityMode)` (line 655) and is **entirely skipped** in post-maturity mode — no check bounds `repaidUnits` against `_position.debt`.
8. Line 676 executes `_position.debt -= UtilsLib.toUint128(repaidUnits)`. Solidity 0.8.34 checked arithmetic reverts on underflow.

**Attacker-controlled inputs:** `seizedAssets` set to the full collateral balance; `postMaturityMode = true`; timing chosen so `lif < maxLif`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

### Impact Explanation

Any liquidation call on a position with `badDebt > 0` in early post-maturity mode (`lif < maxLif`) that also passes a non-zero `seizedAssets` will revert unconditionally. The position cannot be liquidated via the collateral-seizure path; only a pure bad-debt realization call (`seizedAssets = 0`, `repaidUnits = 0`) succeeds. This freezes the normal liquidation flow, leaving the protocol holding undercollateralized debt with no mechanism to recover collateral until `lif` reaches `maxLif` (i.e., after `TIME_TO_MAX_LIF` seconds post-maturity).

### Likelihood Explanation

The preconditions are reachable without any privileged action:
- A position becomes undercollateralized (oracle price drop or interest accrual) and enters post-maturity.
- Any liquidator calls `liquidate` with `postMaturityMode = true` and `seizedAssets > 0` before `TIME_TO_MAX_LIF` elapses.
- The revert is deterministic and repeatable for every block in the window `[maturity, maturity + TIME_TO_MAX_LIF)`.

### Recommendation

After computing `repaidUnits` in the post-maturity branch, cap it at the remaining `_position.debt` before applying the subtraction:

```solidity
// After line 653, before line 670:
if (repaidUnits > _position.debt) {
    // Adjust seizedAssets proportionally or simply cap repaidUnits
    repaidUnits = _position.debt;
}
```

Alternatively, compute `badDebt` using the actual `lif` (not `maxLif`) so that the two subtractions are consistent, or move the `maxRepaid` / debt-cap check outside the `!postMaturityMode` guard.

    // 3. Advance time past maturity but less than TIME_TO_MAX_LIF
    //    so lif = WAD + (maxLif - WAD) * dt / TIME_TO_MAX_LIF < maxLif
    // 4. Liquidator calls liquidate(... seizedAssets = C, postMaturityMode = true ...)
    // Expected: vm.expectRevert() due to arithmetic underflow at _position.debt -= repaidUnits
    // Assertion: repaidUnits > (_position.debt - badDebt) before the call
}
