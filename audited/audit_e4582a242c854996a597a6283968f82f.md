Audit Report

## Title
Unconditional zero-value `safeTransfer` on collateral token blocks bad-debt realization when token reverts on zero transfers - (File: src/Midnight.sol)

## Summary

`Midnight.liquidate` explicitly documents that passing `seizedAssets=0, repaidUnits=0` realizes bad debt "with 0 token transferred" (NatSpec line 577). However, `SafeTransferLib.safeTransfer(collateralToken, receiver, seizedAssets)` is called unconditionally at line 696 regardless of whether `seizedAssets` is zero. If the collateral token reverts on zero-value `transfer` calls — a known behavior of several deployed ERC20s — the entire transaction reverts, permanently preventing bad-debt realization for that market.

## Finding Description

**Exact code path:**

1. Caller invokes `liquidate(market, collateralIndex, 0, 0, borrower, ...)`.
2. Line 595: `atMostOneNonZero(0, 0)` returns `true` — both-zero is explicitly permitted. [1](#0-0) 
3. Line 596: `_position.debt > 0` passes — borrower has outstanding debt.
4. Lines 607–618: while-loop computes `badDebt > 0` (borrower's collateral at `maxLif` is insufficient to cover debt). [2](#0-1) 
5. Lines 620–624: `!liquidationLocked && originalDebt > maxDebt` passes — position is unhealthy.
6. Lines 626–641: `if (badDebt > 0)` block executes — `_position.debt`, `_marketState.lossFactor`, `_marketState.totalUnits`, and `_marketState.continuousFeeCredit` are all updated correctly. [3](#0-2) 
7. Line 643: `if (repaidUnits > 0 || seizedAssets > 0)` is **false** — accounting block is correctly skipped; `seizedAssets` remains `0`. [4](#0-3) 
8. **Line 696:** `SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets)` is called **unconditionally** with `seizedAssets = 0`. No guard exists. [5](#0-4) 
9. Inside `safeTransfer` (lines 15–20): `token.call(abi.encodeCall(IERC20.transfer, (receiver, 0)))` is issued. If the token reverts on zero-value transfers, `success == false`, and the assembly block re-reverts with the token's revert data. [6](#0-5) 
10. The entire transaction reverts — including all state mutations from step 6 — so bad debt is never realized.

**Secondary issue:** Line 717 has the same pattern for `safeTransferFrom(loanToken, payer, address(this), 0)`, but the collateral transfer at line 696 is reached first and is the primary blocker. [7](#0-6) 

**Root cause:** Missing `if (seizedAssets > 0)` guard before line 696, directly contradicting the NatSpec promise at lines 577–579. [8](#0-7) 

**Existing checks reviewed and insufficient:**
- `atMostOneNonZero` (line 595) explicitly permits both-zero — it is not a guard against this path.
- `if (repaidUnits > 0 || seizedAssets > 0)` (line 643) correctly skips accounting but does not guard the transfer at line 696.

## Impact Explanation

For any market whose collateral token reverts on `transfer(addr, 0)`, bad-debt realization via `liquidate(0, 0, ...)` is permanently and completely blocked. The `lossFactor` and `totalUnits` are never updated, so lenders' credit is never written down to reflect the loss. The borrower's position remains technically liquidatable but every realization attempt reverts. This is a permanent freeze of a core protocol invariant: *unhealthy positions must remain liquidatable*. The collateral token is immutable after market creation, making the block irreversible for the affected market.

## Likelihood Explanation

Two preconditions are required: (1) a market is created with a collateral token that reverts on `transfer(addr, 0)` — this is a documented, deployed behavior (e.g., LEND, some cTokens, and others); (2) a borrower accumulates bad debt in that market. Both are reachable without any privileged action. Market creation is permissionless; bad debt accumulates naturally from price movements. Once the collateral token is set, it cannot be changed, making the block permanent and repeatable on every subsequent liquidation attempt.

## Recommendation

Add a zero-value guard before both unconditional transfer calls:

```solidity
// Line 696 — guard collateral transfer
if (seizedAssets > 0) {
    SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets);
}

// Line 717 — guard loan token transfer
if (repaidUnits > 0) {
    SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), repaidUnits);
}
```

This matches the NatSpec promise at lines 577–579 and is consistent with the existing `if (repaidUnits > 0 || seizedAssets > 0)` accounting guard at line 643.

## Proof of Concept

**Minimal Foundry test plan:**

1. Deploy a mock ERC20 collateral token whose `transfer` reverts when `amount == 0`.
2. Create a Midnight market using this token as collateral.
3. Have a borrower supply collateral and borrow; then simulate a price drop so the borrower's collateral value at `maxLif` is less than their debt (`badDebt > 0`).
4. Call `liquidate(market, collateralIndex, 0, 0, borrower, false, receiver, address(0), "")` from any unprivileged address.
5. **Expected (per NatSpec):** transaction succeeds, `lossFactor` and `totalUnits` are updated, bad debt is realized.
6. **Actual:** transaction reverts with the token's zero-transfer revert reason; no state is updated; bad debt is never realized.
7. Confirm by asserting `marketState.lossFactor` is unchanged after the call.
