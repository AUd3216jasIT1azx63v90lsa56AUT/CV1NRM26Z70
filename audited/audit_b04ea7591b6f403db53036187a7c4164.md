Audit Report

## Title
Arithmetic Underflow in `buyerAssetsToUnits`/`sellerAssetsToUnits` via `tick=0` Buy Offer DoSes Bundle Sell Flows - (File: src/periphery/ConsumableUnitsLib.sol)

## Summary
`TickLib.tickToPrice(0)` returns `0` due to integer rounding. A maker can post a buy offer with `tick=0`, `maxUnits=0`, `buy=true`. When such an offer appears in the `takes` array of `supplyCollateralAndSellWithUnitsTarget` or `supplyCollateralAndSellWithAssetsTarget`, the checked subtraction `offerPrice - settlementFee = 0 - positive` in `TakeAmountsLib` underflows and reverts. Because these calls occur outside the `try/catch` block, the revert propagates and aborts the entire bundle transaction.

## Finding Description

**Root cause chain:**

**1. `tickToPrice(0) == 0`**

`tickToPrice` computes:
```
1e36 / (1e18 + wExp(LN_ONE_PLUS_DELTA * (2910 - 0)))
```
With `LN_ONE_PLUS_DELTA ≈ 4.987e15` and exponent `≈ 14.5e18`, `wExp(14.5e18) ≈ 2e24`, so the division yields `≈ 5e11`. The subsequent `divHalfDownUnchecked(PRICE_ROUNDING_STEP)` where `PRICE_ROUNDING_STEP = 1e12` rounds `5e11 / 1e12 = 0`, and `0 * 1e12 = 0`. [1](#0-0) 

**2. `consumableUnits` routes to `buyerAssetsToUnits` when `maxUnits=0` and `buy=true`** [2](#0-1) 

**3. Underflow in `buyerAssetsToUnits` line 26**

With `offerPrice = 0` and any `settlementFee > 0`, the checked subtraction `offerPrice - settlementFee` underflows and reverts under Solidity 0.8. [3](#0-2) 

**4. Same underflow in `sellerAssetsToUnits` line 44** [4](#0-3) 

**5. In `supplyCollateralAndSellWithUnitsTarget`, `consumableUnits` is called at line 150, before the `try` block at line 152 — the revert is not caught** [5](#0-4) 

**6. In `supplyCollateralAndSellWithAssetsTarget`, both `sellerAssetsToUnits` (line 286) and `consumableUnits` (line 290) are outside the `try` block at line 292** [6](#0-5) 

**7. NatSpec explicitly acknowledges the gap**

`supplyCollateralAndSellWithUnitsTarget` documents `/// @dev Reverts if ConsumableUnitsLib reverts.` and `supplyCollateralAndSellWithAssetsTarget` documents `/// @dev Reverts if TakeAmountsLib or ConsumableUnitsLib reverts.` [7](#0-6) [8](#0-7) 

The guard inside `Midnight.take` against `offerPrice < settlementFee` is irrelevant — `consumableUnits`/`sellerAssetsToUnits` fire and revert before `take` is ever reached.

## Impact Explanation
Any call to `supplyCollateralAndSellWithUnitsTarget` or `supplyCollateralAndSellWithAssetsTarget` that includes the crafted buy offer anywhere in its `takes` array reverts entirely. Legitimate borrowers attempting to sell units against a bundle of buy offers are completely blocked whenever the attacker's offer appears in the bundle. This is a targeted, repeatable DoS on core bundle sell functionality.

## Likelihood Explanation
All preconditions are trivially achievable by any unprivileged maker: `offer.buy = true`, `offer.tick = 0` (valid since `0 <= MAX_TICK = 5820` and `0 % tickSpacing == 0` for any spacing), `offer.maxUnits = 0`. Every market with a non-zero settlement fee satisfies `settlementFee > 0`. The attacker risks no capital — the offer itself would also revert inside `Midnight.take`, but its mere presence in a bundle is sufficient to abort the whole transaction. Off-chain aggregators that enumerate all posted buy offers would naturally include this offer. The attack is repeatable at gas cost only.

## Recommendation
Move the `consumableUnits` and `sellerAssetsToUnits` calls inside the `try/catch` block, or wrap them in a separate `try/catch` that skips the offer on revert. Alternatively, add a pre-check that skips any offer where `TickLib.tickToPrice(offer.tick) < settlementFee` before invoking `TakeAmountsLib` functions, consistent with the guard already present inside `Midnight.take`.

## Proof of Concept
1. Deploy a market with any non-zero settlement fee.
2. Attacker calls `Midnight.makeOffer` (or equivalent) with `tick=0`, `maxUnits=0`, `maxAssets=0`, `buy=true`.
3. A legitimate taker constructs a `takes` array that includes the attacker's offer alongside valid buy offers and calls `supplyCollateralAndSellWithUnitsTarget` or `supplyCollateralAndSellWithAssetsTarget`.
4. The transaction reverts at the `ConsumableUnitsLib.consumableUnits` / `TakeAmountsLib.sellerAssetsToUnits` call with an arithmetic underflow, before any `try/catch` is reached.
5. Confirm: a unit test calling `TickLib.tickToPrice(0)` returns `0`; calling `TakeAmountsLib.buyerAssetsToUnits` or `sellerAssetsToUnits` with `offerPrice=0` and any `settlementFee > 0` reverts.
