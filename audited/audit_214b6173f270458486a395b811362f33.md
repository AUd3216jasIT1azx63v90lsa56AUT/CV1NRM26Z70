### Title
Taker-Supplied `proof.length > 20` Causes `TreeTooHigh` to Propagate Through `Midnight.take` Instead of `RatifierFail` - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
An unprivileged taker can supply `ratifierData` encoding a 21-element Merkle proof, causing `HashLib.offerTreeTypeHash(21)` to revert with `TreeTooHigh` inside `EcrecoverRatifier.isRatified`. Because `Midnight.take` wraps the ratifier call with a plain `require(... == CALLBACK_SUCCESS, RatifierFail())`, a revert inside `isRatified` is not caught and re-thrown as `RatifierFail`; it propagates verbatim as `TreeTooHigh`. Callers that only catch `RatifierFail` (or a known set of `Midnight` selectors) will encounter an unhandled revert.

### Finding Description

**Exact code path:**

`Midnight.take` at line 356 calls the ratifier:

```solidity
require(IRatifier(offer.ratifier).isRatified(offer, ratifierData) == CALLBACK_SUCCESS, RatifierFail());
``` [1](#0-0) 

The `require` only converts a *wrong return value* into `RatifierFail`. A revert originating inside `isRatified` bypasses this conversion entirely and propagates as-is.

Inside `EcrecoverRatifier.isRatified`, the execution order is:

1. **Line 37** — `HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof)` — the taker controls all four arguments via `ratifierData`.
2. **Line 38** — `isRootCanceled[offer.maker][root]` — taker controls `root`.
3. **Line 39** — `HashLib.offerTreeTypeHash(proof.length)` — **reverts with `TreeTooHigh()` when `proof.length > 20`**. [2](#0-1) 

`HashLib.offerTreeTypeHash` has no branch for values above 20 and unconditionally reverts: [3](#0-2) 

**Why `isLeaf` does not block this:**

`HashLib.isLeaf` only checks `leafIndex >> proof.length == 0` (the `LeafIndexOutOfRange` guard) and then iterates over the proof. It imposes **no upper bound on `proof.length`**. With `leafIndex = 0` and `proof.length = 21`, the guard passes (`0 >> 21 == 0`). The attacker precomputes a valid 21-level Merkle root by starting from `HashLib.hashOffer(offer)` and applying 21 chosen sibling hashes; `isLeaf` returns `true`. [4](#0-3) 

**Why the signature check does not block this:**

`offerTreeTypeHash` is called at line 39, **before** `ecrecover` at line 42. The revert fires before any signature validation, so the attacker needs no knowledge of the maker's private key. [5](#0-4) 

**Attacker-controlled inputs:**
- `ratifierData` is fully supplied by the taker (any address that calls `Midnight.take` with `taker == msg.sender`).
- `proof` is decoded from `ratifierData`; setting `proof.length = 21` is trivially encoded.

### Impact Explanation
`Midnight.take` reverts with `HashLib.TreeTooHigh` instead of `IMidnight.RatifierFail`. Any smart-contract caller (router, aggregator, multicall wrapper) that uses `try/catch` or selector-based error handling expecting only documented `Midnight` errors will encounter an unhandled revert. The protocol's implicit invariant — that all ratifier-related failures surface as `RatifierFail` — is broken for `EcrecoverRatifier`.

### Likelihood Explanation
The precondition is minimal: the taker must call `Midnight.take` with an offer whose `ratifier` is `EcrecoverRatifier` and supply `ratifierData` encoding a 21-element proof array. No privileged access, no special state, and no valid signature are required. The trigger is deterministic and repeatable on every call.

### Recommendation
Add a `proof.length <= 20` guard in `EcrecoverRatifier.isRatified` before calling `offerTreeTypeHash`, reverting with a ratifier-specific error (e.g., `InvalidProof()`) so the failure is caught by `Midnight.take`'s `RatifierFail` path. Alternatively, wrap the `offerTreeTypeHash` call in a check and revert with `InvalidProof()` on out-of-range lengths. The fix should be placed before the `isLeaf` call to short-circuit early:

```solidity
require(proof.length <= 20, InvalidProof());
```

### Proof of Concept

```solidity
function testTreeTooHighPropagates() public {
    // Setup: valid offer with EcrecoverRatifier, maker authorizes ratifier
    Offer memory offer = makeOffer(lender);
    vm.prank(lender);
    midnight.setIsAuthorized(address(ecrecoverRatifier), true, lender);

    // Build a 21-element proof with a self-consistent root
    bytes32[] memory proof = new bytes32[](21);
    bytes32 current = HashLib.hashOffer(offer);
    for (uint256 i = 0; i < 21; i++) {
        proof[i] = bytes32(uint256(i + 1)); // arbitrary siblings
        current = keccak256(abi.encodePacked(current, proof[i])); // leafIndex=0 → always left
    }
    bytes32 root = current; // isLeaf will return true

    // Dummy signature (ecrecover is never reached)
    Signature memory sig = Signature({v: 27, r: bytes32(0), s: bytes32(0)});
    bytes memory ratifierData = abi.encode(sig, root, uint256(0), proof);

    // Assert: Midnight.take reverts with TreeTooHigh, NOT RatifierFail
    vm.expectRevert(HashLib.TreeTooHigh.selector);
    vm.prank(address(this)); // unprivileged taker
    midnight.take(offer, ratifierData, 0, address(this), address(this), address(0), hex"");
}
```

Expected assertions:
- Revert selector is `HashLib.TreeTooHigh` (not `IMidnight.RatifierFail`).
- `Midnight.take` reverts before any state change.
- Replacing `proof.length` with 20 causes the call to proceed past `offerTreeTypeHash` (reaching the signature check instead).

    function isLeaf(bytes32 root, bytes32 leafHash, uint256 leafIndex, bytes32[] memory proof)
        internal
        pure
        returns (bool)
    {
        require(leafIndex >> proof.length == 0, LeafIndexOutOfRange());
        bytes32 currentHash = leafHash;
        for (uint256 i = 0; i < proof.length; i++) {
            currentHash = (leafIndex >> i) & 1 == 0 ? hashNode(currentHash, proof[i]) : hashNode(proof[i], currentHash);
        }
        return currentHash == root;
    }
