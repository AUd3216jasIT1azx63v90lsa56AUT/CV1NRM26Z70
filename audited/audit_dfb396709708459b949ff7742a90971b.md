### Title
`HashLib.TreeTooHigh` Leaks Through `take()` Instead of Wrapping as `RatifierFail` - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
In `EcrecoverRatifier.isRatified`, `HashLib.isLeaf` is called before `HashLib.offerTreeTypeHash`. An unprivileged taker can supply a `ratifierData` with a self-constructed valid Merkle proof of length 21, causing `isLeaf` to return `true`, after which `offerTreeTypeHash(21)` reverts with `TreeTooHigh`. Because `Midnight.take` invokes `isRatified` via a direct external call and only wraps a *return-value mismatch* in `RatifierFail`, the raw `TreeTooHigh` panic propagates to the caller instead of the expected `RatifierFail`.

### Finding Description

**Exact code path:**

`Midnight.sol` line 356 calls `isRatified` directly: [1](#0-0) 

Inside `EcrecoverRatifier.isRatified`, the order of operations is:
1. Line 37 — `HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof)` is evaluated first.
2. Line 39 — `HashLib.offerTreeTypeHash(proof.length)` is evaluated second. [2](#0-1) 

`isLeaf` imposes only one guard on `proof.length`: `require(leafIndex >> proof.length == 0, LeafIndexOutOfRange())`. With `proof.length = 21` and `leafIndex = 0`, this check passes (`0 >> 21 == 0`). The loop then runs 21 iterations over attacker-supplied siblings. [3](#0-2) 

Because the taker fully controls `root`, `leafIndex`, and `proof` inside `ratifierData`, they can pre-compute a valid 21-element Merkle chain:
- Set `leafIndex = 0`.
- Choose 21 arbitrary `proof[i]` values.
- Compute `h_0 = hashOffer(offer)`, then iteratively `h_{i+1} = hashNode(h_i, proof[i])`.
- Set `root = h_21`.

`isLeaf` returns `true`. Execution reaches line 39, where `offerTreeTypeHash(21)` hits the `revert TreeTooHigh()` branch: [4](#0-3) 

This revert propagates out of `isRatified` as `TreeTooHigh`. Back in `take()`, the `require(..., RatifierFail())` wrapper only fires when `isRatified` *returns* a wrong value — it does not catch a revert from within `isRatified`. So `TreeTooHigh` bubbles all the way to the external caller. [1](#0-0) 

No state is mutated before this point; `consumed` is updated only after line 356. [5](#0-4) 

### Impact Explanation
The concrete scoped impact is confirmed: any taker calling `take()` with a crafted 21-element proof receives a `HashLib.TreeTooHigh` revert instead of `IMidnight.RatifierFail`. This violates the protocol's error-handling contract. Smart-contract integrators that catch `RatifierFail` to distinguish "bad ratifier data" from other failures will silently miss this case, potentially causing unexpected control-flow in downstream contracts. Internal library error selectors are also exposed to external callers, leaking implementation details.

### Likelihood Explanation
No privileged access, no special market state, and no prior setup is required. Any taker interacting with any offer that uses `EcrecoverRatifier` can trigger this with a single crafted `ratifierData`. The construction of a valid 21-element proof is trivial off-chain arithmetic. The condition is repeatable on every call.

### Recommendation
Add an explicit length guard at the top of `isRatified` (or at the top of `isLeaf`) before any Merkle traversal:

```solidity
require(proof.length <= 20, TreeTooHigh());
```

Alternatively, reorder the calls so `offerTreeTypeHash(proof.length)` is evaluated *before* `isLeaf`, so the height cap is enforced first and the revert is still an `EcrecoverRatifier`-owned error that `take()` can wrap correctly.

### Proof of Concept

```solidity
function testTreeTooHighLeaksNotRatifierFail() public {
    // Arrange: valid offer with EcrecoverRatifier as ratifier (standard setup).
    // Build a 21-element proof where isLeaf returns true.
    bytes32[] memory proof = new bytes32[](21);
    // proof[i] can be any value; we derive root from them.
    bytes32 leafHash = HashLib.hashOffer(lenderOffer);
    bytes32 current = leafHash;
    for (uint256 i = 0; i < 21; i++) {
        proof[i] = bytes32(uint256(i + 1)); // arbitrary siblings
        current = keccak256(abi.encodePacked(current, proof[i])); // leafIndex=0 → always left
    }
    bytes32 craftedRoot = current;

    Signature memory sig; // zero sig — irrelevant, revert happens before sig check
    bytes memory ratifierData = abi.encode(sig, craftedRoot, uint256(0), proof);

    // Act + Assert: revert must be TreeTooHigh, NOT RatifierFail
    vm.expectRevert(HashLib.TreeTooHigh.selector);
    vm.prank(borrower);
    midnight.take(lenderOffer, ratifierData, 0, borrower, borrower, address(0), hex"");

    // Negative assertion: confirm it is NOT wrapped as RatifierFail
    // (vm.expectRevert above would fail if selector were IMidnight.RatifierFail)
}
```

Expected assertion: the call reverts with `HashLib.TreeTooHigh` (not `IMidnight.RatifierFail`), confirming the invariant violation.
