Audit Report

## Title
Authorized Operator Can Grief Victim's Pre-Signed EcrecoverAuthorizer Nonce via No-Op Authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` allows any address already authorized by the victim in Midnight to sign and submit an `Authorization` struct on the victim's behalf. Because no check enforces a meaningful state change, an authorized operator can craft a no-op `Authorization` (e.g., `isAuthorized=false` for an address that was never authorized) to consume the victim's current nonce, permanently invalidating any pre-signed `Authorization{nonce=N}` the victim intended to submit.

## Finding Description
**Root cause:** `EcrecoverAuthorizer.setIsAuthorized` unconditionally increments `nonce[authorization.authorizer]` on every successful call regardless of whether the downstream `Midnight.setIsAuthorized` call produces any state change.

**Code path:**

`src/periphery/EcrecoverAuthorizer.sol` line 26:
```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```
The nonce is incremented before any check that the call is meaningful. [1](#0-0) 

Lines 33–36 allow the signer to be either the authorizer themselves **or** any address already authorized by the authorizer in Midnight:
```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [2](#0-1) 

The downstream call at lines 46–47 simply assigns the mapping value in Midnight with no prior-state check: [3](#0-2) 

`Midnight.setIsAuthorized` at line 733 performs an unconditional assignment, and the contract explicitly documents "No-ops are allowed" at line 168: [4](#0-3) [5](#0-4) 

**Exploit flow:**
1. Victim `V` authorizes attacker `A` in Midnight: `isAuthorized[V][A] = true` (normal relayer/adapter setup).
2. `V` pre-signs off-chain: `Authorization{authorizer=V, authorized=criticalContract, isAuthorized=true, nonce=N}`.
3. `A` crafts: `Authorization{authorizer=V, authorized=neverAuthorizedAddr, isAuthorized=false, nonce=N, deadline=future}` and signs it with `A`'s own key.
4. `A` calls `ecrecoverAuthorizer.setIsAuthorized(noOpAuth, aSig)`.
5. All checks pass: deadline valid; `nonce[V]==N` (incremented to N+1); `ecrecover` returns `A`; `isAuthorized[V][A]==true`.
6. `Midnight.setIsAuthorized(neverAuthorizedAddr, false, V)` executes as a no-op.
7. `nonce[V]` is now `N+1`. `V`'s pre-signed `Authorization{nonce=N}` now reverts with `InvalidNonce`.

**Why existing checks fail:** The Certora spec `certora/specs/EcrecoverAuthorizer.spec` only verifies that a successful call increments the authorizer's nonce and that expired/wrong/reused nonces revert. It does not assert that nonce consumption requires a meaningful state change in Midnight. [6](#0-5) 

## Impact Explanation
The victim's pre-signed `Authorization{nonce=N}` is permanently invalidated. If the victim cannot immediately re-sign (hardware wallet unavailable, pre-signed batch, or a contract authorizer that pre-committed to nonce `N`), their ability to delegate via `EcrecoverAuthorizer` is frozen until a new signature is produced. The attack is repeatable: the attacker can consume every subsequent nonce the victim pre-signs for as long as the attacker remains authorized in Midnight, creating a sustained griefing/DoS on the victim's off-chain signing workflow.

## Likelihood Explanation
**Required preconditions:**
- Victim has authorized the attacker in Midnight (`isAuthorized[victim][attacker] = true`) — a routine protocol interaction when setting up relayers, smart contract wallets, or protocol adapters.
- Victim has a pre-signed `Authorization` in flight (in a mempool, stored in a UI, or shared with a relayer).

Both conditions are common in normal protocol usage. The attack costs one transaction, requires no special privileges beyond being an authorized operator, and is repeatable indefinitely until the victim revokes the attacker's Midnight authorization.

## Recommendation
Add a meaningful-state-change guard in `EcrecoverAuthorizer.setIsAuthorized` before incrementing the nonce. Specifically, read the current value of `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, authorization.authorized)` and require that `authorization.isAuthorized != currentValue`. This ensures the nonce is only consumed when the call actually changes authorization state:

```solidity
bool current = IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, authorization.authorized);
require(authorization.isAuthorized != current, NoStateChange());
// ... then increment nonce and proceed
```

Alternatively, move the nonce increment to after the downstream call and revert if no state change occurred.

## Proof of Concept
Minimal Foundry test (extend `SetIsAuthorizedWithSigTest`):

```solidity
function testNonceGriefByAuthorizedOperator() public {
    // Step 1: borrower authorizes ecrecoverAuthorizer and attacker in Midnight
    vm.prank(borrower);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
    vm.prank(borrower);
    midnight.setIsAuthorized(attacker, true, borrower);

    // Step 2: borrower pre-signs Authorization{nonce=0} to grant lender
    Authorization memory victimAuth = makeAuthorization(borrower, lender, true); // nonce=0
    Signature memory victimSig = signAuthorization(victimAuth, borrower);

    // Step 3: attacker crafts no-op Authorization{nonce=0} signed with attacker's key
    Authorization memory noOpAuth = Authorization({
        authorizer: borrower,
        authorized: address(0xdead), // never authorized
        isAuthorized: false,         // no-op: already false
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory attackerSig = signAuthorization(noOpAuth, attacker);

    // Step 4: attacker submits no-op, consuming nonce 0
    ecrecoverAuthorizer.setIsAuthorized(noOpAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(borrower), 1); // nonce consumed

    // Step 5: victim's pre-signed auth now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

    // Midnight state unchanged — pure grief
    assertEq(midnight.isAuthorized(borrower, lender), false);
}
```

/// EcrecoverAuthorizer increments nonce on success and does not change other nonces.
rule effects(env e, EcrecoverAuthorizer.Authorization authorization, EcrecoverAuthorizer.Signature signature, address other) {
    require other != authorization.authorizer;
    uint256 nonceBefore = nonce(authorization.authorizer);
    uint256 otherNonceBefore = nonce(other);

    setIsAuthorized(e, authorization, signature);

    assert nonce(authorization.authorizer) == nonceBefore + 1;
    assert nonce(other) == otherNonceBefore;
}
