Audit Report

## Title
Missing `loanToken != address(0)` validation in `touchMarket` allows creation of permanently broken, undeletable markets - (`src/Midnight.sol`)

## Summary
`touchMarket` validates collateral tokens, LLTV, maxLif, and maturity but contains no guard against `market.loanToken == address(0)`. Any unprivileged caller can create a market with `loanToken = address(0)` that succeeds silently and is permanently recorded in state. Every subsequent `take()`, `withdraw()`, and `repay()` on that market reverts with `SafeTransferLib.NoCode()` because `address(0).code.length == 0`, and the market cannot be deleted once created. Additionally, `TakeAmountsLib.buyerAssetsToUnits` returns nonzero values for offers on this market, falsely signaling to integrators that takes are executable.

## Finding Description

**Root cause — `touchMarket` (`src/Midnight.sol`, lines 755–791):**

The creation guard checks maturity, collateral count, sort order, LLTV tier, and maxLif, but contains no `require(market.loanToken != address(0))`: [1](#0-0) 

The collateral sort check (`collateralToken > previousCollateralToken`, where `previousCollateralToken` starts at `address(0)`) only prevents collateral tokens from being `address(0)` — it does not protect the loan token field. [2](#0-1) 

When `market.loanToken = address(0)`, the market is created with all-zero settlement fees (since `defaultSettlementFeeCbp[address(0)]` returns a zero-initialized array) and is permanently stored: [3](#0-2) 

**Trigger — `take()` (`src/Midnight.sol`, lines 455–456):**

Every `take()` call reaches the token transfer step and calls `SafeTransferLib.safeTransferFrom` with `offer.market.loanToken`: [4](#0-3) 

`SafeTransferLib.safeTransferFrom` immediately reverts because `address(0).code.length == 0`: [5](#0-4) 

The same revert occurs in `withdraw()` (line 499) and `repay()` (line 520), which also call `SafeTransferLib.safe{Transfer,TransferFrom}` on `market.loanToken`.

**Market immutability confirmed by Certora spec:** [6](#0-5) 

**Misleading helper — `TakeAmountsLib.buyerAssetsToUnits` (`src/periphery/TakeAmountsLib.sol`, lines 17–30):**

With `settlementFee = 0` (from the zero-initialized market) and `buyerPrice = offerPrice` (nonzero for any valid tick), the function returns a nonzero unit count — signaling to callers that a take is feasible when it will always revert: [7](#0-6) 

**Confirmed reachable — `test/SettersTest.sol`, lines 267–276:**

The test suite explicitly creates a market with `loanToken: address(0)` and calls `midnight.touchMarket(market)` successfully, confirming the path is exercised without error: [8](#0-7) 

## Impact Explanation

Any market created with `loanToken = address(0)` is permanently broken and unrecoverable: all `take()`, `withdraw()`, and `repay()` calls revert with `NoCode()`, the market state is immutable (no deletion mechanism exists per the Certora invariant), and `TakeAmountsLib.buyerAssetsToUnits` returns nonzero values that falsely indicate takes are executable. Note: because debt can only be created via `take()` (which always reverts), no debt positions can form, so collateral deposited to such a market remains withdrawable. The primary concrete impacts are (1) permanent unrecoverable corruption of protocol state — a broken market ID is permanently occupied and cannot be reclaimed — and (2) integrators relying on `TakeAmountsLib` will receive misleading feasibility signals, potentially causing failed transactions and degraded service. This matches the "permanent lock/freeze or unrecoverable corruption of user/project state" and "service unavailability or severe degradation" impact classes in RESEARCHER.md.

## Likelihood Explanation

`touchMarket` is a public, permissionless function — no role, signature, or special permission is required. The precondition (supply `loanToken = address(0)` with any valid collateral/LLTV/maturity) is trivially satisfiable in a single transaction. The attack is repeatable for any `(address(0), maturity)` pair not yet created, and the broken market is immutable once deployed. The test suite already exercises this exact path without error, confirming zero friction for an attacker.

## Recommendation

Add a `require(market.loanToken != address(0), InvalidLoanToken())` check at the top of the market-creation branch in `touchMarket`, before any state is written:

```solidity
if (marketState[id].tickSpacing == 0) {
    require(market.loanToken != address(0), InvalidLoanToken()); // add this
    require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
    // ...
}
```

Additionally, consider adding a `require(market.loanToken.code.length > 0)` check to ensure the loan token is a deployed contract, which would catch any non-contract address (not just `address(0)`).

function testBrokenMarketWithZeroLoanToken() public {
    CollateralParams[] memory cols = new CollateralParams[](1);
    cols[0] = CollateralParams({
        token: address(collateralToken1),
        lltv: 0.77e18,
        maxLif: maxLif(0.77e18, 0.25e18),
        oracle: address(oracle1)

    Market memory market = Market({
        loanToken: address(0),       // zero loan token
        maturity: block.timestamp + 1 days,
        collateralParams: cols,
        rcfThreshold: 0,
        enterGate: address(0),
        liquidatorGate: address(0)
    });

    // Step 1: market creation succeeds silently
    bytes32 id = midnight.touchMarket(market);
    assert(midnight.tickSpacing(id) > 0); // market is created

    // Step 2: any take() on this market reverts with NoCode()
    Offer memory offer = /* valid offer on this market */;
    vm.expectRevert(SafeTransferLib.NoCode.selector);
    midnight.take(offer, "", 1, address(this), address(this), address(0), "");

    // Step 3: market cannot be deleted — tickSpacing remains > 0 forever
    assert(midnight.tickSpacing(id) > 0);
}

    function buyerAssetsToUnits(address midnight, bytes32 id, Offer memory offer, uint256 targetBuyerAssets)
        internal
        view
        returns (uint256)
    {
        uint256 offerPrice = TickLib.tickToPrice(offer.tick);
        uint256 settlementFee =
            IMidnight(midnight).settlementFee(id, UtilsLib.zeroFloorSub(offer.market.maturity, block.timestamp));
        // Mirrors Midnight's computation to revert if offerPrice < settlementFee in case of a buy offer.
        uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
        uint256 buyerPrice = sellerPrice + settlementFee;
        require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne());
        return offer.buy ? targetBuyerAssets.mulDivUp(WAD, buyerPrice) : targetBuyerAssets.mulDivDown(WAD, buyerPrice);
    }
