Looking at the exact code path in `take()` for a sell offer with a fee-on-transfer loan token.

### Title
Fee-on-Transfer Loan Token Causes Seller to Receive Less Than `sellerAssets` in `take()` - (File: src/Midnight.sol)

### Summary
When `take()` is called on a sell offer (`offer.buy == false`) in a market whose `loanToken` is a fee-on-transfer ERC20, the direct `safeTransferFrom(loanToken, payer, receiver, sellerAssets)` at line 456 delivers fewer tokens to the seller's receiver than `sellerAssets` because the token silently deducts a fee in transit. The protocol's accounting (credit/debt) is already committed before the transfer and is never reconciled against the actual received amount, so the seller permanently loses the fee-deducted portion.

### Finding Description

**Code path:**

`take()` resolves `sellerAssets` at line 364: [1](#0-0) 

The seller's position is updated (credit decremented, debt incremented) at lines 412–414 before any token movement: [2](#0-1) 

Then two sequential `safeTransferFrom` calls are made at lines 455–456: [3](#0-2) 

`SafeTransferLib.safeTransferFrom` only verifies the call did not revert and returned `true`; it performs no balance-delta check: [4](#0-3) 

With a fee-on-transfer token, `transferFrom(payer, receiver, sellerAssets)` succeeds and returns `true`, but `receiver` receives `sellerAssets × (1 − fee%)`. The protocol never checks the actual balance increase at `receiver`.

**Attacker-controlled inputs:**
- Any unprivileged taker calls `take()` with a valid sell offer in a market where `loanToken` is a fee-on-transfer token.
- No special permissions, no oracle manipulation, no reentrancy required.

**Why existing checks fail:**
- `touchMarket()` imposes no restriction on the `loanToken` type; any ERC20 address is accepted: [5](#0-4) 
- There is no pre/post balance snapshot anywhere in `take()`.
- The `live_context.json` core invariant "ERC20 transfer deltas must match accounting deltas" is violated, and the same file notes fee-on-transfer tokens "should be tested if not explicitly excluded" — confirming no explicit exclusion exists.

### Impact Explanation
The seller (maker of the sell offer) has their credit reduced by `sellerCreditDecrease` units and/or their debt increased by `sellerDebtIncrease` units — the full accounting cost of the trade — but receives only `sellerAssets × (1 − fee%)` tokens instead of `sellerAssets`. The difference (`sellerAssets × fee%`) is permanently lost: it is neither held by the protocol nor returned to the seller. This is a direct, irreversible loss of funds for the seller on every take in such a market.

### Likelihood Explanation
- **Precondition:** A market must exist with a fee-on-transfer token as `loanToken`. Market creation is permissionless (`touchMarket` is `public` with no token restrictions), so any user can create such a market.
- **Feasibility:** Tokens with transfer fees (e.g., STA, PAXG, or any custom ERC20 with a fee switch) are real and deployable. The taker needs only a valid sell offer and sufficient allowance.
- **Repeatability:** Every `take()` call on a sell offer in such a market triggers the loss. The loss scales linearly with `sellerAssets` and the token's fee rate.

### Recommendation
Record the receiver's token balance before the transfer and assert it increased by exactly `sellerAssets`:

```solidity
uint256 balanceBefore = IERC20(offer.market.loanToken).balanceOf(receiver);
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
require(
    IERC20(offer.market.loanToken).balanceOf(receiver) - balanceBefore == sellerAssets,
    InsufficientReceivedAssets()
);
```

Alternatively, explicitly document that fee-on-transfer tokens are unsupported and add a token registry or validation step in `touchMarket`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
// ... standard test imports

contract FeeOnTransferToken is ERC20 {
    uint256 public feeBps = 100; // 1% fee
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount * feeBps / 10000;
        super.transferFrom(from, to, amount - fee);
        // fee is burned / kept by token
        _burn(from, fee); // or transfer to fee collector
        return true;
    }
}

contract FeeOnTransferTakeTest is Test {
    function testSellerReceivesLessThanSellerAssets() public {
        // 1. Deploy FeeOnTransferToken as loanToken
        // 2. Create market with this token
        // 3. Seller (maker) creates sell offer with receiverIfMakerIsSeller = sellerReceiver
        // 4. Taker calls take() with the sell offer
        // 5. Record sellerReceiver balance before and after
        
        uint256 balanceBefore = feeToken.balanceOf(sellerReceiver);
        (uint256 buyerAssets, uint256 sellerAssets) = midnight.take(
            sellOffer, "", units, taker, address(0), address(0), ""
        );
        uint256 balanceAfter = feeToken.balanceOf(sellerReceiver);
        
        // ASSERTION: seller received less than sellerAssets — proves the bug
        assertLt(balanceAfter - balanceBefore, sellerAssets, "seller shortchanged by fee");
        // ASSERTION: the deficit equals the fee
        assertEq(balanceAfter - balanceBefore, sellerAssets * 99 / 100, "1% fee deducted");
    }
}
```

Expected result: `balanceAfter - balanceBefore < sellerAssets`, confirming the seller is shortchanged by the token transfer fee while their on-chain credit/debt accounting reflects the full `sellerAssets` amount.

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
