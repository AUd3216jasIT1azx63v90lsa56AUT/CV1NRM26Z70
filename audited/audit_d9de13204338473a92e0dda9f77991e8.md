### Title
Fee-on-Transfer Loan Token Breaks `withdrawable` Accounting in `liquidate()`, Causing Lender Insolvency - (File: src/Midnight.sol)

### Summary

In `liquidate()`, `marketState[id].withdrawable` and `position[id][borrower].debt` are updated by the full `repaidUnits` value before `SafeTransferLib.safeTransferFrom` is called. `SafeTransferLib.safeTransferFrom` only verifies that `transferFrom` does not revert and returns `true`; it does not measure the actual tokens received. If the loan token charges a fee on transfer, Midnight receives fewer tokens than `repaidUnits`, permanently inflating `withdrawable` beyond the contract's real balance and making the market insolvent.

`SafeTransferLib.safeTransferFrom` only checks for revert and a `bool` return value; it performs no before/after balance measurement: [3](#0-2) 

**Market creation has no loan token restriction.** `touchMarket()` validates collateral params, LLTV, and maturity, but imposes no constraint on `market.loanToken`: [4](#0-3) 

**Exploit flow:**

1. Attacker (or any user) calls `touchMarket(market)` where `market.loanToken` is a fee-on-transfer ERC20 (e.g., 1% fee on every `transferFrom`).
2. Lenders supply credit and borrowers take debt in this market through normal `take()` calls.
3. The borrower's position becomes liquidatable (price drop or post-maturity).
4. Liquidator calls `liquidate(market, collateralIndex, 0, repaidUnits, borrower, false, receiver, address(0), "")`.
5. At line 675: `_marketState.withdrawable += repaidUnits` — accounting records full `repaidUnits`.
6. At line 676: `_position.debt -= repaidUnits` — borrower's debt is fully reduced.
7. At line 717: `safeTransferFrom(loanToken, payer, address(this), repaidUnits)` — the fee-on-transfer token delivers only `repaidUnits * (1 - fee_rate)` to Midnight. The call succeeds and returns `true`.
8. Post-call: `marketState[id].withdrawable` exceeds `loanToken.balanceOf(address(midnight))` by `repaidUnits * fee_rate`.

**Why existing checks do not stop it:**

- `SafeTransferLib` only checks for revert and `bool` return — it cannot detect a fee-on-transfer shortfall.
- There is no balance-before/after guard anywhere in `liquidate()`.
- The Certora formal verification in `certora/specs/Solvency.spec` explicitly assumes tokens transfer without fees (`"no fee taking from sender or receiver"`), so the solvency invariant is not proven for fee-on-transfer tokens: [5](#0-4) 

- The same pattern exists in `repay()` (lines 508–520), compounding the exposure. [6](#0-5) 

### Impact Explanation

After one or more liquidations with a fee-on-transfer loan token, `marketState[id].withdrawable` exceeds the actual `loanToken` balance held by Midnight. Lenders calling `withdraw()` reduce `withdrawable` and trigger `safeTransfer(loanToken, receiver, units)`. The first lenders to withdraw succeed; later lenders find the contract's real balance insufficient and their `withdraw()` calls revert. The market is insolvent: lenders collectively cannot redeem all their credit. The shortfall equals the cumulative fee taken across all affected `liquidate()` (and `repay()`) calls.

### Likelihood Explanation

**Preconditions:**
- A market must exist with a fee-on-transfer loan token. Market creation is permissionless (`touchMarket` is public), so any user can create such a market.
- At least one liquidation (or repayment) must occur in that market.

**Feasibility:** High. No privileged action is required. The attacker can be the market creator, a lender, a borrower, or a liquidator. The exploit is repeatable: every `liquidate()` call with `repaidUnits > 0` widens the gap. The `live_context.json` explicitly lists fee-on-transfer tokens as a concern that "should be tested if not explicitly excluded," and no code-level exclusion exists. [7](#0-6) 

### Recommendation

Add a balance-before/after check in both `liquidate()` and `repay()` to measure the actual tokens received, and use that measured amount for accounting updates instead of the nominal `repaidUnits`/`units` parameter. Alternatively, explicitly document and enforce (via a registry or allowlist) that only standard ERC20 tokens without transfer fees may be used as loan tokens, and add an integration test with a fee-on-transfer mock to enforce this invariant.

### Proof of Concept

```solidity
// Foundry unit test sketch
contract FeeOnTransferLoanToken is ERC20 {
    uint256 public feeBps = 100; // 1% fee
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount * feeBps / 10000;
        super.transferFrom(from, to, amount - fee); // deliver less
        _burn(from, fee);                            // fee destroyed
        return true;
    }
}

function testFeeOnTransferLiquidateInsolvency() public {
    FeeOnTransferLoanToken feeToken = new FeeOnTransferLoanToken();
    // Build market with feeToken as loanToken
    // Setup: lender supplies credit, borrower takes debt (units = 1000e18)
    // Make position liquidatable (drop oracle price)
    uint256 repaidUnits = 500e18;
    uint256 balanceBefore = feeToken.balanceOf(address(midnight));
    midnight.liquidate(market, 0, 0, repaidUnits, borrower, false, receiver, address(0), "");
    uint256 balanceAfter = feeToken.balanceOf(address(midnight));
    // Assert: withdrawable increased by repaidUnits but balance only increased by repaidUnits * 0.99
    assertEq(midnight.withdrawable(id), withdrawableBefore + repaidUnits);          // accounting: full amount
    assertEq(balanceAfter - balanceBefore, repaidUnits * 99 / 100);                 // actual: 1% short
    // Invariant violation: withdrawable > actual balance
    assertGt(midnight.withdrawable(id), feeToken.balanceOf(address(midnight)));
}
```

```text
    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
```
