### Title
Factory Allowlist Bypassed via Direct `initialize()` on Cloned Implementation — (File: `src/ConfidencePool.sol`)

### Summary
`ConfidencePool.initialize()` carries no factory-only access control. Any caller can clone the public `poolImplementation` address and call `initialize()` directly, bypassing the factory's `allowedStakeToken` guard and creating a rogue pool that silently under-pays stakers through fee-on-transfer mechanics.

### Finding Description
`ConfidencePoolFactory.createPool()` enforces a critical allowlist check before deploying any pool:

```solidity
if (!allowedStakeToken[stakeToken]) revert StakeTokenNotAllowed();
```

This guard exists specifically because fee-on-transfer and rebasing tokens cause the pool's internal accounting to diverge from its actual token balance, silently under-paying every claim and withdraw. The factory is the intended and documented sole entry point for pool creation.

However, `ConfidencePool.initialize()` is declared `external` and carries only OpenZeppelin's `initializer` modifier — a one-time-call guard, not a caller restriction:

```solidity
function initialize(
    address agreement_,
    address stakeToken_,
    ...
) external initializer {
```

Because `poolImplementation` is a public state variable on the factory, any caller can:
1. Read the implementation address from `factory.poolImplementation()`
2. Deploy a clone via `Clones.clone(poolImplementation)` (or any EIP-1167 cloner)
3. Call `initialize()` on that clone with an arbitrary `stakeToken_` — including a fee-on-transfer token not in the allowlist

The rogue pool passes all internal `initialize()` validations (agreement validity is checked against the real registry), uses identical bytecode to a legitimate pool, and is indistinguishable to a staker who does not verify the factory's `_poolsByAgreement` registry.

Three factory-level guards are bypassed simultaneously:
- `allowedStakeToken[stakeToken]` — the primary allowlist
- `whenNotPaused` — factory pause is irrelevant to a direct clone
- `IAgreement(agreement).owner() != msg.sender` — agreement ownership check

### Impact Explanation
With a fee-on-transfer token admitted through the bypass, every outbound transfer from the pool (withdraw, claimSurvived, claimExpired, claimAttackerBounty, claimCorrupted, sweepUnclaimedBonus) silently deducts the fee from the staker's proceeds. The inbound `stake()` and `contributeBonus()` paths use balance-diff accounting and correctly record only `received`, but the claim/withdraw paths transfer the recorded `eligibleStake` amount directly:

```solidity
stakeToken.safeTransfer(msg.sender, amount);   // withdraw
stakeToken.safeTransfer(msg.sender, payout);   // claimSurvived / claimExpired
```

Each outbound transfer triggers the fee, so every staker receives less than their recorded entitlement. With multiple stakers and a bonus pool, the cumulative fee drain can leave the last claimant unable to recover their full principal. Additionally, the attacker who initializes the rogue pool can set themselves as `outcomeModerator_`, gaining unilateral control over outcome flagging (subject to live registry state) and the ability to name themselves as the good-faith attacker to redirect the entire pool via `claimAttackerBounty`.

### Likelihood Explanation
The `poolImplementation` address is publicly readable. Cloning an EIP-1167 proxy is a single library call. No privileged access, special token, or registry manipulation is required for the base exploit (fee-on-transfer token + real registry). The only friction is attracting stakers to the rogue pool, but a pool created for a real, high-profile agreement with a plausible-looking token address is a credible lure. The existing fee-on-transfer test suite (`ConfidencePool.fot.t.sol`) demonstrates the exact clone-then-initialize pattern works without the factory.

### Recommendation
Restrict `initialize()` to the factory by recording the factory address in the implementation at construction time and asserting `msg.sender == factory` inside `initialize()`. Because the implementation is deployed before the factory, the factory address can be passed as a constructor argument to the implementation or set via a one-time `setFactory` call gated by `_disableInitializers`-compatible logic. Alternatively, validate `stakeToken_` against the factory's allowlist inside `initialize()` itself by accepting the factory address as a parameter and calling `IConfidencePoolFactory(factory_).allowedStakeToken(stakeToken_)`.

### Proof of Concept
```solidity
function test_bypass_factory_allowlist() public {
    // 1. Read the public implementation address
    address impl = factory.poolImplementation();

    // 2. Clone it directly — no factory involvement
    address roguePool = Clones.clone(impl);

    // 3. Initialize with a fee-on-transfer token not in the allowlist
    address[] memory accounts = new address[](1);
    accounts[0] = someInScopeAccount; // valid agreement account
    IConfidencePool(roguePool).initialize(
        validAgreement,          // real, registry-valid agreement
        address(feeOnTransferToken), // NOT in allowedStakeToken
        address(realRegistry),
        attacker,                // attacker is moderator — controls flagOutcome
        block.timestamp + 31 days,
        1e18,
        attacker,                // attacker is recoveryAddress
        attacker,                // attacker is pool owner
        accounts
    );

    // 4. Alice stakes 200 tokens; pool records 198 (1% fee in)
    feeToken.mint(alice, 200e18);
    vm.startPrank(alice);
    feeToken.approve(roguePool, 200e18);
    IConfidencePool(roguePool).stake(200e18);
    vm.stopPrank();
    // eligibleStake[alice] == 198e18

    // 5. Attacker flags SURVIVED (registry is PRODUCTION)
    attackRegistry.setAgreementState(IAttackRegistry.ContractState.PRODUCTION);
    vm.prank(attacker);
    IConfidencePool(roguePool).flagOutcome(PoolStates.Outcome.SURVIVED, false, address(0));

    // 6. Alice claims — pool transfers 198e18 but fee deducts 1%
    //    Alice receives only 196.02e18; 1.98e18 lost to fee
    vm.prank(alice);
    IConfidencePool(roguePool).claimSurvived();
    // alice's actual balance < eligibleStake recorded
}
