### Title
`EcrecoverAuthorizer::setIsAuthorized` Always Reverts for New Users Due to Unpermitted Call into `Midnight::setIsAuthorized` â€” (File: `src/periphery/EcrecoverAuthorizer.sol`)

---

### Summary

`EcrecoverAuthorizer` is designed to enable gasless, signature-based authorization on Midnight. However, it calls `IMidnight(MIDNIGHT).setIsAuthorized(...)` with `onBehalf = authorization.authorizer`, which requires `EcrecoverAuthorizer` to already be authorized by that user on Midnight. Since no new user has pre-authorized `EcrecoverAuthorizer`, every call reverts unconditionally, making the entire gasless authorization mechanism non-functional.

---

### Finding Description

**Root cause â€” the call chain:**

`EcrecoverAuthorizer.setIsAuthorized` verifies the off-chain signature and then calls:

```solidity
IMidnight(MIDNIGHT)
    .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [1](#0-0) 

Inside `Midnight.setIsAuthorized`, the access check is:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [2](#0-1) 

At the point of that call:
- `msg.sender` = `EcrecoverAuthorizer` (the contract)
- `onBehalf` = `authorization.authorizer` (the user)

The check therefore resolves to:

```
authorization.authorizer == EcrecoverAuthorizer   // always false
  OR
isAuthorized[authorization.authorizer][EcrecoverAuthorizer] == true  // false for every new user
```

Neither condition can be true for a user who has not already called `Midnight.setIsAuthorized` directly to grant `EcrecoverAuthorizer` permission. The transaction reverts with `Unauthorized()`.

**The chicken-and-egg problem:**

To use `EcrecoverAuthorizer` (gasless), the user must first authorize it on Midnight. To authorize it on Midnight, the user must call `Midnight.setIsAuthorized` directly â€” which requires gas. This completely defeats the purpose of the gasless authorizer. [3](#0-2) 

---

### Impact Explanation

The `EcrecoverAuthorizer` contract is entirely non-functional for any user who has not already authorized it on Midnight via a direct on-chain call. The gasless authorization feature â€” the sole purpose of this contract â€” is permanently broken for all new users. Any relayer or front-end that submits signed authorizations on behalf of users will always receive a revert. [4](#0-3) 

---

### Likelihood Explanation

This affects **every new user** of `EcrecoverAuthorizer` without exception. There is no code path that allows the call to succeed without prior on-chain authorization. The failure is deterministic and reproducible with any valid signature from a user who has never interacted with Midnight. [5](#0-4) 

---

### Recommendation

`Midnight.setIsAuthorized` should accept calls from a trusted, immutable `EcrecoverAuthorizer` address without requiring prior authorization, analogous to how the fixed Meta protocol moved the call to a context where the caller already held the required permission. Concretely, Midnight could expose a dedicated entry-point:

```solidity
function setIsAuthorizedByTrustedAuthorizer(
    address authorized,
    bool newIsAuthorized,
    address onBehalf,
    /* verified proof / signature already checked by caller */
) external {
    require(msg.sender == ecrecoverAuthorizer, OnlyTrustedAuthorizer());
    isAuthorized[onBehalf][authorized] = newIsAuthorized;
}
```

Alternatively, `EcrecoverAuthorizer` could be redesigned so that it calls `Midnight.setIsAuthorized` with `onBehalf = address(this)` and Midnight's authorization model is adjusted accordingly.

### Proof of Concept

1. Alice (new user, never interacted with Midnight) signs an `Authorization` struct off-chain granting some address permission.
2. A relayer calls `EcrecoverAuthorizer.setIsAuthorized(authorization, signature)`.
3. The signature check at line 31â€“36 passes â€” `signer == authorization.authorizer`.
4. The contract calls `IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, alice)`.
5. Inside Midnight: `msg.sender = EcrecoverAuthorizer`, `onBehalf = alice`.
6. Check: `alice == EcrecoverAuthorizer` â†’ **false**. `isAuthorized[alice][EcrecoverAuthorizer]` â†’ **false** (never set).
7. Transaction reverts: `Unauthorized()`.

Alice cannot use the gasless path without first spending gas to call `Midnight.setIsAuthorized(EcrecoverAuthorizer, true, alice)` directly â€” making `EcrecoverAuthorizer` permanently useless for its intended purpose. [3](#0-2) [4](#0-3)
