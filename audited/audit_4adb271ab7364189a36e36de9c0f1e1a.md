### Title
Co-authorized agent can deauthorize victim's take callback via EcrecoverAuthorizer - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any agent already authorized by the `authorization.authorizer`, not just the authorizer themselves. This allows an attacker who holds `isAuthorized[victim][attacker] == true` to sign and submit an `Authorization` that sets `isAuthorized[victim][callbackAgent] = false`, stripping the callback contract of its ability to call `isAuthorized`-gated functions on behalf of the victim. Any subsequent `take` of victim's callback-based offers will revert inside the callback, permanently DoS-ing those offers until the victim re-authorizes the callback — which the attacker can immediately undo again.

### Finding Description
**Root cause — `EcrecoverAuthorizer.sol` line 34:**

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The second branch permits any agent already authorized by the victim to sign an `Authorization` struct on the victim's behalf, including one that sets `isAuthorized = false` for a third party. There is no restriction preventing a co-authorized agent from deauthorizing other agents.

**Exploit flow:**

1. Victim calls `midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` — standard setup to use the periphery.
2. Victim calls `midnight.setIsAuthorized(callbackAgent, true, victim)` — authorizes the callback contract (e.g. a `BorrowCallback` that calls `supplyCollateral` on victim's behalf inside `onSell`).
3. Victim calls `midnight.setIsAuthorized(attacker, true, victim)` — authorizes the attacker for some other purpose.
4. Attacker constructs `Authorization { authorizer: victim, authorized: callbackAgent, isAuthorized: false, nonce: nonce[victim], deadline: ... }`, signs it with their own key, and calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
5. The check at line 34 passes because `isAuthorized[victim][attacker] == true`.
6. `EcrecoverAuthorizer` calls `midnight.setIsAuthorized(callbackAgent, false, victim)` (line 47); `Midnight.setIsAuthorized` passes its own check because `isAuthorized[victim][ecrecoverAuthorizer] == true` (line 732).
7. `isAuthorized[victim][callbackAgent]` is now `false`.
8. Any taker calling `midnight.take(victimOffer, ...)` reaches the callback invocation (lines 445–452 or 458–473 of `Midnight.sol`). The callback internally calls `midnight.supplyCollateral(market, idx, amount, victim)`, which requires `isAuthorized[victim][callbackAgent]` and reverts with `Unauthorized()`. This revert propagates out of `take`.

**Why existing checks do not stop it:**
- The nonce check (line 26) prevents replay of the same authorization but does not prevent the attacker from issuing a fresh deauthorization each time the victim re-authorizes the callback.
- `Midnight.setIsAuthorized` (line 732) only checks that the caller (`EcrecoverAuthorizer`) is authorized by `onBehalf` (victim), which it is.
- There is no check that the signer is the authorizer themselves when the action is a deauthorization of a third party.

### Impact Explanation
All of victim's active offers that embed `offer.callback = callbackAgent` become permanently untakeable. Any call to `midnight.take` on those offers reverts inside the callback due to `Unauthorized()`. The victim's liquidity is locked in place: offers cannot be filled, and the victim cannot benefit from the callback-based borrow/lend flow. The attacker can repeat the deauthorization immediately after every re-authorization by the victim, making recovery impossible without revoking the attacker's own authorization first.

### Likelihood Explanation
Preconditions are realistic: users of `EcrecoverAuthorizer` must authorize it, and granting authorization to multiple agents (e.g., a keeper and a callback contract) is a normal usage pattern. The attacker needs only one prior authorization from the victim. The attack costs only gas (one `ecrecoverAuthorizer.setIsAuthorized` call per deauthorization). It is fully repeatable as long as the attacker remains authorized.

### Recommendation
Restrict the signer check in `EcrecoverAuthorizer.setIsAuthorized` so that only the `authorization.authorizer` themselves (not a delegated agent) may sign an authorization that modifies another party's authorization status:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegation of this action is intentionally desired, add a separate flag in the `Authorization` struct or a separate entry-point that explicitly limits delegated signers to only granting (not revoking) authorizations, or only authorizing themselves.

### Proof of Concept
```solidity
// Foundry unit test
function testAttackerDeauthorizesCallback() public {
    // Setup: victim authorizes EcrecoverAuthorizer, callbackAgent, and attacker
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    vm.prank(victim);
    midnight.setIsAuthorized(address(callbackAgent), true, victim);
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim posts a sell offer with offer.callback = callbackAgent
    // (callbackAgent.onSell calls midnight.supplyCollateral on behalf of victim)

    // Attacker deauthorizes callbackAgent via EcrecoverAuthorizer
    Authorization memory auth = Authorization({
        authorizer: victim,
        authorized: address(callbackAgent),
        isAuthorized: false,
        nonce: ecrecoverAuthorizer.nonce(victim),
        deadline: block.timestamp + 1 days
    });
    Signature memory sig = signAuthorization(auth, attackerPrivKey);
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);

    // Assert: callbackAgent is now deauthorized
    assertFalse(midnight.isAuthorized(victim, address(callbackAgent)));

    // Assert: take of victim's offer reverts (Unauthorized inside callback)
    vm.expectRevert(IMidnight.Unauthorized.selector);
    vm.prank(taker);
    midnight.take(victimOffer, hex"", units, taker, taker, address(0), hex"");
}
```

Expected assertions:
- `midnight.isAuthorized(victim, callbackAgent) == false` after attacker's call.
- `midnight.take(victimOffer, ...)` reverts (with `Unauthorized()` propagated from inside the callback).
