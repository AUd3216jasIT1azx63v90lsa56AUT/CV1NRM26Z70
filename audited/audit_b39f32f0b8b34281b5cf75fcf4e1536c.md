### Title
`buyWithAssetsTargetAndWithdrawCollateral` unconditionally reverts when any sell offer has `tick = MAX_TICK` and non-zero settlement fee due to `TakeAmountsLib.buyerAssetsToUnits` revert outside try/catch - (`File: src/periphery/MidnightBundles.sol`)

### Summary

`buyWithAssetsTargetAndWithdrawCollateral` calls `TakeAmountsLib.buyerAssetsToUnits` at line 209 of `MidnightBundles.sol` to compute `unitsToTake`, but this call is placed **outside** the `try/catch` block that wraps the actual `take()` call. For any sell offer with `tick = MAX_TICK` (5820) and a non-zero settlement fee, `buyerPrice = tickToPrice(MAX_TICK) + settlementFee = WAD + settlementFee > WAD`, triggering the `require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne())` guard in `TakeAmountsLib.buyerAssetsToUnits` and reverting the entire bundle transaction. The NatSpec on line 175 acknowledges this ("Reverts if TakeAmountsLib or ConsumableUnitsLib reverts"), but this is in direct tension with line 176 ("If taking an offer reverts, the bundler will completely skip this offer"), since `Midnight.take()` itself has no `buyerPrice <= WAD` guard and would not revert for this reason.

### Finding Description

**Exact code path:**

1. Taker calls `buyWithAssetsTargetAndWithdrawCollateral` with a `takes[]` array containing a sell offer (`offer.buy = false`) at `tick = MAX_TICK` (5820). [1](#0-0) 

2. The loop reaches that offer and evaluates the `min(...)` expression. The first argument calls `TakeAmountsLib.buyerAssetsToUnits` at line 209 — **before** the `try` keyword on line 215. [2](#0-1) 

3. Inside `buyerAssetsToUnits`, for a sell offer: `sellerPrice = offerPrice`, `buyerPrice = offerPrice + settlementFee`. The proven invariant `tickToPrice(MAX_TICK) == WAD` (1e18) is confirmed by both the unit test and the Certora spec. [3](#0-2) [4](#0-3) 

4. With `offerPrice = WAD` and any `settlementFee > 0`, `buyerPrice = WAD + settlementFee > WAD`, so `require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne())` reverts. [5](#0-4) 

5. This revert propagates out of the `min(...)` call, past the `try` keyword, and unwinds the entire transaction. The `try/catch` on lines 215–221 never executes. [6](#0-5) 

**Why existing checks do not stop it:**

- `Midnight.take()` itself has **no** `require(buyerPrice <= WAD)` guard. It computes `buyerAssets = units.mulDivUp(buyerPrice, WAD)` and proceeds. The `take()` call would not revert for this reason. [7](#0-6) 

- The settlement fee is non-zero by default for any market with non-zero fee settings. The maximum at TTM=0 is 0.14 bps (1.4e12), and at TTM=360 days is 50 bps (5e15). Any non-zero value suffices. [8](#0-7) 

- `tickToPrice` is proven to return exactly `WAD` at `tick = MAX_TICK` and at most `WAD` for all valid ticks, so the only triggerable tick is `MAX_TICK`. [9](#0-8) 

- The `ConsumableUnitsLib.consumableUnits` call at line 213 does **not** call `buyerAssetsToUnits` for sell offers — it calls `sellerAssetsToUnits`, which has no `PriceGreaterThanOne` guard. The revert originates solely from the direct `TakeAmountsLib.buyerAssetsToUnits` call at line 209. [10](#0-9) 

**Attacker-controlled inputs:** The taker constructs the `takes[]` array. A maker can legitimately post a sell offer at `tick = MAX_TICK` (a valid tick, accessible at any spacing that divides 5820). Any taker who includes such an offer in a bundle — whether knowingly or because it appeared on an order book — triggers the unconditional revert.

### Impact Explanation

The entire `buyWithAssetsTargetAndWithdrawCollateral` transaction reverts. All other valid offers in the `takes[]` array are blocked. The taker's `targetBuyerAssets` transfer is also reverted (since it occurs before the revert point at line 197), so funds are not permanently frozen, but the taker's intended purchase is completely blocked for that bundle invocation. The taker must reconstruct the call without the offending offer.

### Likelihood Explanation

The precondition is: any sell offer with `tick = MAX_TICK` in a market with non-zero settlement fee. `tick = MAX_TICK` is a valid, accessible tick (5820 is divisible by the default spacing). Settlement fees are non-zero by default in any deployed market. The condition is permanent — it does not require the market to be near maturity; it holds at any TTM where `settlementFee > 0`. A maker can deliberately post such an offer to disrupt any taker bundle that includes it, or the condition can arise organically from order-book aggregation.

### Recommendation

Before calling `TakeAmountsLib.buyerAssetsToUnits`, compute `buyerPrice` inline and `continue` (skip the offer) if `buyerPrice > WAD`:

```solidity
uint256 offerPrice = TickLib.tickToPrice(takes[i].offer.tick);
uint256 fee = IMidnight(MIDNIGHT).settlementFee(
    id, UtilsLib.zeroFloorSub(takes[i].offer.market.maturity, block.timestamp)
);
if (offerPrice + fee > WAD) continue;
uint256 unitsToTake = min(
    TakeAmountsLib.buyerAssetsToUnits(...),
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(...)
);
```

Alternatively, modify `TakeAmountsLib.buyerAssetsToUnits` to return `0` instead of reverting when `buyerPrice > WAD`, which would cause `min(...)` to return `0` and the subsequent `take(0, ...)` to be a no-op or be caught by the try/catch.

    function buyerAssetsToUnits(address midnight, bytes32 id, Offer memory offer, uint256 targetBuyerAssets)
        internal
        view
        returns (uint256)
    {
        uint256 offerPrice = TickLib.tickToPrice(offer.tick);
        uint256 settlementFee =
            IMidnight(midnight).settlementFee(id, UtilsLib.zeroFloorSub(offer.market.maturity, block.timestamp));
        // Mirrors Midnight's computation to revert if offerPrice < settlementFee in case of a buy offer.
        uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
        uint256 buyerPrice = sellerPrice + settlementFee;
        require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne());
        return offer.buy ? targetBuyerAssets.mulDivUp(WAD, buyerPrice) : targetBuyerAssets.mulDivDown(WAD, buyerPrice);

    function settlementFee(bytes32 id, uint256 timeToMaturity) public view returns (uint256) {
        MarketState storage _marketState = marketState[id];
        require(_marketState.tickSpacing > 0, MarketNotCreated());

        if (timeToMaturity >= 360 days) return _marketState.settlementFeeCbp6 * CBP;

        // forgefmt: disable-start
        (uint256 start, uint256 end, uint256 feeLower, uint256 feeUpper) =
            timeToMaturity < 1 days   ? (  0 days,   1 days, _marketState.settlementFeeCbp0 * CBP, _marketState.settlementFeeCbp1 * CBP) :
            timeToMaturity < 7 days   ? (  1 days,   7 days, _marketState.settlementFeeCbp1 * CBP, _marketState.settlementFeeCbp2 * CBP) :
            timeToMaturity < 30 days  ? (  7 days,  30 days, _marketState.settlementFeeCbp2 * CBP, _marketState.settlementFeeCbp3 * CBP) :
            timeToMaturity < 90 days  ? ( 30 days,  90 days, _marketState.settlementFeeCbp3 * CBP, _marketState.settlementFeeCbp4 * CBP) :
            timeToMaturity < 180 days ? ( 90 days, 180 days, _marketState.settlementFeeCbp4 * CBP, _marketState.settlementFeeCbp5 * CBP) :
                                        (180 days, 360 days, _marketState.settlementFeeCbp5 * CBP, _marketState.settlementFeeCbp6 * CBP);
        // forgefmt: disable-end

        return (feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start)) / (end - start);
    }
