### Title
Authorized party can revoke EcrecoverAuthorizer's Midnight delegation, permanently blocking signature-based revocation - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
When a victim uses `EcrecoverAuthorizer.setIsAuthorized` to grant authorization to address X in Midnight, X gains the ability to call `Midnight.setIsAuthorized` on behalf of the victim — including revoking `EcrecoverAuthorizer`'s own Midnight authorization. Once revoked, any subsequent call through `EcrecoverAuthorizer` on behalf of the victim will revert with `Unauthorized()`, because `EcrecoverAuthorizer` is no longer in the victim's `isAuthorized` mapping. The victim is then forced to submit a direct on-chain transaction to Midnight to recover control.

### Finding Description

**Code path and root cause:**

`EcrecoverAuthorizer.setIsAuthorized` (line 46–47) calls:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [1](#0-0) 

`Midnight.setIsAuthorized` (line 731–734) checks:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
isAuthorized[onBehalf][authorized] = newIsAuthorized;
``` [2](#0-1) 

For `EcrecoverAuthorizer` to act on behalf of `victim`, `isAuthorized[victim][ecrecoverAuthorizer]` must be `true`. This is the precondition. Once X is authorized in Midnight for victim, X has the same power as victim over the `isAuthorized` mapping — including the ability to set `isAuthorized[victim][ecrecoverAuthorizer] = false`.

**Exploit flow:**

1. **Precondition:** `midnight.isAuthorized(victim, ecrecoverAuthorizer) == true`. Victim has signed `Authorization{authorizer=victim, authorized=X, isAuthorized=true, nonce=0, deadline=T}`.

2. **Attacker (anyone) calls** `ecrecoverAuthorizer.setIsAuthorized(authorization, sig)`:
   - Nonce check passes (nonce=0).
   - Signature check passes (`signer == victim == authorization.authorizer`). [3](#0-2) 
   - Calls `midnight.setIsAuthorized(X, true, victim)` — succeeds because `isAuthorized[victim][ecrecoverAuthorizer] == true`.
   - **Result:** `midnight.isAuthorized(victim, X) == true`.

3. **X calls** `midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim)` directly:
   - Check: `isAuthorized[victim][X] == true` ✓ — passes.
   - **Result:** `midnight.isAuthorized(victim, ecrecoverAuthorizer) == false`.

4. **Victim signs** `Authorization{authorizer=victim, authorized=X, isAuthorized=false, nonce=1}` and anyone submits it to `ecrecoverAuthorizer.setIsAuthorized`:
   - Nonce=1 passes.
   - Signature valid, `signer == victim` passes.
   - Calls `midnight.setIsAuthorized(X, false, victim)` with `msg.sender = ecrecoverAuthorizer`.
   - Check: `victim == ecrecoverAuthorizer`? No. `isAuthorized[victim][ecrecoverAuthorizer]`? **False**.
   - **REVERTS with `Unauthorized()`.**

**Why existing checks fail:** The authorization check in `EcrecoverAuthorizer` (line 33–36) only verifies the signer is the authorizer or is authorized by the authorizer in Midnight. It does not verify that the `authorized` address being granted cannot subsequently revoke `EcrecoverAuthorizer`'s own Midnight delegation. There is no protection in `Midnight.setIsAuthorized` preventing an authorized party from revoking the delegator's authorization of the periphery contract. [4](#0-3) 

### Impact Explanation
Victim cannot use `EcrecoverAuthorizer` to revoke X's Midnight authorization. Every call through `EcrecoverAuthorizer` on behalf of victim will revert with `Unauthorized()` until victim submits a direct on-chain transaction to `midnight.setIsAuthorized` to re-authorize `EcrecoverAuthorizer`. For users who rely on `EcrecoverAuthorizer` as their sole authorization management interface (e.g., systems built around meta-transactions or off-chain signing flows), this constitutes a loss of authorization control over their Midnight account for the duration of the lock.

### Likelihood Explanation
**Preconditions:**
- Victim must have authorized `EcrecoverAuthorizer` in Midnight (required for normal use of the periphery).
- Victim must have signed an authorization for X (the attacker's address) with `isAuthorized=true`.

Both preconditions are the normal intended usage of `EcrecoverAuthorizer`. The attack is executable by X immediately after the signed authorization is submitted. It is repeatable: every time victim re-authorizes `EcrecoverAuthorizer` via a direct on-chain call, X can revoke it again using the same pattern (X is still authorized). The attack requires no special privileges — X is just a signature user.

### Recommendation
`EcrecoverAuthorizer` should not depend on `midnight.isAuthorized(victim, ecrecoverAuthorizer)` being maintained after granting authorization to a third party. Two concrete options:

1. **Restrict what can be authorized via EcrecoverAuthorizer:** Add a check in `EcrecoverAuthorizer.setIsAuthorized` that `authorization.authorized != address(this)`, preventing any party from being granted authorization that could then revoke `EcrecoverAuthorizer`'s own delegation. This is a partial fix only.

2. **Decouple EcrecoverAuthorizer from Midnight's `isAuthorized` mapping:** Have `EcrecoverAuthorizer` maintain its own per-user authorization state (e.g., `mapping(address => bool) public ecrecoverEnabled`), and have Midnight's authorization check also consult `EcrecoverAuthorizer.ecrecoverEnabled[onBehalf]` — or alternatively, have Midnight expose a dedicated signature-based `setIsAuthorized` entry point that accepts an EIP-712 signature directly, removing the need for `EcrecoverAuthorizer` to be in the `isAuthorized` mapping at all.

### Proof of Concept

```solidity
// Foundry unit test
function testEcrecoverAuthorizerRevocationLock() public {
    address victim = makeAddr("victim");
    address X = makeAddr("X");
    uint256 victimKey = /* victim's private key */;

    // Precondition: victim authorizes EcrecoverAuthorizer in Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    assertTrue(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));

    // Victim signs Authorization{authorizer=victim, authorized=X, isAuthorized=true, nonce=0}
    Authorization memory auth = Authorization({
        authorizer: victim, authorized: X, isAuthorized: true,
        nonce: 0, deadline: block.timestamp + 1 days
    });
    Signature memory sig = signAuthorization(victimKey, auth);

    // Anyone submits the signed authorization
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);
    assertTrue(midnight.isAuthorized(victim, X));

    // X revokes EcrecoverAuthorizer's Midnight delegation for victim
    vm.prank(X);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim);
    assertFalse(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));

    // Victim signs revocation of X
    Authorization memory revoke = Authorization({
        authorizer: victim, authorized: X, isAuthorized: false,
        nonce: 1, deadline: block.timestamp + 1 days
    });
    Signature memory revokeSig = signAuthorization(victimKey, revoke);

    // EcrecoverAuthorizer cannot revoke X on behalf of victim — reverts
    vm.expectRevert(IMidnight.Unauthorized.selector);
    ecrecoverAuthorizer.setIsAuthorized(revoke, revokeSig);

    // Assert: X is still authorized, victim is locked out of EcrecoverAuthorizer
    assertTrue(midnight.isAuthorized(victim, X));
    assertFalse(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));
}
```

**Expected assertions:** The `vm.expectRevert(IMidnight.Unauthorized.selector)` passes, confirming victim cannot use `EcrecoverAuthorizer` to revoke X. `midnight.isAuthorized(victim, X)` remains `true`.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );
