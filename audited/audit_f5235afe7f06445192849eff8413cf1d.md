Audit Report

## Title
Authorized-Signer Front-Run Advances Victim's Nonce and Invalidates Pending Signed Authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts any address that `authorization.authorizer` has previously authorized in Midnight as a valid signer for *any* `Authorization` struct at the current nonce — including one with attacker-chosen `authorized` and `isAuthorized` values. An authorized attacker can front-run the victim's pending transaction, consume nonce N with a different struct, and permanently invalidate the victim's signed authorization. The attacker can repeat this indefinitely and can also set arbitrary `isAuthorized` entries on behalf of the victim.

## Finding Description
**Root cause:**

The nonce in `EcrecoverAuthorizer` is keyed only on `authorization.authorizer` and increments on any valid call from any authorized signer. There is no binding between the signer's identity and the `authorized` field, and no constraint that a delegated signer may only submit a struct the authorizer themselves pre-approved.

```solidity
// src/periphery/EcrecoverAuthorizer.sol line 26
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// lines 33-36
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The `Authorization` struct (`src/periphery/interfaces/IEcrecoverAuthorizer.sol`) contains five independently attacker-controllable fields: `authorizer`, `authorized`, `isAuthorized`, `nonce`, `deadline`. A delegated signer can freely choose any values for `authorized` and `isAuthorized` — there is no check that the struct content was pre-approved by the authorizer.

**Exploit flow:**

1. **Precondition:** `IMidnight.isAuthorized(victim, attacker) == true` (victim previously authorized attacker as an operator/relayer — a routine, expected state).
2. Victim signs `Authorization{authorizer=victim, authorized=X, isAuthorized=true, nonce=N}` and broadcasts.
3. Attacker observes victim's tx in mempool; crafts `Authorization{authorizer=victim, authorized=attacker, isAuthorized=true, nonce=N}` (same nonce, different `authorized`).
4. Attacker signs this struct with their own private key → `ecrecover(digest, ...) == attacker`.
5. Attacker submits with higher gas priority.
6. Contract execution:
   - Line 25: deadline check passes (attacker sets any valid future deadline).
   - Line 26: `N == nonce[victim]` → passes; nonce increments to N+1.
   - Line 31: `signer = attacker`.
   - Lines 33–36: `IMidnight.isAuthorized(victim, attacker) == true` → passes.
   - Line 47: `IMidnight.setIsAuthorized(attacker, true, victim)` executes — attacker self-authorizes or sets arbitrary entries.
7. Victim's tx arrives: `N != N+1` → reverts with `InvalidNonce()`.

**Why existing checks fail:**

- The nonce check prevents replay of the *same* signed message but does not prevent a *different* message at the same nonce from a delegated signer.
- The `isAuthorized` delegation path was designed to allow authorized agents to act on behalf of the authorizer, but it does not scope what struct content the agent may sign — the agent can sign any `authorized`/`isAuthorized` combination.
- There is no binding between the signer's identity and the `authorized` field in the struct.

## Impact Explanation
The victim's pending signed `Authorization` is permanently invalidated (DoS of delegation). The victim must sign a new message at nonce N+1, which the attacker can front-run again indefinitely. Additionally, the attacker can set arbitrary `isAuthorized` entries on behalf of the victim — authorizing malicious addresses or revoking existing authorizations — which constitutes unauthorized state mutation of the victim's authorization state in Midnight. This maps to: unauthorized state changes, privilege escalation from authorized-agent to full authorization-manager, and permanent service degradation for the victim's signature-gated delegation path.

## Likelihood Explanation
The precondition (`isAuthorized[victim][attacker] == true`) is a common, expected state — users routinely authorize operators, bots, and relayers. Once met, the attack requires only mempool monitoring and a higher-gas competing transaction, which is standard MEV infrastructure. The attack is trivially repeatable on every future signed authorization the victim broadcasts, making it a persistent griefing vector with no natural expiry.

## Recommendation
Bind the signer's identity to the struct content. The simplest fix is to require that the `authorized` field in the struct equals the signer when the signer is not the authorizer themselves:

```solidity
require(
    signer == authorization.authorizer
        || (
            IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)
            && authorization.authorized == signer  // signer can only act for themselves
        ),
    Unauthorized()
);
```

Alternatively, include the signer's address in the signed struct (add a `signer` field to `Authorization`) and verify it matches `ecrecover` output, so the authorizer explicitly pre-approves which agent may submit the struct. A third option is to make the nonce per `(authorizer, signer)` pair rather than per `authorizer` alone, so a delegated signer cannot consume the authorizer's nonce for other signers.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
