### Title
Front-Running `consumableUnits` Snapshot Enables Griefing DoS on `buyWithUnitsTargetAndWithdrawCollateral` - (File: src/periphery/MidnightBundles.sol)

### Summary
`buyWithUnitsTargetAndWithdrawCollateral` computes each offer's fillable amount via a live `consumableUnits` read, then enforces an exact equality `require(filledUnits == targetUnits, OutOfOffers())`. Because any unprivileged taker can call `Midnight.take` directly on the same offer using the maker's signature extracted from the victim's pending mempool transaction, an attacker can reduce `consumed[maker][group]` by as little as 1 unit before the bundle executes, causing `filledUnits < targetUnits` and a guaranteed `OutOfOffers()` revert at negligible cost.

### Finding Description

**Code path:**

`buyWithUnitsTargetAndWithdrawCollateral` (MidnightBundles.sol lines 69–88):

```solidity
for (uint256 i; i < takes.length && filledUnits < targetUnits; i++) {
    uint256 unitsToTake = min(
        targetUnits - filledUnits,
        takes[i].units,
        ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)  // live read
    );
    try IMidnight(MIDNIGHT).take(..., unitsToTake, ...) returns (...) {
        filledUnits += unitsToTake;
        ...
    } catch {}
}
require(filledUnits == targetUnits, OutOfOffers());  // exact equality
```

`ConsumableUnitsLib.consumableUnits` (ConsumableUnitsLib.sol lines 14–17):

```solidity
uint256 consumed = IMidnight(midnight).consumed(offer.maker, offer.group);
if (offer.maxUnits > 0) {
    return offer.maxUnits.zeroFloorSub(consumed);
```

`Midnight.take` (Midnight.sol lines 346, 355–356, 371–372):

```solidity
require(taker == msg.sender || isAuthorized[taker][msg.sender], TakerUnauthorized());
require(isAuthorized[offer.maker][offer.ratifier], RatifierUnauthorized());
require(IRatifier(offer.ratifier).isRatified(offer, ratifierData) == CALLBACK_SUCCESS, RatifierFail());
...
newConsumed = consumed[offer.maker][offer.group] += units;
require(newConsumed <= offer.maxUnits, ConsumedUnits());
```

**Root cause:** The `consumableUnits` snapshot is taken inside the bundle's execution loop, but `consumed[maker][group]` is a shared, globally writable counter. Any address can call `Midnight.take` directly with `taker = msg.sender` (no maker permission needed for the taker side). The `EcrecoverRatifier` signature in `ratifierData` is over the offer struct only — it does not bind the taker address or the units amount — so the same signature extracted from the victim's pending mempool transaction is reusable by the attacker verbatim.

**Exploit flow:**

1. Maker (borrower) creates a sell offer: `offer.maxUnits = N`, `offer.ratifier = EcrecoverRatifier`, signs it.
2. Victim (lender) constructs bundle: `buyWithUnitsTargetAndWithdrawCollateral(targetUnits=N, ..., takes=[{offer, ratifierData=makerSig, units=N}])` and broadcasts to mempool.
3. Attacker observes the pending transaction, extracts `offer` and `ratifierData` (the maker's reusable signature).
4. Attacker calls `Midnight.take(offer, ratifierData, 1, attacker, address(0), address(0), "")` with higher gas, paying 1 unit of loan tokens.
5. `consumed[maker][group]` becomes 1.
6. Victim's bundle executes: `consumableUnits(offer)` = N − 1; `unitsToTake` = N − 1; `take` succeeds; `filledUnits` = N − 1 < N = `targetUnits`.
7. `require(filledUnits == targetUnits, OutOfOffers())` reverts.

**Why existing checks fail:**

- The `try/catch` at line 85 only catches reverts from `take` itself; it does not compensate for the shortfall in `filledUnits`.
- There is no minimum-fill tolerance; the check at line 88 is strict equality.
- The `TakerUnauthorized` check only requires `taker == msg.sender`, which the attacker satisfies trivially.
- The `EcrecoverRatifier` signature does not bind the taker, so the attacker reuses the victim's extracted `ratifierData` without modification.
- The shared-group mechanism (Midnight.sol lines 84–87) amplifies the attack: a single 1-unit take against any offer in the group reduces `consumableUnits` for every other offer in the same group simultaneously.

### Impact Explanation
Any victim calling `buyWithUnitsTargetAndWithdrawCollateral` with a tight bundle (total offer capacity = `targetUnits`) can be permanently griefed. Every attempt reverts with `OutOfOffers()`. The victim's `maxBuyerAssets` are pulled and then returned (no token loss), but the intended market action — acquiring exactly `targetUnits` of credit — is blocked indefinitely at attacker cost of 1 unit of loan tokens per block (which the attacker recovers as credit).

### Likelihood Explanation
**Preconditions:**
- The offer uses a ratifier whose `isRatified` check the attacker can satisfy (e.g., `EcrecoverRatifier` with a signature visible in the mempool, or any permissive ratifier).
- The victim's bundle has no slack (total offer capacity ≤ `targetUnits`), which is the normal usage pattern.
- The attacker holds enough loan tokens to take ≥ 1 unit (recoverable as credit).

**Feasibility:** High. The victim's `ratifierData` (maker signature) is fully visible in the mempool. The attack requires a single direct `Midnight.take` call with 1 unit. It is repeatable every block at near-zero net cost. The shared-group scenario makes it even cheaper: one 1-unit take can drain `consumableUnits` across all grouped offers simultaneously.

### Recommendation
Replace the strict equality check with a minimum-fill parameter, mirroring the pattern already used in `buyWithAssetsTargetAndWithdrawCollateral` (line 225):

```solidity
// Replace:
require(filledUnits == targetUnits, OutOfOffers());

// With:
require(filledUnits >= minUnits, OutOfOffers());
```

Add `uint256 minUnits` as a parameter to `buyWithUnitsTargetAndWithdrawCollateral` (and symmetrically to `supplyCollateralAndSellWithUnitsTarget`). This allows callers to express a tolerance for partial front-running while still enforcing a meaningful lower bound, eliminating the exact-equality DoS vector without changing the happy-path behavior when `minUnits == targetUnits` is chosen by the caller.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {MidnightBundlesTest} from "test/MidnightBundlesTest.sol";
import {IMidnightBundles, Take, TokenPermit, CollateralWithdrawal} from "src/periphery/interfaces/IMidnightBundles.sol";
import {TickLib, MAX_TICK} from "src/libraries/TickLib.sol";
import {WAD} from "src/libraries/ConstantsLib.sol";
import {UtilsLib} from "src/libraries/UtilsLib.sol";

contract FrontRunDoSTest is MidnightBundlesTest {
    using UtilsLib for uint256;

    function testFrontRunDoS(uint256 partialUnits) public {
        uint256 targetUnits = 100e18;
        partialUnits = bound(partialUnits, 1, targetUnits - 1);

        // Setup: borrower creates sell offer with maxUnits = targetUnits
        offers[0].buy = false;
        offers[0].maker = borrower;
        offers[0].receiverIfMakerIsSeller = borrower;
        offers[0].maxUnits = targetUnits;
        for (uint256 i; i <= 6; i++) midnight.setMarketSettlementFee(id, i, 0);
        collateralize(market, borrower, targetUnits);

        Take[] memory takes = new Take[](1);
        takes[0] = Take({offer: offers[0], units: targetUnits, ratifierData: hex""});

        uint256 price = TickLib.tickToPrice(MAX_TICK);
        uint256 maxBuyerAssets = targetUnits.mulDivUp(price, WAD);

        // Attacker front-runs: takes partialUnits directly from the same offer
        address attacker = makeAddr("attacker");
        deal(address(loanToken), attacker, maxBuyerAssets);
        vm.startPrank(attacker);
        loanToken.approve(address(midnight), maxBuyerAssets);
        midnight.take(offers[0], hex"", partialUnits, attacker, address(0), address(0), "");
        vm.stopPrank();

        // Assert: consumed[maker][group] is now partialUnits
        assertEq(midnight.consumed(borrower, offers[0].group), partialUnits);

        // Victim's bundle now reverts with OutOfOffers
        vm.prank(lender);
        vm.expectRevert(IMidnightBundles.OutOfOffers.selector);
        midnightBundles.buyWithUnitsTargetAndWithdrawCollateral(
            targetUnits, maxBuyerAssets, lender, _noPermit(), takes,
            new CollateralWithdrawal[](0), address(0), 0, address(0)
        );
    }
}
```

**Expected assertions:**
- `midnight.consumed(borrower, offers[0].group) == partialUnits` after attacker's take.
- `buyWithUnitsTargetAndWithdrawCollateral` reverts with `OutOfOffers()` for all `partialUnits ∈ [1, targetUnits − 1]`.
- Fuzz over `partialUnits` confirms the revert is universal across the entire range.
