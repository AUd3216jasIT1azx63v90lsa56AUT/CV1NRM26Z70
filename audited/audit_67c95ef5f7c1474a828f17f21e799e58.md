### Title
Staker principal permanently locked if staker is a smart contract unable to invoke claim functions - (File: src/ConfidencePool.sol)

### Summary
`claimSurvived()` and `claimExpired()` transfer tokens exclusively to `msg.sender` with no permissionless "claim on behalf of" path. If a staker is a smart contract that can call `stake()` but lacks the ability to call the claim functions (e.g., an immutable contract with no outbound call to `claimSurvived`/`claimExpired`, or a proxy that is later bricked), their principal is permanently locked. `sweepUnclaimedBonus()` explicitly reserves `totalEligibleStake` and cannot recover it, leaving the funds with no exit path.

### Finding Description
Both resolution claim paths transfer only to `msg.sender`:

`claimSurvived()` at line 403:
```solidity
stakeToken.safeTransfer(msg.sender, payout);
```
`claimExpired()` at line 601:
```solidity
stakeToken.safeTransfer(msg.sender, payout);
```

There is no "claim for `account`" variant. A smart contract staker that can call `stake()` (e.g., a vault, a DAO treasury, or any contract with a deposit helper but no claim helper) but cannot call `claimSurvived()` or `claimExpired()` will have its `eligibleStake` balance permanently stranded.

`sweepUnclaimedBonus()` cannot recover the principal either. Its reserve calculation at lines 483–488 always holds back `totalEligibleStake` (the full principal of all non-claiming stakers):

```solidity
if (totalEligibleStake != 0) {
    reserved = totalEligibleStake;
    if (riskWindowStart != 0) {
        reserved += snapshotTotalBonus - claimedBonus;
    }
}
```

Because `totalEligibleStake` is decremented only inside `claimSurvived()` and `claimExpired()` (lines 395 and 593), a staker who never claims keeps their share permanently reserved and permanently unswept. No owner, moderator, or admin function exists to force-transfer or recover stranded principal.

### Impact Explanation
A staker whose address is a smart contract incapable of invoking the claim functions loses their entire principal permanently. The funds sit in the pool indefinitely: they cannot be claimed (no `msg.sender` match), cannot be swept (reserved by `sweepUnclaimedBonus`), and cannot be recovered by any privileged actor. This is a complete, irrecoverable loss of principal for the affected staker.

### Likelihood Explanation
Any smart contract that calls `stake()` without a corresponding outbound call to `claimSurvived()`/`claimExpired()` triggers this. Realistic examples: a DAO treasury contract that deposits via governance but whose governance module is later deprecated; a vault or aggregator that wraps `stake()` but has no claim integration; a multisig whose signers become permanently unavailable. The staker chose their own address, but the protocol imposes no requirement that the address be capable of calling claim functions, and provides no fallback if it is not.

### Recommendation
Add a permissionless `claimFor(address staker)` variant that computes the payout for `staker` and transfers to `staker` (not `msg.sender`). This mirrors the TimeLock report's recommendation: allow any caller to trigger the claim on behalf of the beneficiary while still routing funds to the beneficiary. Alternatively, add a time-gated sweep that moves unclaimed principal to `recoveryAddress` after a sufficiently long post-resolution window, analogous to `sweepUnclaimedCorrupted`.

### Proof of Concept
1. Deploy `StuckStaker`, an immutable contract with a `doStake(address pool, uint256 amount)` function that calls `ConfidencePool.stake()` but no function that calls `claimSurvived()` or `claimExpired()`.
2. `StuckStaker` approves the pool and calls `stake(amount)`. `eligibleStake[address(StuckStaker)] = amount`.
3. The pool resolves to `SURVIVED` (moderator calls `flagOutcome(SURVIVED, ...)`).
4. No EOA or contract can call `claimSurvived()` on behalf of `StuckStaker` — `msg.sender` must equal the staker.
5. Call `sweepUnclaimedBonus()`: `reserved = totalEligibleStake = amount`, `freeBalance - reserved = 0` → reverts `NothingToSweep`.
6. `amount` tokens remain in the pool forever with no callable exit path. [1](#0-0) [2](#0-1) [3](#0-2)

    function claimSurvived() external nonReentrant {
        if (outcome != PoolStates.Outcome.SURVIVED) revert OutcomeNotSet();
        if (hasClaimed[msg.sender]) revert InvalidAmount();

        uint256 userEligible = eligibleStake[msg.sender];
        if (userEligible == 0) revert InvalidAmount();

        _clampUserSums(msg.sender);

        hasClaimed[msg.sender] = true;

        uint256 bonusShare = _bonusShare(msg.sender, userEligible);
        uint256 payout = userEligible + bonusShare;
        totalEligibleStake -= userEligible;
        claimedBonus += bonusShare;

        delete eligibleStake[msg.sender];
        delete userSumStakeTime[msg.sender];
        delete userSumStakeTimeSq[msg.sender];

        if (!claimsStarted) claimsStarted = true;
        stakeToken.safeTransfer(msg.sender, payout);
        emit ClaimSurvived(msg.sender, userEligible, bonusShare);
    }
