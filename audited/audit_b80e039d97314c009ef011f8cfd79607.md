### Title
Zero oracle price in `seizedAssets` path allows free collateral seizure with zero debt repaid - (`src/Midnight.sol`)

### Summary
When `liquidate` is called with `seizedAssets > 0` and the oracle for `collateralIndex` returns `price = 0`, the computation `repaidUnits = seizedAssets.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif)` evaluates to exactly `0` due to integer arithmetic. The liquidator receives real collateral tokens while repaying zero debt, violating the core invariant that collateral cannot be seized outside valid liquidation economics.

**Attacker inputs**: `seizedAssets = N > 0`, `repaidUnits = 0`, `collateralIndex` pointing to the zero-price oracle, `postMaturityMode = false` (or `true`).

**Why existing checks fail**: The `InconsistentInput` check (line 595) only requires at most one of `seizedAssets`/`repaidUnits` to be nonzero — passing `seizedAssets > 0, repaidUnits = 0` is explicitly valid. The RCF check passes trivially. No check validates `liquidatedCollatPrice > 0` before the `seizedAssets` branch. The `repaid input` path naturally reverts (division by zero in `mulDivDown(ORACLE_PRICE_SCALE, 0)`), but the `seizedAssets` path does not.

The protocol's own LIVENESS comment at line 146 explicitly documents: *"If the liquidated collateral oracle returns 0 on price, liquidate with repaid input reverts"* — acknowledging 0 prices are possible, but only protecting one path.

### Impact Explanation
An unprivileged liquidator can seize any amount of a borrower's collateral (up to the full balance at `collateralIndex`) while repaying exactly 0 debt units. The borrower loses collateral with no corresponding debt reduction. The lender pool loses collateral backing without any loan repayment, directly violating the invariant that every collateral seizure must correspond to proportional debt repayment.

### Likelihood Explanation
**Preconditions**: (1) borrower has ≥2 active collaterals and is unhealthy (or market is post-maturity); (2) the oracle for the targeted `collateralIndex` returns `0`. The protocol explicitly acknowledges condition (2) as a reachable scenario in its LIVENESS section and in `Reverts.spec` (dedicated ghost axioms for zero-oracle modeling). A zero oracle price can occur via a circuit-breaker oracle, a stale Chainlink feed returning 0, or a custom oracle. Once the oracle returns 0, any liquidator can exploit this repeatedly until the collateral is drained.

### Recommendation
Add an explicit guard in the `seizedAssets > 0` branch:

```solidity
if (seizedAssets > 0) {
    require(liquidatedCollatPrice > 0, ZeroOraclePrice());
    repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
}
```

Alternatively, add a symmetric liveness note and revert path analogous to the `repaid input` path. The LIVENESS section should also be updated to document the `seizedAssets` path behavior.
