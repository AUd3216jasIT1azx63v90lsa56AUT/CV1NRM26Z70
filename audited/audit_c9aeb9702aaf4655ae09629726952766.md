### Title
Zero `sellerAssets` Rounding in `take()` Allows Taker to Incur Debt Without Receiving Loan Tokens - (File: src/Midnight.sol)

---

### Summary

In `Midnight.sol`'s `take()` function, when a taker executes against a buy offer (`offer.buy == true`), the `sellerAssets` amount is computed using `mulDivDown`. Because all tick prices are strictly less than 1 WAD (1e18), taking with `units = 1` always yields `sellerAssets = 0`. The taker's debt is increased by the full `units` amount while they receive zero loan tokens in return — a direct, unguarded loss of value.

---

### Finding Description

In `take()`, the seller's payout is computed at line 364:

```solidity
uint256 sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : units.mulDivUp(sellerPrice, WAD);
``` [1](#0-0) 

When `offer.buy == true` (maker is lender/buyer, taker is borrower/seller), the formula is `units * sellerPrice / WAD` rounded **down**. Since `tickToPrice()` produces values strictly below `1e18` for all valid ticks (0–5820), and `PRICE_ROUNDING_STEP = 1e12` is the minimum representable price: [2](#0-1) 

For `units = 1`: `1 * sellerPrice / 1e18 = 0` for **any** `sellerPrice < 1e18` — which is always true. For the minimum price (`~1e12`), `sellerAssets = 0` for any `units < 1e6`.

There is no guard checking `sellerAssets > 0` before the state mutation and transfer path executes:

```solidity
sellerPos.debt += UtilsLib.toUint128(sellerDebtIncrease);   // taker's debt increases
...
SafeTransferLib.safeTransferFrom(..., receiver, sellerAssets); // transfers 0
``` [3](#0-2) 

Simultaneously, `buyerAssets = units.mulDivDown(buyerPrice, WAD)` also rounds to 0, so the maker (buyer) gains `buyerCreditIncrease` units of credit while paying nothing: [4](#0-3) [5](#0-4) 

---

### Impact Explanation

The taker (seller) incurs real on-chain debt (`position.debt += units`) but receives zero loan tokens. At maturity they must repay this debt or face liquidation — a direct, unrecoverable asset loss. Simultaneously, the maker (buyer) accumulates credit units without paying anything; once `withdrawable` is non-zero (e.g., after any repayment in the market), the maker can withdraw loan tokens at the expense of legitimate lenders.

---

### Likelihood Explanation

The condition is trivially reachable by any unprivileged caller: `units = 1` is a valid input (no minimum-units guard exists), and the rounding-to-zero condition holds for **every** valid tick price. The `supplyCollateralAndSellWithUnitsTarget` bundle in `MidnightBundles.sol` can also route a taker into this path automatically when `targetUnits` is small. [6](#0-5) 

---

### Recommendation

Add a zero-assets guard in `take()` analogous to the fix recommended in the reference report:

```solidity
require(sellerAssets > 0, "zero sellerAssets");
```

Alternatively, enforce a protocol-level minimum `units` per take (e.g., `require(units >= WAD / sellerPrice + 1)`), ensuring `sellerAssets >= 1` always.

---

### Proof of Concept

**Preconditions:** Any market exists. Maker has authorized a ratifier.

1. Maker signs and publishes a buy offer at `tick = 2910` (mid-range price ≈ 0.5 WAD).
2. Taker calls `take(offer, ratifierData, 1, taker, receiver, address(0), "")`.
3. Inside `take()`:
   - `sellerPrice = tickToPrice(2910) - settlementFee ≈ 5e17`
   - `sellerAssets = 1 * 5e17 / 1e18 = 0` (integer division)
   - `sellerDebtIncrease = 1` (taker has no prior credit)
   - `position[id][taker].debt += 1` — taker now owes 1 unit
   - `safeTransferFrom(..., receiver, 0)` — taker receives nothing
   - `buyerAssets = 1 * (5e17 + fee) / 1e18 = 0`
   - `position[id][maker].credit += 1` — maker gains 1 unit free
4. Taker has incurred debt with zero compensation. Maker has gained credit at zero cost.

Repeating with `units = 999_999` at tick 0 (price ≈ 1e12) scales the debt loss to ~1e6 units per call, all with `sellerAssets = 0`.
