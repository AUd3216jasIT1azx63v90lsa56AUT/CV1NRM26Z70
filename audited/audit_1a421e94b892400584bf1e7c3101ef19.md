### Title
Fee-on-Transfer Loan Token Inflates `withdrawable` Beyond Actual Balance in `repay()` - (File: src/Midnight.sol)

### Summary
In `repay()`, `marketState[id].withdrawable` is incremented by the full `units` parameter before the token transfer occurs, and `SafeTransferLib.safeTransferFrom` only validates the boolean return value of `transferFrom` — it does not measure the actual tokens received. When the loan token charges a transfer fee, the contract receives fewer tokens than `units`, but `withdrawable` is inflated by the full amount, permanently breaking the invariant that `withdrawable ≤ actual token balance`.

### Finding Description

The exact code path in `src/Midnight.sol`:

```
repay(market, units=1000, onBehalf, callback=0, data)
  → position[id][onBehalf].debt  -= 1000          // line 508
  → marketState[id].withdrawable += 1000          // line 509  ← inflated by full units
  → SafeTransferLib.safeTransferFrom(
        market.loanToken, payer, address(this), 1000  // line 520
    )
    // fee-on-transfer token: contract receives only 990
    // safeTransferFrom checks returndata bool only — no balance delta check
``` [1](#0-0) 

`SafeTransferLib.safeTransferFrom` performs no before/after balance measurement:

```solidity
(bool success, bytes memory returndata) = token.call(
    abi.encodeCall(IERC20.transferFrom, (from, to, value))
);
// only checks success and returndata bool — amount received is never verified
``` [2](#0-1) 

Market creation via `touchMarket` imposes no restriction on the loan token type — the protocol explicitly supports "arbitrary loan token": [3](#0-2) 

The `withdraw()` function then sends `units` tokens out, trusting `withdrawable` as the accounting source of truth: [4](#0-3) 

**Exploit flow:**
1. Market creator deploys a market with a fee-on-transfer ERC20 (e.g., 1% fee) as `loanToken`.
2. Lender lends 1000 units; borrower takes 1000 units of debt.
3. Borrower calls `repay(market, 1000, borrower, address(0), "")`.
4. `withdrawable` becomes 1000; contract receives only 990 tokens.
5. Lender calls `withdraw(market, 1000, lender, lender)`.
6. `withdraw` attempts `safeTransfer(loanToken, lender, 1000)` — contract only holds 990, so the transfer reverts (or drains 10 tokens from unrelated balances held by the contract for other markets).

### Impact Explanation

The core invariant `contract token balance ≥ withdrawable` is violated. Lenders cannot withdraw the full amount they are owed; the last lender(s) to call `withdraw` will either receive a revert or drain tokens belonging to other markets sharing the same loan token. This constitutes direct, concrete loss of lender funds and protocol insolvency for the affected market.

### Likelihood Explanation

Preconditions: (1) a market is created with a fee-on-transfer loan token — permissionless, no admin required; (2) a borrower calls `repay` — normal user action. Both are fully attacker-controlled without any privileged access. The discrepancy accumulates with every `repay` call, making it repeatable and compounding.

### Recommendation

Measure the actual tokens received by comparing the contract's balance before and after the transfer, and use the delta to update `withdrawable`:

```solidity
uint256 balanceBefore = IERC20(market.loanToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);
uint256 received = IERC20(market.loanToken).balanceOf(address(this)) - balanceBefore;
// use `received` instead of `units` for the withdrawable increment
```

Apply the same fix to `liquidate()` at line 717, which has the identical pattern.

    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;
    function mint(address to, uint256 amt) external { balanceOf[to] += amt; }
    function approve(address sp, uint256 amt) external { allowance[msg.sender][sp] = amt; }
    function transferFrom(address from, address to, uint256 amt) external returns (bool) {
        allowance[from][msg.sender] -= amt;
        balanceOf[from] -= amt;
        uint256 fee = amt / 100;
        balanceOf[to] += amt - fee; // 1% burned/taken
        return true;
    }
    function transfer(address to, uint256 amt) external returns (bool) {
        balanceOf[msg.sender] -= amt;
        balanceOf[to] += amt;
        return true;
    }

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
