Audit Report

## Title
`buyWithAssetsTargetAndWithdrawCollateral` permanently reverts for sell offers near `MAX_TICK` when settlement fee is non-zero - (`File: src/periphery/MidnightBundles.sol`)

## Summary

`TakeAmountsLib.buyerAssetsToUnits` enforces `require(buyerPrice <= WAD)` and reverts with `PriceGreaterThanOne` when `offerPrice + settlementFee > WAD`. This call sits **outside** the try/catch block in `buyWithAssetsTargetAndWithdrawCollateral`, so the entire bundle transaction reverts rather than skipping the offer. The core `Midnight.take()` performs the identical computation with no such guard and would succeed. Any sell offer at a tick near `MAX_TICK` with a non-zero settlement fee triggers this unconditional DoS of the periphery take path.

## Finding Description

**Root cause — `TakeAmountsLib.buyerAssetsToUnits`:**

For a sell offer (`offer.buy = false`):
- `sellerPrice = offerPrice` (line 26)
- `buyerPrice = sellerPrice + settlementFee` (line 27)
- `require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne())` (line 28) — reverts when `offerPrice + settlementFee > WAD` [1](#0-0) 

**Asymmetry with `Midnight.take()`:**

The core performs the identical computation at lines 361–363 but has **no** `buyerPrice <= WAD` guard — it proceeds directly to `units.mulDivUp(buyerPrice, WAD)`. [2](#0-1) 

**Call placement in `buyWithAssetsTargetAndWithdrawCollateral`:**

The `buyerAssetsToUnits` call at lines 209–211 is **outside** the try/catch. If it reverts, the entire transaction reverts. The `take()` call at lines 215–221 is inside try/catch and is never reached. [3](#0-2) 

The natspec at line 175 explicitly documents this: *"Reverts if TakeAmountsLib or ConsumableUnitsLib reverts."* [4](#0-3) 

`ConsumableUnitsLib.consumableUnits` also calls `buyerAssetsToUnits` (line 213), so both calls at lines 209–213 are outside the try/catch and both propagate the same revert. [5](#0-4) 

**Why existing guards are insufficient:**

The only protection is the try/catch around `take()` itself, which does not cover the pre-computation calls. The `TakeAmountsLib` comment at line 14 acknowledges the revert is intentional for the periphery ("not all buyerAssets are reachable then"), but this design choice creates an irrecoverable DoS for a valid subset of core-accepted offers. [6](#0-5) 

## Impact Explanation

Any taker calling `buyWithAssetsTargetAndWithdrawCollateral` with a sell offer at tick T where `tickToPrice(T) + settlementFee > WAD` will have the entire bundle transaction revert. The taker cannot use this periphery function for those offers at all, even though `Midnight.take()` would accept them. This is a permanent, unconditional DoS of the periphery buy-with-assets-target path for all sell offers near `MAX_TICK` whenever the market settlement fee is non-zero.

## Likelihood Explanation

All three preconditions are routine:
1. A market with a non-zero settlement fee at the current TTM breakpoint — standard governance configuration.
2. A maker places a sell offer at `MAX_TICK` (or any tick T where `tickToPrice(T) + settlementFee > WAD`). With any non-zero fee, `MAX_TICK` satisfies this.
3. A taker calls `buyWithAssetsTargetAndWithdrawCollateral` with that offer in `takes[]`.

No privileged action is required. The DoS is deterministic and repeatable: every call with such an offer reverts.

## Recommendation

Wrap the `buyerAssetsToUnits` (and `consumableUnits`) calls in a try/catch so that offers where `buyerPrice > WAD` are skipped rather than causing the entire bundle to revert — consistent with the existing try/catch around `take()`. Alternatively, add a pre-check before calling `buyerAssetsToUnits` that computes `buyerPrice` and skips the offer if `buyerPrice > WAD`, mirroring the skip-on-failure semantics already present for `take()`.

## Proof of Concept

1. Deploy a market with a non-zero settlement fee (e.g., `settlementFee = 1`).
2. Have a maker place a sell offer at `MAX_TICK` (tick = 5820).
3. Verify `tickToPrice(MAX_TICK) + settlementFee > WAD`.
4. Have a taker call `buyWithAssetsTargetAndWithdrawCollateral` with that offer.
5. Observe the transaction reverts with `PriceGreaterThanOne`.
6. Call `Midnight.take()` directly with the same offer — observe it succeeds.

The existing test `testBuyBuyerAssetsTarget` in `test/MidnightBundlesTest.sol` already confirms this implicitly: it resets all settlement fees to 0 before calling `buyWithAssetsTargetAndWithdrawCollateral` with a comment confirming that non-zero fees at high ticks would cause `buyerAssetsToUnits` to revert.
