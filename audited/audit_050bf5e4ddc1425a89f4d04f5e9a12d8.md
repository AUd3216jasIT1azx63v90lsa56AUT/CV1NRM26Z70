### Title
Fee-on-Transfer Token Inflates `claimableSettlementFee` Beyond Actual Balance - (`src/Midnight.sol`)

### Summary

In `take()`, `claimableSettlementFee[offer.market.loanToken]` is incremented by the full nominal spread (`buyerAssets - sellerAssets`) at line 418 before the actual token transfer occurs at line 455. Because `SafeTransferLib.safeTransferFrom` only checks call success and return value — not the amount actually received — a token that deducts a fee when the recipient is a contract causes Midnight to receive less than the credited amount, permanently inflating `claimableSettlementFee` beyond the contract's real balance.

### Finding Description

**Exact code path:**

`src/Midnight.sol` line 418 — state update before transfer: [1](#0-0) 

`src/Midnight.sol` lines 455–456 — the two transfers that follow: [2](#0-1) 

`src/libraries/SafeTransferLib.sol` lines 24–34 — `safeTransferFrom` checks only call success and boolean return; it never compares pre/post balances: [3](#0-2) 

**Root cause:** The accounting write at line 418 uses the *nominal* spread, not the *received* amount. There is no balance-before/balance-after check anywhere in `take()`.

**Attacker-controlled inputs:**

1. Attacker deploys `ContractFeeToken` — an ERC-20 that deducts a fee (e.g., 1%) only when `to` is a contract (`to.code.length > 0`), and returns `true` unconditionally.
2. Attacker (account A = maker) creates an offer whose `market.loanToken = address(ContractFeeToken)`.
3. Attacker (account B = taker, or any taker) calls `take()`.

**Exploit flow:**

| Step | Action | Effect |
|---|---|---|
| Line 418 | `claimableSettlementFee[token] += (buyerAssets - sellerAssets)` | Credited full spread S |
| Line 455 | `safeTransferFrom(token, payer, address(this), S)` | Midnight receives `S - fee` (contract-recipient fee deducted) |
| Line 456 | `safeTransferFrom(token, payer, receiver, sellerAssets)` | Receiver gets full `sellerAssets` (EOA, no fee) |

After the call: `claimableSettlementFee[token]` exceeds the actual balance increase by exactly `fee`.

**Why existing checks fail:**

- `SafeTransferLib.safeTransferFrom` only validates the boolean return value, not the received quantity.
- The Certora formal verification explicitly assumes away this class of token: *"Assume ERC20 tokens transfer correctly: no fee taking from sender or receiver"* — this is a prover assumption, not a runtime guard. [4](#0-3) 
- The `tokenBalanceCorrect` strong invariant and `pendingFeeReceiptZero` weak invariant are only proven under that assumption; they provide no on-chain protection. [5](#0-4) 

### Impact Explanation

After each `take()` with a contract-recipient-fee token, `claimableSettlementFee[token]` exceeds the contract's actual token balance by the fee amount. Repeated calls accumulate the shortfall. When `claimSettlementFee` is eventually called for the full credited amount, the outbound `safeTransfer` reverts (insufficient balance), permanently locking the fee accounting and rendering the protocol insolvent for that token. The core invariant — *contract balances cover fees* — is concretely broken. [6](#0-5) 

### Likelihood Explanation

**Preconditions:**
- Attacker deploys a contract-recipient-fee ERC-20 (trivial, no privileges needed).
- Attacker creates a market with that token as `loanToken` (permissionless via `touchMarket` inside `take()`).
- Attacker controls maker and taker accounts (two EOAs, no admin role required).

**Feasibility:** Fully reachable with no oracle manipulation, no admin key, and no user mistake. Repeatable on every `take()` call, accumulating the shortfall linearly.

### Recommendation

Replace the two sequential `safeTransferFrom` calls with a balance-check pattern:

```solidity
uint256 balanceBefore = IERC20(offer.market.loanToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
require(
    IERC20(offer.market.loanToken).balanceOf(address(this)) - balanceBefore >= buyerAssets - sellerAssets,
    FeeOnTransferToken()
);
```

Alternatively, document and enforce (via a registry or allowlist) that only standard ERC-20 tokens with no transfer fees are permitted as `loanToken`.

### Proof of Concept

// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";

contract ContractFeeToken {
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;
    uint256 constant FEE_BPS = 100; // 1% fee when recipient is a contract

    function mint(address to, uint256 amount) external { balanceOf[to] += amount; }
    function approve(address spender, uint256 amount) external returns (bool) {
        allowance[msg.sender][spender] = amount; return true;
    }
    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        allowance[from][msg.sender] -= amount;
        uint256 fee = to.code.length > 0 ? amount * FEE_BPS / 10000 : 0;
        balanceOf[from] -= amount;
        balanceOf[to] += amount - fee; // fee is burned / kept by token
        return true;
    }
    function transfer(address to, uint256 amount) external returns (bool) {
        return this.transferFrom(msg.sender, to, amount);
    }
}

contract FeeOnTransferTakeTest is Test {
    Midnight midnight;
    ContractFeeToken token;
    address maker = makeAddr("maker");
    address taker = makeAddr("taker");

    function setUp() public { /* deploy midnight, set up roles */ }

    function testClaimableSettlementFeeExceedsBalance() public {
        // 1. Create market with ContractFeeToken as loanToken
        // 2. Maker creates buy offer; taker takes it
        // 3. Record spread S = buyerAssets - sellerAssets
        uint256 spread = /* buyerAssets - sellerAssets from take() return */ 0;
        uint256 credited = midnight.claimableSettlementFee(address(token));
        uint256 actualBalance = token.balanceOf(address(midnight));

        // Key assertion: credited > actual balance received
        assertGt(credited, actualBalance, "claimableSettlementFee exceeds real balance");

        // Attempting to claim the full credited amount reverts
        vm.prank(/* feeClaimer */address(0));
        vm.expectRevert();

**Expected assertions:**
- `claimableSettlementFee[token] > token.balanceOf(address(midnight))` — invariant broken.
- `claimSettlementFee(token, credited, receiver)` reverts — fee is unclaimable, protocol insolvent for this token.
