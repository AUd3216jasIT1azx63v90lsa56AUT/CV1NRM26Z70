### Title
Fee-on-Transfer `loanToken` in `repay()` Inflates `withdrawable` Beyond Actual Contract Balance - (`File: src/Midnight.sol`)

### Summary

`repay()` unconditionally credits `marketState[id].withdrawable` by the full `units` parameter before calling `SafeTransferLib.safeTransferFrom`, which only verifies the boolean return value of `transferFrom` and never checks the actual tokens received. When the market's `loanToken` silently deducts a transfer fee, the contract receives `units * (1 - fee_rate)` but records `units` as withdrawable, permanently inflating the accounting. Because `touchMarket()` is permissionless and imposes no restriction on `loanToken`, any unprivileged actor can create such a market.

### Finding Description

**Reachable code path:**

1. Any caller invokes `touchMarket(market)` with a fee-on-transfer ERC20 as `market.loanToken`. `touchMarket` (lines 755–791) validates only maturity, collateral count, LLTV tiers, and maxLif — no check on `loanToken` properties.

2. A borrower acquires debt in that market (via `take()`).

3. The borrower calls `repay(market, units, onBehalf, address(0), '')`:
   - Line 508: `position[id][onBehalf].debt -= UtilsLib.toUint128(units);`
   - Line 509: `marketState[id].withdrawable += UtilsLib.toUint128(units);`
   - Line 520: `SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);`

4. `SafeTransferLib.safeTransferFrom` (lines 24–34 of `src/libraries/SafeTransferLib.sol`) calls `token.transferFrom(from, to, value)` and checks only that the call did not revert and returned `true`. It does not compare `balanceOf(address(this))` before and after. A fee-on-transfer token returns `true` while delivering only `units * (1 - fee_rate)`.

**Result per repayment of `units` with fee rate `f`:**
- `withdrawable` increases by `units`
- `loanToken.balanceOf(address(midnight))` increases by `units * (1 - f)`
- Deficit per call: `units * f`

After N repayments the cumulative deficit is `N * units * f`. When lenders call `withdraw()` (line 499: `SafeTransferLib.safeTransfer(market.loanToken, receiver, units)`), the contract will revert for the last lenders once the real balance is exhausted, while `withdrawable` still shows a positive value.

**Attacker-controlled inputs:** `market` (with fee-on-transfer `loanToken`), `units` (any value up to `type(uint128).max`), `onBehalf` (self or authorized address), `callback = address(0)`, `data = ''`.

**Existing protections reviewed and found insufficient:**
- `require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender])` — only an authorization check, irrelevant to token accounting.
- `SafeTransferLib` — checks revert and boolean return, not received amount.
- No allowlist or denylist on `loanToken` in `touchMarket`.
- No balance-before/after guard anywhere in `repay()`.

### Impact Explanation

Protocol insolvency scoped to the affected market: `marketState[id].withdrawable` exceeds `loanToken.balanceOf(address(midnight))` by the cumulative fee amount. Lenders who call `withdraw()` last will receive a revert (insufficient token balance), permanently losing their entitled loan tokens. The invariant `withdrawable <= actual loanToken balance held by Midnight` is broken.

### Likelihood Explanation

**Preconditions:**
1. A fee-on-transfer token is used as `loanToken` — permissionlessly achievable by any market creator.
2. At least one borrower has debt and calls `repay()` — normal protocol usage.

No privileged action, oracle manipulation, or user mistake is required. The bug triggers on every `repay()` call in such a market. It is repeatable and cumulative: each repayment widens the deficit. Any borrower, including the market creator themselves, can trigger it.

### Recommendation

Record the contract's `loanToken` balance before and after the `safeTransferFrom` call in `repay()`, and credit `withdrawable` only with the actual received amount (the balance delta), not the nominal `units` parameter. Apply the same fix to `liquidate()` (line 717), which has an identical pattern. Example:

```solidity
uint256 balanceBefore = IERC20(market.loanToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);
uint256 received = IERC20(market.loanToken).balanceOf(address(this)) - balanceBefore;
// use `received` instead of `units` for withdrawable and debt accounting
```

Alternatively, document that fee-on-transfer tokens are unsupported and add an explicit check (e.g., a balance-delta assertion) that reverts if `received < units`.

### Proof of Concept

pragma solidity ^0.8.0;

import "forge-std/Test.sol";

// FeeToken: 1% fee on every transferFrom, returns true
contract FeeToken {
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;
    uint256 constant FEE_BPS = 100; // 1%

    function mint(address to, uint256 amt) external { balanceOf[to] += amt; }
    function approve(address sp, uint256 amt) external returns (bool) {
        allowance[msg.sender][sp] = amt; return true;
    }
    function transferFrom(address from, address to, uint256 amt) external returns (bool) {
        allowance[from][msg.sender] -= amt;
        uint256 fee = amt * FEE_BPS / 10000;
        balanceOf[from] -= amt;
        balanceOf[to] += amt - fee; // fee silently burned
        return true;
    }
    function transfer(address to, uint256 amt) external returns (bool) {
        balanceOf[msg.sender] -= amt; balanceOf[to] += amt; return true;
    }
}

contract FeeOnTransferRepayTest is Test {
    // Deploy Midnight with FeeToken as loanToken,
    // set up a market, give borrower debt of `units`,

    // After lender calls withdraw(market, units, lender, lender):
    //   // Reverts with insufficient balance if deficit >= contract balance,
    //   // or lender receives less than units if partial.

    // Invariant fuzz: after N repayments of random `units`:

    //   // This assertion FAILS, proving the bug.
}

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
