### Title
Missing monotonicity enforcement on settlement fee breakpoints allows inverted fee schedule - (File: src/Midnight.sol)

### Summary
`setDefaultSettlementFee` and `setMarketSettlementFee` validate each breakpoint only against its own per-index ceiling (`maxSettlementFee(index)`) with no cross-breakpoint ordering constraint. When the `feeSetter` sets `defaultSettlementFeeCbp[loanToken][0] > defaultSettlementFeeCbp[loanToken][1]`, any unprivileged market creator calling `touchMarket` snapshots those inverted defaults verbatim into the new `MarketState`. The piecewise linear interpolation in `settlementFee` then produces a fee that strictly decreases as TTM increases from 0 to 1 day, inverting the intended monotonic schedule and letting takers exploit lower fees at longer TTM.

### Finding Description

**Root cause — missing monotonicity check in fee setters:**

`setDefaultSettlementFee` (lines 277–285) accepts any value up to `maxSettlementFee(index)` for each index independently:

```solidity
require(newSettlementFee <= maxSettlementFee(index), SettlementFeeTooHigh());
require(newSettlementFee % CBP == 0, FeeNotMultipleOfFeeCbp());
``` [1](#0-0) 

No check enforces `fee[i] <= fee[i+1]`. The same applies to `setMarketSettlementFee` (lines 258–275). [2](#0-1) 

**Concrete inversion possible within the allowed ranges:**

From `ConstantsLib.sol`:
- `MAX_SETTLEMENT_FEE_0_DAYS = 0.000014e18`
- `MAX_SETTLEMENT_FEE_1_DAY  = 0.000014e18` [3](#0-2) 

Both indices share the same ceiling, so the feeSetter can legally set index 0 to `0.000014e18` and index 1 to `0`, producing `settlementFeeCbp0 = 14, settlementFeeCbp1 = 0`.

**`touchMarket` copies defaults without any ordering check (lines 777–784):**

```solidity
uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
_marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
_marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
...
``` [4](#0-3) 

Any unprivileged caller can trigger `touchMarket` (it is `public` and called internally by `take`, `withdraw`, `repay`, `supplyCollateral`, `withdrawCollateral`, `liquidate`).

**`settlementFee` interpolation inverts for TTM ∈ [0, 1 day) (lines 970–979):**

```solidity
(uint256 start, uint256 end, uint256 feeLower, uint256 feeUpper) =
    timeToMaturity < 1 days ? (0 days, 1 days,
        _marketState.settlementFeeCbp0 * CBP,
        _marketState.settlementFeeCbp1 * CBP) : ...;

return (feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start)) / (end - start);
``` [5](#0-4) 

With `feeLower = 0.000014e18` and `feeUpper = 0`:
- TTM = 0 s → fee = `0.000014e18` (maximum)
- TTM = 43200 s (0.5 day) → fee = `0.000007e18`
- TTM = 86400 s (1 day) → fee = `0`

The fee strictly decreases as TTM increases — the invariant is violated.

**`take` uses this fee directly (lines 360–362):**

```solidity
uint256 _settlementFee = settlementFee(id, timeToMaturity);
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice;
uint256 buyerPrice = sellerPrice + _settlementFee;
``` [6](#0-5) 

No existing check in `take` or `settlementFee` detects or rejects an inverted schedule.

**Exploit flow:**
1. `feeSetter` calls `setDefaultSettlementFee(loanToken, 0, 0.000014e18)` and `setDefaultSettlementFee(loanToken, 1, 0)`.
2. Unprivileged market creator calls `touchMarket(market)` — market is created with `settlementFeeCbp0 = 14, settlementFeeCbp1 = 0`.
3. Taker with TTM near 1 day calls `take` and pays near-zero settlement fee, while a taker with TTM near 0 pays the maximum fee — the opposite of the intended schedule.

### Impact Explanation

The fee schedule stored in `MarketState` is inverted for the [0, 1 day) TTM segment. Takers can time their trades to TTM just below 1 day to pay the minimum possible settlement fee (approaching 0) instead of the maximum. The protocol collects less fee revenue than intended, and the fee invariant (fee non-decreasing with TTM) is broken in on-chain state for the lifetime of the market.

### Likelihood Explanation

The precondition requires the `feeSetter` to set `fee[0] > fee[1]`, which is permitted by the contract. This can happen accidentally (the two indices share the same max ceiling, making it easy to set index 0 high and index 1 low) or deliberately. Once set, the inversion is inherited by every new market created for that loan token until the defaults are corrected. The exploit is repeatable across any number of markets and takes.

### Recommendation

Add a monotonicity check in both `setDefaultSettlementFee` and `setMarketSettlementFee` after updating the breakpoint, asserting that `fee[i] <= fee[i+1]` for all adjacent pairs. Alternatively, enforce the full ordering across all 7 breakpoints atomically when any single index is updated. A simpler approach is to require the caller to supply all 7 values at once and validate the full sorted order in a single transaction.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
// ... standard test setup

contract InvertedFeeInvariantTest is Test {
    Midnight midnight;
    address loanToken = address(0xABC);

    function setUp() public {
        midnight = new Midnight();
        // feeSetter = address(this) via roleSetter
        midnight.setFeeSetter(address(this));
    }

    /// @dev Fuzz all 7 breakpoints within their per-index ceilings.
    /// Assert settlementFee is monotonically non-decreasing with TTM.
    function testFuzz_settlementFeeMonotonicity(
        uint16 cbp0, uint16 cbp1, uint16 cbp2,
        uint16 cbp3, uint16 cbp4, uint16 cbp5, uint16 cbp6,
        uint256 ttm1, uint256 ttm2
    ) public {
        // Bound each cbp to its per-index max (in cbp units)
        cbp0 = uint16(bound(cbp0, 0, maxSettlementFee(0) / CBP));
        cbp1 = uint16(bound(cbp1, 0, maxSettlementFee(1) / CBP));
        // ... similarly for cbp2..cbp6

        midnight.setDefaultSettlementFee(loanToken, 0, uint256(cbp0) * CBP);
        midnight.setDefaultSettlementFee(loanToken, 1, uint256(cbp1) * CBP);
        // ... set cbp2..cbp6

        bytes32 id = midnight.touchMarket(buildMarket(loanToken));

        ttm1 = bound(ttm1, 0, 360 days);
        ttm2 = bound(ttm2, ttm1, 360 days); // ttm2 >= ttm1

        uint256 fee1 = midnight.settlementFee(id, ttm1);
        uint256 fee2 = midnight.settlementFee(id, ttm2);

        // EXPECTED: fee non-decreasing with TTM
        // ACTUAL: fails when cbp0 > cbp1 (e.g., cbp0=14, cbp1=0)
        assertLe(fee1, fee2, "fee must be non-decreasing with TTM");
    }
}
```

**Expected assertion failure:** With `cbp0 = 14, cbp1 = 0`, `ttm1 = 0, ttm2 = 43200` (0.5 day): `fee1 = 0.000014e18 > fee2 = 0.000007e18`, causing `assertLe` to revert.
