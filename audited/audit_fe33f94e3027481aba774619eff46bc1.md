### Title
Unbounded `_poolsByAgreement` array enables sponsor-triggered OOG DoS of `getPoolsByAgreement` in constrained-gas callers — (`File: src/ConfidencePoolFactory.sol`)

### Summary
`createPool` imposes no per-agreement pool cap, so a pool sponsor (agreement owner) can push an unbounded number of entries into `_poolsByAgreement[agreement]`. `getPoolsByAgreement` returns the entire array in a single memory allocation with no pagination or gas guard. Any on-chain consumer that calls `getPoolsByAgreement` inside a callback or with a forwarded gas limit will OOG once the array is large enough, permanently breaking that integration for the targeted agreement.

### Finding Description
`_poolsByAgreement` is declared as an unbounded dynamic array: [1](#0-0) 

Every successful `createPool` call appends one entry unconditionally: [2](#0-1) 

`createPool` enforces no per-agreement limit; the code comment explicitly acknowledges unbounded growth as intentional for off-chain curation: [3](#0-2) 

`getPoolsByAgreement` copies the entire storage array into memory in one shot: [4](#0-3) 

There is no pagination, no length cap, and no gas check. Reading N cold storage slots costs ≈ 2 100 gas each (EIP-2929). At a 100 000 gas budget, the function OOGs at roughly N ≈ 47 pools (100 000 / 2 100 ≈ 47, before accounting for memory expansion and ABI-encoding overhead, which lower the threshold further). A sponsor who owns a valid agreement and uses an allowlisted token can reach N = 47 in a single block sequence; the only prerequisite is paying the clone-deployment gas per call.

The attack sequence:
1. Sponsor owns a valid agreement; factory owner has allowlisted at least one token (normal operation).
2. Sponsor calls `createPool` N times (same agreement, same or different parameters). Each call succeeds; `_poolsByAgreement[agreement]` grows by one.
3. A third-party contract (e.g., a DeFi integration, a settlement hook, a keeper) calls `getPoolsByAgreement(agreement)` with ≤ 100 000 forwarded gas.
4. The call OOGs. The caller's transaction reverts. If the caller is a callback, the entire outer transaction reverts.

`docs/DESIGN.md` does not mention this behavior as accepted. The companion function `poolCountByAgreement` exists and is O(1), confirming the design did not intend `getPoolsByAgreement` to be the only discovery path, but no pagination alternative is provided. [5](#0-4) 

### Impact Explanation
Any on-chain protocol that integrates `getPoolsByAgreement` inside a callback, a low-gas forwarded call, or a `try/catch` with bounded gas is permanently DoS'd for the targeted agreement once N exceeds the OOG threshold. The DoS is permanent (the array never shrinks) and agreement-specific (only the targeted agreement's consumers are affected). No funds are lost within the scoped contracts, but integrations relying on this view for routing, settlement gating, or discovery are bricked.

### Likelihood Explanation
The sponsor is the agreement owner — a semi-trusted actor explicitly listed in the actor model, distinct from the fully-trusted factory owner and moderator. Creating multiple pools per agreement is explicitly permitted by the protocol ("duplicates are allowed and curated off-chain"). A malicious or griefing sponsor needs only to call `createPool` ~47–100 times (cost: ~47 × ~350 000 gas ≈ 16.5 M gas, feasible across a few blocks on an L2 with low base fees). No victim mistake is required; the DoS fires automatically on the next constrained-gas call to `getPoolsByAgreement`. The attack is repeatable (more pools can always be added) and irreversible (no pool removal exists).

### Recommendation
Add a paginated view alongside the existing full-return function, and optionally cap pools per agreement:

```solidity
// Paginated accessor — O(pageSize) gas, safe in callbacks
function getPoolsByAgreement(address agreement, uint256 offset, uint256 limit)
    external view returns (address[] memory page, uint256 total)
{
    address[] storage all = _poolsByAgreement[agreement];
    total = all.length;
    uint256 end = offset + limit > total ? total : offset + limit;
    page = new address[](end - offset);
    for (uint256 i = offset; i < end; i++) {
        page[i - offset] = all[i];
    }
}
```

Alternatively (or additionally), enforce a per-agreement pool cap in `createPool`:

```solidity
uint256 private constant _MAX_POOLS_PER_AGREEMENT = 64;
// inside createPool, before the push:
if (_poolsByAgreement[agreement].length >= _MAX_POOLS_PER_AGREEMENT)
    revert TooManyPoolsForAgreement();
```

The O(1) `poolCountByAgreement` should be retained as-is for cheap length checks.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.26;

import "forge-std/Test.sol";
import {ConfidencePoolFactory} from "src/ConfidencePoolFactory.sol";
// ... standard test setup imports

contract PoolConsumer {
    IConfidencePoolFactory public factory;
    constructor(address f) { factory = IConfidencePoolFactory(f); }

    // Simulates a callback or low-gas integration call
    function process(address agreement) external returns (address[] memory) {
        // Forward only 100_000 gas — realistic for a callback
        return factory.getPoolsByAgreement{gas: 100_000}(agreement);
    }
}

contract GetPoolsByAgreementOOGTest is Test {
    ConfidencePoolFactory factory;
    PoolConsumer consumer;
    address sponsor;
    address agreement; // mock agreement owned by sponsor

    function setUp() public {
        // deploy factory, mock registry, mock agreement, allowlist token
        // sponsor = agreement owner
    }

    function test_OOGAfterNPools() public {
        uint256 N = 60; // expected to exceed OOG threshold at 100k gas
        for (uint256 i = 0; i < N; i++) {
            vm.prank(sponsor);
            factory.createPool(agreement, token, block.timestamp + 31 days, 0, recovery, new address[](0));
        }
        assertEq(factory.poolCountByAgreement(agreement), N);

        // Assert: consumer.process() reverts (OOG) once N is large enough
        vm.expectRevert();
        consumer.process(agreement);

        // Bisect to find exact threshold: repeat with N=40 (should succeed), N=60 (should OOG)
        // assert threshold is between 40 and 60 on the target chain's gas schedule
    }
}
```

**Decisive assertions:**
- `factory.poolCountByAgreement(agreement) == N` — confirms N pools were created.
- `consumer.process(agreement)` reverts with OOG at N ≥ threshold — confirms the DoS.
- `consumer.process(agreement)` succeeds at N < threshold — confirms the threshold is finite and reachable.
