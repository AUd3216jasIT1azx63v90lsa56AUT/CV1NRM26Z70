### Title
Unconditional `safeTransfer` on self-destructed collateral token permanently DoS-es all `liquidate()` paths - (`File: src/libraries/SafeTransferLib.sol` / `src/Midnight.sol`)

### Summary
`liquidate()` calls `SafeTransferLib.safeTransfer(collateralToken, receiver, seizedAssets)` unconditionally at line 696 — outside the `if (repaidUnits > 0 || seizedAssets > 0)` guard — meaning it executes even when `seizedAssets == 0`. `SafeTransferLib.safeTransfer` checks `token.code.length > 0` before every call regardless of value, so if the collateral token's bytecode has been removed (via `SELFDESTRUCT`), every invocation of `liquidate()` reverts with `NoCode()`. Because `touchMarket()` never validates that a collateral token has deployed code, an unprivileged attacker can register a self-destructable token as collateral, open borrower positions, destroy the token, and permanently freeze all liquidation paths for every borrower in that market.

### Finding Description

**Code path and root cause**

`touchMarket()` validates collateral tokens only for sort order, allowed lltv, and valid maxLif: [1](#0-0) 

There is no `require(collateralToken.code.length > 0)` check. Any address — including one whose bytecode will later be erased — is accepted.

`SafeTransferLib.safeTransfer` checks code existence unconditionally: [2](#0-1) 

Inside `liquidate()`, the collateral transfer at line 696 is placed **after** the `if (repaidUnits > 0 || seizedAssets > 0)` block that closes at line 677, making it unconditional: [3](#0-2) [4](#0-3) 

Even the documented "bad-debt realization with 0 tokens transferred" path (both inputs zero) reaches line 696 and calls `safeTransfer(collateralToken, receiver, 0)`, which still executes the `code.length` guard and reverts.

**Exploit flow**

1. Attacker deploys `SelfDestructableToken` — a minimal ERC-20 with a `destroy()` function that calls `SELFDESTRUCT`.
2. Attacker calls `touchMarket()` (or any function that triggers it) with `collateralParams[0].token = address(selfDestructableToken)`. Market is created; no code check is performed.
3. Attacker (or any borrower) calls `supplyCollateral()` and then `take()` to open a debt position against this collateral. Token has code at this point, so `safeTransferFrom` succeeds.
4. Attacker calls `selfDestructableToken.destroy()`. On post-EIP-6780 chains this requires the token to have been created in the same transaction (achievable via a factory contract that creates the token, creates the market, supplies collateral, borrows, and then self-destructs — all atomically). On pre-Cancun chains any `SELFDESTRUCT` removes the code.
5. `collateralToken.code.length == 0` from this point forward.
6. Any call to `liquidate(market, collateralIndex, *, *, borrower, *, *, *, *)` — regardless of `seizedAssets` and `repaidUnits` values — hits line 696 and reverts with `NoCode()`.

**Why existing checks fail**

- `touchMarket` has no `code.length` guard on collateral tokens.
- The `if (repaidUnits > 0 || seizedAssets > 0)` guard at line 643 does not wrap line 696; the transfer is unconditional.
- There is no alternative liquidation path that skips the collateral `safeTransfer`.
- The `liquidationLocked` mechanism and `NotLiquidatable` check are passed before the revert, so the revert is not a false-positive health check — it is a hard transfer failure.

### Impact Explanation
Every borrower in the affected market becomes permanently unliquidatable. Bad debt cannot be realized (the `seizedAssets = 0, repaidUnits = 0` path also reverts). Lender credit is permanently at risk because `withdrawable` can never be increased via liquidation, and the `lossFactor` accounting for bad debt is never applied. The market is effectively bricked for all liquidation-dependent operations.

### Likelihood Explanation
The preconditions are fully attacker-controlled: deploy a self-destructable token (no privilege required), call the permissionless `touchMarket`, supply collateral, borrow, then destroy the token. On post-EIP-6780 chains the self-destruct must occur in the same transaction as token creation, which is achievable with a single factory contract. The attack is one-shot and irreversible; once the token is destroyed the DoS is permanent. The attacker has direct financial incentive: they borrow against collateral they then render unliquidatable, keeping the borrowed assets while their position can never be closed by force.

### Recommendation
Add a `code.length` check inside `touchMarket()` for each collateral token:

```solidity
require(collateralToken.code.length > 0, NoCode());
```

Additionally, guard the collateral `safeTransfer` in `liquidate()` so it is only called when `seizedAssets > 0`:

```solidity
if (seizedAssets > 0) {
    SafeTransferLib.safeTransfer(
        market.collateralParams[collateralIndex].token, receiver, seizedAssets
    );
}
```

Both fixes together close the attack surface: the first prevents registration of codeless tokens; the second ensures the zero-seizure bad-debt path never attempts a transfer on a potentially codeless address.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import "../src/Midnight.sol";
import "../src/libraries/SafeTransferLib.sol";

contract SelfDestructableToken {
    // Minimal ERC-20 that always returns true for transfers
    mapping(address => uint256) public balanceOf;
    function mint(address to, uint256 amt) external { balanceOf[to] += amt; }
    function transfer(address to, uint256 amt) external returns (bool) {
        balanceOf[msg.sender] -= amt; balanceOf[to] += amt; return true;
    }
    function transferFrom(address from, address to, uint256 amt) external returns (bool) {
        balanceOf[from] -= amt; balanceOf[to] += amt; return true;
    }
    function approve(address, uint256) external returns (bool) { return true; }
    function destroy() external { selfdestruct(payable(msg.sender)); }
}

contract SelfDestructDoSTest is Test {
    Midnight midnight;
    SelfDestructableToken collToken;
    ERC20 loanToken;

    function setUp() public {
        midnight = new Midnight(/* constructor args */);
        collToken = new SelfDestructableToken();
        loanToken = new ERC20(/* ... */);
    }

    function testSelfDestructCollateralDoSLiquidate() public {
        // 1. Build market with self-destructable collateral
        Market memory market = _buildMarket(address(collToken), address(loanToken));
        midnight.touchMarket(market);

        // 2. Supply collateral and open a borrow position
        collToken.mint(address(this), 1e18);
        collToken.approve(address(midnight), 1e18);
        midnight.supplyCollateral(market, 0, 1e18, address(this));
        // ... take an offer to create debt ...

        // 3. Self-destruct the collateral token
        collToken.destroy();
        assertEq(address(collToken).code.length, 0, "token code removed");

        // 4. Make position unhealthy (e.g., drop oracle price)
        // oracle.setPrice(0.5e36);

        // 5. Assert ALL liquidate paths revert with NoCode
        vm.expectRevert(SafeTransferLib.NoCode.selector);
        midnight.liquidate(market, 0, 1e18, 0, address(this), false, address(this), address(0), "");

        vm.expectRevert(SafeTransferLib.NoCode.selector);
        midnight.liquidate(market, 0, 0, 1e18, address(this), false, address(this), address(0), "");

        // Bad-debt realization path (both zero) also reverts
        vm.expectRevert(SafeTransferLib.NoCode.selector);
        midnight.liquidate(market, 0, 0, 0, address(this), false, address(this), address(0), "");
    }
}
```

Expected assertions: all three `liquidate` calls revert with `SafeTransferLib.NoCode.selector`; borrower debt remains non-zero; `withdrawable` is unchanged; lender credit is permanently at risk.
