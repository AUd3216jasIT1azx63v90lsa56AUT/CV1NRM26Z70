### Title
`buyerAssetsToUnits` reverts with `PriceGreaterThanOne` for valid sell offers at `MAX_TICK` with non-zero `settlementFeeCbp0` at TTM=0, causing `buyWithAssetsTargetAndWithdrawCollateral` to revert - (`File: src/periphery/TakeAmountsLib.sol`)

### Summary

`TakeAmountsLib.buyerAssetsToUnits` applies `require(buyerPrice <= WAD)` unconditionally for both buy and sell offers. For a sell offer, `buyerPrice = offerPrice + settlementFee`. When `tick = MAX_TICK`, `tickToPrice(MAX_TICK) = WAD` exactly, so `buyerPrice = WAD + settlementFee > WAD` whenever `settlementFee > 0`, triggering the revert. `Midnight.take` has no such guard and executes successfully for the identical inputs. This divergence causes `MidnightBundles.buyWithAssetsTargetAndWithdrawCollateral` to revert entirely for any sell offer at `MAX_TICK` in a market with a non-zero 0-day settlement fee at or after maturity.

### Finding Description

**`tickToPrice(MAX_TICK) = WAD` ŌĆö confirmed arithmetically.**

In `TickLib.tickToPrice` at `tick = MAX_TICK = 5820`:
- Exponent = `LN_ONE_PLUS_DELTA * (2910 ŌłÆ 5820)` ŌĆö a large negative integer.
- `wExp(large_negative)` = `1e36 / wExp(large_positive)` = `0` (integer division truncates to zero).
- Denominator = `1e18 + 0 = 1e18`.
- `divHalfDownUnchecked(1e36, 1e18) = 1e18 = WAD`.
- After `PRICE_ROUNDING_STEP` rounding: still `WAD`.

So `tickToPrice(MAX_TICK) = 1e18 = WAD`.

**`settlementFee(id, 0) = settlementFeeCbp0 * CBP` ŌĆö confirmed from `Midnight.settlementFee`.**

When `timeToMaturity = 0 < 1 days`, the piecewise formula selects `feeLower = settlementFeeCbp0 * CBP`, `feeUpper = settlementFeeCbp1 * CBP`, and returns `(feeLower * 1 days + feeUpper * 0) / 1 days = settlementFeeCbp0 * CBP`.

**Divergence in `buyerAssetsToUnits` (sell offer path):**

```solidity
// TakeAmountsLib.sol lines 22-28
uint256 offerPrice = TickLib.tickToPrice(offer.tick);          // = WAD
uint256 settlementFee = IMidnight(midnight).settlementFee(...); // = settlementFeeCbp0 * CBP > 0
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
// offer.buy = false ŌåÆ sellerPrice = offerPrice = WAD
uint256 buyerPrice = sellerPrice + settlementFee;
// buyerPrice = WAD + settlementFee > WAD
require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne()); // ŌåÉ REVERTS
```

**`Midnight.take` for the identical sell offer (lines 358ŌĆō364):**

```solidity
uint256 offerPrice = TickLib.tickToPrice(offer.tick);   // = WAD
uint256 _settlementFee = settlementFee(id, timeToMaturity); // > 0
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice; // = WAD
uint256 buyerPrice = sellerPrice + _settlementFee;       // = WAD + fee
uint256 buyerAssets = units.mulDivUp(buyerPrice, WAD);   // valid, no guard
```

No `require(buyerPrice <= WAD)` exists in `take`. The take proceeds and returns `(buyerAssets, sellerAssets)` without reverting.

**Bundler propagation ŌĆö `buyWithAssetsTargetAndWithdrawCollateral` (lines 208ŌĆō211):**

```solidity
uint256 unitsToTake = min(
    TakeAmountsLib.buyerAssetsToUnits(MIDNIGHT, id, takes[i].offer, ...),  // ŌåÉ reverts here
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
);
```

`buyerAssetsToUnits` is called outside any `try/catch`. The bundler's own NatSpec at line 175 confirms: *"Reverts if TakeAmountsLib or ConsumableUnitsLib reverts."* The revert propagates to the top-level call.

**The existing test suite independently confirms the gap.** `TakeAmountsTest._maxTick` (lines 69ŌĆō74) explicitly computes the highest tick where `tickToPrice(tick) + settlementFee <= WAD` and all four fuzz tests (lines 99, 121, 145, 168) bound `tick` strictly below `MAX_TICK` when `settlementFee > 0`. Separate `testSnappedBuyerAssets*` tests (lines 185ŌĆō227) handle the `MAX_TICK` case by computing `targetUnits` directly and calling `take` directly ŌĆö deliberately bypassing `buyerAssetsToUnits`. This confirms the library is intentionally not called for this case in unit tests, but the bundler does call it unconditionally.

### Impact Explanation

`MidnightBundles.buyWithAssetsTargetAndWithdrawCollateral` is completely unusable for any sell offer at `MAX_TICK` in a market where `settlementFeeCbp0 > 0` once `block.timestamp >= offer.market.maturity`. The entire transaction reverts; because `pullToken` (line 197) is part of the same transaction, the user's tokens are returned by the EVM revert ŌĆö they are not permanently frozen. The concrete impact is a **DoS of the bundler function** for this valid offer configuration: the user cannot use the assets-target bundler path and must fall back to calling `Midnight.take` directly with a manually computed `units` value. The "freezing" framing in the question overstates the severity; no funds are permanently lost, but the bundler is broken for a reachable and legitimate market state.

### Likelihood Explanation

**Preconditions:**
1. `settlementFeeCbp0 > 0`: The fee setter sets a non-zero default settlement fee for the loan token before market creation (copied into `_marketState.settlementFeeCbp0` in `touchMarket`), or calls `setMarketSettlementFee(id, 0, fee)` directly. `MAX_SETTLEMENT_FEE_0_DAYS = 0.000014e18 = 14 * CBP`, so any value from `1 * CBP` to `14 * CBP` triggers the bug.
2. `tick = MAX_TICK = 5820`: A valid tick (5820 is a multiple of `DEFAULT_TICK_SPACING = 4`). Makers offering at the highest price (WAD) naturally place offers here.
3. `block.timestamp >= offer.market.maturity`: Normal post-maturity state.

All three conditions are routine and can coexist in production. The bug is repeatable for every call to `buyWithAssetsTargetAndWithdrawCollateral` that includes a sell offer at `MAX_TICK` under these conditions.

### Recommendation

In `buyerAssetsToUnits`, the `require(buyerPrice <= WAD)` guard is only meaningful for **buy offers** (where `buyerPrice = offerPrice` and the check mirrors Midnight's own constraint). For **sell offers**, `buyerPrice = offerPrice + settlementFee` can legitimately exceed WAD at `MAX_TICK`, and `Midnight.take` handles this correctly. The fix is to scope the guard to buy offers only:

```solidity
// Only enforce for buy offers; for sell offers buyerPrice > WAD is valid.
require(!offer.buy || buyerPrice <= WAD, TickLib.PriceGreaterThanOne());
```

Alternatively, document and handle the sell-offer case by computing `units = targetBuyerAssets.mulDivDown(WAD, buyerPrice)` even when `buyerPrice > WAD`, since the inverse mapping remains well-defined.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {TakeAmountsLib} from "../src/periphery/TakeAmountsLib.sol";
import {TickLib, MAX_TICK} from "../src/libraries/TickLib.sol";
import {WAD, CBP} from "../src/libraries/ConstantsLib.sol";
import {Market, Offer, CollateralParams} from "../src/interfaces/IMidnight.sol";

contract BugPoC is BaseTest {
    function testBuyerAssetsToUnitsRevertsButTakeSucceeds() public {
        // 1. Set non-zero 0-day settlement fee (fee setter action, normal protocol op)
        uint256 fee0 = 1 * CBP; // 1e12, well within MAX_SETTLEMENT_FEE_0_DAYS
        midnight.setMarketSettlementFee(id, 0, fee0);

        // 2. Advance time to maturity (TTM = 0)
        vm.warp(market.maturity);

        // Confirm: tickToPrice(MAX_TICK) == WAD
        assertEq(TickLib.tickToPrice(MAX_TICK), WAD);

        // Confirm: settlementFee at TTM=0 == fee0
        assertEq(midnight.settlementFee(id, 0), fee0);

        // 3. buyerPrice = WAD + fee0 > WAD
        // Assert buyerAssetsToUnits REVERTS with PriceGreaterThanOne
        Offer memory sellOffer = buildSellOffer(MAX_TICK);
        vm.expectRevert(TickLib.PriceGreaterThanOne.selector);
        TakeAmountsLib.buyerAssetsToUnits(address(midnight), id, sellOffer, 1e18);

        // 4. Assert Midnight.take does NOT revert for the same offer
        uint256 units = 1e18;
        deal(address(loanToken), lender, type(uint256).max);
        collateralize(market, borrower, units);
        sellOffer.maker = borrower;
        sellOffer.receiverIfMakerIsSeller = borrower;
        // take succeeds: buyerAssets = units.mulDivUp(WAD + fee0, WAD) > units
        (uint256 buyerAssets,) = take(units, lender, sellOffer);
        assertGt(buyerAssets, units); // buyer paid more than 1:1 due to settlement fee
    }
}
```

**Expected assertions:**
- `vm.expectRevert(TickLib.PriceGreaterThanOne.selector)` passes ŌĆö `buyerAssetsToUnits` reverts.
- `take(units, lender, sellOffer)` succeeds and returns `buyerAssets > units` ŌĆö `Midnight.take` does not revert.

        // forgefmt: disable-start

            timeToMaturity < 1 days   ? (  0 days,   1 days, _marketState.settlementFeeCbp0 * CBP, _marketState.settlementFeeCbp1 * CBP) :
