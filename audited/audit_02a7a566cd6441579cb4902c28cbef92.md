### Title
Reentrancy in `repay` callback allows authorized operator to drain borrower collateral via debt-before-payment ordering - (`src/Midnight.sol`)

### Summary
`repay` reduces `position[id][onBehalf].debt` before invoking the `IRepayCallback.onRepay` external call and before pulling loan tokens via `safeTransferFrom`. An operator X authorized through `EcrecoverAuthorizer.setIsAuthorized` can supply itself as the callback, re-enter `withdrawCollateral` on behalf of A during `onRepay`, pass the health check (debt is already zero), drain all of A's collateral, and use the proceeds to fund the repayment — all atomically. No reentrancy guard exists on either `repay` or `withdrawCollateral`.

### Finding Description

**Code path — `repay` (src/Midnight.sol:502-521):**

```
502: function repay(Market memory market, uint256 units, address onBehalf, address callback, bytes calldata data) external {
505:     require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
508:     position[id][onBehalf].debt -= UtilsLib.toUint128(units);   // ← debt zeroed HERE
509:     marketState[id].withdrawable += UtilsLib.toUint128(units);
514:     if (callback != address(0)) {
516:         IRepayCallback(callback).onRepay(...)                    // ← external call BEFORE payment
519:     }
520:     SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units); // ← payment LAST
```

**Code path — `withdrawCollateral` (src/Midnight.sol:549-573):**

```
556:     require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
568:     require(isHealthy(market, id, onBehalf), UnhealthyBorrower());  // ← passes because debt=0
572:     SafeTransferLib.safeTransfer(collateralToken, receiver, assets);
```

**Root cause:** `repay` writes `debt = 0` before the callback and before pulling loan tokens. During the callback window, the position appears fully repaid, so `isHealthy` returns `true` for any collateral withdrawal amount.

**Exploit flow:**

1. A signs `Authorization{authorizer=A, authorized=X, isAuthorized=true, nonce=N, deadline=T}` and X (or anyone) submits it to `EcrecoverAuthorizer.setIsAuthorized` → `isAuthorized[A][X] = true`. [1](#0-0) 

2. X (a malicious contract) calls `midnight.repay(market, D, A, X, data)` where X is both `msg.sender` and `callback`.
   - Auth check at line 505: `isAuthorized[A][X] == true` → passes.
   - Line 508: `position[id][A].debt -= D` → debt is now 0. [2](#0-1) 

3. Line 516: `X.onRepay(id, market, D, A, data)` is called. Inside `onRepay`, X re-enters:
   - `midnight.withdrawCollateral(market, 0, C_total, A, X)` — auth check passes (X is authorized), health check passes (debt=0), all of A's collateral is transferred to X. [3](#0-2) 

4. Still inside `onRepay`, X swaps the received collateral C for D loan tokens via an external DEX, then returns `CALLBACK_SUCCESS`.

5. Line 520: `safeTransferFrom(loanToken, X, midnight, D)` — X pays D loan tokens (funded by the swap). Transaction succeeds.

**Net result:** A's debt is 0 and A's collateral is 0. X extracted `value(C) − D` profit. A's overcollateralization is fully drained.

**Why existing checks fail:**
- The authorization check in `withdrawCollateral` passes because X is legitimately authorized.
- The health check `isHealthy` passes because debt was already zeroed at line 508.
- The `LIQUIDATION_LOCK_SLOT` transient lock only protects the seller during `take` callbacks; it is not set during `repay`. [4](#0-3) 
- The Certora formal verification explicitly assumes no reentrancy through callbacks and does not cover this path. [5](#0-4) 

### Impact Explanation
An operator authorized via `EcrecoverAuthorizer` can drain 100% of a borrower's collateral in a single atomic transaction, even when the position is healthy and the collateral is needed to back the debt. The invariant "collateral cannot be withdrawn outside health/liquidation rules" is violated because the health check is evaluated against a transiently zeroed debt that has not yet been backed by received loan tokens. [6](#0-5) 

### Likelihood Explanation
**Preconditions:**
1. A must have signed an `Authorization` for X (e.g., to allow X to repay on their behalf, or via a phishing/malicious UI).
2. A must have a healthy, overcollateralized position (collateral value > debt).
3. The collateral token must be swappable for loan tokens within a single transaction (true for any token with DEX liquidity).

All three are routine conditions. The attack is repeatable for any authorized operator and any market. The EcrecoverAuthorizer signature submission is permissionless (anyone can relay a signed authorization), so the attacker does not need to be the original submitter. [1](#0-0) [7](#0-6) 

### Recommendation
Apply a reentrancy guard to `repay` (and consistently to `withdrawCollateral`, `withdraw`, and `flashLoan`) using a transient storage lock analogous to `LIQUIDATION_LOCK_SLOT`, or restructure `repay` to follow checks-effects-interactions: pull loan tokens **before** reducing debt and before invoking the callback. The callback's purpose (letting the payer source funds) is incompatible with reducing debt first; the correct ordering is:

```
1. pull loan tokens from payer
2. reduce debt / increase withdrawable
3. invoke callback (optional notification, not funding)
``` [8](#0-7)

    function withdrawCollateral(
        Market memory market,
        uint256 collateralIndex,
        uint256 assets,
        address onBehalf,
        address receiver
    ) external {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);
        address collateralToken = market.collateralParams[collateralIndex].token;

        Position storage _position = position[id][onBehalf];
        uint256 newCollateral = _position.collateral[collateralIndex] - assets;
        _position.collateral[collateralIndex] = UtilsLib.toUint128(newCollateral);

        if (newCollateral == 0 && assets > 0) {
            _position.collateralBitmap = _position.collateralBitmap.clearBit(collateralIndex);
        }

        require(isHealthy(market, id, onBehalf), UnhealthyBorrower());

        emit EventsLib.WithdrawCollateral(msg.sender, id, collateralToken, assets, onBehalf, receiver);

        SafeTransferLib.safeTransfer(collateralToken, receiver, assets);
    }
