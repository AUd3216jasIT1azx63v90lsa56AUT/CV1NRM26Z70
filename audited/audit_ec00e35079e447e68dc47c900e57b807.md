### Title
Flash Loan Callback Enables Zero-Capital Spot-Oracle Manipulation to Drain Authorized Borrower Collateral - (File: src/Midnight.sol)

### Summary

`flashLoan` (lines 737–752) has no reentrancy guard and performs no health check before or after its callback. An attacker who holds a borrower's authorization can call `withdrawCollateral` inside `onFlashLoan`, where `isHealthy` reads a transiently inflated spot-oracle price caused by the flash-loaned tokens. After the flash loan is repaid and the oracle price reverts, the borrower is left undercollateralized with bad debt.

### Finding Description

**Code path and root cause.**

`flashLoan` transfers tokens out, fires `onFlashLoan`, then pulls tokens back — with no reentrancy lock and no health assertion at any point:

```
flashLoan (Midnight.sol:737-752)
  → safeTransfer tokens to callback          // Midnight balance drops
  → IFlashLoanCallback(callback).onFlashLoan // arbitrary re-entry allowed
  → safeTransferFrom tokens back             // Midnight balance restored
```

Inside `onFlashLoan`, the attacker calls `withdrawCollateral` (lines 549–573). The only health gate is:

```solidity
require(isHealthy(market, id, onBehalf), UnhealthyBorrower()); // line 568
```

`isHealthy` (lines 944–960) makes a live external call to the oracle:

```solidity
uint256 price = IOracle(collateralParam.oracle).price(); // line 953
```

`IOracle` is a one-function interface with no manipulation-resistance requirement. If the oracle is a spot-price oracle (e.g., Uniswap V2 reserve ratio), its return value is a function of the pool's token balance at call time.

**Attacker inputs and exploit flow.**

Preconditions: `isAuthorized[borrower][attacker] == true`; market oracle is a spot-price oracle; borrower is at or near the LLTV boundary.

1. Attacker calls `flashLoan([loanToken], [largeAmount], attackerContract, data)`.
2. Midnight sends `largeAmount` loan tokens to `attackerContract` (line 743).
3. `onFlashLoan` executes:
   - Attacker swaps flash-loaned loan tokens for collateral tokens in the AMM → collateral spot price inflates.
   - Attacker calls `midnight.withdrawCollateral(market, collateralIndex, assets, borrower, attacker)`:
     - Authorization check (line 556) passes — attacker is authorized.
     - Collateral storage is decremented (line 562).
     - `isHealthy` (line 568) calls oracle → reads inflated price → returns `true`.
     - Collateral tokens transferred to attacker (line 572).
   - Attacker swaps withdrawn collateral back to loan tokens in the AMM → oracle price deflates.
   - Attacker approves Midnight for flash loan repayment.
4. Midnight pulls loan tokens back (line 750) — repayment succeeds.
5. Post-transaction: borrower's collateral is reduced, debt is unchanged, oracle price is back to normal → `isHealthy` now returns `false` → borrower is liquidatable.

**Why existing checks fail.**

- Authorization check (line 556): passes by precondition.
- `isHealthy` check (line 568): reads a live, manipulable oracle price; no snapshot, no TWAP enforcement, no check that price is the same before and after the callback.
- TOKEN SAFETY REQUIREMENTS (lines 133–140): document only token behavior; no oracle manipulation-resistance requirement exists anywhere in the protocol.
- Certora `Healthiness.spec` (line 16) explicitly assumes `price()` is constant per callee (`PER_CALLEE_CONSTANT`) — this is a modeling assumption that does not hold for spot-price oracles and is not enforced on-chain.
- No reentrancy guard exists on `flashLoan` or `withdrawCollateral`.

### Impact Explanation

An authorized attacker can withdraw collateral from a borrower's position while the health check is satisfied only by a transiently inflated oracle price. After the flash loan unwinds, the borrower's `debt > maxDebt`, making the position immediately liquidatable. If the withdrawn collateral exceeds the bad-debt threshold, lenders absorb a loss through the loss-factor mechanism. The attacker profits by the withdrawn collateral minus AMM slippage, with zero net capital required.

### Likelihood Explanation

Preconditions are realistic and repeatable:
- Spot-price oracles (Uniswap V2/V3 instantaneous price) are a common oracle choice; the protocol imposes no oracle requirements.
- Borrower authorization is a standard DeFi pattern (e.g., a borrower authorizes a position-management contract that is later compromised or is itself malicious).
- The flash loan is permissionless and requires no upfront capital.
- The attack is atomic and repeatable in any block.

### Recommendation

Add a transient-storage reentrancy lock (using `tload`/`tstore`, already available per line 176) that is set for the duration of `flashLoan` and checked at the entry of `withdrawCollateral` (and any other health-sensitive function). Alternatively, snapshot the oracle price before the flash loan callback and assert it is unchanged after. At minimum, document a mandatory oracle requirement (e.g., TWAP with a minimum observation window) and enforce it at market creation in `touchMarket`.

contract FlashOracleManipulationTest is BaseTest, IFlashLoanCallback {

        super.setUp();

        setupMarket(market, UNITS);

        // borrower authorizes attacker (this contract)
        vm.prank(borrower);
        midnight.setIsAuthorized(address(this), true, borrower);
    }

    function testFlashOracleManipulation() public {
        uint256 collateralBefore = midnight.collateral(id, borrower, 0);

        // fund AMM and Midnight with loan tokens for flash loan
        deal(address(loanToken), address(midnight), LARGE_AMOUNT);
        deal(address(loanToken), address(amm),      LARGE_AMOUNT);

        address[] memory tokens  = new address[](1);
        uint256[] memory amounts = new uint256[](1);
        tokens[0]  = address(loanToken);
        amounts[0] = LARGE_AMOUNT;

        midnight.flashLoan(tokens, amounts, address(this), "");

        // ASSERTION 1: collateral was withdrawn
        assertLt(midnight.collateral(id, borrower, 0), collateralBefore);

        // ASSERTION 2: borrower is now unhealthy at true oracle price
        assertFalse(midnight.isHealthy(market, id, borrower));

        // ASSERTION 3: position is liquidatable
        // (liquidate should succeed without reverting NotLiquidatable)
    }

    function onFlashLoan(address, address[] memory tokens, uint256[] memory amounts, bytes memory)
        external returns (bytes32)
    {
        // Step 1: inflate spot oracle by buying collateral in AMM
        loanToken.approve(address(amm), amounts[0]);
        amm.buyCollateral(amounts[0]);  // price now inflated

        // Step 2: withdraw collateral — isHealthy reads inflated price
        uint256 collatBal = midnight.collateral(id, borrower, 0);
        midnight.withdrawCollateral(market, 0, collatBal / 2, borrower, address(this));

        // Step 3: sell collateral back to restore loan tokens for repayment
        collateralToken1.approve(address(amm), collatBal / 2);
        amm.sellCollateral(collatBal / 2);

        // Step 4: approve repayment
        loanToken.approve(address(midnight), amounts[0]);
        return CALLBACK_SUCCESS;
    }
}

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
