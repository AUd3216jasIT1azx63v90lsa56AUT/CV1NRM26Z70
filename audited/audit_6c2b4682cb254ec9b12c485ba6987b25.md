### Title
Flash Loan Oracle Manipulation Enables Liquidation of Healthy Borrowers - (File: src/Midnight.sol)

### Summary
The `flashLoan` function in `Midnight.sol` sets no reentrancy guard and does not activate the `LIQUIDATION_LOCK_SLOT` for any borrower during its callback. An attacker can use Midnight's own flash loan to borrow collateral tokens, depress a spot-price oracle within the `onFlashLoan` callback, and immediately call `liquidate` against a healthy borrower whose position appears unhealthy at the manipulated price. The formal verification specs explicitly assume oracle prices do not change during a transaction, confirming this attack surface is unguarded at the protocol level.

### Finding Description

**Code path — `flashLoan`:** [1](#0-0) 

The function transfers tokens to the callback, invokes `onFlashLoan`, then pulls tokens back. There is no reentrancy guard, no transient lock set, and no restriction on what the callback may call. Contrast this with `take`, which explicitly sets `LIQUIDATION_LOCK_SLOT` for the seller before its callbacks: [2](#0-1) 

`flashLoan` has no equivalent protection.

**Code path — `liquidate` health check:** [3](#0-2) 

`liquidate` calls `IOracle(_collateralParam.oracle).price()` live for every activated collateral. If the oracle reads from a manipulable spot source, the returned price is whatever the pool state is at that moment. `maxDebt` is computed from this price; if the price is depressed, `maxDebt` falls below `originalDebt` and the `NotLiquidatable()` guard passes for a borrower who is healthy at the true price.

**`isHealthy` has the same oracle dependency:** [4](#0-3) 

**Formal verification explicitly acknowledges the gap:** [5](#0-4) [6](#0-5) 

Both specs assume `price()` is constant per callee during a transaction (`PER_CALLEE_CONSTANT` / `summaryPrice`). The proofs that healthy borrowers cannot be liquidated are only valid under this assumption. The assumption is not enforced by the protocol.

**Exploit flow:**

1. Attacker calls `midnight.flashLoan([collateralToken], [largeAmount], attackerContract, data)`.
2. Midnight holds collateral deposited by borrowers; it transfers `largeAmount` of `collateralToken` to `attackerContract`.
3. Inside `attackerContract.onFlashLoan`:
   - Attacker dumps `largeAmount` of `collateralToken` into the Uniswap pool that the oracle reads from, depressing the spot price.
   - Attacker calls `midnight.liquidate(market, collateralIndex, seizedAssets, 0, victim, false, attacker, address(0), "")`.
   - `liquidate` reads the depressed price; `maxDebt` < `originalDebt`; `NotLiquidatable` does not revert.
   - Attacker seizes victim's collateral at a discount.
   - Attacker reverses the swap (optional; needed only to repay the flash loan in collateral token).
4. Attacker returns `largeAmount` to Midnight; flash loan repayment succeeds.
5. Transaction completes; victim's collateral is seized despite being healthy at the true price.

**Why existing checks fail:**

- `liquidationLocked` is `false` for the victim (it is only set during `take` callbacks for the seller).
- `postMaturityMode` is `false`; the check `originalDebt > maxDebt` passes because `maxDebt` is computed from the manipulated price.
- No TWAP requirement, no oracle validation, and no manipulation-resistance check exists anywhere in the liquidation path.

### Impact Explanation
A healthy borrower's collateral is seized without cause. The attacker profits by acquiring collateral at the liquidation incentive discount (up to `maxLif`) while the borrower suffers an unauthorized loss. The core invariant — healthy positions are not liquidatable — is broken.

### Likelihood Explanation
Preconditions: (1) a market is created with a collateral oracle that reads from a Uniswap v2/v3 spot price or any other single-block-manipulable source; (2) Midnight holds enough of that collateral token to make the flash loan meaningful for oracle manipulation. Both conditions are realistic: market creation is permissionless, and Midnight accumulates collateral from all borrowers. The attack is atomic, repeatable, and requires no privileged access.

### Recommendation
1. **Require TWAP oracles**: Document and enforce (via oracle interface or market creation checks) that oracles must be manipulation-resistant (e.g., minimum TWAP window). This is the root fix.
2. **Set a flash-loan reentrancy lock**: In `flashLoan`, set a transient flag (analogous to `LIQUIDATION_LOCK_SLOT`) that causes `liquidate` to revert if called during a Midnight flash loan callback. This prevents Midnight's own flash loan from being the manipulation vehicle, though it does not prevent external flash loan providers from being used.
3. **Alternatively, snapshot oracle prices**: Cache the oracle price at the start of each top-level call and reuse it, preventing intra-transaction manipulation.

```solidity
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {IFlashLoanCallback} from "src/interfaces/ICallbacks.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";

// Oracle whose price can be set by anyone (simulates a manipulable spot oracle)
contract ManipulableOracle {
    uint256 public price;
    function setPrice(uint256 p) external { price = p; }
}

contract AttackCallback is IFlashLoanCallback {
    Midnight midnight;
    Market market;
    address victim;
    ManipulableOracle oracle;
    uint256 depressedPrice;
    uint256 seizeAmount;

    constructor(Midnight _m, Market memory _mkt, address _victim,
                ManipulableOracle _oracle, uint256 _dp, uint256 _seize) {
        midnight = _m; market = _mkt; victim = _victim;
        oracle = _oracle; depressedPrice = _dp; seizeAmount = _seize;
    }

    function onFlashLoan(address, address[] calldata, uint256[] calldata, bytes calldata)
        external returns (bytes32)
    {
        // Step 1: depress oracle price
        oracle.setPrice(depressedPrice);
        // Step 2: liquidate healthy victim at manipulated price
        midnight.liquidate(market, 0, seizeAmount, 0, victim, false, address(this), address(0), "");
        // Step 3: restore price (optional)
        oracle.setPrice(1e36); // restore
        return CALLBACK_SUCCESS;
    }
}

contract OracleManipulationTest is Test {
    Midnight midnight;
    ManipulableOracle oracle;

    function testFlashLoanOracleManipulation() public {
        // Setup: deploy Midnight, create market with ManipulableOracle,
        // have victim borrow with collateral such that at true price they are healthy.
        // ...
        // Assert: before attack, isHealthy(market, id, victim) == true
        // Execute: attacker calls midnight.flashLoan([collateralToken], [largeAmount], attackCallback, "")
        // Assert: after attack, victim's collateral[0] < pre-attack value (collateral was seized)
        // Assert: victim's debt decreased (liquidation occurred)
        // Assert: at true oracle price, victim was healthy before attack (invariant violated)
    }
}
```

**Expected assertions:**
- `isHealthy(market, id, victim)` returns `true` before the flash loan call (at true price).
- After the flash loan call completes, `collateral(id, victim, 0)` is reduced by `seizeAmount`.
- The transaction does not revert, proving the liquidation succeeded despite the borrower being healthy at the true price.

```text
    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];

            }
        }
        return maxDebt >= debt;
```
