Audit Report

## Title
Authorized Agent Can Grief Victim's Nonce via Self-Deauthorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that `isAuthorized[victim][attacker]` is `true` for to sign and submit an `Authorization` struct on behalf of the victim, including one that self-deauthorizes the attacker. This consumes `nonce[victim]` from N to N+1, permanently invalidating any pending off-chain signed authorization the victim holds at nonce N, with no mechanism to prevent it.

## Finding Description

**Root cause:** `EcrecoverAuthorizer.setIsAuthorized` allows any currently-authorized agent to sign and submit an `Authorization` struct on behalf of the authorizer — including one naming themselves as `authorized` with `isAuthorized=false`. No check restricts a delegated signer from self-deauthorizing via this path.

**Exact code path:**

`EcrecoverAuthorizer.sol` line 26 — nonce is incremented as part of the `require` evaluation. If the nonce check passes (i.e., `authorization.nonce == nonce[authorization.authorizer]`), the post-increment persists: [1](#0-0) 

`EcrecoverAuthorizer.sol` lines 31–36 — the signer is recovered and checked. The attacker signs with their own key (`signer = attacker`). The first branch `signer == authorization.authorizer` → `attacker == victim` → false. The second branch `isAuthorized(victim, attacker)` → true by precondition. The check passes: [2](#0-1) 

`EcrecoverAuthorizer.sol` lines 46–47 — the downstream call executes `midnight.setIsAuthorized(attacker, false, victim)`: [3](#0-2) 

`Midnight.sol` line 732 — `Midnight.setIsAuthorized` checks `onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender]`. Here `msg.sender` is `EcrecoverAuthorizer` and `onBehalf` is `victim`. Since `isAuthorized[victim][EcrecoverAuthorizer]` must be true for the gasless flow to function at all, this check passes: [4](#0-3) 

**Net state change after the attack:**
- `nonce[victim]` = N+1 (was N)
- `isAuthorized[victim][attacker]` = false (was true)
- Any off-chain signed `Authorization(authorizer=victim, ..., nonce=N)` is permanently invalid (`InvalidNonce()` on submission)

**Why existing checks fail:** The `Unauthorized()` guard at lines 33–36 is the only gate, and it explicitly allows authorized agents through by design. There is no restriction preventing a delegated signer from naming themselves as `authorized` with `isAuthorized=false`. The nonce increment at line 26 occurs before signature verification in execution order, so the nonce is consumed as part of the same atomic check-and-increment. [5](#0-4) 

## Impact Explanation
Any pending signed authorization the victim has distributed off-chain (e.g., to a relayer or bundler) with `nonce=N` is rendered permanently invalid. The victim must re-sign with `nonce=N+1` and redistribute. This constitutes concrete service unavailability for the gasless authorization flow. While the victim can recover by re-signing, the attacker can repeat the pattern each time they are re-authorized, enabling sustained DoS on any gasless authorization flow relying on `EcrecoverAuthorizer`. The impact is not asset theft but targeted disruption of the off-chain signing infrastructure. [6](#0-5) 

## Likelihood Explanation
**Preconditions:** Attacker must hold `isAuthorized[victim][attacker] == true` and `isAuthorized[victim][EcrecoverAuthorizer] == true`. Both are normal operational states — users authorize relayers, bundlers, and smart contracts routinely, and `EcrecoverAuthorizer` must be authorized for the gasless flow to work at all. **Feasibility:** One transaction, no capital required, no oracle dependency. **Repeatability:** One-shot per authorization grant (attacker loses their own authorization), but if the victim re-authorizes the attacker (e.g., a trusted relayer), the attack can be repeated. The attacker can also be any of the many addresses a user might authorize (ratifiers, bundlers, callback contracts). [7](#0-6) 

## Recommendation
Add a check in `EcrecoverAuthorizer.setIsAuthorized` that prevents a delegated signer (i.e., `signer != authorization.authorizer`) from submitting an authorization that targets themselves as `authorized` with `isAuthorized=false`. Specifically, after the `Unauthorized()` guard, add:

```solidity
require(
    signer == authorization.authorizer ||
    !(authorization.authorized == signer && !authorization.isAuthorized),
    Unauthorized()
);
```

Alternatively, restrict delegated signers to only authorizing new agents (not deauthorizing existing ones, including themselves), or require that the `authorized` field cannot equal the `signer` when `isAuthorized=false` and `signer != authorization.authorizer`. A stricter fix would require the `authorizer` themselves to sign any deauthorization. [8](#0-7) 

## Proof of Concept
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. Victim calls `Midnight.setIsAuthorized(EcrecoverAuthorizer, true, victim)` — enables gasless flow.
3. Victim calls `Midnight.setIsAuthorized(attacker, true, victim)` — grants attacker authorization.
4. Victim signs off-chain `Authorization(authorizer=victim, authorized=someRelayer, isAuthorized=true, nonce=0, deadline=...)` and distributes to a bundler.
5. Attacker crafts `Authorization(authorizer=victim, authorized=attacker, isAuthorized=false, nonce=0, deadline=...)`, signs with attacker's key, and calls `EcrecoverAuthorizer.setIsAuthorized(authorization, signature)`.
6. Transaction succeeds: `nonce[victim]` = 1, `isAuthorized[victim][attacker]` = false.
7. Bundler attempts to submit victim's signed authorization from step 4 → reverts with `InvalidNonce()`.
8. Victim must re-sign with `nonce=1` and redistribute.

Expected test assertion: after step 6, `ecrecoverAuthorizer.nonce(victim) == 1` and any submission of the step-4 authorization reverts. [9](#0-8)

```text
    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
```
