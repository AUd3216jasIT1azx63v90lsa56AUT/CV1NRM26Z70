### Title
ERC721 Token Accepted as Collateral via `supplyCollateral` but Irrecoverable — Unliquidatable Positions Enable Theft of Lender Funds - (File: src/Midnight.sol)

---

### Summary

`supplyCollateral` uses `SafeTransferLib.safeTransferFrom`, which calls `transferFrom(from, to, tokenId)` — a function signature shared by both ERC20 and ERC721. An attacker can supply an ERC721 token as collateral, borrow real ERC20 loan tokens against it, and then default with impunity: `withdrawCollateral` and `liquidate` both call `SafeTransferLib.safeTransfer`, which invokes `transfer(to, amount)` — a function that does not exist in ERC721 — causing all collateral recovery paths to revert permanently.

---

### Finding Description

**Root cause:** `touchMarket` performs no validation that collateral tokens are ERC20-compliant. It only checks sorting, LLTV tier, and maxLif. [1](#0-0) 

**Deposit path works with ERC721:** `supplyCollateral` calls `SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets)`. [2](#0-1) 

Inside `SafeTransferLib`, this encodes a call to `IERC20.transferFrom(from, to, value)`: [3](#0-2) 

ERC721 defines `transferFrom(address from, address to, uint256 tokenId)` — identical ABI selector. The call succeeds, the NFT is transferred to `Midnight`, and `_position.collateral[collateralIndex]` is set to the tokenId value.

**Recovery paths fail with ERC721:** Both `withdrawCollateral` and `liquidate` call `SafeTransferLib.safeTransfer(collateralToken, receiver, assets)`: [4](#0-3) [5](#0-4) 

`safeTransfer` encodes a call to `IERC20.transfer(to, value)`: [6](#0-5) 

ERC721 has no `transfer(address, uint256)` function. The call returns `success = false` with empty returndata, causing the assembly block to `revert`. Every attempt to withdraw or seize the collateral reverts permanently.

**Full exploit flow:**
1. Attacker deploys or selects an existing ERC721 token.
2. Attacker calls `touchMarket` with a legitimate ERC20 loan token (e.g., USDC) and the ERC721 as `collateralParams[0].token`.
3. Attacker calls `supplyCollateral` with `assets = tokenId` — succeeds, NFT is held by `Midnight`, position records `collateral[0] = tokenId`.
4. Attacker creates or takes a sell offer in the market, borrowing real USDC against the "collateral".
5. Attacker defaults. Liquidators call `liquidate` → `safeTransfer(erc721, receiver, tokenId)` → reverts. The position is permanently unliquidatable.
6. Bad debt is socialized across all lenders in the market via `lossFactor`, slashing their credit.

---

### Impact Explanation

Lenders who provided USDC (or any ERC20 loan token) into the market suffer permanent, unrecoverable loss. The bad-debt socialization mechanism in `liquidate` will slash `lossFactor` for all lenders in the market, but the collateral can never be seized to offset it. This constitutes direct theft of lender funds with no remediation path, as the contract is immutable. [7](#0-6) 

---

### Likelihood Explanation

Market creation is fully permissionless — any address can call `touchMarket` with arbitrary token addresses. No whitelist, no token-type check, and no admin approval is required. The attacker needs only: (a) ownership of an ERC721 token, (b) a signed or on-chain offer to borrow against it. Both are trivially achievable by an unprivileged external attacker. [8](#0-7) 

---

### Recommendation

Add an ERC20 interface check in `touchMarket` (or in `supplyCollateral`) to reject tokens that do not implement `totalSupply()` or another ERC20-exclusive selector absent from ERC721. Alternatively, maintain a curated token whitelist for collateral assets, similar to the fix recommended in the original OpenQ report (split whitelist by token standard). At minimum, document that ERC721 tokens must never be used as collateral and enforce this with a runtime guard.

    {

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;
            uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];
            _marketState.settlementFeeCbp3 = _defaultSettlementFeeCbp[3];

            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);

        return id;

    function safeTransfer(address token, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transfer, (to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferReturnedFalse());
    }

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
