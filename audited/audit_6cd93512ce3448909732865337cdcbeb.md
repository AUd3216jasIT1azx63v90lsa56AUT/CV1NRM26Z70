### Title
`buyerAssetsToUnits` Division-by-Zero on Zero-Price Sell Offer Causes Bundler DoS - (File: src/periphery/TakeAmountsLib.sol)

### Summary
When `offer.tick = 0` and `settlementFee = 0`, `TickLib.tickToPrice(0)` returns `0` due to rounding, making `buyerPrice = 0`. `buyerAssetsToUnits` then calls `mulDivDown(targetBuyerAssets, WAD, 0)`, which is `(targetBuyerAssets * WAD) / 0` and reverts with a division-by-zero panic. `Midnight.take` handles the same inputs without reverting (it computes `mulDivUp(units, 0, WAD) = 0`), so the bundler path is broken while the core protocol path is not.

### Finding Description

**`TickLib.tickToPrice(0)` returns 0:** [1](#0-0) 

With `tick = 0`, the exponent `LN_ONE_PLUS_DELTA * (2910 - 0) ≈ 14.52e18`, so `wExp(14.52e18) ≈ 2.03e24`. Then `1e36 / (1e18 + 2.03e24) ≈ 4.9e11`, which after `divHalfDownUnchecked(1e12) * 1e12` rounds to `0`. This is confirmed by the existing test: [2](#0-1) 

**Division-by-zero in `buyerAssetsToUnits`:** [3](#0-2) 

With `tick=0`, `settlementFee=0`, `offer.buy=false`:
- `offerPrice = 0`
- `sellerPrice = offerPrice = 0`
- `buyerPrice = 0 + 0 = 0`
- `require(0 <= WAD)` passes
- `return targetBuyerAssets.mulDivDown(WAD, 0)` → `(targetBuyerAssets * WAD) / 0` → **panic revert** [4](#0-3) 

**`Midnight.take` does NOT revert for the same inputs:** [5](#0-4) 

`units.mulDivUp(0, WAD)` = `(units * 0 + WAD - 1) / WAD = 0`, which succeeds.

**Bundler call site is not protected by try/catch:** [6](#0-5) 

The `TakeAmountsLib.buyerAssetsToUnits` call at line 209 is outside the `try/catch` block at line 215. The NoDivisionByZero Certora spec only covers `Midnight.sol`, not the periphery library, so this is not caught by formal verification. [7](#0-6) 

### Impact Explanation
Any lender calling `buyWithAssetsTargetAndWithdrawCollateral` with a zero-price sell offer (`tick=0`, `settlementFee=0`) and `targetBuyerAssets > 0` will have their transaction revert unconditionally. Since the revert unwinds the `pullToken` at line 197, funds are not permanently frozen, but the bundler path is completely unusable for zero-price offers. Lenders must fall back to calling `Midnight.take` directly with a `units` input, losing the asset-denominated targeting functionality of the bundler. [8](#0-7) 

### Likelihood Explanation
Preconditions are minimal and fully attacker-controllable: any unprivileged borrower can post a sell offer with `tick=0`. The settlement fee is a market-level parameter; when it is 0 (the default for many markets as shown in bundler tests that call `setMarketSettlementFee(id, i, 0)`), the bug triggers on every call to `buyWithAssetsTargetAndWithdrawCollateral` targeting that offer. The condition is stable and repeatable. [9](#0-8) 

### Recommendation
Guard against `buyerPrice == 0` in `buyerAssetsToUnits`. When `buyerPrice == 0`, any `targetBuyerAssets > 0` is unreachable (zero assets per unit), so the function should return `type(uint256).max` (or revert with a descriptive error) to signal that no finite unit count can satisfy the target. Alternatively, add an explicit `require(buyerPrice > 0)` before the `mulDivDown` call, consistent with how `Midnight.take` itself documents that zero-price offers transfer zero assets. [10](#0-9) 

### Proof of Concept

```solidity
// Foundry unit test
function testBuyerAssetsToUnitsDivByZeroOnZeroPrice() public {
    // Setup: sell offer with tick=0, settlementFee=0
    offer.tick = 0;
    offer.buy = false;
    for (uint256 i; i <= 6; i++) {
        midnight.setMarketSettlementFee(id, i, 0);
    }
    midnight.touchMarket(offer.market);

    // 1. buyerAssetsToUnits reverts with division-by-zero
    vm.expectRevert(stdError.divisionError);
    TakeAmountsLib.buyerAssetsToUnits(address(midnight), id, offer, 1);

    // 2. Midnight.take with units=1 does NOT revert
    offer.maker = borrower;
    offer.receiverIfMakerIsSeller = borrower;
    offer.maxUnits = 1;
    collateralize(offer.market, borrower, 1);
    (uint256 buyerAssets, uint256 sellerAssets) = take(1, lender, offer);
    assertEq(buyerAssets, 0);
    assertEq(sellerAssets, 0);

    // 3. Bundler reverts for the same offer
    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: offer, units: type(uint256).max, ratifierData: hex""});
    vm.prank(lender);
    vm.expectRevert(); // division by zero inside buyerAssetsToUnits
    midnightBundles.buyWithAssetsTargetAndWithdrawCollateral(
        1, 0, lender, _noPermit(), takes, new CollateralWithdrawal[](0), address(0), 0, address(0)
    );
}
```

Expected assertions: step 1 reverts, step 2 succeeds with `buyerAssets == 0`, step 3 reverts.
