### Title
Unconditional `safeTransfer(collateral, receiver, 0)` in `liquidate` blocks bad debt realization for tokens that revert on zero-value transfers - (`src/Midnight.sol` / `src/libraries/SafeTransferLib.sol`)

### Summary
In `Midnight.liquidate`, the call `SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets)` at line 696 is executed **unconditionally**, even when `seizedAssets == 0`. If the collateral token reverts on zero-value `transfer` calls, the entire liquidation transaction reverts, rolling back all bad-debt state changes (lossFactor, totalUnits, position.debt) that were computed earlier in the same function. The protocol's own NatSpec at line 577 explicitly documents that passing `seizedAssets=0, repaidUnits=0` is the intended path for pure bad-debt realization with no token movement, but the unconditional transfer call violates this guarantee.

### Finding Description

**Code path:**

`liquidate()` in `src/Midnight.sol`:

1. Lines 626–641: Bad debt is realized — `_position.debt` is decremented, `_marketState.lossFactor` is updated, `_marketState.totalUnits` is decremented, `_marketState.continuousFeeCredit` is adjusted. All state changes are committed to storage.

2. Lines 643–677: The collateral-seizure and debt-repayment block is guarded by `if (repaidUnits > 0 || seizedAssets > 0)`. When both are 0, this block is **skipped entirely** — correct.

3. Line 696: `SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets)` — called **unconditionally** with `seizedAssets == 0`.

`SafeTransferLib.safeTransfer` in `src/libraries/SafeTransferLib.sol` lines 12–22:
```solidity
(bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transfer, (to, value)));
if (!success) {
    assembly ("memory-safe") {
        revert(add(returndata, 0x20), mload(returndata))
    }
}
```
If the collateral token's `transfer(receiver, 0)` reverts (a known behavior of certain ERC20 tokens), `success == false` and the assembly block propagates the revert. Because Solidity reverts are atomic, all storage writes from steps 1–2 are rolled back.

**Attacker-controlled inputs:** `seizedAssets=0`, `repaidUnits=0`, `borrower` with `debt > 0` and `collateralValue < debt` (bad debt condition), collateral token that reverts on zero-value transfers.

**Why existing checks do not stop it:** The `if (repaidUnits > 0 || seizedAssets > 0)` guard at line 643 correctly skips collateral accounting, but there is no equivalent guard around the `safeTransfer` at line 696. The Certora formal verification in `certora/specs/LossFactor.spec` rule `liquidateLossFactorDoesNotRevert` (lines 89–106) models `SafeTransferLib.safeTransfer` as `NONDET` (non-reverting), so this revert path is not covered by the formal proofs. ### Impact Explanation

For any market whose collateral token reverts on zero-value `transfer` calls:
- Bad debt can never be socialized: `lossFactor` is never updated, so lenders are never slashed proportionally.
- `totalUnits` is never decremented to reflect the loss, breaking the invariant that every credit unit has matching debt or valid settled/loss state.
- The borrower's `position.debt` is never reduced by the bad-debt amount, leaving the position permanently in an inconsistent state.
- The market becomes permanently insolvent: no liquidator can call `liquidate(0, 0)` to realize bad debt, and the unhealthy position remains unliquidatable via the pure bad-debt path. [5](#0-4) [6](#0-5) 

### Likelihood Explanation

**Preconditions:**
1. A market is created with a collateral token that reverts on zero-value `transfer` calls. This is a known behavior of certain ERC20 tokens (e.g., tokens with strict input validation). The market creator role is permissionless.
2. A borrower accumulates bad debt (collateral value falls below debt value), which is a normal market condition.

**Feasibility:** Any liquidator calling `liquidate(market, idx, 0, 0, borrower, false, receiver, address(0), "")` will trigger the revert. The condition is repeatable on every block as long as the token behavior and bad-debt state persist. No special privilege is required. [7](#0-6) 

### Recommendation

Guard the `safeTransfer` call at line 696 with a zero-amount check, consistent with the existing guard at line 643:

```solidity
// Before (line 696):
SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets);

// After:
if (seizedAssets > 0) {
    SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets);
}
```

Apply the same fix to `SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), repaidUnits)` at line 717 for symmetry, in case the loan token also reverts on zero transfers. [2](#0-1) [8](#0-7)

    function safeTransfer(address token, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transfer, (to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferReturnedFalse());

/// The loss factor arithmetic in liquidate does not revert under valid state. Uses seizedAssets=0, repaidUnits=0 to isolate the bad debt realization path. Uses collateralBitmap=0 to skip the collateral loop, ensuring badDebt == position.debt.
rule liquidateLossFactorDoesNotRevert(env e, Midnight.Market market, address borrower, bytes data) {
    bytes32 id = summaryToId(market);

    require data.length == 0, "no callback to avoid unrelated external call reverts";
    require marketIsCreated(market), "market must be created";
    require market.liquidatorGate == 0, "Assumption:no liquidator gate";
    require market.collateralParams.length > 0, "market has at least one collateral (enforced by touchMarket)";
    require !liquidationLocked(id, borrower), "liquidation not locked (transient storage is zero at transaction start)";
    require currentContract.position[id][borrower].collateralBitmap == 0, "Assumption: no active collaterals: skip loop and maximize badDebt";
    require currentContract.position[id][borrower].debt > 0, "borrower must have debt to enter badDebt > 0 block";
    require currentContract.position[id][borrower].debt <= currentContract.marketState[id].totalUnits, "position debt bounded by totalUnits (see totalUnitsEqualsSumNegativeDebtPlusWithdrawable)";
    require e.msg.value == 0, "Midnight is not payable";

    address zero = 0;
    liquidate@withrevert(e, market, 0, 0, 0, borrower, false, borrower, zero, data);

    assert !lastReverted, "liquidate should not revert under valid state (bad debt realization path)";
