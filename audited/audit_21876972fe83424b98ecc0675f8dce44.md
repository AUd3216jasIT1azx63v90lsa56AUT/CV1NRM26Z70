### Title
O(N) zero-unit `take()` calls with external ratifier/gate invocations when group is fully consumed — (`src/periphery/MidnightBundles.sol`)

### Summary
The loop in `supplyCollateralAndSellWithUnitsTarget` (and the three sibling bundle functions) has no guard to skip iterations where `ConsumableUnitsLib.consumableUnits()` returns 0. When a shared group is fully consumed, every offer in the `takes` array produces `unitsToTake = 0`, yet `try IMidnight.take(..., 0, ...)` is still called for each, performing full validation including external calls to ratifier and gate contracts. The loop condition `filledUnits < targetUnits` never advances when all takes are zero-unit, so all N iterations execute before the transaction reverts with `OutOfOffers`.

### Finding Description
**Code path:**

In `supplyCollateralAndSellWithUnitsTarget` (lines 144–160), for each `i`:
1. `unitsToTake = min(targetUnits - filledUnits, takes[i].units, ConsumableUnitsLib.consumableUnits(...))`.
2. If the group is fully consumed, `consumableUnits()` returns `offer.maxUnits.zeroFloorSub(consumed) = 0`, so `unitsToTake = 0`.
3. `try IMidnight.take(offer, ratifierData, 0, taker, ...)` is called — no skip.
4. `take(0)` validates expiry, start, tick accessibility, ratifier (external call), and gate (external call), then returns `(0, 0)`.
5. `filledUnits += 0` — loop condition `filledUnits < targetUnits` remains true.
6. All N iterations execute; transaction reverts with `OutOfOffers`.

**Attacker-controlled inputs:**

An unprivileged taker (or an authorized agent acting as `msg.sender`) constructs a `takes` array of length N where all offers share a single group. A front-running attacker (or any prior taker) exhausts that group by calling `IMidnight.take()` or `setConsumed()` on any offer in the group. Once `consumed[maker][group] >= maxUnits`, all N offers yield `consumableUnits() = 0`.

**Why existing checks fail:**

The `try/catch` is designed to absorb asynchrony-related reverts, but it also absorbs the zero-unit no-op success. There is no guard `if (unitsToTake == 0) continue;` before the external call. The loop condition does not account for zero-unit iterations.

### Impact Explanation
Each of the N zero-unit `take()` calls performs: signature/ratifier dispatch (external call if `offer.ratifier != 0`), gate check (external call if `offer.market.enterGate != 0`), expiry/start validation, and tick accessibility check. Total gas consumed is O(N × per-take-validation-cost). With ratifier contracts, each iteration costs thousands of additional gas. The transaction always reverts with `OutOfOffers` — no state is corrupted — but the caller loses gas proportional to N.

### Likelihood Explanation
**Preconditions:**
1. A maker creates N offers sharing one group with `maxUnits` set.
2. Any taker (including the attacker) exhausts the group before the victim's bundle transaction lands.

This is achievable via front-running in a competitive mempool. The attacker spends gas to exhaust the group (a legitimate take), then the victim's bundle transaction wastes O(N) gas. The same pattern applies to all four bundle functions (`buyWithUnitsTargetAndWithdrawCollateral`, `supplyCollateralAndSellWithUnitsTarget`, `buyWithAssetsTargetAndWithdrawCollateral`, `supplyCollateralAndSellWithAssetsTarget`).

### Recommendation
Add an early-continue guard before the `try` call in all four bundle loops:

```solidity
if (unitsToTake == 0) continue;
```

This skips the external `take()` call entirely when the offer is fully consumed, reducing wasted gas from O(N × take_cost) to O(N × consumableUnits_cost), which is a pure view call with no external dispatch.

        uint256 consumed = IMidnight(midnight).consumed(offer.maker, offer.group);
