### Title
Division-by-zero revert in `liquidate` when oracle returns 0 for the liquidated collateral (`repaidUnits > 0` path) - (File: src/Midnight.sol)

### Summary

When `IOracle.price()` returns 0 for the liquidated collateral, calling `liquidate` with `repaidUnits > 0` (and `seizedAssets = 0`) unconditionally reverts at `repaidUnits.mulDivDown(lif, WAD).mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice)` because `liquidatedCollatPrice = 0` is used as the denominator. The `seizedAssets > 0` path does **not** revert — it silently yields `repaidUnits = 0`, seizing collateral without reducing any debt. The protocol's own Certora formal verification explicitly acknowledges this gap: `NoDivisionByZero.spec` requires `ghostPrice(...) > 0` as a precondition and `certora/README.md` states division by zero is possible in `liquidate` when the oracle price is 0.

### Finding Description

**Root cause — `src/Midnight.sol` lines 649–653:**

```solidity
if (seizedAssets > 0) {
    repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
} else {
    seizedAssets = repaidUnits.mulDivDown(lif, WAD).mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice);
}
```

**`seizedAssets > 0` branch with `liquidatedCollatPrice = 0`:**
- `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE)` → `(seizedAssets*0 + ORACLE_PRICE_SCALE-1) / ORACLE_PRICE_SCALE` = 0
- `mulDivUp(0, WAD, lif)` → `(lif-1)/lif` = 0 (since `lif ≥ WAD = 1e18`)
- Result: `repaidUnits = 0`. **No revert.** Collateral is seized, debt is unchanged.

**`repaidUnits > 0` branch with `liquidatedCollatPrice = 0`:**
- `mulDivDown(repaidUnits, lif, WAD)` → positive value `X`
- `mulDivDown(X, ORACLE_PRICE_SCALE, 0)` → `(X * ORACLE_PRICE_SCALE) / 0` → **EVM division-by-zero revert**

`mulDivDown` is `return (x * y) / d` (`src/libraries/UtilsLib.sol:30`); Solidity 0.8 reverts on integer division by zero.

**Exploit flow:**
1. Market is created with collateral at index 0 whose oracle can return 0.
2. Borrower supplies collateral 0 and borrows, making the position healthy.
3. Oracle for collateral 0 drops to 0 (legitimate price collapse or oracle edge case). `maxDebt` contribution from collateral 0 becomes 0; if total `maxDebt < debt`, position is unhealthy.
4. Liquidator calls `liquidate(market, collateralIndex=0, seizedAssets=0, repaidUnits=X, borrower, ...)` with `X > 0`.
5. The loop sets `liquidatedCollatPrice = 0`. The `NotLiquidatable` check passes (position is unhealthy). The `else` branch executes `mulDivDown(ORACLE_PRICE_SCALE, 0)` → revert.

**Why existing checks do not stop it:**
- `require(UtilsLib.atMostOneNonZero(repaidUnits, seizedAssets))` passes (only `repaidUnits > 0`).
- `require(_position.debt > 0)` passes.
- `require(!liquidationLocked && originalDebt > maxDebt)` passes (position is unhealthy).
- There is **no guard** on `liquidatedCollatPrice == 0` before the division.

The `seizedAssets > 0` path does not revert but is equally broken: it seizes collateral while repaying 0 debt, leaving the position permanently unhealthy and the seized collateral effectively stolen from the borrower with no protocol benefit.

### Impact Explanation

Any liquidator calling `liquidate` with `repaidUnits > 0` against a collateral whose oracle returns 0 receives an unconditional revert. The unhealthy position cannot be reduced via the `repaidUnits`-input path for that collateral. The `seizedAssets > 0` path succeeds but repays 0 debt, so the position remains unhealthy after "liquidation." If the zero-price collateral is the sole collateral, bad-debt accounting absorbs the debt first, but the revert still blocks the `repaidUnits > 0` call path entirely. The scoped invariant — unhealthy positions must remain liquidatable — is violated for the `repaidUnits > 0` call variant.

### Likelihood Explanation

**Preconditions:**
- A collateral oracle returns 0 (price collapse, stale feed, or oracle misconfiguration — explicitly modeled as possible in the protocol's own Certora specs).
- The borrower has non-zero debt and the position is unhealthy.
- A liquidator calls with `repaidUnits > 0`.

All three are reachable without any privileged action. The oracle returning 0 is not treated as impossible by the protocol (the Certora `NoDivisionByZero.spec` explicitly carves it out). The condition is repeatable: every `repaidUnits > 0` call against that collateral reverts until the oracle recovers.

### Recommendation

Guard against a zero oracle price before entering the seize/repay branch. For example, immediately after `liquidatedCollatPrice` is assigned in the collateral loop, add:

```solidity
if (repaidUnits > 0 || seizedAssets > 0) {
    require(liquidatedCollatPrice > 0, ZeroOraclePrice());
    ...
}
```

Alternatively, skip the `repaidUnits`/`seizedAssets` computation entirely when `liquidatedCollatPrice == 0` (treating it as a bad-debt-only liquidation), consistent with how the bad-debt loop already handles zero-price collateral by contributing 0 to `badDebt` reduction.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
// ... standard test imports

contract OracleZeroPriceLiquidateTest is Test {
    // Setup: single-collateral market, oracle MockOracle
    // 1. MockOracle.setPrice(ORACLE_PRICE_SCALE)  -- healthy price
    // 2. borrower supplies collateral, borrows `units`
    // 3. MockOracle.setPrice(0)                   -- price collapses to 0
    //    => maxDebt = 0 < units => position unhealthy

    function testRepaidUnitsRevertsWhenOracleZero() public {
        // ... setup ...
        MockOracle(oracle).setPrice(0);

        // repaidUnits > 0 path: expect revert (division by zero)
        vm.expectRevert(); // stdError.divisionError or arithmeticError
        midnight.liquidate(market, 0, 0, 1, borrower, false, address(this), address(0), "");
    }

    function testSeizedAssetsYieldsZeroRepaidUnits() public {
        // ... setup ...
        MockOracle(oracle).setPrice(0);

        // seizedAssets > 0 path: does NOT revert, but repaidUnits == 0
        (uint256 seized, uint256 repaid) =
            midnight.liquidate(market, 0, 1, 0, borrower, false, address(this), address(0), "");

        assertEq(repaid, 0, "repaidUnits must be 0 when oracle price is 0");
        assertGt(seized, 0, "collateral is seized");
        // debt is unchanged: position remains unhealthy
        assertEq(midnight.debtOf(id, borrower), units, "debt not reduced");
    }
}
```

- `testRepaidUnitsRevertsWhenOracleZero`: call reverts with division-by-zero.
