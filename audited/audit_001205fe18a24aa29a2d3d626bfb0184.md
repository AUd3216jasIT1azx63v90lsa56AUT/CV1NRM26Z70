### Title
Coarse-grained `isAuthorized` allows any authorized operator to irreversibly cancel a maker's Merkle offer tree — (`File: src/ratifiers/EcrecoverRatifier.sol`)

### Summary
`EcrecoverRatifier.cancelRoot` gates access solely on the global `IMidnight.isAuthorized(maker, msg.sender)` mapping, which is the same flat authorization used for withdraw, repay, setConsumed, and every other Midnight operation. Because `isRootCanceled` can only be set to `true` and never reversed, any operator the maker has authorized for any purpose can permanently destroy all remaining fill capacity in a Merkle offer tree, with no recovery path.

### Finding Description
**Code path:**

`EcrecoverRatifier.cancelRoot` (lines 27–31): [1](#0-0) 

```solidity
function cancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
    isRootCanceled[maker][root] = true;
    emit CancelRoot(msg.sender, maker, root);
}
```

The only guard is `IMidnight.isAuthorized(maker, msg.sender)`. [2](#0-1) 

That mapping is set via `Midnight.setIsAuthorized`, which is a single flat boolean with no scope or purpose restriction. A maker who authorizes an operator for debt repayment, collateral management, or any other reason grants that operator the identical privilege to call `cancelRoot`.

**Irreversibility:** `isRootCanceled` is a `mapping(address => mapping(bytes32 => bool))` with no setter that can write `false`. [3](#0-2) 

Once set, `isRatified` always reverts with `RootCanceled` for every offer in that tree: [4](#0-3) 

**Exploit flow:**
1. Maker builds a large Merkle tree of offers and signs the root.
2. Maker authorizes `operator` via `midnight.setIsAuthorized(operator, true, maker)` for an unrelated purpose (e.g., repayment).
3. Taker partially fills some offers — valid state, partial consumption recorded.
4. Operator calls `ecrecoverRatifier.cancelRoot(maker, root)`.
5. `isRootCanceled[maker][root] = true` — irreversible.
6. All subsequent `take` calls for any offer in that tree revert with `RootCanceled`.
7. Maker cannot re-enable the root; must re-sign an entirely new tree and re-distribute it out-of-band.

**Why existing checks fail:** The authorization check is satisfied by design — the operator is legitimately authorized. There is no scope restriction, no separate permission for destructive/irreversible actions, and no `uncancelRoot` function.

The existing test `testCancelRootAuthorizedOnBehalf` confirms this path is reachable and succeeds: [5](#0-4) 

### Impact Explanation
Any operator authorized by the maker for any purpose can permanently freeze all remaining fill capacity in a Merkle offer tree. Partially-filled offers whose remaining capacity was economically valuable are permanently blocked. The maker cannot recover the same root — they must re-sign and redistribute a new tree, losing all in-flight partial fills and any taker-side state that referenced the old root.

### Likelihood Explanation
The precondition — a maker having authorized any operator for any reason — is a normal, expected usage pattern (e.g., authorizing a repayment bot, a portfolio manager, or a keeper). The attack requires only a single external call with no capital. It is repeatable against every root the maker has ever published. A malicious or compromised operator, or a griefing taker who tricks the maker into authorizing them, can execute this at any time.

### Recommendation
Separate the authorization for irreversible, destructive actions from the general-purpose `isAuthorized` flag. Concretely:

- Restrict `cancelRoot` to `maker == msg.sender` only, removing the `isAuthorized` delegation entirely; or
- Introduce a dedicated, scoped permission (e.g., `isCancelAuthorized`) distinct from the general `isAuthorized` mapping; or
- Mirror `SetterRatifier`'s reversible pattern by adding an `uncancelRoot` function so the maker can recover, reducing the severity of an unauthorized cancellation.

### Proof of Concept
```solidity
function testOperatorCancelsRootAfterPartialFill() public {
    // 1. Maker (lender) builds a tree with two offers
    Offer memory offer1 = makeOffer(lender);
    Offer memory offer2 = makeOffer(lender); offer2.expiry += 1;
    bytes32 h1 = HashLib.hashOffer(offer1);
    bytes32 h2 = HashLib.hashOffer(offer2);
    bytes32 root = HashLib.hashNode(h1, h2);

    // 2. Taker partially fills offer1 (omitted for brevity, assume success)

    // 3. Maker authorizes operator for an unrelated purpose
    address operator = makeAddr("operator");
    vm.prank(lender);
    midnight.setIsAuthorized(operator, true, lender);

    // 4. Operator cancels the root
    vm.prank(operator);
    ecrecoverRatifier.cancelRoot(lender, root);

    // 5. Assert root is permanently canceled
    assertTrue(ecrecoverRatifier.isRootCanceled(lender, root));

    // 6. Assert remaining fills revert
    bytes32[] memory proof = new bytes32[](1);
    proof[0] = h1;
    Signature memory sig = signature(root, privateKey[lender], address(ecrecoverRatifier), 1);
    bytes memory ratifierData = abi.encode(sig, root, 1, proof);
    vm.prank(address(midnight));
    vm.expectRevert(IEcrecoverRatifier.RootCanceled.selector);
    ecrecoverRatifier.isRatified(offer2, ratifierData);

    // 7. Assert no recovery path — no uncancelRoot exists
    // (compilation check: ecrecoverRatifier.uncancelRoot does not exist)
}
```

Expected assertions: `isRootCanceled(lender, root) == true`; `isRatified` reverts with `RootCanceled`; no function exists to reverse the cancellation.
