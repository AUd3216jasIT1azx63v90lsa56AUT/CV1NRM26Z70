### Title
Free Credit Units via `buyerAssets` Rounding to Zero in `take` — (`src/Midnight.sol`)

### Summary

When `offer.buy == true` and `units * buyerPrice < WAD`, the `mulDivDown` in `take` rounds `buyerAssets` to zero. The maker (buyer) receives credit units without transferring any loan tokens. A two-account attacker can exploit this to drain the `withdrawable` pool funded by legitimate repayments, causing bad debt for remaining lenders.

### Finding Description

In `src/Midnight.sol`, the `take` function computes payment amounts as:

```solidity
uint256 buyerAssets = offer.buy ? units.mulDivDown(buyerPrice, WAD) : units.mulDivUp(buyerPrice, WAD);
uint256 sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : units.mulDivUp(sellerPrice, WAD);
``` [1](#0-0) 

When `offer.buy == true`, `buyerAssets = (units * buyerPrice) / WAD` (floor). At the minimum representable price (`tickToPrice(0) = 1e12`, since `PRICE_ROUNDING_STEP = 1e12`), any `units < 1e6` yields `buyerAssets = 0`. [2](#0-1) [3](#0-2) 

The token transfers that follow:

```solidity
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
``` [4](#0-3) 

…both transfer `0` tokens. Yet the credit accounting still executes:

```solidity
buyerPos.credit += UtilsLib.toUint128(buyerCreditIncrease);
sellerPos.debt   += UtilsLib.toUint128(sellerDebtIncrease);
``` [5](#0-4) 

The buyer's credit increases by `units` for free. There is no `require(buyerAssets > 0)` guard anywhere in `take`. [6](#0-5) 

**End-to-end exploit path:**

1. **Attacker A** (maker/buyer) creates a buy offer at `tick = 0` (`price = 1e12`) using any permissionless ratifier (e.g., `EcrecoverRatifier` with their own key).
2. **Attacker B** (taker/seller, a different address to pass `require(offer.maker != taker, SelfTake())`) calls `take` with `units = 999_999`.
3. `buyerAssets = 999_999 * 1e12 / 1e18 = 0` → no loan tokens transferred.
4. Attacker A accumulates `999_999` credit units for free. Attacker B takes on `999_999` debt units, receiving `0` loan tokens.
5. Steps 1–4 are repeated until Attacker A holds a large credit balance.
6. Once legitimate users repay debt (increasing `withdrawable`), Attacker A calls `withdraw` to drain those loan tokens.
7. Attacker B never repays → bad debt is socialized among remaining lenders via `lossFactor` slashing.

### Impact Explanation

- **Direct theft:** Attacker A withdraws real loan tokens from `withdrawable` (funded by legitimate repayments) using credit obtained for free.
- **Undercollateralization:** Attacker B's unpaid debt becomes bad debt, slashing all remaining lenders' credit proportionally via `lossFactor`.
- **Severity: High** — direct loss of funds for honest lenders; no privileged access required.

### Likelihood Explanation

- Any two externally-owned accounts suffice; no privileged keys needed.
- Tick 0 is always accessible (`DEFAULT_TICK_SPACING = 4` divides 0).
- Any market with `withdrawable > 0` (i.e., any market where borrowers have repaid) is a target.
- The attack is repeatable in a loop; gas cost on L2s (Arbitrum, Base, Optimism) is negligible relative to the drained value.

### Recommendation

Add a zero-amount guard after computing `buyerAssets` and `sellerAssets`:

```solidity
require(units == 0 || buyerAssets > 0, ZeroBuyerAssets());
```

Or, equivalently, enforce a minimum `units` such that `units * minPrice >= WAD`. This mirrors the Bond Protocol fix of checking `quoteAmount != 0` after the rounding division.

### Proof of Concept

**Inputs:**
- Market: any loan token (e.g., USDC, 6 decimals), any collateral.
- `offer.buy = true`, `offer.tick = 0` → `buyerPrice = 1e12`.
- `units = 999_999`.

**Calculation:**
```
buyerAssets = mulDivDown(999_999, 1e12, 1e18)
            = (999_999 * 1e12) / 1e18
            = 999_999_000_000_000_000 / 1_000_000_000_000_000_000
            = 0  ✓ (rounds down)
```

**Result:** Attacker A receives `999_999` credit units; `0` loan tokens are transferred. Repeating 1,000 times yields `~1e9` free credit units. If `withdrawable >= 1e9`, Attacker A calls `withdraw` and receives `1e9` loan token atoms. Attacker B's `1e9` debt defaults, socializing the loss.
