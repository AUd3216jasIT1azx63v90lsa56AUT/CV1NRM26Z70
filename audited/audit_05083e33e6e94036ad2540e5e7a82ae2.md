### Title
Authorized operator can activate reverting-oracle collateral slot, permanently freezing victim's position - (File: src/Midnight.sol)

### Summary
`supplyCollateral` relies solely on an authorization check to prevent collateral poisoning, but does not validate that the newly activated collateral's oracle is callable. An authorized attacker can supply 1 wei of a collateral whose oracle always reverts, setting that bit in the victim's `collateralBitmap`. Every subsequent call to `isHealthy`, `liquidate`, and `withdrawCollateral` then reverts unconditionally while the victim carries any debt, permanently freezing their position.

### Finding Description

**Root cause — missing oracle liveness check on bitmap activation.**

`supplyCollateral` (lines 524–546) activates a new collateral slot when `oldCollateral == 0 && assets > 0`:

```
if (oldCollateral == 0 && assets > 0) {
    uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
    _position.collateralBitmap = newCollateralBitmap;
    require(countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER, ...);
}
``` [1](#0-0) 

No oracle call is made here. The only guard is the authorization check at line 527: [2](#0-1) 

The comment at line 523 states "This function checks authorization to prevent activated collateral poisoning," but authorization only blocks *strangers* — it does not block an *authorized* operator. [3](#0-2) 

**Downstream revert paths.**

`isHealthy` iterates every bit in `collateralBitmap` and calls `IOracle(...).price()` for each one whenever `debt > 0`: [4](#0-3) 

`liquidate` does the same unconditionally in its bitmap loop: [5](#0-4) 

`withdrawCollateral` calls `isHealthy` before transferring: [6](#0-5) 

The protocol's own LIVENESS section explicitly documents this consequence: [7](#0-6) [8](#0-7) 

**Exploit flow.**

| Step | Actor | Call |
|------|-------|------|
| 1 | Anyone | Create market with collateral A (working oracle) and collateral B (reverting oracle) |
| 2 | Victim | Supply collateral A, borrow (debt > 0, < 16 active collaterals) |
| 3 | Victim | `midnight.setIsAuthorized(attacker, true, victim)` (or via `EcrecoverAuthorizer.setIsAuthorized`) |
| 4 | Attacker | `midnight.supplyCollateral(market, B_index, 1, victim)` — succeeds; oracle never called; bit B set in victim's bitmap |
| 5 | Anyone | `liquidate(...)` → reverts; `withdrawCollateral(...)` → reverts |

`EcrecoverAuthorizer.setIsAuthorized` (lines 24–48) allows the victim to grant authorization via an off-chain signature, a common UX pattern, making step 3 realistic: [9](#0-8) 

**Why existing checks fail.**

- Authorization check (line 527): passes because attacker is authorized.
- `TooManyActivatedCollaterals` (lines 538–540): passes because victim has < 16 active collaterals.
- No oracle check exists anywhere in `supplyCollateral`.

### Impact Explanation
The victim's position becomes permanently unliquidatable — `liquidate` reverts on every call — and `withdrawCollateral` reverts for every collateral index while debt > 0. The victim's collateral is frozen. The only self-rescue path is repaying the entire debt balance, which may be impossible if the victim lacks funds. Because liquidation is also broken, no external party can reduce the debt on the victim's behalf.

### Likelihood Explanation
Preconditions are achievable without any privileged role:
- A market with a reverting-oracle collateral can be created permissionlessly (`touchMarket` validates LLTV and maxLif but never calls the oracle).
- The victim needs debt and < 16 active collaterals — the normal borrower state.
- The victim must have authorized the attacker, which is a standard UX flow (e.g., delegating position management to a smart contract or keeper).
- The attacker needs only 1 wei of the reverting-oracle collateral token and gas.

The attack is repeatable: even if the victim revokes authorization after the fact, the poisoned bitmap bit persists until the victim repays all debt.

### Recommendation
In `supplyCollateral`, when a new collateral slot is being activated (`oldCollateral == 0 && assets > 0`), call the collateral's oracle and require it to succeed before setting the bitmap bit:

```solidity
if (oldCollateral == 0 && assets > 0) {
    // Validate oracle liveness before activating the slot.
    IOracle(market.collateralParams[collateralIndex].oracle).price();
    uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
    _position.collateralBitmap = newCollateralBitmap;
    require(
        UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER,
        TooManyActivatedCollaterals()
    );
}
```

This ensures that any collateral slot that enters the bitmap has a live oracle at activation time, preserving the invariant that activated collateral oracles must not cause `isHealthy`/`liquidate` to revert.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";

contract RevertingOracle {
    function price() external pure returns (uint256) { revert("oracle down"); }
}

contract CollateralPoisoningTest is Test {
    Midnight midnight;
    address victim  = address(0xBEEF);
    address attacker = address(0xBAD);

    function setUp() public {
        midnight = new Midnight();
    }

    function test_authorizedOperatorPoisonsCollateralBitmap() public {
        // 1. Build market: collateral[0] = working oracle, collateral[1] = reverting oracle
        CollateralParams[] memory params = new CollateralParams[](2);
        params[0] = CollateralParams({ token: address(workingToken), oracle: address(workingOracle), lltv: ..., maxLif: ... });
        params[1] = CollateralParams({ token: address(poisonToken),  oracle: address(new RevertingOracle()), lltv: ..., maxLif: ... });
        Market memory market = Market({ ..., collateralParams: params });

        // 2. Victim supplies collateral[0] and borrows (debt > 0)
        vm.prank(victim);
        midnight.supplyCollateral(market, 0, 1e18, victim);
        // ... victim takes a sell offer to incur debt ...

        // 3. Victim authorizes attacker
        vm.prank(victim);
        midnight.setIsAuthorized(attacker, true, victim);

        // 4. Attacker activates the reverting-oracle slot with 1 wei
        deal(address(poisonToken), attacker, 1);
        vm.prank(attacker);
        poisonToken.approve(address(midnight), 1);
        vm.prank(attacker);
        midnight.supplyCollateral(market, 1, 1, victim); // succeeds — no oracle call

        // 5. Assert: liquidate reverts
        vm.expectRevert();
        midnight.liquidate(market, 0, 0, 1, victim, false, attacker, address(0), "");

        // 6. Assert: withdrawCollateral reverts
        vm.prank(victim);
        vm.expectRevert();
        midnight.withdrawCollateral(market, 0, 1e18, victim, victim);
    }
}
```

Expected assertions: both `liquidate` and `withdrawCollateral` revert with the reverting oracle's error while `victim.debt > 0`, confirming the position is permanently frozen.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
