### Title
Oracle Revert in Any Collateral Bitmap Entry Freezes `withdrawCollateral` for All Collaterals - (File: src/Midnight.sol)

### Summary
`isHealthy` iterates every bit in `collateralBitmap` and calls `IOracle(collateralParam.oracle).price()` for each activated collateral with no error handling. If any oracle in the bitmap reverts (e.g., staleness check, circuit breaker), the entire `isHealthy` call reverts. Because `withdrawCollateral` calls `isHealthy` unconditionally, a borrower with two or more activated collaterals cannot withdraw any collateral — including one whose oracle is perfectly healthy — as long as any other oracle in their bitmap is reverting.

### Finding Description
**Code path:**

`isHealthy` (Midnight.sol:944–960) enters the `while (_collateralBitmap != 0)` loop when `debt > 0` and calls `IOracle(collateralParam.oracle).price()` at line 953 for every set bit, with no `try/catch`: [1](#0-0) 

There is no error isolation around the oracle call. If `oracle[1].price()` reverts for any reason (staleness guard, circuit breaker, paused feed), the `revert` propagates up through `isHealthy` and through `withdrawCollateral`.

**Attacker inputs / exploit flow:**

1. Borrower has `debt > 0` and two activated collaterals: `collateral[0]` (oracle healthy) and `collateral[1]` (oracle has a staleness/circuit-breaker revert path). Both bits are set in `_position.collateralBitmap`.
2. Attacker (or simply time) causes `oracle[1]` to enter its revert state (e.g., TWAP window goes stale, price moves outside a band, feed is paused).
3. Borrower calls `withdrawCollateral(market, 0, assets, borrower, receiver)`.
4. `withdrawCollateral` calls `isHealthy(market, id, borrower)`.
5. `isHealthy` iterates the bitmap; when it reaches bit 1, it calls `oracle[1].price()` → revert.
6. The revert propagates: `withdrawCollateral` reverts.

**Why existing checks do not stop it:**

The only guard in `isHealthy` is `if (debt > 0)` before entering the loop. There is no per-oracle `try/catch`, no bitmap masking to skip the collateral being withdrawn, and no fallback price path. [2](#0-1) 

### Impact Explanation
A borrower with two or more activated collaterals and non-zero debt has their `withdrawCollateral` permanently frozen for all collaterals as long as any single oracle in their bitmap is in a reverting state. The borrower's funds are locked with no recourse through the protocol. This also affects `liquidate` if it calls `isHealthy` post-seizure, potentially blocking liquidation of genuinely unhealthy positions.

### Likelihood Explanation
Preconditions are realistic and common: (1) borrower uses two collateral types — a normal usage pattern; (2) one oracle has a staleness or out-of-range revert — standard for Chainlink, TWAP, or custom oracles; (3) the reverting condition can be triggered passively (waiting for staleness) or actively (large trade to move a TWAP out of band). No privileged access is required. The condition is repeatable and can be sustained indefinitely by the attacker.

### Recommendation
Wrap each oracle call in a `try/catch` inside `isHealthy` and treat a reverting oracle as returning price `0` (making that collateral contribute 0 to `maxDebt`), or alternatively skip oracles for collaterals not relevant to the operation being checked. A more targeted fix for `withdrawCollateral` specifically is to only call oracles for collaterals that remain after the withdrawal, excluding the one being withdrawn. At minimum, add per-oracle `try/catch`:

```solidity
try IOracle(collateralParam.oracle).price() returns (uint256 price) {
    maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
        .mulDivDown(collateralParam.lltv, WAD);
} catch {
    // treat as 0 contribution or revert with a specific error
}
```

```text
    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
```
