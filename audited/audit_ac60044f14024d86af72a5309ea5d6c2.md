### Title
Missing Minimum Maturity Check in `touchMarket` Allows Creation of Pre-Expired Markets with Permanently Frozen Borrow Side - (File: src/Midnight.sol)

### Summary
`touchMarket` enforces only an upper bound on `market.maturity` (`<= block.timestamp + 100 * 365 days`) but has no lower bound, allowing any unprivileged caller to create a market with `maturity = block.timestamp - 1`. Because `take` unconditionally reverts with `CannotIncreaseDebtPostMaturity` whenever `block.timestamp > market.maturity && sellerDebtIncrease > 0`, and because market config is immutable after creation, the borrow side of such a market is permanently frozen from the moment of creation.

### Finding Description
**Root cause — `touchMarket` (src/Midnight.sol:757-758):**

```solidity
if (marketState[id].tickSpacing == 0) {
    require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
```

Only a ceiling is enforced. There is no `require(market.maturity >= block.timestamp)` or any equivalent floor check. A value of `0` or `block.timestamp - 1` passes silently.

**Trigger — `take` (src/Midnight.sol:391):**

```solidity
require(block.timestamp <= offer.market.maturity || sellerDebtIncrease == 0, CannotIncreaseDebtPostMaturity());
```

With `maturity < block.timestamp` from inception, `block.timestamp <= maturity` is always `false`. Any `take` where the seller (borrower) has no pre-existing credit to offset — i.e., `sellerDebtIncrease > 0` — reverts immediately and permanently.

**Exploit flow:**
1. Attacker (unprivileged market creator) calls `touchMarket` with `market.maturity = block.timestamp - 1`.
2. Market is created successfully; `tickSpacing` is set, fees are copied, market is stored immutably.
3. Any subsequent `take` with a sell offer (borrower side) where `sellerDebtIncrease > 0` reverts with `CannotIncreaseDebtPostMaturity`.
4. Because market config is immutable (`market_config_is_immutable_after_creation: true`), the borrow side is frozen forever.

**Existing checks are insufficient:** The only maturity check in `touchMarket` is the `MaturityTooFar` upper-bound guard. No other function in the creation path validates that maturity is in the future.

### Impact Explanation
The borrow side of the market is permanently frozen from inception. No borrower can ever increase debt in this market. Lenders who sign offers referencing this market ID will have those offers permanently unfillable on the debt-increasing path. The market is created in a permanently post-maturity state with no recovery path, constituting a permanent fund-flow freeze for the borrow side.

### Likelihood Explanation
Preconditions: none beyond being able to call `touchMarket` (permissionless). The attacker only needs to supply a valid collateral list, allowed LLTV, valid maxLif, and set `maturity = block.timestamp - 1`. The attack is trivially repeatable for any loan token / collateral combination, creating an unbounded number of permanently broken markets. The existing fuzz test `testPostMaturitySettlementFee` in `test/SettlementFeeTest.sol` already exercises this exact path (`maturity = bound(maturity, 0, vm.getBlockTimestamp() - 1)`) and confirms the revert.

### Recommendation
Add a minimum maturity floor in `touchMarket`, immediately after the `MaturityTooFar` check:

```solidity
require(market.maturity >= block.timestamp, MaturityInPast());
require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
```

Introduce a corresponding `MaturityInPast` error in `IMidnight`.

### Proof of Concept

```solidity
function testExpiredMarketAtCreationFreezesBorrowSide() public {
    // 1. Build a valid market except maturity is already expired
    Market memory m;
    m.loanToken = address(loanToken);
    m.maturity = block.timestamp - 1;          // already expired
    CollateralParams[] memory cp = new CollateralParams[](1);
    cp[0] = CollateralParams({
        token: address(collateralToken1),
        lltv: 0.77e18,
        maxLif: maxLif(0.77e18, 0.25e18),
        oracle: address(oracle1)
    });
    m.collateralParams = cp;

    // 2. Unprivileged caller creates the market — succeeds (no floor check)
    bytes32 mId = midnight.touchMarket(m);
    assertTrue(midnight.tickSpacing(mId) > 0, "market created");

    // 3. Prepare a sell offer (borrower increases debt)
    Offer memory sellOffer;
    sellOffer.buy = false;
    sellOffer.maker = borrower;
    sellOffer.market = m;
    sellOffer.maxUnits = 100;
    sellOffer.tick = MAX_TICK;
    sellOffer.expiry = block.timestamp + 1000;

    collateralize(m, borrower, 100);

    // 4. Any take that would increase sellerDebt reverts immediately
    vm.expectRevert(IMidnight.CannotIncreaseDebtPostMaturity.selector);
    take(100, lender, sellOffer);

    // Assert: borrow side permanently frozen — no debt can ever be created
}
```
