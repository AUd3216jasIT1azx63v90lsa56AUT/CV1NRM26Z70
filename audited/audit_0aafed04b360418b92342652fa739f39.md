### Title
Unbounded `_poolsByAgreement` array enables sponsor-triggered OOG DoS on `getPoolsByAgreement` — (`File: src/ConfidencePoolFactory.sol`)

### Summary

`createPool` imposes no per-agreement pool count limit, so a sponsor (agreement owner) can create an arbitrarily large number of pools for a single agreement. `getPoolsByAgreement` returns the entire `_poolsByAgreement[agreement]` array in one call with no pagination or gas guard, making it possible for a sufficiently large array to exhaust gas in any on-chain caller operating under a constrained gas budget.

### Finding Description

`createPool` (lines 67–114) uses `_poolsByAgreement[agreement].length` as part of the clone salt and pushes each new pool address onto `_poolsByAgreement[agreement]` (line 103) with no upper bound on how many times this can be called for the same agreement. The NatDoc comment at line 33–35 explicitly acknowledges this: *"An agreement may back many pools — each commits to its own (locked) scope, so duplicates are allowed and curated off-chain."*

`getPoolsByAgreement` (lines 117–119) is a single-statement return of the full storage array:

```solidity
function getPoolsByAgreement(address agreement) external view returns (address[] memory) {
    return _poolsByAgreement[agreement];
}
```

There is no pagination, no length cap, and no gas check. Reading a cold dynamic array from storage costs approximately 2,100 gas per element (SLOAD). With a 100,000-gas budget, the function OOGs at roughly N ≈ 47 cold elements. Even with warm storage, the threshold is in the low hundreds. A sponsor who owns a valid agreement and uses an already-allowlisted token can reach this threshold by calling `createPool` N times, each with a distinct `accounts` scope slice (or even an empty one). The only preconditions are:

1. `safeHarborRegistry.isAgreementValid(agreement)` — the sponsor owns a valid agreement (normal operational state).
2. `allowedStakeToken[stakeToken]` — at least one token is on the factory allowlist (required for any pool to exist at all; not a trusted-role compromise by the sponsor).
3. `expiry >= block.timestamp + 30 days` — trivially satisfiable.

No victim mistake or trusted-role compromise is required. The companion function `poolCountByAgreement` (line 122–124) returns only the length and is safe, but it does not mitigate the array-return path.

### Impact Explanation

Any on-chain contract that calls `getPoolsByAgreement` inside a callback, a low-gas forwarded call, or any context with a capped gas stipend will revert with OOG once N exceeds the threshold. This permanently DoS's that integration for the targeted agreement without affecting the core protocol's staking, resolution, or claim paths. The DoS is durable: once the pools are created they cannot be removed, and the array only grows.

### Likelihood Explanation

The sponsor is an unprivileged actor who already owns the agreement and is the natural creator of pools. Creating N pools is expensive in gas but requires no special access beyond what the sponsor already has. The attack is repeatable (more pools can always be added), irreversible (no pool deletion exists), and targeted (only the specific agreement's array grows). A motivated sponsor wishing to grief an integrator or competitor's tooling can execute this at any time the factory is unpaused.

### Recommendation

Add a per-agreement pool count cap enforced inside `createPool`, for example:

```solidity
uint256 private constant _MAX_POOLS_PER_AGREEMENT = 100; // tune to gas budget

if (_poolsByAgreement[agreement].length >= _MAX_POOLS_PER_AGREEMENT) {
    revert TooManyPoolsForAgreement();
}
```

Alternatively, replace `getPoolsByAgreement` with a paginated variant and document that the full-array accessor is off-chain only:

```solidity
function getPoolsByAgreement(address agreement, uint256 offset, uint256 limit)
    external view returns (address[] memory page);
```

Both mitigations should be applied together: the cap prevents unbounded growth; pagination makes the view function safe for on-chain callers regardless.

```solidity
pragma solidity 0.8.26;

import "forge-std/Test.sol";
import {ConfidencePoolFactory} from "src/ConfidencePoolFactory.sol";
// ... standard test setup imports

contract PoolConsumer {
    IConfidencePoolFactory factory;
    constructor(address f) { factory = IConfidencePoolFactory(f); }

    function process(address agreement) external {
        // simulates a callback with a capped gas budget
        address[] memory pools = factory.getPoolsByAgreement{gas: 100_000}(agreement);
        // use pools ...
    }
}

contract GetPoolsByAgreementDoSTest is Test {
    ConfidencePoolFactory factory;
    PoolConsumer consumer;
    address sponsor;
    address agreement;   // mock returning isAgreementValid=true, owner=sponsor
    address stakeToken;  // mock allowlisted ERC20

    function setUp() public {
        // deploy factory, mock agreement + token, allowlist token
        // sponsor = agreement owner
    }

    function test_OOG_after_N_pools() public {
        uint256 N = 60; // tune: expect OOG around 47–60 cold SLOADs at 100k gas
        vm.startPrank(sponsor);
        for (uint256 i = 0; i < N; i++) {
            factory.createPool(
                agreement,
                stakeToken,
                block.timestamp + 31 days,
                0,
                sponsor,
                new address[](0)
            );
        }
        vm.stopPrank();

        assertEq(factory.poolCountByAgreement(agreement), N);

        // PoolConsumer.process() must revert OOG when called with 100k gas
        vm.expectRevert();
        consumer.process(agreement); // internally calls getPoolsByAgreement{gas: 100_000}
    }
}
```

The test asserts that after N `createPool` calls by the sponsor, `consumer.process()` reverts (OOG). The threshold N at which this triggers can be found by bisecting the loop count; empirically it falls in the range 47–100 depending on storage warmth and EVM overhead.
