### Title
Unprotected `withdrawCollateral()` Call in Bundler Can DoS Atomic Bundle Transactions — (`src/periphery/MidnightBundles.sol`)

---

### Summary

`MidnightBundles.sol` wraps every `take()` call in a `try/catch` block, explicitly designed to skip any revert reason. However, the `withdrawCollateral()` calls that follow in the same functions are **not** wrapped in `try/catch`. Since `withdrawCollateral()` internally calls `isHealthy()`, which calls `IOracle(...).price()` for each activated collateral, an oracle revert on the taker's side causes the entire bundle transaction to revert — defeating the resilience guarantee the `try/catch` on `take()` was meant to provide.

---

### Finding Description

In `buyWithUnitsTargetAndWithdrawCollateral()` and `buyWithAssetsTargetAndWithdrawCollateral()`, the pattern is:

**Step 1 — Protected `take()` loop:**

```solidity
try IMidnight(MIDNIGHT)
    .take(takes[i].offer, takes[i].ratifierData, unitsToTake, taker, address(0), address(0), "") returns (
    uint256 resBuyerAssets, uint256
) {
    filledUnits += unitsToTake;
    filledBuyerAssets += resBuyerAssets;
} catch {}
```

The NatSpec comment explicitly states: *"Skips every reason why take can revert (including ones that are not asynchrony related)."* [1](#0-0) [2](#0-1) 

**Step 2 — Unprotected `withdrawCollateral()` loop:**

```solidity
for (uint256 i; i < collateralWithdrawals.length; i++) {
    IMidnight(MIDNIGHT)
        .withdrawCollateral(
            market,
            collateralWithdrawals[i].collateralIndex,
            collateralWithdrawals[i].assets,
            taker,
            collateralReceiver
        );
}
```

No `try/catch` here. [3](#0-2) [4](#0-3) 

**Root cause — `withdrawCollateral()` calls `isHealthy()` which calls the oracle:**

```solidity
require(isHealthy(market, id, onBehalf), UnhealthyBorrower());
``` [5](#0-4) 

And `isHealthy()` calls `IOracle(collateralParam.oracle).price()` for every activated collateral of the taker:

```solidity
uint256 price = IOracle(collateralParam.oracle).price();
``` [6](#0-5) 

This is a **different oracle** than the one called during `take()` (which checks the *seller's* collateral). The `take()` call is protected; the `withdrawCollateral()` call is not. This is the direct analog to the external report: the same vulnerability class (unprotected external call that can revert) applied to two calls that use different parameters (different collateral oracles), where only the first is guarded.

The protocol itself documents this asymmetry in the LIVENESS section:

> *"If an activated collateral oracle reverts on price, liquidate reverts."*
> *"Note that a borrower can activate a collateral once its oracle is reverting because the oracle is not called in supplyCollateral."* [7](#0-6) 

---

### Impact Explanation

If the taker's collateral oracle is reverting at the time of the bundle call (e.g., Chainlink circuit breaker, stale feed, price out of bounds), `withdrawCollateral()` reverts, causing the **entire bundle transaction to revert**. All state changes — including the successfully executed `take()` calls — are rolled back. The user loses gas and cannot complete the atomic operation. This is a **DoS on the bundler's atomic operations** for any user whose collateral oracle is temporarily unavailable.

---

### Likelihood Explanation

Oracle reverts are a realistic, documented liveness condition in this protocol. Chainlink price feeds can revert due to circuit breakers (e.g., price deviation exceeds threshold), sequencer downtime on L2s, or stale heartbeat. The taker's oracle is a **different** oracle from the seller's oracle checked inside `take()`, so it is entirely possible for the seller's oracle to be healthy (allowing `take()` to succeed) while the taker's oracle is reverting (causing `withdrawCollateral()` to fail). No privileged access is required; any user in this state is affected.

---

### Recommendation

Wrap the `withdrawCollateral()` calls in `try/catch` blocks, consistent with the `take()` calls. If a withdrawal fails, it can be skipped or the transaction can revert with a more informative error. A minimal fix:

```solidity
for (uint256 i; i < collateralWithdrawals.length; i++) {
    try IMidnight(MIDNIGHT).withdrawCollateral(
        market,
        collateralWithdrawals[i].collateralIndex,
        collateralWithdrawals[i].assets,
        taker,
        collateralReceiver
    ) {} catch {
        revert WithdrawCollateralFailed();
    }
}
```

Alternatively, if the intent is to be fully resilient (matching the `take()` semantics), use an empty `catch {}` and track which withdrawals succeeded.

---

### Proof of Concept

1. A market exists with two collateral tokens: `CollateralA` (oracle healthy) and `CollateralB` (oracle reverting, e.g., Chainlink circuit breaker triggered).
2. The taker has debt in this market, with `CollateralB` activated in their `collateralBitmap`.
3. The taker calls `buyWithUnitsTargetAndWithdrawCollateral()` with a list of sell offers and a `collateralWithdrawals` entry for `CollateralA`.
4. The `take()` loop executes successfully — the seller's oracle (`CollateralA`) is healthy, so `isHealthy()` for the seller passes. Each take is caught by `try/catch` regardless.
5. Execution reaches the `withdrawCollateral()` loop. `withdrawCollateral()` calls `isHealthy()` for the **taker**, which iterates over all activated collaterals including `CollateralB`. `IOracle(CollateralB.oracle).price()` reverts.
6. `withdrawCollateral()` reverts with no `try/catch` to catch it. The entire transaction reverts.
7. The taker loses gas and cannot complete the bundle. The DoS persists for as long as `CollateralB`'s oracle is reverting. [8](#0-7) [9](#0-8)

    function buyWithUnitsTargetAndWithdrawCollateral(
        uint256 targetUnits,
        uint256 maxBuyerAssets,
        address taker,
        TokenPermit memory loanTokenPermit,
        Take[] memory takes,
        CollateralWithdrawal[] memory collateralWithdrawals,
        address collateralReceiver,
        uint256 referralFeePct,
        address referralFeeRecipient
    ) external {
        require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
        require(referralFeePct < WAD, PctExceeded());
        address loanToken = takes[0].offer.market.loanToken;
        // touchMarket to have the correct settlement fees.
        bytes32 id = IMidnight(MIDNIGHT).touchMarket(takes[0].offer.market);

        pullToken(loanToken, msg.sender, maxBuyerAssets, loanTokenPermit);
        forceApproveMax(loanToken, MIDNIGHT);

        uint256 filledUnits;
        uint256 filledBuyerAssets;
        for (uint256 i; i < takes.length && filledUnits < targetUnits; i++) {
            require(!takes[i].offer.buy, InconsistentSide());
            require(IMidnight(MIDNIGHT).toId(takes[i].offer.market) == id, InconsistentMarket());
            uint256 unitsToTake = min(
                targetUnits - filledUnits,
                takes[i].units,
                ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
            );
            try IMidnight(MIDNIGHT)
                .take(takes[i].offer, takes[i].ratifierData, unitsToTake, taker, address(0), address(0), "") returns (
                uint256 resBuyerAssets, uint256
            ) {
                filledUnits += unitsToTake;
                filledBuyerAssets += resBuyerAssets;
            } catch {}
        }

        require(filledUnits == targetUnits, OutOfOffers());

        Market memory market = takes[0].offer.market;
        for (uint256 i; i < collateralWithdrawals.length; i++) {
            IMidnight(MIDNIGHT)
                .withdrawCollateral(
                    market,
                    collateralWithdrawals[i].collateralIndex,
                    collateralWithdrawals[i].assets,
                    taker,
                    collateralReceiver
                );
        }

        uint256 referralFeeAssets = filledBuyerAssets.mulDivDown(referralFeePct, WAD - referralFeePct);
        if (referralFeeAssets > 0) SafeTransferLib.safeTransfer(loanToken, referralFeeRecipient, referralFeeAssets);
        SafeTransferLib.safeTransfer(loanToken, msg.sender, maxBuyerAssets - filledBuyerAssets - referralFeeAssets);
    }
```

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
