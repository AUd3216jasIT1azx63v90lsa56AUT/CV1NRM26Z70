### Title
Fee-on-Transfer Collateral Inflates Stored Balance, Enabling Undercollateralized Borrowing - (File: src/libraries/SafeTransferLib.sol / src/Midnight.sol)

### Summary

`supplyCollateral` credits `_position.collateral[collateralIndex]` with the caller-supplied `assets` value before calling `SafeTransferLib.safeTransferFrom`, and `safeTransferFrom` only checks the boolean return value of `transferFrom` — it never verifies the actual balance delta. For a fee-on-transfer collateral token, the contract records `assets` but receives only `assets - fee`. `isHealthy` then computes `maxDebt` from the inflated stored value, allowing the borrower to carry debt that exceeds the real collateral value, creating an undercollateralized position and potential bad debt.

### Finding Description

**Exact code path:**

`supplyCollateral` (src/Midnight.sol lines 531–545):

```solidity
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets); // ← full assets credited
// ...bitmap update...
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets); // ← only assets-fee arrives
``` [1](#0-0) 

`SafeTransferLib.safeTransferFrom` (src/libraries/SafeTransferLib.sol lines 24–34) only checks that the call did not revert and returned `true`; it performs no balance-before/balance-after comparison: [2](#0-1) 

`isHealthy` (src/Midnight.sol lines 944–960) reads `_position.collateral[i]` — the stored, inflated value — to compute `maxDebt`:

```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [3](#0-2) 

`touchMarket` is permissionless and validates only LLTV tier, maxLif formula, and sorted token addresses — no check prevents a fee-on-transfer token from being registered as a collateral: [4](#0-3) 

**Attacker inputs and exploit flow:**

1. Attacker (as market creator) calls `touchMarket` with a market whose `collateralParams[i].token` is a fee-on-transfer ERC20 (e.g., 1% fee). This is fully permissionless.
2. Attacker calls `supplyCollateral(market, i, X, attacker)`. Storage records `_position.collateral[i] = X`. The contract receives `X * 0.99`.
3. Attacker calls into the borrow path (via `take` as a seller). `isHealthy` computes `maxDebt` using `X`, not `X * 0.99`. The attacker can carry debt up to `X * price * lltv / ORACLE_PRICE_SCALE / WAD`.
4. Actual collateral value backing that debt is only `(X * 0.99) * price * lltv / ORACLE_PRICE_SCALE / WAD`.
5. The position is undercollateralized from inception. If the oracle price drops even slightly, the position becomes liquidatable with insufficient collateral to cover the debt, realizing bad debt for lenders.

**Why existing checks fail:**

- `safeTransferFrom` does not measure received tokens.
- `isHealthy` trusts stored accounting, not contract balance.
- The Certora formal verification explicitly assumes "ERC20 tokens are assumed well-behaved… no fee taking from sender or receiver, no rebasing" — the proofs do not cover this case. [5](#0-4) [6](#0-5) 

The protocol's own `live_context.json` flags this class of issue as requiring explicit exclusion if not in scope: [7](#0-6) 

`SECURITY.md` does not exclude fee-on-transfer tokens from scope.

### Impact Explanation

A borrower can create a position where `isHealthy` returns `true` but the actual token balance backing the debt is less than the debt. At any oracle price drop, the position becomes liquidatable with insufficient collateral to cover repayment, socializing the shortfall as bad debt among lenders. This directly violates the invariant that every borrow must be backed by correctly valued collateral.

### Likelihood Explanation

- **Preconditions:** A fee-on-transfer token must be usable as a collateral token. Market creation is permissionless; any address can call `touchMarket` with any ERC20 as collateral. No on-chain whitelist exists.
- **Feasibility:** Trivially reachable in a single transaction sequence: create market → supply fee-on-transfer collateral → borrow.
- **Repeatability:** Repeatable on every `supplyCollateral` call with a fee-on-transfer token; the gap compounds with each additional supply.

### Recommendation

Record the contract's token balance before and after the `safeTransferFrom` call in `supplyCollateral`, and credit only the actual received delta:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

Alternatively, explicitly document and enforce (via `touchMarket` validation or a registry) that fee-on-transfer tokens are not permitted as collateral tokens.

### Proof of Concept

// Foundry unit test sketch
contract FeeToken is ERC20 {
    // 1% fee on every transferFrom
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount / 100;
        super.transferFrom(from, to, amount - fee); // only amount-fee arrives
        return true;
    }
}

function testFeeOnTransferCollateralInflation() public {
    FeeToken feeToken = new FeeToken();
    // Build market with feeToken as collateral[0], normalToken as collateral[1]
    Market memory m = buildMarket(address(feeToken), address(normalToken));
    midnight.touchMarket(m);

    uint256 supplyAmount = 1000e18;
    feeToken.mint(borrower, supplyAmount);
    vm.prank(borrower);
    feeToken.approve(address(midnight), supplyAmount);
    vm.prank(borrower);
    midnight.supplyCollateral(m, 0, supplyAmount, borrower);

    // Stored collateral = 1000e18, actual balance received = 990e18
    assertEq(midnight.collateral(id, borrower, 0), 1000e18);
    assertEq(feeToken.balanceOf(address(midnight)), 990e18);

    // isHealthy uses 1000e18 to compute maxDebt
    // Borrow up to maxDebt computed from 1000e18
    // Assert: position is healthy per isHealthy() but actual collateral < debt
    assertTrue(midnight.isHealthy(m, id, borrower));
    // Assert: contract balance (990e18) * price * lltv < debt taken
    // => undercollateralized position, bad debt on any price drop
}
```

Expected assertions: `midnight.collateral(id, borrower, 0) == supplyAmount` (inflated), `feeToken.balanceOf(address(midnight)) == supplyAmount * 99 / 100` (actual), and `isHealthy` returns `true` while actual collateral value is less than outstanding debt.

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
