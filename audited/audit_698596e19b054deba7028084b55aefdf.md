### Title
Zero-price oracle allows collateral seizure with zero loan token repayment - (File: src/interfaces/IMidnight.sol → src/Midnight.sol)

### Summary
When the oracle for a collateral returns `0`, the `liquidate` function computes `repaidUnits = seizedAssets.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif) = 0` via the `seizedAssets > 0` branch. The collateral is fully transferred to the liquidator while `_position.debt` is decremented by zero, violating the invariant that collateral cannot be seized without repaying correct debt value.

### Finding Description

**Exact code path — `src/Midnight.sol`:**

The while loop at lines 607–618 iterates the borrower's `collateralBitmap`, calls `IOracle(_collateralParam.oracle).price()` for each active collateral, and assigns the result to `liquidatedCollatPrice` when `i == collateralIndex`: [1](#0-0) 

If the oracle returns `0`, `liquidatedCollatPrice` is set to `0`. That collateral also contributes `0` to `maxDebt` (line 613), making `originalDebt > maxDebt` true and the borrower appear liquidatable: [2](#0-1) 

The liquidatability check at lines 620–624 then passes: [3](#0-2) 

Inside the `seizedAssets > 0` branch at line 649–650, `repaidUnits` is computed as:
```
repaidUnits = seizedAssets.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif)
``` [4](#0-3) 

The `mulDivUp` implementation is:
```solidity
return (x * y + (d - 1)) / d;
``` [5](#0-4) 

Step 1: `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE)` = `(0 + ORACLE_PRICE_SCALE − 1) / ORACLE_PRICE_SCALE` = `0`.
Step 2: `mulDivUp(0, WAD, lif)` = `(lif − 1) / lif` = `0`.

This is confirmed by the `MulDiv.spec` rule `mulDivZero`: [6](#0-5) 

So `repaidUnits = 0`. The state updates then become:
- `_position.collateral[collateralIndex] -= seizedAssets` (line 670) — collateral is fully removed
- `_position.debt -= 0` (line 676) — debt is unchanged
- `SafeTransferLib.safeTransfer(collateralToken, receiver, seizedAssets)` (line 696) — collateral sent to attacker
- `SafeTransferLib.safeTransferFrom(loanToken, payer, address(this), 0)` (line 717) — zero loan tokens paid [7](#0-6) 

**Attacker-controlled inputs:** `seizedAssets = borrower.collateral[collateralIndex]` (any positive value ≤ borrower's balance), `repaidUnits = 0` (explicitly passed, satisfies `atMostOneNonZero`).

**Why existing checks fail:**
- `atMostOneNonZero(0, seizedAssets)` → `true` (passes)
- `_position.debt > 0` → `true` (passes)
- `NotLiquidatable`: oracle returning `0` zeroes out that collateral's contribution to `maxDebt`, making `originalDebt > maxDebt` trivially true
- `maxRepaid` RCF check (lines 655–668): only applies in `!postMaturityMode`; even then `repaidUnits = 0 ≤ maxRepaid` always passes [8](#0-7) 

**Formal acknowledgment of the gap:** The Certora `NoDivisionByZero.spec` README explicitly states the spec does not cover this case: [9](#0-8) 

The `noDivisionByZeroLiquidate` rule requires `price > 0` as a precondition assumption, meaning the formal proofs do not cover the `price = 0` + `seizedAssets > 0` combination: [10](#0-9) 

The `Reverts.spec` explicitly axiomatizes `mulDivUp(a, 0, d) = 0` as a known oracle-zero modeling requirement, confirming the protocol is aware oracles can return `0`: [11](#0-10) 

### Impact Explanation
A liquidator seizes the borrower's entire collateral at `collateralIndex` while repaying zero loan tokens. The borrower's debt is not reduced, yet their collateral balance drops to zero and the collateral bitmap bit is cleared. The protocol's `withdrawable` pool is not increased. The invariant "collateral cannot be seized without repaying correct debt value" is directly violated, causing direct, irreversible loss of the borrower's collateral with no corresponding debt relief.

### Likelihood Explanation
**Preconditions:**
1. The oracle for the targeted collateral returns `0` — explicitly modeled as a reachable scenario in the Certora specs.
2. The borrower has `debt > 0` and a non-zero balance at `collateralIndex` with the bitmap bit set.
3. The borrower is liquidatable (oracle returning `0` makes `maxDebt` contribution from that collateral `0`, trivially satisfying `originalDebt > maxDebt`; alternatively, post-maturity mode requires only `block.timestamp > market.maturity`).

The exploit is repeatable for any market whose oracle can transiently return `0` (e.g., uninitialized feed, price feed gap, or oracle contract returning `0` on error). No special privilege is required — any address can call `liquidate`.

### Recommendation
Add an explicit guard before computing `repaidUnits` in the `seizedAssets > 0` branch to reject a zero oracle price:

```solidity
if (seizedAssets > 0) {
    require(liquidatedCollatPrice > 0, ZeroOraclePrice());
    repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
}
```

This mirrors the existing Certora assumption (`price > 0`) and makes it an enforced invariant at runtime, preventing collateral seizure when the oracle price is zero. [4](#0-3)

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {IMidnight} from "src/interfaces/IMidnight.sol";
import {Oracle} from "test/mocks/Oracle.sol";
// ... standard test setup imports

contract ZeroOracleLiquidationTest is Test {
    // Setup: deploy Midnight, create market with one collateral,
    // borrower supplies collateral and takes a loan (debt > 0).

    function testZeroOracleSeizesCollateralForFree() public {
        // 1. Setup: borrower has collateral[0] = C, debt = D > 0
        //    Oracle initially returns a healthy price.
        uint256 initialCollateral = midnight.collateral(id, borrower, 0);
        uint256 initialDebt = midnight.debtOf(id, borrower);
        assertGt(initialCollateral, 0);
        assertGt(initialDebt, 0);

        // 2. Oracle returns 0 for collateral[0].
        Oracle(market.collateralParams[0].oracle).setPrice(0);

        // 3. Attacker calls liquidate with seizedAssets = full collateral, repaidUnits = 0.
        uint256 attackerLoanBalanceBefore = loanToken.balanceOf(attacker);
        uint256 attackerCollateralBalanceBefore = collateralToken.balanceOf(attacker);

        vm.prank(attacker);
        (uint256 seized, uint256 repaid) =
            midnight.liquidate(market, 0, initialCollateral, 0, borrower, false, attacker, address(0), "");

        // 4. Assertions:
        assertEq(repaid, 0, "repaidUnits must be 0");
        assertEq(seized, initialCollateral, "all collateral seized");
        assertEq(midnight.collateral(id, borrower, 0), 0, "borrower collateral wiped");
        assertEq(midnight.debtOf(id, borrower), initialDebt, "borrower debt unchanged");
        assertEq(loanToken.balanceOf(attacker), attackerLoanBalanceBefore, "attacker paid 0 loan tokens");
        assertEq(
            collateralToken.balanceOf(attacker),
            attackerCollateralBalanceBefore + initialCollateral,
            "attacker received full collateral"
        );
    }
}
```

Expected result: all four state assertions pass, demonstrating collateral is seized with zero loan token repayment and zero debt reduction.
