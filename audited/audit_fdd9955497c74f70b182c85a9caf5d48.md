### Title
Fee-on-Transfer Loan Token Causes Debt/Withdrawable Over-Accounting in `repay` - (File: src/Midnight.sol)

### Summary

`Midnight.repay` decrements `position.debt` and increments `marketState.withdrawable` by the full `units` value before calling `safeTransferFrom`, which for a fee-on-transfer loan token delivers only `units * (1 - fee)` to the contract. Because `SafeTransferLib.safeTransferFrom` only checks call success and a boolean return value — not the actual balance delta — the shortfall is silently accepted. Each repayment in such a market permanently inflates `withdrawable` relative to the real token balance, accumulating insolvency.

`SafeTransferLib.safeTransferFrom` in `src/libraries/SafeTransferLib.sol` lines 24–34 only verifies the call did not revert and that the return value (if any) is `true`. It performs no before/after balance check and cannot detect that fewer than `units` tokens were actually deposited. [2](#0-1) 

**Attacker-controlled inputs:**

A market creator (unprivileged per the rules) deploys a market whose `loanToken` is a fee-on-transfer ERC-20. Any borrower in that market then calls:

```
repay(market, units, onBehalf, callback, data)
```

where `callback` is optionally a `IRepayCallback` payer. The borrower controls `units`.

**Exploit flow:**

1. Market exists with a fee-on-transfer `loanToken` (e.g., 1% fee).
2. Borrower calls `repay(market, 1000, self, address(0), "")`.
3. Line 508 reduces `position.debt` by 1000.
4. Line 509 increases `marketState.withdrawable` by 1000.
5. Line 520 calls `safeTransferFrom(loanToken, borrower, address(this), 1000)` — the token transfers 1000 but the contract receives only 990 (10 taken as fee). The call returns `true`, no revert.
6. Net effect: Midnight's accounting says 1000 more tokens are withdrawable; its actual balance increased by only 990. A 10-token shortfall is created.

**Why existing checks fail:**

- `SafeTransferLib.safeTransferFrom` has no balance-delta check. [3](#0-2) 
- There is no post-transfer invariant check in `repay`. [4](#0-3) 
- No market creation guard rejects fee-on-transfer tokens.
- The optional `IRepayCallback.onRepay` path does not change this: the transfer at line 520 is always the final step regardless of whether a callback is used. [5](#0-4) 

### Impact Explanation

Every repayment inflates `marketState.withdrawable` by `units * fee` more than the real token balance supports. Lenders calling `withdraw` drain the real balance in FIFO order; the last lenders to withdraw find the contract insolvent by the accumulated fee shortfall. The invariant "contract balances cover withdrawable assets" is broken permanently and grows with each repayment.

### Likelihood Explanation

Preconditions: a market must exist with a fee-on-transfer loan token. Market creation is permissionless (market creator is an unprivileged role). Fee-on-transfer tokens exist in production (e.g., tokens with deflationary mechanics or tax tokens). The exploit is repeatable on every `repay` call in such a market, requires no special privilege, and the shortfall compounds linearly with repayment volume.

### Recommendation

Add a balance check around the transfer in `repay` to verify the actual received amount equals `units`:

```solidity
uint256 balanceBefore = IERC20(market.loanToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);
require(IERC20(market.loanToken).balanceOf(address(this)) - balanceBefore == units, BadTransferAmount());
```

Alternatively, document that fee-on-transfer tokens are unsupported and add a market creation check (e.g., a round-trip transfer test), consistent with how Morpho Blue handles this class of token.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
// ... market setup imports

contract FeeToken is ERC20 {
    uint256 public constant FEE_BPS = 100; // 1%
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount * FEE_BPS / 10000;
        super.transferFrom(from, to, amount - fee); // deliver amount-fee to `to`
        _burn(from, fee);                           // burn the fee
        return true;
    }
}

contract RepayFeeOnTransferTest is Test {
    function test_repayFeeOnTransferShortfall() public {
        // 1. Deploy FeeToken as loanToken
        // 2. Create market with FeeToken as loanToken
        // 3. Supply liquidity, borrow 1000 units
        // 4. Record midnight.balanceOf(feeToken) before repay
        uint256 balBefore = feeToken.balanceOf(address(midnight));
        // 5. Borrower calls repay(market, 1000, borrower, address(0), "")
        midnight.repay(market, 1000, borrower, address(0), "");
        uint256 balAfter = feeToken.balanceOf(address(midnight));
        // 6. Assert: actual balance increase < units
        assertLt(balAfter - balBefore, 1000, "fee-on-transfer shortfall not detected");
        // 7. Assert: withdrawable increased by full 1000
        assertEq(midnight.marketState(id).withdrawable, withdrawableBefore + 1000);
        // 8. Assert: debt decreased by full 1000
        assertEq(midnight.position(id, borrower).debt, debtBefore - 1000);
        // 9. Invariant: withdrawable > actual balance => insolvency
        assertGt(midnight.marketState(id).withdrawable, balAfter);
    }
}
```

Expected: assertions at steps 6–9 all pass, confirming the shortfall and broken invariant.

        external

        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);

        marketState[id].withdrawable += UtilsLib.toUint128(units);

        address payer = callback != address(0) ? callback : msg.sender;
        emit EventsLib.Repay(msg.sender, id, units, onBehalf, payer);

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
