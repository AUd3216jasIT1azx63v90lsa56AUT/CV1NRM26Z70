### Title
First taker in an untouched 128-collateral market pays full market creation gas cost - (File: src/Midnight.sol)

### Summary
`take()` calls `touchMarket(offer.market)` unconditionally at line 347 before any check that the market already exists. When a maker signs an offer for a valid but untouched 128-collateral market, the first taker's transaction triggers the full market initialization: a 128-iteration collateral-validation loop and a SSTORE2 CREATE2 deployment of the entire market struct. The taker bears this gas cost with no recourse.

### Finding Description
**Code path:**
- `take()` → line 347: `bytes32 id = touchMarket(offer.market);`
- `touchMarket()` → line 757: `if (marketState[id].tickSpacing == 0)` → enters initialization branch
- Lines 762–773: for-loop over `market.collateralParams.length` (up to 128), reading each entry and calling `isLltvAllowed`, `maxLif` twice per iteration
- Line 786: `IdLib.storeInCode(market, INITIAL_CHAIN_ID)` — deploys the full ABI-encoded `Market` struct (including all 128 `CollateralParams`) as CREATE2 bytecode; `toMarket()` at lines 879–880 confirms this is read back via `.code`

**Root cause:** No guard in `take()` requires `marketState[id].tickSpacing > 0` before calling `touchMarket`. Every other entry-point (`withdraw`, `repay`, `supplyCollateral`, `withdrawCollateral`, `liquidate`) has the same property, but `take()` is the only one where a third party (the taker) can be forced into the call by a maker-controlled offer.

**Attacker inputs:** Maker constructs a `Market` with exactly 128 valid, sorted `CollateralParams` entries (all passing `isLltvAllowed` and `maxLif` checks), signs an offer for it, and publishes the offer off-chain. The market is never touched before the taker submits.

**Why checks fail:** The only validation in `take()` before `touchMarket` is the authorization check at line 346. After `touchMarket` returns, `_marketState.tickSpacing` is now set, so all subsequent checks pass normally. Nothing prevents the market from being created inside the taker's call.

### Impact Explanation
The first taker in a 128-collateral market pays the full market creation gas cost — the 128-iteration loop plus a SSTORE2 CREATE2 deployment of a ~16 KB market struct — in addition to normal take costs. On mainnet this can amount to several million gas units of unexpected cost. If the taker's gas limit was sized for a normal take, the transaction reverts out-of-gas, effectively blocking the take entirely until the market is separately touched by another transaction.

### Likelihood Explanation
Any unprivileged maker can set this up with zero capital at risk: construct a valid 128-collateral market, sign an offer, and publish it. The precondition (market untouched) is trivially achievable for any new market. The attack is repeatable for every new market a maker creates. Automated takers that do not pre-check `tickSpacing(id) > 0` are silently griefed.

### Recommendation
Add a guard at the start of `take()` (before or replacing the `touchMarket` call) that requires the market to already be touched:

```solidity
bytes32 id = toId(offer.market);
require(marketState[id].tickSpacing > 0, MarketNotCreated());
```

Market creation should remain available via a dedicated `touchMarket` call (already `public`) or via other entry-points, but `take()` should not silently absorb creation cost on behalf of the taker. Alternatively, document and enforce that makers must touch the market before publishing offers, and have the ratifier verify this.

        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
