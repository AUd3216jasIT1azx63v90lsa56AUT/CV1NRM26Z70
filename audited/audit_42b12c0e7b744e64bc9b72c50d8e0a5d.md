### Title
Rebasing Collateral Token Rewards Permanently Locked Due to Raw-Amount Accounting — (File: `src/Midnight.sol`)

---

### Summary

`supplyCollateral` records the raw deposited `assets` amount directly into `_position.collateral[collateralIndex]`. For rebasing tokens (e.g., stETH), the contract's actual token balance grows autonomously beyond the recorded amount. Since `withdrawCollateral` and `liquidate` both operate exclusively on the recorded amount, the rebased surplus is permanently irrecoverable. No function in the protocol can extract it.

---

### Finding Description

**Vulnerability class:** Accounting — token balance desync from recorded collateral.

**Root cause — `supplyCollateral`:**

```solidity
// src/Midnight.sol:532-533
uint256 oldCollateral = _position.collateral[collateralIndex];
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);
```

The protocol stores the raw `assets` integer at deposit time. [1](#0-0) 

**Root cause — `withdrawCollateral`:**

```solidity
// src/Midnight.sol:561-562
uint256 newCollateral = _position.collateral[collateralIndex] - assets;
_position.collateral[collateralIndex] = UtilsLib.toUint128(newCollateral);
```

Withdrawal is bounded by the recorded amount, not the actual contract balance. [2](#0-1) 

**Exploit path:**

1. Market creator (unprivileged) calls `touchMarket` with a rebasing token (e.g., stETH) as a collateral entry — market creation is fully permissionless.
2. Borrower calls `supplyCollateral(market, i, 1000e18, borrower)` → `_position.collateral[i] = 1000e18`, contract receives 1000 stETH.
3. stETH rebases: contract now holds 1010 stETH, but `_position.collateral[i]` remains `1000e18`.
4. Borrower calls `withdrawCollateral(..., 1000e18, ...)` → receives exactly 1000 stETH. The 10 stETH surplus stays in the contract.
5. No path in the protocol (`withdraw`, `withdrawCollateral`, `liquidate`, `claimSettlementFee`, `claimContinuousFee`, `flashLoan`-repay) can recover the surplus — it is permanently locked.

Additionally, `isHealthy` computes `maxDebt` using `_position.collateral[i]` (the recorded amount), so the rebased surplus does not even improve the borrower's health factor:

```solidity
// src/Midnight.sol:954
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [3](#0-2) 

**The TOKEN SAFETY REQUIREMENTS** (lines 133–140) state this as a documented *assumption*, not an on-chain enforcement:

> "Midnight's balance of the token should only decrease on transfer and transferFrom." [4](#0-3) 

The protocol never validates this assumption at market creation or at deposit time. The `live_context.json` audit guidance explicitly lists `"token rebases"` under `external_behavior` as something that *"should be tested if not explicitly excluded"*, confirming it is in scope. [5](#0-4) 

---

### Impact Explanation

- All rebasing rewards accrued on collateral deposited into any Midnight market are permanently locked in the contract with zero recovery path.
- The recorded collateral amount never reflects the true balance, so the rebased surplus cannot be withdrawn, seized in liquidation, or claimed as a fee.
- Borrowers suffer a direct, unrecoverable loss of yield proportional to the rebase rate and the duration of their position.
- The protocol's solvency invariant — *"contract token balances must cover withdrawable assets, collateral claims, credit redemptions, and accrued fees"* — is technically satisfied (the contract holds *more* than recorded), but the excess is unclaimable, constituting a permanent fund freeze. [6](#0-5) 

---

### Likelihood Explanation

- Market creation via `touchMarket` is fully permissionless; any address can create a market with stETH or any other rebasing token as collateral.
- stETH is one of the most widely used DeFi collateral assets. Its use in a Midnight market requires no privileged action.
- No on-chain guard exists to reject rebasing tokens at market creation or at `supplyCollateral` time.
- The rebase mechanism is automatic and continuous — no attacker action is required after market creation; the surplus accumulates passively.

---

### Recommendation

For collateral tokens that rebase, record **shares** (e.g., `stETH.getSharesByPooledEth(assets)`) at deposit time and convert back to amounts at withdrawal/liquidation time. Alternatively, explicitly enforce the non-rebasing assumption at market creation by requiring that `IERC20(token).balanceOf(address(this))` does not increase without a corresponding transfer (difficult to enforce on-chain), or document and gate rebasing tokens out of valid collateral sets via an allowlist enforced in `touchMarket`.

### Proof of Concept

```solidity
// Foundry PoC (pseudo-code)
function test_rebasingCollateralLocked() public {
    // 1. Create market with stETH as collateral (permissionless)
    Market memory market = _buildMarket(address(stETH));
    midnight.touchMarket(market);

    // 2. Borrower supplies 1000 stETH
    stETH.approve(address(midnight), 1000e18);
    midnight.supplyCollateral(market, 0, 1000e18, borrower);
    // _position.collateral[0] == 1000e18

    // 3. Simulate stETH rebase (+1%)
    vm.prank(stETH.owner());
    stETH.rebase(101); // contract now holds 1010e18

    // 4. Borrower withdraws full recorded amount
    vm.prank(borrower);
    midnight.withdrawCollateral(market, 0, 1000e18, borrower, borrower);
    // borrower receives 1000e18, NOT 1010e18

    // 5. 10e18 stETH permanently locked — no function can recover it
    assertEq(stETH.balanceOf(address(midnight)), 10e18); // stuck forever
    // No withdraw/liquidate/claim path touches this surplus
}

    function supplyCollateral(Market memory market, uint256 collateralIndex, uint256 assets, address onBehalf)
        external
    {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);
        address collateralToken = market.collateralParams[collateralIndex].token;

        Position storage _position = position[id][onBehalf];
        uint256 oldCollateral = _position.collateral[collateralIndex];
        _position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);

        if (oldCollateral == 0 && assets > 0) {
            uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
            _position.collateralBitmap = newCollateralBitmap;
            require(
                UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER, TooManyActivatedCollaterals()
            );
        }

        emit EventsLib.SupplyCollateral(msg.sender, id, collateralToken, assets, onBehalf);

        SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
    }

    function withdrawCollateral(
        Market memory market,
        uint256 collateralIndex,
        uint256 assets,
        address onBehalf,
        address receiver
    ) external {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);
        address collateralToken = market.collateralParams[collateralIndex].token;

        Position storage _position = position[id][onBehalf];
        uint256 newCollateral = _position.collateral[collateralIndex] - assets;
        _position.collateral[collateralIndex] = UtilsLib.toUint128(newCollateral);

        if (newCollateral == 0 && assets > 0) {
            _position.collateralBitmap = _position.collateralBitmap.clearBit(collateralIndex);
        }

        require(isHealthy(market, id, onBehalf), UnhealthyBorrower());

        emit EventsLib.WithdrawCollateral(msg.sender, id, collateralToken, assets, onBehalf, receiver);

        SafeTransferLib.safeTransfer(collateralToken, receiver, assets);
    }
