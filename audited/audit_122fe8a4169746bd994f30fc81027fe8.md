### Title
Co-authorized agent can deauthorize peer agents via `setIsAuthorized`, enabling DoS on offer cancellation - (File: src/Midnight.sol)

### Summary
`Midnight.setIsAuthorized` permits any address that passes the `isAuthorized[onBehalf][msg.sender]` check to write any value into `isAuthorized[onBehalf][*]`, including setting a peer agent's authorization to `false`. Because `setConsumed` uses the same `isAuthorized` mapping to gate offer-cancellation calls, a co-authorized attacker can silently strip the `setConsumedAgent`'s permission, permanently blocking offer cancellation without touching the victim's own key.

### Finding Description

**Root cause — `Midnight.setIsAuthorized`:** [1](#0-0) 

The only guard is `onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender]`. There is no restriction on *which* `authorized` address the caller may target, nor on the direction of the change (`true` vs `false`). Any authorized agent therefore has full write access to the entire `isAuthorized[onBehalf][*]` mapping.

**`setConsumed` authorization check:** [2](#0-1) 

Offer cancellation (`amount = type(uint256).max`) is gated on the same mapping. If `isAuthorized[victim][setConsumedAgent]` is flipped to `false`, the cancellation call reverts with `Unauthorized`.

**EcrecoverAuthorizer amplification path:** [3](#0-2) 

The signer check accepts `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)`, so the attacker's own key satisfies the check. The attacker signs `Authorization(authorizer=victim, authorized=setConsumedAgent, isAuthorized=false, nonce=N)` and submits it; `EcrecoverAuthorizer` then calls `Midnight.setIsAuthorized` on behalf of victim, which passes because `isAuthorized[victim][ecrecoverAuthorizer] == true`.

**Exploit flow (direct path, no EcrecoverAuthorizer needed):**

1. Victim calls `midnight.setIsAuthorized(setConsumedAgent, true, victim)` and `midnight.setIsAuthorized(attacker, true, victim)`.
2. Attacker calls `midnight.setIsAuthorized(setConsumedAgent, false, victim)` — check `isAuthorized[victim][attacker] == true` passes.
3. `isAuthorized[victim][setConsumedAgent]` is now `false`.
4. `setConsumedAgent` calls `midnight.setConsumed(group, type(uint256).max, victim)` → reverts `Unauthorized`.
5. Victim's offers remain live and fillable past the intended cancellation point.

The existing test `testSetIsAuthorizedAuthorization` confirms an authorized agent can authorize *new* agents but does not test the deauthorization-of-peers case, leaving the invariant untested and unguarded. [4](#0-3) 

### Impact Explanation
Victim's offer-cancellation agent is silently deauthorized. Every subsequent `setConsumed(group, type(uint256).max, victim)` call reverts. Offers that should have been cancelled remain active and fillable, causing unintended debt creation or credit deployment beyond the victim's intended limits. The attacker pays only one transaction's gas.

### Likelihood Explanation
Preconditions: victim must have authorized both the cancellation agent and the attacker. This is a realistic multi-agent setup (e.g., a keeper bot as `setConsumedAgent` and a separate trading bot as `attacker`). The attack is one transaction, costs only gas, is immediately repeatable (victim re-authorizes → attacker deauthorizes again), and requires no special privileges beyond being an existing authorized agent.

### Recommendation
Restrict `setIsAuthorized` so that only the principal itself (`onBehalf == msg.sender`) may modify the authorization mapping. Authorized agents should be permitted to act *on behalf of* the principal for market operations, but not to alter the principal's trust graph. Concretely:

```solidity
function setIsAuthorized(address authorized, bool newIsAuthorized, address onBehalf) external {
    require(onBehalf == msg.sender, Unauthorized()); // only self
    isAuthorized[onBehalf][authorized] = newIsAuthorized;
    emit EventsLib.SetIsAuthorized(msg.sender, authorized, newIsAuthorized, onBehalf);
}
```

Apply the same restriction in `EcrecoverAuthorizer.setIsAuthorized` by requiring `signer == authorization.authorizer` (drop the `isAuthorized` branch). [3](#0-2) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {IMidnight} from "../src/interfaces/IMidnight.sol";

contract DeauthorizePeerAgentTest is BaseTest {
    function testAttackerDeauthorizesSetConsumedAgent() public {
        address victim          = makeAddr("victim");
        address setConsumedAgent = makeAddr("setConsumedAgent");
        address attacker        = makeAddr("attacker");
        bytes32 group           = bytes32(uint256(1));

        // Victim authorizes both agents
        vm.startPrank(victim);
        midnight.setIsAuthorized(setConsumedAgent, true, victim);
        midnight.setIsAuthorized(attacker,         true, victim);
        vm.stopPrank();

        // Sanity: setConsumedAgent can cancel offers
        vm.prank(setConsumedAgent);
        midnight.setConsumed(group, type(uint256).max, victim);

        // Reset consumed for the test
        // (consumed can only increase, so use a fresh group)
        bytes32 group2 = bytes32(uint256(2));

        // Attacker deauthorizes setConsumedAgent — single tx, no special privilege
        vm.prank(attacker);
        midnight.setIsAuthorized(setConsumedAgent, false, victim);

        assertFalse(midnight.isAuthorized(victim, setConsumedAgent));

        // setConsumedAgent can no longer cancel offers
        vm.prank(setConsumedAgent);
        vm.expectRevert(IMidnight.Unauthorized.selector);
        midnight.setConsumed(group2, type(uint256).max, victim);
    }
}
```

Expected assertions: `assertFalse(midnight.isAuthorized(victim, setConsumedAgent))` passes; the final `setConsumed` call reverts with `Unauthorized`.
