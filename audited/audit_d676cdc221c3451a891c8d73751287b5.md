Audit Report

## Title
Zero-rounding in `maxAssets` branch enables unbounded offer replay with unbacked credit minting — (`src/Midnight.sol`, `src/libraries/UtilsLib.sol`)

## Summary
When a buy offer uses `maxAssets > 0` and the taker supplies `units` small enough that `units * buyerPrice < WAD`, `mulDivDown` returns zero and `consumed` is incremented by zero. The cap check `require(newConsumed <= offer.maxAssets)` passes unchanged on every call, so the offer is never exhausted. The maker accrues `units` of credit and the taker accrues `units` of debt with zero token transfers, creating unbacked credit that can be redeemed at maturity to drain legitimate depositors' funds.

## Finding Description

**Root cause:** `mulDivDown(units, buyerPrice, WAD)` computes `(units * buyerPrice) / WAD`. When `units * buyerPrice < WAD`, the result is 0.

`src/Midnight.sol` line 363 — `buyerAssets` rounds to zero for small `units`: [2](#0-1)

`src/Midnight.sol` lines 382–384 — `buyerCreditIncrease` and `sellerDebtIncrease` are computed from `units` (non-zero), not `buyerAssets` (zero): [4](#0-3)

**No `units > 0` or `buyerAssets > 0` guard exists.** A grep for `require.*units` or `require.*buyerAssets` in `src/Midnight.sol` returns no matches. The only units-related require is `require(newConsumed <= offer.maxUnits)` in the `maxUnits` branch (line 372), which is immune because it increments by `units` directly. [7](#0-6) 

**Tick price range:** `PRICE_ROUNDING_STEP = 1e12` is the minimum representable non-zero price, occurring at tick 2 (not tick 0 — `tickToPrice(0) == 0` per the test suite). At tick 2, `offerPrice = 1e12`. With zero settlement fee, `buyerPrice = 1e12`, so any `units < 1e6` satisfies `units * 1e12 < 1e18 = WAD`, making `buyerAssets = 0`. [8](#0-7) [9](#0-8) 

**Exploit flow:**
1. Attacker controls address A (maker) and address B (taker). `require(offer.maker != taker)` at line 354 is satisfied.
2. A posts a buy offer with `maxAssets = M > 0` at tick 2 (`offerPrice = 1e12`).
3. B calls `take(offer, ..., units = 999_999)` where `999_999 * 1e12 = 999_999e12 < 1e18`.
4. `buyerAssets = mulDivDown(999_999, 1e12, 1e18) = 0`. `sellerAssets = 0`.
5. `consumed += 0` → unchanged; `require(consumed <= M)` passes.
6. A's credit increases by `999_999`; B's debt increases by `999_999`.
7. Both token transfers execute for 0 tokens.
8. Steps 3–7 repeat indefinitely; `consumed` never reaches `M`.
9. A redeems accumulated credit at maturity via `withdraw`, draining tokens deposited by legitimate users.

**Why existing checks fail:**
- `require(newConsumed <= offer.maxAssets)` enforces an upper bound but does not require `consumed` to have increased.
- There is no `require(units == 0 || buyerAssets > 0)` guard.
- The `maxUnits` branch uses `+= units` and is immune; only the `maxAssets` branch is affected.

## Impact Explanation
Each iteration grants the maker `units` of credit with zero token payment and burdens the taker with `units` of debt with zero token receipt. Repeated indefinitely, the maker accumulates unbounded credit not backed by any token deposit. When the maker calls `withdraw` at maturity, the protocol pays out tokens deposited by legitimate users, directly draining the contract's loan-token balance. This is direct theft of funds / protocol insolvency. [10](#0-9) 

## Likelihood Explanation
**Preconditions:** A buy offer with `maxAssets > 0` must exist at any tick where `offerPrice < WAD` (true for all valid ticks). The attacker needs two addresses (trivially created). No admin access, oracle manipulation, or victim cooperation is required. The required `units` value is computed off-chain as any value satisfying `units * offerPrice < WAD`. **Repeatability:** Unlimited — `consumed` never advances, so the offer never closes.

## Recommendation
Add a guard in the `maxAssets` branch requiring that `buyerAssets` (or `sellerAssets` for sell offers) is non-zero when `units > 0`:

```solidity
if (offer.maxAssets > 0) {
    uint256 assetsDelta = offer.buy ? buyerAssets : sellerAssets;
    require(units == 0 || assetsDelta > 0, ZeroAssetsTaken());
    newConsumed = consumed[offer.maker][offer.group] += assetsDelta;
    require(newConsumed <= offer.maxAssets, ConsumedAssets());
}
```

Alternatively, enforce a protocol-wide minimum: `require(units == 0 || buyerAssets > 0)` before the branch, or require `units >= WAD / offerPrice` at the call site.

function testUnboundedReplay() public {

    uint256 tick = 2;

    Offer memory offer;
    offer.buy = true;

    // lender has 999_999 * 1000 = 999_999_000 credit, zero tokens deposited
    assertEq(midnight.creditOf(id, lender), 999_999 * 1000);
    // consumed is still 0
    assertEq(midnight.consumed(lender, offer.group), 0);
}
