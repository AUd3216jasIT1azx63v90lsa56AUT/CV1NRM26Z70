### Title
Authorized Operator Can Consume Victim's EcrecoverAuthorizer Nonces via No-Op Self-Grant - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` increments `nonce[authorization.authorizer]` unconditionally before verifying the downstream `Midnight.setIsAuthorized` call produces any net state change. Because `Midnight` explicitly permits no-op writes to `isAuthorized` (documented at line 168: "No-ops are allowed"), an operator already authorized by the victim can sign a self-grant `Authorization{authorizer=victim, authorized=operator, isAuthorized=true}`, submit it through `EcrecoverAuthorizer`, consume victim's nonce, and leave `isAuthorized[victim][operator]` unchanged. This can be repeated indefinitely, invalidating all of victim's pre-signed authorization messages.

### Finding Description

**Exact code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 25–47):

```
require(block.timestamp <= authorization.deadline, Expired());
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce()); // nonce consumed here
...
require(
    signer == authorization.authorizer
    || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer), // passes if operator is authorized
    Unauthorized()
);
...
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

`Midnight.setIsAuthorized` (lines 731–735):

```
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
isAuthorized[onBehalf][authorized] = newIsAuthorized; // unconditional write, no-op if already equal
```

**Attacker-controlled inputs:**

- `authorization.authorizer = victim`
- `authorization.authorized = operator` (attacker's own address)
- `authorization.isAuthorized = true`
- `authorization.nonce = nonce[victim]` (current nonce, publicly readable)
- `authorization.deadline` = any future timestamp
- `signature` = operator's own ECDSA signature over the above struct

**Why the authorization check passes:**

The signer is `operator`. The check `signer == authorization.authorizer` evaluates to `operator == victim` → false. The fallback `IMidnight(MIDNIGHT).isAuthorized(victim, operator)` evaluates to `true` by precondition. The check passes.

**Why the downstream call is a no-op:**

`Midnight.setIsAuthorized(operator, true, victim)` is called with `msg.sender = address(ecrecoverAuthorizer)`. The authorization check passes because `isAuthorized[victim][ecrecoverAuthorizer] == true` (precondition). The write `isAuthorized[victim][operator] = true` is a no-op because it is already `true`.

**Net result:** `nonce[victim]` advances from N to N+1; `isAuthorized[victim][operator]` is unchanged. Any message victim pre-signed at nonce N (e.g., `{authorizer=victim, authorized=operator, isAuthorized=false}` to revoke the operator) is now permanently invalid. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation

An operator already authorized by the victim (`isAuthorized[victim][operator] == true`) and where the victim has also authorized `ecrecoverAuthorizer` (`isAuthorized[victim][ecrecoverAuthorizer] == true`) can:

1. Consume victim's `EcrecoverAuthorizer` nonces at will, one per transaction, with no gas cost to victim and no state change.
2. Invalidate any pre-signed `Authorization` messages victim has distributed (e.g., to a relayer or keeper) at those nonces.
3. Specifically, prevent victim from revoking the operator's own access via a pre-signed revocation message, since the operator can front-run or race-consume the nonce before the revocation is submitted.

The victim cannot recover pre-signed messages at consumed nonces; they must re-sign at the new nonce, which the operator can again consume. [4](#0-3) [5](#0-4) 

### Likelihood Explanation

**Preconditions:**

- `isAuthorized[victim][operator] == true`: victim has authorized operator — the normal intended use of the protocol.
- `isAuthorized[victim][address(ecrecoverAuthorizer)] == true`: victim has authorized the EcrecoverAuthorizer to act on their behalf — required for any off-chain signed authorization flow to work.

Both preconditions are the standard setup for any user who wants to use `EcrecoverAuthorizer` with an operator. They are not edge cases.

**Feasibility:** The operator needs only their own private key and knowledge of the current `nonce[victim]` (public storage). No tokens, collateral, or special roles are required.

**Repeatability:** The attack can be repeated for every nonce increment. The operator can sustain the grief indefinitely as long as `isAuthorized[victim][operator]` remains true — which the operator can also prevent from being revoked via this same mechanism. [6](#0-5) [7](#0-6) 

### Recommendation

Add a check in `EcrecoverAuthorizer.setIsAuthorized` that the requested authorization state differs from the current on-chain state before consuming the nonce:

```solidity
bool currentState = IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, authorization.authorized);
require(currentState != authorization.isAuthorized, NoOpAuthorization());
```

This ensures every nonce consumption corresponds to a net state change. Alternatively, restrict the signer to be strictly `authorization.authorizer` (i.e., remove the delegate-signer path), but this would break the intended use case of authorized operators managing authorizations on behalf of users. [1](#0-0) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract NopNonceGriefTest is BaseTest {
    function testOperatorConsumesVictimNonceWithNoOp() public {
        address victim = makeAddr("victim");
        uint256 operatorKey = 0xBEEF;
        address operator = vm.addr(operatorKey);

        // Setup: victim authorizes ecrecoverAuthorizer and operator in Midnight
        vm.prank(victim);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
        vm.prank(victim);
        midnight.setIsAuthorized(operator, true, victim);

        // Victim pre-signs a revocation at nonce 0
        Authorization memory victimRevoke = Authorization({
            authorizer: victim,
            authorized: operator,
            isAuthorized: false,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        // (victim's signed revocation is distributed to a relayer but not yet submitted)

        // Operator signs a self-grant (no-op) at nonce 0
        Authorization memory selfGrant = Authorization({
            authorizer: victim,
            authorized: operator,
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        bytes32 digest = ecrecoverAuthorizer.hashAuthorization(selfGrant); // helper
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(operatorKey, digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Operator submits the no-op self-grant
        ecrecoverAuthorizer.setIsAuthorized(selfGrant, sig);

        // Assert: nonce advanced but authorization unchanged
        assertEq(ecrecoverAuthorizer.nonce(victim), 1);
        assertEq(midnight.isAuthorized(victim, operator), true); // unchanged

        // Assert: victim's pre-signed revocation at nonce 0 is now invalid
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimRevoke, signAuthorization(victimRevoke, victim));
    }
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(victim) == 1` after the no-op submission
- `midnight.isAuthorized(victim, operator) == true` (unchanged)
- Submitting victim's pre-signed revocation at nonce 0 reverts with `InvalidNonce` [8](#0-7) [9](#0-8)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
