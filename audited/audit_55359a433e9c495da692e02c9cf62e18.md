### Title
Oracle Price Manipulated via `onSell` Callback Allows Undercollateralized Borrowing - (File: src/Midnight.sol)

### Summary
In `take()`, the seller's debt is increased before the `onSell` callback fires, and the health check (`isHealthy`) runs after the callback. Because `IOracle.price()` is called via `staticcall` inside `isHealthy`, it cannot write state, but it can freely read external state that the `onSell` callback already wrote. An attacker who controls both the oracle (by creating a market with a malicious oracle) and the sell callback can inflate the oracle price during the callback, causing `isHealthy` to pass with a price that does not reflect the real collateral value, leaving the seller with undercollateralized debt.

### Finding Description
**Exact code path:**

1. `take()` is called (`src/Midnight.sol:337`).
2. `sellerPos.debt += sellerDebtIncrease` (`src/Midnight.sol:414`) — seller's debt is committed to storage.
3. Liquidation lock is set for the seller (`src/Midnight.sol:444`).
4. `onSell` callback fires (`src/Midnight.sol:458–474`) — this is a regular `CALL`, not a `staticcall`, so the callback can write to any external contract.
5. Liquidation lock is released (`src/Midnight.sol:475`).
6. `require(liquidationLocked(id, seller) || isHealthy(offer.market, id, seller))` (`src/Midnight.sol:476`).
7. Inside `isHealthy`, `IOracle(collateralParam.oracle).price()` is called via `staticcall` (`src/Midnight.sol:953`). The oracle reads from a `StateContract` whose storage was just written by the callback in step 4, returning an inflated price.
8. `maxDebt` is computed with the inflated price and exceeds the seller's actual debt — health check passes.

**Attacker-controlled inputs:**
- Market creation: anyone can call `touchMarket` with a `CollateralParams.oracle` pointing to a malicious oracle that reads from an attacker-controlled `StateContract`.
- Sell callback: the seller controls `offer.callback` (if `offer.buy == false`, seller = `offer.maker`) or `takerCallback` (if `offer.buy == true`, seller = `taker`).

**Why existing checks fail:**
- The liquidation lock only prevents liquidation during callbacks; it does not prevent oracle price manipulation.
- `staticcall` on `price()` prevents the oracle from writing state during the health check, but the callback already wrote the manipulated value before `isHealthy` is called.
- No snapshot of the oracle price is taken before the callback to compare against.
- The Certora formal verification explicitly assumes `price()` is constant during execution and does not cover this case.

### Impact Explanation
The seller can borrow an arbitrary number of units (`sellerDebtIncrease = units`) backed by a negligible amount of collateral (e.g., 1 wei). After `take()` completes, the seller's position is undercollateralized at the true oracle price. The seller can then withdraw their collateral (since `withdrawCollateral` also calls `isHealthy` with the same manipulable oracle), leaving the protocol with bad debt that is socialized to lenders. This directly violates the invariant that collateral cannot be withdrawn or seized outside health/liquidation rules, and that every credit has matching debt or valid settled/loss state.

### Likelihood Explanation
Preconditions are fully attacker-controlled: the attacker deploys a `StateContract`, a malicious oracle reading from it, and a malicious `onSell` callback writing to it, then creates a market using these. No privileged access is required. The attack is repeatable across any number of takes and is not blocked by any on-chain guard. The only cost is gas and a small collateral deposit.

### Recommendation
Snapshot the oracle price(s) for the seller's activated collaterals **before** any callback executes, and re-read them **after** the callback. If any price differs, revert. Alternatively, call `isHealthy` (or a price-snapshot equivalent) immediately before the `onSell` callback and require the same result after. A simpler mitigation is to call `isHealthy` **before** the `onSell` callback fires (after debt is updated but before the callback), so the oracle is read in a context where no callback has yet had the opportunity to manipulate external state.

contract StateContract {
    uint256 public value;
    function set(uint256 v) external { value = v; }
}

contract MaliciousOracle {
    StateContract public s;
    constructor(StateContract _s) { s = _s; }
    function price() external view returns (uint256) { return s.value(); }
}

contract MaliciousCallback is ISellCallback {
    StateContract public s;
    uint256 public inflatedPrice;
    constructor(StateContract _s, uint256 _p) { s = _s; inflatedPrice = _p; }
    function onSell(...) external returns (bytes32) {
        s.set(inflatedPrice); // write inflated price before isHealthy is called
        return CALLBACK_SUCCESS;
    }
}

function testOraclePriceManipulatedViaOnSell() public {
    // 1. Deploy StateContract, MaliciousOracle, MaliciousCallback
    // 2. Set StateContract.value = REAL_LOW_PRICE (e.g., 1)
    // 3. Create market with MaliciousOracle as oracle
    // 4. Attacker supplies 1 wei of collateral
    // 5. Attacker creates sell offer with MaliciousCallback (inflatedPrice = type(uint128).max)
    // 6. Call take() with units = LARGE_AMOUNT
    // 7. Assert: take() does NOT revert (health check passes with inflated price)
    // 8. Assert: debtOf(id, seller) == LARGE_AMOUNT
    // 9. Assert: collateral(id, seller, 0) == 1 (only 1 wei of collateral)
    // 10. Assert: isHealthy(market, id, seller) == false when StateContract.value is reset to REAL_LOW_PRICE
    //     (proving the position is undercollateralized at the real price)
}

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
