Audit Report

## Title
Missing oracle address validation in `touchMarket` enables permanent liquidation DoS - (File: src/Midnight.sol)

## Summary
The `touchMarket` function validates `token`, `lltv`, and `maxLif` for each collateral slot but imposes no constraint on the `oracle` field. An unprivileged market creator can supply `oracle = address(0)` for any collateral slot. Once a borrower activates that slot, every subsequent call to `liquidate` unconditionally invokes `IOracle(address(0)).price()`, which causes an ABI-decoding revert, permanently freezing lender funds in an unliquidatable position.

## Finding Description

**Root cause â€” `touchMarket` validation loop:**

The loop at `src/Midnight.sol` lines 762â€“772 enforces token ordering, LLTV allowance, and `maxLif` validity, but contains no check on the `oracle` field: [1](#0-0) 

There is no `require(market.collateralParams[i].oracle != address(0))`. The `CollateralParams` struct explicitly includes an `oracle` field: [2](#0-1) 

**Trigger â€” `liquidate` bitmap loop:**

The oracle call at line 610 is unconditional inside the collateral bitmap loop: [3](#0-2) 

When `_collateralParam.oracle == address(0)`, the EVM CALL to `address(0)` returns success with 0 bytes of returndata. Solidity 0.8.x's ABI decoder for `uint256` requires 32 bytes and reverts on empty returndata, causing `liquidate` to always revert for any position that has bit `i` set in `collateralBitmap`.

**Market ID locks in the zero oracle:**

The market ID is derived from a full hash of the market struct (`toId(market)`). A liquidator must pass the exact market struct used at creation time and cannot substitute a valid oracle address without producing a different (uncreated) market ID, making the DoS permanent.

**Certora invariant gap:**

The `createdMarketsHaveNonZeroCollaterals` invariant only proves `token != 0`, not `oracle != 0`: [4](#0-3) 

**Exploit flow:**
1. Attacker calls `touchMarket` with `collateralParams[i].oracle = address(0)` â€” passes all validation.
2. Borrower supplies collateral at index `i` and takes a loan; bit `i` is set in `collateralBitmap`.
3. Position becomes unhealthy (price drop or post-maturity).
4. Any liquidator calls `liquidate` with the correct market struct.
5. The loop reaches bit `i`, executes `IOracle(address(0)).price()`.
6. `address(0)` has no code; CALL returns 0 bytes; ABI decoder reverts.
7. `liquidate` always reverts; the unhealthy position can never be liquidated.

## Impact Explanation
Lenders' funds are frozen in a permanently unliquidatable position. Bad debt can never be realized through `liquidate`. Both "permanent or long-term fund freeze" and "liquidation bypass" are explicitly listed as `best_bug_classes` in `live_context.json`. [5](#0-4) 

This is not a "pure external oracle failure" (which is excluded) â€” it is a protocol-level validation gap that allows a structurally invalid market to be created and permanently traps lender funds.

## Likelihood Explanation
Market creation is fully permissionless â€” "market creator" is listed as an unprivileged attacker role in `live_context.json`. [6](#0-5) 

The attacker only needs to call `touchMarket` with a crafted `CollateralParams` array containing `oracle = address(0)`. The condition is trivially reachable in a single transaction with no capital requirement. The DoS is permanent and repeatable across any number of markets.

## Recommendation
Add a non-zero oracle check inside the `touchMarket` validation loop:

```solidity
require(market.collateralParams[i].oracle != address(0), InvalidOracle());
```

This should be placed alongside the existing checks at lines 764â€“771 in `src/Midnight.sol`. Additionally, update the Certora invariant `createdMarketsHaveNonZeroCollaterals` in `certora/specs/CreatedMarkets.spec` to also assert `market.collateralParams[i].oracle != 0`. [1](#0-0) 

## Proof of Concept

```solidity
// Foundry test sketch
function test_zeroOracleLiquidationDoS() public {
    CollateralParams[] memory params = new CollateralParams[](1);
    params[0] = CollateralParams({
        token: address(collateralToken),
        lltv: 860e15,  // allowed LLTV tier
        maxLif: maxLif(860e15, LIQUIDATION_CURSOR_LOW),
        oracle: address(0)  // zero oracle â€” passes touchMarket validation
    });
    Market memory market = Market({
        loanToken: address(loanToken),
        collateralParams: params,
        maturity: block.timestamp + 30 days,
        rcfThreshold: 0,
        enterGate: address(0),
        liquidatorGate: address(0)
    });

    midnight.touchMarket(market);  // succeeds

    // Borrower supplies collateral and borrows
    vm.startPrank(borrower);
    midnight.supplyCollateral(market, 0, 1000e18, borrower);
    // ... take offer to create debt ...
    vm.stopPrank();

    // Warp past maturity to make position liquidatable
    vm.warp(market.maturity + 1);

    // Liquidation attempt always reverts
    vm.expectRevert();
    midnight.liquidate(market, 0, 0, 1, borrower, true, liquidator, address(0), "");
}
```

The test demonstrates that `touchMarket` accepts `oracle = address(0)`, a borrower can activate the slot, and every subsequent `liquidate` call reverts due to the ABI-decoding failure on the zero-address oracle call.
