### Title
Fee-on-Transfer Collateral Inflates `position.collateral` Beyond Actual Holdings, Enabling Undercollateralized Borrowing - (File: src/Midnight.sol)

### Summary
`supplyCollateral` records the caller-supplied `assets` parameter directly into `position.collateral[collateralIndex]` before calling `SafeTransferLib.safeTransferFrom`, which only validates the boolean return value and never measures the actual tokens received. When the collateral token charges a transfer fee, the protocol records `X` units of collateral while holding only `X*(1-fee%)`, and `isHealthy` computes `maxDebt` from the inflated recorded value, allowing a borrower to carry debt that exceeds the value of collateral actually held.

### Finding Description

**Exact code path:**

`supplyCollateral` (`src/Midnight.sol` lines 531–545):
```
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets); // line 533 — records input, not received
...
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets); // line 545 — only checks bool return
``` [1](#0-0) 

`SafeTransferLib.safeTransferFrom` (`src/libraries/SafeTransferLib.sol` lines 24–34) calls `transferFrom` and checks only the boolean return value; it performs no balance-before/balance-after measurement. [2](#0-1) 

`isHealthy` (`src/Midnight.sol` lines 944–959) reads `_position.collateral[i]` — the recorded (inflated) value — to compute `maxDebt`:
```
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
           .mulDivDown(collateralParam.lltv, WAD);
``` [3](#0-2) 

**Root cause:** State is written from the input parameter `assets`, not from the delta of the contract's actual token balance. There is no balance-before/balance-after guard anywhere in `supplyCollateral`.

**Attacker-controlled inputs:**
- Collateral token: a fee-on-transfer ERC20 (fee `f` in [1%, 50%])
- `assets = X` (any amount)

**Exploit flow:**
1. Market creation is permissionless (`touchMarket` has no restriction on token type). [4](#0-3) 
2. Attacker (as market creator) calls `touchMarket` with a fee-on-transfer ERC20 as `collateralParams[0].token`.
3. Attacker calls `supplyCollateral(market, 0, X, attacker)`. The contract records `position.collateral[0] = X` but receives only `X*(1-f)` tokens.
4. Attacker calls `take` as seller (borrower), borrowing up to `X * price * lltv / WAD` units of the loan token — computed from the inflated `X`.
5. `isHealthy` returns `true` because `maxDebt = X * price * lltv / WAD >= debt`, while the actual collateral backing is only `X*(1-f) * price * lltv / WAD`.

**Why existing checks fail:**
- `SafeTransferLib` only checks `success` and the ERC20 boolean return; it does not verify received quantity.
- `isHealthy` has no access to actual token balances; it reads only `position.collateral[i]`.
- `touchMarket` validates LLTV tiers, sorted tokens, and maturity bounds, but imposes no restriction on token transfer mechanics. [5](#0-4) 

### Impact Explanation
The borrower's recorded collateral exceeds the protocol's actual token holdings by a factor of `1/(1-f)`. `isHealthy` computes `maxDebt` on phantom collateral units never held, so the position is reported healthy while it is structurally undercollateralized. Upon default or attempted liquidation, the seized collateral is worth less than the debt, creating irrecoverable bad debt for lenders and violating the invariant that contract balances cover all collateral obligations.

### Likelihood Explanation
Market creation is permissionless and accepts any ERC20 address as collateral with no token-type validation. Fee-on-transfer tokens exist on mainnet (e.g., tokens with deflationary mechanics or explicit transfer taxes). The attacker needs only to deploy or reuse such a token, create a market, and supply collateral. The exploit is repeatable on every `supplyCollateral` call in any market using such a token. No privileged action is required after market creation.

### Recommendation
Replace the static `assets` write with a balance-delta measurement in `supplyCollateral`:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

This must be applied consistently to `repay` and `flashLoan` return paths if the loan token can also be fee-on-transfer. Alternatively, add an explicit invariant check in `touchMarket` that rejects tokens whose `transferFrom` delivers less than the requested amount (a one-time probe transfer), though the balance-delta approach is more robust.

pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams} from "src/interfaces/IMidnight.sol";
import {ORACLE_PRICE_SCALE, WAD} from "src/libraries/ConstantsLib.sol";

// Fee-on-transfer ERC20: charges `feeBps` basis points on every transferFrom
contract FeeToken is ERC20 {
    uint256 public feeBps; // e.g. 1000 = 10%
    constructor(uint256 _feeBps) { feeBps = _feeBps; }
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount * feeBps / 10_000;
        super.transferFrom(from, to, amount - fee); // only amount-fee arrives
        _burn(from, fee);                           // fee destroyed
        return true;
    }
}

contract FeeOnTransferCollateralTest is Test {
    Midnight midnight;
    FeeToken feeToken;   // collateral with 10% fee
    ERC20    loanToken;
    Oracle   oracle;
    Market   market;
    bytes32  id;
    address  borrower = makeAddr("borrower");

    function setUp() public {
        midnight  = new Midnight(...);
        feeToken  = new FeeToken(1000); // 10% fee
        loanToken = new ERC20();
        oracle    = new Oracle();
        oracle.setPrice(ORACLE_PRICE_SCALE); // 1:1 price

        market.loanToken = address(loanToken);
        market.maturity  = block.timestamp + 365 days;
        market.collateralParams.push(CollateralParams({
            token:  address(feeToken),
            lltv:   0.77e18,
            maxLif: ...,
            oracle: address(oracle)
        }));
        id = midnight.touchMarket(market);
    }

    function testFeeOnTransferInflatesCollateral(uint256 feeBps) public {
        feeBps = bound(feeBps, 100, 5000); // 1%–50%
        feeToken.setFeeBps(feeBps);

        uint256 X = 1e18;
        deal(address(feeToken), borrower, X);
        vm.startPrank(borrower);
        feeToken.approve(address(midnight), X);
        midnight.supplyCollateral(market, 0, X, borrower);

        // Recorded collateral = X (inflated)
        uint256 recorded = midnight.collateral(id, borrower, 0);
        assertEq(recorded, X, "recorded == X");

        // Actual balance = X*(1 - feeBps/10000)
        uint256 actual = feeToken.balanceOf(address(midnight));
        assertLt(actual, recorded, "actual < recorded: phantom collateral exists");

        // Borrow to the oracle-derived limit based on inflated recorded value
        uint256 maxBorrow = recorded * ORACLE_PRICE_SCALE / ORACLE_PRICE_SCALE * 0.77e18 / WAD;
        // ... set up lender offer and take maxBorrow units ...

        // isHealthy returns true even though actual collateral < required
        assertTrue(midnight.isHealthy(market, id, borrower), "healthy on phantom collateral");

        // Invariant violation: actual token balance < collateral value used for health check
        uint256 actualMaxDebt = actual * ORACLE_PRICE_SCALE / ORACLE_PRICE_SCALE * 0.77e18 / WAD;
        assertLt(actualMaxDebt, midnight.debtOf(id, borrower),
            "debt exceeds value of actually-held collateral");
    }
}
```

**Expected assertions that must hold for the bug to be confirmed:**
- `feeToken.balanceOf(address(midnight)) < midnight.collateral(id, borrower, 0)` — phantom collateral exists
- `midnight.isHealthy(market, id, borrower) == true` — position reported healthy
- `midnight.debtOf(id, borrower) > actual_collateral_value` — position is actually undercollateralized

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));

                revert(add(returndata, 0x20), mload(returndata))
