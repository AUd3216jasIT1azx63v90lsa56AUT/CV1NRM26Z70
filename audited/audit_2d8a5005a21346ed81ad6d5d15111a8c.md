### Title
Fee-Free Flash Loan Enables Same-Transaction Oracle Manipulation and Profitable Liquidation of Healthy Positions — (File: `src/Midnight.sol`)

---

### Summary

Midnight's built-in `flashLoan()` is fee-free and can simultaneously lend both collateral tokens (held from borrowers' deposits) and loan tokens (held from lenders' credit). Because `liquidate()` calls `IOracle.price()` with no TWAP, staleness, or manipulation guard, an attacker can borrow collateral via flash loan, crash a spot-price oracle in the same transaction, liquidate positions that are now artificially unhealthy, and repay the flash loan — all atomically and profitably.

---

### Finding Description

**Vulnerability class:** Pricing/oracle manipulation + flash-loan atomicity (direct analog to the ERD Tellor report).

**Root cause — three cooperating weaknesses:**

**1. Fee-free flash loan lends protocol-held collateral**

`flashLoan()` imposes zero fee and can lend any token Midnight holds, including collateral tokens deposited by borrowers via `supplyCollateral()`. [1](#0-0) 

The function accepts an array of tokens and amounts, so collateral tokens *and* loan tokens can be borrowed in a single call.

**2. `liquidate()` reads the oracle price with no manipulation protection**

Inside `liquidate()`, the price of every activated collateral is fetched directly from the oracle contract: [2](#0-1) 

The `IOracle` interface exposes only a bare `price()` view — no TWAP, no staleness window, no circuit-breaker: [3](#0-2) 

If that oracle is a DEX spot-price feed, the value returned is the instantaneous reserve ratio, which is trivially manipulable within a single block.

**3. Market creation imposes no oracle-quality constraint**

`touchMarket()` validates LLTV tiers and collateral ordering but performs no check on the oracle address: [4](#0-3) 

Any address — including a Uniswap V2 spot-price wrapper — is accepted.

**4. `liquidate()` sends collateral before pulling loan tokens (flash-liquidation pattern)** [5](#0-4) 

Collateral is transferred to `receiver` first; the `onLiquidate` callback fires next; loan tokens are pulled last. This lets the attacker sell the seized collateral inside the callback to fund the repayment — no external capital required.

---

**End-to-end exploit path (single transaction):**

```
1. attacker.attack() {
2.   midnight.flashLoan(
         [collateralToken, loanToken],
         [C_amount, L_amount],
         address(this), ""
       );
3. }
4. onFlashLoan(...) {
5.   // Dump C_amount collateral into the DEX pool → oracle price crashes P → P'
6.   dex.swap(collateralToken → loanToken, C_amount);
7.
8.   // Positions that were healthy at P are now unhealthy at P'
9.   midnight.liquidate(market, collatIdx, 0, repaidUnits,
                        victim, false, address(this), address(this), "");
10.  // onLiquidate fires: seized collateral already in attacker's hands
11.  // sell excess collateral to cover repaidUnits + flash loan repayment
12.
13.  // Approve Midnight to pull back C_amount collateral + L_amount loan token
14.  return CALLBACK_SUCCESS;
15. }
```

At the manipulated price `P'`:

```
seizedAssets = repaidUnits * lif / WAD * ORACLE_PRICE_SCALE / P'
```

Because `P' << P`, `seizedAssets` is far larger than the fair-value equivalent of `repaidUnits`. The attacker keeps the spread after repaying the flash loan.

---

### Impact Explanation

- **Direct theft of collateral** from borrowers whose positions are liquidated at an artificially depressed price.
- The liquidation incentive factor (`maxLif`) amplifies the over-seizure: at LLTV = 0.86 with `LIQUIDATION_CURSOR_HIGH = 0.5`, `maxLif ≈ 1.078`. A 20 % oracle depression turns a healthy position into one where the attacker seizes ~28 % more collateral than the debt is worth at true prices.
- Bad debt may be socialized to lenders via the `lossFactor` mechanism if the borrower's remaining collateral is insufficient after the manipulated liquidation. [6](#0-5) 

---

### Likelihood Explanation

- **Preconditions (no privilege required):** A market must exist that uses a DEX spot-price oracle (Uniswap V2/V3 instantaneous price, Curve spot, etc.). Such oracles are common in permissionless lending deployments.
- **Capital requirement:** The attacker needs enough capital to move the DEX pool price sufficiently. For low-liquidity pools this can be modest. Midnight's own flash loan supplies the collateral tokens for free.
- **Atomicity:** The entire attack is one transaction — no multi-block timing risk.
- **Realistic:** This exact attack class has been executed on-chain against Cream Finance, Mango Markets, and others using the same flash-loan + spot-oracle pattern.

---

### Recommendation

1. **Enforce TWAP oracles at the protocol level.** Add a `minTwapWindow` field to `CollateralParams` and require the oracle to expose a `twapPrice(uint32 window)` function. Reject markets where the oracle cannot prove a time-weighted price.

2. **Add a flash-loan fee.** Even a 0.09 % fee makes oracle manipulation unprofitable for most pool depths.

3. **Add a per-block price-change guard in `liquidate()`.** Cache the oracle price at the start of the block (via `tstore`/`tload`) and revert if the current price deviates by more than a configurable threshold (e.g., 5 %) within the same block.

4. **Document the oracle risk prominently.** If TWAP enforcement is deferred, require market creators to attest that their oracle is manipulation-resistant, and surface this in the UI.

### Proof of Concept

**Setup:**
- Market: `loanToken = USDC`, `collateralToken = WETH`, oracle = Uniswap V2 WETH/USDC spot price wrapper.
- Victim borrower: 10 WETH collateral, 15 000 USDC debt, LLTV = 0.86 → healthy at true price $2 000/ETH (maxDebt = 17 200 USDC > 15 000).
- Midnight holds 10 WETH (from victim) and 15 000 USDC (from lenders).

**Attack (one tx):**

```solidity
// Step 1 – borrow 10 WETH + 15 000 USDC from Midnight for free
midnight.flashLoan([WETH, USDC], [10e18, 15_000e6], address(this), "");

// onFlashLoan:
// Step 2 – dump 10 WETH into Uniswap V2 pool
//   Pool: 100 WETH / 200 000 USDC → after swap: 110 WETH / ~181 818 USDC
//   New spot price: 181 818 / 110 ≈ $1 653/ETH  (−17.4 %)
uniV2.swap(10e18 WETH → USDC);

// Step 3 – victim is now unhealthy: maxDebt = 10 * 1653 * 0.86 = 14 215 < 15 000
// Step 4 – liquidate: repay 15 000 USDC, seize all 10 WETH
midnight.liquidate(market, 0, 0, 15_000e6, victim, false, address(this), address(this), "");
// onLiquidate: attacker holds 10 WETH seized

// Step 5 – sell 8.2 WETH back into pool to recover 15 000 USDC for repayment
uniV2.swap(8.2e18 WETH → 15_000 USDC);

// Step 6 – repay flash loan: return 10 WETH + 15 000 USDC to Midnight
WETH.approve(midnight, 10e18);
USDC.approve(midnight, 15_000e6);
return CALLBACK_SUCCESS;

// Net profit: 10 − 8.2 = 1.8 WETH ≈ $3 600 at true price, risk-free
```

The victim loses their entire 10 WETH collateral while their 15 000 USDC debt is repaid at the manipulated price. The attacker profits ~1.8 WETH in a single atomic transaction with no upfront capital.
