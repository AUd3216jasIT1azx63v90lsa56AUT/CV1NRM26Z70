### Title
Fee-on-Transfer Collateral Token Inflates Recorded Collateral Beyond Actual Balance - (File: src/Midnight.sol)

### Summary

`supplyCollateral` credits `_position.collateral[collateralIndex]` with the full caller-supplied `assets` value before calling `SafeTransferLib.safeTransferFrom`, and neither the accounting step nor the transfer step measures the actual tokens received. When a fee-on-transfer token is used as collateral, Midnight records `assets` but receives only `assets * (1 - fee)`, creating phantom collateral that inflates the borrower's health check and allows undercollateralized borrowing.

### Finding Description

**Exact code path** — `src/Midnight.sol` lines 524–546:

```solidity
// line 533: records full `assets` unconditionally
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);

// line 545: transfer may deliver less than `assets` if token charges a fee
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
``` [1](#0-0) 

`SafeTransferLib.safeTransferFrom` only checks that the call did not revert and that the return value (if any) is `true`; it does not compare pre/post balances. [2](#0-1) 

**Market creation has no token-type restriction.** `touchMarket` validates LLTV tiers, sorted collateral addresses, and `maxLif`, but imposes no constraint on token transfer mechanics. [3](#0-2) 

**`isHealthy` uses the recorded (inflated) collateral value directly:**

```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [4](#0-3) 

**Exploit flow:**
1. Attacker deploys a fee-on-transfer ERC20 (e.g., 10% fee) and creates a permissionless Midnight market with it as collateral.
2. Attacker calls `supplyCollateral(market, 0, 1000e18, attacker)`.
3. `_position.collateral[0]` is set to `1000e18`; Midnight receives only `900e18`.
4. `isHealthy` computes `maxDebt` using `1000e18`, allowing the attacker to borrow up to `1000e18 * price * lltv / WAD`.
5. Actual collateral backing is only `900e18 * price * lltv / WAD` — the position is immediately undercollateralized.
6. On liquidation, the protocol attempts to seize up to `1000e18` tokens but only holds `900e18`, causing a shortfall and bad debt.

**Why existing checks fail:** The Certora formal verification explicitly models `safeTransferFrom` as `NONDET` and states *"ERC20 tokens are assumed well-behaved"*, so the solvency invariant (`Solvency.spec`) is not verified for fee-on-transfer tokens. [5](#0-4) 

### Impact Explanation

The invariant `sum(position[id][user].collateral[i] for all users) <= Midnight.balanceOf(collateralToken)` is violated. The phantom collateral allows the attacker to open debt positions that exceed the actual token balance held by the contract. When the position is liquidated, the seized amount cannot be fully transferred out, leaving bad debt that is socialized across lenders via the loss-factor mechanism.

### Likelihood Explanation

Market creation is permissionless — any address can deploy a fee-on-transfer token and create a market. The attacker only needs to be the `onBehalf` address (or authorized for it). The attack is repeatable: each `supplyCollateral` call inflates collateral by `fee * assets`. Tokens with transfer fees exist in production (e.g., USDT with fee enabled, STA, PAXG). No admin action is required.

### Recommendation

Record the actual received amount by measuring the balance delta around the transfer:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

Move the state update **after** the transfer and use `received` instead of `assets`. Alternatively, document that fee-on-transfer tokens are unsupported and add a market-creation guard (e.g., a round-trip balance check in `touchMarket`).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight, Market, CollateralParams} from "src/Midnight.sol";

contract FeeOnTransferToken is ERC20 {
    uint256 public constant FEE_BPS = 1000; // 10%
    constructor() ERC20("FOT", "FOT") { _mint(msg.sender, 1e30); }
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount * FEE_BPS / 10000;
        super.transferFrom(from, to, amount - fee); // only amount*(1-fee) arrives
        return true;
    }
}

contract FeeOnTransferCollateralTest is Test {
    Midnight midnight;
    FeeOnTransferToken fot;
    Market market;

    function setUp() public {
        midnight = new Midnight(/* ... */);
        fot = new FeeOnTransferToken();
        // build market with fot as collateral, valid lltv/maxLif/oracle
        // ...
        midnight.touchMarket(market);
    }

    function testPhantomCollateral() public {
        uint256 assets = 1000e18;
        fot.approve(address(midnight), assets);
        midnight.supplyCollateral(market, 0, assets, address(this));

        bytes32 id = midnight.toId(market);
        uint256 recorded = midnight.collateral(id, address(this), 0);
        uint256 actual = fot.balanceOf(address(midnight));

        // KEY ASSERTIONS:
        assertEq(recorded, assets);           // 1000e18 recorded
        assertEq(actual, assets * 9000 / 10000); // only 900e18 received
        assertGt(recorded, actual);           // phantom collateral confirmed
        // Further: open max debt, assert position is immediately undercollateralized
    }
}
```

Expected: `recorded (1000e18) > actual (900e18)`, confirming the invariant violation. A follow-up fuzz test can assert that for any fee rate `f ∈ (0, 1)`, `sum(collateral) > balanceOf(token)` after a single `supplyCollateral` call.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];

                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
