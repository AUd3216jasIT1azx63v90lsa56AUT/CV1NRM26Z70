Audit Report

## Title
Unconditional zero-value `safeTransferFrom` in `take()` DoS's markets with zero settlement fee when loan token reverts on zero transfers - (`File: src/Midnight.sol`)

## Summary
When all settlement fee breakpoints are zero (the default for any unconfigured loan token), `buyerAssets - sellerAssets` evaluates to exactly `0`. `take()` unconditionally calls `SafeTransferLib.safeTransferFrom(loanToken, payer, address(this), 0)` at line 455 with no zero-value guard. Loan tokens that revert on zero-value `transferFrom` calls will cause every `take()` in that market to revert, permanently blocking valid takers.

## Finding Description

**Root cause — price arithmetic when fee is zero:**

In `src/Midnight.sol` lines 360–364: [4](#0-3) 

When `_settlementFee == 0`, `buyerPrice == sellerPrice`. For a buy offer, both `buyerAssets` and `sellerAssets` use `mulDivDown` with the same price and units, so `buyerAssets - sellerAssets == 0` exactly.

**Unconditional transfer at line 455:** [1](#0-0) 

There is no `if (buyerAssets > sellerAssets)` guard before this call.

**`SafeTransferLib.safeTransferFrom` has no zero-value guard:** [2](#0-1) 

It always issues the low-level `token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)))` regardless of `value`.

**Zero settlement fee is the default state:**

`touchMarket` copies `defaultSettlementFeeCbp[market.loanToken]` into the market state: [3](#0-2) 

For any loan token not explicitly configured by the fee setter, `defaultSettlementFeeCbp` is a mapping that defaults to `0` for all 7 indices. This means every newly created market for an unconfigured token starts with all-zero settlement fees.

**Exploit flow:**
1. Deploy or use a loan token that reverts on zero-value `transferFrom` (e.g., LEND, some rebasing tokens — a known real-world pattern).
2. Create a market using that token without pre-configuring `defaultSettlementFeeCbp` (the default initial state).
3. Call `take()` — execution reaches line 455, `safeTransferFrom(..., 0)` is issued, the token reverts, the entire `take()` reverts.
4. The condition is fully repeatable: every `take()` call in the affected market will revert until the fee setter raises the settlement fee above zero.

**Why existing checks fail:**
The `claimableSettlementFee` accounting at line 418 silently handles the zero case (`+= 0`), but the transfer at line 455 is not similarly guarded: [5](#0-4) 

## Impact Explanation
Any market whose loan token reverts on zero-value transfers is permanently DoS'd for `take()` whenever the settlement fee is zero. All state changes made before line 455 (position updates, consumed tracking, credit/debt accounting) are reverted, so no funds are lost — but valid takes are completely blocked. This is a fund-freeze / liveness failure for affected markets.

## Likelihood Explanation
- Zero settlement fee is the **default** state: `touchMarket` copies `defaultSettlementFeeCbp[loanToken]`, which is `0` for any token not explicitly configured by the fee setter.
- Tokens that revert on zero-value transfers exist in production. The protocol places no restriction on which ERC-20 can be a loan token.
- The condition is fully repeatable: every `take()` call in the affected market will revert until the settlement fee is raised above zero.
- No privileged action by the attacker is required; any taker triggers it.

## Recommendation
Add a zero-value guard before the settlement fee transfer at line 455:

```solidity
uint256 settlementFeeAmount = buyerAssets - sellerAssets;
if (settlementFeeAmount > 0) {
    SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), settlementFeeAmount);
}
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
```

This mirrors how `claimableSettlementFee` already handles the zero case silently at line 418.

## Proof of Concept
1. Deploy a mock ERC-20 loan token that reverts when `transferFrom` is called with `amount == 0`.
2. Do **not** call `setDefaultSettlementFee` for this token (leave all cbps at 0).
3. Call `touchMarket` with this token — all `settlementFeeCbp0..6` will be 0.
4. Place a buy offer and call `take()` with any nonzero `units`.
5. Observe that execution reaches line 455, `safeTransferFrom(..., 0)` is called, the token reverts, and `take()` reverts.
6. Confirm that raising the settlement fee above zero via `setMarketSettlementFee` unblocks `take()`.

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
