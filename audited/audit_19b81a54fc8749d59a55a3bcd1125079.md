### Title
`withdraw()` Missing `whenPoolNotPaused` Guard Allows Stake Withdrawal During Emergency Pause - (File: src/ConfidencePool.sol)

### Summary
`ConfidencePool.withdraw()` does not apply the `whenPoolNotPaused` modifier, while both `stake()` and `contributeBonus()` do. When the pool owner pauses the contract in response to a discovered issue, any staker can still call `withdraw()` and exit with their full principal, defeating the emergency pause.

### Finding Description
`ConfidencePool` inherits OpenZeppelin `Pausable` and defines a `whenPoolNotPaused` modifier that reverts with `PoolPaused` when the contract is paused. [1](#0-0) 

This modifier is applied to both deposit-side entry points: [2](#0-1) [3](#0-2) 

However, `withdraw()` carries no such guard: [4](#0-3) 

Its only guards are the outcome check, the live registry-state check, and `nonReentrant` — none of which consult the paused flag. A paused pool therefore blocks new capital from entering while leaving the exit path fully open.

### Impact Explanation
The `pause()` function exists precisely to give the owner an emergency stop over pool activity. If the owner pauses the pool to contain a discovered defect in the withdrawal path (e.g., an accounting inconsistency in `sumStakeTime`/`sumStakeTimeSq` manipulation, or a token-interaction edge case), every staker can still call `withdraw()` and drain their principal before the owner can remediate. The pause provides no protection over the one function that moves the largest volume of funds out of the contract.

### Likelihood Explanation
The owner must first invoke `pause()` for the gap to matter, which requires an emergency scenario. However, the entire value of a pause mechanism is that it works unconditionally when invoked. The asymmetry — deposits blocked, withdrawals open — is not documented anywhere in `docs/DESIGN.md` or the README as an intentional design choice, making this an implementation omission rather than an accepted trade-off.

### Recommendation
Add `whenPoolNotPaused` to `withdraw()`:

```solidity
function withdraw() external nonReentrant whenPoolNotPaused {
```

This mirrors the guard already present on `stake()` and `contributeBonus()` and makes the pause semantics uniform across all fund-movement entry points.

### Proof of Concept
1. Staker calls `stake(amount)` — succeeds; `eligibleStake[staker] = amount`.
2. Owner discovers an issue and calls `pause()`.
3. Staker calls `stake(amount)` again — reverts with `PoolPaused` ✓.
4. Staker calls `withdraw()` — **succeeds**; `stakeToken.safeTransfer(staker, amount)` executes despite the pool being paused.
5. The emergency pause has no effect on the withdrawal path. [5](#0-4)

    function withdraw() external nonReentrant {
        if (outcome != PoolStates.Outcome.UNRESOLVED) revert OutcomeAlreadySet();
        IAttackRegistry.ContractState state = _observePoolState();
        // `riskWindowStart` is the pool's one-way record that risk has materialised;
        // gate on it so an upstream registry rewind cannot re-open withdrawals.
        if (
            riskWindowStart != 0
                || (state != IAttackRegistry.ContractState.NOT_DEPLOYED
                    && state != IAttackRegistry.ContractState.NEW_DEPLOYMENT
                    && state != IAttackRegistry.ContractState.ATTACK_REQUESTED)
        ) {
            revert WithdrawsDisabled();
        }

        uint256 amount = eligibleStake[msg.sender];
        if (amount == 0) revert InvalidAmount();

        _clampUserSums(msg.sender);
        // Withdrawing forfeits the caller's bonus claim: subtract their full contribution from
        // the global accumulators so honest stakers' shares aren't diluted by the exiter's
        // forfeited weight.
        sumStakeTime -= userSumStakeTime[msg.sender];
        sumStakeTimeSq -= userSumStakeTimeSq[msg.sender];

        eligibleStake[msg.sender] = 0;
        userSumStakeTime[msg.sender] = 0;
        userSumStakeTimeSq[msg.sender] = 0;
        totalEligibleStake -= amount;

        stakeToken.safeTransfer(msg.sender, amount);
        emit Withdrawn(msg.sender, amount);
    }
