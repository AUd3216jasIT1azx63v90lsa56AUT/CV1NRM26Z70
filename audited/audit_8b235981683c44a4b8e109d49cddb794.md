### Title
Authorized Operator Can Permanently Cancel Maker's Merkle Root via Coarse `isAuthorized` Reuse - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
`EcrecoverRatifier.cancelRoot` gates access using `IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender)`, the same coarse boolean that governs all Midnight position-management functions. Because `isRootCanceled` is a one-way latch with no `uncancelRoot` counterpart, any operator the maker has authorized for any purpose can permanently invalidate an entire Merkle tree of offers, making every offer in that tree unfillable forever.

### Finding Description
**Exact code path:**

`EcrecoverRatifier.sol` lines 27–31:
```solidity
function cancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
    isRootCanceled[maker][root] = true;
    emit CancelRoot(msg.sender, maker, root);
}
``` [1](#0-0) 

The guard delegates entirely to `Midnight.isAuthorized[maker][msg.sender]`. That mapping is set by `Midnight.setIsAuthorized` (lines 731–735), which is a flat, scope-free boolean: [2](#0-1) 

Once `isRootCanceled[maker][root] = true`, every subsequent `take` that presents that root hits the hard revert in `isRatified` (line 38):
```solidity
require(!isRootCanceled[offer.maker][root], RootCanceled());
``` [3](#0-2) 

There is no `uncancelRoot` function anywhere in the interface or implementation. [4](#0-3) 

**Attacker-controlled inputs:** `maker` (the victim's address) and `root` (any root the maker has signed over).

**Why existing checks fail:** The only guard is `isAuthorized[maker][msg.sender]`. This is the identical check used for `withdraw`, `repay`, `supplyCollateral`, `setConsumed`, and `setIsAuthorized` on Midnight itself. There is no secondary check that restricts `cancelRoot` to the maker alone or to a narrower authorization scope. The protocol documentation acknowledges the reuse (lines 106–108) but provides no code-level enforcement: [5](#0-4) 

### Impact Explanation
All offers whose Merkle root is `root` become permanently unfillable. The maker cannot undo the cancellation; they must generate a new key tree, re-sign every offer, and redistribute the new proofs to all potential takers. Any in-flight or resting orders referencing the old root are dead. This is a permanent, targeted denial-of-service against the maker's entire active offer set with zero cost to the attacker beyond gas.

### Likelihood Explanation
The precondition — maker has authorized an operator — is common in practice (e.g., authorizing a repayment helper, a portfolio manager contract, or a keeper). The authorization is a single `setIsAuthorized` call with no expiry. The operator needs only to know one of the maker's active roots (observable on-chain from past `take` events or off-chain distribution channels). The attack is a single transaction, is repeatable for every root the maker ever publishes, and cannot be front-run-defended by the maker once the operator is authorized.

### Recommendation
Restrict `cancelRoot` to the maker only — remove the `isAuthorized` branch entirely:

```solidity
function cancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender, Unauthorized());
    isRootCanceled[maker][root] = true;
    emit CancelRoot(msg.sender, maker, root);
}
```

If delegation is desired, introduce a separate, narrowly scoped authorization mapping (e.g., `isCancelAuthorized[maker][operator]`) that the maker must set explicitly and independently of the broad Midnight authorization.

### Proof of Concept
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {EcrecoverRatifier} from "src/ratifiers/EcrecoverRatifier.sol";

contract CancelRootDoSTest is Test {
    Midnight midnight;
    EcrecoverRatifier ratifier;

    address maker   = makeAddr("maker");
    address operator = makeAddr("operator");
    bytes32 root    = keccak256("makerActiveTree");

    function setUp() public {
        midnight = new Midnight();
        ratifier = new EcrecoverRatifier(address(midnight));

        // maker authorizes operator for an unrelated purpose (e.g., repayment)
        vm.prank(maker);
        midnight.setIsAuthorized(operator, true, maker);
    }

    function test_operatorCancelsRoot() public {
        // precondition: root is not yet canceled
        assertFalse(ratifier.isRootCanceled(maker, root));

        // operator cancels maker's root without maker's knowledge
        vm.prank(operator);
        ratifier.cancelRoot(maker, root);

        // assertion 1: root is now permanently canceled
        assertTrue(ratifier.isRootCanceled(maker, root));

        // assertion 2: no uncancelRoot path exists — state is irreversible
        // (compile-time: IEcrecoverRatifier has no uncancelRoot selector)

        // assertion 3: any take using this root would revert RootCanceled
        // (verified by inspecting isRatified line 38 — require(!isRootCanceled[offer.maker][root]))
    }
}
```

**Expected assertions:**
- `isRootCanceled(maker, root)` flips from `false` to `true` after the operator's call.
- No transaction can reset it.
- Any `Midnight.take` presenting `root` for `maker` reverts with `RootCanceled`.
