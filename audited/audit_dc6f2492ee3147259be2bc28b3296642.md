### Title
`setIsAuthorized` Allows Any Authorized Account to Delegate Full User Authorization to Arbitrary Third Parties, Over-Extending Permission Scope — (File: src/Midnight.sol)

### Summary
The `setIsAuthorized` function in `Midnight.sol` uses the same `isAuthorized` mapping to gate both position-management operations and the authorization-granting operation itself. This means any account a user authorizes — even for a narrow purpose such as bundled trading — inherits the unrestricted ability to grant the user's authorization to any arbitrary address. A user who authorizes a contract expecting it to manage their positions has unknowingly also authorized it to hand full control of their account to a third party.

### Finding Description

`setIsAuthorized` at `src/Midnight.sol` lines 731–735:

```solidity
function setIsAuthorized(address authorized, bool newIsAuthorized, address onBehalf) external {
    require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
    isAuthorized[onBehalf][authorized] = newIsAuthorized;
    emit EventsLib.SetIsAuthorized(msg.sender, authorized, newIsAuthorized, onBehalf);
}
``` [1](#0-0) 

The guard `isAuthorized[onBehalf][msg.sender]` is the same mapping used to protect every other user-facing function (`withdraw`, `withdrawCollateral`, `repay`, `take`, `setConsumed`). There is no separate, narrower permission that restricts which authorized accounts may further delegate authorization. Consequently, any authorized account can call `setIsAuthorized(attacker, true, alice)` and the check passes.

The protocol's own inline documentation acknowledges this at lines 101–110:

> *"authorized accounts can authorize other accounts on behalf of the user"* [2](#0-1) 

The test `testSetIsAuthorizedAuthorization` in `test/AuthorizationTest.sol` (lines 290–304) explicitly confirms the behavior: once `authorized` is set for `user`, `authorized` can immediately call `setIsAuthorized(newAuthorized, true, user)` and it succeeds. [3](#0-2) 

The same over-extension is present in `EcrecoverAuthorizer.setIsAuthorized` (lines 33–36), where the signer check accepts any already-authorized account, allowing it to gaslessly sign an EIP-712 authorization that grants a third party full access on the user's behalf:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [4](#0-3) 

**Root cause**: The authorization check for `setIsAuthorized` does not distinguish between "authorized to manage positions" and "authorized to delegate authorization." Both are gated by the same flat `isAuthorized` mapping, so every authorized account automatically inherits the ability to escalate privileges.

**Exploit flow**:
1. Alice authorizes `MidnightBundles` (or any other contract) on Midnight for trading purposes.
2. `MidnightBundles` is authorized: `isAuthorized[alice][MidnightBundles] = true`.
3. A malicious or compromised path inside the authorized contract calls `midnight.setIsAuthorized(attacker, true, alice)`. The guard `isAuthorized[alice][MidnightBundles]` is `true`, so the call succeeds.
4. `isAuthorized[alice][attacker]` is now `true`.
5. Attacker calls `withdraw(market, units, alice, attacker)` — drains Alice's credit.
6. Attacker calls `withdrawCollateral(market, index, assets, alice, attacker)` — drains Alice's collateral.
7. Attacker calls `setConsumed(group, type(uint256).max, alice)` — cancels all of Alice's resting offers. [5](#0-4) [6](#0-5) [7](#0-6) 

### Impact Explanation
Full theft of a victim's credit (loan tokens at maturity), collateral assets, and cancellation of all active offers. Any user who has ever authorized a contract — including the in-scope `MidnightBundles` periphery — is exposed if that contract contains any path (reentrancy, public function, callback abuse) that can be triggered to call `setIsAuthorized` on their behalf. The impact is total loss of position.

### Likelihood Explanation
Medium. The `MidnightBundles` contract itself does not currently expose a direct call to `setIsAuthorized`, but the authorization model is flat and permanent: every user who authorizes any contract for any reason silently grants that contract the power to re-delegate their authorization. As the ecosystem grows and more integrators build on top of Midnight's authorization mapping (the NatSpec explicitly anticipates this: *"other contracts might re-use Midnight's authorization mapping"*), the attack surface widens. Users authorizing bundlers, ratifiers, or third-party integrators will not expect that action to also grant those contracts the ability to hand full account control to an arbitrary address. [8](#0-7) 

### Recommendation
Restrict `setIsAuthorized` so that only `onBehalf == msg.sender` (the user themselves) can grant new authorizations. Authorized accounts should be able to act *within* the scope of their authorization (position management, trading, repayment) but should not be able to *extend* that authorization to third parties. If delegation is a desired feature, introduce a separate, explicit permission bit (e.g., `canDelegate`) that users must consciously set, distinct from the general `isAuthorized` flag.

### Proof of Concept

```solidity
// Alice authorizes MidnightBundles for trading
vm.prank(alice);
midnight.setIsAuthorized(address(midnightBundles), true, alice);

// MidnightBundles (or any authorized contract) grants attacker full access

midnight.setIsAuthorized(attacker, true, alice);

// Attacker drains Alice's credit

assertGt(collateralToken.balanceOf(attacker), 0);

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
