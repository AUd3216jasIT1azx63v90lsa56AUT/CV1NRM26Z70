Now I have all the information needed. Let me trace the exact code path.### Title
Missing `loanToken != address(0)` validation in `touchMarket` permanently freezes all takes on the created market - (File: src/Midnight.sol)

### Summary

`touchMarket` validates collateral tokens, LLTV, maxLif, and maturity, but performs no check that `market.loanToken != address(0)`. An unprivileged market creator can call `touchMarket` with `loanToken = address(0)`, which succeeds and snapshots `defaultSettlementFeeCbp[address(0)]` — all zeros if never set by governance. Every subsequent `take()` on this market reaches `SafeTransferLib.safeTransferFrom(address(0), ...)`, which unconditionally reverts with `NoCode()` because `address(0).code.length == 0`, permanently freezing the market.

### Finding Description

**Root cause — missing check in `touchMarket`:**

`touchMarket` at `src/Midnight.sol:755-791` validates:
- `market.maturity <= block.timestamp + 100 * 365 days`
- `market.collateralParams.length > 0`
- each `collateralToken > previousCollateralToken` (enforces sorted, non-zero collateral tokens)
- `isLltvAllowed(lltv)` and `maxLif` for each collateral

There is **no** `require(market.loanToken != address(0))` check. [1](#0-0) 

**Fee snapshot with zero values:**

On creation, the market snapshots `defaultSettlementFeeCbp[market.loanToken]`. With `loanToken = address(0)` and no governance call to `setDefaultSettlementFee(address(0), ...)`, all seven breakpoints are zero. [2](#0-1) 

**`TakeAmountsLib.buyerAssetsToUnits` with zero settlement fee:**

`buyerAssetsToUnits` reads `settlementFee(id, ttm)` from the market state. With all-zero cbp values, `settlementFee = 0`. The function then computes `buyerPrice = offerPrice + 0 = offerPrice` and returns `targetBuyerAssets.mulDivDown(WAD, offerPrice)` — a nonzero result for any nonzero `offerPrice`. The function does not revert; it silently returns a unit count as if the market were valid. [3](#0-2) 

**`take()` reverts unconditionally at token transfer:**

`take()` calls `touchMarket` internally (line 347), which is a no-op for an already-created market. All position accounting, consumed tracking, and state mutations proceed normally. At lines 455–456, `SafeTransferLib.safeTransferFrom(offer.market.loanToken, ...)` is called with `offer.market.loanToken = address(0)`. [4](#0-3) 

`SafeTransferLib.safeTransferFrom` begins with `require(token.code.length > 0, NoCode())`. Since `address(0).code.length == 0`, this reverts unconditionally regardless of the transfer amount — even when `buyerAssets - sellerAssets == 0`. [5](#0-4) 

**Why existing checks do not stop it:**

The sorted-collateral check (`collateralToken > previousCollateralToken`) enforces that collateral tokens are non-zero (since `address(0)` is the zero value and `previousCollateralToken` starts at `address(0)`, the first collateral must be strictly greater). This protects collateral tokens but is entirely independent of `loanToken`. No other check in `touchMarket` or `take()` validates `loanToken`. [6](#0-5) 

### Impact Explanation

Every call to `take()` on a market created with `loanToken = address(0)` reverts with `SafeTransferLib.NoCode()` at the token transfer step. The market is permanently frozen: no take can ever succeed. `TakeAmountsLib.buyerAssetsToUnits` returns a nonzero unit count (zero settlement fee, nonzero offer price), so the `MidnightBundles` bundler computes a valid `unitsToTake`, submits the take via `try/catch`, catches the revert, and ultimately fails with `OutOfOffers()` — the bundler cannot fill any target on this market. [7](#0-6) 

### Likelihood Explanation

The precondition is trivially reachable: `touchMarket` is `public` with no access control, and the only inputs required are valid collateral params, a valid LLTV tier, a valid maxLif, and a maturity within range. Setting `loanToken = address(0)` requires no privilege. The market ID is deterministic and unique, so the broken market cannot be overwritten. The condition is permanent and repeatable for any distinct `(address(0), maturity, collateralParams)` tuple.

### Recommendation

Add a non-zero check for `loanToken` at the top of the market-creation branch in `touchMarket`:

```solidity
require(market.loanToken != address(0), InvalidLoanToken());
```

This mirrors the implicit protection already applied to collateral tokens via the sorted-greater-than check, and closes the gap for the loan token. [8](#0-7)

        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
