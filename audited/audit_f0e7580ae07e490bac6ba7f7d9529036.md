### Title
Unbounded gas on `canLiquidate` + 16-oracle bitmap loop makes unhealthy positions permanently unliquidatable - (`src/Midnight.sol`)

### Summary
`touchMarket` is fully permissionless and imposes no constraints on the `liquidatorGate` or oracle addresses beyond structural validity (sorted tokens, allowed LLTV, valid `maxLif`). The `liquidate` function makes an unbounded external call to `ILiquidatorGate.canLiquidate` at line 598, then iterates over up to 16 activated collaterals calling `IOracle.price()` on each at line 610, also without any gas cap. A malicious market creator can deploy a gate and oracles that are expensive-but-non-reverting, causing every liquidation attempt to exhaust the block gas limit and permanently violating the invariant that unhealthy positions remain liquidatable.

### Finding Description

**Reachable Solidity path:**

`liquidate()` at `src/Midnight.sol:581` calls:

1. `ILiquidatorGate(market.liquidatorGate).canLiquidate(msg.sender)` — line 598, unbounded gas forwarded.
2. A `while (_collateralBitmap != 0)` loop — lines 607–618 — that calls `IOracle(_collateralParam.oracle).price()` for every bit set in `_position.collateralBitmap`, up to `MAX_COLLATERALS_PER_BORROWER = 16` iterations, each also unbounded. [1](#0-0) [2](#0-1) 

**Market creation — no gate/oracle gas validation:**

`touchMarket` validates only that collateral tokens are sorted, LLTV is in the allowed set, and `maxLif` matches the formula. It performs zero checks on `market.liquidatorGate` or any `collateralParams[i].oracle` address. [3](#0-2) 

**Attacker-controlled inputs:**

- `market.liquidatorGate` → attacker-deployed contract implementing `ILiquidatorGate`
- `market.collateralParams[i].oracle` (up to 16 entries) → attacker-deployed contracts implementing `IOracle`

Both interfaces are minimal (one function each), so the attacker controls the full implementation. [4](#0-3) 

**Exploit flow:**

1. Attacker deploys `MaliciousGate` implementing `canLiquidate(address) → true` but burning ~15 M gas internally (e.g., a tight SLOAD loop over a large mapping).
2. Attacker deploys 16 `ExpensiveOracle` contracts each implementing `price() → validPrice` but burning ~1 M gas each.
3. Attacker calls `touchMarket` with `liquidatorGate = MaliciousGate` and 16 collateral entries pointing to the 16 `ExpensiveOracle` contracts. Market is created; no check fails.
4. Attacker (as borrower) calls `supplyCollateral` for all 16 collateral tokens, setting all 16 bits in `collateralBitmap`, then borrows via `take`.
5. Position becomes unhealthy (price movement or deliberate under-collateralization).
6. Any liquidator calls `liquidate()`. Gas consumed: ~15 M (gate) + 16 × ~1 M (oracles) + base overhead > 30 M block limit → out-of-gas revert every time.

**Why existing checks fail:**

- The LIVENESS section (lines 142–152) only documents that *reverting* oracles/gates break liquidation. It does not address expensive-but-non-reverting external calls.
- There is no `gasleft()` guard, no `{gas: N}` cap on either external call site, and no oracle/gate allowlist.
- The `require` at line 597–600 only checks the boolean return value; it cannot prevent gas exhaustion before the check completes. [5](#0-4) 

### Impact Explanation

The core invariant "unhealthy positions remain liquidatable" is permanently broken for any market where the attacker controls the gate and/or oracle contracts. Lenders in such a market cannot recover their funds through liquidation; bad debt accrues and is socialized, but the bad-debt realization path (`liquidate` with `seizedAssets = repaidUnits = 0`) is equally blocked because it traverses the same gate check and the same oracle bitmap loop.

### Likelihood Explanation

**Preconditions:**
- Attacker must be a market creator (permissionless).
- At least one lender must supply credit in the market. This is feasible because: (a) the gate appears legitimate for `canIncreaseCredit`/`canIncreaseDebt` (different functions on `IEnterGate`, not `ILiquidatorGate`), and (b) oracles can return valid prices cheaply in off-chain view calls while burning gas on-chain via storage-heavy paths.
- Borrower must activate 16 collaterals (attacker can be the borrower).

**Feasibility:** Fully on-chain, no privileged role required, no impossible oracle values, no user mistake required. Repeatable across any number of markets.

### Recommendation

Apply a fixed gas cap on both external calls:

```solidity
// At line 598 — cap gate call
require(
    market.liquidatorGate == address(0)
        || ILiquidatorGate(market.liquidatorGate).canLiquidate{gas: MAX_GATE_GAS}(msg.sender),
    LiquidatorGatedFromLiquidating()
);

// At line 610 — cap each oracle call
uint256 price = IOracle(_collateralParam.oracle).price{gas: MAX_ORACLE_GAS}();
```

Define `MAX_GATE_GAS` and `MAX_ORACLE_GAS` as protocol constants (e.g., 100 000 gas each) chosen so that `MAX_GATE_GAS + 16 * MAX_ORACLE_GAS + base_liquidate_gas` stays well below the block gas limit. Alternatively, enforce an oracle/gate allowlist at market creation time in `touchMarket`.

contract GasGriefGate is ILiquidatorGate {
    function canLiquidate(address) external view returns (bool) {
        // burn ~15 M gas via cold SLOAD loop

        for (uint256 i = 1; i <= 500; i++) {
            assembly { dummy := sload(i) }
        }
        return true; // always allows liquidation — gate check passes
    }
}

contract GasGriefOracle is IOracle {
    function price() external view returns (uint256) {
        // burn ~1 M gas per call
        uint256 dummy;
        for (uint256 i = 1; i <= 30; i++) {
            assembly { dummy := sload(i) }
        }
        return 1e36; // valid price
    }
}

contract GasGriefTest is Test {
    Midnight midnight;

    function testLiquidationOutOfGas() public {
        GasGriefGate gate = new GasGriefGate();
        CollateralParams[] memory params = new CollateralParams[](16);
        for (uint256 i = 0; i < 16; i++) {
            params[i] = CollateralParams({
                token: address(uint160(i + 1)),   // dummy sorted tokens
                lltv: 0.77e18,
                maxLif: maxLif(0.77e18, 0.25e18),
                oracle: address(new GasGriefOracle())
            });

        Market memory market = Market({
            loanToken: address(loanToken),
            collateralParams: params,
            maturity: block.timestamp + 365 days,
            rcfThreshold: 0,
            enterGate: address(0),
            liquidatorGate: address(gate)
        });

        midnight.touchMarket(market);
        // ... setup borrower with 16 active collaterals, make position unhealthy ...

        uint256 gasBefore = gasleft();
        try midnight.liquidate{gas: 30_000_000}(
            market, 0, 0, 0, borrower, false, address(this), address(0), ""
        ) {
            revert("should have OOG");
        } catch {
            // assertion: liquidation of unhealthy position failed — invariant violated
            assertLt(gasleft(), gasBefore / 10, "nearly all gas consumed");
        }
    }
}

- `liquidate` reverts with out-of-gas even when the position is provably unhealthy.
- Fuzz: vary `gate_gas` (0–29 M) and `oracle_gas` (0–2 M per oracle); assert `gate_gas + 16 * oracle_gas + BASE > BLOCK_GAS_LIMIT` implies liquidation always reverts.
- Invariant test: after any sequence of `supplyCollateral` / `take` / price-drop steps, if `isHealthy == false` then `liquidate` must not revert with OOG.

    function touchMarket(Market memory market) public returns (bytes32) {
