### Title
Borrower Can Activate Collateral With Reverting Oracle to Create a Non-Liquidatable Position — (File: src/Midnight.sol)

---

### Summary

`supplyCollateral` activates a collateral in the borrower's bitmap without ever calling its oracle. The `liquidate` function unconditionally calls `IOracle.price()` for every activated collateral. If any one oracle reverts, the entire liquidation transaction reverts. A borrower can therefore supply a dust amount of a collateral whose oracle is reverting (or can be made to revert) to permanently block liquidation of their underwater position.

---

### Finding Description

**Root cause — `supplyCollateral` performs no oracle liveness check:**

`supplyCollateral` validates authorization and the collateral count, then sets the bitmap bit and pulls tokens. It never calls the oracle. [1](#0-0) 

**Liquidation unconditionally calls every activated collateral's oracle:**

Inside `liquidate`, the loop at lines 607–618 iterates over the full `collateralBitmap` and calls `IOracle(_collateralParam.oracle).price()` for each entry with no `try/catch`. A single revert from any oracle propagates and reverts the entire `liquidate` call. [2](#0-1) 

**The protocol itself documents this gap:**

The NatSpec at lines 34–36 explicitly states: *"Liquidation reverts if any of the activated collaterals' oracle reverts … Note that a borrower can activate a collateral once its oracle is reverting because the oracle is not called in supplyCollateral."* [3](#0-2) 

**`withdrawCollateral` is also blocked once debt exists:**

`withdrawCollateral` calls `isHealthy`, which calls every activated oracle when `debt > 0`. So the borrower cannot self-rescue by withdrawing the bad collateral either. [4](#0-3) 

---

### Impact Explanation

An underwater borrower whose position cannot be liquidated accumulates bad debt that is socialized across all lenders in the market via the `lossFactor` slashing mechanism. The position is permanently frozen: liquidators cannot seize collateral, lenders cannot recover principal, and the market's `withdrawable` pool is never replenished from that debt. This is a **permanent, unrecoverable loss of lender funds** for any market where the attack succeeds.

---

### Likelihood Explanation

The attack requires a market whose collateral list includes at least one oracle that can be made to revert. This is realistic because:

- **Chainlink feeds** revert when the aggregator is paused, the sequencer is down (L2), or the answer is stale and the integration enforces a staleness check.
- **Custom/permissionless oracles** — markets are created permissionlessly via `touchMarket`; a market creator can embed a malicious oracle that reverts on demand.
- The borrower needs only to supply **1 wei** of the affected collateral token to set the bitmap bit; the cost is negligible.
- Once the bit is set and the oracle is reverting, neither the liquidator nor the borrower can clear the bit (both `liquidate` and `withdrawCollateral` revert).

No privileged access is required. Any borrower in any market containing a revert-capable oracle can execute this.

---

### Recommendation

1. **Short term**: Wrap each oracle call in `liquidate` (and `isHealthy`) with a `try/catch`. If an oracle reverts, treat the collateral price as 0 (worst-case for health) or skip it for `maxDebt` computation while still allowing seizure of other collaterals.
2. **Long term**: Add an oracle liveness check in `supplyCollateral` before setting the bitmap bit, so a collateral whose oracle is already reverting cannot be activated.

---

### Proof of Concept

**Setup**: A market `M` with two collateral entries — `C0` (oracle `O0`, always works) and `C1` (oracle `O1`, can be paused/made to revert).

1. **Eve** calls `take` as seller in market `M`, acquiring debt backed by `C0` collateral. Position is healthy.
2. Eve calls `supplyCollateral(M, 1 /*index of C1*/, 1 wei, Eve)`. No oracle is called; `C1` is added to Eve's `collateralBitmap`. Cost: 1 wei of `C1` token.
3. `O1` begins reverting (e.g., Chainlink sequencer goes down, or Eve triggers a pause on a custom oracle she controls).
4. Eve's position goes underwater (price of `C0` drops).
5. **Alice** (liquidator) calls `liquidate(M, 0, ..., Eve, ...)`. The loop at line 607 reaches `C1`, calls `O1.price()`, which reverts. Alice's transaction reverts.
6. Eve's position is permanently non-liquidatable. Bad debt accrues to all lenders in `M`. [2](#0-1) [5](#0-4)

    function supplyCollateral(Market memory market, uint256 collateralIndex, uint256 assets, address onBehalf)
        external
    {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);
        address collateralToken = market.collateralParams[collateralIndex].token;

        Position storage _position = position[id][onBehalf];
        uint256 oldCollateral = _position.collateral[collateralIndex];
        _position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);

        if (oldCollateral == 0 && assets > 0) {
            uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
            _position.collateralBitmap = newCollateralBitmap;
            require(
                UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER, TooManyActivatedCollaterals()
            );
        }

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
