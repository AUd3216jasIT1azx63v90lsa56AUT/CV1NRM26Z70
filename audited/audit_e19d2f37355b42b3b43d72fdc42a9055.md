### Title
Missing `loanToken` zero-address validation in `touchMarket` permanently freezes loan-side operations - (File: src/Midnight.sol)

### Summary
`touchMarket` validates collateral tokens, LLTV, maxLif, and maturity, but performs no check on `market.loanToken`. An unprivileged caller can pass `loanToken = address(0)`, which passes all guards and creates a live market (`tickSpacing = DEFAULT_TICK_SPACING > 0`). Every subsequent `take` and `repay` in that market unconditionally reverts because `SafeTransferLib.safeTransferFrom` opens with `require(token.code.length > 0, NoCode())`, and `address(0).code.length == 0` always.

### Finding Description
**Root cause — `src/Midnight.sol` lines 755–791:**

`touchMarket` iterates over `collateralParams` and enforces `collateralToken > previousCollateralToken` (where `previousCollateralToken` starts at `address(0)`), so collateral tokens are implicitly non-zero. `market.loanToken` is never read for validation; it is only used to copy default settlement-fee and continuous-fee values into storage (lines 777–785), both of which silently return zero for `address(0)`.

**Exploit flow:**

1. Attacker constructs a `Market` with `loanToken = address(0)`, a valid maturity, and at least one collateral entry satisfying the LLTV/maxLif/sort checks.
2. Attacker calls `touchMarket(market)`. All `require` guards pass; `_marketState.tickSpacing` is set to `DEFAULT_TICK_SPACING` (line 776). Market is live.
3. Any call to `take` (lines 455–456) or `repay` (line 520) reaches `SafeTransferLib.safeTransferFrom(address(0), ...)`.
4. `SafeTransferLib.safeTransferFrom` (line 25): `require(token.code.length > 0, NoCode())` — `address(0).code.length == 0` — reverts unconditionally.

**Why existing checks are insufficient:**

The collateral-sort guard (`collateralToken > previousCollateralToken`) only protects collateral token addresses. There is no analogous guard for `loanToken`. The `SafeTransferLib` code-length check is a transfer-time guard, not a market-creation guard, so it fires on every loan-side call rather than at market creation.

### Impact Explanation
Every `take` and `repay` call in the affected market reverts with `SafeTransferLib.NoCode`. No credit or debt can ever be created (both require a successful `take`), and no debt can ever be repaid. All loan-side operations are permanently and irrecoverably frozen for the lifetime of the market. `supplyCollateral` and `withdrawCollateral` are unaffected (they use the collateral token), so collateral deposited into this market can still be recovered; however, the market is rendered entirely non-functional for its intended purpose.

### Likelihood Explanation
The precondition is trivially satisfiable: any address can call `touchMarket` with `loanToken = address(0)` and otherwise-valid parameters. No privilege, signature, or prior state is required. The attack is a single transaction and is permanent — once the market is created, its `tickSpacing` is non-zero and `touchMarket` will never re-enter the initialization branch, so the zero `loanToken` cannot be corrected.

### Recommendation
Add an explicit non-zero check for `market.loanToken` at the top of the market-initialization branch in `touchMarket`, mirroring the implicit protection already applied to collateral tokens:

```solidity
require(market.loanToken != address(0), ZeroLoanToken());
```

This should be placed alongside the existing `maturity`, `collateralParams.length`, and per-collateral checks (lines 758–773).

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;
            uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];
            _marketState.settlementFeeCbp3 = _defaultSettlementFeeCbp[3];
            _marketState.settlementFeeCbp4 = _defaultSettlementFeeCbp[4];
            _marketState.settlementFeeCbp5 = _defaultSettlementFeeCbp[5];
            _marketState.settlementFeeCbp6 = _defaultSettlementFeeCbp[6];
            _marketState.continuousFee = defaultContinuousFee[market.loanToken];
            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);
        }
        return id;
    }
