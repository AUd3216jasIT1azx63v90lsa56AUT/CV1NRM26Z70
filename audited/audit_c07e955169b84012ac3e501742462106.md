### Title
Nonce Not Committed on Failed `setIsAuthorized` Enables Replay of Signed Authorizations — (File: `src/periphery/EcrecoverAuthorizer.sol`)

---

### Summary

`EcrecoverAuthorizer.setIsAuthorized` increments the authorizer's nonce inside the same atomic transaction as the downstream `IMidnight(MIDNIGHT).setIsAuthorized(...)` call. If the Midnight call reverts (because `EcrecoverAuthorizer` is not currently authorized in Midnight for the user), the entire transaction reverts, including the nonce increment. The signed authorization remains replayable at nonce `N` indefinitely until the user submits another transaction that consumes that nonce.

---

### Finding Description

In `EcrecoverAuthorizer.setIsAuthorized`, the nonce is incremented on line 26:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
``` [1](#0-0) 

This increment is not committed independently — it lives in the same transaction as the final call on line 47:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [2](#0-1) 

That Midnight call enforces:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [3](#0-2) 

Here `msg.sender` is the `EcrecoverAuthorizer` contract and `onBehalf` is `authorization.authorizer`. The call reverts whenever `isAuthorized[authorization.authorizer][EcrecoverAuthorizer]` is `false`. This is a **mutable condition** — the user can authorize or deauthorize `EcrecoverAuthorizer` at any time via a direct call to `Midnight.setIsAuthorized`.

The test suite confirms the required setup pattern: the user must first call `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, ...)` before a signed authorization can be executed. [4](#0-3) 

**Exploit path:**

1. Alice calls `Midnight.setIsAuthorized(ecrecoverAuthorizer, true, Alice)` — authorizes the periphery contract.
2. Alice signs `Authorization{authorizer=Alice, authorized=Bob, isAuthorized=true, nonce=0, deadline=T}`.
3. Alice later calls `Midnight.setIsAuthorized(ecrecoverAuthorizer, false, Alice)` — revokes the periphery contract's permission (e.g., to block a pending transaction she no longer wants executed).
4. An attacker (or the original relayer) submits the signed authorization from step 2. The `nonce` check passes and increments to 1 in memory, but `IMidnight.setIsAuthorized` reverts with `Unauthorized`. The entire transaction reverts; `nonce[Alice]` stays at `0`.
5. Alice later re-authorizes `EcrecoverAuthorizer` in Midnight for legitimate use.
6. The attacker replays the step-2 authorization. `nonce[Alice]` is still `0`, so the check passes. `IMidnight.setIsAuthorized` now succeeds. Bob is authorized for Alice against her current intent.

---

### Impact Explanation

An authorized account in Midnight can call `take`, `withdraw`, `withdrawCollateral`, `repay`, `supplyCollateral`, `setConsumed`, and `setIsAuthorized` on behalf of the victim. [5](#0-4) [6](#0-5) 

Replaying a stale authorization to grant access to a malicious or compromised account can lead to direct theft of lender credit, unauthorized collateral withdrawal, or further authorization escalation (the newly authorized account can authorize additional accounts).

---

### Likelihood Explanation

The trigger condition — `EcrecoverAuthorizer` being deauthorized in Midnight at the time of submission, then re-authorized later — is a realistic operational pattern. A user who wants to "pause" gasless operations revokes the periphery contract, then re-enables it. Any signed-but-unexecuted authorization from before the pause remains replayable. The attacker needs only to monitor the mempool or chain history for failed `setIsAuthorized` calls and wait for the re-authorization event. No privileged access is required.

---

### Recommendation

Increment the nonce unconditionally before any external call that can revert, and do so in a way that is not rolled back on downstream failure. The standard pattern is to increment the nonce first in a separate storage write, then perform the external call:

```solidity
function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
    require(block.timestamp <= authorization.deadline, Expired());
    uint256 currentNonce = nonce[authorization.authorizer];
    require(authorization.nonce == currentNonce, InvalidNonce());
    nonce[authorization.authorizer] = currentNonce + 1; // committed before any revert path

    // ... signature verification ...

    IMidnight(MIDNIGHT).setIsAuthorized(...);
}
```

This mirrors the fix recommended in the referenced report and the pattern used by OpenZeppelin's `MinimalForwarder`. The nonce increment must be placed before any call that can revert so that a failed downstream call does not undo the nonce advance. [7](#0-6)

```

  nonce[Alice] = 0
  isAuthorized[Alice][EcrecoverAuthorizer] = false  (not yet set up)

        → sig_A

Step 2: Alice calls midnight.setIsAuthorized(ecrecoverAuthorizer, true, Alice)
        → isAuthorized[Alice][EcrecoverAuthorizer] = true

Step 3: Alice calls midnight.setIsAuthorized(ecrecoverAuthorizer, false, Alice)
        → isAuthorized[Alice][EcrecoverAuthorizer] = false
        (Alice wants to block any pending gasless tx)

Step 4: Attacker calls ecrecoverAuthorizer.setIsAuthorized(auth_A, sig_A)
        → nonce check: 0 == 0 ✓ (increments to 1 in memory)
        → signature check: passes
        → IMidnight.setIsAuthorized reverts: isAuthorized[Alice][EcrecoverAuthorizer] = false
        → entire tx reverts; nonce[Alice] stays 0

Step 5: Alice calls midnight.setIsAuthorized(ecrecoverAuthorizer, true, Alice)
        → isAuthorized[Alice][EcrecoverAuthorizer] = true  (re-enables for new use)

Step 6: Attacker replays ecrecoverAuthorizer.setIsAuthorized(auth_A, sig_A)
        → nonce check: 0 == 0 ✓
        → signature check: passes
        → IMidnight.setIsAuthorized succeeds
        → isAuthorized[Alice][Bob] = true  ← unauthorized

Result: Bob is now authorized to act on Alice's positions (withdraw, take, withdrawCollateral, etc.)
```

        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
