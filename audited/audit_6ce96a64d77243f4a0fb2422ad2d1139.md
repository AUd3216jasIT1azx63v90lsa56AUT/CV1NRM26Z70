### Title
Non-standard `transfer` returndata (64 bytes, first word zero) causes permanent fund freeze via `SafeTransferLib` false-decode — (File: src/libraries/SafeTransferLib.sol)

### Summary
`SafeTransferLib.safeTransfer` and `safeTransferFrom` use `abi.decode(returndata, (bool))` which reads only the first 32 bytes of any non-empty returndata. A malicious loan token that returns 64 bytes with the first 32 bytes as zero from `transfer` (while returning standard `true` from `transferFrom`) causes `safeTransfer` to decode `false` and revert with `TransferReturnedFalse()`. Because `touchMarket` imposes no validation on the loan token address or its return behavior, an unprivileged market creator can deploy such a token, attract lender deposits via `take` (which uses `safeTransferFrom`), and permanently freeze all withdrawals.

### Finding Description

**Root cause — `SafeTransferLib.sol` line 21:**

```solidity
require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferReturnedFalse());
```

The guard accepts only two cases: empty returndata, or returndata whose first 32 bytes ABI-decode to `true`. It does not constrain `returndata.length == 32`. When a token returns 64 bytes (e.g., `abi.encode(uint256(0), uint256(1))`):

1. EVM call returns `success = true` (the transfer "succeeded" at the EVM level).
2. `returndata.length == 64 ≠ 0` — the short-circuit fails.
3. `abi.decode(returndata, (bool))` reads the first 32 bytes (all zero) → decodes as `false`. Solidity 0.8.x `abi.decode` for static types does not revert on trailing data; it reads the required 32 bytes and ignores the rest.
4. `require(false, TransferReturnedFalse())` → reverts.

**Identical logic applies to `safeTransferFrom` line 33** with `TransferFromReturnedFalse()`.

**Exploit path:**

| Step | Actor | Action |
|------|-------|--------|
| 1 | Market creator | Deploys `MaliciousToken`: `transferFrom` returns `abi.encode(true)` (32 bytes); `transfer` returns `abi.encode(uint256(0), uint256(1))` (64 bytes, first word zero). |
| 2 | Market creator | Calls any function that invokes `touchMarket(market)` with `market.loanToken = MaliciousToken`. `touchMarket` validates only maturity, collateral params, LLTV, and maxLif — **no loan token behavior check**. |
| 3 | Lender | Calls `take(...)`. Internally `safeTransferFrom(MaliciousToken, payer, address(this), ...)` is called. `transferFrom` returns 32 bytes of `true` → passes. Credit is recorded. |
| 4 | Lender | Calls `withdraw(market, units, onBehalf, receiver)`. State is updated (credit decremented, withdrawable decremented) **before** the transfer. Then `safeTransfer(MaliciousToken, receiver, units)` is called. `transfer` returns 64 bytes with first word zero → `abi.decode` → `false` → reverts with `TransferReturnedFalse()`. |

Because the state update precedes the external call in `withdraw` (lines 493–499), the revert rolls back the state change, so the lender's credit is not lost — but every subsequent `withdraw` attempt hits the same revert. Funds are permanently locked.

**`touchMarket` has no loan token guard** (lines 757–789): it only checks collateral params, maturity, LLTV, and maxLif. Any address with `code.length > 0` is accepted as a loan token.

### Impact Explanation
All calls to `withdraw` for the affected market permanently revert with `TransferReturnedFalse()`. Lenders who supplied credit via `take` cannot recover their funds. If the malicious token also returns 64 bytes (first word zero) from `transferFrom` after initial deposits, `repay` and `liquidate` also revert, completing the freeze. The contract balance is non-zero but inaccessible.

### Likelihood Explanation
Preconditions are minimal: deploy a token contract (trivial, no privilege required), call `touchMarket` (permissionless, no admin approval), and attract lenders (e.g., via a front-end or social engineering). The token behavior is deterministic and repeatable — every `withdraw` call reverts identically. No oracle manipulation, no admin key, no impossible state is required.

### Recommendation
Restrict the returndata length check to exactly 0 or 32 bytes before decoding:

```solidity
// safeTransfer
require(
    returndata.length == 0 || (returndata.length == 32 && abi.decode(returndata, (bool))),
    TransferReturnedFalse()
);

// safeTransferFrom
require(
    returndata.length == 0 || (returndata.length == 32 && abi.decode(returndata, (bool))),
    TransferFromReturnedFalse()
);
```

This rejects tokens returning non-standard-length data at the point of the first interaction (e.g., during `take`), preventing funds from ever entering a market whose loan token has broken return semantics, rather than silently accepting deposits and then freezing withdrawals.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import "../src/Midnight.sol";
import "../src/libraries/SafeTransferLib.sol";

/// transferFrom returns true (32 bytes); transfer returns abi.encode(0, 1) (64 bytes, first word zero)
contract MaliciousLoanToken {
    mapping(address => mapping(address => uint256)) public allowance;
    mapping(address => uint256) public balanceOf;

    function mint(address to, uint256 amount) external { balanceOf[to] += amount; }
    function approve(address spender, uint256 amount) external returns (bool) {
        allowance[msg.sender][spender] = amount;
        return true;
    }
    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        balanceOf[from] -= amount;
        balanceOf[to] += amount;
        return true; // standard 32-byte true
    }
    function transfer(address to, uint256 amount) external {
        balanceOf[msg.sender] -= amount;
        balanceOf[to] += amount;
        // Return 64 bytes: first word = 0 (false), second word = 1
        assembly {
            mstore(0x00, 0)
            mstore(0x20, 1)
            return(0x00, 0x40)
        }
    }
}

contract FundFreezeTest is Test {
    MaliciousLoanToken token;
    // ... setup Midnight, market, lender

    function testWithdrawAlwaysReverts() public {
        // 1. Deploy malicious token
        token = new MaliciousLoanToken();

        // 2. Create market with malicious loan token (touchMarket accepts any token)
        // ... construct Market struct with token as loanToken, valid collateral params

        // 3. Lender supplies via take (safeTransferFrom succeeds — transferFrom returns true)
        // ... take(...) succeeds, lender credit recorded

        // 4. Lender attempts withdraw
        vm.expectRevert(SafeTransferLib.TransferReturnedFalse.selector);
        // withdraw(...) — safeTransfer calls token.transfer → 64 bytes, first word 0 → false → revert
        midnight.withdraw(market, units, lender, lender);

        // Assert: lender credit still non-zero (state rolled back), funds locked
        assertGt(midnight.creditOf(marketId, lender), 0);
    }
}
```

**Expected assertion:** `withdraw` always reverts with `TransferReturnedFalse.selector`; lender credit remains non-zero; contract token balance is non-zero; funds are permanently inaccessible.
