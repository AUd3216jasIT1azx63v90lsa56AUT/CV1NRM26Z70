### Title
Unbounded `tokens[]` loop in `flashLoan` enables block-gas-consuming DoS - ([File: src/Midnight.sol])

### Summary
The `flashLoan` function accepts a caller-supplied `tokens[]` array of arbitrary length and iterates over it twice with no upper-bound check. Because each iteration performs an external ERC20 call, a sufficiently large array can consume the entire block gas limit. Unlike `touchMarket`, which enforces `MAX_COLLATERALS = 128` on its collateral array, `flashLoan` has no analogous cap.

### Finding Description
**Code path:**

`flashLoan` (src/Midnight.sol:737-752) is a permissionless `external` function. Its only input validation is:

```solidity
require(tokens.length == assets.length, InconsistentInput()); // line 740
```

No upper bound on `tokens.length` exists. The function then executes two full iterations:

- **Loop 1** (lines 742-744): `SafeTransferLib.safeTransfer(tokens[i], callback, assets[i])` — one external CALL per token.
- **Loop 2** (lines 749-751): `SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i])` — one external CALL per token.

**Attacker inputs:**
- `tokens`: array of N attacker-deployed minimal ERC20 stubs (each `transfer`/`transferFrom` returns `true` cheaply).
- `assets`: array of N zeros (zero-value transfers are accepted by standard ERC20s, so no protocol balance is required).
- `callback`: attacker-controlled contract that returns `CALLBACK_SUCCESS` immediately.
- `data`: empty.

**Exploit flow:**
1. Attacker deploys N cheap stub tokens and a stub callback.
2. Attacker calls `midnight.flashLoan(tokens_N, zeros_N, stubCallback, "")`.
3. Loop 1 fires N `safeTransfer` calls; each cold CALL costs ≥2,100 gas.
4. Callback returns `CALLBACK_SUCCESS`.
5. Loop 2 fires N `safeTransferFrom` calls; same cost.
6. At N ≈ 1,400 (2 × 1,400 × 2,100 ≈ 5.88 M gas, plus overhead), the transaction approaches the 30 M Ethereum block gas limit. At N ≈ 7,000 it fills a block entirely.

**Why existing checks fail:**
`touchMarket` enforces `MAX_COLLATERALS = 128` on its loop, but `flashLoan` has no equivalent constant. [1](#0-0) [2](#0-1) 

The Certora specs model `onFlashLoan` as a pure CVL stub and do not reason about gas consumption, so formal verification provides no protection here. [3](#0-2) 

### Impact Explanation
An unprivileged attacker can craft a single `flashLoan` call that consumes the full block gas limit (~30 M gas on Ethereum mainnet). Repeated across consecutive blocks, this prevents any other transaction — including time-critical liquidations, repayments, and settlements — from being included, constituting a sustained protocol-level DoS. The attacker pays their own gas but needs no protocol funds or permissions. [4](#0-3) 

### Likelihood Explanation
- **Preconditions:** None beyond deploying cheap stub contracts. No protocol balance, role, or signature is required.
- **Feasibility:** Fully reachable on any EVM chain where Midnight is deployed. The attacker controls all inputs.
- **Repeatability:** The attack can be repeated every block indefinitely at the cost of gas fees. On L2s with low gas prices the cost is negligible. [5](#0-4) 

### Recommendation
Add a `MAX_FLASH_TOKENS` constant (analogous to `MAX_COLLATERALS`) and enforce it at the top of `flashLoan`:

```solidity
uint256 constant MAX_FLASH_TOKENS = 32; // or another safe bound

function flashLoan(...) external {
    require(tokens.length == assets.length, InconsistentInput());
    require(tokens.length <= MAX_FLASH_TOKENS, TooManyTokens());
    ...
}
```

This mirrors the existing pattern in `touchMarket`: [6](#0-5) [7](#0-6)

```solidity
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";
import {IFlashLoanCallback} from "src/interfaces/ICallbacks.sol";

contract StubToken {
    function transfer(address, uint256) external pure returns (bool) { return true; }
    function transferFrom(address, address, uint256) external pure returns (bool) { return true; }
}

contract StubCallback is IFlashLoanCallback {
    function onFlashLoan(address, address[] memory, uint256[] memory, bytes memory)
        external pure returns (bytes32) { return CALLBACK_SUCCESS; }
}

contract FlashLoanGasGriefTest is Test {
    Midnight midnight;
    StubCallback cb;

    function setUp() public { /* deploy midnight, cb */ }

    /// @dev Fuzz tokens.length from 1 to 1000; assert gas stays under block limit.
    function testFuzz_flashLoanGas(uint16 n) public {
        n = uint16(bound(n, 1, 1000));
        address[] memory tokens = new address[](n);
        uint256[] memory assets = new uint256[](n);
        for (uint256 i = 0; i < n; i++) {
            tokens[i] = address(new StubToken());
            assets[i] = 0;
        }
        uint256 gasBefore = gasleft();
        midnight.flashLoan(tokens, assets, address(cb), "");
        uint256 gasUsed = gasBefore - gasleft();
        // Assertion: no single call should approach the 30M block gas limit.
        assertLt(gasUsed, 25_000_000, "flashLoan consumed near-block gas");
    }
}
```

**Expected result without fix:** the assertion fails for large `n` (e.g., n ≈ 7,000 fills a block; even n = 1,000 uses several million gas). **Expected result with fix:** the `require(tokens.length <= MAX_FLASH_TOKENS)` reverts before the loops for any `n` above the cap, and the gas assertion holds for all valid inputs.
