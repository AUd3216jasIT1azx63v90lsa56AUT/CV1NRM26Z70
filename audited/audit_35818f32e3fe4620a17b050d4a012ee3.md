### Title
Unguarded `initialize` allows anyone to deploy pools with arbitrary parameters, bypassing all factory safety checks — (`src/ConfidencePool.sol`)

### Summary
`ConfidencePool.initialize()` is `external` and protected only by the `initializer` modifier, which prevents re-initialization but does not restrict *who* may call it. Because `poolImplementation` is a public state variable on the factory, any attacker can read it, deploy their own EIP-1167 clone of the implementation, and call `initialize` directly with fully attacker-controlled parameters — including a malicious `safeHarborRegistry_` and a self-appointed `outcomeModerator_`. This bypasses every safety check the factory enforces and enables complete theft of staker funds.

### Finding Description
`ConfidencePoolFactory.createPool()` enforces four critical invariants before deploying and initializing a pool:

1. **Token allowlist** — `allowedStakeToken[stakeToken]` must be true.
2. **Agreement-owner gate** — `IAgreement(agreement).owner() == msg.sender`.
3. **Trusted registry** — always passes `address(safeHarborRegistry)` (the DAO-controlled singleton).
4. **Trusted moderator** — always passes `defaultOutcomeModerator` (the DAO). [1](#0-0) 

None of these invariants are enforced inside `ConfidencePool.initialize()` itself. The function is `external` with only the `initializer` modifier: [2](#0-1) 

`poolImplementation` is a public state variable readable by anyone: [3](#0-2) 

An attacker can therefore:

1. Read `factory.poolImplementation()`.
2. Call `Clones.clone(poolImplementation)` (or `cloneDeterministic` with any salt) to deploy a fresh, uninitialized clone.
3. Call `initialize()` on that clone with:
   - `safeHarborRegistry_` → a malicious contract whose `isAgreementValid` returns `true` and whose `getAttackRegistry()` returns a malicious registry that always reports `CORRUPTED`.
   - `outcomeModerator_` → the attacker's own address.
   - `recoveryAddress_` → the attacker's own address.
   - Any valid-looking `agreement_`, `stakeToken_`, `expiry_`, etc.

The only check `initialize` performs on the registry is `isAgreementValid`, which the malicious registry trivially satisfies: [4](#0-3) 

Once stakers have deposited, the attacker (as self-appointed moderator) calls `flagOutcome(CORRUPTED, false, address(0))`. The malicious registry returns `CORRUPTED`, so the state check passes: [5](#0-4) 

The attacker then calls `claimCorrupted()`, which sweeps the entire pool balance to `recoveryAddress` — the attacker's own address: [6](#0-5) 

### Impact Explanation
Complete theft of all staker principal and bonus funds deposited into the attacker-created pool. The pool is indistinguishable at the contract level from a legitimately factory-created pool (same bytecode, same interface, same events). The only difference is that it does not appear in `factory.getPoolsByAgreement()`, but a UI or aggregator scanning for `PoolCreated`-like events or checking the implementation address would surface it as a valid pool.

### Likelihood Explanation
Medium. The attack requires social engineering to attract stakers to a non-factory pool. However, the attacker controls all pool parameters and can make the pool appear legitimate (real agreement address, real stake token, plausible expiry). The `poolImplementation` address is public, and deploying an EIP-1167 clone requires no special access. A technically sophisticated attacker can execute this with a single script.

### Recommendation
Restrict `initialize` so it can only be called by the factory. The cleanest approach is to store the factory address in the implementation and add a check:

```solidity
// In initialize():
if (msg.sender != factory) revert NotFactory();
```

Alternatively, add a `factory` immutable set in the constructor (before `_disableInitializers`) and validate `msg.sender` against it in `initialize`. A lighter-weight option is to validate that `safeHarborRegistry_` matches a known-good address passed in from the factory, but this is weaker than a direct caller check.

### Proof of Concept

```solidity
// Attacker script (pseudocode)
address impl = factory.poolImplementation();

// 1. Deploy a clone without going through the factory
address maliciousPool = Clones.clone(impl);

// 2. Deploy a malicious registry that always returns CORRUPTED
MaliciousRegistry reg = new MaliciousRegistry(); // isAgreementValid→true, getAgreementState→CORRUPTED

// 3. Initialize with attacker-controlled parameters
IConfidencePool(maliciousPool).initialize(
    realAgreement,       // any valid-looking agreement address
    realStakeToken,      // any ERC20 to attract stakers
    address(reg),        // malicious registry
    attacker,            // attacker is the moderator
    block.timestamp + 31 days,
    1e18,
    attacker,            // attacker is the recovery address
    attacker,            // attacker is the pool owner
    scopeAccounts
);

// 4. Wait for stakers to deposit, then:
IConfidencePool(maliciousPool).flagOutcome(
    PoolStates.Outcome.CORRUPTED, false, address(0)
); // succeeds: malicious registry returns CORRUPTED

IConfidencePool(maliciousPool).claimCorrupted();
// All staker funds transferred to attacker's address
```

    function initialize(
        address agreement_,
        address stakeToken_,
        address safeHarborRegistry_,
        address outcomeModerator_,
        uint256 expiry_,
        uint256 minStake_,
        address recoveryAddress_,
        address owner_,
        address[] calldata accounts
    ) external initializer {
        if (agreement_ == address(0)) revert ZeroAddress();
        if (stakeToken_ == address(0)) revert ZeroAddress();
        if (safeHarborRegistry_ == address(0)) revert ZeroAddress();
        if (outcomeModerator_ == address(0)) revert ZeroAddress();
        if (owner_ == address(0)) revert ZeroAddress();
        if (recoveryAddress_ == address(0)) revert InvalidRecoveryAddress();
        if (expiry_ < block.timestamp + _MIN_EXPIRY_LEAD) revert ExpiryTooSoon();
        if (expiry_ > type(uint32).max) revert ExpiryTooFar();
        if (minStake_ == 0) revert InvalidAmount();
        // aderyn-fp-next-line(reentrancy-state-change)
        if (!IBattleChainSafeHarborRegistry(safeHarborRegistry_).isAgreementValid(agreement_)) {
            revert InvalidAgreement();
        }

        agreement = agreement_;
        stakeToken = IERC20(stakeToken_);
        safeHarborRegistry = IBattleChainSafeHarborRegistry(safeHarborRegistry_);
        outcomeModerator = outcomeModerator_;
        // forge-lint: disable-next-line(unsafe-typecast)
        expiry = uint32(expiry_);
        minStake = minStake_;
        recoveryAddress = recoveryAddress_;
        outcome = PoolStates.Outcome.UNRESOLVED;

        _replaceScope(accounts);

        // Direct assignment (skipping Ownable2Step's two-step) so no `owner() == initializer-caller`
        // window exists between init and the new owner accepting. Two-step still applies to later transfers.
        _transferOwnership(owner_);
    }
