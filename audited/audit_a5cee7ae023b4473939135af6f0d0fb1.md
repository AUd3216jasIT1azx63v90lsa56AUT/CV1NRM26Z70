### Title
Post-maturity no-op `liquidate(0,0)` repeatable via multicall triggers O(N×16) oracle calls with zero state change - (`src/Midnight.sol`)

### Summary
In post-maturity mode, after the first `liquidate(seizedAssets=0, repaidUnits=0)` call realizes partial bad debt, subsequent identical calls pass all guards (`_position.debt > 0`, `block.timestamp > market.maturity`) and each unconditionally execute the full `collateralBitmap` while-loop (up to 16 `IOracle.price()` calls) before returning `(0, 0)` with zero state change. Because `multicall` propagates reverts immediately and calls 2..N never revert, an attacker can compose a multicall of N such calls, causing exactly N×16 oracle calls in a single transaction bounded only by the block gas limit.

### Finding Description

**Exact code path:**

`multicall` at [1](#0-0)  uses `delegatecall` and propagates reverts with no try/catch, so every sub-call must succeed for the transaction to land.

Inside `liquidate`, the while-loop at [2](#0-1)  runs **unconditionally** before any liquidatability check, calling `IOracle(_collateralParam.oracle).price()` for every bit set in `_position.collateralBitmap` (up to `MAX_COLLATERALS_PER_BORROWER = 16`).

The `NotLiquidatable()` guard at [3](#0-2)  in post-maturity mode reduces to `block.timestamp > market.maturity` — a condition that is permanently true once maturity has passed, regardless of debt vs. maxDebt.

**Call-by-call trace (post-maturity, 16 collaterals, partial bad debt):**

- **Call 1**: `_position.debt > 0` ✓ → 16 oracle calls → `badDebt > 0` → `_position.debt -= badDebt` (partial: debt remains > 0) → `seizedAssets == 0 && repaidUnits == 0` skips seize block → `safeTransfer(token, receiver, 0)` and `safeTransferFrom(loanToken, payer, this, 0)` both succeed on well-behaved ERC20s → returns `(0, 0)`. [4](#0-3) [5](#0-4) [6](#0-5) 

- **Calls 2..N**: `_position.debt > 0` ✓ (partial bad debt left) → 16 oracle calls → `badDebt = 0` (remaining debt ≈ `collateral × price / maxLif`, the floor already realized) → `NotLiquidatable()` passes (`block.timestamp > market.maturity`) → `if (badDebt > 0)` skipped → `if (repaidUnits > 0 || seizedAssets > 0)` skipped → 0-amount token transfers succeed → returns `(0, 0)`. **Zero state change, 16 oracle calls consumed.**

**Why existing checks fail:**

| Check | Why it does not stop calls 2..N |
|---|---|
| `require(_position.debt > 0)` | Partial bad debt leaves debt > 0 after call 1 |
| `NotLiquidatable()` post-maturity | Reduces to `block.timestamp > market.maturity`, always true |
| `if (badDebt > 0)` | `badDebt == 0` on calls 2..N, block is skipped but loop already ran |
| `if (repaidUnits > 0 \|\| seizedAssets > 0)` | Both are 0, block skipped |
| `multicall` revert propagation | Calls 2..N do not revert, so multicall succeeds |

The `EcrecoverAuthorizer.setIsAuthorized` step in the stated call sequence is not required; `liquidate` has no authorization check on the caller beyond the optional `liquidatorGate`. [7](#0-6) 

### Impact Explanation
A single multicall transaction with N entries of `liquidate(0, 0, postMaturityMode=true)` against a borrower with 16 activated collaterals and partial bad debt triggers exactly N×16 external `IOracle.price()` calls. This is bounded only by the block gas limit (~30 M gas on Ethereum mainnet), not by any protocol-level cap. The attacker can saturate a block with oracle computation at the cost of their own gas, constituting concrete gas griefing of the liquidation path and potential block-space denial-of-service against competing liquidators.

### Likelihood Explanation
**Preconditions:**
1. Market is post-maturity (`block.timestamp > market.maturity`) — inevitable for every market.
2. Borrower has exactly partial bad debt: `debt > sum(collateral_i × price_i / maxLif_i)` but `debt < originalDebt` (i.e., not full bad debt where all oracle prices are 0). This is the common case when collateral prices drop significantly but not to zero.
3. Borrower has 16 activated collaterals — the maximum allowed, achievable by any borrower.
4. No `liquidatorGate` (or attacker passes it).

All preconditions are reachable by an unprivileged liquidator with no special access. The attack is repeatable across any post-maturity market with partial bad debt and is not blocked by any nonce, consumed counter, or state transition after call 1.

### Recommendation
Add an explicit no-op guard after the while-loop. If `badDebt == 0` and both `seizedAssets` and `repaidUnits` are zero, revert:

```solidity
// After the while-loop, before the NotLiquidatable check:
require(badDebt > 0 || repaidUnits > 0 || seizedAssets > 0, NoOpLiquidation());
```

This preserves the intentional "realize bad debt with 0 tokens transferred" feature (call 1 still works because `badDebt > 0`) while making calls 2..N revert before any oracle is queried on subsequent identical invocations.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
// ... standard test imports

contract GasGriefLiquidateTest is Test {
    // Setup: market with 16 collaterals, borrower with partial bad debt, post-maturity

    function testOracleCallsMultiplyWithMulticall(uint256 N) public {
        N = bound(N, 2, 50); // fuzz N

        // 1. Create market with 16 collaterals, each with a mock oracle that counts calls
        // 2. Supply all 16 collaterals for borrower, borrow units
        // 3. Drop oracle prices so debt > sum(collateral*price/maxLif) but not to 0
        //    (partial bad debt: some collateral value remains)
        // 4. Warp past maturity

        // 5. Build multicall with N liquidate(0, 0, postMaturityMode=true) calls
        bytes[] memory calls = new bytes[](N);
        for (uint256 i = 0; i < N; i++) {
            calls[i] = abi.encodeCall(
                midnight.liquidate,
                (market, 0, 0, 0, borrower, true, address(this), address(0), "")
            );
        }

        // 6. Execute multicall — must not revert
        midnight.multicall(calls);

        // 7. Assert oracle was called exactly N*16 times
        assertEq(mockOracle.callCount(), N * 16,
            "oracle called N*16 times: O(N) scaling confirmed");

        // 8. Assert no net state change after call 1
        // (debt unchanged from after call 1, collateral unchanged, lossFactor unchanged)
        assertEq(midnight.debtOf(id, borrower), debtAfterCall1,
            "debt unchanged after calls 2..N");
        assertEq(midnight.lossFactor(id), lossFactorAfterCall1,
            "lossFactor unchanged after calls 2..N");
    }
}
```

**Expected assertions:** `mockOracle.callCount() == N * 16` confirms O(N×16) scaling. `debtOf` and `lossFactor` unchanged after call 1 confirms calls 2..N are pure oracle-consuming no-ops.
