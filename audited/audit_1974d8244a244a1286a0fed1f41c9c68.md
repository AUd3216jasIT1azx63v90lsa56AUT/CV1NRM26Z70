Audit Report

## Title
`maxAssets` cap on buy offers bypassed via zero-rounding when `buyerPrice < WAD` - (File: `src/Midnight.sol`)

## Summary
When a buy offer uses `maxAssets > 0` as its fill cap, the consumed delta is `buyerAssets = mulDivDown(units, buyerPrice, WAD)`. For any tick where `buyerPrice < WAD` (all ticks below `MAX_TICK`), a taker submitting `units = 1` causes `buyerAssets` to floor to zero, so `consumed` never advances and the `ConsumedAssets` guard never fires. The offer can be filled an unlimited number of times, inflating the maker's credit and `totalUnits` without any corresponding asset transfer.

**Exploit flow:**
1. Maker posts a buy offer with `maxAssets = M > 0`, `maxUnits = 0`, and any tick where `buyerPrice < WAD` (e.g., `MAX_TICK - 16`).
2. Taker calls `take(offer, ..., units=1, ...)` N times.
3. Each call: `buyerAssets = mulDivDown(1, buyerPrice, WAD) = 0`; `consumed` stays at 0; guard passes; `buyerPos.credit += 1`; `sellerPos.debt += 1`; `totalUnits` incremented; zero tokens transferred.
4. After N calls: `consumed[maker][group] == 0`, maker's credit has grown by N, taker's debt has grown by N, with zero asset transfer.

## Impact Explanation

The `maxAssets` cap — the protocol's primary mechanism for limiting a buy offer's total fill exposure — is completely ineffective whenever `buyerPrice < WAD`, which is the normal operating range for all ticks below `MAX_TICK`. Consequences:

- **Unbacked credit inflation:** The maker accumulates credit (units) without any corresponding loan token deposit. At maturity, credit entitles the holder to receive loan tokens; unbacked credit inflates claims against the protocol's asset pool.
- **`totalUnits` inflation:** `_marketState.totalUnits` grows without bound, corrupting market-wide accounting used in fee calculations and withdrawals.
- **Accounting invariant broken:** `consumed` is supposed to track and bound total fill activity, but it never advances, so the invariant `consumed <= maxAssets` is vacuously satisfied while actual fill activity is unbounded.
- **Taker self-harm / griefing vector:** A malicious actor controlling both sides can manufacture unbounded credit for the maker at the cost of unbounded debt on the taker address, then exploit the maker's inflated credit in downstream interactions.

## Likelihood Explanation

**Preconditions:**
1. A buy offer with `maxAssets > 0` and any tick below `MAX_TICK` — the default operating range for all practical prices below 1 WAD per unit.
2. The taker is any unprivileged address — no special role, no admin key, no oracle manipulation required.
3. The taker must supply collateral for the debt they take on, but the token cost per fill is zero.

**Feasibility:** Trivially repeatable. The only cost is gas. The condition `buyerPrice < WAD` is the default operating range. No external dependencies or timing constraints are required.

## Recommendation

Add a minimum-assets floor guard before the consumed update. Specifically, for a buy offer with `maxAssets > 0`, require that `buyerAssets > 0` before proceeding, or require that `units` is large enough that `mulDivDown(units, buyerPrice, WAD) > 0`. A concrete fix:

```solidity
if (offer.maxAssets > 0) {
    uint256 delta = offer.buy ? buyerAssets : sellerAssets;
    require(delta > 0, ZeroAssetDelta()); // prevent zero-rounding bypass
    newConsumed = consumed[offer.maker][offer.group] += delta;
    require(newConsumed <= offer.maxAssets, ConsumedAssets());
}
```

Alternatively, enforce a minimum `units` value such that the asset delta is always nonzero when `maxAssets > 0`.

## Proof of Concept

The existing test `testBugBuyMaxAssetsBypass` in `test/TakeTest.sol` at lines 858–889 is a complete, passing proof of concept. It:
1. Sets `maxAssets = 1`, `maxUnits = 0`, `tick = MAX_TICK - 16`.
2. Pre-sets `consumed` to `maxAssets` via `setConsumed`.
3. Calls `take(1, borrower, lenderOffer)`.
4. Asserts `buyerAssets == 0`, `sellerAssets == 0`, `consumed` unchanged at `maxAssets`.
5. Asserts `creditOf(lender)`, `debtOf(borrower)`, and `totalUnits` all strictly increased.
