### Title
Fee-on-Transfer `loanToken` Breaks Balance Invariant in `take()` and `repay()` - (`src/Midnight.sol`)

### Summary

`touchMarket()` is permissionless and validates only collateral parameters, maturity, LLTV, and `maxLif` — it performs no check on `loanToken` behavior. `SafeTransferLib.safeTransferFrom()` only validates the ERC20 return value, not the actual amount received. When a fee-on-transfer token is used as `loanToken`, every inbound transfer to `address(this)` delivers less than the requested amount, while `claimableSettlementFee` and `withdrawable` are incremented by the full nominal amount, permanently breaking the invariant that contract token balances must cover all withdrawable assets and claimable settlement fees.

### Finding Description

**Code path — `take()`:** [1](#0-0) 

`touchMarket()` is `public` with no restriction on `loanToken`. Any caller can create a market whose `loanToken` is a fee-on-transfer token. [2](#0-1) 

`claimableSettlementFee[offer.market.loanToken]` is incremented by the full nominal `buyerAssets - sellerAssets` before the transfer executes. [3](#0-2) 

`SafeTransferLib.safeTransferFrom()` is then called for `buyerAssets - sellerAssets` into `address(this)`. With a 1% fee-on-transfer token, only `(buyerAssets - sellerAssets) * 0.99` arrives. [4](#0-3) 

`SafeTransferLib` checks only the boolean return value — it does not compare pre/post balances. The shortfall is silently accepted.

**Code path — `repay()`:** [5](#0-4) 

`marketState[id].withdrawable += units` is recorded before the transfer. With a fee-on-transfer token, only `units * (1 - fee%)` arrives, but `withdrawable` reflects the full `units`.

**Root cause:** `IdLib.toId()` encodes `loanToken` by address only. [6](#0-5) 

There is no mechanism anywhere in the protocol to reject or detect fee-on-transfer tokens at market creation or at transfer time.

**Exploit flow:**
1. Attacker deploys `FeeToken` (ERC20 with 1% transfer fee, returns `true`).
2. Calls `midnight.touchMarket(market)` with `market.loanToken = address(feeToken)` — succeeds.
3. Lender signs a buy offer; taker calls `take()` with `units = 1e18`.
   - `claimableSettlementFee[feeToken] += settlementFeeAmount` (full amount).
   - Contract receives only `settlementFeeAmount * 0.99`.
   - Deficit after `take()`: `settlementFeeAmount * 0.01`.
4. Borrower calls `repay(market, units, ...)`.
   - `withdrawable += units` (full amount).
   - Contract receives only `units * 0.99`.
   - Deficit after `repay()`: `units * 0.01`.
5. `withdrawable + claimableSettlementFee[feeToken] > IERC20(feeToken).balanceOf(address(midnight))`.
6. Last lenders to call `withdraw()` or feeClaimer to call `claimSettlementFee()` receive a revert due to insufficient balance.

### Impact Explanation

The contract's accounting records more `loanToken` as owed (via `withdrawable` and `claimableSettlementFee`) than it actually holds. This is protocol insolvency: the sum `withdrawable + claimableSettlementFee[loanToken]` exceeds the actual ERC20 balance. Lenders and the fee claimer compete for a shortfall; the last to withdraw lose funds. The deficit compounds with every `take()`, `repay()`, and `liquidate()` call on the affected market.

### Likelihood Explanation

The precondition is entirely attacker-controlled: deploy a fee-on-transfer ERC20 (trivial), call the permissionless `touchMarket()`, and execute normal protocol operations. No privileged access, no oracle manipulation, and no user mistake is required. The attack is repeatable on every `take()` and `repay()` call in the market. Any market participant (lender, borrower, or third party) can trigger it.

### Recommendation

Add a balance-before / balance-after check in every function that pulls `loanToken` into the contract (`take()`, `repay()`, `liquidate()`), and require that the received amount equals the expected amount:

```solidity
uint256 balanceBefore = IERC20(token).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(token, payer, address(this), amount);
require(IERC20(token).balanceOf(address(this)) - balanceBefore == amount, UnexpectedTokenAmount());
```

Alternatively, document that fee-on-transfer tokens are unsupported and add a check in `touchMarket()` that performs a test transfer to detect fee-on-transfer behavior, or maintain an allowlist of valid `loanToken` addresses.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, Offer, CollateralParams} from "src/interfaces/IMidnight.sol";

contract FeeToken is ERC20 {
    constructor() ERC20("FeeToken", "FT") { _mint(msg.sender, 1e30); }
    function transferFrom(address from, address to, uint256 amount) public override returns (bool) {
        uint256 fee = amount / 100; // 1% fee burned
        super.transferFrom(from, to, amount - fee);
        _burn(to, 0); // fee is simply not transferred (simulates deflation)
        return true;
    }
    // transfer similarly deducts 1%
}

contract FeeOnTransferTest is Test {
    Midnight midnight;
    FeeToken feeToken;
    // ... setup market, lender offer, borrower take ...

    function testFeeOnTransferBreaksInvariant() public {
        // 1. Create market with feeToken as loanToken
        // 2. Lender approves, taker calls take(offer, ..., 1e18, ...)
        // 3. Borrower calls repay(market, 1e18, ...)
        
        uint256 balance = feeToken.balanceOf(address(midnight));
        uint256 withdrawableAmt = midnight.withdrawable(id);
        uint256 claimableFee = midnight.claimableSettlementFee(address(feeToken));
        
        // Assert: invariant broken
        assertLt(balance, withdrawableAmt + claimableFee,
            "balance must be less than withdrawable + claimableSettlementFee");
    }
}
```

Expected: `balance < withdrawable + claimableSettlementFee[feeToken]`, confirming protocol insolvency.

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }

        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
