### Title
Unprivileged taker can drain arbitrary victim's loanToken by setting takerCallback to victim on sell offers - (File: src/Midnight.sol)

### Summary
In `Midnight.take()`, for sell offers (`offer.buy = false`), the `takerCallback` parameter supplied by the taker is used directly as both the `buyerCallback` (callback recipient) and the `payer` for all loanToken transfers. There is no check that `takerCallback` is the taker or authorized by the taker. An attacker can set `takerCallback` to any victim contract that has approved Midnight and returns `CALLBACK_SUCCESS` from `onBuy`, causing the protocol to pull the full `buyerAssets` from the victim without the victim's consent for this specific take.

### Finding Description
**Code path:**

1. Attacker calls `Midnight.take(offer, ratifierData, units, taker, receiver, victimAddress, callbackData)` where `offer.buy = false` and `offer.callback = address(0)`.
2. Line 346 checks `taker == msg.sender` — passes (attacker is taker).
3. Line 356 calls `EcrecoverRatifier.isRatified(offer, ratifierData)` — passes because the ratifier only validates the signed offer fields; `takerCallback` is not part of the offer struct and is never signed.
4. Line 420: `buyerCallback = takerCallback = victimAddress`.
5. Line 422: `payer = buyerCallback = victimAddress` (since `buyerCallback != address(0)`).
6. Lines 445–452: `IBuyCallback(victimAddress).onBuy(...)` is called. If victim returns `CALLBACK_SUCCESS` (precondition), execution continues.
7. Lines 455–456: `safeTransferFrom(loanToken, victimAddress, address(this), buyerAssets - sellerAssets)` and `safeTransferFrom(loanToken, victimAddress, receiver, sellerAssets)` — full `buyerAssets` drained from victim.

**Root cause:** `takerCallback` is an unconstrained attacker-controlled input. For sell offers it becomes `payer` with no authorization check tying it to `taker` or any consenting party.

**Why existing checks fail:** The taker authorization check (line 346) only validates `msg.sender` vs `taker`. The ratifier (EcrecoverRatifier) only validates the signed offer struct — `takerCallback` is not a field in `Offer` and is never included in the EIP-712 digest. No other check in `take()` validates `takerCallback`.

### Impact Explanation
Direct loss of victim's loanToken balance up to `buyerAssets` per take call. The attacker receives `sellerAssets` at `receiver` (their chosen address) and the protocol receives the settlement fee portion (`buyerAssets - sellerAssets`). The victim's balance decreases by `buyerAssets` without having consented to this specific take. The core invariant — "signatures, gates, ratifiers, callbacks, and approvals bind the right user/market/action/amount/deadline" — is violated.

### Likelihood Explanation
**Preconditions:**
- A valid sell offer signed via EcrecoverRatifier must exist (normal protocol operation).
- The victim must be a contract that: (a) has `loanToken.approve(midnight, type(uint256).max)` or sufficient allowance, and (b) implements `IBuyCallback` and returns `CALLBACK_SUCCESS` from `onBuy` without validating `buyer == address(this)`.

Condition (b) is realistic for any protocol integration designed to act as a buyer in Midnight (e.g., vaults, aggregators, router contracts). Such contracts naturally pre-approve Midnight and implement the callback interface. If their `onBuy` does not assert `buyer == address(this)`, they are exploitable. The attack is repeatable as long as the victim's allowance and balance remain, and requires no special privileges — any address can be the taker.

### Recommendation
Add a validation that `takerCallback` must be either `address(0)`, `taker`, or an address authorized by `taker`:

```solidity
require(
    takerCallback == address(0) || takerCallback == taker || isAuthorized[taker][takerCallback],
    TakerCallbackUnauthorized()
);
```

This ensures the payer for a taker-side callback is always a party that has explicitly consented (by being the taker or by being authorized by the taker).

### Proof of Concept
```solidity

pragma solidity ^0.8.0;

contract VictimBuyer is IBuyCallback {
    // Pre-condition: loanToken.approve(midnight, type(uint256).max) called in setup
    function onBuy(...) external returns (bytes32) {
        return CALLBACK_SUCCESS; // naive implementation, no buyer == address(this) check
    }
}

contract ExploitTest is Test {
    function test_drainVictimViaTakerCallback() public {
        // Setup: deploy victim, fund with loanToken, approve Midnight
        uint256 victimBalanceBefore = loanToken.balanceOf(address(victim));

        // Attacker takes a valid sell offer, setting takerCallback = victim
        midnight.take(
            sellOffer,       // offer.buy = false, offer.callback = address(0)
            ratifierData,    // valid EcrecoverRatifier proof
            units,
            attacker,        // taker = msg.sender
            attackerReceiver,
            address(victim), // takerCallback = victim (payer)
            ""
        );

        uint256 victimBalanceAfter = loanToken.balanceOf(address(victim));

        // Assert victim lost buyerAssets without consent
        assertEq(victimBalanceBefore - victimBalanceAfter, buyerAssets);
        // Assert attacker's receiver received sellerAssets
        assertEq(loanToken.balanceOf(attackerReceiver), sellerAssets);
    }
}
```

**Expected assertions:** `victim.balanceBefore - victim.balanceAfter == buyerAssets`; attacker's receiver balance increases by `sellerAssets`; Midnight's balance increases by `buyerAssets - sellerAssets` (settlement fee). A fuzz variant should iterate `takerCallback` over all addresses with nonzero allowance and assert the invariant holds — it will fail for any victim satisfying the preconditions.

    function isRatified(Offer memory offer, bytes memory ratifierData) external view returns (bytes32) {
        require(msg.sender == MIDNIGHT, NotMidnight());
        (Signature memory sig, bytes32 root, uint256 leafIndex, bytes32[] memory proof) =
            abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]));
        require(HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof), InvalidProof());
        require(!isRootCanceled[offer.maker][root], RootCanceled());
        bytes32 structHash = keccak256(abi.encode(HashLib.offerTreeTypeHash(proof.length), root));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, structHash));
        address _signer = ecrecover(digest, sig.v, sig.r, sig.s);
        require(_signer != address(0), InvalidSignature());
        require(_signer == offer.maker || IMidnight(MIDNIGHT).isAuthorized(offer.maker, _signer), Unauthorized());
        return CALLBACK_SUCCESS;
    }
