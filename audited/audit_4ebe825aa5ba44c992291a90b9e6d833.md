### Title
Unbounded `takes[0]` Access Without Length Guard Causes Panic Revert in All Bundle Functions - (File: src/periphery/MidnightBundles.sol)

### Summary
All four `takes`-accepting functions in `MidnightBundles` unconditionally dereference `takes[0]` immediately after the authorization and fee checks, with no prior `takes.length > 0` guard. Passing an empty `takes[]` array triggers a Solidity array-out-of-bounds panic (error code `0x32`) rather than a protocol-defined revert, wasting all gas supplied to the call. Because Midnight exposes a `multicall` function, any transaction that sequences state-changing Midnight calls before a MidnightBundles call will have those changes rolled back by the propagating panic.

### Finding Description
**Affected lines (all in `src/periphery/MidnightBundles.sol`):**

| Function | Unguarded access |
|---|---|
| `buyWithUnitsTargetAndWithdrawCollateral` | line 62: `address loanToken = takes[0].offer.market.loanToken;` |
| `supplyCollateralAndSellWithUnitsTarget` | line 129: `address loanToken = takes[0].offer.market.loanToken;` |
| `buyWithAssetsTargetAndWithdrawCollateral` | line 193: `address loanToken = takes[0].offer.market.loanToken;` |
| `supplyCollateralAndSellWithAssetsTarget` | line 264: `address loanToken = takes[0].offer.market.loanToken;` |

In every case the execution path is:

1. `require(taker == msg.sender || isAuthorized(taker, msg.sender))` — passes when attacker is an authorized operator.
2. `require(referralFeePct < WAD)` — passes with `referralFeePct = 0`.
3. `takes[0].offer.market.loanToken` — **panics** (EVM reverts with `Panic(0x32)`) when `takes.length == 0`.

No existing check stops this. The `OutOfOffers` guard at line 88 / 163 / 224 / 303 is never reached.

**Exploit flow:**
- Attacker is an authorized operator for `taker` (granted via `midnight.setIsAuthorized`).
- Attacker calls `buyWithUnitsTargetAndWithdrawCollateral(targetUnits, maxBuyerAssets, taker, permit, new Take[](0), new CollateralWithdrawal[](0), receiver, 0, address(0))`.
- Execution panics at line 62 with `Panic(0x32)`.
- All gas forwarded to the call is consumed; none is refunded.
- If this call is embedded in a `midnight.multicall(data)` sequence (confirmed present in `src/Midnight.sol` and tested in `test/MulticallTest.sol`), the panic propagates and rolls back every state change made by earlier entries in the multicall batch. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation
An authorized operator can force any of the four bundle functions to revert with a raw panic at negligible cost (empty calldata array, no token approval needed, no collateral required). When composed inside a `midnight.multicall` batch, the panic rolls back all preceding state changes in that batch — collateral supplies, repayments, or other operations already executed — while consuming all forwarded gas. The result is a low-cost, repeatable denial-of-service against any multicall that includes a MidnightBundles call. [5](#0-4) 

### Likelihood Explanation
**Preconditions:** The attacker must hold operator authorization for the target taker (`isAuthorized(taker, attacker) == true`). This is a user-granted privilege, not a protocol role, so it is a realistic precondition in any integration where a third-party contract or EOA is authorized as an operator. The call itself requires no tokens, no collateral, and no signatures beyond the authorization already in place. It is repeatable at will for as long as the authorization holds.

### Recommendation
Add an explicit length guard before the first `takes[0]` access in each of the four functions:

```solidity
require(takes.length > 0, EmptyTakes());
```

Place this check immediately after the `referralFeePct` check and before the `takes[0]` dereference (lines 62, 129, 193, 264). Define a corresponding `error EmptyTakes()` in `IMidnightBundles`. [6](#0-5) 

### Proof of Concept
```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {MidnightBundlesTest} from "./MidnightBundlesTest.sol";
import {Take, CollateralWithdrawal, TokenPermit} from "../src/periphery/interfaces/IMidnightBundles.sol";

contract EmptyTakesPanicTest is MidnightBundlesTest {

    // Attacker is an authorized operator for `lender` (set up in setUp()).
    // Calls buyWithUnitsTargetAndWithdrawCollateral with takes = [].
    // Expected: revert with Panic(0x32) — NOT OutOfOffers, NOT Unauthorized.
    function testEmptyTakesPanics_buy_units() public {
        vm.prank(address(this)); // address(this) is authorized for lender in setUp
        vm.expectRevert(); // Panic(0x32) — no selector, raw panic
        midnightBundles.buyWithUnitsTargetAndWithdrawCollateral(
            1, 0, lender, _noPermit(),
            new Take[](0), new CollateralWithdrawal[](0),
            address(0), 0, address(0)
        );
    }

    function testEmptyTakesPanics_sell_units() public {
        vm.prank(borrower);
        vm.expectRevert();
        midnightBundles.supplyCollateralAndSellWithUnitsTarget(
            1, 0, borrower, borrower,
            new CollateralSupply[](0), new Take[](0),
            0, address(0)
        );
    }

    // Assert the revert is a panic, not a named error:
    function testEmptyTakesPanicSelector() public {
        (bool ok, bytes memory ret) = address(midnightBundles).call(
            abi.encodeCall(
                midnightBundles.buyWithUnitsTargetAndWithdrawCollateral,
                (1, 0, lender, _noPermit(),
                 new Take[](0), new CollateralWithdrawal[](0),
                 address(0), 0, address(0))
            )
        );
        assertFalse(ok);
        // Panic(0x32) ABI-encodes to: 0x4e487b71 ++ abi.encode(0x32)
        assertEq(bytes4(ret), bytes4(0x4e487b71), "must be Panic selector");
        assertEq(abi.decode(ret[4:], (uint256)), 0x32, "must be out-of-bounds panic");
    }
}
```

**Expected assertions:**
- All calls revert.
- The revert data carries selector `0x4e487b71` (Solidity `Panic`) with code `0x32` (array out-of-bounds), not `OutOfOffers` (`IMidnightBundles` custom error).
- After the fix (`require(takes.length > 0, EmptyTakes())`), the same calls revert with `EmptyTakes()` instead of a panic.
