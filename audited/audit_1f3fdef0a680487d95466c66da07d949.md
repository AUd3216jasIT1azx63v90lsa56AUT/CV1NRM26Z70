### Title
Double-Rounding in `isHealthy()` Causes False Unhealthy Result, Blocking Valid `take()` - (`src/Midnight.sol`)

### Summary

`isHealthy()` computes `maxDebt` using two sequential `mulDivDown` calls — first dividing by `ORACLE_PRICE_SCALE`, then by `WAD` — introducing a compounded truncation error of up to 1 unit per collateral. When a seller's position sits at the exact health boundary, this double-rounding can produce `maxDebt = maxDebt_exact - 1 < debt`, causing `take()` to revert with `SellerIsLiquidatable` even though the position is healthy under single-step exact arithmetic.

### Finding Description

**Code path:**

`isHealthy()` at `src/Midnight.sol:954-955`:
```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
```

`mulDivDown` is `(x * y) / d` (plain integer division, `src/libraries/UtilsLib.sol:29-31`).

The two-step computation is:
1. `intermediate = ⌊collateral × price / ORACLE_PRICE_SCALE⌋`
2. `maxDebt = ⌊intermediate × lltv / WAD⌋`

The exact single-step value is:
`maxDebt_exact = ⌊collateral × price × lltv / (ORACLE_PRICE_SCALE × WAD)⌋`

**Why double-rounding loses 1 unit:**

Let `collateral × price = q₁ × ORACLE_PRICE_SCALE + r₁` (r₁ > 0) and `q₁ × lltv = q₂ × WAD + r₂` (r₂ > 0). The single-step numerator expands to `q₂ × ORACLE_PRICE_SCALE × WAD + r₂ × ORACLE_PRICE_SCALE + r₁ × lltv`. When `r₂ × ORACLE_PRICE_SCALE + r₁ × lltv ≥ ORACLE_PRICE_SCALE × WAD`, the single-step floor is `q₂ + 1` but the double-step gives `q₂`.

**Concrete counterexample** (lltv = `LLTV_7 = 0.98e18`, allowed by `isLltvAllowed`):
- `collateral = 2`, `price = 1e36 − 1`
- Step 1: `⌊2 × (1e36−1) / 1e36⌋ = ⌊2 − 2/1e36⌋ = 1`
- Step 2: `⌊1 × 0.98e18 / 1e18⌋ = ⌊0.98⌋ = 0`
- Exact: `⌊2 × (1e36−1) × 0.98e18 / (1e36 × 1e18)⌋ = ⌊1.96 − ε⌋ = 1`

With `debt = 1`: `maxDebt = 0 < 1 = debt` → `isHealthy()` returns `false`, but the position is healthy by exact arithmetic.

**Exploit flow:**
1. Seller (or authorized operator via `EcrecoverAuthorizer.setIsAuthorized` → `Midnight.setIsAuthorized`) supplies `collateral = 2` wei of collateral token.
2. Oracle is set to return `price = 1e36 − 1`.
3. Seller calls (or operator calls on behalf of seller) `take()` with a buy offer for `units = 1`.
4. `sellerDebtIncrease = 1`; seller's debt becomes 1.
5. `take()` reaches line 476: `require(liquidationLocked(id, seller) || isHealthy(...), SellerIsLiquidatable())`.
6. `liquidationLocked` is `false` (lock was released at line 475 since `wasLocked = false`).
7. `isHealthy()` returns `false` due to double-rounding → revert with `SellerIsLiquidatable`.

**Why existing checks do not stop it:**

The `liquidationLocked` bypass at line 475-476 only helps during callbacks. The `CannotIncreaseDebtPostMaturity` check is unrelated. There is no tolerance or single-step fallback in `isHealthy()`. The Certora `Healthiness.spec` proofs axiomatize `mulDivDown` as a ghost function and do not capture the concrete two-step rounding loss.

### Impact Explanation

A seller whose position is exactly at the health boundary after a take — healthy by exact arithmetic — has their `take()` call reverted with `SellerIsLiquidatable`. The seller cannot borrow the maximum amount permitted by their collateral. With multiple activated collaterals the rounding loss accumulates (up to 1 unit per collateral, up to 16 collaterals = 16 wei), widening the gap between the exact boundary and what `isHealthy()` accepts.

### Likelihood Explanation

Requires the seller's post-take debt to equal exactly `maxDebt_exact` (the single-step floor). This is a boundary condition that can be engineered: the seller controls collateral amount, the oracle price is observable on-chain, and `units` in `take()` is caller-chosen. With small token amounts (wei-scale) or tokens with few decimals the boundary is easy to hit. The condition is repeatable across any market with any allowed LLTV tier.

### Recommendation

Replace the two-step `mulDivDown` chain with a single-step computation using a 512-bit intermediate (e.g., Solady `FullMath.mulDiv`) to avoid the intermediate truncation:

```solidity
// Instead of:
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);

// Use (requires overflow-safe 512-bit mulDiv):
maxDebt += FullMath.mulDiv(
    _position.collateral[i],
    price * collateralParam.lltv,   // may overflow — use 512-bit path
    ORACLE_PRICE_SCALE * WAD
);
```

Alternatively, apply the same fix used in Morpho Blue: compute `collateral × price × lltv` in a single `mulDiv` with a 512-bit intermediate, or add a per-collateral tolerance of `+1` to `maxDebt` before comparing with `debt` (accepting the minor loosening of the health boundary).

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {IMidnight, Market, CollateralParams, Offer} from "src/interfaces/IMidnight.sol";
import {ORACLE_PRICE_SCALE, WAD, LLTV_7} from "src/libraries/ConstantsLib.sol";
import {UtilsLib} from "src/libraries/UtilsLib.sol";

contract DoubleRoundingPoC is Test {
    using UtilsLib for uint256;

    // Fuzz: find (collateral, price, lltv) where double-rounding loses 1 unit
    function testFuzz_doubleRoundingFalseUnhealthy(
        uint128 collateral,
        uint256 price,
        uint8 lltvIndex
    ) public pure {
        uint256[9] memory tiers = [
            uint256(0.385e18), 0.625e18, 0.77e18, 0.86e18,
            0.915e18, 0.945e18, 0.965e18, 0.98e18, 1e18
        ];
        uint256 lltv = tiers[lltvIndex % 9];
        vm.assume(price > 0 && collateral > 0);

        uint256 intermediate = uint256(collateral).mulDivDown(price, ORACLE_PRICE_SCALE);
        uint256 doubleStep   = intermediate.mulDivDown(lltv, WAD);

        // Single-step exact (may overflow for large values — bound inputs accordingly)
        // Use 512-bit math in real test; here illustrate the gap
        uint256 singleStep;
        unchecked {
            // Only valid when collateral*price fits uint256 — fuzz will find small cases
            if (price <= type(uint256).max / collateral) {
                singleStep = (uint256(collateral) * price).mulDivDown(lltv, ORACLE_PRICE_SCALE * WAD);
            } else {
                return; // skip overflow cases
            }
        }

        // Assert: double-step must equal single-step (invariant under test)
        // This WILL fail for e.g. collateral=2, price=1e36-1, lltv=0.98e18
        assertEq(doubleStep, singleStep, "double-rounding loss detected");
    }

    // Unit test with the concrete counterexample
    function test_concreteDoubleRoundingLoss() public pure {
        uint256 collateral = 2;
        uint256 price      = 1e36 - 1;
        uint256 lltv       = 0.98e18; // LLTV_7

        uint256 intermediate = collateral.mulDivDown(price, ORACLE_PRICE_SCALE);
        assertEq(intermediate, 1, "step1");

        uint256 doubleStep = intermediate.mulDivDown(lltv, WAD);
        assertEq(doubleStep, 0, "step2 double-rounding gives 0");

        // Exact single-step: floor(2*(1e36-1)*0.98e18 / (1e36*1e18)) = floor(1.96-eps) = 1
        // With debt=1: isHealthy returns false (0 < 1) but exact says healthy (1 >= 1)
        uint256 debt = 1;
        bool healthyExact  = (doubleStep + 1) >= debt; // single-step = 1
        bool healthyActual = doubleStep >= debt;        // double-step = 0

        assertTrue(healthyExact,   "healthy by exact arithmetic");
        assertFalse(healthyActual, "unhealthy by double-rounding — take() reverts");
    }
}
```

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
