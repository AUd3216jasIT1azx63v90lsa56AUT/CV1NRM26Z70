### Title
Dust-collateral rounding in `isHealthy` enables liquidation of economically healthy positions - (`src/Midnight.sol`)

### Summary

`isHealthy` computes `maxDebt` via two chained `mulDivDown` calls: `collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE).mulDivDown(lltv, WAD)`. When `collateral[i] = 1` wei, the intermediate result of the first division can be a small integer (e.g., 1) that the second division rounds to 0, yielding `maxDebt = 0` even though the position's true economic collateral value exceeds its debt. Because `liquidate` uses the identical rounding path and checks `originalDebt > maxDebt`, a position with 1 wei of collateral and 1 unit of debt passes the liquidatability check when it is in fact overcollateralized.

### Finding Description

**Exact code path:**

`isHealthy` at `src/Midnight.sol:954–955`:
```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [1](#0-0) 

`withdrawCollateral` at `src/Midnight.sol:564–568` only clears the bitmap bit when `newCollateral == 0`; leaving 1 wei keeps the bit set and the oracle is queried in every subsequent health check: [2](#0-1) 

`liquidate` at `src/Midnight.sol:620–624` uses the same rounding and checks `originalDebt > maxDebt`: [3](#0-2) 

`ORACLE_PRICE_SCALE = 1e36`, `WAD = 1e18`: [4](#0-3) 

**Concrete arithmetic (lltv = 0.77e18, LLTV_2):**

*At withdrawal time — price = 2e36:*
- `mulDivDown(1, 2e36, 1e36) = 2`
- `mulDivDown(2, 0.77e18, 1e18) = floor(1.54) = 1`
- `maxDebt = 1 >= debt = 1` → `isHealthy` returns true → `withdrawCollateral` succeeds, leaving 1 wei of collateral with bitmap bit set.

*After oracle price drops to 1.5e36:*
- `mulDivDown(1, 1.5e36, 1e36) = 1`
- `mulDivDown(1, 0.77e18, 1e18) = floor(0.77) = 0`
- `maxDebt = 0 < debt = 1` → `isHealthy` returns false.

*True economic health at 1.5e36:*
- Collateral value = 1 × 1.5e36 / 1e36 = 1.5 loan-token units
- True max debt = 1.5 × 0.77 = 1.155 loan-token units
- Debt = 1 unit → position IS genuinely overcollateralized (1 < 1.155).

**Why existing checks fail:**

`withdrawCollateral` checks `isHealthy` only at withdrawal time (price = 2e36), which passes. There is no minimum-collateral floor, no dust guard, and no check that prevents a subsequent price drop from triggering the rounding collapse. The `liquidate` path recomputes `maxDebt` with the same two-step `mulDivDown` and checks `originalDebt > maxDebt` (1 > 0), which passes unconditionally.

**Attacker inputs:**
- `collateralIndex` = any valid index
- `assets` = `initial_collateral − 1` (leaving 1 wei)
- Oracle price at withdrawal ≥ 2e36; oracle price at liquidation in `(1e36 / lltv, 2e36)` — a normal market price drop, not an impossible value.

### Impact Explanation

A liquidator can seize 1 wei of collateral (worth 1.5 loan-token units at the example price) by repaying 1 unit of debt, netting a profit of ~0.5 loan-token units. The borrower loses collateral that the protocol's own health formula should have protected. The invariant "healthy positions are not liquidatable" is violated: the position satisfies `collateral_value × lltv > debt` in real arithmetic but fails the on-chain check due to integer rounding on dust collateral.

### Likelihood Explanation

**Preconditions:**
1. Borrower reduces collateral to 1 wei via `withdrawCollateral` — reachable by any borrower who supplies a large amount and withdraws `amount − 1`.
2. Oracle price at withdrawal ≥ `ceil(WAD / lltv) × ORACLE_PRICE_SCALE` (e.g., ≥ 2e36 for lltv = 0.77e18).
3. Oracle price later drops into `(ORACLE_PRICE_SCALE / lltv, ceil(WAD/lltv) × ORACLE_PRICE_SCALE)` — a ~33% price drop for lltv = 0.77e18, a routine market event.

The vulnerability is repeatable for any market with lltv < WAD (all tiers except LLTV_8 = 1e18) and any collateral token whose oracle price can be in the described range. No privileged access is required.

### Recommendation

Add a minimum collateral threshold (dust guard) in `withdrawCollateral` so that if the borrower has debt, `collateral[i]` after withdrawal must either be zero (bitmap cleared) or exceed a protocol-defined minimum (e.g., `ORACLE_PRICE_SCALE / lltv` wei) sufficient to produce a non-zero `maxDebt` contribution. Alternatively, compute `maxDebt` using a single combined `mulDivDown(collateral * lltv, price, ORACLE_PRICE_SCALE * WAD)` to avoid the intermediate truncation, or add a check that any non-zero collateral with a non-zero oracle price contributes at least 1 to `maxDebt`.

contract DustCollateralRoundingTest is BaseTest {
    function testDustCollateralRoundingLiquidation() public {
        // Setup: lltv = 0.77e18, oracle price = 2e36

        Market memory mkt = /* build market with lltv=0.77e18 */;
        bytes32 id = toId(mkt);

        // Borrower supplies 1e18 collateral
        deal(address(collateralToken1), borrower, 1e18);
        vm.prank(borrower);
        midnight.supplyCollateral(mkt, 0, 1e18, borrower);

        // Borrower borrows 1 unit of debt (via take)
        // ... setup offer and take to get debt = 1

        // Borrower withdraws 1e18 - 1, leaving 1 wei
        // At price=2e36: mulDivDown(1,2e36,1e36)=2, mulDivDown(2,0.77e18,1e18)=1 >= debt=1 → passes
        vm.prank(borrower);
        midnight.withdrawCollateral(mkt, 0, 1e18 - 1, borrower, borrower);

        assertEq(midnight.collateral(id, borrower, 0), 1, "collateral is 1 wei");
        assertTrue(midnight.isHealthy(mkt, id, borrower), "healthy at price=2e36");

        assertFalse(midnight.isHealthy(mkt, id, borrower), "appears unhealthy due to rounding");

        // But position IS economically healthy: 1 * 1.5 * 0.77 = 1.155 > 1 = debt
        // True maxDebt = 1.155 > debt = 1

        // Liquidator liquidates — should revert but does not
        uint256 liquidatorBalBefore = collateralToken1.balanceOf(address(this));
        midnight.liquidate(mkt, 0, 1, 0, borrower, false, address(this), address(0), "");
        uint256 liquidatorBalAfter = collateralToken1.balanceOf(address(this));

        // Assert: liquidation succeeded (invariant violated)
        assertGt(liquidatorBalAfter, liquidatorBalBefore, "liquidator seized collateral from healthy position");
        // Assert: borrower's debt was reduced despite position being healthy
        assertEq(midnight.debtOf(id, borrower), 0, "debt wiped from healthy position");
    }
}
