### Title
Oracle returning type(uint256).max causes arithmetic overflow in isHealthy, permanently DoS-ing take() for any seller with collateral ≥ 2 - (File: src/Midnight.sol)

### Summary
`isHealthy` computes `_position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)` where `mulDivDown` is implemented as the unchecked-free expression `(x * y) / d`. When an oracle returns `type(uint256).max` and the seller holds collateral ≥ 2, the multiplication overflows and reverts. Because `take()` unconditionally calls `isHealthy` for the seller at line 476, every `take()` against that seller permanently reverts, blocking all sell-side market activity for that position.

`mulDivDown()` → `UtilsLib.sol` line 30:
```solidity

```

**Root cause:** `mulDivDown` performs `collateral[i] * price` in checked arithmetic. `collateral[i]` is `uint128` (max ≈ 3.4×10³⁸). For `price = type(uint256).max ≈ 1.15×10⁷⁷`, any `collateral[i] ≥ 2` causes `collateral[i] * price > type(uint256).max`, triggering an arithmetic panic revert. The protocol's formal verification (`NoMultiplicationOverflow.spec`) explicitly treats bounded oracle prices as an **external assumption** (`boundedPrice` ghost), not an enforced invariant.

**Attacker inputs:**
- Market creator deploys a market whose `collateralParams[i].oracle` is a contract returning `type(uint256).max`
- Seller supplies ≥ 2 units of collateral and opens debt via a prior `take()`
- Any taker subsequently calling `take()` with this seller triggers the overflow

**Why existing checks fail:**
- `liquidationLocked` is only `true` during the seller's own callback window; it is `false` at line 476 for all normal taker calls
- There is no on-chain oracle price cap or sanity check anywhere in the call path
- The `isHealthy` short-circuit (`if (debt > 0)`) does not help because the seller has debt by construction

### Impact Explanation
All `take()` calls against the affected seller permanently revert with an arithmetic overflow. The seller's sell offers become permanently unfillable. The seller cannot reduce their debt via `take()` (since they are the seller), and the DoS persists as long as the oracle returns the overflow-inducing price. This matches the scoped impact: DoS on `take()` for all sell offers from the affected seller.

### Likelihood Explanation
**Preconditions:**
1. A market is created with a malicious or manipulable oracle — market creator is a valid unprivileged attacker role
2. The seller has `debt > 0` and `collateral[i] ≥ 2` — trivially achievable via normal protocol use
3. The oracle returns `type(uint256).max` — directly controlled by the market creator's oracle contract

The attack is repeatable and permanent until the oracle is changed (which the market creator controls). Any market with an untrusted oracle is vulnerable.

### Recommendation
Add an on-chain oracle price bound check in `isHealthy` (and in `liquidate`'s equivalent loop). Specifically, enforce that `price ≤ type(uint256).max / type(uint128).max` (approximately `2^128`) before performing the multiplication, consistent with the assumption already documented in `NoMultiplicationOverflow.spec`:

```solidity
uint256 price = IOracle(collateralParam.oracle).price();
require(price <= type(uint256).max / type(uint128).max, OraclePriceOverflow());
```

Alternatively, use a `mulDivDown` variant that performs the multiplication in 512-bit intermediate space (as used in Uniswap v3's `FullMath`), eliminating the overflow entirely.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
