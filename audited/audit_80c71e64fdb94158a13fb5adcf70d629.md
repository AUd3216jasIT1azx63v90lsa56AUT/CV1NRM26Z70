### Title
Missing `collateralIndex` bounds check in bundler causes Midnight-level panic revert after takes execute - (`src/periphery/MidnightBundles.sol`)

### Summary
Both `buyWithUnitsTargetAndWithdrawCollateral` and `buyWithAssetsTargetAndWithdrawCollateral` execute the takes loop first, then iterate over `collateralWithdrawals` and call `IMidnight.withdrawCollateral` with the caller-supplied `collateralIndex` without validating it against `market.collateralParams.length`. If `collateralIndex >= market.collateralParams.length`, the call into `Midnight.withdrawCollateral` panics with a Solidity array-out-of-bounds error (0x32) rather than a bundler-defined error, after the takes loop has already run. The entire transaction reverts atomically, so no state persists, but the revert origin and error type diverge from expected bundler behavior.

### Finding Description
**Code path — `buyWithUnitsTargetAndWithdrawCollateral`:**

1. Authorization check at line 60; takes loop at lines 71–86 calls `IMidnight.take` for each offer.
2. `require(filledUnits == targetUnits, OutOfOffers())` at line 88 passes once the target is met.
3. Collateral withdrawal loop at lines 91–100 calls `IMidnight(MIDNIGHT).withdrawCollateral(market, collateralWithdrawals[i].collateralIndex, ...)` with no prior bounds check.

**Inside `Midnight.withdrawCollateral` (line 558):**
```solidity
address collateralToken = market.collateralParams[collateralIndex].token;
```
If `collateralIndex >= market.collateralParams.length`, Solidity emits a panic 0x32 (array index out of bounds). There is no explicit `InvalidCollateralIndex` guard anywhere in `Midnight.sol` or `MidnightBundles.sol`.

**Identical path in `buyWithAssetsTargetAndWithdrawCollateral`:** takes loop at lines 205–222, checks at lines 224–225, then withdrawal loop at lines 228–237 — same missing guard.

**Attacker-controlled inputs:** The caller (who must be `taker` or authorized by `taker`) supplies the `CollateralWithdrawal[] memory collateralWithdrawals` array. `CollateralWithdrawal.collateralIndex` is a plain `uint256` with no upper-bound validation anywhere in the bundler before it is forwarded to Midnight.

**Why existing checks fail:** The only input checks in the bundler are `Unauthorized()`, `PctExceeded()`, `InconsistentSide()`, `InconsistentMarket()`, and `OutOfOffers()`. None of these guard `collateralIndex` against `market.collateralParams.length`.

### Impact Explanation
The transaction reverts atomically, so no state change persists. The concrete divergence is: (a) the revert originates inside `Midnight.withdrawCollateral` as a Solidity panic rather than a bundler-defined error selector, making the failure opaque to callers and off-chain tooling; (b) the takes loop has already executed within the same call frame before the revert, consuming gas unnecessarily; (c) the error type is non-deterministic from the bundler's perspective — a caller expecting a clean `IMidnightBundles` error receives a raw EVM panic instead.

### Likelihood Explanation
The precondition is that the caller is the taker or an address authorized by the taker on Midnight. Any such caller can trivially construct a `CollateralWithdrawal` with `collateralIndex` set to any value ≥ `market.collateralParams.length` (e.g., `type(uint256).max`). The condition is reachable on every market regardless of state, and is repeatable without any special setup beyond valid takes.

### Recommendation
Add an explicit bounds check in the bundler's collateral withdrawal loop before forwarding to Midnight, in both functions:

```solidity
require(
    collateralWithdrawals[i].collateralIndex < market.collateralParams.length,
    InvalidCollateralIndex()
);
```

This should be placed at the top of the loop body in lines 91–100 and 228–237 of `MidnightBundles.sol`, and a corresponding `error InvalidCollateralIndex()` added to `IMidnightBundles`.

### Proof of Concept
```solidity
function testInvalidCollateralIndexRevertsWithPanic() public {
    // Setup: valid market with 1 collateral (index 0 only)
    // Borrower supplies collateral and creates a sell offer
    // Lender has sufficient loan tokens approved

    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: sellOffer, units: targetUnits, ratifierData: hex""});

    // Pass collateralIndex = 1, but market only has index 0
    CollateralWithdrawal[] memory withdrawals = new CollateralWithdrawal[](1);
    withdrawals[0] = CollateralWithdrawal({collateralIndex: 1, assets: 0});

    vm.prank(lender);
    // Expect a Solidity panic (array OOB = 0x32), NOT a bundler error selector
    vm.expectRevert(stdError.indexOOBError);
    midnightBundles.buyWithUnitsTargetAndWithdrawCollateral(
        targetUnits, maxBuyerAssets, lender, _noPermit(), takes, withdrawals, address(this), 0, address(0)
    );

    // Assert: taker's credit is 0 (takes were rolled back atomically)
    assertEq(midnight.creditOf(id, lender), 0, "credit must be zero after atomic revert");
    // Assert: borrower's debt is 0 (take rolled back)
    assertEq(midnight.debtOf(id, borrower), 0, "debt must be zero after atomic revert");
}
```
