Audit Report

## Title
Unvalidated Non-Contract `liquidatorGate` Permanently Bricks Liquidations - (File: src/Midnight.sol)

## Summary
`touchMarket` accepts any `address` for `market.liquidatorGate` without verifying it is `address(0)` or a deployed contract. An unprivileged attacker can create a market with `liquidatorGate` set to a codeless address, causing every subsequent `liquidate()` call to unconditionally revert via Solidity 0.8.34 ABI-decoding failure, permanently disabling liquidations and enabling unbounded bad-debt accumulation at lenders' expense.

## Finding Description

**Root cause — no contract check in `touchMarket`:**

`touchMarket` validates `maturity`, collateral count, token ordering, `lltv`, and `maxLif`, but never inspects `market.liquidatorGate`:

```solidity
// src/Midnight.sol lines 758–773
require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
require(market.collateralParams.length > 0, NoCollateralParams());
// ... collateral checks only ...
// liquidatorGate is never inspected
```

The `Market` struct (`src/interfaces/IMidnight.sol` line 11) accepts any `address` for `liquidatorGate` with no constraint. After `touchMarket`, the market is immutable via `IdLib.storeInCode` (line 786).

**Trigger — `liquidate()` calls the interface on the codeless address:**

```solidity
// src/Midnight.sol lines 597–600
require(
    market.liquidatorGate == address(0) || ILiquidatorGate(market.liquidatorGate).canLiquidate(msg.sender),
    LiquidatorGatedFromLiquidating()
);
```

When `liquidatorGate = address(0xdead)`:
1. `market.liquidatorGate == address(0)` → `false`; short-circuit does not fire.
2. `ILiquidatorGate(address(0xdead)).canLiquidate(msg.sender)` is evaluated.
3. The EVM `CALL` to a codeless address returns `(success=true, returndata=<empty>)`.
4. Solidity 0.8.34's ABI decoder expects ≥ 32 bytes for the `bool` return declared in `ILiquidatorGate.canLiquidate` (`src/interfaces/IGate.sol` line 11); it receives 0 bytes → **unconditional revert** (ABI decoding error).
5. The `require` guard cannot catch this: a revert thrown inside the condition expression propagates unconditionally; `require` only catches a `false` return.

**Existing protections are insufficient:**

The LIVENESS comment at line 152 (`"If liquidatorGate.canLiquidate reverts or returns false, liquidate reverts"`) documents the behavior of a *legitimate* gate that may transiently revert. It does not constitute acceptance of a permanently broken non-contract gate created by an adversary. No `extcodesize` or equivalent guard exists anywhere in the creation or liquidation path.

**Exploit flow:**
1. Attacker calls `touchMarket(market)` with `market.liquidatorGate = address(0xdead)` and otherwise valid parameters. Market is created successfully.
2. Attacker seeds the market with attractive offers; lenders and borrowers interact normally (`take`, `repay`, `supplyCollateral` are unaffected).
3. Borrower's position becomes unhealthy (e.g., oracle price drop).
4. Any call to `liquidate(market, ...)` hits the `canLiquidate` call → ABI-decode revert → liquidation permanently impossible.
5. Bad debt accumulates; lenders cannot recover funds.

## Impact Explanation
Unhealthy borrowers in any market created with a non-contract, non-zero `liquidatorGate` can never be liquidated. Bad debt accumulates without bound. Lenders suffer permanent, unrecoverable fund loss. The core protocol invariant — "unhealthy positions remain liquidatable" — is permanently violated for the affected market. This constitutes direct, concrete loss of lender funds: permanent freeze/loss of user assets, an in-scope impact class.

## Likelihood Explanation
- **Privilege required:** None. `touchMarket` is public and permissionless.
- **Feasibility:** Trivial. One transaction suffices to create the malformed market.
- **Repeatability:** The market ID is deterministic from its parameters; once created, `liquidatorGate` is immutable. The DoS is permanent.
- **Victim attraction:** The attacker can seed the market with competitive offers to draw in lenders and borrowers before positions become unhealthy.

## Recommendation
In `touchMarket`, add a contract-existence check for `liquidatorGate` when it is non-zero:

```solidity
if (market.liquidatorGate != address(0)) {
    uint256 size;
    assembly { size := extcodesize(market.liquidatorGate) }
    require(size > 0, InvalidLiquidatorGate());
}
```

Alternatively, apply the same pattern used for oracle addresses if one exists, or add a try/catch in `liquidate()` around the `canLiquidate` call to treat a revert as `false` rather than propagating it. The former (creation-time check) is strongly preferred as it prevents the malformed market from ever being created.

## Proof of Concept
1. Deploy `Midnight` on a local fork.
2. Call `touchMarket(market)` where `market.liquidatorGate = address(0xdead)` and all other parameters are valid. Observe success.
3. Have a lender supply credit and a borrower take debt in this market.
4. Manipulate the oracle to make the borrower's position unhealthy.
5. Call `liquidate(market, ...)`. Observe unconditional revert (ABI decoding error, not `LiquidatorGatedFromLiquidating`).
6. Confirm no liquidation path exists; bad debt accumulates permanently.
