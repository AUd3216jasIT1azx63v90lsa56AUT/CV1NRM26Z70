### Title
Borrower can self-immunize against liquidation by activating a reverting-oracle collateral via `supplyCollateral` - (File: `src/Midnight.sol`)

### Summary
`supplyCollateral` contains an authorization check specifically described as preventing "activated collateral poisoning," but this only blocks third-party poisoning — it does not prevent the borrower from activating a reverting-oracle collateral on their own position. Because `liquidate` iterates over every bit in the borrower's `collateralBitmap` and calls `IOracle.price()` on each with no try/catch, activating even one reverting oracle causes every subsequent `liquidate` call to revert, permanently blocking liquidation while `repay` (which never calls any oracle) remains available to the borrower.

### Finding Description
**Root cause — `supplyCollateral` activates a bitmap bit without any oracle liveness check:**

`supplyCollateral` at `src/Midnight.sol:535-540` sets the collateral bitmap bit whenever `oldCollateral == 0 && assets > 0`:

```solidity
if (oldCollateral == 0 && assets > 0) {
    uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
    _position.collateralBitmap = newCollateralBitmap;
    require(
        UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER, TooManyActivatedCollaterals()
    );
}
```

No oracle is called, no health check is performed, and no check verifies that the newly activated oracle is functional. [1](#0-0) 

**`liquidate` calls every activated oracle with no try/catch:**

`liquidate` at `src/Midnight.sol:607-618` iterates the full `collateralBitmap` and calls `IOracle(_collateralParam.oracle).price()` unconditionally. A single reverting oracle causes the entire call to revert before the `NotLiquidatable` check is even reached. [2](#0-1) 

**`isHealthy` has the same unconditional oracle call:**

`isHealthy` at `src/Midnight.sol:949-957` also iterates the bitmap and calls `.price()` with no try/catch, so any caller that depends on `isHealthy` (e.g., `withdrawCollateral`) is equally blocked. [3](#0-2) 

**The authorization check only stops third-party poisoning:**

The NatSpec comment at line 523 reads *"This function checks authorization to prevent activated collateral poisoning."* The check `require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender])` at line 527 prevents a stranger from activating a reverting oracle on someone else's position, but the borrower calling on their own behalf satisfies this check trivially. [4](#0-3) 

**`repay` never calls any oracle:**

`repay` at `src/Midnight.sol:502-521` only decrements `_position.debt` and increments `marketState[id].withdrawable`. The borrower can therefore repay their debt at any time after self-immunizing, choosing the most favorable moment. [5](#0-4) 

**Exploit call sequence:**
1. Borrower calls `take(sell, borrow)` to open a debt position.
2. Oracle price drops → borrower becomes unhealthy (`debt > maxDebt`).
3. Borrower calls `supplyCollateral(market, idx_reverting, 1 wei, borrower)` where `market.collateralParams[idx_reverting].oracle` always reverts on `.price()`.
4. Bit `idx_reverting` is now set in `_position.collateralBitmap`.
5. Any call to `liquidate(market, ...)` for this borrower enters the bitmap loop, hits the reverting oracle at line 610, and reverts.
6. Borrower repays at leisure when market conditions are favorable, or never repays if the position eventually becomes bad debt that cannot be realized.

**The Certora spec confirms this is a real execution path:**

`Reverts.spec:183-193` (`oracleRevertCausesLiquidateRevert`) formally proves that if any activated collateral oracle reverts, `liquidate` reverts — treating it as a guaranteed property of the system, not a guarded edge case. [6](#0-5) 

### Impact Explanation
An unhealthy borrower can block all liquidation attempts indefinitely by activating a single reverting-oracle collateral with 1 wei. This directly violates the core invariant *"unhealthy positions remain liquidatable."* The borrower retains the ability to repay on their own schedule, effectively converting a forced liquidation into a voluntary repayment, eliminating the protocol's ability to protect lenders from bad debt accumulation.

### Likelihood Explanation
**Preconditions:**
- The market must include at least one `collateralParams` entry whose oracle can revert (e.g., a circuit-breaker oracle, a deprecated feed, or any oracle that reverts under certain conditions).
- The borrower must hold 1 wei of the corresponding collateral token (trivially acquirable).
- The borrower must be unhealthy (the attack is only useful at that point).

**Feasibility:** The Certora specs explicitly model reverting oracles as a realistic scenario. Any market with a collateral whose oracle has a circuit breaker or can be paused is vulnerable. The attack requires a single `supplyCollateral` call with 1 wei and is repeatable across any such market. The borrower has a direct financial incentive (avoiding forced liquidation at a loss).

### Recommendation
When `supplyCollateral` activates a new collateral bit (`oldCollateral == 0 && assets > 0`), call the oracle for that specific collateral index to verify it is currently functional:

```solidity
if (oldCollateral == 0 && assets > 0) {
    // Verify the oracle is live before activating it in the bitmap.
    IOracle(market.collateralParams[collateralIndex].oracle).price();
    uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
    _position.collateralBitmap = newCollateralBitmap;
    require(
        UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER,
        TooManyActivatedCollaterals()
    );
}
```

This ensures that a collateral whose oracle is currently reverting cannot be activated, preventing self-immunization. The check is scoped to the single newly activated oracle, so it does not add overhead for collaterals already in the bitmap.

```solidity
import "forge-std/Test.sol";
import {Midnight, Market, CollateralParams} from "src/Midnight.sol";
import {IMidnight} from "src/interfaces/IMidnight.sol";

contract RevertingOracle {
    function price() external pure returns (uint256) {
        revert("oracle down");
    }
}

contract SelfImmunizationTest is Test {
    Midnight midnight;
    Market market; // two collateral params: [0] normal oracle, [1] reverting oracle
    address borrower = address(0xB0);

    function setUp() public {
        // Deploy Midnight, two ERC20 tokens, one normal oracle, one reverting oracle.
        // Create market with collateralParams[0] = normal, collateralParams[1] = reverting.
        // Fund borrower with collateral token[0] and 1 wei of collateral token[1].
        // Borrower supplies collateral[0], takes a borrow position.
    }

    function testSelfImmunization() public {
        // 1. Drop oracle[0] price so borrower is unhealthy.
        //    assert: isHealthy(market, id, borrower) == false (or reverts after step 3)

        // 2. Borrower activates the reverting oracle collateral.
        vm.prank(borrower);
        midnight.supplyCollateral(market, 1 /*idx_reverting*/, 1 wei, borrower);
        // assert: collateralBitmap bit 1 is now set

        // 3. Liquidation must revert.
        vm.expectRevert();
        midnight.liquidate(market, 0, 0, 1, borrower, false, address(this), address(0), "");

        // 4. isHealthy must also revert.
        vm.expectRevert();
        midnight.isHealthy(market, id, borrower);

        // 5. Repay succeeds (no oracle call).
        vm.prank(borrower);
        midnight.repay(market, 1, borrower, address(0), "");
        // assert: debtOf(id, borrower) decreased
    }
}
```

**Expected assertions:**
- After step 2: `collateralBitmap` for borrower has bit 1 set.
- Step 3: `liquidate` reverts (oracle revert propagates through the bitmap loop at `src/Midnight.sol:610`).
- Step 4: `isHealthy` reverts (same oracle call at `src/Midnight.sol:953`).
- Step 5: `repay` succeeds and `debtOf` decreases, confirming the borrower retains full control over debt repayment while liquidation is blocked.

```text
    function supplyCollateral(Market memory market, uint256 collateralIndex, uint256 assets, address onBehalf)
        external
    {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);
        address collateralToken = market.collateralParams[collateralIndex].token;

        Position storage _position = position[id][onBehalf];
        uint256 oldCollateral = _position.collateral[collateralIndex];
        _position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);

        if (oldCollateral == 0 && assets > 0) {
            uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
            _position.collateralBitmap = newCollateralBitmap;
            require(
                UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER, TooManyActivatedCollaterals()
            );
        }

        emit EventsLib.SupplyCollateral(msg.sender, id, collateralToken, assets, onBehalf);

        SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
    }
```

```text
    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;

            }
        }
```

```text
/// If any activated collateral oracle reverts on price, liquidate reverts.
rule oracleRevertCausesLiquidateRevert(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, uint256 revertingCollateralIndex, bool postMaturityMode) {
    require singleRevertingOracle == market.collateralParams[revertingCollateralIndex].oracle, "oracle is reverting";

    bytes32 id = summaryToId(market);
    uint128 bitmap = collateralBitmap(id, borrower);
    require summaryGetBit(bitmap, revertingCollateralIndex), "revertingCollateralIndex is activated";

    liquidate@withrevert(e, market, collateralIndex, seizedAssets, repaidUnits, borrower, postMaturityMode, receiver, callback, data);

    assert lastReverted;
}
```
