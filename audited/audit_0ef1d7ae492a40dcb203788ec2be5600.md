### Title
Seller Self-Activates Reverting-Oracle Collateral to DoS All Takers on Their Offers - (`src/Midnight.sol`)

### Summary
`supplyCollateral` guards against third-party "activated collateral poisoning" via an authorization check, but the seller themselves can call it to activate a collateral index whose oracle reverts. Because `isHealthy` iterates every bit in the seller's `collateralBitmap` and calls `IOracle.price()` on each, a single reverting oracle causes `isHealthy` to revert, which propagates through the unconditional `require` at `Midnight.sol:476`, making every subsequent `take()` against any of that seller's offers revert for as long as the seller carries debt.

### Finding Description

**Code path:**

`isHealthy` (`Midnight.sol:944-960`) iterates the seller's `collateralBitmap`:

```solidity
while (_collateralBitmap != 0) {
    uint256 i = UtilsLib.msb(_collateralBitmap);
    CollateralParams memory collateralParam = market.collateralParams[i];
    uint256 price = IOracle(collateralParam.oracle).price(); // bare external call, no try/catch
    ...
    _collateralBitmap = _collateralBitmap.clearBit(i);
}
``` [1](#0-0) 

If any oracle in the bitmap reverts, `isHealthy` reverts. `take()` ends with:

```solidity
if (!wasLocked) UtilsLib.tExchange(LIQUIDATION_LOCK_SLOT, id, seller, false);
require(liquidationLocked(id, seller) || isHealthy(offer.market, id, seller), SellerIsLiquidatable());
``` [2](#0-1) 

`liquidationLocked` is false in the normal (non-reentrant) path because `wasLocked` was false and the lock was just cleared on line 475. Therefore `isHealthy` is always called, and its revert propagates.

**How the seller activates the reverting oracle:**

`supplyCollateral` sets a bit in `collateralBitmap` when `oldCollateral == 0 && assets > 0`:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
...
if (oldCollateral == 0 && assets > 0) {
    uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
    _position.collateralBitmap = newCollateralBitmap;
    ...
}
``` [3](#0-2) 

The authorization check (and the comment "to prevent activated collateral poisoning") blocks a *third party* from poisoning the seller's bitmap. It does **not** block the seller from calling `supplyCollateral(market, j, 1, seller)` on themselves to activate index `j` whose oracle is known to revert or will later revert.

**Exploit flow:**

1. Seller has existing debt (acquired via a prior `take`).
2. Seller calls `supplyCollateral(market, j, 1, seller)` where `collateralParams[j].oracle` is a reverting oracle (or becomes one). Cost: 1 wei of the collateral token + gas.
3. Bit `j` is set in `position[id][seller].collateralBitmap`.
4. Any taker calls `take(offer, ...)` where `offer.maker == seller` (sell offer) or `offer.buy == true && taker == seller` (buy offer where seller is taker).
5. `isHealthy(market, id, seller)` is called at line 476.
6. The bitmap loop reaches index `j`, calls `IOracle(collateralParams[j].oracle).price()`, which reverts.
7. `isHealthy` reverts → `take` reverts with no useful error.
8. All takers are blocked from filling any of the seller's offers for as long as the seller maintains debt.

**Why existing checks fail:**

- The `liquidationLocked` short-circuit at line 476 only fires when `wasLocked` was already true (i.e., a reentrant `take` inside a callback). In the normal external call path `wasLocked = false`, the lock is cleared on line 475, and `liquidationLocked` returns false, so `isHealthy` is always reached.
- There is no `try/catch` around `IOracle.price()` in `isHealthy`.
- There is no check in `supplyCollateral` that the oracle for the activated index is currently non-reverting.
- The Certora spec itself formally documents this as a reachable property (`oracleRevertPreventsTakeWhenSellerHasDebt`, `Reverts.spec:224-241`), confirming the revert path is real. [4](#0-3) 

### Impact Explanation
Every `take()` call against any offer where the poisoned seller is the seller reverts at `Midnight.sol:476`. Takers cannot fill those offers at all. The seller's entire offer book is effectively frozen for takers. The seller retains their debt position and collateral; only the taker-side liquidity is blocked. The cost to the seller is 1 wei of the collateral token for index `j`.

### Likelihood Explanation
**Preconditions:** (a) market has at least two collateral params; (b) seller has non-zero debt; (c) seller can obtain 1 wei of the token for a collateral index whose oracle reverts (or will revert). Condition (c) is feasible whenever a market lists a collateral whose oracle is upgradeable, pauseable, or otherwise fallible, or when the seller deploys a trivially reverting oracle contract and the market creator lists it (market creation is permissionless). The seller has direct economic incentive: if market rates move against them after signing offers, freezing takers prevents further debt accrual at unfavorable terms. The attack is repeatable and cheap.

### Recommendation
Wrap the oracle call in `isHealthy` with a `try/catch`, treating a reverting oracle as returning price `0` (which makes the collateral contribute zero to `maxDebt`, preserving the conservative safety direction):

```solidity
try IOracle(collateralParam.oracle).price() returns (uint256 price) {
    maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
        .mulDivDown(collateralParam.lltv, WAD);
} catch {
    // reverting oracle contributes 0 to maxDebt (conservative)
}
```

This ensures `isHealthy` never reverts due to an oracle failure, so `take()` can always complete the health check. A reverting oracle simply makes that collateral worthless for health purposes, which is the safe default. The same `try/catch` should be applied consistently in `liquidate`'s health check for symmetry.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {Market, CollateralParams, Offer} from "src/interfaces/IMidnight.sol";

contract RevertingOracle {
    bool public shouldRevert;
    uint256 public storedPrice;
    function setRevert(bool v) external { shouldRevert = v; }
    function setPrice(uint256 p) external { storedPrice = p; }
    function price() external view returns (uint256) {
        require(!shouldRevert, "oracle down");
        return storedPrice;
    }
}

contract OracleDoSTakeTest is Test {
    // Setup: two-collateral market; oracle0 healthy, oracle1 = RevertingOracle
    // Step 1: seller takes a buy offer → seller acquires debt
    // Step 2: seller calls supplyCollateral(market, 1, 1, seller)
    //         → bit 1 set in seller's collateralBitmap
    // Step 3: oracle1.setRevert(true)
    // Step 4: taker attempts take(sellOffer) → assert it reverts
    // Step 5: oracle1.setRevert(false) → assert take succeeds
    //
    // Key assertions:
    //   vm.expectRevert() on take() in step 4
    //   take() succeeds in step 5 (no revert)
    //
    // Fuzz variant: fuzz collateralIndex j in [0, numCollaterals),
    //   for each j set oracle[j] to revert, assert take always reverts,
    //   then unset revert, assert take succeeds.
}

/// If an activated collateral oracle reverts on price and take succeeds, the seller must have no debt.
rule oracleRevertPreventsTakeWhenSellerHasDebt(env e, Midnight.Offer offer, bytes ratifierData, uint256 units, address taker, address receiver, address takerCallback, bytes takerCallbackData, uint256 collateralIndex) {
    require singleRevertingOracle == offer.market.collateralParams[collateralIndex].oracle, "oracle is reverting";

    bytes32 id = summaryToId(offer.market);
    address seller = offer.buy ? taker : offer.maker;

    // Without this, take's liquidatability check short-circuits to false (without calling isHealthy) because
    // take's tExchange keeps the lock set when wasLocked is true, so the oracle is never queried.
    require !liquidationLocked(id, seller), "seller is not liquidation locked";

    uint128 bitmap = collateralBitmap(id, seller);
    require summaryGetBit(bitmap, collateralIndex), "collateralIndex is activated";

    take(e, offer, ratifierData, units, taker, receiver, takerCallback, takerCallbackData);

    assert debtOf(id, seller) == 0;
}
