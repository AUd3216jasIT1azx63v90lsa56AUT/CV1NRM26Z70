### Title
Authorized address can pre-cancel unpublished Merkle roots, permanently blocking maker's future offer fills - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
`cancelRoot` permits any address that passes the `isAuthorized(maker, msg.sender)` check to cancel an arbitrary `bytes32` root value, including roots for offer trees the maker has not yet signed or published. Because `isRootCanceled` is a write-once mapping (no `uncancelRoot` exists anywhere in the codebase), a pre-cancellation is irreversible. An authorized attacker who can compute the deterministic root from known offer parameters can therefore permanently block the maker from filling any offer whose ratifier data references that root.

### Finding Description
**Exact code path:**

`cancelRoot` (line 27–31) performs a single authorization check and unconditionally sets the canceled flag:

```solidity
function cancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
    isRootCanceled[maker][root] = true;   // permanent, no inverse operation
    emit CancelRoot(msg.sender, maker, root);
}
``` [1](#0-0) 

`isRatified` (line 38) then hard-reverts on any canceled root, regardless of whether a valid signature is present:

```solidity
require(!isRootCanceled[offer.maker][root], RootCanceled());
``` [2](#0-1) 

**Root predictability:** Every root is a pure deterministic function of public offer parameters. A single-offer tree root equals `HashLib.hashOffer(offer)`, which is a `keccak256` over all `Offer` struct fields (market, buy, maker, start, expiry, tick, group, callback, callbackData, receiverIfMakerIsSeller, ratifier, reduceOnly, maxUnits, maxAssets). Multi-leaf roots are built by `HashLib.hashNode` over those leaf hashes. All inputs are observable on-chain or off-chain before the maker signs. [3](#0-2) 

**No uncancelRoot exists:** A grep across the entire repository confirms `isRootCanceled` is only ever written to `true`; there is no reversal path. [4](#0-3) 

**Exploit flow:**
1. Attacker holds `isAuthorized(maker, attacker) == true` (maker granted this, e.g., to allow a signing bot).
2. Attacker observes or predicts the offer parameters the maker intends to sign.
3. Attacker computes `predicted_root = HashLib.hashOffer(offer)` (or the multi-leaf equivalent) off-chain.
4. Attacker calls `cancelRoot(maker, predicted_root)` — the `isAuthorized` check passes, `isRootCanceled[maker][predicted_root]` is set to `true`.
5. Maker signs `predicted_root` and publishes the offer.
6. Taker calls `take(offer, ratifierData, ...)` → Midnight calls `isRatified` → line 38 reverts with `RootCanceled()`.

**Why existing checks fail:** The only guard in `cancelRoot` is the authorization check. There is no requirement that the root correspond to an already-signed or already-published tree, no timestamp or nonce binding, and no way to reverse the cancellation. The authorization check is intentionally broad (same scope as signing), so it does not distinguish between "cancel my own root" and "cancel a root I predicted for the maker."

### Impact Explanation
Every offer fill that uses `EcrecoverRatifier` requires a valid, non-canceled root. Once `isRootCanceled[maker][root]` is `true`, no taker can ever successfully call `take` with ratifier data referencing that root for that maker. Because there is no `uncancelRoot`, the block is permanent for that root value. If the attacker can continuously predict and pre-cancel roots as the maker generates new offer parameters (feasible while authorization persists), the maker's ability to fill offers through this ratifier is durably disrupted.

### Likelihood Explanation
**Preconditions:**
- Attacker must hold `isAuthorized(maker, attacker) == true`. This is a realistic precondition: makers routinely authorize bots, relayers, or co-signers for offer management.
- Attacker must predict the root. Roots are fully deterministic from public offer parameters; no secret is required.

**Feasibility:** Both preconditions are independently achievable and commonly co-occur in production (e.g., a market-making bot authorized by a lender). The attack requires a single transaction per root and can be scripted to front-run any observed offer parameter set.

**Repeatability:** The attack can be repeated for every new root the maker attempts to use, for as long as the authorization persists. Revoking authorization stops future pre-cancellations but does not restore already-canceled roots.

### Recommendation
Restrict `cancelRoot` so that only the maker themselves (`maker == msg.sender`) can cancel a root, removing the `isAuthorized` branch entirely:

```solidity
function cancelRoot(bytes32 root) external {
    isRootCanceled[msg.sender][root] = true;
    emit CancelRoot(msg.sender, msg.sender, root);
}
```

If delegation is a desired feature, require the maker to co-sign the cancellation (e.g., provide an EIP-712 signature from `maker` over `(maker, root, nonce)`) so an authorized party cannot act unilaterally on roots the maker has not explicitly chosen to cancel.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {EcrecoverRatifier} from "src/ratifiers/EcrecoverRatifier.sol";
import {HashLib} from "src/ratifiers/libraries/HashLib.sol";
import {IMidnight, Offer} from "src/interfaces/IMidnight.sol";
import {IEcrecoverRatifier, Signature} from "src/ratifiers/interfaces/IEcrecoverRatifier.sol";

contract PreCancelRootTest is Test {
    EcrecoverRatifier ratifier;
    address midnight;
    uint256 makerKey = 1;
    uint256 attackerKey = 2;
    address maker;
    address attacker;

    function setUp() public {
        maker    = vm.addr(makerKey);
        attacker = vm.addr(attackerKey);
        // Deploy a mock Midnight that returns isAuthorized(maker, attacker) == true
        // and allows setIsAuthorized.
        midnight = address(new MockMidnight(maker, attacker));
        ratifier = new EcrecoverRatifier(midnight);
    }

    function testPreCancelBlocksFutureOfferFill() public {
        // 1. Build the offer the maker intends to sign (all params known).
        Offer memory offer;
        offer.maker = maker;
        // ... populate remaining fields deterministically ...

        // 2. Attacker computes the deterministic root off-chain.
        bytes32 predictedRoot = HashLib.hashOffer(offer);

        // 3. Attacker pre-cancels the root (passes isAuthorized check).
        vm.prank(attacker);
        ratifier.cancelRoot(maker, predictedRoot);

        // Assert: root is now permanently canceled.
        assertTrue(ratifier.isRootCanceled(maker, predictedRoot));

        // 4. Maker signs the root (EIP-712 over predictedRoot).
        bytes32 structHash = keccak256(
            abi.encode(HashLib.offerTreeTypeHash(0), predictedRoot)
        );
        bytes32 domainSep = keccak256(
            abi.encode(
                0x47e79534a245952e8b16893a336b85a3d9ea9fa8c573f3d803afb92a79469218,
                block.chainid,
                address(ratifier)
            )
        );
        bytes32 digest = keccak256(
            bytes.concat("\x19\x01", domainSep, structHash)
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(makerKey, digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        bytes memory ratifierData = abi.encode(
            sig, predictedRoot, uint256(0), new bytes32[](0)
        );

        // 5. Taker attempts take → isRatified → RootCanceled revert.
        vm.prank(midnight);
        vm.expectRevert(IEcrecoverRatifier.RootCanceled.selector);
        ratifier.isRatified(offer, ratifierData);
        // ASSERTION: valid signature + valid proof still cannot fill the offer.
    }
}
```

**Expected assertions:**
- `isRootCanceled[maker][predictedRoot] == true` after step 3.
- `isRatified` reverts with `RootCanceled` in step 5, even though the signature is cryptographically valid.
- No path exists to set `isRootCanceled[maker][predictedRoot]` back to `false`.
