### Title
Unvalidated Oracle Address in Permissionless Market Creation Allows Bypassing Health Checks to Borrow Without Adequate Collateral - (File: src/Midnight.sol)

### Summary

Markets in Midnight are created permissionlessly via `touchMarket()`, which accepts any oracle address in `CollateralParams` without validation. The oracle's return value is used directly in `isHealthy()` to determine a seller's borrowing capacity. An attacker can deploy a malicious oracle returning an arbitrarily inflated price, create a market with it, and place sell offers. When victims take these offers as buyers, the attacker receives loan tokens while the malicious oracle makes the attacker's undercollateralized position appear healthy, permanently preventing liquidation.

### Finding Description

**Root Cause**: `touchMarket()` validates collateral token ordering, LLTV tiers, and `maxLif` values, but performs **no validation on the oracle address** in each `CollateralParams` entry. [1](#0-0) 

The oracle is called unconditionally in `isHealthy()`: [2](#0-1) 

At the end of every `take()` call, the seller's health is checked using the oracle embedded in `offer.market`: [3](#0-2) 

**Exploit Flow**:

1. Attacker deploys `MaliciousOracle` whose `price()` returns a large value (e.g., `1e54`) that does not cause overflow in `mulDivDown(collateral, price, ORACLE_PRICE_SCALE)`.
2. Attacker calls `touchMarket()` with `collateralParams[0].oracle = MaliciousOracle`. No check rejects this.
3. Attacker calls `supplyCollateral()` with 1 token (e.g., 1e18 wei of WETH) to activate the collateral bitmap. This is required so `isHealthy` iterates over the collateral and calls the oracle.
4. Attacker (as maker, `offer.buy = false`, i.e., maker is seller) signs and publishes a sell offer with an attractive price in this market.
5. Victim (taker/buyer) calls `take()`, paying `buyerAssets` loan tokens. Victim's credit increases; attacker's debt increases; attacker receives `sellerAssets` loan tokens.
6. At line 476, `isHealthy(offer.market, id, attacker)` is called. With `collateral = 1e18` and `price = 1e54`: `maxDebt = 1e18 * 1e54 / 1e36 * lltv / 1e18 = 1e18 * lltv` — astronomically larger than any realistic debt. Health check passes.
7. Attacker has received loan tokens and has debt. The malicious oracle permanently prevents liquidation. Attacker never repays. [4](#0-3) [5](#0-4) 

### Impact Explanation

Victims who take sell offers in the attacker's market permanently lose their loan tokens. The attacker's debt position can never be liquidated because `isHealthy` always returns `true` via the malicious oracle. The attacker can repeat this across multiple victims and multiple markets. This constitutes direct theft of user funds with no recovery path.

### Likelihood Explanation

The protocol is designed to be permissionless and offer-book based — users are expected to evaluate offers off-chain. A sophisticated attacker can make the malicious oracle indistinguishable from a legitimate one (e.g., a proxy contract that initially behaves correctly, then switches). Since the protocol itself provides no oracle whitelist or on-chain signal that an oracle is trustworthy, users have no protocol-level protection. The attack requires no privileged access and only a small amount of collateral to set up.

### Recommendation

Introduce an oracle whitelist managed by a trusted role (e.g., `feeSetter` or a dedicated `oracleSetter`), and require that every oracle address in `CollateralParams` is on the whitelist before a market can be created. This mirrors the LLTV tier validation already present in `touchMarket()`:

```solidity
require(isOracleAllowed(market.collateralParams[i].oracle), OracleNotAllowed());
```

Alternatively, validate oracle behavior at market creation time by calling `oracle.price()` and requiring it to return a value within a sane range.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
