Looking at the exact code path in `src/periphery/EcrecoverAuthorizer.sol` and `src/Midnight.sol`.

### Title
Authorized Delegate Can Transitively Grant Arbitrary Authorizations via EcrecoverAuthorizer - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any account that `authorization.authorizer` has authorized on Midnight, not just from the authorizer themselves. Because the signer controls the full `Authorization` struct (including the `authorized` field), any authorized delegate can sign and submit a new authorization granting an arbitrary third party full control over the authorizer's Midnight state. This violates the invariant that only the authorizer or their explicitly designated delegate can modify authorization state.

### Finding Description
**Code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 33–36) performs:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The `signer` is recovered from the EIP-712 signature over the full `Authorization` struct, which includes `authorizer`, `authorized`, `isAuthorized`, `nonce`, and `deadline` — all attacker-controlled fields. The second branch of the `||` passes whenever `midnight.isAuthorized(authorizer, signer) == true`, i.e., whenever `signer` is any account the authorizer has previously authorized on Midnight for *any* purpose.

**Exploit flow:**

1. **Precondition A:** `authorizer` calls `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, authorizer)` — required for `EcrecoverAuthorizer` to operate on behalf of `authorizer` at all.
2. **Precondition B:** `authorizer` calls `midnight.setIsAuthorized(signer, true, authorizer)` — grants `signer` some legitimate authorization (e.g., as a ratifier or operator).
3. **Attack:** `signer` reads the public `ecrecoverAuthorizer.nonce(authorizer)` → value `N`. Constructs `Authorization{authorizer: authorizer, authorized: attacker, isAuthorized: true, nonce: N, deadline: future}`. Signs it with their own private key. Calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
4. **Check 1 in EcrecoverAuthorizer (line 34):** `signer == authorizer` → false; `isAuthorized[authorizer][signer]` → **true** → check passes.
5. **EcrecoverAuthorizer calls** `midnight.setIsAuthorized(attacker, true, authorizer)` with `msg.sender = EcrecoverAuthorizer`.
6. **Check in Midnight (line 732):** `isAuthorized[authorizer][EcrecoverAuthorizer]` → **true** (precondition A) → check passes.
7. **State after:** `midnight.isAuthorized[authorizer][attacker] = true`.

**Why existing checks fail:** The nonce replay protection is keyed to `authorization.authorizer` and is correctly incremented, but it does not prevent the delegate from spending that nonce slot to authorize an arbitrary third party. The `Unauthorized()` revert is bypassed because the second branch of the `||` is satisfied by the pre-existing Midnight authorization of `signer`.

### Impact Explanation
`attacker` gains `isAuthorized` status over `authorizer`'s Midnight account. Per `Midnight.setIsAuthorized` and `Midnight.setConsumed`, an authorized account can act on behalf of `authorizer` in all protocol operations: taking/making offers, managing collateral, cancelling offers, and further modifying authorization state. This constitutes full account takeover of `authorizer` without their direct consent.

### Likelihood Explanation
Preconditions are normal protocol usage: any user who has authorized `EcrecoverAuthorizer` (required to use it) and has also authorized any other account (e.g., a ratifier, a keeper, or another operator) is vulnerable. Both authorizations are routine and expected. The attack requires no special privileges, no oracle manipulation, and no tokens — only knowledge of the victim's current `EcrecoverAuthorizer` nonce (a public mapping). It is repeatable as long as the delegate's authorization on Midnight has not been revoked.

### Recommendation
Remove the `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` branch entirely. The `EcrecoverAuthorizer` is a signature-relay mechanism: only the authorizer themselves should be able to produce a valid `Authorization` signature. The delegate path conflates "authorized to act on Midnight" with "authorized to sign new authorization grants," which are fundamentally different trust levels. The fix:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If relay by a delegate is intentionally desired, it must be scoped to submission of a pre-signed message (where the authorizer signed it), not to signing new messages — which is already satisfied by the `signer == authorization.authorizer` path when the authorizer signs off-chain and a relayer submits.

### Proof of Concept

```solidity
function testTransitiveDelegateEscalation() public {
    address authorizer = makeAddr("authorizer");
    address delegate   = makeAddr("delegate");
    address attacker   = makeAddr("attacker");
    uint256 delegateKey = /* boundPrivateKey */ 0xDEAD;
    // give delegate a known key
    vm.label(vm.addr(delegateKey), "delegate");

    // Precondition A: authorizer enables EcrecoverAuthorizer
    vm.prank(authorizer);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, authorizer);

    // Precondition B: authorizer grants delegate some authorization
    vm.prank(authorizer);
    midnight.setIsAuthorized(vm.addr(delegateKey), true, authorizer);

    // Delegate constructs and signs an Authorization granting attacker
    Authorization memory auth = Authorization({
        authorizer:   authorizer,
        authorized:   attacker,
        isAuthorized: true,
        nonce:        ecrecoverAuthorizer.nonce(authorizer), // 0
        deadline:     block.timestamp + 1 days
    });
    bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
    bytes32 domainSep  = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
    bytes32 digest     = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(delegateKey, digest);
    Signature memory sig = Signature({v: v, r: r, s: s});

    // Delegate submits — should revert if invariant holds, but does NOT
    vm.prank(vm.addr(delegateKey));
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);

    // Assert: attacker now controls authorizer's account — invariant violated
    assertTrue(midnight.isAuthorized(authorizer, attacker));
}
```
