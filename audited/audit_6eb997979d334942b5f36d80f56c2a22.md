### Title
Authorized agent can increment victim's EcrecoverAuthorizer nonce without victim's consent, invalidating pending pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `IMidnight.isAuthorized(authorizer, signer)` returns true for, not only the authorizer themselves. An attacker who holds an active Midnight authorization from the victim can self-sign a no-op `Authorization` struct using the victim's current nonce, submit it, and cause `nonce[victim]` to increment — permanently invalidating any pending pre-signed authorization the victim had distributed for a future `take`, `repay`, `withdraw`, `liquidate`, or `claimFee` operation.

### Finding Description
**Code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 24–48, `src/periphery/EcrecoverAuthorizer.sol`):

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
...
Line 34: signer == authorization.authorizer
         || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)
```

**Root cause:** The signature validity check on line 34 accepts any signer for whom `isAuthorized[authorizer][signer]` is true in Midnight. This means an authorized agent can produce a valid signature over an `Authorization` struct that names the victim as `authorizer`, consuming the victim's current nonce.

**Attacker inputs:**
- `authorization.authorizer = victim`
- `authorization.authorized = victim` (or any address; the Midnight call is a no-op)
- `authorization.isAuthorized = true`
- `authorization.nonce = N` (current `nonce[victim]`)
- `authorization.deadline = block.timestamp + 1` (any future timestamp)
- `signature` = attacker's ECDSA signature over the EIP-712 digest of the above struct

**Exploit flow:**
1. Victim pre-signs `Authorization(authorizer=victim, authorized=X, isAuthorized=true, nonce=N, deadline=T)` and hands it to a relayer for a future `take`/`repay`/`withdraw`/`liquidate`.
2. Attacker (who holds `isAuthorized[victim][attacker] == true`) constructs `Authorization(authorizer=victim, authorized=victim, isAuthorized=true, nonce=N, deadline=T')` and signs it with their own key.
3. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`.
4. Line 26 passes: `N == nonce[victim]`, nonce incremented to `N+1`.
5. Line 34 passes: `ecrecover(...)` returns attacker's address; `isAuthorized[victim][attacker]` is true.
6. Line 47 executes: `Midnight.setIsAuthorized(victim, true, victim)` — sets `isAuthorized[victim][victim] = true`, a functionally irrelevant state change.
7. Victim's pre-signed authorization with nonce `N` now reverts with `InvalidNonce()` when submitted.

**Why existing checks fail:** The `Unauthorized()` guard was designed to prevent strangers from acting on behalf of an authorizer, but it does not distinguish between "the authorizer consented to this specific nonce consumption" and "the signer happens to be an authorized agent." There is no requirement that the signer be the authorizer themselves when the purpose is to consume a nonce.

### Impact Explanation
Any pending pre-signed `Authorization` with nonce `N` that the victim distributed — intended to enable a counterparty to call `take`, `repay`, `withdraw`, `liquidate`, or `claimFee` on their behalf via `EcrecoverAuthorizer` — is permanently invalidated at the cost of a single transaction. The victim must re-sign and redistribute a new authorization. The attacker can repeat this every time the victim issues a new pre-signed authorization, creating a sustained, low-cost denial-of-service against all signature-gated market actions.

### Likelihood Explanation
**Preconditions:**
- Attacker must hold `isAuthorized[victim][attacker] == true` in Midnight. This is a realistic scenario: users authorize relayers, smart contracts, or counterparties as part of normal protocol usage (e.g., authorizing `EcrecoverRatifier`, a callback contract, or a trading bot).
- Victim must have a pending pre-signed authorization in circulation (e.g., a signed permit handed to a relayer).

**Feasibility:** Single transaction, no capital required, no oracle manipulation, no flash loan. Repeatable indefinitely as long as the authorization relationship persists. The attacker does not need to front-run; they only need to submit before the relayer does.

### Recommendation
Restrict the signer check so that only the `authorization.authorizer` themselves can produce a signature that consumes their nonce. Remove the `isAuthorized` delegation path from `EcrecoverAuthorizer.setIsAuthorized`:

```solidity
// Replace lines 33-36 with:
require(signer == authorization.authorizer, Unauthorized());
```

This preserves the intended design — the authorizer signs off-chain and anyone can submit the transaction — while ensuring that only the authorizer's own key can consume their nonce. Authorized agents who need to act on behalf of the authorizer should call `Midnight.setIsAuthorized` directly (which requires `msg.sender` to be the authorizer or an authorized agent, not a signature).

### Proof of Concept
```solidity
// Foundry unit test
function testNonceGriefByAuthorizedAgent() public {
    // Setup: victim authorizes attacker in Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim pre-signs an authorization for a relayer (nonce = 0)
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: relayer,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victim); // victim's key

    // Attacker constructs a no-op authorization using victim's current nonce
    Authorization memory attackAuth = Authorization({
        authorizer: victim,
        authorized: victim,   // no-op target
        isAuthorized: true,
        nonce: 0,             // same nonce as victim's pending auth
        deadline: block.timestamp + 1 days
    });
    Signature memory attackSig = signAuthorization(attackAuth, attacker); // attacker's key

    // Attacker submits first — consumes nonce 0
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);

    // Assert: nonce[victim] is now 1
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);

    // Assert: victim's pre-signed authorization is now invalid
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}
```

```text
    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );
```
