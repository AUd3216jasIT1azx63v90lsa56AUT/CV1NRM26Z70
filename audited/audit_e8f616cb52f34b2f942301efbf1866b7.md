### Title
Fee-on-Transfer / Partial-Transfer `loanToken` Inflates `claimableSettlementFee` Beyond Actual Contract Balance ‚Äî (`src/libraries/SafeTransferLib.sol`, `src/Midnight.sol`)

### Summary
`SafeTransferLib.safeTransferFrom` only checks that the call does not revert and that the return value is `true`; it never verifies that the amount actually received equals the requested `value`. In `take()`, `claimableSettlementFee[loanToken]` is incremented by `buyerAssets - sellerAssets` before the transfer, and the subsequent `safeTransferFrom` call is trusted to deliver exactly that amount. A market creator who deploys a partial-transfer token (one that silently transfers `min(requested, cap)` and returns `true`) as the `loanToken` can cause the recorded `claimableSettlementFee` to permanently exceed the contract's actual token balance, breaking the core solvency invariant for that token.

### Finding Description

**Root cause ‚Äî `SafeTransferLib.safeTransferFrom` (lines 24‚Äì34, `src/libraries/SafeTransferLib.sol`):**

```solidity
function safeTransferFrom(address token, address from, address to, uint256 value) internal {
    require(token.code.length > 0, NoCode());
    (bool success, bytes memory returndata) =
        token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
    if (!success) { assembly { revert(...) } }
    require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
}
```

No balance-before/after check. A token that executes `transfer(to, min(value, cap))` and returns `true` passes every guard.

**Accounting in `take()` (`src/Midnight.sol`, lines 418, 455‚Äì456):**

```solidity
// Step 1 ‚Äî accounting credited unconditionally
claimableSettlementFee[offer.market.loanToken] += buyerAssets - sellerAssets;   // line 418

// Step 2 ‚Äî transfer (no amount verification)
SafeTransferLib.safeTransferFrom(loanToken, payer, address(this), buyerAssets - sellerAssets); // line 455
SafeTransferLib.safeTransferFrom(loanToken, payer, receiver,      sellerAssets);               // line 456
```

The fee credit at line 418 is written before the transfer at line 455. If the token delivers only `X < (buyerAssets - sellerAssets)`, the contract's balance increases by `X` while `claimableSettlementFee` increases by `buyerAssets - sellerAssets`. The shortfall `(buyerAssets - sellerAssets) - X` is permanently unrecoverable.

**Market creation ‚Äî `touchMarket()` (`src/Midnight.sol`, lines 755‚Äì791):**

`touchMarket` imposes no restriction on `loanToken`; any address with deployed code is accepted. An unprivileged actor (market creator role) can register a market whose `loanToken` is a partial-transfer contract they control.

**Exploit flow:**
1. Attacker deploys `PartialToken`: `transferFrom(from, to, amount)` transfers `min(amount, CAP)` and returns `true`.
2. Attacker calls `touchMarket(market)` with `market.loanToken = address(PartialToken)`.
3. Attacker (as maker or taker) calls `take()` with `units` chosen so that `buyerAssets - sellerAssets > CAP`.
4. Line 418 credits `claimableSettlementFee[PartialToken] += buyerAssets - sellerAssets`.
5. Line 455 calls `safeTransferFrom(..., buyerAssets - sellerAssets)` ‚Äî token delivers only `CAP`; call succeeds.
6. Contract balance of `PartialToken` is `CAP`; `claimableSettlementFee[PartialToken]` is `buyerAssets - sellerAssets`. Invariant violated.
7. Repeating the attack accumulates an unbounded shortfall.

**Existing protections reviewed and found insufficient:**
- `NoCode` check: passes (attacker deploys real contract).
- `TransferFromReturnedFalse` check: passes (token returns `true`).
- No balance-delta check anywhere in `SafeTransferLib` or `take()`.
- The Certora spec (`certora/specs/Role.spec`, lines 45‚Äì48) explicitly models `cvlSafeTransferFrom` as transferring the exact amount ‚Äî this assumption is the gap the attack exploits.

### Impact Explanation
`claimableSettlementFee[token]` exceeds the contract's actual balance of that token. `claimSettlementFee` will revert when the fee claimer attempts to withdraw the full recorded amount, permanently locking or losing fee revenue. If the same token is used as `loanToken` across multiple markets, the shortfall compounds across all `take()` calls, making the recorded fee obligation completely uncollectable. This directly violates the invariant: *contract balances cover collateral, credit redemption, fees, and withdrawable assets*.

### Likelihood Explanation
- **Preconditions:** Attacker must deploy a partial-transfer ERC20 and create a market with it. Both are permissionless.
- **Feasibility:** Trivial ‚Äî no privileged access, no oracle manipulation, no user mistake required.
- **Repeatability:** The attack can be repeated across multiple `take()` calls and multiple markets using the same token, compounding the shortfall indefinitely.
- **Constraint:** The impact is scoped to the specific `loanToken` used; legitimate markets with standard tokens are unaffected. However, the protocol makes no distinction and the invariant is still broken.

### Recommendation
Add a balance-before/after check in `SafeTransferLib.safeTransferFrom`, or add a dedicated wrapper used at every inbound transfer site in `Midnight.sol`:

```solidity
function safeTransferFromExact(address token, address from, address to, uint256 value) internal {
    uint256 balanceBefore = IERC20(token).balanceOf(to);
    safeTransferFrom(token, from, to, value);
    require(IERC20(token).balanceOf(to) - balanceBefore == value, InsufficientTransfer());
}
```

Apply this to both `safeTransferFrom` calls in `take()` (lines 455‚Äì456) and to all other inbound transfer sites (`repay`, `liquidate`, `flashLoan`, `supplyCollateral`).

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight, Market, Offer, CollateralParams} from "src/Midnight.sol";

contract PartialToken {
    uint256 public constant CAP = 1; // transfers at most 1 token per call
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;

    function mint(address to, uint256 amount) external { balanceOf[to] += amount; }
    function approve(address spender, uint256 amount) external returns (bool) {
        allowance[msg.sender][spender] = amount; return true;
    }
    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        uint256 actual = amount < CAP ? amount : CAP; // silent partial transfer
        balanceOf[from] -= actual;
        balanceOf[to]   += actual;
        return true; // always returns true
    }
    function transfer(address to, uint256 amount) external returns (bool) {
        uint256 actual = amount < CAP ? amount : CAP;
        balanceOf[msg.sender] -= actual;
        balanceOf[to]         += actual;
        return true;
    }
}

contract PartialTransferFeeTest is Test {
    Midnight midnight;
    PartialToken token;

    function setUp() public {
        midnight = new Midnight(/* constructor args */);
        token = new PartialToken();
    }

    function testClaimableSettlementFeeExceedsBalance() public {
        // 1. Create market with partial-transfer loanToken
        Market memory market;
        market.loanToken = address(token);
        market.maturity  = block.timestamp + 30 days;
        // ... set collateralParams, etc.
        midnight.touchMarket(market);

        // 2. Fund payer with enough tokens and approve
        token.mint(address(this), type(uint256).max);
        token.approve(address(midnight), type(uint256).max);

        // 3. Execute take() ‚Äî choose units so buyerAssets - sellerAssets >> CAP
        // ... set up offer, call midnight.take(...)

        // 4. Assert invariant violation
        uint256 recorded = midnight.claimableSettlementFee(address(token));
        uint256 actual   = token.balanceOf(address(midnight));
        assertGt(recorded, actual, "claimableSettlementFee exceeds actual balance ‚Äî invariant broken");

        // 5. Assert fee claimer cannot withdraw full amount
        vm.expectRevert();
        midnight.claimSettlementFee(address(token), recorded, address(this));
    }
}
```
