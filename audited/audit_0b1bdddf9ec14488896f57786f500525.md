### Title
Authorized operator can self-revoke via EcrecoverAuthorizer, consuming authorizer's nonce and invalidating pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` validates the signer as either the `authorizer` themselves or any address that `isAuthorized[authorizer][signer] == true` in Midnight. There is no check preventing the signer from being the same address as `authorization.authorized` when `isAuthorized=false`. This allows an operator (`operatorA`) who is already authorized by a victim in Midnight to sign an `Authorization{authorizer=victim, authorized=operatorA, isAuthorized=false}`, pass the authorization check (because `isAuthorized[victim][operatorA] == true`), advance victim's nonce in `EcrecoverAuthorizer`, and revoke their own Midnight authorization on behalf of victim.

### Finding Description
**Code path:**

In `EcrecoverAuthorizer.setIsAuthorized` (lines 33–36), the signer validity check is:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

If `signer == operatorA` and `isAuthorized[victim][operatorA] == true`, the second branch passes unconditionally — regardless of whether `signer == authorization.authorized` (i.e., the operator is revoking themselves).

The call then executes:
```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
// → setIsAuthorized(operatorA, false, victim)
``` [2](#0-1) 

**Attacker inputs:**
- `authorization.authorizer = victim`
- `authorization.authorized = operatorA` (attacker = signer)
- `authorization.isAuthorized = false`
- `authorization.nonce = nonce[victim]` (current)
- `authorization.deadline` = any future timestamp
- `signature` = valid EIP-712 signature by `operatorA`

**Exploit flow:**
1. Precondition: `isAuthorized[victim][operatorA] == true` in Midnight; `EcrecoverAuthorizer` is authorized by victim in Midnight (so victim can use it normally).
2. `operatorA` signs the above `Authorization` struct.
3. Any unprivileged caller submits it to `EcrecoverAuthorizer.setIsAuthorized`.
4. Nonce check passes and `nonce[victim]` increments.
5. Signer check passes because `isAuthorized[victim][operatorA] == true`.
6. `midnight.setIsAuthorized(operatorA, false, victim)` executes — `operatorA` is revoked.

**Why existing checks fail:** The `Unauthorized()` guard only verifies that the signer has authority to act on behalf of `authorization.authorizer`. It does not check whether the signer is self-revoking (i.e., `signer == authorization.authorized && !authorization.isAuthorized`). There is no such guard anywhere in the function. [3](#0-2) 

### Impact Explanation
`operatorA` can unilaterally consume `victim`'s nonce in `EcrecoverAuthorizer` without victim's consent. If victim has pre-signed and distributed an `Authorization` at nonce `N` (e.g., granting `operatorB`), `operatorA` can front-run or race it by submitting a self-revocation at nonce `N`, advancing victim's nonce to `N+1` and permanently invalidating victim's pre-signed authorization to `operatorB`. Additionally, `operatorA` is removed from `isAuthorized[victim]` in Midnight, which may disrupt victim's intended delegation setup.

### Likelihood Explanation
Preconditions are the normal operating state of the protocol: victim has authorized `EcrecoverAuthorizer` and `operatorA` in Midnight. No special privilege is required — any address can call `EcrecoverAuthorizer.setIsAuthorized` with a valid signature. `operatorA` can generate the self-revoking signature at any time. The attack is repeatable at every nonce increment (if `operatorA` is re-authorized). It is also front-runnable: if victim broadcasts a pre-signed authorization, `operatorA` can observe it and submit the self-revocation first.

### Recommendation
Add a check that prevents a non-authorizer signer from revoking their own authorization. Specifically, when `signer != authorization.authorizer`, disallow `signer == authorization.authorized && !authorization.isAuthorized`:

```solidity
require(
    signer == authorization.authorizer ||
    (IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer) &&
     !(signer == authorization.authorized && !authorization.isAuthorized)),
    Unauthorized()
);
```

This ensures that only the `authorizer` themselves can revoke an operator's access, while operators can still use the EcrecoverAuthorizer to grant or manage other authorizations on behalf of the authorizer.

function test_operatorA_selfRevoke_consumesVictimNonce() public {

    Authorization memory auth = Authorization({
        authorizer: victim,
        authorized: operatorA,
        isAuthorized: false,
        nonce: nonceBefore,
        deadline: block.timestamp + 1 days
    });
    bytes32 digest = /* EIP-712 digest */;

    ecrecoverAuthorizer.setIsAuthorized(auth, Signature(v, r, s));

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
