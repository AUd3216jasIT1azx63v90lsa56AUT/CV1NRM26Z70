### Title
Non-`stakeToken` ERC20 assets sent to `ConfidencePool` are permanently locked with no recovery path - (File: src/ConfidencePool.sol)

### Summary
Every token-moving function in `ConfidencePool` operates exclusively on `stakeToken`. Any other ERC20 token transferred directly to the pool — whether by a bonus contributor using the wrong token address, a user mistake, or an airdrop — is permanently locked. Because `ConfidencePool` clones are explicitly non-upgradeable, there is no upgrade-path escape hatch.

### Finding Description
All claim and sweep entrypoints in `ConfidencePool` hard-code `stakeToken` as the only asset they move:

- `claimSurvived` / `claimExpired` / `withdraw` — `stakeToken.safeTransfer(msg.sender, payout)`
- `claimCorrupted` — `uint256 toSweep = stakeToken.balanceOf(address(this)); … stakeToken.safeTransfer(recoveryAddress, toSweep)`
- `claimAttackerBounty` — `stakeToken.safeTransfer(attacker, payout)`
- `sweepUnclaimedCorrupted` — `uint256 amount = stakeToken.balanceOf(address(this)); … stakeToken.safeTransfer(recoveryAddress, amount)`
- `sweepUnclaimedBonus` — `uint256 freeBalance = stakeToken.balanceOf(address(this)); … stakeToken.safeTransfer(recoveryAddress, amount)`

There is no `rescueToken`, generic sweep, or any other function that can move a token whose address differs from `stakeToken`. A direct `IERC20(otherToken).transfer(pool, amount)` call by any external party deposits that token into the pool with zero recovery path.

### Impact Explanation
Any ERC20 token other than `stakeToken` that reaches the pool balance is permanently frozen. Unlike the InsuranceFund in the reference report — which was upgradeable and could be patched to sweep tokens later — `ConfidencePool` clones are non-upgradeable by design. Once a non-`stakeToken` asset lands in the pool, it is irrecoverable regardless of the pool's outcome or lifecycle stage.

### Likelihood Explanation
Low. No protocol-internal flow routes non-`stakeToken` assets to the pool; the only vectors are user error (wrong token address in a direct transfer), a bonus contributor calling `transfer` instead of `contributeBonus`, or an unsolicited airdrop. These are uncommon but realistic, especially given that the factory allowlist may contain multiple tokens and a contributor could easily confuse addresses.

### Recommendation
Add an owner-restricted rescue function that transfers any token **other than** `stakeToken` to `recoveryAddress` (or a caller-supplied address):

```solidity
function rescueToken(address token, uint256 amount) external onlyOwner {
    if (token == address(stakeToken)) revert CannotRescueStakeToken();
    IERC20(token).safeTransfer(recoveryAddress, amount);
}
```

Restricting it to non-`stakeToken` assets ensures it cannot be used to bypass the pool's accounting invariants.

### Proof of Concept
1. A pool is deployed with `stakeToken = USDC`.
2. A bonus contributor intends to call `contributeBonus(1000e6)` but instead calls `WETH.transfer(pool, 1e18)` directly (wrong token, direct transfer).
3. The pool now holds 1 WETH.
4. At any resolution outcome (SURVIVED, EXPIRED, CORRUPTED), every sweep and claim function calls `stakeToken.balanceOf(address(this))` — i.e., `USDC.balanceOf(pool)` — and transfers only USDC.
5. The 1 WETH remains in the pool forever. No function in `ConfidencePool` can move it, and the clone is non-upgradeable, so no future code path can be added to recover it. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    function sweepUnclaimedBonus() external nonReentrant {
        if (outcome != PoolStates.Outcome.SURVIVED && outcome != PoolStates.Outcome.EXPIRED) {
            revert OutcomeNotEligibleForSweep();
        }

        // Reserve principal still owed to non-claimers plus any bonus they're entitled to. When
        // `riskWindowStart == 0` (no observable risk), `_bonusShare` returns 0 for everyone, so
        // the bonus is not owed to any staker and the entire snapshotTotalBonus is sweepable.
        uint256 reserved;
        if (totalEligibleStake != 0) {
            reserved = totalEligibleStake;
            if (riskWindowStart != 0) {
                reserved += snapshotTotalBonus - claimedBonus;
            }
        }

        // aderyn-fp-next-line(reentrancy-state-change)
        uint256 freeBalance = stakeToken.balanceOf(address(this));
        uint256 amount = freeBalance > reserved ? freeBalance - reserved : 0;
        if (amount == 0) revert NothingToSweep();

        // Bonus is only unreserved when no staker is owed it (no risk window, or no stakers left).
        // In that case the sweep removes it from the pool, so drop it from the live `totalBonus`
        // too — keeping the accounting honest for any later re-snapshot. Clamp to `totalBonus` so
        // swept donations/dust (never counted in it) can't over-decrement or underflow.
        if (totalEligibleStake == 0 || riskWindowStart == 0) {
            totalBonus -= amount <= totalBonus ? amount : totalBonus;
        }

        // Intentionally does NOT set claimsStarted. A direct-transfer donation of as little as 1
        // wei would otherwise let anyone flip the flag post-flagOutcome and block the moderator's
        // documented pre-claim re-flag window. Genuine reliance only comes from claim entrypoints.
        stakeToken.safeTransfer(recoveryAddress, amount);

        emit BonusSwept(msg.sender, recoveryAddress, amount);
    }
