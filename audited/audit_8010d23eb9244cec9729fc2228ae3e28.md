### Title
Fee-on-Transfer Collateral Inflates Stored Collateral Without Actual Token Receipt, Enabling Uncollateralized Debt - (File: src/libraries/SafeTransferLib.sol / src/Midnight.sol)

### Summary
`supplyCollateral` writes `_position.collateral[collateralIndex]` and sets the `collateralBitmap` bit **before** calling `SafeTransferLib.safeTransferFrom`. `safeTransferFrom` only validates call success and the boolean return value; it never checks the actual balance delta. A fee-on-transfer collateral token with a 100% fee rate returns `true` and does not revert, yet delivers zero tokens to Midnight. `isHealthy` then computes `maxDebt` from the phantom stored collateral, allowing the borrower to pass the health check in `take` and incur real debt with zero actual collateral.

### Finding Description
**Code path:**

1. Attacker deploys a token whose `transferFrom` deducts 100% as a fee, returns `true`, and does not revert.
2. Attacker (or any market creator) calls `touchMarket` with this token as `collateralParams[0].token`. `touchMarket` validates LLTV tier, maxLif, and sort order — it does not inspect token behavior.
3. Attacker calls `supplyCollateral(market, 0, 1e18, borrower)`:
   - Line 533: `_position.collateral[0] = 1e18` — stored.
   - Lines 535-540: `oldCollateral == 0 && assets > 0` → bitmap bit 0 set.
   - Line 545: `SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), 1e18)` — token returns `true`, 0 tokens arrive.
4. State after: `position[id][borrower].collateral[0] == 1e18`, `collateralBitmap == 1`, `token.balanceOf(midnight) == 0`.
5. A lender's buy offer exists in the market (placed by any third party, or by the attacker using a separate account with real loan tokens).
6. Attacker calls `take(buyOffer, ..., units, borrower, ...)`:
   - Line 414: `sellerPos.debt += units` — borrower now has debt.
   - Line 476: `isHealthy` is called. It reads `_position.collateral[0] == 1e18`, multiplies by oracle price and LLTV, computes `maxDebt >= debt` → returns `true`.
   - Take succeeds; borrower receives `sellerAssets` loan tokens.

**Why existing checks fail:**
- `safeTransferFrom` (SafeTransferLib.sol:27-33) checks only `success` and `returndata` boolean — no balance delta check.
- `isHealthy` (Midnight.sol:954-955) reads `_position.collateral[i]`, not `IERC20(token).balanceOf(address(this))`.
- The Certora `nonZeroCollateralsAreActivated` invariant (CollateralBitmap.spec:39-40) proves bitmap ↔ stored-collateral consistency, not stored-collateral ↔ actual-balance consistency.
- No whitelist or token-behavior validation exists in `touchMarket` or `supplyCollateral`.

### Impact Explanation
The borrower holds real loan tokens received from the sell side of `take` and has recorded debt, but Midnight holds zero collateral tokens. The debt is permanently uncollateralized. If the borrower does not repay, lenders absorb the loss through the `lossFactor` bad-debt socialization mechanism. This is pure bad debt creation with no collateral backing, directly violating the core invariant that every borrow must have correct collateral obligation.

### Likelihood Explanation
Market creation via `touchMarket` is permissionless — any address can register a market with an arbitrary collateral token. The attacker only needs to deploy a 100%-fee token, create a market, and attract or self-fund a lender position. The attack is repeatable across any number of markets and requires no privileged access. The only friction is sourcing a lender; with a self-funded lender account the attacker moves their own loan tokens but creates permanent bad debt for that position, which socializes losses to all lenders in the market.

### Recommendation
After `safeTransferFrom`, verify the actual balance increase:
```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
require(IERC20(collateralToken).balanceOf(address(this)) - balanceBefore == assets, FeeOnTransferToken());
```
Alternatively, record only the actual received amount and use that for the collateral update, or explicitly document and enforce that fee-on-transfer tokens are not supported collateral.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
