### Title
Mutable `recoveryAddress` After Staking Allows Sponsor to Redirect Full Pool Sweep to Attacker — (File: src/ConfidencePool.sol)

### Summary
`setRecoveryAddress` imposes no post-stake lock, so the pool sponsor can silently replace the CORRUPTED sweep destination at any time — including after stakers have committed funds and after the outcome has been mechanically set. Under bad-faith CORRUPTED resolution (including the fully permissionless auto-CORRUPTED backstop), the entire pool is swept to whatever `recoveryAddress` holds at the moment `claimCorrupted` executes, not the address stakers observed when they deposited.

### Finding Description
`expiry` is explicitly locked after the first stake via the one-way `expiryLocked` flag, with the rationale documented in `docs/DESIGN.md §10`: *"This protects staker reliance: once anyone has deposited against a given deadline … the sponsor cannot move it."* No equivalent protection exists for `recoveryAddress`. [1](#0-0) 

`setRecoveryAddress` is gated only by `onlyOwner`. It checks neither `expiryLocked`, nor `outcome`, nor `claimsStarted`. The sponsor can call it at any point in the pool's lifecycle — before staking, during the risk window, after `claimExpired` has mechanically set the outcome to CORRUPTED, and even after `claimsStarted` is latched — as long as `claimCorrupted` has not yet drained the balance.

The permissionless auto-CORRUPTED path in `claimExpired` makes this exploitable without any moderator involvement: [2](#0-1) 

After `claimExpired` sets `outcome = CORRUPTED` and latches `claimsStarted = true`, the sponsor still has an open window to call `setRecoveryAddress` before `claimCorrupted` executes. `claimCorrupted` then sweeps the entire balance to the newly set address: [3](#0-2) 

### Impact Explanation
Under bad-faith CORRUPTED (auto or moderator-flagged), `claimCorrupted` transfers `stakeToken.balanceOf(address(this))` — the full pool, comprising all staker principal plus all bonus — to `recoveryAddress`. A sponsor who changes this address to an attacker-controlled account causes total loss of all staker principal and bonus. There is no recovery path once the sweep executes.

### Likelihood Explanation
The sponsor is explicitly listed as an untrusted actor (distinct from the trusted moderator/DAO and trusted factory owner). The auto-CORRUPTED backstop path requires only: (1) the registry reaching `CORRUPTED`, (2) `riskWindowStart != 0` (sealable permissionlessly via `pokeRiskWindow`), and (3) `block.timestamp >= expiry + MODERATOR_CORRUPTED_GRACE`. These are the exact conditions the protocol is designed to handle. The sponsor has the entire 180-day grace window — plus any time before expiry — to substitute `recoveryAddress`. No moderator action or collusion is required.

`docs/DESIGN.md §10` explicitly documents the `expiryLocked` staker protection for `expiry` but contains no analogous accepted trade-off for `recoveryAddress` mutability post-stake. The general disclaimer "verify pool parameters before depositing" does not constitute acceptance of this specific post-deposit substitution attack. [4](#0-3) 

### Recommendation
Mirror the `expiryLocked` pattern for `recoveryAddress`: add a `recoveryAddressLocked` one-way flag (or reuse `expiryLocked`) that is set on the first stake, and revert in `setRecoveryAddress` once it is true. Alternatively, lock `recoveryAddress` at outcome resolution time by snapshotting it into an immutable `snapshotRecoveryAddress` variable (analogous to the stake/bonus snapshots), and use that snapshot in `claimCorrupted` and `sweepUnclaimedCorrupted` instead of reading the live `recoveryAddress`.

### Proof of Concept

1. Sponsor creates pool with `recoveryAddress = 0xLegit` (a legitimate multisig).
2. Stakers deposit; `expiryLocked` is set. Stakers observe `recoveryAddress = 0xLegit` on-chain.
3. Registry transitions to `UNDER_ATTACK`; anyone calls `pokeRiskWindow()` → `riskWindowStart` sealed.
4. Registry transitions to `CORRUPTED`; `riskWindowEnd` sealed.
5. Pool expires. Moderator is absent (or slow). Sponsor calls `setRecoveryAddress(attackerEOA)`.
6. After `expiry + 180 days`, anyone calls `claimExpired()`:
   - `state == CORRUPTED && riskWindowStart != 0` → auto-CORRUPTED branch executes.
   - `outcome = CORRUPTED`, `claimsStarted = true` latched.
   - Function returns early; no funds move yet.
7. Sponsor (or anyone) calls `claimCorrupted()`:
   - `toSweep = stakeToken.balanceOf(address(this))` = entire pool.
   - `stakeToken.safeTransfer(recoveryAddress, toSweep)` → transfers to `attackerEOA`.
8. All staker principal and bonus are lost. No recourse exists. [3](#0-2) [1](#0-0)
