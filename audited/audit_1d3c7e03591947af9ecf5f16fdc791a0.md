### Title
Blacklisted `attacker` address blocks bounty claim and locks all staker principal for 180 days — (File: src/ConfidencePool.sol)

### Summary
`claimAttackerBounty()` pushes the stake token directly to the hardcoded `attacker` address. If the stake token implements address-level blacklisting (e.g., USDC) and the named `attacker` is blacklisted, the transfer reverts. Because `claimCorrupted()` is gated by `MustClaimBountyFirst`, the entire pool — all stakers' principal plus bonus — is frozen for the full 180-day `CORRUPTED_CLAIM_WINDOW`. Stakers cannot recover their funds until `sweepUnclaimedCorrupted()` becomes callable, at which point the attacker's bounty is forfeited to `recoveryAddress`.

### Finding Description
In the good-faith CORRUPTED resolution path, `flagOutcome()` sets `bountyEntitlement = snapshotTotalStaked + snapshotTotalBonus` (the entire pool) and records the moderator-named `attacker` address. [1](#0-0) 

`claimAttackerBounty()` then transfers directly to that address: [2](#0-1) 

`claimCorrupted()` — the only path that sweeps the pool to `recoveryAddress` — is hard-gated: [3](#0-2) 

If the stake token is USDC (a standard ERC20 on the factory allowlist) and the named `attacker` is blacklisted by USDC, `safeTransfer(attacker, payout)` reverts unconditionally. Because `bountyClaimed` never reaches `bountyEntitlement`, the `MustClaimBountyFirst` guard permanently blocks `claimCorrupted()`. The only remaining exit is `sweepUnclaimedCorrupted()`, which requires `block.timestamp > corruptedClaimDeadline`: [4](#0-3) 

`corruptedClaimDeadline` is `_firstGoodFaithCorruptedAt + CORRUPTED_CLAIM_WINDOW` (180 days): [5](#0-4) 

So all staker principal is locked for up to 180 days with no recourse.

DESIGN.md §12 acknowledges token-level stuck-states only for tokens that "freeze the pool's own balance," which the factory allowlist is said to exclude. Recipient-level blacklisting (the pool's balance is intact; only the transfer *to* `attacker` fails) is a distinct scenario not covered by that defense. [6](#0-5) 

### Impact Explanation
All stakers' principal and bonus (the entire pool snapshot) is inaccessible for up to 180 days. After the window, `sweepUnclaimedCorrupted()` redirects the attacker's full entitlement to `recoveryAddress`, so the named whitehat loses their entire bounty. Stakers suffer a 180-day forced lock of capital they are owed, with no ability to withdraw or claim through any other path.

### Likelihood Explanation
USDC is the canonical "standard ERC20" used in DeFi and is a plausible factory-allowlisted stake token. USDC blacklisting of specific addresses (e.g., following regulatory action or sanctions) is a documented, real-world occurrence. The `attacker` address is a named whitehat whose on-chain identity is public; if that address is ever sanctioned or blacklisted, the condition triggers automatically with no action required from any unprivileged party.

### Recommendation
Replace the push pattern in `claimAttackerBounty()` with a pull-payment pattern:

1. Credit the bounty amount to a `pendingBounty[attacker]` mapping instead of transferring immediately.
2. Add a separate `withdrawBounty()` function that lets the attacker pull their credited amount.

This isolates a blacklisted attacker's inability to receive tokens to themselves alone, leaving `claimCorrupted()` and the rest of the resolution path unaffected for stakers.

### Proof of Concept
1. Pool is deployed with USDC as the stake token (factory-allowlisted).
2. Multiple stakers deposit USDC; `snapshotTotalStaked = 1_000_000e6`.
3. Agreement is marked `CORRUPTED`; moderator calls `flagOutcome(CORRUPTED, true, 0xAttacker)`.
   - `bountyEntitlement = 1_000_000e6`, `corruptedClaimDeadline = block.timestamp + 180 days`.
4. USDC blacklists `0xAttacker` (e.g., regulatory action).
5. `0xAttacker` calls `claimAttackerBounty()` → `stakeToken.safeTransfer(attacker, payout)` reverts with USDC blacklist error. `bountyClaimed` remains `0`.
6. Any caller tries `claimCorrupted()` → reverts `MustClaimBountyFirst` because `goodFaith == true && bountyClaimed (0) < bountyEntitlement (1_000_000e6)`.
7. Stakers have no path to recover principal for 180 days.
8. After 180 days, `sweepUnclaimedCorrupted()` is called → entire pool swept to `recoveryAddress`; `0xAttacker` receives nothing.

```text
    function sweepUnclaimedCorrupted() external nonReentrant {
        if (outcome != PoolStates.Outcome.CORRUPTED) revert OutcomeNotSet();
        if (!goodFaith) revert NotGoodFaithCorrupted();
        if (block.timestamp <= corruptedClaimDeadline) revert ClaimWindowNotExpired();

        // aderyn-fp-next-line(reentrancy-state-change)
        uint256 amount = stakeToken.balanceOf(address(this));
        if (amount == 0) revert NothingToSweep();

        corruptedReserve = 0;
        bountyClaimed = bountyEntitlement;
        if (!claimsStarted) claimsStarted = true;
        stakeToken.safeTransfer(recoveryAddress, amount);
```
