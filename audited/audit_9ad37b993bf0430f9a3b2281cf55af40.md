Audit Report

## Title
Authorized-Signer Front-Run Advances Victim's Nonce and Invalidates Pending Signed Authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` keys the nonce solely on `authorization.authorizer` and permits any address already authorized in Midnight to sign an arbitrary `Authorization` struct at that nonce. A delegated signer (attacker) can observe a victim's pending signed authorization in the mempool, craft a different struct at the same nonce, sign it with their own key, and submit it with higher gas priority. This consumes nonce N and causes the victim's transaction to revert with `InvalidNonce()`, permanently invalidating the victim's signed message. The attack is indefinitely repeatable on every future signed authorization the victim broadcasts.

## Finding Description

**Root cause:**

In `EcrecoverAuthorizer.setIsAuthorized`, the nonce is incremented keyed only on `authorization.authorizer`:

```solidity
// src/periphery/EcrecoverAuthorizer.sol line 26
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

The signer check accepts any address for which `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns `true`:

```solidity
// lines 33-36
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

There is no binding between the signer's identity and the `authorized` field in the struct. The `Authorization` struct (`src/periphery/interfaces/IEcrecoverAuthorizer.sol` lines 11-17) contains five independently attacker-controllable fields: `authorizer`, `authorized`, `isAuthorized`, `nonce`, `deadline`. A delegated signer is permitted to sign any combination of these fields, including a struct that differs from the victim's intended struct but shares the same `authorizer` and `nonce`.

**Exploit flow:**

1. **Precondition**: `isAuthorized[victim][attacker] == true` in Midnight (victim previously authorized attacker as an operator — a normal, expected state confirmed by `testSetIsAuthorizedAuthorization` in `test/AuthorizationTest.sol` lines 290–304).
2. Victim signs `Authorization{authorizer=victim, authorized=X, isAuthorized=true, nonce=N}` and broadcasts.
3. Attacker observes victim's tx in mempool; crafts `Authorization{authorizer=victim, authorized=attacker, isAuthorized=true, nonce=N}` (same nonce, different `authorized`).
4. Attacker signs this struct with their own private key → `ecrecover(digest, ...) == attacker`.
5. Attacker submits with higher gas priority.
6. Contract execution:
   - Line 25: deadline check passes (attacker sets any valid future deadline).
   - Line 26: `N == nonce[victim]` → passes; nonce increments to N+1.
   - Line 31: `signer = attacker`.
   - Lines 33–36: `isAuthorized[victim][attacker] == true` → passes.
   - Line 47: `Midnight.setIsAuthorized(attacker, true, victim)` executes.
7. Victim's tx arrives: `N != N+1` → reverts with `InvalidNonce()`.

**Why existing checks fail:**

- The nonce check prevents replay of the *same* signed message but does not prevent a *different* message at the same nonce from a delegated signer.
- The `isAuthorized` delegation path was designed to allow authorized agents to act on behalf of the authorizer, but it does not scope what struct content the agent may sign — the agent can sign any `authorized`/`isAuthorized` combination.
- There is no binding between the signer's identity and the `authorized` field in the struct.

## Impact Explanation

The victim's pending signed `Authorization` is permanently invalidated (DoS of the gasless delegation path). The victim must sign a new message at nonce N+1, which the attacker can front-run again indefinitely. This constitutes a persistent, unrecoverable service degradation for the victim's signature-gated delegation path through `EcrecoverAuthorizer`. Additionally, the attacker can set arbitrary `isAuthorized` entries on behalf of the victim via `EcrecoverAuthorizer` — though note that an already-authorized operator can also call `Midnight.setIsAuthorized` directly (as confirmed by `testSetIsAuthorizedAuthorization` in `test/AuthorizationTest.sol` lines 290–304). The primary unique impact is the indefinite DoS of the gasless authorization flow.

## Likelihood Explanation

The precondition (`isAuthorized[victim][attacker] == true`) is a common, expected state — users routinely authorize operators, bots, and relayers to act on their behalf. Once met, the attack requires only mempool monitoring and a higher-gas competing transaction, which is standard MEV infrastructure. The attack is trivially repeatable on every future signed authorization the victim broadcasts, making it a persistent griefing vector with no natural expiry.

## Recommendation

Bind the nonce to the signer rather than solely to the authorizer, so that each signer has an independent nonce namespace. Alternatively, require that a delegated signer may only sign structs where `authorized == signer` (i.e., a delegate may only authorize themselves, not arbitrary third parties). A third option is to include the signer's address as a field in the `Authorization` struct and verify it matches `ecrecover` output, preventing a delegate from signing a struct intended for a different signer identity.

## Proof of Concept

```solidity
// Foundry test outline
function testFrontRunNonceDoS() public {
    // Setup: victim authorizes attacker as operator in Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim also authorizes EcrecoverAuthorizer
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);

    // Victim signs Authorization{authorizer=victim, authorized=thirdParty, isAuthorized=true, nonce=0}
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: thirdParty,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victim);

    // Attacker front-runs with different struct at same nonce=0, signed by attacker
    Authorization memory attackerAuth = Authorization({
        authorizer: victim,
        authorized: attacker,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory attackerSig = signAuthorization(attackerAuth, attacker);

    // Attacker submits first — succeeds, nonce advances to 1
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);

    // Victim's tx now reverts with InvalidNonce
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

    // thirdParty was never authorized
    assertEq(midnight.isAuthorized(victim, thirdParty), false);
}
```
