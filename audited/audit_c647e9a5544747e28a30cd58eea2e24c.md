### Title
Authorized agent can exhaust victim's `EcrecoverAuthorizer` nonce to invalidate pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for, not only the authorizer themselves. An attacker who holds a Midnight authorization from the victim can sign and submit sequential no-op `Authorization` structs naming the victim as `authorizer`, incrementing `nonce[victim]` on each call without the victim's per-call consent. This burns through all nonces the victim embedded in pre-signed authorizations distributed to other agents, rendering those signatures permanently invalid.

### Finding Description
**Exact code path:**

In `EcrecoverAuthorizer.setIsAuthorized` the nonce is incremented unconditionally before signature verification:

```solidity
// src/periphery/EcrecoverAuthorizer.sol line 26
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

Then the signer check at lines 33–36 is:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

The second branch allows *any* address that the victim has authorized on Midnight to act as a valid signer for the victim's `Authorization` struct, including incrementing `nonce[victim]`.

**Attacker-controlled inputs:**
- `authorization.authorizer` = victim address
- `authorization.authorized` = attacker address (or any address)
- `authorization.isAuthorized` = `true` (no-op if already true)
- `authorization.nonce` = current `nonce[victim]` (attacker reads this on-chain)
- `authorization.deadline` = far future
- `signature` = attacker's own valid ECDSA signature over the above struct

**Exploit flow:**
1. Victim calls `midnight.setIsAuthorized(attacker, true, victim)` — attacker is now authorized on Midnight. [2](#0-1) 
2. Victim pre-signs `K` `Authorization` structs (nonces 0 through K−1) and distributes them to K agents for future use (e.g., automated repay, liquidation protection, withdraw).
3. Attacker reads `nonce[victim] == 0`, constructs `Authorization{authorizer: victim, authorized: attacker, isAuthorized: true, nonce: 0, deadline: far}`, signs it with attacker's own key, and calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
4. The check `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` passes; `nonce[victim]` becomes 1.
5. Attacker repeats with nonce 1, 2, … K−1 in a tight loop.
6. All K pre-signed authorizations held by the victim's agents now revert with `InvalidNonce()` when submitted. [3](#0-2) 

**Why existing checks fail:**
The `Unauthorized()` guard is satisfied by the attacker's Midnight authorization. There is no check that the signer is the `authorizer` themselves when the nonce is being consumed. The nonce is incremented at line 26 before the signature is even verified, so the state change is committed regardless. [4](#0-3) 

### Impact Explanation
Any Midnight operation gated on `isAuthorized[victim][agent]` that the victim intended to enable via pre-signed `EcrecoverAuthorizer` authorizations — `take`, `repay`, `withdraw`, `withdrawCollateral`, `liquidate`, `claimFee`, `setConsumed` — is blocked. The victim's agents cannot submit their pre-signed authorizations because every nonce they hold has been consumed by the attacker's no-ops. The attacker can sustain the DoS indefinitely by continuing to burn newly generated nonces. [5](#0-4) 

### Likelihood Explanation
**Precondition:** The attacker must hold `isAuthorized[victim][attacker] == true` on Midnight. This is a realistic precondition: victims routinely authorize agents (smart contracts, keepers, relayers) for legitimate purposes such as automated repayment or liquidation protection. Once that authorization exists, the attacker can exploit it at any time.

**Feasibility:** Each attack call costs only one ECDSA signature verification and one storage write. On a cheap-gas chain the cost per nonce burn is negligible. The attacker can front-run any submission of a victim's pre-signed authorization by watching the mempool and burning the nonce first.

**Repeatability:** The attack is fully repeatable. Every time the victim generates and distributes new pre-signed authorizations, the attacker can burn the new nonces again.

### Recommendation
Remove the delegated-signer branch from `EcrecoverAuthorizer.setIsAuthorized`. The nonce is a replay-protection mechanism tied to the authorizer's identity; only the authorizer's own key should be able to consume it:

```solidity
// Replace lines 33-36 with:
require(signer == authorization.authorizer, Unauthorized());
```

If delegated signing is intentionally desired, it must be gated on a separate, explicit consent mechanism (e.g., a dedicated nonce or a separate on-chain approval) that cannot be abused by an already-authorized agent. [6](#0-5)

// Foundry fuzz/stateful test plan
// File: test/EcrecoverAuthorizerNonceExhaustion.t.sol

function testNonceExhaustionDoS(uint8 K) public {
    K = uint8(bound(K, 1, 20));

    // Precondition: victim authorizes attacker on Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim pre-signs K authorizations for K different agents (nonces 0..K-1)
    Signature[] memory victimSigs = new Signature[](K);
    Authorization[] memory victimAuths = new Authorization[](K);
    for (uint8 i = 0; i < K; i++) {
        victimAuths[i] = Authorization({
            authorizer: victim, authorized: agents[i],
            isAuthorized: true, nonce: i, deadline: block.timestamp + 1 days
        });
        victimSigs[i] = signAuthorization(victimAuths[i], victim);
    }

    // Attacker burns nonces 0..K-1 using their own signatures
    for (uint8 i = 0; i < K; i++) {
        Authorization memory attackAuth = Authorization({
            authorizer: victim, authorized: attacker,
            isAuthorized: true, nonce: i, deadline: block.timestamp + 1 days
        });
        Signature memory attackSig = signAuthorization(attackAuth, attacker);
        ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);
        // Assert: nonce[victim] incremented each time
        assertEq(ecrecoverAuthorizer.nonce(victim), i + 1);
    }

    // Assert: all victim's pre-signed authorizations now revert
    for (uint8 i = 0; i < K; i++) {
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuths[i], victimSigs[i]);
    }
}
```

Expected assertions: all `K` victim pre-signed authorizations revert with `InvalidNonce()`, confirming sustained DoS on any downstream Midnight action those authorizations were intended to enable. [7](#0-6)

        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
