### Title
Authorized operator can force-activate a reverting-oracle collateral index on a victim's bitmap, permanently blocking liquidation and collateral withdrawal - (`src/Midnight.sol`)

### Summary
`supplyCollateral` gates bitmap activation behind the same `isAuthorized[onBehalf][msg.sender]` check used for all other on-behalf operations. An authorized operator can therefore call `supplyCollateral(market, poisonedIndex, 1, victim)` to set a new bit in the victim's `collateralBitmap` for a collateral whose oracle always reverts. Once that bit is set and the victim has debt, every code path that iterates the bitmap (`liquidate`, `isHealthy` inside `withdrawCollateral`) will revert, making the position permanently unliquidatable and unwithdrawable while the victim remains unhealthy.

### Finding Description

**Exact code path**

`supplyCollateral` (line 527) passes the authorization check when `isAuthorized[onBehalf][msg.sender]` is true:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [1](#0-0) 

When `oldCollateral == 0 && assets > 0`, the function unconditionally sets the bit:

```solidity
if (oldCollateral == 0 && assets > 0) {
    uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
    _position.collateralBitmap = newCollateralBitmap;
    ...
}
``` [2](#0-1) 

No check is made on whether the oracle at `collateralIndex` is functional.

**Why `liquidate` is blocked**

`liquidate` iterates the full bitmap and calls `IOracle(_collateralParam.oracle).price()` for every set bit before any bit is cleared: [3](#0-2) 

If any oracle in the bitmap reverts, the entire `liquidate` call reverts. The Certora spec formally proves this: [4](#0-3) 

**Why `withdrawCollateral` is blocked**

`withdrawCollateral` clears the bit for the *withdrawn* index before calling `isHealthy`, but `isHealthy` still iterates all *remaining* set bits: [5](#0-4) 

The Certora spec proves: if the reverting oracle is at a *different* index than the one being withdrawn, `withdrawCollateral` reverts whenever the borrower has debt: [6](#0-5) 

**Why the victim cannot self-rescue when unhealthy**

The victim can withdraw the poisoned index directly (clearing its bit before `isHealthy` runs), but only if `isHealthy` returns `true` afterward. When the victim is unhealthy (debt > maxDebt from remaining collateral), `isHealthy` returns `false` regardless, so `withdrawCollateral` reverts with `UnhealthyBorrower`. The victim is fully stuck unless they can repay all debt.

`isHealthy` only skips oracle calls when `debt == 0`: [7](#0-6) 

**Exploit flow**

1. Victim has debt in market M; victim's bitmap has bit 0 set (working oracle).
2. Attacker holds `isAuthorized[victim][attacker] == true`.
3. Attacker deploys `RevertingOracle` (always reverts on `price()`).
4. Attacker creates or uses a market M where `collateralParams[1].oracle == RevertingOracle`.
5. Attacker calls `supplyCollateral(M, 1, 1 wei, victim)` — bit 1 is set in victim's bitmap.
6. Victim's price drops; victim becomes unhealthy.
7. Any call to `liquidate(M, *, *, *, victim, ...)` reverts (hits reverting oracle at bit 1).
8. `withdrawCollateral(M, 0, *, victim, *)` reverts (isHealthy hits bit 1).
9. `withdrawCollateral(M, 1, 1 wei, victim, *)` clears bit 1, but `isHealthy` returns false (victim is unhealthy on bit 0 alone) → reverts.
10. Victim is permanently stuck; bad debt accumulates.

### Impact Explanation

An unhealthy borrower whose bitmap contains a reverting-oracle collateral index cannot be liquidated (all `liquidate` calls revert) and cannot withdraw any collateral (all `withdrawCollateral` calls revert). The position becomes permanently insolvent with no recovery path short of full debt repayment, which may be impossible for a deeply underwater position. Bad debt accrues to the protocol's lenders.

### Likelihood Explanation

**Preconditions:**
- `isAuthorized[victim][attacker] == true` — common in DeFi where users authorize smart-contract position managers, routers, or aggregators.
- Attacker holds a trivial amount of any ERC-20 token accepted as collateral in the market.
- Victim has outstanding debt (is a borrower).

**Feasibility:** The attacker needs only one `supplyCollateral` call. The reverting oracle can be any contract that always reverts on `price()`. The attack can be set up in advance (while the victim is healthy) and becomes effective the moment the victim's position turns unhealthy. It is repeatable and cheap.

### Recommendation

Restrict bitmap activation to the position owner only. Split the authorization check: allow authorized operators to top up *existing* collateral, but require `onBehalf == msg.sender` when `oldCollateral == 0` (i.e., when a new bit would be set):

```solidity
function supplyCollateral(...) external {
    if (oldCollateral == 0 && assets > 0) {
        require(onBehalf == msg.sender, Unauthorized()); // only owner may activate a new index
        ...setBit...
    } else {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
    }
    ...
}
```

Alternatively, restructure so the outer authorization check is applied only after determining whether a new bit would be set, and tighten it to `msg.sender == onBehalf` in that branch.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight, Market, CollateralParams} from "src/Midnight.sol";

contract RevertingOracle {
    function price() external pure returns (uint256) { revert("oracle down"); }
}

contract PoisonCollateralTest is Test {
    Midnight midnight;
    address victim  = makeAddr("victim");
    address attacker = makeAddr("attacker");

    function setUp() public {
        midnight = new Midnight(...);
        // victim authorizes attacker (e.g. as a position manager)
        vm.prank(victim);
        midnight.setIsAuthorized(attacker, true, victim);
    }

    function testPoisonCollateralBlocksLiquidation() public {
        // 1. Build a two-collateral market: index 0 = working oracle, index 1 = reverting oracle
        RevertingOracle badOracle = new RevertingOracle();
        Market memory market = _buildMarket(workingOracle, badOracle);
        midnight.touchMarket(market);

        // 2. Victim supplies index-0 collateral and takes on debt (becomes borrower)
        _supplyAndBorrow(victim, market, /*index=*/0, /*collateral=*/1e18, /*debt=*/0.9e18);

        // 3. Attacker activates index 1 (reverting oracle) on victim's bitmap
        deal(address(market.collateralParams[1].token), attacker, 1);
        vm.startPrank(attacker);
        IERC20(market.collateralParams[1].token).approve(address(midnight), 1);
        midnight.supplyCollateral(market, 1, 1, victim);
        vm.stopPrank();

        // 4. Price drops — victim is now unhealthy
        workingOracle.setPrice(0.5e36); // drop price so debt > maxDebt

        // 5. Assert: liquidate reverts
        vm.expectRevert();
        midnight.liquidate(market, 0, 0, 1, victim, false, attacker, address(0), "");

        // 6. Assert: withdrawCollateral on index 0 reverts (isHealthy hits reverting oracle at bit 1)
        vm.prank(victim);
        vm.expectRevert();
        midnight.withdrawCollateral(market, 0, 1e18, victim, victim);

        // 7. Assert: withdrawCollateral on index 1 (poisoned) also reverts (victim is unhealthy on index 0 alone)
        vm.prank(victim);
        vm.expectRevert();
        midnight.withdrawCollateral(market, 1, 1, victim, victim);
    }
}
```

**Expected assertions:** all three calls in steps 5–7 revert, confirming the victim's position is permanently unliquidatable and unwithdrawable.

rule oracleRevertCausesLiquidateRevert(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, uint256 revertingCollateralIndex, bool postMaturityMode) {

/// If an activated collateral oracle reverts on price different than withdrawn collateral, withdrawCollateral reverts when the borrower has debt.
rule oracleRevertCausesWithdrawCollateralRevert(env e, Midnight.Market market, uint256 collateralIndex, uint256 assets, address onBehalf, address receiver, uint256 revertingCollateralIndex) {
    require singleRevertingOracle == market.collateralParams[revertingCollateralIndex].oracle, "oracle is reverting";
    require revertingCollateralIndex < 128, "clearBit produces a new bitmap whose summaryGetBit is unconstrained for indices >= 128";
    require revertingCollateralIndex != collateralIndex, "withdrawCollateral may clear the bit at collateralIndex before calling isHealthy";

    bytes32 id = summaryToId(market);
    uint128 bitmap = collateralBitmap(id, onBehalf);
    require summaryGetBit(bitmap, revertingCollateralIndex), "revertingCollateralIndex is activated";

    withdrawCollateral@withrevert(e, market, collateralIndex, assets, onBehalf, receiver);
    bool reverted = lastReverted;

    assert debtOf(id, onBehalf) > 0 => reverted;
}
