### Title
Authorized agent can exhaust victim's EcrecoverAuthorizer nonce to invalidate all pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that is authorized by `authorization.authorizer` on Midnight, not only from the authorizer themselves. Because the nonce increment at line 26 fires before the signature check and is keyed on `authorization.authorizer`, an attacker who holds a Midnight authorization from the victim can sign arbitrary `Authorization` structs naming the victim as `authorizer`, submit them in a loop, and burn through `nonce[victim]` at will. Every pre-signed authorization the victim distributed to other agents becomes permanently invalid.

### Finding Description

**Exact code path:**

`EcrecoverAuthorizer.setIsAuthorized` (src/periphery/EcrecoverAuthorizer.sol:24-48):

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
...
Line 31: address signer = ecrecover(digest, signature.v, signature.r, signature.s);
Line 33-36:
    require(
        signer == authorization.authorizer
            || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
        Unauthorized()
    );
``` [1](#0-0) [2](#0-1) 

**Root cause:** The `Unauthorized` check at line 34 has two branches. The second branch ŌĆö `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` ŌĆö allows any address that the victim has authorized on Midnight to act as the signer for an `Authorization` struct that names the victim as `authorizer`. This means the attacker's own private-key signature on a struct with `authorizer = victim` passes the check, and the nonce increment on line 26 fires against `nonce[victim]`. [3](#0-2) 

**Attacker-controlled inputs:**
- `authorization.authorizer` = victim address
- `authorization.authorized` = attacker address (or any address)
- `authorization.isAuthorized` = true (no-op)
- `authorization.nonce` = current `nonce[victim]` (attacker reads this from chain)
- `authorization.deadline` = any future timestamp
- `signature` = attacker's own ECDSA signature over the above struct

**Exploit flow:**
1. Precondition: `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` (victim previously authorized attacker on Midnight, e.g., to act as a trading agent).
2. Victim pre-signs N authorizations with nonces 0ŌĆ”N-1 and distributes them to N agents for use in `take`, `repay`, `withdraw`, `liquidate`, or `claimFee` flows.
3. Attacker constructs `Authorization(authorizer=victim, authorized=attacker, isAuthorized=true, nonce=0, deadline=far_future)`, signs it with their own key, and calls `EcrecoverAuthorizer.setIsAuthorized`.
4. Line 26 passes (`0 == nonce[victim]`), increments `nonce[victim]` to 1.
5. Line 34 passes because `isAuthorized[victim][attacker] == true`.
6. Attacker repeats with nonce=1, 2, ŌĆ”, N-1 in a tight loop.
7. All N pre-signed authorizations from the victim now revert with `InvalidNonce` when any agent tries to submit them.

**Why existing checks fail:**
- The `InvalidNonce` check (line 26) only enforces sequential ordering; it does not verify that the authorizer themselves signed this particular nonce consumption.
- The `Unauthorized` check (lines 33-36) is evaluated *after* the nonce is already incremented and does not restrict who may consume the authorizer's nonces ŌĆö only that the signer is either the authorizer or someone the authorizer trusts on Midnight.
- There is no check that `msg.sender == authorization.authorizer` or that the signature is from the authorizer's own key specifically. [4](#0-3) 

### Impact Explanation
Any agent holding a Midnight authorization from the victim can unilaterally burn the victim's sequential nonces in `EcrecoverAuthorizer`. This permanently invalidates any batch of pre-signed `Authorization` structs the victim distributed off-chain. Downstream, every protocol action that relies on those pre-signed authorizations ŌĆö `take`, `repay`, `withdraw`, `liquidate`, `claimFee` ŌĆö reverts with `InvalidNonce` when submitted by the intended agents, causing a sustained DoS on all those core market actions for the victim.

### Likelihood Explanation
**Precondition:** The attacker must hold `isAuthorized[victim][attacker] == true` on Midnight. This is a normal operational state: victims routinely authorize bots, routers, or agents to act on their behalf. The attacker does not need any tokens, collateral, or special role beyond this existing authorization.

**Feasibility:** The attack requires only repeated calls to `EcrecoverAuthorizer.setIsAuthorized` with self-signed structs. Each call costs one ECDSA signature (computed off-chain) and one transaction. On any L2 or low-fee chain, the cost per nonce burned is negligible. The attacker can front-run or race any legitimate submission.

**Repeatability:** The attack is indefinitely repeatable as long as the attacker remains authorized on Midnight. Even if the victim revokes the attacker's Midnight authorization, any nonces already burned are permanently consumed.

### Recommendation
Restrict nonce consumption to signatures from the `authorization.authorizer` exclusively. Remove the second branch of the `Unauthorized` check, so only the authorizer's own private key can sign an `Authorization` struct that increments their nonce:

```solidity
// Before (vulnerable):
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// After (fixed):
require(signer == authorization.authorizer, Unauthorized());
```

This preserves the EIP-712 intent (the authorizer signs their own delegation) while eliminating the ability of any third party to burn the authorizer's nonces. Authorized agents who need to submit on behalf of the authorizer should relay the authorizer's own pre-signed message, not produce a new signature.

    function testNonceDrain(uint8 n) public {
        n = uint8(bound(n, 1, 50));

        // Victim pre-signs N authorizations for N legitimate agents

        Authorization[] memory victimAuths = new Authorization[](n);
        Signature[] memory victimSigs = new Signature[](n);

            agents[i] = makeAddr(string(abi.encode("agent", i)));

                authorized: agents[i],

                nonce: i,

        assertEq(ecrecoverAuthorizer.nonce(borrower), n);

        // Assert: all victim's pre-signed authorizations now revert
        for (uint8 i = 0; i < n; i++) {
            vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
            ecrecoverAuthorizer.setIsAuthorized(victimAuths[i], victimSigs[i]);
        }

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
