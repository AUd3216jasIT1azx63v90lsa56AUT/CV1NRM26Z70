### Title
Missing `loanToken != address(0)` validation in `touchMarket` permanently freezes all takes on the market - (`src/periphery/TakeAmountsLib.sol`)

### Summary
`touchMarket` in `src/Midnight.sol` validates collateral tokens, LLTV, maxLif, and maturity, but performs no check that `market.loanToken != address(0)`. Any unprivileged caller can create a market with `loanToken = address(0)`, which silently inherits all-zero settlement fees from `defaultSettlementFeeCbp[address(0)]`. `TakeAmountsLib.buyerAssetsToUnits` then returns a nonzero unit count computed against a zero settlement fee, but every subsequent `take()` call reverts with `SafeTransferLib.NoCode()` because `address(0).code.length == 0`, permanently freezing all takes on that market.

### Finding Description
**Root cause — `touchMarket` (lines 755–791, `src/Midnight.sol`):**

The creation guard checks maturity, collateral count, collateral sort order, LLTV tier, and maxLif, but contains no `require(market.loanToken != address(0))`:

```solidity
if (marketState[id].tickSpacing == 0) {
    require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
    require(market.collateralParams.length > 0, NoCollateralParams());
    // ... collateral loop: only checks token > previousCollateralToken, lltv, maxLif
    uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken]; // address(0) → all zeros
    ...
}
```

The collateral sort check (`collateralToken > previousCollateralToken`, where `previousCollateralToken` starts at `address(0)`) only prevents collateral tokens from being `address(0)`, not the loan token.

**Exploit path:**

1. Attacker calls `touchMarket(market)` with `market.loanToken = address(0)` and any valid collateral/LLTV/maturity. Market is created with `settlementFeeCbp0..6 = 0` and `continuousFee = 0`.

2. Attacker (or any user) calls `TakeAmountsLib.buyerAssetsToUnits(midnight, id, offer, targetBuyerAssets)`. The function reads `settlementFee = IMidnight(midnight).settlementFee(id, ...) = 0`, computes `buyerPrice = offerPrice` (nonzero for any valid tick), and returns `targetBuyerAssets.mulDivDown(WAD, buyerPrice)` — a nonzero value.

3. Any caller attempts `take(offer, ...)` on this market. `take()` reaches line 455:
   ```solidity
   SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
   ```
   `SafeTransferLib.safeTransferFrom` immediately reverts:
   ```solidity
   require(token.code.length > 0, NoCode()); // address(0).code.length == 0 → revert
   ```

**Why existing checks fail:**
- `touchMarket` has no `loanToken` zero-address guard.
- `SafeTransferLib` guards against zero-code addresses, but this check fires at execution time, not at market creation time.
- The existing Certora spec `createdMarketsHaveNonZeroCollaterals` only covers collateral tokens, not the loan token.
- The test at `test/SettersTest.sol:267–276` explicitly creates a market with `loanToken: address(0)` and calls `touchMarket` successfully, confirming the path is reachable.

### Impact Explanation
Every `take()` on a market created with `loanToken = address(0)` reverts with `SafeTransferLib.NoCode()` at the token transfer step. The market state is permanently created (markets cannot be deleted), so all takes on this market are frozen forever. `TakeAmountsLib.buyerAssetsToUnits` returns a nonzero unit count (computed with zero settlement fee), misleading callers into believing a take is feasible when it will always revert.

### Likelihood Explanation
Any unprivileged address can call `touchMarket` with `loanToken = address(0)` — no role, signature, or special permission is required. The precondition is trivially satisfiable in a single transaction. The market, once created, is immutable and permanently broken. The attack is repeatable for any `(address(0), maturity)` pair not yet created.

### Recommendation
Add a zero-address check for `loanToken` at the top of the market-creation branch in `touchMarket`:

```solidity
require(market.loanToken != address(0), InvalidLoanToken());
```

Optionally, also add `require(market.loanToken.code.length > 0, InvalidLoanToken())` to enforce that the loan token is a deployed contract at creation time, consistent with the `SafeTransferLib` invariant.

    function buyerAssetsToUnits(address midnight, bytes32 id, Offer memory offer, uint256 targetBuyerAssets)
        internal
        view
        returns (uint256)
    {
        uint256 offerPrice = TickLib.tickToPrice(offer.tick);
        uint256 settlementFee =
            IMidnight(midnight).settlementFee(id, UtilsLib.zeroFloorSub(offer.market.maturity, block.timestamp));
        // Mirrors Midnight's computation to revert if offerPrice < settlementFee in case of a buy offer.
        uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
        uint256 buyerPrice = sellerPrice + settlementFee;
        require(buyerPrice <= WAD, TickLib.PriceGreaterThanOne());
        return offer.buy ? targetBuyerAssets.mulDivUp(WAD, buyerPrice) : targetBuyerAssets.mulDivDown(WAD, buyerPrice);
    }
