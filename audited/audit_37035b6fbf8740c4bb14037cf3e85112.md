### Title
Settlement Fee Interpolation Truncation Causes Systematic Protocol Fee Undercollection - (`src/Midnight.sol`)

### Summary

`settlementFee()` computes a piecewise linear interpolation using plain integer (floor) division. Because `feeLower` and `feeUpper` are integer multiples of `CBP = 1e12` but the interval widths in seconds are not multiples of `CBP`, the division can produce a result that is not a multiple of `CBP`, truncating up to `CBP − 1 = 999 999 999 999` wei below the true interpolated value. Every `take()` call at a non-breakpoint TTM with `feeLower ≠ feeUpper` collects a settlement fee that is strictly less than the true linear interpolation, and a taker can freely choose the block timestamp to land at the TTM that maximises this truncation.

### Finding Description

**Code path:**

`settlementFee()` at `src/Midnight.sol:979`:
```solidity
return (feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start)) / (end - start);
``` [1](#0-0) 

`feeLower` and `feeUpper` are constructed as `settlementFeeCbpN * CBP` where `CBP = 1e12` and `settlementFeeCbpN` is a `uint16` integer. [2](#0-1) 

`CBP` is defined as `1e12`: [3](#0-2) 

The numerator factors as `CBP × (A × (end − TTM) + B × (TTM − start))` where A, B are uint16 integers. The denominator `(end − start)` is a pure second-count (e.g. 518 400 for the [1 d, 7 d] interval). Because `gcd(1e12, 518400) = 86400`, the division truncates whenever `(A × (end − TTM) + B × (TTM − start))` is not divisible by `518400 / 86400 = 6`.

**Concrete example** — interval [1 d, 7 d], `feeLower = 1 × CBP`, `feeUpper = 2 × CBP`, TTM = 3 days:
- numerator = `1e12 × (345600 + 345600) = 1e12 × 691200`
- denominator = `518400`
- exact result = `1e12 × 4/3 = 1 333 333 333 333.33…`
- Solidity integer result = `1 333 333 333 333`
- truncation = `333 333 333 333` wei ≈ 0.33 × CBP

**Exploit flow:**

1. Taker observes `feeLower ≠ feeUpper` for the relevant breakpoint interval.
2. Taker computes the TTM (i.e. the `block.timestamp`) that maximises `(numerator mod denominator)`, giving the largest truncation.
3. Taker submits `take()` at that block timestamp. `timeToMaturity = zeroFloorSub(offer.market.maturity, block.timestamp)` is fully determined by the taker's choice of when to transact.
4. `_settlementFee` is the truncated value; `buyerPrice = sellerPrice + _settlementFee` is lower than the true interpolated price.
5. `claimableSettlementFee` is incremented by `buyerAssets − sellerAssets`, which is computed from the truncated fee. [4](#0-3) [5](#0-4) 

**Why existing checks do not stop it:**

The `setMarketSettlementFee` / `setDefaultSettlementFee` functions enforce `newSettlementFee % CBP == 0` on the stored breakpoint values, but impose no constraint on the interpolated result. [6](#0-5) 

There is no rounding-up or ceiling division anywhere in `settlementFee()`.

### Impact Explanation

On every `take()` where TTM falls strictly between two breakpoints and `feeLower ≠ feeUpper`, the protocol collects up to `CBP − 1 = 999 999 999 999` wei less settlement fee than the true linear interpolation. The shortfall accumulates in `claimableSettlementFee` as a permanent deficit. For a high-volume market the aggregate loss is unbounded (loss per fill × number of fills).

### Likelihood Explanation

Preconditions are minimal: any market where the fee setter has configured at least two adjacent breakpoints with different values (the common case). The taker needs only to choose the right `block.timestamp`, which is freely controllable. The condition is met on every fill in the interior of any breakpoint interval, making it continuous and repeatable.

### Recommendation

Replace the floor division in `settlementFee()` with ceiling division (round up):

```solidity
// current (floor):
return (feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start)) / (end - start);

// fix (ceiling):
uint256 num = feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start);
uint256 denom = end - start;
return (num + denom - 1) / denom;
```

This ensures the returned fee is always ≥ the true interpolated value, preserving the protocol's fee-collection invariant while the maximum over-collection is bounded by `denom − 1` wei (sub-wei for any reasonable unit count).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
// ... standard test setup

contract SettlementFeeTruncationTest is Test {
    function testFuzz_interpolationTruncation(
        uint16 cbpLower,
        uint16 cbpUpper,
        uint256 ttm
    ) public {
        // Bound to [1d, 7d) interval
        ttm = bound(ttm, 1 days + 1, 7 days - 1);
        vm.assume(cbpUpper > cbpLower); // feeLower != feeUpper

        // Set breakpoints
        midnight.setMarketSettlementFee(id, 1, uint256(cbpLower) * CBP);
        midnight.setMarketSettlementFee(id, 2, uint256(cbpUpper) * CBP);

        uint256 reported = midnight.settlementFee(id, ttm);

        // True interpolated value (in 256-bit rational arithmetic)
        uint256 start = 1 days;
        uint256 end   = 7 days;
        uint256 feeLower = uint256(cbpLower) * CBP;
        uint256 feeUpper = uint256(cbpUpper) * CBP;
        uint256 trueNum  = feeLower * (end - ttm) + feeUpper * (ttm - start);
        uint256 trueDenom = end - start;

        // Assert: reported fee must be >= true interpolated fee (ceiling)
        // i.e. reported * trueDenom >= trueNum
        assertGe(
            reported * trueDenom,
            trueNum,
            "fee undercollection: reported < true interpolated fee"
        );
    }
}
```

The fuzz test will find counterexamples immediately (e.g. `cbpLower=1, cbpUpper=2, ttm=3 days`) where `reported * trueDenom < trueNum`, confirming systematic undercollection.

    /// @dev Returns the settlement fee using piecewise linear interpolation between breakpoints.
    function settlementFee(bytes32 id, uint256 timeToMaturity) public view returns (uint256) {
        MarketState storage _marketState = marketState[id];
        require(_marketState.tickSpacing > 0, MarketNotCreated());

        if (timeToMaturity >= 360 days) return _marketState.settlementFeeCbp6 * CBP;

        // forgefmt: disable-start
        (uint256 start, uint256 end, uint256 feeLower, uint256 feeUpper) =
            timeToMaturity < 1 days   ? (  0 days,   1 days, _marketState.settlementFeeCbp0 * CBP, _marketState.settlementFeeCbp1 * CBP) :
            timeToMaturity < 7 days   ? (  1 days,   7 days, _marketState.settlementFeeCbp1 * CBP, _marketState.settlementFeeCbp2 * CBP) :
            timeToMaturity < 30 days  ? (  7 days,  30 days, _marketState.settlementFeeCbp2 * CBP, _marketState.settlementFeeCbp3 * CBP) :
            timeToMaturity < 90 days  ? ( 30 days,  90 days, _marketState.settlementFeeCbp3 * CBP, _marketState.settlementFeeCbp4 * CBP) :
            timeToMaturity < 180 days ? ( 90 days, 180 days, _marketState.settlementFeeCbp4 * CBP, _marketState.settlementFeeCbp5 * CBP) :
                                        (180 days, 360 days, _marketState.settlementFeeCbp5 * CBP, _marketState.settlementFeeCbp6 * CBP);
        // forgefmt: disable-end

        return (feeLower * (end - timeToMaturity) + feeUpper * (timeToMaturity - start)) / (end - start);
    }
