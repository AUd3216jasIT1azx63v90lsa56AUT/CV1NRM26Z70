### Title
Missing minimum maturity check allows creation of already-expired markets - (File: src/Midnight.sol)

### Summary
`touchMarket` enforces only an upper bound on `market.maturity` (`<= block.timestamp + 100 * 365 days`) but no lower bound. Any unprivileged caller can create a market with `market.maturity < block.timestamp`, causing the market to be born in post-maturity state. Because `take()` unconditionally enforces `block.timestamp <= offer.market.maturity || sellerDebtIncrease == 0`, and in a freshly created market every take produces `sellerDebtIncrease = units > 0`, the market is immediately and permanently unusable for any new position.

### Finding Description
**Root cause — `touchMarket`, `src/Midnight.sol:755-791`:**

```solidity
function touchMarket(Market memory market) public returns (bytes32) {
    bytes32 id = toId(market);
    if (marketState[id].tickSpacing == 0) {
        require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar()); // only upper bound
        // no require(market.maturity >= block.timestamp, ...)
        ...
        _marketState.tickSpacing = DEFAULT_TICK_SPACING;
        ...
    }
    return id;
}
``` [1](#0-0) 

Setting `market.maturity = block.timestamp - 1` satisfies the only maturity guard and the market is created.

**Trigger — `take()`, `src/Midnight.sol:384,391`:**

```solidity
uint256 sellerDebtIncrease = units - sellerCreditDecrease; // = units in a fresh market (sellerPos.credit == 0)
...
require(block.timestamp <= offer.market.maturity || sellerDebtIncrease == 0, CannotIncreaseDebtPostMaturity());
``` [2](#0-1) 

In a freshly created market every position has `credit = 0`, so `sellerCreditDecrease = 0` and `sellerDebtIncrease = units`. With `block.timestamp > market.maturity` the condition is `false || (units == 0)`, which reverts for any non-zero `units`. This applies to both buy and sell offers, making the market completely inoperable for new positions.

**Exploit path:**
1. Attacker constructs a valid `Market` struct with `maturity = block.timestamp - 1` (all other params valid: sorted collaterals, allowed LLTV, correct `maxLif`).
2. Calls `midnight.touchMarket(market)` — succeeds, market is created with `tickSpacing = DEFAULT_TICK_SPACING`.
3. Any subsequent `take()` on this market with `units > 0` reverts with `CannotIncreaseDebtPostMaturity`.

**Existing protections reviewed and insufficient:**
- `MaturityTooFar` only blocks maturities more than 100 years in the future. [3](#0-2) 
- No other guard in `touchMarket` checks that maturity is in the future.
- The `CannotIncreaseDebtPostMaturity` check in `take()` is a consequence, not a protection. [4](#0-3) 

### Impact Explanation
A market is created and stored on-chain with `tickSpacing > 0` (i.e., `marketIsCreated == true`), yet every `take()` call that would open any new position immediately reverts. The market is permanently locked in post-maturity state from block 0 of its existence. Lenders who supply collateral or credit to this market (e.g., via `supplyCollateral`, which calls `touchMarket` and succeeds) cannot pair with any borrower. The state is inconsistent: the market exists and accepts some operations (collateral supply, `repay`, `withdraw` on zero balances) but its core function — matching lenders and borrowers — is permanently broken.

### Likelihood Explanation
The precondition is trivially achievable: any EOA can call `touchMarket` with `maturity = block.timestamp - 1`. No privilege, no capital, no oracle manipulation is required. The call is permissionless and costs only gas. It is repeatable for any unique set of market parameters. The only constraint is that the attacker must choose parameters no one else has already used to create a legitimate market with the same ID.

### Recommendation
Add a minimum maturity check inside the market-creation branch of `touchMarket`:

```solidity
require(market.maturity >= block.timestamp, MaturityInThePast());
```

This should be placed alongside the existing `MaturityTooFar` check at `src/Midnight.sol:758`, inside the `if (marketState[id].tickSpacing == 0)` block, so it only applies on first creation and does not affect already-created markets. [5](#0-4) 

### Proof of Concept
```solidity
function testExpiredMarketCreation() public {
    // Warp to a non-zero timestamp so maturity - 1 is valid
    vm.warp(1000);

    // Build a valid market with maturity already in the past
    Market memory expiredMarket;
    expiredMarket.loanToken = address(loanToken);
    expiredMarket.maturity = block.timestamp - 1; // expired
    expiredMarket.collateralParams = market.collateralParams; // reuse valid params from BaseTest

    // Step 1: market creation succeeds — no revert expected
    bytes32 id = midnight.touchMarket(expiredMarket);
    assertGt(midnight.tickSpacing(id), 0, "market must be created");

    // Step 2: attempt a take with sellerDebtIncrease > 0
    // Construct a minimal sell offer (maker = lender, taker = borrower)
    Offer memory offer = _buildSellOffer(expiredMarket, /* units */ 1e18);
    deal(address(loanToken), borrower, 1e18);

    vm.prank(borrower);
    vm.expectRevert(IMidnight.CannotIncreaseDebtPostMaturity.selector);
    midnight.take(offer, "", 1e18, borrower, address(0), address(0), "");

    // Assertion: market exists but borrowing is permanently impossible
}
```

Expected assertions:
- `touchMarket` does **not** revert → market is created.
- `take` with `units > 0` reverts with `CannotIncreaseDebtPostMaturity` → borrowing is impossible from block 0.
- `midnight.tickSpacing(id) > 0` → market is marked as created despite being unusable.
