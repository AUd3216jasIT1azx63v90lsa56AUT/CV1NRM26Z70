### Title
Authorized party can escalate authorization to arbitrary third addresses and erase own trace — (`src/periphery/EcrecoverAuthorizer.sol` / `src/Midnight.sol`)

### Summary
After an authorized party is granted access via a victim-signed `Authorization` message processed through `EcrecoverAuthorizer`, they satisfy Midnight's standard `isAuthorized[onBehalf][msg.sender]` guard and can call `Midnight.setIsAuthorized` directly — without any further signature — to grant authorization to an arbitrary second address and then revoke their own access. The victim's signed message authorized only the attacker, but the protocol allows that authorization to be used as a key to modify the entire authorization state, violating the invariant that authorization changes must reflect only the authorizer's intent.

### Finding Description
**Code path:**

1. Precondition: `isAuthorized[victim][EcrecoverAuthorizer] = true` (victim has authorized the periphery contract).
2. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(Authorization{authorizer=victim, authorized=attacker, isAuthorized=true, nonce=N}, sig)`.
   - Deadline and nonce checks pass. [4](#0-3) 
   - `ecrecover` recovers `victim`; the check `signer == authorization.authorizer` passes. [5](#0-4) 
   - `IMidnight(MIDNIGHT).setIsAuthorized(attacker, true, victim)` is called → `isAuthorized[victim][attacker] = true`. [1](#0-0) 
3. Attacker directly calls `Midnight.setIsAuthorized(attacker2, true, victim)`.
   - Guard: `msg.sender == victim` → false; `isAuthorized[victim][attacker]` → **true** (set in step 2). Call succeeds. `isAuthorized[victim][attacker2] = true`.
4. Attacker directly calls `Midnight.setIsAuthorized(attacker, false, victim)`.
   - Same guard passes (attacker is still authorized at call time). `isAuthorized[victim][attacker] = false`.

**Root cause:** `Midnight.setIsAuthorized` allows any currently-authorized party to modify the full authorization state of the authorizer, including granting new authorizations to arbitrary addresses. There is no restriction preventing an authorized party from delegating or expanding authorization beyond what the authorizer's signature expressed.

**Why existing checks fail:** The nonce in `EcrecoverAuthorizer` prevents replay of the original signature, but steps 3 and 4 bypass `EcrecoverAuthorizer` entirely — they call `Midnight.setIsAuthorized` directly. The only guard on that function is the `isAuthorized` mapping, which the attacker satisfies after step 2.

### Impact Explanation
`attacker2` gains permanent `isAuthorized[victim][attacker2] = true` status, enabling them to withdraw victim's credit, repay/increase victim's debt, withdraw/supply victim's collateral, and modify victim's authorization state further — all without any consent from the victim. The attacker's own address is removed, making forensic detection harder.

### Likelihood Explanation
Preconditions are realistic: a victim who uses `EcrecoverAuthorizer` to delegate actions (e.g., to a keeper or relayer) must authorize the periphery contract and sign authorization messages. Any counterparty who receives such a signed message (e.g., a relayer who is later compromised, or a malicious relayer) can execute this chain. The attack is atomic, repeatable for any victim who has authorized `EcrecoverAuthorizer`, and requires no special role.

### Recommendation
Restrict `Midnight.setIsAuthorized` so that only `msg.sender == onBehalf` (the authorizer themselves) can modify the authorization state — remove the `isAuthorized[onBehalf][msg.sender]` branch from the guard on this specific function. Authorized parties should be able to act *as* the authorizer for protocol operations (borrow, repay, withdraw, etc.) but must not be able to *expand* the authorization set. Alternatively, introduce a separate, more privileged "authorization manager" role that is distinct from general authorization.

### Proof of Concept
```solidity
// Foundry unit test
function test_authEscalation() public {
    // Setup: victim authorizes EcrecoverAuthorizer on Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);

    // Attacker has victim's signed Authorization{authorizer=victim, authorized=attacker, isAuthorized=true, nonce=0}
    Authorization memory auth = Authorization({
        authorizer: victim, authorized: attacker,
        isAuthorized: true, nonce: 0, deadline: type(uint256).max
    });
    Signature memory sig = _signAuthorization(victimKey, auth);

    // Step 1: attacker submits victim's signature
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);
    assertTrue(midnight.isAuthorized(victim, attacker));

    // Step 2: attacker grants attacker2 directly on Midnight
    vm.prank(attacker);
    midnight.setIsAuthorized(attacker2, true, victim);

    // Step 3: attacker removes themselves
    vm.prank(attacker);
    midnight.setIsAuthorized(attacker, false, victim);

    // Assertions
    assertTrue(midnight.isAuthorized(victim, attacker2));   // attacker2 has persistent access
    assertFalse(midnight.isAuthorized(victim, attacker));   // attacker covered tracks
}
```
