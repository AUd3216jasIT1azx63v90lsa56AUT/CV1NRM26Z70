### Title
Unbounded `tokens[]` loop in `flashLoan` enables block-gas-consuming DoS - (File: src/Midnight.sol)

### Summary
`flashLoan` imposes no upper bound on `tokens.length`, while two sequential loops each make a full external call per token via `SafeTransferLib`. An unprivileged caller can supply an arbitrarily large `tokens[]` array with zero `assets[]` values, causing both loops to exhaust block gas and prevent any other transaction from being included in the same block.

### Finding Description
**Code path:**

`flashLoan` at [1](#0-0)  accepts `tokens[]` and `assets[]` as calldata. The only input validation is the length-equality check at line 740. [2](#0-1) 

Loop 1 (lines 742-744) calls `SafeTransferLib.safeTransfer(tokens[i], callback, assets[i])` for every element. [3](#0-2) 

Loop 2 (lines 749-751) calls `SafeTransferLib.safeTransferFrom(tokens[i], callback, address(this), assets[i])` for every element. [4](#0-3) 

Each `safeTransfer` / `safeTransferFrom` call first checks `token.code.length > 0` and then issues a full external `.call()` to the token contract. [5](#0-4) 

**Root cause:** No `tokens.length` cap exists in `flashLoan`. Compare `touchMarket`, which enforces `require(market.collateralParams.length <= MAX_COLLATERALS)`. [6](#0-5)  The constant `MAX_COLLATERALS = 128` is defined but never applied to `flashLoan`. [7](#0-6) 

**Attacker inputs:**
- `tokens[]`: N addresses of deployed minimal ERC20 contracts (or any existing tokens that accept zero-amount transfers)
- `assets[]`: all zeros — `safeTransfer(token, callback, 0)` still executes the full external call path
- `callback`: attacker-controlled contract that returns `CALLBACK_SUCCESS`

**Exploit flow:**
1. Attacker pre-deploys N minimal ERC20 contracts (one-time cost, reusable).
2. Attacker calls `flashLoan(tokens, zeros, callback, "")` with `tokens.length = N`.
3. Loop 1 makes N external calls (cold address: ~2 600 gas each + ERC20 dispatch).
4. Callback returns `CALLBACK_SUCCESS` cheaply.
5. Loop 2 makes another N external calls.
6. At ~5 000–10 000 gas per call × 2N calls, N ≈ 1 500–3 000 fills a 30 M gas block.

**Why existing checks fail:** The only guard is `tokens.length == assets.length`, which the attacker trivially satisfies. `SafeTransferLib` requires `token.code.length > 0`, but the attacker supplies real contract addresses. No reentrancy guard, no length cap, no fee, and no access control on `flashLoan` stops this. [8](#0-7) 

### Impact Explanation
An unprivileged caller can repeatedly submit transactions that each consume the full block gas limit, effectively blocking all other protocol interactions (borrows, repayments, liquidations, withdrawals) for as long as the attacker is willing to pay their own gas. This is a sustained, permissionless DoS against the entire protocol.

### Likelihood Explanation
- **Preconditions:** Caller needs only valid ERC20 contract addresses and a callback that returns `CALLBACK_SUCCESS`. No protocol position, no tokens, no approval required.
- **Feasibility:** Deploying a batch of minimal ERC20s is a one-time setup. The attack is then repeatable every block.
- **Repeatability:** High — the attacker pays their own gas but can fill every block indefinitely.

### Recommendation
Add a maximum-length guard on `tokens` in `flashLoan`, consistent with the existing `MAX_COLLATERALS` pattern used in `touchMarket`:

```solidity
require(tokens.length <= MAX_FLASH_LOAN_TOKENS, TooManyTokens());
```

Define `MAX_FLASH_LOAN_TOKENS` in `ConstantsLib.sol` at a value that keeps worst-case gas well below the block limit (e.g., 32 or 64). [9](#0-8) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import "../src/Midnight.sol";

contract MinimalERC20 {
    function transfer(address, uint256) external pure returns (bool) { return true; }
    function transferFrom(address, address, uint256) external pure returns (bool) { return true; }
}

contract MaliciousCallback {
    bytes32 constant SUCCESS = keccak256("morpho.midnight.callbackSuccess");
    function onFlashLoan(address, address[] calldata, uint256[] calldata, bytes calldata)
        external pure returns (bytes32) { return SUCCESS; }
}

contract FlashLoanGasGriefTest is Test {
    Midnight midnight;
    MaliciousCallback cb;

    function setUp() public {
        midnight = new Midnight(...);
        cb = new MaliciousCallback();
    }

    /// @dev Fuzz tokens.length from 1 to 1000
    function testFuzz_flashLoanGasGrief(uint16 n) public {
        n = uint16(bound(n, 1, 1000));
        address[] memory tokens = new address[](n);
        uint256[] memory assets = new uint256[](n);
        for (uint256 i = 0; i < n; i++) {
            tokens[i] = address(new MinimalERC20());
            assets[i] = 0;
        }
        uint256 gasBefore = gasleft();
        midnight.flashLoan(tokens, assets, address(cb), "");
        uint256 gasUsed = gasBefore - gasleft();
        // Assert: no single call should be able to fill a 30M gas block
        assertLt(gasUsed, 30_000_000, "flashLoan consumed full block gas");
    }
}
```

**Expected assertion failure:** At `n` ≈ 1 500–3 000 the `assertLt(gasUsed, 30_000_000)` assertion trips, confirming the block-gas-consuming vector. A fixed implementation with a `tokens.length` cap would keep `gasUsed` bounded regardless of `n`.
