Audit Report

## Title
Authorized Operator Can Grief Victim's Pre-Signed EcrecoverAuthorizer Nonce via No-Op Authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` unconditionally increments `nonce[authorization.authorizer]` on every successful call, regardless of whether the downstream `Midnight.setIsAuthorized` call produces any state change. Because any address already authorized by the victim in Midnight can sign and submit an `Authorization` struct on the victim's behalf, an authorized operator can craft a no-op `Authorization` to consume the victim's current nonce, permanently invalidating any pre-signed `Authorization{nonce=N}` the victim intended to submit.

## Finding Description

**Root cause:** The nonce increment at `EcrecoverAuthorizer.sol` line 26 is unconditional:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
``` [1](#0-0) 

The signer check at lines 33–36 permits any address already authorized by the authorizer in Midnight to submit on their behalf:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [2](#0-1) 

`Midnight.setIsAuthorized` at line 733 performs an unconditional assignment with no prior-state check:

```solidity
isAuthorized[onBehalf][authorized] = newIsAuthorized;
``` [3](#0-2) 

The contract explicitly documents "No-ops are allowed" at line 168, confirming this is an intentional design choice that enables the attack surface: [4](#0-3) 

**Exploit flow:**
1. Victim `V` authorizes attacker `A` in Midnight: `isAuthorized[V][A] = true` (normal relayer/adapter setup).
2. `V` pre-signs off-chain: `Authorization{authorizer=V, authorized=criticalContract, isAuthorized=true, nonce=N}`.
3. `A` crafts: `Authorization{authorizer=V, authorized=neverAuthorizedAddr, isAuthorized=false, nonce=N, deadline=future}` and signs it with `A`'s own key.
4. `A` calls `ecrecoverAuthorizer.setIsAuthorized(noOpAuth, aSig)`.
5. All checks pass: deadline valid; `nonce[V]==N` (incremented to N+1); `ecrecover` returns `A`; `isAuthorized[V][A]==true`.
6. `Midnight.setIsAuthorized(neverAuthorizedAddr, false, V)` executes as a no-op (sets `false` for an address never authorized).
7. `nonce[V]` is now `N+1`. `V`'s pre-signed `Authorization{nonce=N}` now reverts with `InvalidNonce`.

**Why existing checks fail:** The Certora spec only verifies that a successful call increments the authorizer's nonce and that expired/wrong/reused nonces revert. It does not assert that nonce consumption requires a meaningful state change in Midnight: [5](#0-4) 

## Impact Explanation
The victim's pre-signed `Authorization{nonce=N}` is permanently invalidated. If the victim cannot immediately re-sign (hardware wallet unavailable, pre-signed batch, or a contract authorizer that pre-committed to nonce `N`), their ability to delegate via `EcrecoverAuthorizer` is frozen until a new signature is produced. The attack is repeatable: the attacker can consume every subsequent nonce the victim pre-signs for as long as the attacker remains authorized in Midnight, creating a sustained griefing/DoS on the victim's off-chain signing workflow.

## Likelihood Explanation
Required preconditions are both common in normal protocol usage:
- Victim has authorized the attacker in Midnight (`isAuthorized[victim][attacker] = true`) — a routine interaction when setting up relayers, smart contract wallets, or protocol adapters.
- Victim has a pre-signed `Authorization` in flight (in a mempool, stored in a UI, or shared with a relayer).

The attack costs one transaction, requires no special protocol-level privileges beyond being an authorized operator (a user-level delegation, not a governance/admin role), and is repeatable indefinitely until the victim revokes the attacker's Midnight authorization. [6](#0-5) 

## Recommendation
Add a meaningful-state-change check before consuming the nonce. Either:
1. **Require the authorization to produce a state change:** Before calling `Midnight.setIsAuthorized`, check that `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, authorization.authorized) != authorization.isAuthorized` and revert if it would be a no-op.
2. **Restrict signers to the authorizer only:** Remove the `isAuthorized` signer check so only the authorizer themselves can consume their own nonce, eliminating the third-party griefing vector entirely.
3. **Move nonce increment after validation:** Increment the nonce only after confirming the downstream call will produce a state change.

Option 2 is the simplest and most conservative fix, as it aligns with the typical EIP-712 permit pattern where only the key owner can consume their own nonce.

## Proof of Concept
```solidity
// 1. V authorizes A in Midnight
midnight.setIsAuthorized(attacker, true, victim); // called by victim

// 2. V pre-signs (off-chain) Authorization{authorizer=V, authorized=criticalContract, isAuthorized=true, nonce=0}
// sigV = sign(Authorization{V, criticalContract, true, 0, deadline})

// 3. A crafts and submits a no-op with nonce=0
Authorization memory noOp = Authorization({
    authorizer: victim,
    authorized: address(0xdead), // never authorized
    isAuthorized: false,          // no-op: setting false for never-authorized addr
    nonce: 0,
    deadline: block.timestamp + 1 hours
});
Signature memory sigA = sign(noOp, attackerPrivKey);
ecrecoverAuthorizer.setIsAuthorized(noOp, sigA);
// nonce[victim] is now 1

// 4. V's pre-signed tx now reverts
ecrecoverAuthorizer.setIsAuthorized(victimAuth, sigV); // reverts: InvalidNonce
``` [7](#0-6)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }

/// EcrecoverAuthorizer increments nonce on success and does not change other nonces.
rule effects(env e, EcrecoverAuthorizer.Authorization authorization, EcrecoverAuthorizer.Signature signature, address other) {
    require other != authorization.authorizer;
    uint256 nonceBefore = nonce(authorization.authorizer);
    uint256 otherNonceBefore = nonce(other);

    setIsAuthorized(e, authorization, signature);

    assert nonce(authorization.authorizer) == nonceBefore + 1;
    assert nonce(other) == otherNonceBefore;
}
