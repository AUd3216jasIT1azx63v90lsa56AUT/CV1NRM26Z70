### Title
Free Collateral Seizure via Zero-Price Oracle in `seizedAssets` Input Path - (File: src/Midnight.sol)

### Summary

When `liquidate` is called with `seizedAssets > 0` and the oracle for `collateralIndex` returns `0`, the computation `repaidUnits = seizedAssets.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif)` evaluates to `0` without reverting. The liquidator receives the full `seizedAssets` of collateral tokens while repaying zero debt and transferring zero loan tokens. This is an asymmetry the protocol itself acknowledges: the `repaidUnits > 0` path is protected by a division-by-zero revert, but the `seizedAssets > 0` path silently produces `repaidUnits = 0`.

### Finding Description

**Code path in `src/Midnight.sol`:**

The while loop at lines 607–618 iterates over the borrower's active collateral bitmap and calls each oracle:

```solidity
uint256 price = IOracle(_collateralParam.oracle).price();
if (i == collateralIndex) liquidatedCollatPrice = price;
```

If the oracle for `collateralIndex` returns `0`, `liquidatedCollatPrice` is set to `0`. The loop still completes normally; the other collateral's non-zero price contributes to `maxDebt`, so the `NotLiquidatable` check at line 620–624 can pass (borrower is unhealthy based on the other collateral).

At line 649–650 (the `seizedAssets > 0` branch):

```solidity
repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
```

With `liquidatedCollatPrice = 0` and `mulDivUp` implemented as `(x * y + (d - 1)) / d` (`src/libraries/UtilsLib.sol` line 35):
- `seizedAssets.mulDivUp(0, 1e36)` = `(seizedAssets * 0 + (1e36 - 1)) / 1e36` = `0`
- `0.mulDivUp(WAD, lif)` = `(0 + (lif - 1)) / lif` = `0` (since `lif >= WAD = 1e18`)

So `repaidUnits = 0`. The `maxRepaid` check at lines 662–667 passes trivially because `0 <= maxRepaid`. Then:

```solidity
_position.collateral[collateralIndex] -= seizedAssets;   // collateral reduced
_position.debt -= 0;                                      // debt unchanged
SafeTransferLib.safeTransfer(collateralToken, receiver, seizedAssets);  // collateral sent out
SafeTransferLib.safeTransferFrom(loanToken, payer, address(this), 0);   // 0 loan tokens paid
```

The liquidator receives `seizedAssets` collateral tokens for free.

**Contrast with the `repaidUnits > 0` path** (line 652):

```solidity
seizedAssets = repaidUnits.mulDivDown(lif, WAD).mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice);
```

This divides by `liquidatedCollatPrice`, so when price = 0 it reverts with a division-by-zero panic. The `repaidUnits` path is therefore protected; the `seizedAssets` path is not.

**The protocol explicitly models oracle-returns-zero as a valid scenario.** `certora/specs/Reverts.spec` lines 245–253 (`oracleZeroCausesLiquidateWithRepaidRevert`) proves the `repaidUnits` path reverts on zero price. `certora/README.md` line 98 states: *"NoDivisionByZero.spec checks that division by zero never occur, **except for liquidate if the liquidated collateral is not activated or has a price of 0**."* `certora/specs/NoDivisionByZero.spec` line 124 explicitly assumes `ghostPrice(...) > 0` to prove no division by zero in `liquidate`. There is no Certora rule proving that the `seizedAssets` path reverts or produces correct behavior when price = 0.

### Impact Explanation

An unprivileged liquidator can seize any amount of a borrower's collateral (up to the borrower's full balance at `collateralIndex`) while repaying zero debt. The borrower loses collateral but their debt is unchanged. Lenders lose because collateral backing is destroyed without any corresponding debt reduction, worsening the pool's solvency. The invariant *"collateral cannot be seized outside health/liquidation rules"* and *"every credit has matching debt or valid settled/loss state"* are both violated.

### Likelihood Explanation

**Preconditions:**
1. A market exists with at least two collateral params.
2. The borrower has non-zero collateral at both indices (both bits set in `collateralBitmap`).
3. The oracle for the target `collateralIndex` returns `0` (the protocol explicitly models this as a reachable state, not a failure).
4. The borrower is unhealthy based on the other collateral's price (so `NotLiquidatable` passes).

The protocol's own Certora specs treat oracle-returns-zero as a first-class scenario with dedicated rules. A price feed that returns 0 for a delisted, deprecated, or uninitialized asset is a realistic condition. The attack is repeatable as long as the oracle continues returning 0 and the borrower retains collateral at that index.

### Recommendation

Add an explicit check that `liquidatedCollatPrice > 0` before entering the seizure/repayment block when `seizedAssets > 0`, mirroring the implicit protection the `repaidUnits` path already has via division-by-zero:

```solidity
if (repaidUnits > 0 || seizedAssets > 0) {
    require(liquidatedCollatPrice > 0, ZeroPriceCollateral());
    // ... rest of the block
}
```

Alternatively, enforce the same revert behavior symmetrically by computing `repaidUnits` via a path that also divides by `liquidatedCollatPrice` when `seizedAssets > 0`.

```text
// Show that liquidate does not cause a division by zero, in case the oracle price is non-zero and the collateral is active.
rule noDivisionByZeroLiquidate(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, bool postMaturityMode) {
    require equalsGlobalMarket(market);

    // Needed for the bitmap loop which calls mulDivUp(WAD, maxLif) for every activated collateral.
    require forall uint256 i. i < market.collateralParams.length => market.collateralParams[i].maxLif >= WAD(), "see maxLifIsAtLeastWad in ExactMath.spec";

    require market.collateralParams[collateralIndex].lltv < WAD() => to_mathint(market.collateralParams[collateralIndex].maxLif) * to_mathint(market.collateralParams[collateralIndex].lltv) <= to_mathint(WAD()) * (to_mathint(WAD()) - 1), "see lifTimesLltvStrictBound in ExactMath.spec";

    // Assume that the collateral price is non-zero and the collateral is active. Otherwise, liquidate may revert with div by zero.
    require ghostPrice(market.collateralParams[collateralIndex].oracle) > 0, "Assumption: the collateral price is not zero";
    require summaryGetBit(currentContract.position[globalId][borrower].collateralBitmap, collateralIndex), "Assumption: liquidated collateral was activated";

    liquidate(e, market, collateralIndex, seizedAssets, repaidUnits, borrower, postMaturityMode, receiver, callback, data);
    assert true;
```
