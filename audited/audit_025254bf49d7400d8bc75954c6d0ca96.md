Audit Report

## Title
Authorized Midnight Delegate Can Consume Authorizer's EcrecoverAuthorizer Nonce, Permanently Invalidating Pre-Signed Authorization Messages - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` increments `nonce[authorization.authorizer]` at line 26 before the signer is recovered, and the subsequent signer check at lines 33–36 explicitly accepts any address for which `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns true. Any Midnight-level delegate of Alice can therefore construct and submit a valid `Authorization` struct with `authorizer=Alice` and Alice's current nonce, permanently consuming it and invalidating all of Alice's pre-signed messages carrying that nonce. The attack is repeatable on every subsequent nonce.

## Finding Description

**Root cause** — `src/periphery/EcrecoverAuthorizer.sol`, lines 24–48:

The nonce is incremented at line 26 via post-increment before the signer is recovered:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

The signer check at lines 33–36 explicitly permits any Midnight delegate of the authorizer:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

Because the nonce increment is a state change that is only rolled back on revert, and because the signer check passes for any valid delegate, a delegate can submit a well-formed transaction that passes every check and permanently consumes the authorizer's nonce.

**Exploit flow:**

1. Alice calls `Midnight.setIsAuthorized(EcrecoverAuthorizer, true, Alice)` — required to use the peripheral contract at all; `isAuthorized[Alice][EcrecoverAuthorizer] = true`.
2. Alice calls `Midnight.setIsAuthorized(Bob, true, Alice)` — for any legitimate purpose; `isAuthorized[Alice][Bob] = true`.
3. Alice off-chain pre-signs `Authorization(authorizer=Alice, authorized=Dave, isAuthorized=true, nonce=0, deadline=T)`.
4. Bob constructs `Authorization(authorizer=Alice, authorized=Charlie, isAuthorized=true, nonce=0, deadline=T')` and signs it with his own key.
5. Bob calls `EcrecoverAuthorizer.setIsAuthorized(auth_bob, sig_bob)`.
6. Line 26: `nonce[Alice] == 0` matches; nonce advances to 1.
7. Lines 33–36: `ecrecover` returns Bob; `isAuthorized[Alice][Bob] == true`; check passes.
8. Lines 46–47: `Midnight.setIsAuthorized(Charlie, true, Alice)` succeeds because `isAuthorized[Alice][EcrecoverAuthorizer] == true`.
9. Transaction commits. Alice's pre-signed message `(authorizer=Alice, nonce=0)` now reverts with `InvalidNonce()` on any future submission.
10. Bob immediately repeats with nonce=1, and so on.

**Why existing checks fail:**

- The nonce check at line 26 only verifies the value and increments — it does not restrict *who* may consume it.
- The signer check at lines 33–36 is evaluated after the nonce is already incremented; even if reordered, it still explicitly permits authorized delegates.
- `Midnight.sol` line 192 confirms the delegation mapping is flat: `mapping(address authorizer => mapping(address authorized => bool)) public isAuthorized`.
- `Midnight.sol` lines 731–734 confirm any authorized account can call `setIsAuthorized` on behalf of the authorizer.
- The NatSpec at lines 105–108 warns that "other contracts might re-use Midnight's authorization mapping too" and that "authorized accounts can authorize other accounts on behalf of the user," but does not document nonce-consumption in `EcrecoverAuthorizer` as an accepted risk, and users cannot reasonably anticipate this specific consequence.

## Impact Explanation
Any address Alice has authorized in Midnight can consume Alice's current `EcrecoverAuthorizer` nonce by signing and submitting an `Authorization` struct with `authorizer=Alice` and the current nonce value. This permanently invalidates all of Alice's pre-signed authorization messages carrying that nonce. Alice must re-sign with the new nonce, but the attacker can immediately repeat the attack, creating a sustained, repeatable DoS on Alice's ability to use gasless/pre-signed delegations via `EcrecoverAuthorizer`. The impact is a permanent, repeatable freeze of a core protocol feature for any user with at least one Midnight delegate.

## Likelihood Explanation
Preconditions are: (1) Alice has authorized `EcrecoverAuthorizer` in Midnight — a necessary step to use the feature; (2) Alice has authorized Bob in Midnight for any purpose — a normal, expected protocol operation. Bob must act adversarially or be compromised. No funds are at risk directly, but the DoS is persistent and requires active remediation (revoking Bob's Midnight authorization, which itself requires a live transaction). No special on-chain privileges, admin keys, or leaked credentials are required. The attack is repeatable: after each nonce consumption, Bob can immediately consume the next nonce.

## Recommendation
Validate the signer *before* consuming the nonce. Move the nonce increment to after the signer check, or use a check-effects pattern where the nonce is only written after all validation passes. Additionally, consider whether delegates should be permitted to consume nonces at all in `EcrecoverAuthorizer` — if the intent is that only the authorizer themselves can submit pre-signed messages, the signer check should be restricted to `signer == authorization.authorizer` only, removing the delegate path entirely from this peripheral contract.

```solidity
// Recommended order:
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
require(signer != address(0), InvalidSignature());
require(
    signer == authorization.authorizer, // remove delegate path, or validate before nonce bump
    Unauthorized()
);
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

## Proof of Concept

**Minimal manual steps:**

1. Deploy `Midnight` and `EcrecoverAuthorizer(address(midnight))`.
2. As Alice: `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice)`.
3. As Alice: `midnight.setIsAuthorized(bob, true, alice)`.
4. Alice off-chain signs `Authorization(authorizer=alice, authorized=dave, isAuthorized=true, nonce=0, deadline=block.timestamp+1000)` — stores for later submission.
5. Bob constructs `Authorization(authorizer=alice, authorized=charlie, isAuthorized=true, nonce=0, deadline=block.timestamp+1000)` and signs with Bob's key.
6. Bob calls `ecrecoverAuthorizer.setIsAuthorized(bobAuth, bobSig)` — succeeds; `nonce[alice]` is now 1.
7. Attempt to submit Alice's pre-signed message from step 4 — reverts with `InvalidNonce()`.
8. Repeat from step 5 with nonce=1 to demonstrate repeatability.

**Fuzz/invariant test plan:** Assert that `nonce[authorizer]` can only increase when `msg.sender == authorizer` or when the recovered signer equals `authorizer` (not a delegate).     function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
```text
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );
```
