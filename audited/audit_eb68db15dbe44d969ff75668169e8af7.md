Audit Report

## Title
Missing oracle address validation in `touchMarket` allows permanent position freeze - (File: `src/Midnight.sol`)

## Summary
`touchMarket` is a permissionless public function that validates collateral token ordering, LLTV tiers, and `maxLif` values but never checks that `collateralParams[i].oracle != address(0)`. An attacker can create a market with a zero oracle address. Any borrower who supplies collateral to that index and acquires debt will have `isHealthy`, `withdrawCollateral`, and `liquidate` permanently revert, freezing their collateral and debt with no recovery path.

## Finding Description
**Root cause:** In `src/Midnight.sol`, `touchMarket` (lines 762–773) iterates over `collateralParams` and enforces sorted/non-zero tokens via `require(collateralToken > previousCollateralToken)` (line 764), allowed LLTV tiers (line 766), and valid `maxLif` values (lines 767–771). There is no `require(market.collateralParams[i].oracle != address(0))` check anywhere in the creation path.

**Exploit flow:**
1. Attacker calls `touchMarket` with `collateralParams[0].token = <valid token>`, `collateralParams[0].oracle = address(0)`, valid LLTV and `maxLif`. The call succeeds; the market is stored.
2. Victim calls `supplyCollateral(market, 0, amount, victim)` — no oracle is consulted here; `_position.collateralBitmap` is set (lines 535–537).
3. Victim borrows via `take`, acquiring non-zero `_position.debt`.
4. Any subsequent call reaching `isHealthy` with `debt > 0` enters the bitmap loop (lines 950–956), reads `collateralParam.oracle == address(0)`, and executes `IOracle(address(0)).price()` (line 953). Since `address(0)` has no code, the low-level call returns empty bytes; Solidity's ABI decoder reverts trying to decode a `uint256` from zero bytes.
5. `withdrawCollateral` (line 568: `require(isHealthy(...))`) reverts permanently.
6. `liquidate` (lines 607–618) has its own oracle loop that also calls `IOracle(_collateralParam.oracle).price()` directly (line 610) — it reverts identically.

**Why existing checks fail:** The sorted-token check (`collateralToken > previousCollateralToken`, line 764) only prevents `token == address(0)` and duplicates; it says nothing about the `oracle` field. No other check in `touchMarket` or anywhere else in the creation path validates the oracle address.

## Impact Explanation
Any borrower who activates the poisoned collateral index and holds debt has their position permanently frozen: they cannot withdraw collateral, cannot be liquidated (their debt cannot be recovered by the protocol), and `isHealthy` always reverts for them. Their collateral is locked in the contract with no recovery path, and their debt accrues indefinitely. This is a permanent, irreversible freeze of user funds — a concrete in-scope impact per `RESEARCHER.md` ("Permanent lock, freeze, or unrecoverable corruption of user/project state").

## Likelihood Explanation
The precondition is trivially achievable: `touchMarket` is a `public` function with no access control, requires no tokens or privileges, and the only constraint on the oracle field is that it be ABI-encodable as an address. A single transaction creates the trap. Victims are any borrowers who interact with the market without inspecting the raw `collateralParams[i].oracle` field off-chain. The freeze is permanent and irreversible once a borrower has debt with the activated index.

## Recommendation
Add an explicit non-zero oracle check inside the `touchMarket` loop, immediately after the token check:

```solidity
require(market.collateralParams[i].oracle != address(0), InvalidOracle());
```

Optionally, also verify that the oracle address contains code (`market.collateralParams[i].oracle.code.length > 0`) to guard against non-contract addresses more broadly.

## Proof of Concept
```solidity
// 1. Attacker creates a market with oracle = address(0)
CollateralParams[] memory params = new CollateralParams[](1);
params[0] = CollateralParams({
    token:   address(validToken),
    oracle:  address(0),          // zero oracle — passes all touchMarket checks
    lltv:    allowedLltv,
    maxLif:  midnight.maxLif(allowedLltv, LIQUIDATION_CURSOR_LOW)
});
Market memory market = Market({ ..., collateralParams: params });
midnight.touchMarket(market);

// 2. Victim supplies collateral and borrows
validToken.approve(address(midnight), 1e18);
midnight.supplyCollateral(market, 0, 1e18, victim);
midnight.take(...);  // victim acquires debt

// 3. Any health-check call now reverts permanently
midnight.isHealthy(market, id, victim);       // reverts: IOracle(address(0)).price()
midnight.withdrawCollateral(market, 0, ...);  // reverts at line 568
midnight.liquidate(market, 0, ...);           // reverts at line 610
// Victim's collateral and debt are permanently frozen.
```
