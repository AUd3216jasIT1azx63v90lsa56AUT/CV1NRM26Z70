### Title
Fee-on-transfer loan token causes `repayAndWithdrawCollateral` to always revert due to bundler balance shortfall - (File: src/periphery/MidnightBundles.sol)

### Summary
`repayAndWithdrawCollateral` computes `units = assets - referralFeeAssets` and then calls `pullToken(loanToken, msg.sender, assets, ...)`, expecting the bundler to hold exactly `assets` tokens. For a fee-on-transfer loan token the bundler only receives `assets - transferFee`. When `Midnight.repay` subsequently calls `SafeTransferLib.safeTransferFrom(loanToken, bundler, midnight, units)`, the bundler's balance is insufficient whenever `transferFee > referralFeeAssets`, causing an unconditional revert and permanently blocking the bundler's combined repay-and-withdraw path.

### Finding Description
**Exact code path:**

In `repayAndWithdrawCollateral` (lines 329–334 of `src/periphery/MidnightBundles.sol`):

```solidity
uint256 referralFeeAssets = assets.mulDivDown(referralFeePct, WAD);  // line 329
uint256 units = assets - referralFeeAssets;                           // line 330
pullToken(loanToken, msg.sender, assets, loanTokenPermit);            // line 331
forceApproveMax(loanToken, MIDNIGHT);                                 // line 332
IMidnight(MIDNIGHT).repay(market, units, onBehalf, address(0), "");  // line 334
```

`pullToken` calls `SafeTransferLib.safeTransferFrom(token, from, address(this), amount)` unconditionally for the `PermitKind.None` path. [1](#0-0) 

For a fee-on-transfer token with per-transfer fee `f`, the bundler receives `assets - f`, not `assets`.

Inside `Midnight.repay` (line 520):

```solidity
SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);
```

where `payer = msg.sender = bundler` (because `callback == address(0)`). [2](#0-1) 

**Root cause:** The bundler assumes `pullToken` delivers exactly `assets` tokens. No post-pull balance check is performed. The repay amount `units = assets - referralFeeAssets` is computed from the nominal `assets`, not from the actual received balance.

**Failure condition:** `units > bundler_balance_after_pull`
→ `assets - referralFeeAssets > assets - f`
→ `f > referralFeeAssets`
→ `f > assets * referralFeePct / WAD`

**Simplest trigger:** `referralFeePct = 0` → `referralFeeAssets = 0` → `units = assets`, but bundler holds `assets - f`. Any nonzero transfer fee causes revert.

**Attacker-controlled inputs:** The borrower (unprivileged) simply calls `repayAndWithdrawCollateral` on a market whose `loanToken` is a fee-on-transfer token with `referralFeePct = 0` (or any value where `f > assets * referralFeePct / WAD`).

**Existing checks:** `require(referralFeePct < WAD)` (line 326) and `require(onBehalf == msg.sender || isAuthorized(...))` (line 325) do not address the balance shortfall. [3](#0-2) 

### Impact Explanation
The bundler's `repayAndWithdrawCollateral` function is permanently non-functional for any market whose loan token charges a transfer fee. Every call reverts at the `Midnight.repay` → `safeTransferFrom` step. The borrower cannot use the bundler to atomically repay debt and withdraw collateral; they must fall back to separate direct calls to `Midnight.repay` and `Midnight.withdrawCollateral`, which is a broken user-facing path for this token class.

### Likelihood Explanation
Fee-on-transfer ERC-20 tokens (e.g., tokens with a built-in tax) are a well-known token class. Any market created with such a token as `loanToken` will make this function permanently unusable. The condition is deterministic and repeatable: every call with `referralFeePct = 0` (the default/common case) will revert. No special attacker setup is required beyond having a position in such a market.

### Recommendation
After `pullToken`, measure the actual received amount and use it to recompute `units`:

```solidity
uint256 balanceBefore = IERC20(loanToken).balanceOf(address(this));
pullToken(loanToken, msg.sender, assets, loanTokenPermit);
uint256 received = IERC20(loanToken).balanceOf(address(this)) - balanceBefore;

uint256 referralFeeAssets = received.mulDivDown(referralFeePct, WAD);
uint256 units = received - referralFeeAssets;
```

This ensures `units` never exceeds the bundler's actual balance, making the function safe for fee-on-transfer tokens.

// Foundry unit test
function testRepayAndWithdrawCollateral_FeeOnTransfer_Reverts() public {
    // 1. Deploy a fee-on-transfer loan token (1% fee per transfer).
    FeeOnTransferToken feeToken = new FeeOnTransferToken(100); // 1% fee

    // 2. Create a Midnight market with feeToken as loanToken.
    //    Set up borrower with debt = 1000e18 units.

    // 3. Give borrower 1000e18 feeToken, approve bundler.
    uint256 assets = 1000e18;
    deal(address(feeToken), borrower, assets);
    vm.prank(borrower);
    feeToken.approve(address(midnightBundles), assets);

    // 4. Call repayAndWithdrawCollateral with referralFeePct = 0.
    //    pullToken pulls 1000e18 but bundler receives 990e18 (1% fee).
    //    units = 1000e18 - 0 = 1000e18.
    //    repay tries safeTransferFrom(bundler, midnight, 1000e18) but bundler has 990e18.
    vm.prank(borrower);
    vm.expectRevert(); // SafeTransferLib revert: insufficient balance
    midnightBundles.repayAndWithdrawCollateral(
        market, assets, borrower, _noPermit(),
        new CollateralWithdrawal[](0), address(0), 0, address(0)
    );

    // 5. Assert debt is unchanged (repay never happened).
    assertEq(midnight.debtOf(id, borrower), 1000e18, "debt unchanged");
}

Expected assertion: the call reverts and `midnight.debtOf(id, borrower)` remains at the original debt, confirming the repay path is blocked. [4](#0-3) [5](#0-4)

    function repay(Market memory market, uint256 units, address onBehalf, address callback, bytes calldata data)
        external
    {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);

        position[id][onBehalf].debt -= UtilsLib.toUint128(units);
        marketState[id].withdrawable += UtilsLib.toUint128(units);

        address payer = callback != address(0) ? callback : msg.sender;
        emit EventsLib.Repay(msg.sender, id, units, onBehalf, payer);

        if (callback != address(0)) {
            require(
                IRepayCallback(callback).onRepay(id, market, units, onBehalf, data) == CALLBACK_SUCCESS,
                WrongRepayCallbackReturnValue()
            );
        }
        SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);
    }
