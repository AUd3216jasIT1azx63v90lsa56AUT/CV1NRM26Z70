### Title
Authorized operator can consume victim's `EcrecoverAuthorizer` nonces and revoke delegations via operator-signed `Authorization` structs - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that Midnight considers authorized for the `authorization.authorizer`, not only the authorizer themselves. Because the nonce is keyed by `authorization.authorizer` (the victim), an operator who holds `isAuthorized[victim][operator] == true` can sign and submit `Authorization` structs naming `authorizer = victim`, consuming victim's sequential nonces and revoking victim's existing delegations. This violates the invariant that only the authorizer's own signed intent should advance their `EcrecoverAuthorizer` nonce.

### Finding Description
**Code path:**

`EcrecoverAuthorizer.setIsAuthorized` (lines 24–48):

```
Line 26: nonce[authorization.authorizer]++   // victim's nonce consumed
Line 34: signer == authorization.authorizer
         || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)
``` [1](#0-0) 

The nonce is incremented for `authorization.authorizer` (victim) before the signer check. The signer check at line 33–36 passes if `isAuthorized[victim][operator]` is true on Midnight — it does not require the signer to equal `authorization.authorizer`. [2](#0-1) 

The downstream call at line 46–47 then calls `Midnight.setIsAuthorized(X, false, victim)` with `msg.sender = EcrecoverAuthorizer`. That call passes because `isAuthorized[victim][EcrecoverAuthorizer]` is true (second precondition). [3](#0-2) [4](#0-3) 

**Attacker-controlled inputs:**
- `authorization.authorizer` = victim address
- `authorization.authorized` = any address victim has authorized (e.g., another operator, a ratifier)
- `authorization.isAuthorized` = false
- `authorization.nonce` = current `nonce[victim]`
- `signature` = operator's own ECDSA signature over the struct

**Exploit flow:**
1. Victim has authorized `EcrecoverAuthorizer` on Midnight and has authorized `operator` on Midnight.
2. Victim pre-signs `N` Authorization structs with nonces `0..N-1` (e.g., to grant future delegations off-chain).
3. Operator constructs `N` Authorization structs with `authorizer=victim`, `authorized=<victim's other delegates>`, `isAuthorized=false`, nonces `0..N-1`, signed by operator's own key.
4. Operator submits each via `EcrecoverAuthorizer.setIsAuthorized`.
5. Each call: nonce check passes (sequential), signer check passes (`isAuthorized[victim][operator]`), `nonce[victim]` advances, and `isAuthorized[victim][X]` is set to false.
6. Victim's pre-signed Authorization structs now all fail `InvalidNonce`; victim's existing delegations are revoked.

**Why existing checks fail:**
The `Unauthorized` check at line 33–36 is designed to mirror Midnight's own authorization model, but this conflates two distinct roles: (a) who may act on behalf of the authorizer for protocol operations, and (b) whose signed intent should advance the authorizer's EIP-712 nonce. The nonce is incremented unconditionally at line 26 before the signer identity is resolved, and the signer check does not distinguish between "authorizer signed this" and "an operator signed this on authorizer's behalf." [5](#0-4) 

### Impact Explanation
An authorized operator can: (1) consume all of victim's `EcrecoverAuthorizer` nonces, permanently invalidating any pre-signed Authorization structs the victim distributed off-chain; (2) revoke victim's authorizations of other operators and ratifiers (e.g., `EcrecoverRatifier`, `MidnightBundles`), temporarily blocking victim's ability to use signature-gated protocol flows until victim re-authorizes on-chain. This is a concrete griefing path with no financial loss but real access disruption.

### Likelihood Explanation
**Preconditions:**
- Victim must have authorized `EcrecoverAuthorizer` on Midnight (common for any user of the signature-based flow).
- Victim must have authorized the attacker-operator on Midnight (e.g., a delegated keeper, a bundle executor, or any address the victim trusted).

Both preconditions are normal operational states for users of the periphery. The attack is permissionless once preconditions hold, costs only gas, and is repeatable (operator can keep consuming nonces as victim re-signs).

### Recommendation
Restrict nonce advancement to the authorizer's own signature only. Remove the `isAuthorized` branch from the signer check, or split the check so that the nonce is only incremented when `signer == authorization.authorizer`:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegated signing is intentionally supported, use a separate nonce namespace keyed by `(authorizer, signer)` rather than `authorizer` alone, so an operator's submissions do not consume the authorizer's nonce sequence.

### Proof of Concept
```solidity
function testOperatorConsumesVictimNonce() public {
    // Setup: victim authorizes ecrecoverAuthorizer and operator on Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    vm.prank(victim);
    midnight.setIsAuthorized(operator, true, victim);

    // Victim pre-signs an Authorization at nonce 0
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: someDelegate,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victim);

    // Operator signs a revocation Authorization at nonce 0 (operator's own key, victim as authorizer)
    Authorization memory attackAuth = Authorization({
        authorizer: victim,
        authorized: someDelegate,
        isAuthorized: false,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory attackSig = signAuthorization(attackAuth, operator);

    // Operator submits — passes both nonce and signer checks
    vm.prank(operator);
    ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);

    // Victim's nonce is now 1; victim's pre-signed auth at nonce 0 is invalid
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

    // Victim's delegation was also revoked
    assertEq(midnight.isAuthorized(victim, someDelegate), false);
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.nonce(victim) == 1` after operator's submission
- Victim's pre-signed `victimAuth` reverts with `InvalidNonce`
- `midnight.isAuthorized(victim, someDelegate) == false`

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );
