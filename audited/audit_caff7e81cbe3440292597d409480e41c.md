### Title
Fee-on-Transfer Tokens Inflate Collateral Credit and `withdrawable`, Breaking Solvency Invariants — (File: `src/Midnight.sol`)

---

### Summary

`supplyCollateral()` and `repay()` both credit accounting state with the nominal `assets`/`units` argument before the actual token transfer occurs, with no balance-delta check. If a fee-on-transfer token is used as a collateral or loan token, the contract receives fewer tokens than it records, permanently inflating the borrower's collateral credit or the market's `withdrawable` pool. Because markets are created permissionlessly with no on-chain token validation, any actor can open a market with such a token and exploit the discrepancy.

---

### Finding Description

**Root cause — `supplyCollateral()`** [1](#0-0) 

```solidity
uint256 oldCollateral = _position.collateral[collateralIndex];
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);  // credited in full
``` [2](#0-1) 

```solidity
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
```

`SafeTransferLib` does not measure the balance delta; it only checks the boolean return value. [3](#0-2) 

If `collateralToken` deducts a fee, the contract receives `assets × (1 − fee)` but records `assets` in `_position.collateral`. The borrower's effective LTV is therefore understated, allowing them to take on more debt than the actual collateral can cover.

**Root cause — `repay()`** [4](#0-3) 

```solidity
position[id][onBehalf].debt -= UtilsLib.toUint128(units);
marketState[id].withdrawable += UtilsLib.toUint128(units);   // credited in full
``` [5](#0-4) 

```solidity
SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), units);
```

If the loan token has a transfer fee, `withdrawable` is inflated by `units × fee` relative to actual tokens held. Lenders calling `withdraw()` can drain the shortfall from other lenders' funds.

**Why the documented assumption does not close the issue**

The `TOKEN SAFETY REQUIREMENTS` comment states "Midnight must send/receive exactly the requested amount on transfers." [6](#0-5) 

However, `touchMarket()` performs no on-chain token validation whatsoever — any address can be supplied as `loanToken` or `collateralParams[i].token`. [7](#0-6) 

The assumption is therefore unenforced, and the attack surface is fully permissionless.

---

### Impact Explanation

- **`supplyCollateral()` path**: Borrower's on-chain collateral balance exceeds actual tokens held. The borrower can borrow against phantom collateral. When liquidated, the seized collateral is less than what the accounting expects, producing bad debt that is socialized across all lenders in the market via `lossFactor`.
- **`repay()` path**: `withdrawable` exceeds actual loan tokens held. Lenders who withdraw first receive full value; later lenders find the pool short, suffering a direct loss proportional to `units × fee_rate` per repayment.

Both paths break the core solvency invariant: the contract's token balance must be sufficient to honour all recorded obligations.

---

### Likelihood Explanation

- Market creation is fully permissionless; no governance approval or token whitelist is enforced on-chain.
- Fee-on-transfer tokens are a well-known token class (USDT fee mode, STA, PAXG, etc.).
- An attacker needs only to deploy or point to such a token, call `touchMarket()`, and interact normally — no privileged keys required.
- The `MidnightBundles` periphery also calls `supplyCollateral()` with the raw `assets` value, compounding the surface. [8](#0-7) 

---

### Recommendation

Replace the fixed-amount credit with a balance-delta measurement in both `supplyCollateral()` and `repay()`:

```solidity
uint256 before = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - before;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

Apply the same pattern to `repay()` so that `withdrawable` is incremented only by the tokens actually received. Alternatively, enforce the token safety requirement on-chain by maintaining a registry of approved tokens checked during `touchMarket()`.

---

### Proof of Concept

1. Deploy `FeeToken` — an ERC-20 that deducts 1 % on every `transferFrom`.
2. Call `touchMarket()` with `collateralParams[0].token = address(FeeToken)`.
3. Attacker calls `supplyCollateral(market, 0, 10_000e18, attacker)`.
   - Contract records `_position.collateral[0] = 10_000e18`.
   - Contract actually receives `9_900e18` (`10_000 × 0.99`).
4. Attacker calls `take()` to borrow the maximum units permitted by the inflated collateral.
5. Attacker defaults; liquidator seizes collateral but the contract only holds `9_900e18`, not `10_000e18`.
6. The `100e18` shortfall is realized as bad debt and socialized via `lossFactor`, slashing all lenders in the market.

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;

            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];

            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);
        }
        return id;
    }

    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
        if (!success) {
            assembly ("memory-safe") {
                revert(add(returndata, 0x20), mload(returndata))
            }
        }
        require(returndata.length == 0 || abi.decode(returndata, (bool)), TransferFromReturnedFalse());
    }
