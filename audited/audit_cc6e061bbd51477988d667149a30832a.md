### Title
Division by Zero in `sellerAssetsToUnits` via `tickToPrice(0) = 0` Causes DoS of Sell-Side Bundle Flows - (File: src/periphery/ConsumableUnitsLib.sol)

### Summary

`tickToPrice(0)` provably returns `0` due to rounding by `PRICE_ROUNDING_STEP`, as confirmed by both a passing unit test and a Certora formal rule. When `consumableUnits` is called with a sell offer (`offer.buy = false`) that has `offer.maxUnits = 0` and `offer.maxAssets > 0` at `offer.tick = 0`, it delegates to `sellerAssetsToUnits`, which computes `sellerPrice = tickToPrice(0) = 0` and then calls `mulDivDown(targetSellerAssets, WAD, 0)`, triggering an unconditional division-by-zero revert. No existing guard in `consumableUnits` or `sellerAssetsToUnits` prevents this path.

### Finding Description

**Confirmed zero-price fact:**

`test/TickLibTest.sol` line 16 asserts `tickToPrice(0) == 0` and passes: [1](#0-0) 

The Certora spec independently proves the same invariant: [2](#0-1) 

The root cause is the double rounding in `tickToPrice`: the raw `1e36 / (1e18 + wExp(...))` result for tick=0 is smaller than `PRICE_ROUNDING_STEP = 1e12`, so `divHalfDownUnchecked(..., PRICE_ROUNDING_STEP) * PRICE_ROUNDING_STEP` collapses to `0`: [3](#0-2) 

**Reachable division-by-zero path:**

`consumableUnits` reaches `sellerAssetsToUnits` when `offer.maxUnits == 0`, `offer.buy == false`, and `offer.maxAssets > 0`: [4](#0-3) 

Inside `sellerAssetsToUnits`, for a sell offer (`offer.buy = false`), `sellerPrice` is set directly to `offerPrice = tickToPrice(0) = 0`, then `mulDivDown(targetSellerAssets, WAD, sellerPrice)` is called with denominator `0`: [5](#0-4) 

`mulDivDown` is implemented as plain `(x * y) / d`: [6](#0-5) 

Division by zero in Solidity 0.8.x is an unconditional revert, confirmed by the existing test: [7](#0-6) 

**Attacker-controlled inputs:**

An unprivileged maker crafts an `Offer` struct with:
- `offer.tick = 0` (valid: `0 <= MAX_TICK`, and `0 % spacing == 0` for any spacing, so always accessible)
- `offer.buy = false`
- `offer.maxUnits = 0`
- `offer.maxAssets > 0` (e.g., `1`)

This offer is passed to any bundle function that calls `consumableUnits` on sell offers. The `MidnightBundles` NatSpec explicitly states it "Reverts if TakeAmountsLib or ConsumableUnitsLib reverts": [8](#0-7) 

**Why existing checks fail:**

- `tickToPrice` only guards `tick > MAX_TICK`; tick=0 is in range.
- `consumableUnits` has no guard for `sellerPrice == 0`.
- `sellerAssetsToUnits` has no guard for `sellerPrice == 0`.
- The `NoDivisionByZero.spec` Certora spec covers only the core `Midnight.sol` contract and uses `NONDET` for `tickToPrice`, so it does not catch this periphery path: [9](#0-8) 

### Impact Explanation

Any bundle transaction that calls `consumableUnits` on a sell offer with `tick=0`, `maxUnits=0`, `maxAssets>0` reverts unconditionally. Because `MidnightBundles` does not wrap `consumableUnits` in a `try/catch`, the entire bundle call fails. This is a concrete DoS of sell-side bundle flows (e.g., `buyWithAssetsTargetAndWithdrawCollateral` and any other bundle path that processes sell offers through `consumableUnits`).

### Likelihood Explanation

The preconditions are trivially satisfiable by any unprivileged maker: tick=0 is always a valid, accessible tick (divisible by any spacing), and setting `maxUnits=0` with `maxAssets>0` is a normal asset-capped offer configuration. The attack is zero-cost (no funds required, no state setup), repeatable on every block, and requires no special role or privilege.

### Recommendation

Add a zero-price guard in `sellerAssetsToUnits` (and symmetrically in `buyerAssetsToUnits` for the sell-offer branch) before performing the division:

```solidity
require(sellerPrice > 0, ZeroPrice());
```

Alternatively, guard at the `consumableUnits` level before delegating to `sellerAssetsToUnits`:

```solidity
uint256 offerPrice = TickLib.tickToPrice(offer.tick);
require(offerPrice > 0, ZeroPrice());
```

A complementary fix is to enforce `tickToPrice(tick) > 0` for all ticks accepted by the protocol (e.g., require `tick >= 2`, since `tickToPrice(2) = 1e12` is the first non-zero price per the existing test at line 17 of `test/TickLibTest.sol`). [10](#0-9) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {TickLib} from "src/libraries/TickLib.sol";
import {TakeAmountsLib} from "src/periphery/TakeAmountsLib.sol";
import {ConsumableUnitsLib} from "src/periphery/ConsumableUnitsLib.sol";
import {Offer} from "src/interfaces/IMidnight.sol";

contract PoCDivByZeroTick0 is Test {
    // Step 1: confirm tickToPrice(0) == 0
    function testTickToPriceZeroAtTickZero() public pure {
        assertEq(TickLib.tickToPrice(0), 0);
    }

    // Step 2: confirm sellerAssetsToUnits reverts with division error
    // when offer.buy=false, offer.tick=0, targetSellerAssets>0
    function testSellerAssetsToUnitsRevertsDivByZero(
        address midnight,
        bytes32 id,
        Offer memory offer
    ) public {
        offer.buy = false;
        offer.tick = 0;
        // targetSellerAssets = 1 > 0
        vm.expectRevert(); // stdError.divisionError
        TakeAmountsLib.sellerAssetsToUnits(midnight, id, offer, 1);
    }

    // Step 3: confirm consumableUnits reverts via the same path
    function testConsumableUnitsRevertsDivByZero(
        address midnight,
        bytes32 id,
        Offer memory offer
    ) public {
        offer.buy = false;
        offer.tick = 0;
        offer.maxUnits = 0;
        offer.maxAssets = 1; // > 0 so targetSellerAssets > 0
        // mock consumed() to return 0 so zeroFloorSub(1, 0) = 1
        vm.mockCall(midnight, abi.encodeWithSignature("consumed(address,bytes32)", offer.maker, offer.group), abi.encode(0));
        vm.expectRevert(); // division by zero
        ConsumableUnitsLib.consumableUnits(midnight, id, offer);
    }
}
```

**Expected assertions:**
- `testTickToPriceZeroAtTickZero`: passes (already confirmed by existing test suite).
- `testSellerAssetsToUnitsRevertsDivByZero`: reverts with `Panic(0x12)` (division by zero).
- `testConsumableUnitsRevertsDivByZero`: reverts with `Panic(0x12)`, demonstrating the full DoS path through `consumableUnits`.
