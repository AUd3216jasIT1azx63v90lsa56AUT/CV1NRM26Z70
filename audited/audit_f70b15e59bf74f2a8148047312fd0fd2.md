### Title
Missing `loanToken` Validation in `touchMarket` Allows Permanently Unusable Market Creation - (`src/Midnight.sol`)

### Summary

`touchMarket` validates collateral tokens, LLTV, maxLif, and maturity, but performs no check on `market.loanToken`. An unprivileged caller can create a market with `loanToken = address(0)`, which succeeds and sets `tickSpacing > 0`. Any subsequent `take` unconditionally calls `SafeTransferLib.safeTransferFrom(address(0), ...)`, which immediately reverts with `NoCode()` because `address(0).code.length == 0`, making the market permanently unusable.

### Finding Description

**Root cause — missing check in `touchMarket`:**

`touchMarket` in `src/Midnight.sol` (lines 755–791) validates:
- `market.maturity` (line 758)
- `market.collateralParams.length` (lines 759–760)
- each `collateralToken > previousCollateralToken` (line 764)
- `isLltvAllowed(lltv)` (line 766)
- `maxLif` (lines 767–771) [1](#0-0) 

There is **no check on `market.loanToken`**. Passing `loanToken = address(0)` satisfies every existing guard (collateral tokens only need to be `> address(0)`, which any valid collateral token satisfies).

**State change:** `_marketState.tickSpacing = DEFAULT_TICK_SPACING` is written, permanently marking the market as created. [2](#0-1) 

**Trigger — `take` unconditionally calls `safeTransferFrom` with `loanToken`:**

After all position accounting, `take` calls: [3](#0-2) 

`SafeTransferLib.safeTransferFrom` begins with an unconditional code-length guard: [4](#0-3) 

`address(0).code.length == 0`, so this reverts with `NoCode()` on every `take` call, regardless of `units` (even `units = 0` triggers the check because the calls are not guarded by `if (value > 0)`).

**Exploit flow:**
1. Attacker constructs a `Market` with `loanToken = address(0)`, valid collateral params, valid LLTV/maxLif, and valid maturity.
2. Attacker calls `touchMarket(market)` — succeeds, `tickSpacing > 0`, market ID slot consumed.
3. Any call to `take` with this market's offer reverts at `safeTransferFrom(address(0), ...)` with `NoCode()`.

### Impact Explanation

The market is created (state written, `tickSpacing > 0`) but every `take` reverts permanently. The market ID slot for this `(address(0), collaterals, maturity)` tuple is consumed and can never be re-initialized (the `if (marketState[id].tickSpacing == 0)` guard prevents re-creation). The market is permanently bricked. [5](#0-4) 

### Likelihood Explanation

No preconditions. `touchMarket` is `public` with no access control. Any address can call it with `loanToken = address(0)` and valid collateral/LLTV/maxLif parameters. The attack is free (only gas cost), deterministic, and repeatable for any distinct collateral/maturity combination.

### Recommendation

Add a `require(market.loanToken != address(0), InvalidLoanToken())` check (or equivalently `require(market.loanToken.code.length > 0)`) inside the `if (marketState[id].tickSpacing == 0)` block in `touchMarket`, before writing state. [6](#0-5)

function testLoanTokenZeroMarketBricked() public {
    // Build a market with loanToken = address(0) but otherwise valid params
    Market memory badMarket;
    badMarket.loanToken = address(0);
    badMarket.maturity = block.timestamp + 100;
    badMarket.collateralParams = new CollateralParams[](1);
    badMarket.collateralParams[0] = CollateralParams({
        token: address(collateralToken1),
        lltv: 0.77e18,
        maxLif: maxLif(0.77e18, 0.25e18),
        oracle: address(oracle1)
    });

    // Step 1: touchMarket succeeds — market is created
    bytes32 id = midnight.touchMarket(badMarket);
    assertGt(midnight.tickSpacing(id), 0, "market created with loanToken=0");

    // Step 2: construct a minimal offer for this market
    Offer memory offer;
    offer.market = badMarket;
    offer.buy = true;
    offer.maker = lender;
    offer.ratifier = address(dummyRatifier);
    offer.maxUnits = type(uint256).max;
    offer.expiry = block.timestamp + 200;
    offer.tick = 0; // tick divisible by tickSpacing

    vm.prank(lender);
    midnight.setIsAuthorized(address(dummyRatifier), true, lender);

    // Step 3: take reverts with NoCode — market permanently unusable
    vm.expectRevert(SafeTransferLib.NoCode.selector);
    vm.prank(borrower);
    midnight.take(offer, "", 0, borrower, borrower, address(0), "");
}
