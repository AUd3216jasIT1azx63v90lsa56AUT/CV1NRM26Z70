### Title
Authorized delegate can consume victim's EcrecoverAuthorizer nonce via direct call, permanently invalidating pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any party that Midnight's `isAuthorized` mapping recognizes as a delegate of the `authorization.authorizer`. Because the nonce is incremented unconditionally on success, any attacker holding `isAuthorized[victim][attacker] == true` can sign and submit an Authorization naming the victim as authorizer, consuming the victim's current nonce. No `take` callback is required; the attacker calls `EcrecoverAuthorizer.setIsAuthorized` directly. Any pre-signed authorization the victim holds for that nonce is permanently invalidated.

### Finding Description
The root cause is in `EcrecoverAuthorizer.setIsAuthorized`:

```
Line 26: require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
...
Line 33-36:
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) [2](#0-1) 

The nonce for `authorization.authorizer` is incremented as part of the `require` on line 26 — before the signature and authorization checks. If the full transaction succeeds, the nonce is permanently consumed. The authorization check on lines 33–36 explicitly permits any signer for whom `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns `true`.

**Exact exploit path (no take callback needed):**

1. **Precondition**: `isAuthorized[victim][attacker] == true` in Midnight (victim previously called `Midnight.setIsAuthorized(attacker, true, victim)`). Victim has a pre-signed `Authorization{authorizer=victim, nonce=N, ...}` they intend to submit later.
2. **Attacker constructs**: `Authorization{authorizer=victim, authorized=anyAddress, isAuthorized=true/false, nonce=N, deadline=block.timestamp+1}`.
3. **Attacker signs** this struct with their own private key → `ecrecover` returns `attacker`.
4. **Attacker calls** `EcrecoverAuthorizer.setIsAuthorized(auth, attackerSig)` directly.
5. Line 26 passes: `N == nonce[victim]`, nonce becomes `N+1`.
6. Lines 33–36 pass: `signer == attacker` and `isAuthorized[victim][attacker] == true`.
7. `nonce[victim]` is now `N+1`.
8. Victim's pre-signed authorization with nonce `N` now reverts with `InvalidNonce()`.

The `take` callback path described in the question (`take → onBuy → setIsAuthorized`) is also reachable — the `onBuy` callback at lines 445–452 of `Midnight.sol` has no reentrancy guard preventing calls to `EcrecoverAuthorizer` — but it is unnecessary overhead. The direct call is sufficient. [3](#0-2) 

**Why existing checks fail**: The `isAuthorized` delegation branch was added to allow authorized agents to submit authorizations on behalf of the authorizer. However, it conflates Midnight-level delegation (intended for protocol actions) with EcrecoverAuthorizer nonce authority (intended to prove the authorizer's own intent via their private key). There is no check preventing an authorized delegate from consuming nonces the authorizer never intended them to touch.

### Impact Explanation
The victim's pre-signed `Authorization` with nonce `N` becomes permanently unsubmittable — any call to `setIsAuthorized` with that message reverts with `InvalidNonce()`. If the victim distributed the signed message off-chain (e.g., for a gasless relay, a multisig flow, or a time-delayed authorization), that message is irrecoverably burned. The attacker can repeat this for every subsequent nonce as long as they remain authorized, permanently blocking the victim from using EcrecoverAuthorizer's signature-based flow.

### Likelihood Explanation
The precondition — `isAuthorized[victim][attacker] == true` — requires the victim to have previously authorized the attacker in Midnight. This is a realistic scenario: users authorize bots, relayers, or counterparties for protocol operations. The victim would not expect that granting Midnight-level authorization also grants the delegate the ability to consume EcrecoverAuthorizer nonces. The attack is cheap (one transaction), repeatable, and requires no special on-chain state beyond the existing authorization.

### Recommendation
Remove the `isAuthorized` delegation branch from `EcrecoverAuthorizer.setIsAuthorized`. Only the authorizer's own signature should be accepted:

```solidity
require(signer == authorization.authorizer, Unauthorized());
``` [2](#0-1) 

The EIP-712 signature already cryptographically proves the authorizer's intent. Allowing Midnight delegates to sign on the authorizer's behalf is unnecessary for the contract's purpose and creates this griefing vector. If delegation is desired, it should use a separate, explicit mechanism scoped to EcrecoverAuthorizer rather than inheriting Midnight's broad `isAuthorized` mapping.

### Proof of Concept
```solidity
// Foundry unit test
function testNonceGriefByDelegate() public {
    // Setup: victim authorizes attacker in Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim pre-signs Authorization with nonce 0
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: someAddress,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victim);

    // Attacker crafts a different Authorization with same nonce, signs with own key
    Authorization memory attackerAuth = Authorization({
        authorizer: victim,
        authorized: address(0xdead),
        isAuthorized: false,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory attackerSig = signAuthorization(attackerAuth, attacker);

    // Attacker submits directly — no take callback needed
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);

    // Assert: nonce consumed
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);

    // Assert: victim's pre-signed message is now invalid
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}
```

Expected assertions: `nonce[victim] == 1` after the attacker's call; victim's pre-signed authorization reverts with `InvalidNonce`.
