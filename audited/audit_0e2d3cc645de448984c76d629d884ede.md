Audit Report

## Title
Co-authorized agent can deauthorize peer agents via `EcrecoverAuthorizer.setIsAuthorized` - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a valid signature from any address that `authorization.authorizer` has already authorized in Midnight, not exclusively from the authorizer themselves. A co-authorized attacker can sign an `Authorization` struct targeting a peer agent (e.g., a `setConsumedAgent`) with `isAuthorized=false`, stripping that agent's ability to cancel the victim's offers and leaving them permanently active and fillable.

## Finding Description
**Root cause — `EcrecoverAuthorizer.sol` lines 33–36:**
```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```
The second branch grants any address already authorized by `authorization.authorizer` in Midnight the ability to sign *any* `Authorization` struct on the authorizer's behalf — including ones that revoke other agents.

**Exploit path:**

Preconditions (all set by victim, all realistic):
- `isAuthorized[victim][ecrecoverAuthorizer] = true`
- `isAuthorized[victim][setConsumedAgent] = true`
- `isAuthorized[victim][attacker] = true`

Steps:
1. Attacker constructs `Authorization { authorizer: victim, authorized: setConsumedAgent, isAuthorized: false, nonce: nonce[victim], deadline: T+1 }`.
2. Attacker signs the EIP-712 digest with their own private key.
3. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(auth, attackerSig)`.
4. `ecrecover` returns `attacker`; `IMidnight(MIDNIGHT).isAuthorized(victim, attacker)` → `true` → check passes (line 34).
5. `EcrecoverAuthorizer` calls `Midnight.setIsAuthorized(setConsumedAgent, false, victim)` (lines 46–47).
6. `Midnight.setIsAuthorized` verifies `isAuthorized[victim][ecrecoverAuthorizer]` → `true` → passes.
7. Result: `isAuthorized[victim][setConsumedAgent] = false`.

`setConsumedAgent` can no longer call `Midnight.setConsumed` on behalf of the victim. No existing check prevents this; the nonce is publicly readable on-chain.

## Impact Explanation
`setConsumedAgent` is permanently blocked from calling `Midnight.setConsumed(group, type(uint256).max, victim)`. All offers in the targeted group remain active and fillable, violating the invariant that offers cannot be filled after cancellation. Takers can continue filling those offers up to their original limits, causing unintended credit/debt changes for the victim.

## Likelihood Explanation
Any user who uses `EcrecoverAuthorizer` for signature-based delegation and has more than one authorized agent is vulnerable. The attacker only needs to hold one of those authorizations — a role reachable as a counterparty, relayer, or any other protocol participant. The attack requires no capital, is fully on-chain, and is repeatable (each deauthorization consumes one nonce, but the attacker can repeat if the victim re-authorizes).

## Recommendation
Restrict the signer check in `EcrecoverAuthorizer.setIsAuthorized` to only accept signatures from `authorization.authorizer` directly. Remove the second branch (`IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)`) entirely, or introduce a separate, narrowly scoped authorization role (e.g., `isAuthorizationManager`) that is distinct from general agent authorization. This ensures that only the account whose authorization state is being modified can sign the corresponding `Authorization` struct.

## Proof of Concept
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. As `victim`: call `Midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)`, `Midnight.setIsAuthorized(setConsumedAgent, true, victim)`, `Midnight.setIsAuthorized(attacker, true, victim)`.
3. As `attacker`: construct `Authorization { authorizer: victim, authorized: setConsumedAgent, isAuthorized: false, nonce: 0, deadline: block.timestamp + 1 }`, sign the EIP-712 digest with the attacker's private key.
4. Call `ecrecoverAuthorizer.setIsAuthorized(auth, attackerSig)` — expect success.
5. Assert `Midnight.isAuthorized(victim, setConsumedAgent) == false`.
6. As `setConsumedAgent`: call `Midnight.setConsumed(group, type(uint256).max, victim)` — expect revert with `Unauthorized`.
