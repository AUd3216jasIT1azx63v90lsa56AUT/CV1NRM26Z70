Audit Report

## Title
Authorized Agent Can Grief Victim's Nonce via Self-Deauthorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address `A` for which `isAuthorized[victim][A] == true` to sign an `Authorization(authorizer=victim, authorized=A, isAuthorized=false, nonce=N)` with its own key, pass all on-chain checks, atomically consume `nonce[victim]` (N â†’ N+1), and remove its own authorization. Any off-chain `Authorization` the victim has already signed and distributed with `nonce=N` becomes permanently unsubmittable with `InvalidNonce()`.

## Finding Description

**Root cause:** `EcrecoverAuthorizer.setIsAuthorized` (line 34) accepts any signer for whom `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns `true`, without restricting what the signed payload may contain. Nothing prevents an authorized signer from crafting a payload that targets itself as `authorized` with `isAuthorized=false`, thereby consuming the authorizer's nonce while removing its own delegation.

**Exact code path:**

1. **Line 26 â€” nonce consumed:**
```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```
`nonce[victim]` advances from N to N+1 as part of the `require` expression. If the rest of the transaction succeeds, this is permanent.

2. **Lines 31â€“36 â€” signature check passes:**
```solidity
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
require(signer != address(0), InvalidSignature());
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```
Attacker signs with their own key â†’ `signer = attacker`. Branch 1 (`attacker == victim`) is false. Branch 2 (`isAuthorized[victim][attacker]`) is **true** by precondition. Check passes.

3. **Lines 46â€“47 â€” downstream call:**
```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```
Resolves to `Midnight.setIsAuthorized(attacker, false, victim)`.

4. **Midnight line 732 â€” authorization check:**
```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
```
`msg.sender = EcrecoverAuthorizer`, `onBehalf = victim`. For the EcrecoverAuthorizer flow to function at all, `isAuthorized[victim][EcrecoverAuthorizer]` must be `true`. Check passes. Result: `isAuthorized[victim][attacker] = false`.

**Why existing checks fail:** The signature check only verifies that the signer is the authorizer or an authorized delegate. It does not verify that the payload's `authorized` field is not the signer itself, nor that `isAuthorized=false` self-removals are prohibited. The nonce is consumed unconditionally once the transaction succeeds.

## Impact Explanation
Any off-chain `Authorization(authorizer=victim, ..., nonce=N)` the victim has signed and distributed (e.g., to a relayer, bundler, or ratifier) is permanently invalidated. The victim must detect the invalidation out-of-band, re-sign with `nonce=N+1`, and redistribute. This constitutes a targeted, repeatable DoS on the victim's gasless authorization workflow. No funds are directly stolen, but the integrity of the off-chain signing flow is broken and can be continuously disrupted at negligible cost to the attacker.

## Likelihood Explanation
**Preconditions:** (1) `isAuthorized[victim][EcrecoverAuthorizer] == true` â€” required for any EcrecoverAuthorizer usage; (2) `isAuthorized[victim][attacker] == true` â€” standard operational state for any user who has authorized a relayer, bundler, ratifier, or callback contract. Both are normal for users of the gasless authorization flow. **Feasibility:** One transaction, no capital, no oracle dependency. **Repeatability:** One-shot per authorization grant (attacker loses their own authorization), but if the victim re-authorizes the attacker (e.g., a trusted relayer), the attack is repeatable indefinitely, sustaining the DoS.

## Recommendation
Add a check in `EcrecoverAuthorizer.setIsAuthorized` that prevents a delegated signer from signing a payload that targets itself as `authorized` with `isAuthorized=false`:

```solidity
require(
    signer == authorization.authorizer ||
    authorization.authorized != signer ||
    authorization.isAuthorized,
    Unauthorized()
);
```

Alternatively, restrict delegated signers from signing self-deauthorization payloads entirely: require that if `signer != authorization.authorizer`, then `authorization.authorized != signer`. A stricter fix would require the nonce check to occur after signature verification, though this does not eliminate the root cause.

## Proof of Concept
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. Victim calls `Midnight.setIsAuthorized(EcrecoverAuthorizer, true, victim)` and `Midnight.setIsAuthorized(attacker, true, victim)`.
3. Victim signs off-chain `Authorization(authorizer=victim, authorized=someAddress, isAuthorized=true, nonce=0, deadline=...)` and distributes to a relayer.
4. Attacker signs `Authorization(authorizer=victim, authorized=attacker, isAuthorized=false, nonce=0, deadline=...)` with attacker's key.
5. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(authorization, signature)`.
6. Transaction succeeds: `nonce[victim] == 1`, `isAuthorized[victim][attacker] == false`.
7. Relayer attempts to submit victim's off-chain authorization â†’ reverts with `InvalidNonce()`.
8. Victim must re-sign with `nonce=1` and redistribute.
