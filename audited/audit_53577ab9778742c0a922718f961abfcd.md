### Title
Zero `targetSellerAssets` bypasses sell loop, deposits collateral with no borrow - (`File: src/periphery/MidnightBundles.sol`)

### Summary
`supplyCollateralAndSellWithAssetsTarget` does not guard against `targetSellerAssets = 0`. When called with `targetSellerAssets = 0` and `referralFeePct = 0`, `targetFilledSellerAssets` resolves to `0`, the take loop's entry condition `filledSellerAssets < 0` is immediately false, all post-loop `require` checks pass trivially, and the function returns successfully after having deposited the caller's collateral into Midnight with no sell executed.

### Finding Description
**Code path** (`src/periphery/MidnightBundles.sol` lines 252–308):

1. **Line 262–263**: Authorization and `referralFeePct < WAD` checks pass for `referralFeePct = 0`.
2. **Lines 269–275**: Collateral supply loop runs unconditionally. For each entry in `collateralSupplies`, tokens are pulled from `msg.sender` and deposited into Midnight for `taker`.
3. **Line 277**: `referralFeeAssets = 0.mulDivDown(0, WAD) = 0`.
4. **Line 278**: `targetFilledSellerAssets = 0 + 0 = 0`.
5. **Line 282**: Loop condition `filledSellerAssets < targetFilledSellerAssets` → `0 < 0` → **false**. Loop body never executes; no `take` is called.
6. **Line 303**: `require(0 == 0, OutOfOffers())` → passes.
7. **Line 304**: `require(0 <= maxUnits, UnitsTooHigh())` → passes (uint256 ≥ 0 always).
8. **Line 306**: `if (0 > 0)` → skipped.
9. **Line 307**: `safeTransfer(loanToken, receiver, 0)` → executes (zero-value transfer succeeds on standard ERC20s).

**Root cause**: No `require(targetSellerAssets > 0)` (or equivalent `require(targetFilledSellerAssets > 0)`) guard before the take loop. The collateral supply loop and the take loop are sequenced independently, so collateral is committed before the zero-target is detected — but with a zero target, nothing is detected at all.

**Attacker-controlled inputs**: `targetSellerAssets = 0`, `referralFeePct = 0`, non-empty `collateralSupplies`. The caller must satisfy line 262 (`taker == msg.sender` or authorized), so the taker themselves is the natural caller.

### Impact Explanation
Collateral is deposited into Midnight for the taker (state change is real and irreversible without a separate `withdrawCollateral` call), but no debt is created and no loan tokens are received. The transaction succeeds silently, consuming gas for the collateral pull, approval, `supplyCollateral` call, and a zero-value `safeTransfer`, with no meaningful economic output. This matches the scoped impact: unnecessary gas cost, non-critical behavior divergence, no direct loss or freezing (collateral remains withdrawable by the taker under normal health rules).

### Likelihood Explanation
Reachable by any caller who is the taker or is authorized by the taker. No special state is required. The input `targetSellerAssets = 0` is a valid `uint256`. Repeatable on every call. No oracle manipulation or privileged access needed.

### Recommendation
Add an input guard immediately after the `referralFeePct` check:

```solidity
require(targetSellerAssets > 0, ZeroTarget());
```

Alternatively, guard on `targetFilledSellerAssets > 0` after line 278, which also covers the edge case where `targetSellerAssets > 0` but rounding produces `targetFilledSellerAssets == 0` (not possible here given integer arithmetic, but defensive). The analogous `supplyCollateralAndSellWithUnitsTarget` should be audited for the same pattern with `targetUnits = 0`.
