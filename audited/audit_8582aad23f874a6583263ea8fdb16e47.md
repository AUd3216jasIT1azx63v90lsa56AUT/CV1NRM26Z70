### Title
Missing zero-units guard in bundler loop causes no-op `take` calls on fully-consumed offers — (`src/periphery/MidnightBundles.sol`)

### Summary

When `offer.maxUnits > 0` and `consumed[maker][group] == maxUnits`, `ConsumableUnitsLib.consumableUnits` returns exactly 0 via `zeroFloorSub`. The bundler computes `unitsToTake = min(remaining, takes[i].units, 0) = 0` and then unconditionally issues `try take(offer, ratifierData, 0, taker, ...)` with no guard. `Midnight.take` accepts `units = 0` as a valid no-op (confirmed by the Certora rule `fullyConsumedOfferRevertsOnNonTrivialTake` and `takeInputOutputConsistency`), so the `try` succeeds, `filledUnits += 0`, and the loop advances to the next offer without making progress. Every offer slot in the `takes` array that is fully consumed burns gas on a full external call — including `touchMarket`, authorization checks, and a ratifier call — before the loop either finds a live offer or exhausts the array and reverts with `OutOfOffers`.

### Finding Description

**Code path:**

`ConsumableUnitsLib.consumableUnits` (line 16-17):
```solidity
if (offer.maxUnits > 0) {
    return offer.maxUnits.zeroFloorSub(consumed); // returns 0 when consumed >= maxUnits
}
``` [1](#0-0) 

`zeroFloorSub` is `mul(gt(x, y), sub(x, y))` — returns 0 when `x <= y`. [2](#0-1) 

`MidnightBundles.buyWithUnitsTargetAndWithdrawCollateral` loop (lines 74-85):
```solidity
uint256 unitsToTake = min(
    targetUnits - filledUnits,
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer) // = 0
);
// unitsToTake = min(x, y, 0) = 0 — no guard here
try IMidnight(MIDNIGHT).take(takes[i].offer, takes[i].ratifierData, unitsToTake, ...) returns (...) {
    filledUnits += unitsToTake; // += 0
    ...
} catch {}
``` [3](#0-2) 

`Midnight.take` with `units = 0` and `consumed == maxUnits` (lines 371-372):
```solidity
newConsumed = consumed[offer.maker][offer.group] += units; // += 0 → newConsumed = maxUnits
require(newConsumed <= offer.maxUnits, ConsumedUnits()); // maxUnits <= maxUnits → passes
``` [4](#0-3) 

All downstream asset and position deltas are zero when `units = 0`, so `take` returns `(0, 0)` without reverting. The Certora spec formally confirms this: [5](#0-4) [6](#0-5) 

**Root cause:** There is no `if (unitsToTake == 0) continue;` guard before the `try` block in any of the four bundler loops. The same pattern exists in `supplyCollateralAndSellWithUnitsTarget`, `buyWithAssetsTargetAndWithdrawCollateral`, and `supplyCollateralAndSellWithAssetsTarget`. [7](#0-6) [8](#0-7) 

**Exploit flow:**
1. Attacker (or any prior taker) fully consumes an offer so `consumed[maker][group] == maxUnits`.
2. A victim calls `buyWithUnitsTargetAndWithdrawCollateral` with a `takes` array that includes this offer.
3. For that slot: `consumableUnits = 0` → `unitsToTake = 0` → `take(..., 0, ...)` is dispatched.
4. `take` executes `touchMarket`, authorization checks, and `IRatifier.isRatified` (external call), then returns `(0, 0)`.
5. `filledUnits` is unchanged; the loop advances to the next offer.
6. If all offers in `takes` are fully consumed, the loop exhausts the array and reverts with `OutOfOffers`, having wasted gas on `N` unnecessary external calls.

A front-running attacker can deliberately consume all offers in a victim's pending `takes` array to maximize the gas waste before the `OutOfOffers` revert.

### Impact Explanation

Every fully-consumed offer slot in `takes` causes one unnecessary cross-contract call to `Midnight.take` (including a ratifier call), burning gas proportional to `takes.length`. If all offers are consumed, the transaction reverts with `OutOfOffers` after wasting the maximum gas. This constitutes concrete gas griefing: the victim pays for `N` no-op external calls that produce no state change and no filled units.

### Likelihood Explanation

**Preconditions:** An offer with `maxUnits > 0` must be fully consumed before the bundler call executes. This is a normal market condition — offers are consumed over time by any taker. A targeted attacker can front-run a specific victim's transaction to consume the exact offers in their `takes` array. The condition is reachable without any privileged access and is repeatable across any market.

### Recommendation

Add a zero-units guard immediately before each `try take(...)` call in all four bundler loops:

```solidity
if (unitsToTake == 0) continue;
try IMidnight(MIDNIGHT).take(...) ...
```

This skips the external call entirely when the offer is fully consumed (or when `takes[i].units == 0`), eliminating the unnecessary ratifier invocation, gas waste, and spurious zero-unit `Take` events.

### Proof of Concept

```solidity
// Foundry unit test
function testNoOpTakeOnFullyConsumedOffer() public {
    // Setup: maker offer with maxUnits = 100, fully consumed
    Offer memory offer = ...; // maxUnits = 100, maxAssets = 0
    // Consume the offer fully via a prior take
    midnight.take(offer, "", 100, taker1, ...);
    assertEq(midnight.consumed(offer.maker, offer.group), 100);

    // Now call bundler with this fully-consumed offer as the only take
    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: offer, ratifierData: "", units: 100});

    uint256 gasBefore = gasleft();
    vm.expectRevert(IMidnightBundles.OutOfOffers.selector);
    bundles.buyWithUnitsTargetAndWithdrawCollateral(
        100, maxBuyerAssets, taker2, emptyPermit, takes, new CollateralWithdrawal[](0), receiver, 0, address(0)
    );
    uint256 gasUsed = gasBefore - gasleft();

    // Assert: gas was wasted on a take(offer, ..., 0, ...) call
    // With the fix (continue on unitsToTake==0), gasUsed should be significantly lower
    // Key assertion: consumableUnits returned 0, unitsToTake was 0, take was still called
    assertEq(ConsumableUnitsLib.consumableUnits(address(midnight), id, offer), 0);
    // Verify take(offer, ..., 0, ...) would succeed (no-op)
    (uint256 ba, uint256 sa) = midnight.take(offer, "", 0, taker2, ...);
    assertEq(ba, 0);
    assertEq(sa, 0);
}
```

**Expected assertions:**
- `consumableUnits == 0` for the fully-consumed offer
- `take(offer, ..., 0, ...)` returns `(0, 0)` without reverting
- The bundler reverts with `OutOfOffers` after making the unnecessary `take` call
- Gas used with the fix is materially lower than without it

/// A fully-consumed offer in units mode only allows no-op takes.
rule fullyConsumedOfferRevertsOnNonTrivialTake(env e, Midnight.Offer offer, bytes ratifierData, uint256 units, address taker, address receiver, address takerCallback, bytes takerCallbackData) {
    require offer.maxAssets == 0;

    uint256 consumedBefore = consumed(offer.maker, offer.group);

    require offer.maxUnits > 0 && consumedBefore >= offer.maxUnits, "assume the offer is fully consumed";

    take(e, offer, ratifierData, units, taker, receiver, takerCallback, takerCallbackData);

    // If take does not revert, its input has to be zero.
    assert units == 0;
}
