### Title
Authorized operator can redirect victim's sell proceeds to arbitrary address via unvalidated `receiver` parameter - (`File: src/periphery/MidnightBundles.sol`)

### Summary
`supplyCollateralAndSellWithUnitsTarget` checks that `msg.sender` is authorized to act on behalf of `taker`, but places no restriction on the `receiver` parameter. An attacker who holds a Midnight authorization from the victim can pass `taker=victim, receiver=attacker`, causing the victim's debt to increase while all loan-token proceeds flow to the attacker.

### Finding Description

**Exact code path:**

Step 1 — Authorization gate in `MidnightBundles.supplyCollateralAndSellWithUnitsTarget` (line 127): [1](#0-0) 

```solidity
require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
```
With `taker=victim` and `msg.sender=attacker`, `isAuthorized[victim][attacker]=true` satisfies this check. No check is ever performed on `receiver`.

Step 2 — The bundler calls `Midnight.take` with `receiverIfTakerIsSeller = address(this)`: [2](#0-1) 

Step 3 — Inside `Midnight.take`, the taker-authorization check passes because the bundler is authorized by the victim: [3](#0-2) 

Step 4 — Because `offer.buy=true`, `receiver` inside `Midnight.take` resolves to `receiverIfTakerIsSeller = address(bundler)`: [4](#0-3) 

Step 5 — `sellerAssets` are transferred from the offer maker (lender) directly to the bundler: [5](#0-4) 

Step 6 — The bundler forwards the accumulated `filledSellerAssets` to the attacker-supplied `receiver`: [6](#0-5) 

**Root cause:** The `receiver` parameter is a free attacker input. The only authorization check (line 127) guards the right to act *as* the taker, not the right to designate *where proceeds go*. There is no `require(receiver == taker || isAuthorized(taker, receiver))` guard anywhere in the function.

**Attacker inputs:**
- `taker = victim` (victim has pre-authorized attacker and bundler on Midnight)
- `receiver = attacker`
- `collateralSupplies = []` (victim already has sufficient collateral/credit)
- `takes = [valid buy offers]`
- `minSellerAssets = 0` (no slippage protection needed by attacker)

### Impact Explanation
The victim's on-chain debt increases by `targetUnits` and their collateral remains locked, while every loan token received from the sell is transferred to the attacker. The victim suffers a direct, irreversible financial loss equal to `filledSellerAssets` in loan tokens, with no corresponding benefit to their position.

### Likelihood Explanation
**Preconditions:**
1. Victim has authorized the attacker on Midnight (`setIsAuthorized(attacker, true, victim)`) — this is the normal flow for delegating position management to a third-party operator or smart contract.
2. Victim has authorized the bundler on Midnight — also the normal setup for using `MidnightBundles`.
3. Victim has existing collateral sufficient to remain healthy after the sell.

All three preconditions are standard operational states. The attack is repeatable as long as the authorization persists and the victim has collateral. It requires no privileged access, no oracle manipulation, and no token-level exploits.

### Recommendation
Add a receiver authorization check immediately after the existing taker check:

```solidity
require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
require(receiver == taker || receiver == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, receiver), Unauthorized());
```

The same fix must be applied to `supplyCollateralAndSellWithAssetsTarget`, which has an identical structure. [7](#0-6) 

### Proof of Concept

```solidity
function testAttackerRedirectsSellProceeds() public {
    // Setup: victim (borrower) has collateral and has authorized both bundler and attacker.
    uint256 units = 100e18;
    offers[0].maxUnits = units;
    collateralize(market, borrower, units);

    address attacker = makeAddr("attacker");

    // Victim authorizes attacker on Midnight (normal operator delegation).
    vm.prank(borrower);
    midnight.setIsAuthorized(attacker, true, borrower);
    // Victim also authorizes bundler (already done in setUp).

    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: offers[0], units: units, ratifierData: hex""});

    uint256 attackerBalanceBefore = loanToken.balanceOf(attacker);
    uint256 borrowerBalanceBefore = loanToken.balanceOf(borrower);

    // Attacker calls with taker=victim, receiver=attacker.
    vm.prank(attacker);
    midnightBundles.supplyCollateralAndSellWithUnitsTarget(
        units,
        0,
        borrower,   // taker = victim
        attacker,   // receiver = attacker (malicious redirect)
        new CollateralSupply[](0),
        takes,
        0,
        address(0)
    );

    // Assertions:
    // Victim's debt increased — victim bears the liability.
    assertGt(midnight.debtOf(id, borrower), 0, "victim debt increased");
    // Victim received nothing.
    assertEq(loanToken.balanceOf(borrower), borrowerBalanceBefore, "victim got nothing");
    // Attacker received all proceeds.
    assertGt(loanToken.balanceOf(attacker), attackerBalanceBefore, "attacker stole proceeds");
}
```

**Expected result:** Test passes, confirming attacker's loan token balance increases while victim's debt increases and victim receives zero proceeds.
