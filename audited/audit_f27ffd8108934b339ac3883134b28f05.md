### Title
Midnight-Authorized Agent Can Increment Victim's EcrecoverAuthorizer Nonce, Invalidating All Pending Signed Authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary

`EcrecoverAuthorizer.setIsAuthorized` permits any signer satisfying `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer) == true` to submit a self-signed `Authorization` struct where `authorizer` is set to the victim. Because the nonce check at line 26 unconditionally increments `nonce[authorization.authorizer]` on success, and the authorization check at lines 33–36 accepts any Midnight-authorized agent as a valid signer for the authorizer's namespace, an attacker who holds any Midnight authorization from the victim can burn the victim's nonce at will. Every pre-signed authorization the victim has distributed to legitimate agents is thereby rendered permanently invalid.

**Exploit flow:**

1. Victim previously called `IMidnight.setIsAuthorized(attacker, true, victim)` (or the attacker obtained Midnight authorization through any legitimate path).
2. Victim off-chain signs and distributes `Authorization(authorizer=victim, authorized=legitimateAgent, isAuthorized=true, nonce=N, deadline=T)` to legitimate agents.
3. Attacker constructs `Authorization(authorizer=victim, authorized=attacker, isAuthorized=true, nonce=N, deadline=T')` and signs it with the attacker's own key.
4. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(authorization, attackerSignature)`.
5. `ecrecover` returns `attacker`; the check `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` passes.
6. `nonce[victim]` is incremented from `N` to `N+1`.
7. All legitimate agents holding victim's pre-signed authorization at nonce `N` now receive `InvalidNonce` on submission.
8. Attacker repeats step 3–6 with `nonce=N+1`, `N+2`, … indefinitely.

**Why existing checks fail:**

The `Unauthorized` check only verifies that the signer is either the authorizer themselves or a Midnight-authorized agent of the authorizer. It does not restrict *which* `Authorization` struct that agent may sign — in particular, it does not prevent the agent from signing a struct where `authorizer` is the victim rather than the agent themselves. The nonce increment is gated only on the nonce matching, not on the signer being the authorizer. [3](#0-2) 

### Impact Explanation

Any function in Midnight that relies on `EcrecoverAuthorizer`-mediated authorization (take, repay, withdraw, liquidate, claimFee) can be blocked for the victim. Legitimate agents holding pre-signed authorizations cannot act on the victim's behalf. The attacker can sustain the DoS at the cost of one transaction per nonce increment, with no capital at risk beyond gas.

### Likelihood Explanation

**Preconditions:** The attacker must hold a Midnight authorization from the victim (`IMidnight.isAuthorized(victim, attacker) == true`). This is a realistic precondition — users routinely authorize agents (bots, routers, relayers) on Midnight. Once the precondition holds, the attack is trivially repeatable with no additional trust or capital requirements. The attacker can front-run any legitimate submission to keep the nonce ahead of all distributed pre-signed authorizations.

### Recommendation

Restrict the signer's authority to sign only on their own behalf. Specifically, require that `authorization.authorizer == signer` when the signer is not the authorizer themselves, or equivalently, disallow Midnight-authorized agents from acting as signers for a different `authorizer` address. The simplest fix is to remove the `IMidnight.isAuthorized` branch from the signer check entirely and require `signer == authorization.authorizer`, relying solely on EIP-712 self-signed authorizations. If delegation is desired, add an explicit check that `authorization.authorizer == msg.sender` (direct call path) or that the signer is signing for themselves only. [3](#0-2)

```solidity
// Foundry unit test sketch
function test_nonce_dos() public {
    // Setup: victim authorizes attacker on Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim distributes pre-signed auth at nonce 0 to legitimateAgent
    Authorization memory legitAuth = Authorization({
        authorizer: victim, authorized: legitimateAgent,
        isAuthorized: true, nonce: 0, deadline: block.timestamp + 1 days
    });
    Signature memory legitSig = _sign(victimKey, legitAuth);

    // Attacker constructs and signs a competing auth at nonce 0
    Authorization memory attackAuth = Authorization({
        authorizer: victim, authorized: attacker,
        isAuthorized: true, nonce: 0, deadline: block.timestamp + 1 days
    });
    Signature memory attackSig = _sign(attackerKey, attackAuth);

    // Attacker submits first — increments nonce[victim] to 1
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);

    assertEq(ecrecoverAuthorizer.nonce(victim), 1);

    // Legitimate agent's submission now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    vm.prank(legitimateAgent);
    ecrecoverAuthorizer.setIsAuthorized(legitAuth, legitSig);
}
```

**Expected assertions:** `nonce[victim] == 1` after attacker's call; legitimate agent's call reverts with `InvalidNonce`. Repeat with incrementing nonces to demonstrate sustained DoS.

```text
    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
```
