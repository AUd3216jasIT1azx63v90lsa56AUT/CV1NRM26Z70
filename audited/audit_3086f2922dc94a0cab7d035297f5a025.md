### Title
Authorized operator can permanently cancel maker's Merkle root with no reversal mechanism - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
`cancelRoot` permits any address holding `isAuthorized[maker][msg.sender] == true` to set `isRootCanceled[maker][root] = true` for any root. Because no `uncancelRoot` function exists and the flag is never reset, a malicious-or-turned-hostile authorized operator can permanently destroy the maker's entire offer tree in a single call. Revoking the operator's authorization afterward does not undo the cancellation.

### Finding Description
**Exact code path:**

`EcrecoverRatifier.cancelRoot` (line 27–31):
```solidity
function cancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
    isRootCanceled[maker][root] = true;          // permanent, no inverse
    emit CancelRoot(msg.sender, maker, root);
}
``` [1](#0-0) 

The guard delegates to `IMidnight.isAuthorized`, the same flat boolean used for `withdraw`, `repay`, `setConsumed`, and `setIsAuthorized`. All of those actions are reversible; root cancellation is not. [2](#0-1) 

`isRatified` hard-blocks any offer whose root is canceled:
```solidity
require(!isRootCanceled[offer.maker][root], RootCanceled());
``` [3](#0-2) 

There is no function anywhere in `EcrecoverRatifier` that sets `isRootCanceled` back to `false`. [4](#0-3) 

**Attacker-controlled inputs:** `maker` (the victim's address), `root` (any Merkle root the victim signed).

**Exploit flow:**
1. Maker legitimately authorizes `attacker` for any purpose (e.g., to sign offers on their behalf, manage positions): `midnight.setIsAuthorized(attacker, true, maker)`.
2. Attacker calls `ecrecoverRatifier.cancelRoot(maker, root)` — passes the `isAuthorized` check.
3. `isRootCanceled[maker][root]` is set to `true` permanently.
4. Every subsequent `take` that references any offer in that tree reverts with `RootCanceled`.
5. Maker revokes attacker's authorization: `midnight.setIsAuthorized(attacker, false, maker)` — has no effect on the already-set flag.
6. Maker has no recourse; the entire tree is dead.

**Why existing checks fail:** The only guard is the `isAuthorized` check, which is intentionally satisfied by the precondition. The protocol has no per-action permission granularity and no reversal path.

The existing test `testCancelRootAuthorizedOnBehalf` confirms the path is reachable and produces exactly this state: [5](#0-4) 

### Impact Explanation
Every offer committed to the canceled Merkle tree becomes permanently unfillable. The maker cannot re-enable them; they must re-sign and redistribute an entirely new tree. If the attacker repeats the call for each new root the maker publishes, they can continuously grief the maker's offer capacity at negligible cost (one transaction per root).

### Likelihood Explanation
**Preconditions:** Maker must have authorized the attacker via `midnight.setIsAuthorized`. This is a normal operational step — makers authorize bots, co-signers, or portfolio managers. The authorization is a single flat boolean with no scope restriction.

**Feasibility:** One external call with two attacker-controlled arguments. No special tokens, oracle values, or flash loans required.

**Repeatability:** The attacker can cancel every new root the maker publishes, making the griefing indefinitely sustainable.

### Recommendation
Add a reversal path so makers can un-cancel roots:
```solidity
function uncancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
    isRootCanceled[maker][root] = false;
    emit UncancelRoot(msg.sender, maker, root);
}
```
Alternatively, restrict `cancelRoot` so that only `maker == msg.sender` (not any authorized operator) can cancel, matching the irreversible nature of the action. Authorized operators should only be permitted to cancel if the protocol explicitly intends to grant them that destructive power, which should be documented and ideally gated by a separate, narrower permission.

### Proof of Concept
```solidity
function testAuthorizedOperatorGriefsMakerRoot() public {
    // Setup: lender creates an offer tree
    Offer memory offer = makeOffer(lender);
    bytes32 _root = HashLib.hashOffer(offer);
    bytes memory ratifierData = buildRatifierData(_root, lender);

    // Lender authorizes borrower (e.g., as a co-signer)
    vm.prank(lender);
    midnight.setIsAuthorized(borrower, true, lender);

    // Sanity: offer is valid before attack
    vm.prank(address(midnight));
    bytes32 result = ecrecoverRatifier.isRatified(offer, ratifierData);
    assertEq(result, CALLBACK_SUCCESS);

    // Attack: borrower cancels lender's root
    vm.prank(borrower);
    ecrecoverRatifier.cancelRoot(lender, _root);

    // Assert: root is permanently canceled
    assertTrue(ecrecoverRatifier.isRootCanceled(lender, _root));

    // Lender revokes borrower's authorization — too late
    vm.prank(lender);
    midnight.setIsAuthorized(borrower, false, lender);

    // Assert: offer is still blocked despite revocation
    vm.prank(address(midnight));
    vm.expectRevert(IEcrecoverRatifier.RootCanceled.selector);
    ecrecoverRatifier.isRatified(offer, ratifierData);

    // Assert: lender has no way to restore the root
    // (no uncancelRoot function exists — confirmed by absence in interface)
}
```
Expected assertions: `isRootCanceled(lender, _root) == true` after the attack; `isRatified` reverts with `RootCanceled` even after authorization is revoked; no function call can restore the root.

```text
contract EcrecoverRatifier is IEcrecoverRatifier {
    address public immutable MIDNIGHT;

    mapping(address maker => mapping(bytes32 root => bool)) public isRootCanceled;

    constructor(address _midnight) {
        MIDNIGHT = _midnight;
    }

    function cancelRoot(address maker, bytes32 root) external {
        require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
        isRootCanceled[maker][root] = true;
        emit CancelRoot(msg.sender, maker, root);
    }

    function isRatified(Offer memory offer, bytes memory ratifierData) external view returns (bytes32) {
        require(msg.sender == MIDNIGHT, NotMidnight());
        (Signature memory sig, bytes32 root, uint256 leafIndex, bytes32[] memory proof) =
            abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]));
        require(HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof), InvalidProof());
        require(!isRootCanceled[offer.maker][root], RootCanceled());
        bytes32 structHash = keccak256(abi.encode(HashLib.offerTreeTypeHash(proof.length), root));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, structHash));
        address _signer = ecrecover(digest, sig.v, sig.r, sig.s);
        require(_signer != address(0), InvalidSignature());
        require(_signer == offer.maker || IMidnight(MIDNIGHT).isAuthorized(offer.maker, _signer), Unauthorized());
        return CALLBACK_SUCCESS;
    }
```
