### Title
Taker-Crafted 21-Element Proof Leaks `TreeTooHigh` Instead of `InvalidProof` in `isRatified` - (File: src/ratifiers/EcrecoverRatifier.sol)

### Summary
In `EcrecoverRatifier.isRatified`, the Merkle proof validity check (`HashLib.isLeaf`) executes before the type-hash lookup (`HashLib.offerTreeTypeHash`). Because a taker fully controls `ratifierData`, they can supply a self-consistent proof of length 21 that passes `isLeaf` and then causes `offerTreeTypeHash(21)` to revert with `HashLib.TreeTooHigh` rather than `IEcrecoverRatifier.InvalidProof`. This produces an observable error-selector divergence that callers cannot distinguish from a structurally invalid proof.

### Finding Description

**Execution order in `isRatified`:**

```
Line 37: require(HashLib.isLeaf(root, HashLib.hashOffer(offer), leafIndex, proof), InvalidProof());
Line 38: require(!isRootCanceled[offer.maker][root], RootCanceled());
Line 39: bytes32 structHash = keccak256(abi.encode(HashLib.offerTreeTypeHash(proof.length), root));
``` [1](#0-0) 

`HashLib.offerTreeTypeHash` explicitly reverts with `TreeTooHigh()` for any `height > 20`:

```
if (height == 20) return 0x824e...;
revert TreeTooHigh();
``` [2](#0-1) 

**Attacker-controlled inputs:** The taker supplies `ratifierData`, which `abi.decode`s into `(Signature, bytes32 root, uint256 leafIndex, bytes32[] proof)`. The taker controls all four fields. [3](#0-2) 

**Exploit construction:**

1. Taker knows the offer (it is passed into `isRatified` by Midnight).
2. Taker computes `leafHash = HashLib.hashOffer(offer)`.
3. Taker builds `proof` of length 21 with arbitrary `bytes32` siblings and sets `leafIndex = 0`.
4. Taker computes the resulting Merkle root by hashing up 21 levels and sets `root` to that value.
5. `HashLib.isLeaf` checks `leafIndex >> proof.length == 0` (satisfied: `0 >> 21 == 0`) then iterates 21 times and returns `currentHash == root` â€” **true** by construction. [4](#0-3) 

6. Line 38 passes because the taker chose a fresh `root` not in `isRootCanceled`.
7. Line 39 calls `HashLib.offerTreeTypeHash(21)`, which falls through all `if` branches and hits `revert TreeTooHigh()`.

The revert propagates through `isRatified` â†’ `Midnight.take` â†’ caller, exposing selector `HashLib.TreeTooHigh` instead of `IEcrecoverRatifier.InvalidProof`.

### Impact Explanation

Any off-chain system (keeper, relayer, SDK, indexer) that catches `InvalidProof` to classify a failed take as "bad proof" will instead receive `TreeTooHigh`, an error defined in a library not part of the ratifier's public interface. This breaks error-based branching logic: a caller cannot tell whether the proof was structurally wrong or merely oversized. No funds are moved and no state is written; the impact is confined to observable error-selector divergence on the revert path.

### Likelihood Explanation

The precondition is trivially satisfiable: any taker can craft `ratifierData` with `proof.length == 21` and a self-consistent root at zero cost, with no privileged access required. The path is deterministic and repeatable on every call. The only constraint is that the taker must be able to call `Midnight.take` with an offer whose `ratifier` is `EcrecoverRatifier`, which is the normal usage path.

### Recommendation

Move the `offerTreeTypeHash` call â€” or an explicit length guard â€” **before** the `isLeaf` check, so an oversized proof is rejected with a clear, ratifier-level error before any proof arithmetic runs:

```solidity
// Add before line 37:
require(proof.length <= 20, InvalidProof()); // or a dedicated OversizedProof error
```

Alternatively, wrap the `offerTreeTypeHash` call in a try/catch and re-throw as `InvalidProof()`, keeping the public error surface uniform.

### Proof of Concept

```solidity
// Foundry unit test sketch
function test_treeTooHigh_leaks_wrong_selector() public {
    // 1. Create a valid offer with ratifier = address(ecrecoverRatifier)
    Offer memory offer = _buildOffer(address(ecrecoverRatifier));

    // 2. Compute leaf hash
    bytes32 leafHash = HashLib.hashOffer(offer);

    // 3. Build a 21-element proof with arbitrary siblings, leafIndex = 0
    bytes32[] memory proof = new bytes32[](21);
    for (uint i = 0; i < 21; i++) proof[i] = keccak256(abi.encode(i));

    // 4. Compute self-consistent root
    bytes32 root = leafHash;
    for (uint i = 0; i < 21; i++) {
        root = keccak256(abi.encodePacked(root, proof[i]));
    }

    // 5. Encode ratifierData (signature can be anything; revert happens before ecrecover)
    Signature memory sig = Signature({v: 27, r: bytes32(0), s: bytes32(0)});
    bytes memory ratifierData = abi.encode(sig, root, uint256(0), proof);

    // 6. Expect revert with TreeTooHigh, NOT InvalidProof
    vm.expectRevert(HashLib.TreeTooHigh.selector);
    midnight.take(offer, ratifierData, ...);

    // 7. Contrast: proof.length == 20, wrong root â†’ InvalidProof
    bytes32[] memory proof20 = new bytes32[](20);
    bytes memory ratifierData20 = abi.encode(sig, bytes32(uint256(1)), uint256(0), proof20);
    vm.expectRevert(IEcrecoverRatifier.InvalidProof.selector);
    midnight.take(offer, ratifierData20, ...);
}
```

Expected assertions: first call reverts with `TreeTooHigh.selector`; second call reverts with `InvalidProof.selector` â€” confirming the divergence.

    function isLeaf(bytes32 root, bytes32 leafHash, uint256 leafIndex, bytes32[] memory proof)
        internal
        pure
        returns (bool)
    {
        require(leafIndex >> proof.length == 0, LeafIndexOutOfRange());
        bytes32 currentHash = leafHash;
        for (uint256 i = 0; i < proof.length; i++) {
            currentHash = (leafIndex >> i) & 1 == 0 ? hashNode(currentHash, proof[i]) : hashNode(proof[i], currentHash);
        }
