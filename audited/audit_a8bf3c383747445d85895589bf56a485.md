### Title
Missing Oracle Address Validation in `touchMarket()` Enables Zero-Price Oracle Exploitation — (File: src/Midnight.sol)

---

### Summary

`touchMarket()` validates collateral token addresses, LLTV values, and `maxLif` parameters during market creation, but performs **no validation on `collateralParams[i].oracle`**. A market can be created with `oracle = address(0)`. In Solidity 0.8.34, calling `IOracle(address(0)).price()` does not revert — it returns `0` (empty returndata decoded as `uint256`). When `liquidatedCollatPrice = 0`, the `liquidate()` path that takes `seizedAssets > 0` computes `repaidUnits = 0`, allowing a liquidator to seize a borrower's entire collateral while repaying zero debt.

---

### Finding Description

**Root cause — `touchMarket()` does not validate `collateralParams[i].oracle`:** [1](#0-0) 

The loop checks `collateralToken`, `lltv`, and `maxLif` but never asserts `market.collateralParams[i].oracle != address(0)`.

**Oracle call with zero address returns 0:** [2](#0-1) 

`IOracle(address(0)).price()` is a low-level CALL to an address with no code. The EVM returns `(success=true, data=0x)`. Solidity decodes empty bytes as `uint256(0)`. No revert occurs.

**Consequence in `liquidate()` — free collateral seizure:** [3](#0-2) 

When `seizedAssets > 0`:
```
repaidUnits = seizedAssets.mulDivUp(0, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif)
            = 0
```
The liquidator receives `seizedAssets` of collateral token while `repaidUnits = 0` — zero debt is repaid.

**Debt accounting after the exploit:** [4](#0-3) 

`_marketState.withdrawable += 0` and `_position.debt -= 0` — the borrower's debt is untouched. The borrower loses all collateral but still owes the full debt obligation.

**Secondary effect — `isHealthy` always returns false:** [5](#0-4) 

With `price = 0`, `maxDebt = 0` for every collateral, so `maxDebt >= debt` is always false. `withdrawCollateral` reverts for any borrower, permanently locking collateral until the attacker liquidates it for free.

**`repaidUnits > 0` path reverts (division by zero):** [6](#0-5) 

`seizedAssets = repaidUnits * lif / WAD * ORACLE_PRICE_SCALE / 0` — panics. Only the `seizedAssets > 0` path is usable, which is the free-seizure path.

---

### Impact Explanation

A liquidator calling `liquidate(market, collateralIndex, seizedAssets=X, repaidUnits=0, ...)` on a market whose `collateralParams[collateralIndex].oracle = address(0)`:

1. Receives `X` collateral tokens transferred to `receiver`.
2. Repays `0` loan tokens (`SafeTransferLib.safeTransferFrom(..., 0)` is a no-op).
3. The borrower's debt position is unchanged — they still owe the full debt but hold no collateral.

This is **direct theft of user collateral** with no capital requirement. The borrower is left insolvent with no recourse.

---

### Likelihood Explanation

`touchMarket()` is a `public` function callable by anyone with no access control. [7](#0-6) 

An attacker can:
1. Deploy a market with `oracle = address(0)` at zero cost.
2. Advertise it through a malicious front-end or integrate it into a bundle.
3. Wait for victims to `supplyCollateral` and borrow.
4. Drain all collateral via `liquidate`.

The protocol is permissionless and immutable — there is no admin mechanism to invalidate a created market. Once a victim's collateral is in the market, it is irrecoverable except by the attacker.

---

### Recommendation

Add a non-zero oracle address check inside the `touchMarket()` collateral validation loop:

```solidity
for (uint256 i = 0; i < market.collateralParams.length; i++) {
    address collateralToken = market.collateralParams[i].token;
    require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
+   require(market.collateralParams[i].oracle != address(0), ZeroOracle());
    uint256 lltv = market.collateralParams[i].lltv;
    require(isLltvAllowed(lltv), LltvNotAllowed());
    ...
}
```

Also add a corresponding error to `IMidnight.sol`:
```solidity
error ZeroOracle();
```

Similarly, consider adding `require(market.loanToken != address(0))` to prevent markets with a zero loan token from being created (though `SafeTransferLib`'s `NoCode()` check provides a secondary guard there).

### Proof of Concept

**Setup:**
```solidity
CollateralParams[] memory params = new CollateralParams[](1);
params[0] = CollateralParams({
    token: address(collateralToken),
    lltv: LLTV_0,           // 0.385e18 — valid
    maxLif: maxLif(LLTV_0, LIQUIDATION_CURSOR_LOW),
    oracle: address(0)      // ← zero oracle, no validation
});

Market memory market = Market({
    loanToken: address(loanToken),
    collateralParams: params,
    maturity: block.timestamp + 30 days,
    rcfThreshold: 1e18,
    enterGate: address(0),
    liquidatorGate: address(0)
});
```

**Victim flow:**
```solidity
midnight.supplyCollateral(market, 0, 1000e18, victim);
// victim takes a sell offer → debt = 500e18
```

**Attacker exploit:**
```solidity
// repaidUnits = seizedAssets * 0 / ORACLE_PRICE_SCALE * WAD / lif = 0
midnight.liquidate(market, 0, 1000e18, 0, victim, false, attacker, address(0), "");
// attacker receives 1000e18 collateral, pays 0 loan tokens
// victim still has debt = 500e18, collateral = 0
```

**Expected outcome:** Attacker holds `1000e18` collateral tokens. Victim holds `0` collateral and `500e18` debt — fully insolvent with no recovery path.
