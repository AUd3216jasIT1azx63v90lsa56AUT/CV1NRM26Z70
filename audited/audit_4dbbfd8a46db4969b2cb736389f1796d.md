### Title
Unvalidated `oracle` Address in `CollateralParams` During `touchMarket` Permanently Breaks Liquidations and Health Checks — (File: `src/Midnight.sol`)

---

### Summary

The `touchMarket` function in `Midnight.sol` creates markets without validating the `oracle` field in each `CollateralParams` entry. If `oracle` is set to `address(0)` (or any non-contract address), every downstream call to `IOracle(oracle).price()` will revert. Because markets are immutable once created, this permanently breaks `liquidate`, `isHealthy`, `take` (for sellers with debt), and `withdrawCollateral` (for borrowers with debt) for the affected market — an exact structural analog to the Celo `[M01]` finding where an unvalidated registry identifier broke `mint` and `burn`.

---

### Finding Description

**Root cause — missing oracle validation in `touchMarket`**

`touchMarket` is a public, permissionless function. When a new market is created (i.e., `marketState[id].tickSpacing == 0`), the function iterates over `collateralParams` and validates only three things per entry: [1](#0-0) 

- `collateralToken > previousCollateralToken` (sorted, no duplicates)
- `isLltvAllowed(lltv)` (valid LLTV value)
- `maxLif` matches one of two allowed formulas

There is **no check** that `market.collateralParams[i].oracle != address(0)` or that it is a deployed contract. The oracle address is stored immutably as part of the market identity (via `IdLib.storeInCode`) and can never be changed after creation. [2](#0-1) 

**Downstream breakage — four critical functions**

Every function that must assess collateral value calls `IOracle(collateralParam.oracle).price()`. A call to `address(0)` reverts unconditionally in the EVM.

1. **`liquidate`** — iterates all activated collaterals and calls `.price()` on each oracle: [3](#0-2) 

2. **`isHealthy`** — same pattern: [4](#0-3) 

3. **`take`** — calls `isHealthy` on the seller at the end of every execution; if the seller has debt, the oracle is invoked and the entire `take` reverts: [5](#0-4) 

4. **`withdrawCollateral`** — calls `isHealthy` when the borrower has debt: [6](#0-5) 

---

### Impact Explanation

A market created with `oracle = address(0)` for any collateral is **permanently and irrecoverably broken** for all participants who hold debt in it:

- **Liquidations are permanently disabled.** Borrowers with activated collateral at that index can never be liquidated, regardless of how far underwater their position goes.
- **Lenders' capital is permanently at risk.** If borrowers refuse to repay, lenders have no recourse — `liquidate` always reverts, and there is no admin override or market upgrade path.
- **Sellers with debt cannot take offers.** The `take` function reverts at the final health check, freezing the borrower's ability to trade out of their position.
- **Borrowers with debt cannot withdraw collateral.** `withdrawCollateral` reverts on the health check, trapping collateral.

The `CollateralParams` struct is part of the market's identity hash; the oracle address cannot be patched post-creation. [7](#0-6) 

---

### Likelihood Explanation

`touchMarket` is **public and permissionless** — any externally owned account can create a market. No privileged role is required. The attack path requires no leaked keys, no governance access, and no economic capital beyond gas.

**Realistic scenarios:**

1. **Deliberate exploit**: An attacker creates a market with `oracle = address(0)`, posts attractive buy offers (maker = borrower), waits for lenders to fill them, supplies worthless collateral, borrows, and never repays. Since liquidation is permanently broken, the attacker faces no forced repayment.

2. **Accidental misconfiguration**: A naive integrator or user creates a market with a zero or undeployed oracle address. All participants who interact with that market before discovering the error have their positions permanently frozen.

Both scenarios are reachable with no privilege assumptions. The permissionless nature of market creation is the direct enabler.

---

### Recommendation

Add an explicit non-zero oracle address check inside the `touchMarket` validation loop, mirroring the existing checks for `lltv` and `maxLif`:

```solidity
// In touchMarket, inside the collateralParams loop:
require(market.collateralParams[i].oracle != address(0), InvalidOracle());
```

Optionally, also verify that the oracle address has deployed code (`oracle.code.length > 0`) to catch non-contract addresses that would also cause a revert on `.price()`. A corresponding `InvalidOracle` error should be added to `IMidnight.sol`. [1](#0-0)

### Proof of Concept

**Setup**: Deploy `Midnight`. Construct a `Market` with one `CollateralParams` entry where `oracle = address(0)`, a valid `loanToken`, valid `lltv`, and valid `maxLif`.

**Step 1 — Create the broken market** (no privilege required):
```solidity
CollateralParams[] memory params = new CollateralParams[](1);
params[0] = CollateralParams({
    token:   address(someCollateralToken),
    lltv:    0.8e18,          // any allowed LLTV
    maxLif:  maxLif(0.8e18, LIQUIDATION_CURSOR_LOW),
    oracle:  address(0)       // zero address — not validated
});
Market memory market = Market({
    loanToken:       address(loanToken),
    collateralParams: params,
    maturity:        block.timestamp + 365 days,
    rcfThreshold:    1e18,
    enterGate:       address(0),
    liquidatorGate:  address(0)
});
midnight.touchMarket(market); // succeeds — no revert
```

**Step 2 — Borrow in the market**:
```solidity
// Attacker supplies collateral and takes a sell offer, acquiring debt
midnight.supplyCollateral(market, 0, 1000e18, attacker);
// ... take a sell offer to acquire debt ...
```

**Step 3 — Attempt liquidation** (always reverts):
```solidity
// Reverts: IOracle(address(0)).price() → EVM call to zero address
midnight.liquidate(market, 0, 0, debtAmount, attacker, false, receiver, address(0), "");
// → revert (zero-address call)
```

**Expected outcome**: `liquidate` reverts unconditionally. The borrower's position is permanently unliquidatable. Lenders cannot recover their funds if the borrower does not voluntarily repay. [8](#0-7) [9](#0-8)
