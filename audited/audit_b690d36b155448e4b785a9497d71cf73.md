### Title
Zero-unit take unconditionally invokes buyerCallback with buyerAssets=0 and sets seller liquidation lock at no economic cost - (`src/Midnight.sol`)

### Summary
`take()` contains no guard against `units == 0`. When called with `units=0`, `offer.buy=True`, and `offer.maxAssets > 0`, all accounting deltas are zero yet the function still sets the seller's liquidation lock in transient storage and unconditionally dispatches the `buyerCallback` (`offer.callback`) with `buyerAssets=0`. An unprivileged taker can trigger the maker's callback at zero token cost, violating the invariant that a zero-unit take is a no-op.

### Finding Description

**Exact code path** (`src/Midnight.sol`):

1. No `units > 0` guard exists anywhere in `take()`. The grep for `require.*units` returns zero matches.

2. With `units=0`, `offer.buy=True`, `offer.maxAssets > 0`:
   - Line 363: `buyerAssets = units.mulDivDown(buyerPrice, WAD) = 0`
   - Line 364: `sellerAssets = 0`
   - Lines 367-369: `consumed[offer.maker][offer.group] += 0`; `require(newConsumed <= offer.maxAssets)` passes trivially
   - Lines 382-389: `buyerCreditIncrease=0`, `sellerCreditDecrease=0`, `sellerDebtIncrease=0`, all fee deltas = 0
   - Lines 408-418: every position write is `+= 0` or `-= 0`; `claimableSettlementFee += 0`

3. Line 444: `wasLocked = UtilsLib.tExchange(LIQUIDATION_LOCK_SLOT, id, seller, true)` — the lock is set for `seller = taker` **unconditionally**, regardless of `units`.

4. Lines 445-453: `if (buyerCallback != address(0))` — `buyerCallback = offer.callback` for a buy offer. The callback is dispatched with `buyerAssets=0, units=0, pendingFeeIncrease=0`. No guard on `buyerAssets > 0` before this branch.

5. Lines 455-456: zero-amount `safeTransferFrom` calls execute (no revert on zero).

6. Line 475: lock is restored; line 476: health check passes because `sellerDebtIncrease=0` left the seller's position unchanged.

**Attacker-controlled inputs**: `offer.buy=True`, `offer.maxAssets > 0` (any positive value), `offer.callback = <attacker-controlled or victim callback>`, `units=0`. The taker supplies these directly to `take()`.

**Why existing checks fail**:
- `require(UtilsLib.atMostOneNonZero(offer.maxAssets, offer.maxUnits))` — only checks mutual exclusivity, not that `units > 0`.
- `require(newConsumed <= offer.maxAssets)` — `newConsumed` is unchanged when `buyerAssets=0`, so this always passes.
- `require(offer.maker != taker)` — prevents self-take but does not prevent a third-party taker from calling with `units=0`.
- No `require(units > 0)` exists anywhere in the function. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation

The concrete scoped impact is confirmed:

1. **Callback invoked at zero economic cost**: `offer.callback.onBuy(id, market, buyerAssets=0, units=0, 0, buyer, data)` is called. The callback must return `CALLBACK_SUCCESS` but is not required to transfer any tokens (since `buyerAssets - sellerAssets = 0`). Any side effects inside the callback (state writes, external calls, event emissions, interactions with other protocols) execute for free.

2. **Liquidation lock observable during callback**: At the moment `onBuy` executes, `liquidationLocked(id, seller) == true` (transient storage). The callback can read this via `IMidnight.liquidationLocked(id, taker)`. While the lock itself blocks a re-entrant liquidation attempt, the callback can branch on this observable state or relay it to other contracts.

3. **No token cost to attacker**: `safeTransferFrom(..., 0)` succeeds silently. The taker pays only gas. [4](#0-3) [5](#0-4) 

### Likelihood Explanation

**Preconditions**:
- A live offer with `offer.buy=True`, `offer.maxAssets > 0`, and `offer.callback != address(0)` must exist (common for lend-side offers using flash-lend callbacks).
- The taker must not equal `offer.maker` (trivially satisfied by any third party).
- The offer must not be expired and must pass ratifier check (standard conditions for any take).

**Feasibility**: Fully reachable with a single external call. No privileged access, no oracle manipulation, no special token behavior required. The existing fuzz test `testBuyBuyerCallback` already bounds `units` from 0, confirming the protocol does not reject `units=0` at the test level.

**Repeatability**: Unlimited — each call with `units=0` passes the `consumed` check trivially (adds 0), so the same offer can be targeted indefinitely. [6](#0-5) 

### Recommendation

Add a non-zero units guard at the top of `take()`, immediately after the authorization check:

```solidity
require(units > 0, ZeroUnits());
```

This ensures the callback is only dispatched when an economically meaningful take occurs, restoring the no-op invariant for `units=0` and eliminating the zero-cost callback trigger vector. [7](#0-6) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {IMidnight, Offer} from "src/interfaces/IMidnight.sol";
import {IBuyCallback} from "src/interfaces/ICallbacks.sol";
import {CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";

contract ZeroUnitCallbackSpy is IBuyCallback {
    bool public wasCalled;
    uint256 public recordedBuyerAssets;
    bool public liquidationLockObserved;
    IMidnight public midnight;
    bytes32 public marketId;
    address public seller;

    constructor(address _midnight, bytes32 _id, address _seller) {
        midnight = IMidnight(_midnight);
        marketId = _id;
        seller = _seller;
    }

    function onBuy(bytes32, Market memory, uint256 buyerAssets, uint256, uint256, address, bytes memory)
        external returns (bytes32)
    {
        wasCalled = true;
        recordedBuyerAssets = buyerAssets;
        liquidationLockObserved = midnight.liquidationLocked(marketId, seller);
        return CALLBACK_SUCCESS;
    }
}

contract ZeroUnitTakeCallbackTest is BaseTest {
    function testZeroUnitTakeInvokesBuyerCallback() public {
        // Setup: lender (maker) creates buy offer with callback, maxAssets > 0
        ZeroUnitCallbackSpy spy = new ZeroUnitCallbackSpy(address(midnight), id, borrower);
        
        lenderOffer.buy = true;
        lenderOffer.callback = address(spy);
        lenderOffer.maxAssets = 1e18; // non-zero maxAssets branch
        lenderOffer.maxUnits = 0;
        lenderOffer.tick = MAX_TICK;
        
        // Borrower (taker, seller) calls take with units=0
        vm.prank(borrower);
        (uint256 buyerAssets, uint256 sellerAssets) = midnight.take(
            lenderOffer, hex"", 0, borrower, borrower, address(0), hex""
        );
        
        // Assertions
        assertEq(buyerAssets, 0, "buyerAssets must be 0");
        assertEq(sellerAssets, 0, "sellerAssets must be 0");
        assertTrue(spy.wasCalled(), "callback must have been invoked");
        assertEq(spy.recordedBuyerAssets(), 0, "callback received buyerAssets=0");
        assertTrue(spy.liquidationLockObserved(), "liquidation lock was set for seller during callback");
        // consumed unchanged
        assertEq(midnight.consumed(lenderOffer.maker, lenderOffer.group), 0, "consumed unchanged");
    }
}
```

**Expected assertions**: all pass, confirming the callback is invoked with `buyerAssets=0` at zero token cost and the seller's liquidation lock is observable inside the callback. [8](#0-7) [9](#0-8)
