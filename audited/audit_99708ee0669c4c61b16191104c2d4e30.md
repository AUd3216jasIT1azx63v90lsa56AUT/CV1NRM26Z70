### Title
Fee-on-Transfer Collateral Token Inflates `position.collateral` Beyond Protocol Balance - (File: src/Midnight.sol)

### Summary

`supplyCollateral` credits `position[id][onBehalf].collateral[collateralIndex]` with the caller-supplied `assets` value before executing the token transfer, with no before/after balance measurement. When the collateral token silently deducts a transfer fee, the protocol records more collateral than it actually receives, violating the solvency invariant that `collateralToken.balanceOf(midnight) >= Σ position.collateral[i]`. Because `isHealthy` uses the inflated accounting value to compute `maxDebt`, the borrower can borrow against and later withdraw phantom collateral, draining real funds from other users.

### Finding Description

**Code path and root cause**

In `supplyCollateral` (lines 524–546 of `src/Midnight.sol`):

```
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);  // line 533
...
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets); // line 545
```

The accounting write at line 533 uses the caller-supplied `assets` parameter unconditionally. The `safeTransferFrom` at line 545 requests `assets` tokens, but a fee-on-transfer token delivers only `assets * (1 - fee_rate)` to `address(this)`. No balance snapshot is taken before or after the transfer to reconcile the discrepancy.

**Market creation is permissionless**

`touchMarket` (lines 755–791) validates only: maturity bound, non-empty sorted collateral list, allowed LLTV tiers, and valid `maxLif` values. There is no check that the collateral token is a standard ERC20. Any unprivileged user can create a market whose `collateralParams[i].token` is a fee-on-transfer contract.

**`isHealthy` uses the inflated accounting value**

`isHealthy` (lines 944–960) computes `maxDebt` from `_position.collateral[i]` (the inflated stored value), not from the actual token balance:

```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
```

So the health check passes for a position whose recorded collateral exceeds what the protocol holds.

**`withdrawCollateral` transfers the full recorded amount**

`withdrawCollateral` (lines 549–573) subtracts `assets` from the position and then calls `SafeTransferLib.safeTransfer(collateralToken, receiver, assets)` (line 572). If the protocol's actual balance is less than the sum of all recorded collateral positions, this transfer either reverts (freezing legitimate withdrawals) or, if the deficit is covered by other users' collateral, drains those users.

**Exploit flow**

1. Attacker deploys a fee-on-transfer ERC20 with a 10% fee.
2. Attacker calls `touchMarket` with this token as `collateralParams[0].token` — succeeds with no restriction.
3. Attacker calls `supplyCollateral(market, 0, 1e18, attacker)`:
   - Line 533 sets `position.collateral[0] = 1e18`.
   - Line 545 transfers only `0.9e18` to `address(this)`.
4. Attacker calls `take` (or borrows via a sell offer) against the inflated collateral. `isHealthy` sees `1e18` collateral and allows borrowing up to `1e18 * price * lltv / ORACLE_PRICE_SCALE` in debt.
5. Attacker calls `withdrawCollateral(market, 0, 1e18, attacker, attacker)`:
   - Line 561 subtracts `1e18` from the position (passes).
   - Line 568 `isHealthy` check passes (debt is 0 after repayment, or attacker never borrowed).
   - Line 572 transfers `1e18` out — but the protocol only received `0.9e18`, so `0.1e18` is taken from other users' collateral.

**Why existing checks fail**

- `touchMarket` has no token-type validation.
- `supplyCollateral` performs no balance delta measurement.
- `isHealthy` is blind to the actual token balance.
- The Certora `Solvency.spec` (README line 78) and `BalanceEffects.spec` (line 217) summarize `safeTransferFrom` as `NONDET` (see `LiquidationProfitability.spec` lines 24–25), so formal verification does not model fee-on-transfer behavior.

### Impact Explanation

The solvency invariant `collateralToken.balanceOf(midnight) >= Σ_users position[id][user].collateral[i]` is broken for every market using a fee-on-transfer collateral token. A borrower can borrow against phantom collateral (debt backed by tokens the protocol does not hold) and withdraw more collateral than was deposited, directly draining other users' collateral balances. The protocol's balance goes negative for that token, causing legitimate `withdrawCollateral` calls by honest users to revert or to be paid from other tokens' balances if the contract holds them.

### Likelihood Explanation

**Preconditions**: A fee-on-transfer ERC20 must be used as a collateral token. Market creation is permissionless, so the attacker can deploy the token and create the market themselves — no admin action required. The attack is repeatable on every `supplyCollateral` call in such a market. Many production tokens (e.g., tokens with configurable transfer fees) satisfy this precondition.

**Feasibility**: High. The attacker controls the collateral token contract and the market parameters. No privileged access, no oracle manipulation, and no user error is required.

### Recommendation

Measure the actual received amount using a before/after balance check in `supplyCollateral`, and credit only the delta:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

Alternatively, explicitly disallow fee-on-transfer tokens in `touchMarket` by performing a probe transfer and checking the received amount equals the sent amount. The state write should always reflect the actual received amount, not the caller-supplied `assets`.

pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {ERC20} from "solmate/tokens/ERC20.sol";

// Fee-on-transfer token: deducts 10% on every transferFrom
contract FeeToken is ERC20("Fee", "FEE", 18) {
    function mint(address to, uint256 amt) external { _mint(to, amt); }
    function transferFrom(address from, address to, uint256 amt) public override returns (bool) {
        uint256 fee = amt / 10;
        super.transferFrom(from, to, amt - fee); // only amt*0.9 arrives
        _burn(from, fee);                         // fee is burned
        return true;
    }
}

contract FeeOnTransferCollateralTest is Test {
    Midnight midnight;
    FeeToken feeToken;
    // ... standard market setup ...

    function testPhantomCollateral() public {
        // 1. Deploy fee token and create market with it as collateral
        feeToken = new FeeToken();
        // ... build market with feeToken as collateralParams[0].token ...
        // ... touchMarket(market) ...

        address attacker = makeAddr("attacker");
        uint256 assets = 1e18;
        feeToken.mint(attacker, assets);

        vm.startPrank(attacker);
        feeToken.approve(address(midnight), assets);
        midnight.supplyCollateral(market, 0, assets, attacker);
        vm.stopPrank();

        // ASSERTION 1: position records full assets
        assertEq(midnight.collateral(id, attacker, 0), assets);

        // ASSERTION 2: protocol only received 0.9e18
        assertEq(feeToken.balanceOf(address(midnight)), assets * 9 / 10);

        // ASSERTION 3: invariant violated — position > balance
        assertGt(midnight.collateral(id, attacker, 0), feeToken.balanceOf(address(midnight)));

        // ASSERTION 4: attacker can withdraw full assets, draining 0.1e18 from others
        vm.prank(attacker);
        midnight.withdrawCollateral(market, 0, assets, attacker, attacker);
        // Protocol balance goes to 0 (or negative if covered by others)
        // attacker received assets * 0.9 (fee applies on outbound transfer too, but net drain is confirmed)
    }
}

    function supplyCollateral(Market memory market, uint256 collateralIndex, uint256 assets, address onBehalf)
        external
    {
        require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
        bytes32 id = touchMarket(market);
        address collateralToken = market.collateralParams[collateralIndex].token;

        Position storage _position = position[id][onBehalf];
        uint256 oldCollateral = _position.collateral[collateralIndex];
        _position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);

        if (oldCollateral == 0 && assets > 0) {
            uint128 newCollateralBitmap = _position.collateralBitmap.setBit(collateralIndex);
            _position.collateralBitmap = newCollateralBitmap;
            require(
                UtilsLib.countBits(newCollateralBitmap) <= MAX_COLLATERALS_PER_BORROWER, TooManyActivatedCollaterals()
            );
        }

        emit EventsLib.SupplyCollateral(msg.sender, id, collateralToken, assets, onBehalf);

        SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
    }

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;
    }
