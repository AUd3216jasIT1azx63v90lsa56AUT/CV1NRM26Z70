### Title
Oracle price manipulation via seller callback bypasses `SellerIsLiquidatable` health check — (File: `src/Midnight.sol`)

### Summary
The `take` function commits all seller state updates (debt increase) to storage before firing the seller callback, then performs the `isHealthy` check only after the callback returns. Because `isHealthy` reads oracle prices live at call time and the seller callback receives the full `offer.market` struct (including every `CollateralParams.oracle` address), a seller callback can call `oracle.setPrice(inflated_price)` on a market-creator-supplied manipulable oracle, causing the post-callback health check to pass even though the seller was unhealthy before and after the take in economic terms.

### Finding Description

**Exact code path:**

Step 1 — Seller debt is written to storage before any callback: [1](#0-0) 

Step 2 — Seller callback fires, receiving `offer.market` (which carries every `CollateralParams.oracle` address): [2](#0-1) 

The `ISellCallback.onSell` signature confirms the callback receives the full `Market` struct: [3](#0-2) 

`CollateralParams.oracle` is a plain `address` field with no access-control enforcement: [4](#0-3) 

Step 3 — Liquidation lock is cleared, then `isHealthy` is called with a live oracle read: [5](#0-4) 

`isHealthy` calls `IOracle(collateralParam.oracle).price()` at the moment of the check — no pre-callback snapshot exists: [6](#0-5) 

**Root cause:** The health check is intentionally placed after the seller callback (to allow legitimate collateral supply during the callback), but the oracle price is read live with no snapshot taken before the callback window opens. Any oracle whose `price()` return value can be altered by the callback contract will produce an incorrect health result.

**Attacker-controlled inputs:**
- Market creator deploys a market whose `CollateralParams.oracle` is a contract with an unrestricted `setPrice` function (identical in structure to the test helper `test/helpers/Oracle.sol`, which allows anyone to call `setPrice`).
- Seller registers a sell callback contract that, inside `onSell`, calls `Oracle(market.collateralParams[i].oracle).setPrice(ORACLE_PRICE_SCALE * 1000)` and then returns `CALLBACK_SUCCESS`.

The existing test infrastructure already demonstrates this exact pattern — `ReentrantLiquidateBorrowCallback.onSell` calls `oracle.setPrice(healthyPrice / 2)` during a sell callback to test the liquidation lock: [7](#0-6) 

The proposed attack is the mirror image: inflate rather than deflate.

**Why existing checks fail:**
- The liquidation lock (line 444) only blocks re-entrant `liquidate` calls; it does not prevent oracle writes.
- The `SellerIsLiquidatable` check at line 476 reads the oracle after the callback has already written the inflated price, so it sees `maxDebt >= debt` and passes.
- There is no oracle allowlist, no price-snapshot mechanism, and no validation of oracle implementations at market creation.

### Impact Explanation
An economically unhealthy seller (debt > maxDebt at the true oracle price) can successfully call `take`, increasing their debt further, while the protocol records them as healthy. Buyers who filled the offer have extended credit to a borrower who was already undercollateralized and whose position is now even more so. The invariant "unhealthy positions remain liquidatable" is violated for the duration between the take and the next honest oracle update, and the seller's debt has grown beyond what their collateral can cover.

### Likelihood Explanation
Preconditions: (1) a market exists whose oracle allows external price writes — achievable by any market creator, who is listed as a valid unprivileged attacker; (2) the seller has a registered sell callback (standard borrow-via-callback pattern, already tested in the codebase); (3) the seller's position is unhealthy before the take. All three are reachable without any privileged role. The attack is repeatable on every `take` call in such a market.

### Recommendation
Snapshot all oracle prices for the seller's activated collaterals **before** the seller callback is invoked, and use those cached values inside the post-callback `isHealthy` check. Concretely, introduce an internal `isHealthyWithCachedPrices(prices[], id, seller)` variant, populate `prices[]` immediately after the state updates at line 414 and before the callback at line 458, and pass it to the check at line 476. This preserves the legitimate use case (supplying collateral during the callback changes storage balances, not oracle prices) while closing the manipulation window.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.34;

import "forge-std/Test.sol";
import {Midnight} from "src/Midnight.sol";
import {ISellCallback} from "src/interfaces/ICallbacks.sol";
import {Market, Offer, CollateralParams} from "src/interfaces/IMidnight.sol";
import {ORACLE_PRICE_SCALE, CALLBACK_SUCCESS} from "src/libraries/ConstantsLib.sol";

// Manipulable oracle — identical to test/helpers/Oracle.sol
contract ManipulableOracle {
    uint256 public price = ORACLE_PRICE_SCALE; // 1:1 initially
    function setPrice(uint256 p) external { price = p; }
}

// Malicious sell callback: inflates oracle price during onSell
contract OracleManipulatorCallback is ISellCallback {
    function onSell(
        bytes32, Market memory market, uint256, uint256, uint256,
        address, address, bytes memory
    ) external returns (bytes32) {
        // Inflate oracle price 1000x so maxDebt >> debt
        ManipulableOracle(market.collateralParams[0].oracle)
            .setPrice(ORACLE_PRICE_SCALE * 1000);
        return CALLBACK_SUCCESS;
    }
}

contract OracleManipulationTest is Test {
    function testUnhealthySellerTakesViaOracleManipulation() public {
        // 1. Deploy protocol and market with manipulable oracle
        // 2. Collateralize seller at exactly the LLTV boundary (healthy)
        // 3. Drop oracle price so seller becomes unhealthy
        //    (debt > maxDebt at real price)
        // 4. Deploy OracleManipulatorCallback; register as seller's callback
        // 5. Call midnight.take(..., takerCallback = address(manipulatorCallback))
        //    where the taker is the seller (sell offer path)
        // 6. Assert: take() does NOT revert (SellerIsLiquidatable not triggered)
        // 7. Assert: seller.debt increased (debt grew while unhealthy)
        // 8. Assert: isHealthy returns false at the real oracle price
        //    (confirming the position is actually unhealthy post-take)
    }
}
```

Expected assertions:
- `take` succeeds (no revert) despite seller being unhealthy at the real oracle price.
- `midnight.debtOf(id, seller)` is larger after the take.
- Calling `oracle.setPrice(ORACLE_PRICE_SCALE)` (restore real price) then `midnight.isHealthy(market, id, seller)` returns `false`, confirming the invariant was violated.
