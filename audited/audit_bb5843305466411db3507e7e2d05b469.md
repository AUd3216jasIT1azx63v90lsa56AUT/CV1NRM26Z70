### Title
Taker-Controlled `takerCallback` Sets `payer` to Arbitrary Third Party on Sell Offers - (`src/Midnight.sol`)

### Summary
In `Midnight.take`, when processing a sell offer (`offer.buy = false`), the `payer` address is derived directly from `buyerCallback`, which equals the taker-supplied `takerCallback`. There is no check that `takerCallback` is the taker themselves or authorized by the taker to pay. An attacker can set `takerCallback = victimContract`, causing `payer = victimContract` and all loan token transfers to be pulled from the victim rather than the attacker.

### Finding Description

The relevant code path in `Midnight.sol`:

```solidity
// Line 420-422
address buyerCallback = offer.buy ? offer.callback : takerCallback;   // sell: buyerCallback = takerCallback
address sellerCallback = offer.buy ? takerCallback : offer.callback;   // sell: sellerCallback = offer.callback
address payer = buyerCallback != address(0) ? buyerCallback : (offer.buy ? buyer : msg.sender);
``` [1](#0-0) 

For a sell offer (`offer.buy = false`):
- `buyerCallback = takerCallback` — fully attacker-controlled, no authorization check
- `payer = buyerCallback` if non-zero → `payer = victimContract`

The `onBuy` callback is then invoked on `victimContract`:

```solidity
// Lines 445-452
if (buyerCallback != address(0)) {
    require(
        IBuyCallback(buyerCallback).onBuy(...) == CALLBACK_SUCCESS,
        WrongBuyCallbackReturnValue()
    );
}
``` [2](#0-1) 

If `victimContract.onBuy` returns `CALLBACK_SUCCESS`, execution continues to the transfers:

```solidity
// Lines 455-456
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets);
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);
``` [3](#0-2) 

Both transfers pull from `payer = victimContract`. The maker's `offer.callback` is then invoked as `sellerCallback` (line 458–473), but the maker's callback is irrelevant to who pays — the payment has already been extracted from the victim. [4](#0-3) 

There is no check anywhere in `take` that `takerCallback` must equal `taker`, `msg.sender`, or any address authorized by the taker. The `EcrecoverRatifier.isRatified` only validates the maker's signature over `offer` fields; it does not constrain `takerCallback` at all. [5](#0-4) 

**Exploit flow:**
1. Maker signs a sell offer with `offer.callback = makerCallbackContract`, `offer.buy = false`.
2. Attacker calls `take(offer, ratifierData, units, attacker, receiver, victimContract, data)`.
3. `buyerCallback = victimContract`, `payer = victimContract`.
4. `victimContract.onBuy(...)` is called; it returns `CALLBACK_SUCCESS` (precondition: victim implements `IBuyCallback` and returns success, e.g. a general-purpose callback contract).
5. `safeTransferFrom(loanToken, victimContract, ...)` pulls `buyerAssets` from `victimContract` (precondition: victim has approved Midnight).
6. `makerCallbackContract.onSell(...)` is called — the maker's callback executes, but the victim paid.

### Impact Explanation
A victim contract that (a) implements `IBuyCallback` returning `CALLBACK_SUCCESS` and (b) has a token approval to Midnight can have its tokens drained by any attacker who takes a sell offer with `takerCallback = victimContract`. The victim never authorized this payment. This breaks the invariant that `payer` must be the party responsible for `buyerAssets` (i.e., the taker or a callback the taker authorized to pay on their behalf).

### Likelihood Explanation
Preconditions are realistic: any contract designed to act as a buy callback in Midnight (e.g., a flash-loan wrapper, a router, or another user's callback contract) will implement `IBuyCallback` and hold or have approval over loan tokens. The attacker needs no special privilege — only the ability to call `take` as a taker. The attack is repeatable as long as the victim contract retains its approval and callback interface.

### Recommendation
Enforce that `takerCallback`, when used as `payer`, is authorized to pay on behalf of the taker. The simplest fix is to require that `takerCallback` is either `msg.sender`, `taker`, or an address authorized by `taker` in `isAuthorized`. Alternatively, derive `payer` from `taker`/`msg.sender` unconditionally and only use `takerCallback` for the callback invocation, not for payment sourcing — mirroring how `repay` handles its `payer`:

```solidity
// repay (line 511): payer = callback != address(0) ? callback : msg.sender
// take should similarly require takerCallback == msg.sender or isAuthorized[taker][takerCallback]
require(takerCallback == address(0) || takerCallback == msg.sender || isAuthorized[taker][takerCallback], TakerCallbackUnauthorized());
``` [6](#0-5) 

### Proof of Concept

```solidity
// Foundry unit test
function test_payerConfusion_sellOffer() public {
    // Setup: victimContract implements IBuyCallback (returns CALLBACK_SUCCESS), 
    //        has approved Midnight for loanToken
    // maker signs sell offer: offer.buy=false, offer.callback=makerCallbackContract

    uint256 victimBalanceBefore = loanToken.balanceOf(address(victimContract));

    // Attacker calls take with takerCallback = victimContract
    vm.prank(attacker);
    midnight.take(offer, ratifierData, units, attacker, receiver, address(victimContract), "");

    uint256 victimBalanceAfter = loanToken.balanceOf(address(victimContract));

    // Assert: victim paid, not attacker
    assertLt(victimBalanceAfter, victimBalanceBefore, "victim tokens were pulled");
    assertEq(loanToken.balanceOf(attacker), attackerBalanceBefore, "attacker paid nothing");
    // Assert: payer in emitted Take event == victimContract
}
```

Expected assertion: `victimBalanceAfter < victimBalanceBefore` and attacker balance unchanged, confirming the victim was charged for the attacker's trade.

        require(msg.sender == MIDNIGHT, NotMidnight());

            abi.decode(ratifierData, (Signature, bytes32, uint256, bytes32[]));

        require(!isRootCanceled[offer.maker][root], RootCanceled());

        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, structHash));

        require(_signer != address(0), InvalidSignature());
