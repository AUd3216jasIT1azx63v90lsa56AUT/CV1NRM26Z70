### Title
Malicious Maker Can Permanently DoS Takers via Unbounded Gas Consumption in `offer.callback` — (File: src/Midnight.sol)

---

### Summary

In `Midnight.sol`'s `take()` function, the maker-controlled `offer.callback` address is invoked without any gas limit. A malicious maker can deploy a callback contract that runs an infinite loop or otherwise exhausts all forwarded gas, causing every taker's `take()` transaction against that offer to revert. This is the direct analog of the 1inch `notifyFillOrder` DoS: an attacker-controlled hook is called with unbounded gas, enabling permanent griefing of any taker or bot that attempts to fill the offer.

---

### Finding Description

**Root cause — no gas cap on maker callback invocation**

The `Offer` struct includes a maker-supplied `callback` field: [1](#0-0) 

Inside `take()`, the maker/taker callback routing is: [2](#0-1) 

When `offer.buy == true`, `buyerCallback = offer.callback` (the maker's address). It is then called with no gas limit: [3](#0-2) 

When `offer.buy == false`, `sellerCallback = offer.callback` (again the maker's address), called with no gas limit: [4](#0-3) 

Neither call site forwards a bounded gas stipend (e.g., `{gas: N}`). The full remaining gas of the transaction is available to the callback.

**Exploit path**

1. Attacker (maker) deploys `MaliciousCallback` implementing `IBuyCallback.onBuy` / `ISellCallback.onSell` with `while (true) {}`.
2. Attacker signs an `Offer` with `offer.callback = address(MaliciousCallback)` and a competitive price to attract takers.
3. Any taker or bot that calls `take()` against this offer will have the callback consume all remaining gas.
4. The transaction reverts with out-of-gas; the taker loses their gas cost and the offer is never fillable.

The `offer.callback` is part of the signed `Offer` struct, so the ratifier validates the whole struct including the malicious callback address — the signature check at line 356 passes normally: [5](#0-4) 

---

### Impact Explanation

Every `take()` call against the poisoned offer reverts. Takers and arbitrage bots waste gas on each attempt. A maker can post many such offers at attractive prices to systematically drain competing bots' ETH and degrade protocol usability. Because the offer is never consumed (`consumed` is only incremented before the callback), the attack is self-renewing — the same offer can grief an unlimited number of takers.

---

### Likelihood Explanation

Any permissionless maker can execute this with zero upfront capital. Deploying a looping contract costs ~50k gas once. The attacker then signs offers off-chain at no cost. Bots that rely on on-chain simulation may not catch this if the simulation gas limit differs from execution, or if the callback is conditionally malicious (e.g., behaves normally in `eth_call` but loops in real execution via `tx.origin` checks). Likelihood is **high** given zero barrier to entry.

---

### Recommendation

Forward a bounded gas stipend when invoking `offer.callback`, analogous to the 1inch recommendation. For example:

```solidity
// Instead of:
IBuyCallback(buyerCallback).onBuy{...}(...)

// Use:
IBuyCallback(buyerCallback).onBuy{gas: MAX_CALLBACK_GAS}(...)
```

Define `MAX_CALLBACK_GAS` as a protocol constant (e.g., 100_000 gas) sufficient for legitimate flash-loan / sourcing callbacks but insufficient for an infinite loop. Alternatively, document that `offer.callback` is fully trusted and takers must verify callback contracts before filling — but this is a weaker mitigation.

---

### Proof of Concept

```solidity
// MaliciousCallback.sol
contract MaliciousCallback {
    function onBuy(
        bytes32, Market memory, uint256, uint256, uint256, address, bytes memory
    ) external returns (bytes32) {
        while (true) {} // consumes all gas
    }
}

// Attack:
// 1. Deploy MaliciousCallback → address(mc)
// 2. Maker signs Offer { buy: true, callback: address(mc), tick: <competitive>, ... }
// 3. Taker calls midnight.take(offer, ratifierData, units, ...)
// 4. Execution reaches IBuyCallback(mc).onBuy(...) at Midnight.sol:448
// 5. Infinite loop exhausts all gas → entire take() reverts
// 6. Taker loses gas; offer.consumed unchanged; attack repeats indefinitely
