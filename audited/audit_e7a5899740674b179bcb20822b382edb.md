Audit Report

## Title
Authorized Signer Can Unilaterally Consume Authorizer's Nonce via Self-Deauthorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` increments the authorizer's nonce unconditionally before the signer check, and imposes no restriction preventing an authorized signer from naming themselves as `authorization.authorized` with `isAuthorized = false`. An authorized operator (Bob) can craft a self-deauthorization that atomically consumes Alice's current nonce in `EcrecoverAuthorizer`, permanently invalidating any pre-signed `Authorization` Alice has distributed at that nonce.

## Finding Description

**Root cause** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:

```solidity
// line 26 – nonce consumed before signer check; reverts only on mismatch, not on later auth failure
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// lines 33–36 – signer check: authorizer OR any address currently authorized by authorizer in Midnight
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// lines 46–47 – executes whatever the struct says; no guard on authorized == signer with isAuthorized = false
IMidnight(MIDNIGHT).setIsAuthorized(
    authorization.authorized, authorization.isAuthorized, authorization.authorizer
);
```

The nonce increment at line 26 is permanent once the transaction succeeds. The `isAuthorized` check at line 34 queries Midnight's state *before* the deauthorization executes at line 47 (TOCTOU). There is no guard preventing `authorization.authorized == signer` combined with `authorization.isAuthorized = false`.

**Preconditions (all normal operational states):**
1. Alice calls `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice)` — required to use `EcrecoverAuthorizer` at all.
2. Alice calls `midnight.setIsAuthorized(bob, true, alice)` — normal delegation.
3. Alice off-chain signs `Authorization{authorizer:alice, authorized:charlie, isAuthorized:true, nonce:0, deadline:T}` and distributes it to Charlie.

**Exploit flow:**
1. Bob constructs `Authorization{authorizer:alice, authorized:bob, isAuthorized:false, nonce:0, deadline:T2}` and signs it with his own private key.
2. Bob calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
3. Line 26: `0 == nonce[alice]++` → passes; `nonce[alice]` becomes `1`.
4. Lines 33–36: `ecrecover` returns Bob; `IMidnight.isAuthorized(alice, bob)` is `true` at this point (deauthorization has not yet executed) → passes.
5. Lines 46–47: `midnight.setIsAuthorized(bob, false, alice)` executes with `msg.sender = ecrecoverAuthorizer`, which Alice has authorized → Bob is deauthorized from Midnight.
6. Charlie submits Alice's pre-signed message (nonce=0): `require(0 == nonce[alice])` → `0 == 1` → **reverts with `InvalidNonce`**.

**Why existing checks fail:**
The signer check (lines 33–36) is satisfied because `isAuthorized[alice][bob]` is read before the deauthorization executes on line 47. The nonce increment is unconditional and permanent once the transaction succeeds. No guard exists on `authorization.authorized == signer` with `authorization.isAuthorized = false`.

## Impact Explanation
Alice's nonce in `EcrecoverAuthorizer` advances from N to N+1 without her consent. Any pre-signed `Authorization` she has distributed at nonce N — to Charlie, a relayer, or a smart contract — is permanently invalidated and cannot be submitted. Alice must re-sign at nonce N+1 and redistribute. If Alice is offline, unavailable, or the signed message was embedded in a time-sensitive workflow, the authorization is lost for that window. This constitutes unauthorized state corruption (nonce manipulation) and service disruption of the gasless authorization flow. The attack is repeatable each time Alice re-authorizes Bob.

## Likelihood Explanation
All three preconditions are normal operational states. Authorizing `EcrecoverAuthorizer` in Midnight is required to use the gasless flow at all. Authorizing Bob is a standard delegation. Having a pending pre-signed authorization is the core use case of `EcrecoverAuthorizer`. Bob requires no funds, no special protocol role beyond being authorized by Alice, and no external oracle. The attack is a single transaction executable by any authorized operator.

## Recommendation
Add a guard preventing a signer from submitting an authorization that deauthorizes themselves (or any address that would pass the signer check) when `isAuthorized = false`, or — more robustly — move the nonce increment to after all checks and the external call, or use a commit-reveal / check-effects-interactions pattern. The simplest targeted fix is to add:

```solidity
require(
    authorization.isAuthorized || signer != authorization.authorized,
    SelfDeauthorizationForbidden()
);
```

A more complete fix is to move the nonce increment to after the signer check so that a failed authorization check does not consume the nonce (though in this exploit the check passes, so the deeper fix is the guard above or restricting what an authorized signer can set `authorization.authorized` to).

## Proof of Concept

**Minimal manual steps:**
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. As Alice: `midnight.setIsAuthorized(ecrecoverAuthorizer, true, alice)`.
3. As Alice: `midnight.setIsAuthorized(bob, true, alice)`.
4. Alice off-chain signs `Authorization{authorizer:alice, authorized:charlie, isAuthorized:true, nonce:0, deadline:block.timestamp+1h}`.
5. As Bob: sign `Authorization{authorizer:alice, authorized:bob, isAuthorized:false, nonce:0, deadline:block.timestamp+1h}` with Bob's key.
6. As Bob: call `ecrecoverAuthorizer.setIsAuthorized(bobAuth, bobSig)` → succeeds; `nonce[alice]` = 1.
7. As Charlie: call `ecrecoverAuthorizer.setIsAuthorized(aliceAuth, aliceSig)` → reverts with `InvalidNonce`.

**Forge test plan:**
```solidity
function testNonceGriefByAuthorizedSigner() public {
    vm.prank(alice);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice);
    vm.prank(alice);
    midnight.setIsAuthorized(bob, true, alice);

    // Alice pre-signs for Charlie at nonce 0
    Authorization memory charlieAuth = Authorization(alice, charlie, true, 0, block.timestamp + 1 hours);
    bytes32 digest = ecrecoverAuthorizer.hashAuthorization(charlieAuth);
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(aliceKey, digest);

    // Bob self-deauthorizes at nonce 0
    Authorization memory bobAuth = Authorization(alice, bob, false, 0, block.timestamp + 1 hours);
    bytes32 digest2 = ecrecoverAuthorizer.hashAuthorization(bobAuth);
    (uint8 v2, bytes32 r2, bytes32 s2) = vm.sign(bobKey, digest2);
    vm.prank(bob);
    ecrecoverAuthorizer.setIsAuthorized(bobAuth, Signature(v2, r2, s2));

    // Charlie's message is now invalid
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(charlieAuth, Signature(v, r, s));
}
```
