### Title
Authorized Operator Can Unilaterally Consume Authorizer's Nonce via Self-Deauthorization - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that passes the delegated-signer check (`isAuthorized[authorizer][signer] == true`) to sign and submit an `Authorization` struct on behalf of the authorizer. Because there is no restriction preventing the signer from naming themselves as `authorization.authorized` with `isAuthorized = false`, an authorized operator (Bob) can craft and submit a self-deauthorization that atomically consumes the authorizer's (Alice's) current nonce, invalidating any pre-signed authorization Alice has already distributed at that nonce.

### Finding Description

**Exact code path** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:

```solidity
// line 26 – nonce consumed unconditionally on match
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// lines 31–36 – signer check: authorizer OR any address authorized by authorizer
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
require(signer != address(0), InvalidSignature());
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// line 46–47 – executes whatever the struct says
IMidnight(MIDNIGHT).setIsAuthorized(
    authorization.authorized, authorization.isAuthorized, authorization.authorizer
);
```

**Attacker-controlled inputs:**
- `authorization.authorizer = alice`
- `authorization.authorized = bob` (himself)
- `authorization.isAuthorized = false`
- `authorization.nonce = N` (Alice's current nonce)
- `authorization.deadline = T` (any future timestamp)
- `signature` = Bob's own ECDSA signature over the above struct

**Exploit flow:**
1. Alice calls `midnight.setIsAuthorized(bob, true, alice)` → `isAuthorized[alice][bob] = true`.
2. Alice off-chain signs `Authorization{authorizer:alice, authorized:charlie, isAuthorized:true, nonce:0, deadline:T}` and hands the signed message to Charlie for later submission.
3. Bob constructs `Authorization{authorizer:alice, authorized:bob, isAuthorized:false, nonce:0, deadline:T2}` and signs it with **Bob's own private key**.
4. Bob calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
5. Line 26: `0 == nonce[alice]++` → passes; `nonce[alice]` is now `1`.
6. Lines 33–36: `ecrecover` returns `bob`; `bob == alice` is false; `isAuthorized[alice][bob]` is `true` → passes.
7. Line 46–47: `midnight.setIsAuthorized(bob, false, alice)` executes → Bob is deauthorized.
8. Charlie submits Alice's pre-signed message (nonce=0): `0 == nonce[alice]` → `0 == 1` → **reverts with `InvalidNonce`**.

**Why existing checks fail:**
The signer check (lines 33–36) is satisfied because `isAuthorized[alice][bob]` is queried *before* the deauthorization executes (line 47). There is no guard preventing `signer == authorization.authorized` combined with `isAuthorized = false`. The nonce increment on line 26 is permanent once the transaction succeeds.

### Impact Explanation
Alice's nonce advances from N to N+1 without her consent. Any pre-signed `Authorization` she has distributed at nonce N — to Charlie, a relayer, or a smart contract — is permanently invalidated and cannot be submitted. Alice must re-sign at nonce N+1 and redistribute. If Alice is offline, unavailable, or the signed message was embedded in a time-sensitive workflow, the authorization is lost for that window.

### Likelihood Explanation
Preconditions: (1) Alice has authorized Bob (`isAuthorized[alice][bob] == true`), and (2) Alice has a pending pre-signed authorization at nonce N that has not yet been submitted. Both are normal operational states for any user relying on `EcrecoverAuthorizer` for gasless/delegated authorization. Bob needs no funds, no special role, and no external oracle. The attack is a single transaction, repeatable each time Alice re-authorizes Bob.

### Recommendation
Restrict the delegated-signer path so that a non-authorizer signer cannot deauthorize themselves. The minimal fix is to add a check that the signer is not simultaneously the `authorized` target being removed:

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
// Prevent a delegated signer from consuming the authorizer's nonce to self-deauthorize
require(
    signer == authorization.authorizer
        || !(signer == authorization.authorized && !authorization.isAuthorized),
    Unauthorized()
);
```

A cleaner architectural fix is to require that only `authorization.authorizer` themselves may sign through `EcrecoverAuthorizer` (removing the delegated-signer path entirely), since the delegation check creates a transitive nonce-consumption surface. Alternatively, key the nonce on `(authorizer, signer)` rather than `authorizer` alone, so a delegated signer's actions cannot advance the authorizer's nonce sequence.

### Proof of Concept

```solidity
// Foundry unit test
function testBobSelfDeauthorizationConsumesAliceNonce() public {
    // Setup: Alice authorizes EcrecoverAuthorizer and Bob
    vm.prank(alice);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice);
    vm.prank(alice);
    midnight.setIsAuthorized(bob, true, alice);

    // Alice pre-signs an authorization for Charlie at nonce 0
    Authorization memory aliceAuth = Authorization({
        authorizer: alice,
        authorized: charlie,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory aliceSig = signAuthorization(aliceAuth, alice);

    // Bob self-deauthorizes, consuming Alice's nonce 0
    Authorization memory bobAuth = Authorization({
        authorizer: alice,
        authorized: bob,
        isAuthorized: false,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory bobSig = signAuthorization(bobAuth, bob);

    vm.prank(bob);
    ecrecoverAuthorizer.setIsAuthorized(bobAuth, bobSig);

    // Assert: Alice's nonce advanced without her consent
    assertEq(ecrecoverAuthorizer.nonce(alice), 1);
    // Assert: Bob is deauthorized
    assertFalse(midnight.isAuthorized(alice, bob));

    // Assert: Alice's pre-signed message at nonce 0 is now invalid
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(aliceAuth, aliceSig);
}
```
