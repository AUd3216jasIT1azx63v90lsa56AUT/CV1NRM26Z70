### Title
`consumableUnits` Missing Tick Accessibility Check Enables Malicious Maker to Grief Bundler Takers via Silent `OutOfOffers` - (File: src/periphery/ConsumableUnitsLib.sol)

### Summary
`ConsumableUnitsLib.consumableUnits` returns a positive unit count for any offer with remaining capacity, without checking whether `offer.tick` is divisible by the market's `tickSpacing`. When the bundler uses this value to call `take()`, the core contract's `TickNotAccessible` revert is silently swallowed by the `catch {}` block, the offer is skipped, and if insufficient valid offers remain the bundler reverts with `OutOfOffers`. A malicious maker can deliberately sign and publish offers at inaccessible ticks to pollute an order book and reliably grief takers using `MidnightBundles`.

### Finding Description

**Root cause — missing check in `consumableUnits`:**

`consumableUnits` computes remaining capacity purely from `consumed`, `maxUnits`, and `maxAssets`:

```solidity
// src/periphery/ConsumableUnitsLib.sol:14-23
function consumableUnits(address midnight, bytes32 id, Offer memory offer) internal view returns (uint256) {
    uint256 consumed = IMidnight(midnight).consumed(offer.maker, offer.group);
    if (offer.maxUnits > 0) {
        return offer.maxUnits.zeroFloorSub(consumed);   // no tick check
    } else if (offer.buy) {
        return TakeAmountsLib.buyerAssetsToUnits(...);   // no tick check
    } else {
        return TakeAmountsLib.sellerAssetsToUnits(...);  // no tick check
    }
}
``` [1](#0-0) 

There is no `offer.tick % tickSpacing == 0` guard anywhere in this function or in `TakeAmountsLib`.

**Core contract enforces the check — but only inside `take()`:**

```solidity
// src/Midnight.sol:351
require(offer.tick % _marketState.tickSpacing == 0, TickNotAccessible());
``` [2](#0-1) 

This check fires unconditionally before any units-related logic, so even a call with `units=0` reverts for an inaccessible tick.

**Bundler swallows the revert:**

```solidity
// src/periphery/MidnightBundles.sol:74-85
uint256 unitsToTake = min(
    targetUnits - filledUnits,
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)  // returns non-zero
);
try IMidnight(MIDNIGHT).take(takes[i].offer, ..., unitsToTake, ...) returns (...) {
    filledUnits += unitsToTake;
    ...
} catch {}   // TickNotAccessible silently swallowed
``` [3](#0-2) 

After the loop, `filledUnits < targetUnits` triggers:

```solidity
require(filledUnits == targetUnits, OutOfOffers());
``` [4](#0-3) 

**Attacker-controlled inputs:**

Midnight offers are off-chain signed messages; there is no on-chain validation at offer creation time. Any maker can sign an `Offer` struct with an arbitrary `tick` value (e.g., `tick=2921` when `tickSpacing=4`). The signed offer is valid EIP-712 data and will pass ratifier checks; the only on-chain enforcement is the `TickNotAccessible` guard inside `take()`. A malicious maker publishes such offers to an off-chain order book. An automated taker system (aggregator, frontend) fetches them and includes them in the `takes[]` array passed to the bundler.

**Exploit flow:**
1. Market has `tickSpacing=4` (default).
2. Malicious maker signs `Offer{tick: 2921, maxUnits: type(uint256).max, ...}` and posts it to the order book.
3. Taker's system calls `buyWithUnitsTargetAndWithdrawCollateral` (or any bundler variant) with this offer in `takes[]`.
4. `consumableUnits` returns `type(uint256).max` (no tick check).
5. `unitsToTake` is non-zero.
6. `take()` reverts with `TickNotAccessible`.
7. `catch {}` swallows the revert; `filledUnits` is not incremented.
8. If no other offers cover `targetUnits`, the bundler reverts with `OutOfOffers`.

The same pattern applies to all four bundler entry-points (`buyWithUnitsTarget`, `supplyCollateralAndSellWithUnitsTarget`, `buyWithAssetsTarget`, `supplyCollateralAndSellWithAssetsTarget`). [5](#0-4) [6](#0-5) 

### Impact Explanation
A malicious maker can reliably cause any taker's bundler transaction to revert with `OutOfOffers` by polluting the order book with inaccessible-tick offers. The taker loses gas and fails to complete their intended borrow or lend operation. Because the bundler is documented as the intended integration path for takers, this constitutes concrete user loss through normal protocol usage.

### Likelihood Explanation
**Preconditions:** (1) A market exists with `tickSpacing > 1` (all markets start at `DEFAULT_TICK_SPACING = 4`). (2) A malicious maker signs and publishes at least one offer with `tick % tickSpacing != 0`. (3) A taker uses a bundler entry-point with an `takes[]` array that includes the poisoned offer (e.g., via an aggregator or frontend that fetches from an open order book).

All three preconditions are trivially satisfiable by any unprivileged actor. The attack is repeatable at zero cost beyond signing a message, requires no on-chain state, and is not detectable by `consumableUnits` as currently written.

### Recommendation
Add a tick accessibility guard at the top of `consumableUnits`:

```solidity
function consumableUnits(address midnight, bytes32 id, Offer memory offer) internal view returns (uint256) {
    (, , , , , , , , , , , , uint8 spacing) = IMidnight(midnight).marketState(id);
    if (offer.tick % spacing != 0) return 0;
    ...
}
```

Additionally, add a zero-units guard in every bundler loop to avoid calling `take()` when `unitsToTake == 0` (which would still revert for an inaccessible tick even after the above fix):

```solidity
if (unitsToTake == 0) continue;
```

Both fixes are required: the `consumableUnits` fix makes the pre-filter accurate, and the `continue` guard prevents a wasted `take()` call when the pre-filter returns 0 for any reason.

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import {BaseTest} from "./BaseTest.sol";
import {ConsumableUnitsLib} from "../src/periphery/ConsumableUnitsLib.sol";
import {IMidnight, IMidnightBundles} from "../src/interfaces/IMidnight.sol";
import {Take} from "../src/periphery/MidnightBundles.sol";

contract TickAccessibilityBundlerTest is BaseTest {
    // Market with DEFAULT_TICK_SPACING = 4.
    // Inaccessible tick: 2921 (2921 % 4 == 1 != 0).

    function testConsumableUnitsNonZeroForInaccessibleTick() public {
        bytes32 id = midnight.touchMarket(market);
        Offer memory badOffer = _makeOffer(2921); // tick not divisible by 4
        badOffer.maxUnits = 1000;

        uint256 units = ConsumableUnitsLib.consumableUnits(address(midnight), id, badOffer);
        // ASSERTION 1: consumableUnits returns non-zero despite inaccessible tick
        assertGt(units, 0, "consumableUnits should return 0 for inaccessible tick but does not");
    }

    function testBundlerOutOfOffersViaPoisonedOffer() public {
        bytes32 id = midnight.touchMarket(market);
        uint256 targetUnits = 100;

        // Only offer in the takes[] array has an inaccessible tick.
        Offer memory badOffer = _makeOffer(2921);
        badOffer.maxUnits = type(uint256).max;
        deal(address(loanToken), lender, type(uint128).max);
        collateralize(market, borrower, targetUnits);

        Take[] memory takes = new Take[](1);
        takes[0] = Take({offer: badOffer, units: targetUnits, ratifierData: hex""});

        vm.prank(borrower);
        // ASSERTION 2: bundler reverts with OutOfOffers because take() reverts
        // with TickNotAccessible (caught by catch{}) and filledUnits stays 0.
        vm.expectRevert(IMidnightBundles.OutOfOffers.selector);
        midnightBundles.buyWithUnitsTargetAndWithdrawCollateral(
            targetUnits, type(uint256).max, borrower,
            emptyPermit(), takes, new CollateralWithdrawal[](0),
            address(0), 0, address(0)
        );
    }
}
```

**Expected assertions:**
- `consumableUnits` returns `> 0` for `tick=2921` with `tickSpacing=4` (proves the missing check).
- The bundler call reverts with `OutOfOffers` (proves the end-to-end griefing path).
- Replacing the poisoned offer with `tick=2920` (divisible by 4) causes the bundler to succeed (proves the tick is the sole cause).
