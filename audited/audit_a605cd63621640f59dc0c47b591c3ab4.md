Audit Report

## Title
Authorized delegate can front-run and consume victim's EcrecoverAuthorizer nonce with arbitrary payload - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts signatures from any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for, not only the authorizer's own key. Because the nonce is shared across the authorizer and all their Midnight-authorized delegates, a malicious delegate can front-run a victim's pending `setIsAuthorized` transaction, burn nonce N with an attacker-chosen payload, and both invalidate the victim's signature and silently substitute a different `authorized` address in the victim's name.

## Finding Description
In `EcrecoverAuthorizer.setIsAuthorized` (lines 24–48 of `src/periphery/EcrecoverAuthorizer.sol`):

- Line 26 increments `nonce[authorization.authorizer]` unconditionally before the signer is verified.
- Lines 33–36 accept `signer == authorization.authorizer` **or** `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)`.

These two checks together mean any address the victim has previously authorized in Midnight (for any purpose — taking, withdrawing, repaying, etc.) can construct a fully attacker-controlled `Authorization{authorizer: victim, authorized: <any>, isAuthorized: <any>, nonce: N, deadline: <any valid>}`, sign it with their own key, and call `setIsAuthorized`. The nonce check passes (N matches current nonce), the nonce is incremented to N+1, and the signer check passes because `isAuthorized(victim, attacker) == true`. The victim's pending signature over nonce N now fails with `InvalidNonce`. Additionally, line 47 executes `IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer)` using the attacker-supplied `authorized` field, silently changing who gets authorized in the victim's Midnight mapping. [1](#0-0) 

The `Authorization` struct confirms all fields — including `authorized` and `isAuthorized` — are fully attacker-controlled inputs. [2](#0-1) 

## Impact Explanation
Two concrete impacts:

1. **DoS on EcrecoverAuthorizer flow**: The victim's signed-and-broadcast `setIsAuthorized` transaction is permanently invalidated. The attacker can repeat this for every subsequent nonce (N+1, N+2, …), permanently blocking the victim from using the EcrecoverAuthorizer.
2. **Unauthorized state change / gate bypass**: The attacker substitutes their own `authorized` address in the victim's Midnight `isAuthorized` mapping. This is a concrete privilege escalation: the attacker can grant or revoke authorization for arbitrary addresses in the victim's name, bypassing any gate logic that depends on `isAuthorized`. [3](#0-2) 

## Likelihood Explanation
Preconditions: (1) the attacker has previously been authorized by the victim in Midnight for any reason (a routine pattern for operators and bundlers, as shown in `AuthorizationTest.sol` lines 100–103 and 237–239); (2) the victim has a pending `EcrecoverAuthorizer.setIsAuthorized` transaction visible in the mempool. Both conditions are normal in protocol usage. The attack requires no privileged protocol access — only a standard user-level authorization relationship. It is repeatable indefinitely. [4](#0-3) [5](#0-4) 

## Recommendation
Move the nonce increment to **after** the signer check so that a failed authorization attempt does not consume the nonce. More importantly, restrict the signer check in `EcrecoverAuthorizer` to only accept `signer == authorization.authorizer`. The purpose of the EcrecoverAuthorizer is to allow off-chain signatures from the authorizer themselves to be submitted on-chain; delegated submission should not be permitted here, or if it is, the delegate must sign the exact same struct the authorizer signed (which is impossible to enforce without a separate commitment scheme). A minimal fix:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegate submission is intentionally desired, the nonce must be scoped per `(authorizer, signer)` pair, not per `authorizer` alone, to prevent cross-delegate nonce interference.

## Proof of Concept
1. Victim (`V`) has previously called `midnight.setIsAuthorized(attacker, true, V)` for any operational reason.
2. Victim signs `Authorization{authorizer: V, authorized: A, isAuthorized: true, nonce: 0, deadline: T}` and broadcasts the `setIsAuthorized` call.
3. Attacker observes the mempool, constructs `Authorization{authorizer: V, authorized: B, isAuthorized: false, nonce: 0, deadline: T}`, signs it with attacker's key, and submits with higher gas.
4. Attacker's transaction lands first: nonce 0 is consumed, `midnight.isAuthorized(V, B)` is set to `false`.
5. Victim's transaction reverts with `InvalidNonce`.
6. Repeat for nonce 1, 2, … to permanently block the victim.

Minimal Foundry test: deploy `EcrecoverAuthorizer`, have `victim` authorize `attacker` in Midnight, have `attacker` call `setIsAuthorized` with `authorizer = victim, nonce = 0` signed by attacker, then assert victim's call with the same nonce reverts with `InvalidNonce` and that `midnight.isAuthorized(victim, B)` reflects the attacker-chosen value.     function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
