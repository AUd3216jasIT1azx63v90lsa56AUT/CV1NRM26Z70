### Title
Unbounded `offer.maxAssets` causes `mulDivUp` overflow in `buyerAssetsToUnits`, DoS-ing all bundler calls that include the offer - (`File: src/periphery/ConsumableUnitsLib.sol`)

### Summary

`ConsumableUnitsLib.consumableUnits` passes `offer.maxAssets.zeroFloorSub(consumed)` directly into `TakeAmountsLib.buyerAssetsToUnits`, which calls `mulDivUp(targetBuyerAssets, WAD, buyerPrice)`. Because `offer.maxAssets` is an unbounded `uint256` and `UtilsLib.mulDivUp` uses plain checked Solidity arithmetic (`x * y + (d - 1)`), any offer with `maxAssets > type(uint256).max / WAD` causes an arithmetic overflow revert. All four bundler functions in `MidnightBundles.sol` explicitly document "Reverts if ConsumableUnitsLib reverts," and the `consumableUnits` call is outside the `try/catch` block, so the entire bundler transaction reverts.

### Finding Description

**Code path:**

`MidnightBundles.supplyCollateralAndSellWithUnitsTarget` (and three other bundler functions) → `ConsumableUnitsLib.consumableUnits` → `TakeAmountsLib.buyerAssetsToUnits` → `UtilsLib.mulDivUp`

**Root cause — `UtilsLib.mulDivUp` uses unchecked-style plain multiplication in checked Solidity 0.8:**

```solidity
// src/libraries/UtilsLib.sol:34-36
function mulDivUp(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y + (d - 1)) / d;   // overflows if x * y > type(uint256).max
}
``` [1](#0-0) 

**`offer.maxAssets` is `uint256` with no upper-bound check anywhere:**

```solidity
// src/interfaces/IMidnight.sol:35
uint256 maxAssets; // buyerAssets if offer.buy else sellerAssets
``` [2](#0-1) 

`Midnight.take()` only checks `atMostOneNonZero(offer.maxAssets, offer.maxUnits)` — no magnitude bound. [3](#0-2) 

**`consumableUnits` passes the raw remaining capacity into `buyerAssetsToUnits`:**

```solidity
// src/periphery/ConsumableUnitsLib.sol:18-19
} else if (offer.buy) {
    return TakeAmountsLib.buyerAssetsToUnits(midnight, id, offer, offer.maxAssets.zeroFloorSub(consumed));
``` [4](#0-3) 

**`buyerAssetsToUnits` calls `mulDivUp(targetBuyerAssets, WAD, buyerPrice)`:**

```solidity
// src/periphery/TakeAmountsLib.sol:29
return offer.buy ? targetBuyerAssets.mulDivUp(WAD, buyerPrice) : ...
``` [5](#0-4) 

The `require(buyerPrice <= WAD)` guard on line 28 only bounds `d`, not `x`. It does not prevent `x * WAD` from overflowing.

**The `consumableUnits` call is NOT inside the bundler's `try/catch`:**

```solidity
// src/periphery/MidnightBundles.sol:147-160
uint256 unitsToTake = min(
    targetUnits - filledUnits,
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)  // ← reverts here
);
try IMidnight(MIDNIGHT).take(...) { ... } catch {}   // ← never reached
``` [6](#0-5) 

**Exploit flow:**

1. Attacker (maker) creates a buy offer: `offer.buy = true`, `offer.maxUnits = 0`, `offer.maxAssets = type(uint256).max`, any valid ratifier.
2. Victim taker calls any bundler function with this offer in the `takes[]` array.
3. Bundler computes `consumableUnits` → `buyerAssetsToUnits(..., type(uint256).max)` → `mulDivUp(type(uint256).max, 1e18, buyerPrice)` → `type(uint256).max * 1e18` overflows → revert.
4. Entire bundler transaction reverts.

**Why existing checks fail:**

- `atMostOneNonZero` only checks mutual exclusivity, not magnitude.
- `require(buyerPrice <= WAD)` bounds the denominator, not the multiplicand.
- The Certora `NoMultiplicationOverflow.spec` covers only `Midnight.sol` core functions, not the periphery library.
- The bundler's `try/catch` wraps only `IMidnight.take()`, not `consumableUnits`.

### Impact Explanation

Any bundler call (`supplyCollateralAndSellWithUnitsTarget`, `buyWithAssetsTargetAndWithdrawCollateral`, `sellWithAssetsTargetAndSupplyCollateral`, `buyWithUnitsTargetAndWithdrawCollateral`) that includes a buy offer with `maxAssets > type(uint256).max / WAD` (≈ 1.157 × 10⁵⁹) in its `takes[]` array reverts entirely. A malicious maker can publish such an offer to permanently DoS any bundler-based taker who attempts to fill it, without any cost beyond the gas to create the offer.

### Likelihood Explanation

**Preconditions:** Attacker must be a maker (unprivileged). Creating an offer requires no special permission. Setting `maxAssets = type(uint256).max` is a valid `uint256` value accepted by `take()`. The overflow threshold (`> 1.157e59`) is far below `type(uint256).max` (`≈ 1.157e77`), so any value above `~1.16e59` triggers it. The attack is repeatable and free to sustain (no on-chain state change needed after offer creation).

### Recommendation

Cap `targetBuyerAssets` before passing it to `mulDivUp`, or add an overflow-safe guard in `buyerAssetsToUnits`:

```solidity
// In TakeAmountsLib.buyerAssetsToUnits, before the mulDivUp call:
if (targetBuyerAssets > type(uint256).max / WAD) return type(uint256).max;
return offer.buy
    ? targetBuyerAssets.mulDivUp(WAD, buyerPrice)
    : targetBuyerAssets.mulDivDown(WAD, buyerPrice);
```

Alternatively, enforce `offer.maxAssets <= type(uint128).max` in `Midnight.take()` (consistent with how `consumed` increments are bounded by `uint128` position values), or wrap the `consumableUnits` call in the bundler with a `try/catch` and treat a revert as zero consumable units (skip the offer).

### Proof of Concept

```solidity
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {ConsumableUnitsLib} from "src/periphery/ConsumableUnitsLib.sol";
import {TakeAmountsLib} from "src/periphery/TakeAmountsLib.sol";
import {Offer} from "src/interfaces/IMidnight.sol";

contract OverflowDoSTest is Test {
    // Fuzz: assert consumableUnits never reverts for any (maxAssets, consumed, tick) triple.
    function testFuzz_consumableUnitsNeverOverflows(
        uint256 maxAssets,
        uint256 consumed,
        uint256 tick
    ) public {
        // Bound tick to valid range so tickToPrice doesn't revert for unrelated reasons.
        tick = bound(tick, 0, MAX_TICK);
        consumed = bound(consumed, 0, maxAssets);

        Offer memory offer;
        offer.buy = true;
        offer.maxUnits = 0;
        offer.maxAssets = maxAssets;  // attacker sets this to type(uint256).max
        offer.tick = tick;
        // ... set up market, ratifier, etc.

        // This should NOT revert — but it DOES when maxAssets > type(uint256).max / WAD
        // Expected assertion failure: consumableUnits reverts with arithmetic overflow
        // when maxAssets = type(uint256).max and consumed = 0.
        uint256 result = ConsumableUnitsLib.consumableUnits(address(midnight), id, offer);
        // assert result is a valid uint256 (no overflow)
    }

    // Unit test: concrete overflow at boundary.
    function test_unit_overflowAtMaxAssets() public {
        Offer memory offer = _makeBuyOffer(type(uint256).max, validTick);
        // consumableUnits must revert with arithmetic overflow:
        vm.expectRevert(stdError.arithmeticError);
        ConsumableUnitsLib.consumableUnits(address(midnight), id, offer);
    }

    // Unit test: bundler reverts when malicious offer is in takes[].
    function test_unit_bundlerDoSWithMaliciousOffer() public {
        Take[] memory takes = new Take[](1);
        takes[0].offer = _makeBuyOffer(type(uint256).max, validTick);
        takes[0].units = 1;
        // Bundler call must revert entirely (not just skip the offer):
        vm.expectRevert(stdError.arithmeticError);
        bundler.supplyCollateralAndSellWithUnitsTarget(
            1, 0, taker, receiver, new CollateralSupply[](0), takes, 0, address(0)
        );
    }
}
```

**Expected assertions:**
- `test_unit_overflowAtMaxAssets`: reverts with `Arithmetic over/underflow` when `maxAssets = type(uint256).max`.
- `test_unit_bundlerDoSWithMaliciousOffer`: entire bundler call reverts (not silently skipped).
- `testFuzz_consumableUnitsNeverOverflows`: fuzzer finds a counterexample at `maxAssets > type(uint256).max / 1e18`.
