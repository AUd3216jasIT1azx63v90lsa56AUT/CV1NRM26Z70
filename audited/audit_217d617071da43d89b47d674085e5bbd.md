### Title
Authorized operator can consume victim's EcrecoverAuthorizer nonce via no-op Authorization, permanently invalidating victim's pre-signed delegation - (File: src/periphery/EcrecoverAuthorizer.sol)

### Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that is already authorized by the victim in Midnight to sign and submit an `Authorization` struct on the victim's behalf. Because the contract imposes no requirement that the submitted `Authorization` produces a meaningful state change, an authorized operator can craft and submit `Authorization{authorizer=victim, authorized=alreadyUnauthorizedAddress, isAuthorized=false, nonce=N}`, consuming nonce `N` while leaving `isAuthorized` unchanged. Any pre-signed `Authorization{nonce=N}` the victim intended to submit is then permanently invalidated.

### Finding Description
The full execution path in `EcrecoverAuthorizer.setIsAuthorized`: [1](#0-0) 

Line 26 unconditionally increments `nonce[authorization.authorizer]` on every successful call, regardless of whether the resulting `Midnight.setIsAuthorized` call changes any state. [2](#0-1) 

The signer check passes if `signer == authorization.authorizer` **or** `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)`. An attacker who is already authorized by the victim in Midnight satisfies the second branch with their own key — they do not need the victim's private key. [3](#0-2) 

The downstream call `Midnight.setIsAuthorized(alreadyUnauthorizedAddress, false, victim)` is explicitly a no-op: the Midnight contract documents "No-ops are allowed" and its implementation simply assigns the mapping value without checking the prior state. [4](#0-3) 

**Exploit flow:**
1. Victim (`V`) has authorized attacker (`A`) in Midnight: `isAuthorized[V][A] = true`.
2. `V` pre-signs off-chain: `Authorization{authorizer=V, authorized=criticalContract, isAuthorized=true, nonce=N}`.
3. `A` crafts: `Authorization{authorizer=V, authorized=someNeverAuthorizedAddr, isAuthorized=false, nonce=N, deadline=future}` and signs it with `A`'s own key.
4. `A` calls `ecrecoverAuthorizer.setIsAuthorized(noOpAuth, aSig)`.
5. Checks pass: deadline valid, `nonce[V]==N` (incremented to N+1), `ecrecover` returns `A`, `isAuthorized[V][A]==true`.
6. `Midnight.setIsAuthorized(someNeverAuthorizedAddr, false, V)` executes as a no-op.
7. `nonce[V]` is now `N+1`. `V`'s pre-signed `Authorization{nonce=N}` now reverts with `InvalidNonce`.

No existing check prevents this. The Certora spec `EcrecoverAuthorizer.spec` only verifies that a *successful* call increments the signer's nonce and that expired/wrong/reused nonces revert — it does not verify that nonce consumption requires a meaningful state change. [5](#0-4) 

### Impact Explanation
Victim's pre-signed `Authorization{nonce=N}` (intended to grant a critical contract) is permanently invalidated. The victim must produce a new signature over nonce `N+1`. If the victim cannot re-sign (e.g., hardware wallet unavailable, pre-signed batch, or the authorizer is a contract that pre-committed to nonce `N`), their ability to delegate via `EcrecoverAuthorizer` is frozen until a new signature is produced. The attack is repeatable: the attacker can consume every subsequent nonce the victim pre-signs, as long as the attacker remains authorized.

### Likelihood Explanation
**Preconditions:**
- Victim must have authorized the attacker in Midnight (`isAuthorized[victim][attacker] = true`). This is a normal, expected protocol interaction (e.g., authorizing a relayer, smart contract wallet, or protocol adapter).
- Victim must have a pre-signed `Authorization` in flight (e.g., submitted to a mempool, stored in a UI, or shared with a relayer).

Both conditions are routine. The attack requires no special privileges beyond being an authorized operator, is cheap (single transaction), and is repeatable indefinitely.

### Recommendation
Add a check in `EcrecoverAuthorizer.setIsAuthorized` that the submitted `Authorization` produces a real state change before consuming the nonce:

```solidity
bool current = IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, authorization.authorized);
require(current != authorization.isAuthorized, NoOpAuthorization());
```

This check must be placed **before** the nonce increment (line 26) to prevent no-op submissions from consuming nonces. Alternatively, restrict the signer to `authorization.authorizer` only (removing the delegated-signer path), which eliminates the third-party submission vector entirely.

### Proof of Concept
```solidity
// Foundry unit test
function testNoOpNonceGrief() public {
    // Setup: victim authorizes attacker in Midnight
    vm.prank(victim);
    midnight.setIsAuthorized(attacker, true, victim);

    // Victim pre-signs Authorization{nonce=0} to grant criticalContract
    Authorization memory victimAuth = Authorization({
        authorizer: victim,
        authorized: criticalContract,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory victimSig = signAuthorization(victimAuth, victimPrivKey);

    // Attacker crafts no-op: set already-false address to false, nonce=0
    Authorization memory noOpAuth = Authorization({
        authorizer: victim,
        authorized: address(0xdead), // never authorized
        isAuthorized: false,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    Signature memory attackerSig = signAuthorization(noOpAuth, attackerPrivKey);

    // Attacker submits no-op — succeeds, consumes nonce 0
    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(noOpAuth, attackerSig);

    // Assert: nonce is now 1
    assertEq(ecrecoverAuthorizer.nonce(victim), 1);

    // Assert: victim's original Authorization{nonce=0} now reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);

    // Assert: criticalContract was never granted
    assertEq(midnight.isAuthorized(victim, criticalContract), false);
}
```

**Expected assertions:** `nonce[victim] == 1` after no-op submission; victim's original `Authorization{nonce=0}` reverts with `InvalidNonce`; `isAuthorized[victim][criticalContract] == false`.

rule effects(env e, EcrecoverAuthorizer.Authorization authorization, EcrecoverAuthorizer.Signature signature, address other) {
    require other != authorization.authorizer;
    uint256 nonceBefore = nonce(authorization.authorizer);
