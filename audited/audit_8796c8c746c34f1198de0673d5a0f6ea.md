### Title
Unbounded `canLiquidate` External Call Enables OOG Gas Griefing to Permanently Block Liquidations - (File: src/Midnight.sol)

### Summary
Any unprivileged actor can deploy a market with an arbitrary `liquidatorGate` address because the `Market` struct accepts any `address liquidatorGate` with no validation. The `liquidate` function forwards all remaining gas to `ILiquidatorGate(market.liquidatorGate).canLiquidate(msg.sender)` with no gas cap, so a malicious gate whose `canLiquidate` iterates over a caller-grown list can be made to consume enough gas to revert every liquidation attempt OOG, permanently freezing undercollateralized debt.

### Finding Description

**Code path:**

`liquidate` at [1](#0-0)  performs:

```solidity
require(
    market.liquidatorGate == address(0) || ILiquidatorGate(market.liquidatorGate).canLiquidate(msg.sender),
    LiquidatorGatedFromLiquidating()
);
```

This external call carries no `{gas: N}` limit. The `Market` struct definition at [2](#0-1)  places `liquidatorGate` as a plain `address` field with no whitelist, no code-size check, and no interface validation. `touchMarket` is callable by anyone and accepts any market parameters, so any user can register a market whose `liquidatorGate` is an attacker-controlled contract.

**Attacker-controlled inputs:**
- `market.liquidatorGate` — set to a `GrowingListGate` contract the attacker deployed.
- The gate's internal approved-liquidator array — grown by the attacker at will after market creation and after borrower positions are opened.

**Exploit flow:**
1. Attacker deploys `GrowingListGate` with a `canLiquidate(address)` that linearly scans an `address[]` storage array.
2. Attacker calls `touchMarket(market)` with `liquidatorGate = address(GrowingListGate)` to register the market. [3](#0-2) 
3. A borrower (attacker or victim) opens a position in this market (supplies collateral, borrows).
4. Attacker repeatedly calls a public `addApproved(address)` function on `GrowingListGate`, growing the array until `canLiquidate` consumes ~30 M gas per call.
5. Collateral price drops; borrower becomes unhealthy.
6. Any liquidator calls `liquidate(...)`. Execution reaches line 598, forwards all remaining gas to `canLiquidate`, which exhausts it before the bitmap health loop at line 607 is ever reached. The transaction reverts OOG.
7. No liquidator can ever succeed; bad debt accumulates and lenders are permanently impaired.

**Why existing checks fail:**
- The `market.liquidatorGate == address(0)` short-circuit only skips the call when the gate is the zero address; it does nothing for a non-zero malicious gate. [1](#0-0) 
- `ILiquidatorGate` is a one-function interface with no gas or complexity constraints. [4](#0-3) 
- There is no registry, whitelist, or code-hash check on `liquidatorGate` anywhere in `Midnight.sol`.
- The `NotLiquidatable` check at line 620 is never reached because the OOG happens at line 598. [5](#0-4) 

### Impact Explanation
Unhealthy positions in any market whose `liquidatorGate` is a gas-exhausting contract cannot be liquidated. The invariant "unhealthy positions remain liquidatable" is broken. Lenders in that market suffer permanent bad debt with no recovery path, constituting a fund freeze.

### Likelihood Explanation
**Preconditions:** Attacker must create the market and either be the borrower or convince a borrower to use it. The attacker needs no special privilege — `touchMarket` is permissionless. Growing the list costs only calldata gas per entry. The attack is repeatable: even if the list is partially trimmed, the attacker can re-grow it. The only social precondition is that someone borrows in the attacker's market, which is realistic if the market parameters (lltv, oracle, maturity) appear legitimate.

### Recommendation
Cap the gas forwarded to `canLiquidate` to a fixed, conservative budget (e.g., 100 000 gas):

```solidity
require(
    market.liquidatorGate == address(0)
        || ILiquidatorGate(market.liquidatorGate).canLiquidate{gas: 100_000}(msg.sender),
    LiquidatorGatedFromLiquidating()
);
```

Alternatively, enforce that `liquidatorGate` must be set at market creation time and cannot be changed, and document a maximum acceptable complexity for conforming gate implementations. A stricter fix is to require `liquidatorGate` to be drawn from a protocol-approved registry.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "forge-std/Test.sol";
import {IMidnight, Market, CollateralParams} from "src/interfaces/IMidnight.sol";
import {ILiquidatorGate} from "src/interfaces/IGate.sol";

contract GrowingListGate is ILiquidatorGate {
    address[] public approved;
    function addApproved(address a) external { approved.push(a); }
    // Linear scan — O(n) SLOAD per call
    function canLiquidate(address who) external view returns (bool) {
        for (uint256 i; i < approved.length; i++) {
            if (approved[i] == who) return true;
        }
        return false;
    }
}

contract GasGriefLiquidationTest is Test {
    function testOOGLiquidation() public {
        GrowingListGate gate = new GrowingListGate();

        // 1. Build market with malicious gate
        Market memory market = /* ... standard params ... */;
        market.liquidatorGate = address(gate);

        // 2. touchMarket, supply collateral, borrow (standard setup)
        // ...

        // 3. Grow the list — add N dummy addresses (not the liquidator)
        for (uint256 i; i < 50_000; i++) {
            gate.addApproved(address(uint160(i + 1)));
        }

        // 4. Make position unhealthy (oracle price drop)
        // oracle.setPrice(ORACLE_PRICE_SCALE / 2);

        // 5. Assert liquidate reverts OOG
        vm.expectRevert(); // OOG
        midnight.liquidate{gas: 500_000}(
            market, 0, 1, 0, borrower, false, address(this), address(0), ""
        );

        // 6. Assert bad debt is not cleared — position still has debt
        (, , , , uint128 debt, ) = midnight.position(id, borrower);
        assertGt(debt, 0, "debt must remain: liquidation was blocked");
    }
}
```

**Expected assertions:** `liquidate` reverts (OOG) before reaching the bitmap loop at line 607; `position.debt > 0` after the attempted liquidation confirms the invariant violation.
