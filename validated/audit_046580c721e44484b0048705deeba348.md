Audit Report

## Title
Callback-Driven Bad Debt Over-Realization via `supplyCollateral` During `onLiquidate` - (File: `src/Midnight.sol`)

## Summary
In `liquidate`, bad debt is fully committed to state (position debt zeroed, `lossFactor` and `totalUnits` updated) before the `onLiquidate` callback fires. Because `liquidate` never sets the `LIQUIDATION_LOCK_SLOT` transient storage before invoking the callback, and `supplyCollateral` has no guard against mid-liquidation calls, a borrower-controlled callback can supply collateral on behalf of the borrower after bad debt is already socialized. The borrower then withdraws that collateral (debt is now 0, so `isHealthy` returns true unconditionally), recovering their assets at zero net cost while lenders permanently absorb the full debt as bad debt.

## Finding Description

**Root cause — bad debt committed before callback, no lock set:**

`liquidate` snapshots `_position.collateralBitmap` at line 606 and, when it is 0, leaves `maxDebt = 0` and `badDebt = originalDebt`. The liquidatability check `originalDebt > maxDebt` passes trivially. Bad debt is then fully committed to state: [1](#0-0) 

Only after this does the callback fire: [2](#0-1) 

The `LIQUIDATION_LOCK_SLOT` transient-storage lock is set exclusively inside `take` (line 444) and is never written in `liquidate`: [3](#0-2) 

`supplyCollateral` has no reentrancy guard and no check against the liquidation lock: [4](#0-3) 

`isHealthy` returns `true` unconditionally when `debt == 0`, because the collateral-valuation loop is gated on `debt > 0`: [5](#0-4) 

**Exploit flow:**

1. **Setup:** Borrower reaches state `debt = D > 0`, `collateralBitmap = 0` (natural after a prior liquidation that seized all collateral but left residual debt). Borrower holds X collateral tokens externally and calls `setIsAuthorized(callbackContract, true, borrower)`.
2. **Trigger:** Borrower calls `liquidate(market, 0, 0, 0, borrower, false, borrower, callbackContract, data)`. `liquidatorGate == address(0)` so no gate blocks this.
3. **Bad debt realized:** Bitmap snapshot is 0 → `badDebt = D`, `maxDebt = 0`. Check `D > 0` passes. State committed: `_position.debt = 0`, `totalUnits -= D`, `lossFactor` updated to socialize D across all lenders.
4. **Callback:** `callbackContract.onLiquidate(...)` calls `supplyCollateral(market, idx, X, borrower)`. Authorization passes because `isAuthorized[borrower][callbackContract]` is true. Borrower's position now has `debt = 0`, `collateral[idx] = X`, `collateralBitmap ≠ 0`.
5. **Recovery:** `repaidUnits = 0` so the post-callback `safeTransferFrom` transfers nothing. Borrower calls `withdrawCollateral`; `isHealthy` returns `true` (debt = 0), and X collateral tokens are returned.

**Net result:** Debt D is wiped at zero net cost. Lenders absorb D units of bad debt even though the borrower possessed collateral sufficient to cover it.

**Why existing checks fail:**

- `liquidationLocked` in `liquidate` (line 621) is a *precondition* check that prevents liquidating a locked position; it does not set the lock during the callback window.
- The lock is only set via `tExchange` in `take` (line 444), never in `liquidate`.
- `supplyCollateral` contains no guard referencing the liquidation lock or any reentrancy protection. [6](#0-5) 

## Impact Explanation
Lenders' credit is permanently reduced via `lossFactor` socialization by the full original debt D, even when the borrower possessed collateral sufficient to cover some or all of that debt. The invariant `badDebt = max(0, debt − Σ(collateral_i · price_i / maxLif_i))` is violated because the final collateral state (post-callback) is not reflected in the realized bad debt. The excess loss per attack equals `min(D, X · price / maxLif)`, which equals D if the borrower supplies enough collateral to fully cover the debt. This constitutes direct, permanent theft of lender value with no recovery path.

## Likelihood Explanation
All preconditions are reachable without privilege:
- `collateralBitmap = 0` with `debt > 0` arises naturally after a partial liquidation that seizes all collateral without repaying all debt — a documented bad-debt scenario.
- The borrower holds collateral tokens externally (purchasable on open market) and pre-authorizes the callback contract via a single permissionless `setIsAuthorized` call.
- The borrower calls `liquidate` on themselves — permissionless when `liquidatorGate == address(0)`.
- The attack is repeatable across any market where the borrower can reach the zero-bitmap/nonzero-debt state.

## Recommendation
Set the `LIQUIDATION_LOCK_SLOT` for the borrower before firing the `onLiquidate` callback in `liquidate`, mirroring the pattern used in `take`. Specifically, add a `tExchange(LIQUIDATION_LOCK_SLOT, id, borrower, true)` call before the callback invocation and restore it afterward. This prevents `supplyCollateral` (and any other state-mutating call that checks `liquidationLocked`) from being called on the borrower's position during the callback window. Alternatively, add an explicit check in `supplyCollateral` that reverts if `liquidationLocked(id, onBehalf)` is true.

## Proof of Concept
**Minimal Foundry test outline:**
1. Deploy a market with `liquidatorGate = address(0)` and one collateral.
2. Lender supplies D loan tokens; borrower takes D units of debt, then a prior liquidation seizes all collateral leaving `debt = D`, `collateralBitmap = 0`.
3. Deploy `MaliciousCallback` that implements `onLiquidate` to call `midnight.supplyCollateral(market, 0, X, borrower)` (pre-funded with X collateral tokens, pre-approved).
4. Borrower calls `midnight.setIsAuthorized(address(maliciousCallback), true, borrower)`.
5. Borrower calls `midnight.liquidate(market, 0, 0, 0, borrower, false, borrower, address(maliciousCallback), "")`.
6. Assert: `midnight.debtOf(id, borrower) == 0`, `midnight.lossFactor(id) > 0` (lenders absorbed D).
7. Borrower calls `midnight.withdrawCollateral(market, 0, X, borrower, borrower)`.
8. Assert: borrower's collateral token balance restored to X, lenders permanently bear D bad debt.
