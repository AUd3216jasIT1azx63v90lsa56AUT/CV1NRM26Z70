Audit Report

## Title
Authorized-Party Nonce Burn via Delegated Signature Consumes Authorizer's Nonce — (`src/periphery/EcrecoverAuthorizer.sol`)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts signatures from any party already authorized by `authorization.authorizer` in Midnight, not only from the authorizer themselves. Because `nonce[authorization.authorizer]` is incremented on every successful call regardless of who signed, an authorized party can craft and submit their own EIP-712 signature over an `Authorization` struct naming the victim as `authorizer`, consuming the victim's current nonce and permanently invalidating any pending off-chain authorization the victim has already distributed with that nonce.

## Finding Description
**Code path** — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:

The nonce is keyed to `authorization.authorizer` (victim A) and is incremented unconditionally at line 26 before any signature check:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

The `Unauthorized` guard at lines 33–36 has a delegation branch:

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

**Root cause** — There is no requirement that the signer is the same party who generated the `Authorization` struct, nor that the struct was produced by the authorizer. An authorized party (ATTACKER) can independently construct `Authorization{authorizer=A, authorized=ATTACKER, isAuthorized=true, nonce=N}` and sign it with their own private key. `ecrecover` returns ATTACKER's address. The delegation branch `isAuthorized[A][ATTACKER] == true` passes. The nonce `nonce[A]` advances N → N+1. The call succeeds as a no-op (ATTACKER re-authorizes themselves).

**Exploit flow:**

| Step | Action |
|------|--------|
| 0 | A authorizes `ecrecoverAuthorizer` and ATTACKER in Midnight (normal protocol setup). |
| 1 | A signs off-chain: `Authorization{authorizer=A, authorized=LEGITIMATE_OPERATOR, isAuthorized=true, nonce=N}` and hands it to `LEGITIMATE_OPERATOR`. |
| 2 | ATTACKER independently signs: `Authorization{authorizer=A, authorized=ATTACKER, isAuthorized=true, nonce=N}` with ATTACKER's own key. |
| 3 | ATTACKER calls `ecrecoverAuthorizer.setIsAuthorized(authAttacker, sigAttacker)`. |
| 4 | Line 26: `nonce[A]` advances N → N+1. Lines 33–36: `isAuthorized[A][ATTACKER] == true` → passes. Call succeeds. |
| 5 | `LEGITIMATE_OPERATOR` submits A's signed authorization → reverts `InvalidNonce` because `nonce[A]` is now N+1. |

**Why existing checks fail** — The `Unauthorized` check is satisfied by the delegation branch; it does not distinguish between "the authorizer is signing their own nonce" and "a delegate is consuming the authorizer's nonce on a struct the delegate constructed themselves." There is no guard preventing an authorized party from targeting `authorization.authorizer = A` with a nonce they did not generate.

## Impact Explanation
A's off-chain signed authorization with nonce=N is permanently invalidated. `LEGITIMATE_OPERATOR` cannot submit it; A must re-sign with nonce=N+1 and redistribute. ATTACKER can repeat this indefinitely (each time submitting a no-op self-authorization `isAuthorized=true` for themselves), making it impossible for A to successfully delegate to any third party through `EcrecoverAuthorizer` as long as ATTACKER remains authorized. A's only escape is to revoke ATTACKER's authorization via a direct on-chain call to `midnight.setIsAuthorized`, but ATTACKER can front-run that revocation with another nonce burn. This constitutes a persistent, repeatable DoS on A's authorization mechanism.

## Likelihood Explanation
**Preconditions:**
1. A has authorized `ecrecoverAuthorizer` in Midnight — required for any EcrecoverAuthorizer user.
2. A has authorized ATTACKER in Midnight at any prior point (e.g., a market maker, keeper, or liquidation bot).
3. A has a pending off-chain signed authorization that ATTACKER knows about (e.g., via mempool observation).

All three are realistic for any active protocol user. `Midnight.sol` lines 101–110 document that "authorized accounts can authorize other accounts on behalf of the user," meaning users who authorize counterparties for position management do not expect those counterparties to also gain nonce-burning capability in `EcrecoverAuthorizer`. The attack is repeatable with zero cost beyond gas.

## Recommendation
Restrict the delegation branch so that a delegate may only sign on behalf of the authorizer if the `Authorization` struct was produced by the authorizer (i.e., require `signer == authorization.authorizer` always, removing the delegation branch entirely from `EcrecoverAuthorizer`). If delegation is intentionally supported, introduce a separate nonce namespace per signer (e.g., `nonce[signer]` instead of `nonce[authorization.authorizer]`) so that a delegate's signature consumes only the delegate's own nonce, not the authorizer's. Alternatively, add an explicit check that `signer == authorization.authorizer` to enforce that only the authorizer can consume their own nonce.

## Proof of Concept
```solidity
// 1. A authorizes ecrecoverAuthorizer and ATTACKER in Midnight
midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, A); // from A
midnight.setIsAuthorized(ATTACKER, true, A);                     // from A

// 2. A signs off-chain authorization for LEGITIMATE_OPERATOR at nonce=0
// (Authorization{authorizer=A, authorized=LEGITIMATE_OPERATOR, isAuthorized=true, nonce=0, deadline=...})

// 3. ATTACKER constructs and signs their own Authorization at nonce=0
Authorization memory authAttacker = Authorization({
    authorizer: A,
    authorized: ATTACKER,
    isAuthorized: true,
    nonce: 0,
    deadline: block.timestamp + 1 hours
});
// ATTACKER signs authAttacker with ATTACKER's private key → sigAttacker

// 4. ATTACKER submits — succeeds, nonce[A] advances 0 → 1
ecrecoverAuthorizer.setIsAuthorized(authAttacker, sigAttacker);

// 5. LEGITIMATE_OPERATOR submits A's authorization — reverts InvalidNonce
ecrecoverAuthorizer.setIsAuthorized(authLegitimate, sigA); // reverts
```
