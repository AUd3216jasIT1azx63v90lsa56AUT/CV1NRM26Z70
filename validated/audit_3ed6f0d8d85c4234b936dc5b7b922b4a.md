### Title
Pre-Signed `Authorization` in `EcrecoverAuthorizer` Remains Valid After On-Chain Revocation, Enabling Replay to Re-Authorize Ratifier â€” (File: `src/periphery/EcrecoverAuthorizer.sol`)

---

### Summary

`EcrecoverAuthorizer` uses a sequential per-authorizer nonce to prevent replay of signed `Authorization` structs. However, there is no mechanism to invalidate (skip/burn) a pending nonce without consuming it. If a maker signs an `Authorization` for gasless use but later revokes the ratifier's authorization directly on `Midnight`, the pre-signed `Authorization` remains valid. Any party in possession of that signature can replay it to re-establish the ratifier's authorization, allowing the maker's offers to be taken after the maker believed they were canceled.

---

### Finding Description

**Root cause â€” no nonce invalidation path in `EcrecoverAuthorizer`:**

`EcrecoverAuthorizer.setIsAuthorized` enforces sequential nonces:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

The nonce is only advanced when a signed authorization is consumed. There is no `invalidateNonce` (or equivalent) function that lets a maker advance their nonce to burn pending signed authorizations. [1](#0-0) 

The nonce storage is a plain mapping with no skip/invalidation path: [2](#0-1) 

**Contrast with `EcrecoverRatifier`:**

`EcrecoverRatifier` â€” the analogous component for offer signatures â€” explicitly provides a `cancelRoot` function so a maker can invalidate signed offer trees:

```solidity
function cancelRoot(address maker, bytes32 root) external {
    require(maker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(maker, msg.sender), Unauthorized());
    isRootCanceled[maker][root] = true;
    ...
}
``` [3](#0-2) 

`EcrecoverAuthorizer` has no equivalent. [4](#0-3) 

**Attack path:**

1. Alice (maker) signs `Authorization(authorized=EcrecoverRatifier, isAuthorized=true, nonce=0, deadline=T+365d)` for gasless/relayer use and shares the signature off-chain.
2. Alice directly calls `Midnight.setIsAuthorized(EcrecoverRatifier, true, Alice)` â€” nonce `0` in `EcrecoverAuthorizer` is **never consumed**.
3. Alice places offers signed via `EcrecoverRatifier`.
4. Market conditions change; Alice wants to cancel all her offers. She calls `Midnight.setIsAuthorized(EcrecoverRatifier, false, Alice)` â€” `EcrecoverRatifier` is now unauthorized, and `take` will revert at:

```solidity
require(isAuthorized[offer.maker][offer.ratifier], RatifierUnauthorized());
``` [5](#0-4) 

5. An attacker who obtained the pre-signed `Authorization` (from the relayer, mempool, or off-chain storage) calls `EcrecoverAuthorizer.setIsAuthorized(authorization, signature)` with nonce=0.
6. The check `authorization.nonce == nonce[authorization.authorizer]++` passes (nonce is still 0), `EcrecoverRatifier` is re-authorized for Alice. [6](#0-5) 
7. Alice's offers are now takeable again at prices that are unfavorable to her.

The test suite confirms that `setIsAuthorized` is **permissionless** â€” any caller can submit a valid signed authorization: [7](#0-6) 

---

### Impact Explanation

A maker who revokes a ratifier's authorization (intending to cancel all their active offers) can have that authorization silently re-established by any party holding a pre-signed `Authorization` with an unconsumed nonce. This allows the attacker to force execution of the maker's offers at prices that are now unfavorable to the maker (e.g., after adverse market movement), resulting in direct financial loss to the maker. The maker has no on-chain recourse once the pre-signed authorization exists and the deadline has not passed.

---

### Likelihood Explanation

The gasless/relayer workflow is the primary use case for `EcrecoverAuthorizer` â€” makers sign authorizations off-chain and share them with relayers or front-end services. It is realistic that:
- A maker signs an authorization for a relayer but later authorizes directly on-chain (leaving nonce=0 unconsumed).
- The relayer's database, a failed transaction in the mempool, or a compromised front-end exposes the signed authorization to an attacker.
- The attacker replays it within the deadline window.

No privileged access is required. The attacker only needs the signed `Authorization` struct and its signature.

---

### Recommendation

Add an `invalidateNonce` function to `EcrecoverAuthorizer` that allows a maker to advance their nonce past any pending signed authorizations, mirroring the `cancelRoot` pattern in `EcrecoverRatifier`:

```solidity
function invalidateNonce(address authorizer, uint256 newNonce) external {
    require(authorizer == msg.sender || IMidnight(MIDNIGHT).isAuthorized(authorizer, msg.sender), Unauthorized());
    require(newNonce > nonce[authorizer], InvalidNonce());
    nonce[authorizer] = newNonce;
    emit NonceInvalidated(msg.sender, authorizer, newNonce);
}
```

This allows a maker to burn all pending signed authorizations up to `newNonce` in a single transaction, consistent with the pattern used by Permit2's `AllowanceTransfer.invalidateNonces`. [8](#0-7)

contract EcrecoverAuthorizer is IEcrecoverAuthorizer {
    address public immutable MIDNIGHT;
    mapping(address => uint256) public nonce;

    constructor(address _midnight) {
        MIDNIGHT = _midnight;
    }

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
