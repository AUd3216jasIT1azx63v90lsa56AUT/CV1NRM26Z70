Audit Report

## Title
`sellerAssetsToUnits` Division-by-Zero When `sellerPrice == 0` DoS's Periphery Bundler - (File: src/periphery/TakeAmountsLib.sol)

## Summary
In `TakeAmountsLib.sellerAssetsToUnits`, when `offer.buy = true` and `offerPrice == settlementFee`, `sellerPrice` computes to `0`, causing an unconditional Solidity 0.8 division-by-zero panic on `mulDivUp(WAD, 0)`. The core `Midnight.take()` does not revert in this state — it computes `sellerAssets = 0` via `mulDivDown(0, WAD)` — creating a documented core/periphery divergence. Because `sellerAssetsToUnits` is called outside the `try/catch` block in `supplyCollateralAndSellWithAssetsTarget`, the panic propagates and permanently DoS's that bundler function for any offer in this state.

## Finding Description

**Root cause — `TakeAmountsLib.sol` lines 44–46:**

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
return offer.buy ? targetSellerAssets.mulDivUp(WAD, sellerPrice) : ...;
```

When `offer.buy = true` and `offerPrice == settlementFee`, `sellerPrice = 0`. The return statement immediately calls `targetSellerAssets.mulDivUp(WAD, 0)` — a Solidity 0.8 panic. There is no zero-guard on `sellerPrice`.

The NatSpec at line 34 states *"Reverts if offerPrice < settlementFee in case of a buy offer (midnight reverts too)"* — but this only covers the strict underflow case. For the equal case (`offerPrice == settlementFee`), midnight does **not** revert, while the periphery panics. The NatSpec's claim that "midnight reverts too" is false for the equal case.

**Core does not revert — `Midnight.sol` lines 361–364:**

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice;
uint256 sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : ...;
```

With `sellerPrice = 0`, `units.mulDivDown(0, WAD) = 0`. No revert. `take()` proceeds normally, transferring 0 assets to the seller.

**Bundler call site — `MidnightBundles.sol` lines 285–300:**

`TakeAmountsLib.sellerAssetsToUnits(...)` is called at line 286, **outside** the `try/catch` block that wraps `IMidnight(MIDNIGHT).take(...)` at lines 292–300. The panic propagates uncaught and reverts the entire `supplyCollateralAndSellWithAssetsTarget` call.

**Simplest exploit path (no governance required):**
1. A market exists with `settlementFee = 0` (default if no fee has been set, or at any breakpoint where the fee is 0).
2. An unprivileged lender creates a buy offer at tick 0, where `tickToPrice(0) = 0` (confirmed by Certora spec `tickToPriceIsZeroAtZero`). Thus `offerPrice = 0 = settlementFee`.
3. Any caller invokes `supplyCollateralAndSellWithAssetsTarget` with `takes` containing that offer and `targetSellerAssets > 0`.
4. `sellerAssetsToUnits` panics with division-by-zero; the entire bundler transaction reverts.

**Alternative path (natural fee drift):** Both `tickToPrice` prices and settlement fees are multiples of `PRICE_ROUNDING_STEP = 1e12` (CBP). The settlement fee is a piecewise-linear function of TTM that changes continuously with time. As time passes, the interpolated fee value naturally drifts and can pass through a tick price value, triggering the same condition without any governance action.

**Why existing checks fail:** The NatSpec guard only documents the `offerPrice < settlementFee` case (underflow revert in both core and periphery). The equal case is unguarded. The bundler NatSpec at line 246 states "Reverts if TakeAmountsLib or ConsumableUnitsLib reverts" — acknowledging propagation but not the specific undocumented trigger.

## Impact Explanation
Any call to `supplyCollateralAndSellWithAssetsTarget` that includes a buy offer where `tickToPrice(offer.tick) == settlementFee` and `targetSellerAssets > 0` reverts unconditionally. The bundler is DoS'd for that offer. Because the `try/catch` only wraps `take()` and not `sellerAssetsToUnits`, there is no graceful fallback. Funds are not frozen in the core (`take()` still works directly), but the periphery path is completely blocked for affected offers. The divergence between core and periphery violates the NatSpec contract that "midnight reverts too" for the equal case.

## Likelihood Explanation
**Preconditions:**
- Settlement fee must equal `tickToPrice(T)` for some tick `T`. Both quantities are multiples of `1e12`, so exact equality is structurally possible.
- The simplest case requires no governance: tick 0 gives `offerPrice = 0`, and any market with `settlementFee = 0` (default) satisfies the condition.
- The settlement fee is a piecewise-linear function of TTM that changes continuously with time. Even without malicious governance action, the interpolated fee value naturally drifts and can pass through a tick price value.
- Once the condition holds, any unprivileged lender can create a buy offer at tick `T`, and any caller of the bundler triggers the DoS.
- `targetSellerAssets > 0` — always true for any real bundler call.

**Repeatability:** Deterministic — every bundler call with that offer reverts until the offer expires or the fee changes.

## Recommendation
Add a zero-guard on `sellerPrice` in `sellerAssetsToUnits`. If `sellerPrice == 0`, either revert with a descriptive error or return `type(uint256).max` (indicating no finite number of units can yield a positive `targetSellerAssets`). Update the NatSpec to document the equal case explicitly. Optionally, move `sellerAssetsToUnits` inside the `try/catch` block in `supplyCollateralAndSellWithAssetsTarget`, or add a pre-check that skips offers where `sellerPrice == 0`.
