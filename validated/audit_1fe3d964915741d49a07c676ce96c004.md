Audit Report

## Title
Rounding in `lossFactor` update prematurely saturates market when `_lossFactor = type(uint128).max - 1`, permanently blocking all `take()` calls via `MarketLossFactorMaxedOut` - (File: src/Midnight.sol)

## Summary
When `_marketState.lossFactor` equals `type(uint128).max - 1`, the `mulDivDown` inner term in the lossFactor update formula evaluates to `0` for any `badDebt >= 1`, regardless of how small `badDebt` is relative to `totalUnits`. This causes `lossFactor` to be set to `type(uint128).max`, permanently freezing the market: all subsequent `take()` calls revert with `MarketLossFactorMaxedOut`, even though the market still has outstanding debt and credit.

## Finding Description

**Exact code path — `src/Midnight.sol` lines 626–633:**

```solidity
if (badDebt > 0) {
    _position.debt -= uint128(badDebt);
    uint256 _totalUnits = _marketState.totalUnits;
    uint256 _lossFactor = _marketState.lossFactor;
    _marketState.lossFactor = UtilsLib.toUint128(
        type(uint128).max - (type(uint128).max - _lossFactor).mulDivDown(_totalUnits - badDebt, _totalUnits)
    );
```

`mulDivDown` at `src/libraries/UtilsLib.sol` lines 29–31 is plain integer division:

```solidity
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;
}
```

**Root cause arithmetic:**

When `_lossFactor = type(uint128).max - 1`:
- `type(uint128).max - _lossFactor = 1`
- The call becomes `mulDivDown(1, _totalUnits - badDebt, _totalUnits)` = `(1 * (_totalUnits - badDebt)) / _totalUnits`
- For any `badDebt >= 1`, `_totalUnits - badDebt < _totalUnits`, so integer division yields `0`
- Therefore `newLossFactor = type(uint128).max - 0 = type(uint128).max`

This holds even when `badDebt = 1` and `totalUnits = 1_000_000` (0.0001% bad debt).

**Reachability of `_lossFactor = type(uint128).max - 1`:**

The state is reachable through normal market operation. Concrete example: `oldLossFactor = type(uint128).max - 2`, `badDebt = 1`, `totalUnits = 3`:
- `mulDivDown(2, 2, 3) = 4/3 = 1`
- `newLossFactor = type(uint128).max - 1` ✓

**Exploit flow:**
1. Market accumulates bad debt over its lifetime, driving `_lossFactor` to `type(uint128).max - 1` via the monotonically increasing update formula.
2. Any unprivileged liquidator calls `liquidate(market, 0, 0, 0, borrower, false, receiver, address(0), "")` on any borrower with `badDebt >= 1`. The protocol explicitly documents this zero-cost path at lines 577–578: *"Passing both 0 for seizedAssets and repaidUnits allows to realize bad debt with 0 token transferred."*
3. The formula evaluates `mulDivDown(1, totalUnits - badDebt, totalUnits) = 0`.
4. `_marketState.lossFactor` is set to `type(uint128).max`.
5. All subsequent `take()` calls revert at line 349: `require(_marketState.lossFactor < type(uint128).max, MarketLossFactorMaxedOut())`.

**Why existing checks fail:**

There is no guard preventing `lossFactor` from reaching `type(uint128).max` when `badDebt < totalUnits`. The `liquidate()` function only requires `badDebt > 0` to enter the update block. The Certora formal verification rule `lossFactorChangesIffBadDebt` (`certora/specs/LossFactor.spec` line 53) explicitly preconditions on `lossFactorBefore < max_uint128`, meaning it does not cover the near-max regime and provides no protection here.

## Impact Explanation
Once `lossFactor = type(uint128).max`, `take()` permanently reverts with `MarketLossFactorMaxedOut` for the affected market. Any remaining lenders with non-zero credit cannot take new positions or reduce their exposure through the normal take mechanism. The market is frozen by a single liquidation call that realizes as little as 1 unit of bad debt, even when `totalUnits - 1` units of outstanding debt remain. This constitutes a permanent, irreversible freeze of a live market — a concrete in-scope impact matching bad debt misallocation and position desynchronization.

## Likelihood Explanation
The precondition (`_lossFactor = type(uint128).max - 1`) is reachable through normal liquidation activity in any distressed market without any privileged action. Once the precondition is met, any unprivileged liquidator can trigger the saturation with a single zero-cost call (`seizedAssets = 0`, `repaidUnits = 0`), requiring no token transfers. The trigger is a single transaction, and the resulting state change is permanent and irreversible.

## Recommendation
Cap the new `lossFactor` to avoid premature saturation when `badDebt < totalUnits`. One approach: only set `lossFactor = type(uint128).max` when `badDebt >= totalUnits` (i.e., full bad debt), and otherwise clamp the result to `type(uint128).max - 1` to preserve market liveness:

```solidity
uint256 newLossFactor = badDebt >= _totalUnits
    ? type(uint128).max
    : type(uint128).max - (type(uint128).max - _lossFactor).mulDivDown(_totalUnits - badDebt, _totalUnits);
// Optionally: clamp to type(uint128).max - 1 if badDebt < totalUnits to prevent rounding-induced saturation
newLossFactor = (badDebt < _totalUnits && newLossFactor == type(uint128).max)
    ? type(uint128).max - 1
    : newLossFactor;
_marketState.lossFactor = UtilsLib.toUint128(newLossFactor);
```

Alternatively, use `mulDivUp` for the inner term so that rounding favors preserving a non-zero remainder, preventing the result from collapsing to `0` when `x = 1` and `y < d`.

## Proof of Concept

**Minimal manual steps:**

1. Deploy a market and drive `lossFactor` to `type(uint128).max - 1` via repeated bad debt liquidations (e.g., using the concrete path: `oldLossFactor = type(uint128).max - 2`, `badDebt = 1`, `totalUnits = 3`).
2. Confirm a borrower exists with `debt > 0` and `badDebt >= 1` (e.g., zero collateral, post-maturity).
3. Call `liquidate(market, 0, 0, 0, borrower, false, receiver, address(0), "")`.
4. Assert `marketState[id].lossFactor == type(uint128).max`.
5. Attempt any `take()` call on the market — assert it reverts with `MarketLossFactorMaxedOut`.

**Fuzz/invariant test plan:**

Write an invariant: `if marketState[id].totalUnits > 0 && marketState[id].lossFactor == type(uint128).max, then the last liquidation must have had badDebt == totalUnitsBefore`. rule lossFactorChangesIffBadDebt(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, bool postMaturityMode) {
    bytes32 id = summaryToId(market);
    uint128 lossFactorBefore = currentContract.marketState[id].lossFactor;
    uint256 totalUnitsBefore = totalUnits(id);

    require lossFactorBefore < max_uint128, "market lossFactor must not be saturated";

    assert lossFactorChanged <=> badDebtOccurred;
}
