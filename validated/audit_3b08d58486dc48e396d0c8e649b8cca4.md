Audit Report

## Title
Arithmetic Underflow in `liquidate` When Seizing All Collateral at Price Just Above Bad-Debt Threshold - (File: `src/Midnight.sol`)

## Summary

When a liquidator passes `seizedAssets = collateral` (full collateral seizure) at a price exactly one unit above `badDebtPriceDown`, the bad-debt block is skipped (`badDebt = 0`, so `_position.debt` remains `originalDebt`), but `repaidUnits`, computed with the identical ceiling expression (`mulDivUp`), can be strictly greater than `originalDebt`. The subsequent checked subtraction `_position.debt -= UtilsLib.toUint128(repaidUnits)` at line 676 reverts with an arithmetic underflow, blocking the `seizedAssets` input path at this price boundary. No privileged role is required.

## Finding Description

**Root cause — rounding asymmetry between bad-debt check and `repaidUnits` computation.**

The bad-debt amount is computed with ceiling rounding via `zeroFloorSub`:

```solidity
// src/Midnight.sol lines 614-616
badDebt = badDebt.zeroFloorSub(
    _collateral.mulDivUp(price, ORACLE_PRICE_SCALE).mulDivUp(WAD, _collateralParam.maxLif)
);
```

`badDebt = 0` whenever `_collateral.mulDivUp(price, ORACLE_PRICE_SCALE).mulDivUp(WAD, maxLif) >= originalDebt`.

When the liquidator passes `seizedAssets = collateral`, `repaidUnits` is computed with the **identical ceiling expression**:

```solidity
// src/Midnight.sol line 650
repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
```

In normal pre-maturity mode, `lif = _maxLif = maxLif`. So `repaidUnits = _collateral.mulDivUp(price, ORACLE_PRICE_SCALE).mulDivUp(WAD, maxLif)`.

This is the same expression as in the bad-debt computation. Therefore, when `badDebt = 0`, `repaidUnits >= originalDebt`. When the real-valued result of `_collateral * price / ORACLE_PRICE_SCALE * WAD / maxLif` lies strictly in `(originalDebt, originalDebt + 1)`, the ceiling rounds it up to `originalDebt + 1`, so `repaidUnits = originalDebt + 1 > _position.debt`. The checked subtraction at line 676 then reverts:

```solidity
// src/Midnight.sol line 676
_position.debt -= UtilsLib.toUint128(repaidUnits); // underflow: repaidUnits > _position.debt
```

The existing test `testBadDebtPriceDownIsMaximal` (lines 336–343) confirms that at `price = badDebtPriceDown(units) + 1`, `_badDebt() = 0`, establishing the precondition.

**Why the RCF check does not prevent this.**

The RCF check at lines 655–668 computes:

```solidity
uint256 maxRepaid = lltv < WAD
    ? (_position.debt - maxDebt).mulDivUp(WAD * WAD, WAD * WAD - lif * lltv)
    : type(uint256).max;
require(
    repaidUnits <= maxRepaid || ...zeroFloorSub(maxRepaid) < market.rcfThreshold,
    RecoveryCloseFactorConditionsViolated()
);
```

`maxRepaid` is a multiple of `(debt - maxDebt)`. For a position modestly underwater (e.g., 2%), `maxRepaid` can be a large multiple of `debt`, so `repaidUnits = debt + 1 <= maxRepaid` is satisfied. Alternatively, markets with `rcfThreshold = type(uint256).max` (as used in `testLiquidateFullyRepayOrFullySeizeWhenRcfDeactivated`) trivially pass the second condition. Neither path checks `repaidUnits <= _position.debt`.

**Exploit flow:**

1. Market: single collateral, any LLTV < WAD, normal pre-maturity mode.
2. Borrower has `debt = units`, `collateral = units.mulDivUp(WAD, lltv)`.
3. Oracle set to `price = badDebtPriceDown(units) + 1`.
4. At this price: `badDebt = 0` (position unhealthy but no bad debt).
5. Liquidator calls `liquidate(seizedAssets = collateral, repaidUnits = 0, postMaturityMode = false)`.
6. `repaidUnits = collateral.mulDivUp(price, ORACLE_PRICE_SCALE).mulDivUp(WAD, maxLif) = units + 1`.
7. RCF check passes (`repaidUnits <= maxRepaid` or `rcfThreshold = type(uint256).max`).
8. `_position.debt -= UtilsLib.toUint128(repaidUnits)` → arithmetic underflow → revert.

## Impact Explanation

The `seizedAssets = collateral` liquidation path reverts unconditionally at this price boundary. The position is genuinely unhealthy (`originalDebt > maxDebt`) and the liquidator is entitled to seize collateral, but the transaction always reverts. This is a service availability failure: a valid, documented liquidation input mode is permanently broken at a deterministic price value. Note that the `repaidUnits = debt` input path is unaffected (it uses `mulDivDown` at line 652), so the position is not permanently unliquidatable, but the full-collateral-seizure path is blocked.

## Likelihood Explanation

The price window is a single integer value (`badDebtPriceDown + 1`), which is narrow in relative terms but deterministic and repeatable. Any oracle returning integer prices (e.g., Chainlink-style with 1e36 scaling) will hit it with nonzero probability. No privileged access is required. Markets with `rcfThreshold = type(uint256).max` are unconditionally vulnerable whenever the oracle returns this exact price. The condition is reachable by any liquidator observing the oracle price.

## Recommendation

Before the subtraction at line 676, cap `repaidUnits` to `_position.debt`:

```solidity
repaidUnits = UtilsLib.min(repaidUnits, _position.debt);
_position.debt -= UtilsLib.toUint128(repaidUnits);
```

Alternatively, add an explicit guard:

```solidity
require(repaidUnits <= _position.debt, RepaidExceedsDebt());
_position.debt -= UtilsLib.toUint128(repaidUnits);
```

The capping approach is preferable as it preserves the liquidator's intent (seize all collateral, repay as much debt as the collateral value covers) without reverting.

## Proof of Concept

Minimal Foundry test (add to `LiquidationTest.sol`):

```solidity
function testUnderflowAtBadDebtBoundary(uint256 units) public {
    units = bound(units, 100, MAX_UNITS);
    collateralize(market, borrower, units);
    setupMarket(market, units);

    uint256 price = badDebtPriceDown(units) + 1;
    Oracle(market.collateralParams[0].oracle).setPrice(price);

    // Confirm no bad debt at this price
    assertEq(_badDebt(), 0, "no bad debt");

    // Confirm position is unhealthy (liquidatable)
    uint256 collateral = midnight.collateral(id, borrower, 0);
    uint256 maxDebt = collateral.mulDivDown(price, ORACLE_PRICE_SCALE)
        .mulDivDown(market.collateralParams[0].lltv, WAD);
    assertGt(units, maxDebt, "position is unhealthy");

    // Compute expected repaidUnits
    uint256 expectedRepaid = collateral.mulDivUp(price, ORACLE_PRICE_SCALE)
        .mulDivUp(WAD, market.collateralParams[0].maxLif);

    // If repaidUnits > debt, the seizedAssets path reverts
    if (expectedRepaid > units) {
        vm.expectRevert(stdError.arithmeticError);
        midnight.liquidate(market, 0, collateral, 0, borrower, false, address(this), address(0), "");
    }
}

    function testLiquidateFullyRepayOrFullySeizeWhenRcfDeactivated(
        uint256 units,
        uint256 collateral1,
        uint256 collateral2
    ) public {
        collateral1 = bound(collateral1, 1, MAX_UNITS);
        collateral2 = bound(collateral2, 1, MAX_UNITS);

        // Deactivate RCF.
        market.rcfThreshold = type(uint256).max;
        id = toId(market);

        // Price is 1 initially, assume liquidatable but no bad debt.
        uint256 maxDebt = collateral1.mulDivDown(market.collateralParams[0].lltv, WAD)
            + collateral2.mulDivDown(market.collateralParams[1].lltv, WAD);
        uint256 repayableDebt = collateral1.mulDivDown(WAD, market.collateralParams[0].maxLif)
            + collateral2.mulDivDown(WAD, market.collateralParams[1].maxLif);
        units = bound(units, maxDebt, repayableDebt);
        vm.assume(units > maxDebt);

        // Write debt into Position storage.
        // Layout: slot 0 = credit | pendingFee, slot 1 = lastLossFactor | lastAccrual,
        // slot 2 = debt | collateralBitmap.
        // Debt is in the lower 128 bits of slot 2.
        uint256 mappingSlot = 0;
        bytes32 intermediateSlot = keccak256(abi.encode(id, mappingSlot));
        bytes32 borrowerSlot = keccak256(abi.encode(borrower, intermediateSlot));
        vm.store(address(midnight), bytes32(uint256(borrowerSlot) + 2), bytes32(units));

        assertEq(midnight.debtOf(id, borrower), units, "debt");

        // Collateralize with both collateralParams.

        vm.prank(borrower);

        midnight.setIsAuthorized(address(this), true, borrower);

        deal(market.collateralParams[0].token, address(this), collateral1);
        midnight.supplyCollateral(market, 0, collateral1, borrower);

        deal(market.collateralParams[1].token, address(this), collateral2);
        midnight.supplyCollateral(market, 1, collateral2, borrower);

        // Check that the position has no bad debt.
        // If it had bad debt, this can be taken into account separately.
        assertEq(_badDebt(), 0, "no bad debt");

        uint256 collateralNeededToRepayAll = units.mulDivDown(market.collateralParams[0].maxLif, WAD);
        if (collateralNeededToRepayAll <= collateral1) {
            midnight.liquidate(market, 0, 0, units, borrower, false, address(this), address(0), "");
        } else {
            midnight.liquidate(market, 0, collateral1, 0, borrower, false, address(this), address(0), "");
        }

        uint256 debtAfter = midnight.debtOf(id, borrower);
        uint256 collateralAfter = midnight.collateral(id, borrower, 0);
        assertTrue(debtAfter == 0 || collateralAfter == 0, "either debt repaid or collateral seized");
    }
