### Title
Gas Griefing via Uncapped External Calls to Maker-Controlled Callbacks in `take()` — (File: src/Midnight.sol)

---

### Summary

In `Midnight.sol`'s `take()` function, external calls are made to maker-controlled callback contracts (`offer.callback`) and a maker-authorized ratifier (`offer.ratifier`) without any gas limit. A malicious maker can deploy a callback or ratifier contract that burns all remaining gas, causing the taker's transaction to revert out-of-gas and lose their gas costs with no protocol recourse.

---

### Finding Description

The `take()` function makes multiple unbounded external calls to addresses that are fully controlled by the **maker** (an unprivileged, permissionless role):

**1. Ratifier call (line 356):**
```solidity
require(IRatifier(offer.ratifier).isRatified(offer, ratifierData) == CALLBACK_SUCCESS, RatifierFail());
```
The ratifier is embedded in the signed offer (`offer.ratifier`) and must only be authorized by the maker itself: [1](#0-0) 

**2. Buyer callback call (lines 447–452):**
```solidity
require(
    IBuyCallback(buyerCallback)
        .onBuy(id, offer.market, buyerAssets, units, buyerPendingFeeIncrease, buyer, buyerCallbackData)
    == CALLBACK_SUCCESS,
    WrongBuyCallbackReturnValue()
);
```
`buyerCallback` resolves to `offer.callback` when `offer.buy == true`: [2](#0-1) 

**3. Seller callback call (lines 458–473):**
```solidity
require(
    ISellCallback(sellerCallback)
        .onSell(...) == CALLBACK_SUCCESS,
    WrongSellCallbackReturnValue()
);
```
`sellerCallback` resolves to `offer.callback` when `offer.buy == false`: [3](#0-2) 

The `Offer` struct embeds both `callback` and `ratifier` as maker-supplied fields: [4](#0-3) 

The only authorization check on the ratifier is that the maker has authorized it — meaning the maker can authorize their own malicious contract: [1](#0-0) 

None of these three calls use a gas stipend (e.g., `{gas: N}`). All forward the full remaining gas to the external contract.

---

### Impact Explanation

A malicious maker deploys a callback/ratifier that executes a gas-burning loop (e.g., `while(true) {}`). When a taker calls `take()` against the offer, the unbounded external call consumes all remaining gas. The transaction reverts out-of-gas. The taker:
- Loses all gas paid (can be significant at high gas prices)
- Receives no tokens and no position update
- Can be repeatedly targeted across many blocks at negligible cost to the attacker (signing a new offer is free off-chain)

**Impact: Medium** — direct financial loss (gas) to the taker; no token theft, but repeated griefing is cheap for the attacker.

---

### Likelihood Explanation

- Offers are **permissionless**: any address can be a maker and sign offers with arbitrary `callback` and `ratifier` fields.
- The taker cannot easily distinguish a malicious callback from a legitimate one on-chain without simulating the transaction.
- Automated market-takers (bots) are especially vulnerable as they may not simulate every offer before submitting.
- The attacker's cost is essentially zero: signing an off-chain offer requires no on-chain transaction.

**Likelihood: Medium.**

---

### Recommendation

Apply a fixed gas cap to all external calls to maker-supplied or market-supplied contracts in `take()`:

1. **Ratifier call** — use a fixed gas stipend (e.g., 100k gas):
   ```solidity
   require(IRatifier(offer.ratifier).isRatified{gas: RATIFIER_GAS_LIMIT}(offer, ratifierData) == CALLBACK_SUCCESS, RatifierFail());
   ```

2. **Buyer/Seller callback calls** — allow the taker to specify a gas cap, or enforce a protocol-level maximum:
   ```solidity
   IBuyCallback(buyerCallback).onBuy{gas: MAX_CALLBACK_GAS}(...)
   ```

3. Similarly cap the `IEnterGate` calls at lines 399 and 404, which are also unbounded calls to market-creator-supplied addresses. [5](#0-4) 

---

### Proof of Concept

**Setup:**
```solidity
contract MaliciousCallback is IBuyCallback, ISellCallback {
    function onBuy(...) external returns (bytes32) {
        while (true) {} // burns all gas
    }
    function onSell(...) external returns (bytes32) {
        while (true) {}
    }
}

contract MaliciousRatifier is IRatifier {
    function isRatified(Offer memory, bytes memory) external view returns (bytes32) {
        while (true) {} // burns all gas
    }
}
```

**Attack steps:**
1. Attacker deploys `MaliciousCallback` and `MaliciousRatifier`.
2. Attacker (maker) calls `setIsAuthorized(address(maliciousRatifier), true, attackerAddress)` on Midnight.
3. Attacker signs an `Offer` with `offer.callback = address(maliciousCallback)` and `offer.ratifier = address(maliciousRatifier)`.
4. Victim taker calls `take(offer, ratifierData, units, ...)`.
5. Midnight calls `IRatifier(maliciousRatifier).isRatified(...)` (line 356) — all gas consumed, tx reverts.
6. Alternatively, if the ratifier passes, Midnight calls `IBuyCallback(maliciousCallback).onBuy(...)` (line 448) — all gas consumed, tx reverts.
7. Taker loses all gas. Attacker can repeat indefinitely with new signed offers at zero cost. [6](#0-5) [7](#0-6)
