Audit Report

## Title
Unvalidated Non-Contract `liquidatorGate` Permanently Bricks Liquidations - (File: src/Midnight.sol)

## Summary

`touchMarket` accepts any `address` for `market.liquidatorGate` without validating it is `address(0)` or a deployed contract. An unprivileged attacker can create a market with `liquidatorGate` set to a codeless address. Every subsequent `liquidate()` call unconditionally reverts due to Solidity 0.8.34 ABI-decoding failure when calling `ILiquidatorGate.canLiquidate` on a codeless address, permanently disabling liquidations and enabling unbounded bad-debt accumulation at lenders' expense.

## Finding Description

**Root cause — no `liquidatorGate` validation in `touchMarket`:**

`touchMarket` validates `maturity`, collateral count, token ordering, `lltv`, and `maxLif`, but performs zero validation on `market.liquidatorGate`: [1](#0-0) 

The `Market` struct accepts any `address` for `liquidatorGate` with no constraint: [2](#0-1) 

**Trigger — `liquidate()` calls the interface unconditionally on the non-contract address:** [3](#0-2) 

When `liquidatorGate = address(0xdead)`:
1. `market.liquidatorGate == address(0)` → `false`; short-circuit does not fire.
2. `ILiquidatorGate(address(0xdead)).canLiquidate(msg.sender)` is evaluated.
3. The EVM `CALL` to a codeless address returns `(success=true, returndata=<empty>)`.
4. Solidity 0.8.34's ABI decoder expects ≥ 32 bytes for the `bool` return value declared in `ILiquidatorGate.canLiquidate`; it receives 0 bytes → unconditional revert (ABI decoding error). [4](#0-3) 

5. The `require` guard cannot catch this: a revert thrown *inside* the condition expression propagates unconditionally; `require` only catches a `false` return.

**Why the LIVENESS comment is insufficient:**

The LIVENESS section at line 152 states: "If `liquidatorGate.canLiquidate` reverts or returns false, `liquidate` reverts." [5](#0-4) 

This documents the behavior of a *legitimate* gate that may transiently revert (e.g., during a pause). It does not constitute acceptance of an adversary-created market with a permanently broken non-contract gate. No `extcodesize` or equivalent guard exists anywhere in the creation or liquidation path.

**Exploit flow:**
1. Attacker calls `touchMarket(market)` with `market.liquidatorGate = address(0xdead)` and otherwise valid parameters. Market is created successfully.
2. Attacker seeds the market with competitive offers; lenders and borrowers interact normally (`take`, `repay`, `supplyCollateral` are unaffected).
3. Borrower's position becomes unhealthy (e.g., oracle price drop).
4. Any call to `liquidate(market, ...)` hits the `canLiquidate` call → ABI-decode revert → liquidation permanently impossible.
5. Bad debt accumulates; lenders cannot recover funds.

## Impact Explanation

Unhealthy borrowers in any market created with a non-contract, non-zero `liquidatorGate` can never be liquidated. Bad debt accumulates without bound. Lenders suffer permanent, unrecoverable fund loss. The core protocol invariant — "unhealthy positions remain liquidatable" — is permanently violated for the affected market. This constitutes direct, concrete loss of lender funds: a permanent freeze/loss of user assets.

## Likelihood Explanation

- **Privilege required:** None. `touchMarket` is public and permissionless.
- **Feasibility:** Trivial. One transaction suffices to create the malformed market.
- **Repeatability:** The market ID is deterministic from its parameters; once created, `liquidatorGate` is immutable (encoded via `IdLib.storeInCode`). The DoS is permanent.
- **Victim attraction:** The attacker can seed the market with competitive offers to draw in lenders and borrowers before positions become unhealthy.

## Recommendation

In `touchMarket`, validate that `market.liquidatorGate` is either `address(0)` or a deployed contract with code:

```solidity
require(
    market.liquidatorGate == address(0) || market.liquidatorGate.code.length > 0,
    InvalidLiquidatorGate()
);
```

This check should be placed alongside the existing collateral parameter validations inside the `if (marketState[id].tickSpacing == 0)` block in `touchMarket`. [6](#0-5) 

## Proof of Concept

**Minimal Foundry test:**

```solidity
function test_liquidatorGateDos() public {
    // 1. Create market with non-contract liquidatorGate
    Market memory market = _buildValidMarket();
    market.liquidatorGate = address(0xdead); // codeless address
    bytes32 id = midnight.touchMarket(market);

    // 2. Supply collateral and borrow (normal flow works)
    _supplyCollateralAndBorrow(market, borrower, id);

    // 3. Drop oracle price to make position unhealthy
    oracle.setPrice(0.5e18);

    // 4. Attempt liquidation — must revert with ABI decode error, not LiquidatorGatedFromLiquidating
    vm.expectRevert(); // ABI decoding revert, not the custom error
    midnight.liquidate(market, 0, 0, repaidUnits, borrower, false, receiver, address(0), "");
}
```

The test confirms: (a) market creation succeeds with `liquidatorGate = address(0xdead)`; (b) normal operations proceed; (c) `liquidate` reverts unconditionally with an ABI decoding error rather than `LiquidatorGatedFromLiquidating`, proving the `require` cannot catch it and liquidations are permanently bricked.
