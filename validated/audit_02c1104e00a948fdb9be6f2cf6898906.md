### Title
Permissionless `EcrecoverAuthorizer.setIsAuthorized()` Allows Frontrunning to Grief Honest Senders — (File: `src/periphery/EcrecoverAuthorizer.sol`)

---

### Summary

`EcrecoverAuthorizer.setIsAuthorized()` imposes no restriction on `msg.sender`. Any observer can extract a valid `(authorization, signature)` pair from the mempool and rebroadcast it with a higher gas price. The frontrunner's call succeeds, consuming the nonce, and the original sender's transaction reverts with `InvalidNonce()`, wasting their gas. Unlike the Futureswap analog, there is no direct financial reward for the frontrunner, but the attack is a realistic griefing vector with zero cost beyond gas.

---

### Finding Description

`EcrecoverAuthorizer.setIsAuthorized()` is a gasless-authorization relay: a user signs an `Authorization` struct off-chain and any party can submit it on-chain.

```solidity
// src/periphery/EcrecoverAuthorizer.sol  lines 24-48
function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
    require(block.timestamp <= authorization.deadline, Expired());
    require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
    ...
    IMidnight(MIDNIGHT)
        .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
}
```

There is **no `msg.sender` check**. The full `authorization` struct and `signature` are calldata, visible to every mempool observer.

**Attack path:**

1. User (or their relayer) broadcasts `setIsAuthorized(auth, sig)`.
2. Attacker copies `auth` and `sig` verbatim, rebroadcasts with a higher gas price.
3. Attacker's transaction lands first; `nonce[authorization.authorizer]` is incremented.
4. Original sender's transaction reverts at `require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce())`.
5. The authorization itself is correctly applied (the attacker's call did the right thing), but the original sender paid gas for a reverted transaction.

The nonce check at line 26 is the only replay guard, and it is consumed by whoever submits first.

---

### Impact Explanation

- The honest sender (user or relayer) wastes gas on every frontrun event.
- If the user is operating a time-sensitive workflow (e.g., authorizing a bundler just before taking an expiring offer), repeated griefing can cause them to miss the window entirely.
- The authorization itself is not blocked — the attacker cannot alter `authorized`, `isAuthorized`, or any other field because the signature covers the full struct. The user's intent is fulfilled, but at the user's expense.
- No direct asset theft occurs; severity is **Low / Griefing**.

---

### Likelihood Explanation

- The attack requires zero privilege: any Ethereum address can call `setIsAuthorized()`.
- The `authorization` and `signature` are fully public in the mempool.
- There is no financial incentive for a random attacker, so opportunistic bots are unlikely. However, a targeted competitor or a DoS-motivated actor can execute this at negligible cost (only the gas for the frontrunning call itself).
- Likelihood is **Low** in practice but the attack surface is permanently open.

---

### Recommendation

Bind the submission to a specific caller by adding a `relayer` field to the `Authorization` struct and enforcing it:

```solidity
require(
    authorization.relayer == address(0) || authorization.relayer == msg.sender,
    UnauthorizedRelayer()
);
```

Alternatively, allow only `authorization.authorizer` (or an address they have already authorized on Midnight) to submit the signed message, mirroring the fix applied in Futureswap PR #494 where only a trusted "stamper"-approved relayer may submit.

---

### Proof of Concept

1. Alice signs `Authorization { authorizer: Alice, authorized: BundlerBot, isAuthorized: true, nonce: 0, deadline: T+1h }` and broadcasts `EcrecoverAuthorizer.setIsAuthorized(auth, sig)`.
2. Attacker observes the pending transaction, copies `auth` and `sig`, and calls `setIsAuthorized(auth, sig)` with `gasPrice + 1 wei`.
3. Attacker's transaction is mined first. `nonce[Alice]` becomes `1`. `BundlerBot` is authorized on Midnight.
4. Alice's transaction is mined next. `require(0 == 1, InvalidNonce())` reverts. Alice pays gas for nothing.
5. Repeat for every subsequent authorization Alice attempts. [1](#0-0) [2](#0-1)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
