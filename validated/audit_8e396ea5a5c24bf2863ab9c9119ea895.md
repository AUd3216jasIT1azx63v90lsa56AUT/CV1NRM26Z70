Audit Report

## Title
Midnight-authorized agent can exhaust victim's `EcrecoverAuthorizer` nonce to invalidate pre-signed authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a valid ECDSA signature from any address for which `IMidnight.isAuthorized(authorizer, signer)` returns `true`, not only the authorizer themselves. An attacker who holds a Midnight authorization from the victim can sign and submit sequential `Authorization` structs naming the victim as `authorizer`, consuming `nonce[victim]` on each call without the victim's per-call consent. This permanently invalidates all pre-signed authorizations the victim distributed to other agents.

## Finding Description
**Root cause:**

In `EcrecoverAuthorizer.setIsAuthorized`, the signer check at lines 33–36 permits two valid signers:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

The second branch allows any address the victim has authorized on Midnight to act as a valid signer for the victim's `Authorization` struct. Combined with the nonce increment at line 26:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
``` [2](#0-1) 

a Midnight-authorized agent can consume `nonce[victim]` at will.

**Exploit flow:**
1. Victim calls `midnight.setIsAuthorized(attacker, true, victim)` — a normal, expected protocol action granting the attacker an agent role.
2. Victim pre-signs `K` `Authorization` structs (nonces 0 through K−1) and distributes them to K agents for future use (automated repay, liquidation protection, withdraw, etc.).
3. Attacker reads `nonce[victim] == 0` on-chain, constructs `Authorization{authorizer: victim, authorized: attacker, isAuthorized: true, nonce: 0, deadline: far}`, signs it with their own key, and calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
4. Line 26: nonce check passes (`0 == 0`), `nonce[victim]` becomes 1. Lines 31–36: `ecrecover` returns attacker's address; `IMidnight.isAuthorized(victim, attacker) == true` satisfies the guard.
5. Line 47 executes `midnight.setIsAuthorized(attacker, true, victim)` — a no-op since the authorization already exists.
6. Attacker repeats for nonces 1, 2, … K−1 in a tight loop or via `multicall`.
7. All K pre-signed authorizations held by the victim's agents now revert with `InvalidNonce()`.

**Why existing checks fail:**
The `Unauthorized()` guard is fully satisfied by the attacker's Midnight authorization. There is no requirement that the signer be the `authorizer` themselves when consuming a nonce. The `Authorization` struct fields `authorized` and `isAuthorized` are attacker-controlled and can be set to a no-op value. [3](#0-2) 

Note: the claim's assertion that "the nonce is committed regardless of signature verification" is technically imprecise — a failed `require` after line 26 would revert the increment. However, in the actual exploit path the signature check succeeds, so the nonce is permanently consumed.

## Impact Explanation
Any Midnight operation gated on `isAuthorized[victim][agent]` that the victim intended to enable via pre-signed `EcrecoverAuthorizer` authorizations — `take`, `repay`, `withdraw`, `withdrawCollateral`, `liquidate`, `claimFee`, `setConsumed` — is blocked. The most severe concrete outcome is that a borrower's pre-signed liquidation-protection or automated-repay authorizations are silently invalidated, leaving the position exposed to liquidation. The attacker can sustain the DoS indefinitely by front-running any new pre-signed authorization submission observed in the mempool. [4](#0-3) 

## Likelihood Explanation
**Precondition:** The attacker must hold `isAuthorized[victim][attacker] == true` on Midnight. This is a realistic, common precondition: users routinely authorize keeper contracts, relayers, and automated agents for legitimate protocol operations.

**Feasibility:** Each nonce-burn call costs one ECDSA verification and one storage write. On a low-gas chain the cost per nonce consumed is negligible. The attacker can front-run any mempool submission of a victim's pre-signed authorization.

**Repeatability:** Fully repeatable. Every time the victim generates and distributes new pre-signed authorizations, the attacker can burn the new nonces again.

## Recommendation
Restrict the signer to the `authorizer` themselves when consuming a nonce in `EcrecoverAuthorizer`. The `isAuthorized` branch should not be permitted to consume nonces on behalf of the authorizer. Concretely, change the signer check to:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegation is intentionally desired, introduce a separate, explicit per-delegation nonce or require the authorizer to co-sign any nonce-consuming delegation.

## Proof of Concept
**Minimal Foundry test outline:**
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. As `victim`, call `midnight.setIsAuthorized(attacker, true, victim)`.
3. As `victim`, sign `K` `Authorization` structs (nonces 0..K−1) off-chain and store them.
4. As `attacker`, for each nonce `i` in 0..K−1: construct `Authorization{authorizer: victim, authorized: attacker, isAuthorized: true, nonce: i, deadline: type(uint256).max}`, sign with attacker's key, call `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`. Assert each call succeeds and `nonce[victim]` increments.
5. Attempt to submit each of the victim's K pre-signed authorizations. Assert all revert with `InvalidNonce()`.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
