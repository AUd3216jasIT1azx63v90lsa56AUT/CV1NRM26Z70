Audit Report

## Title
Authorized Agent Can Grief Victim's Pending Pre-Signed Authorization by Consuming Nonce via No-Op Re-Authorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts signatures from any address that is an authorized agent of the `authorization.authorizer` on Midnight, not just the authorizer themselves. Because the nonce is incremented unconditionally on every successful call regardless of whether the underlying `Midnight.setIsAuthorized` produces any state change, an authorized agent can burn the authorizer's current nonce with a single no-op transaction, permanently invalidating any pending pre-signed authorization the authorizer has distributed off-chain. The attack is cheap, requires no privileged protocol access, and is repeatable indefinitely.

## Finding Description

**Exact code path — `src/periphery/EcrecoverAuthorizer.sol` lines 24–48:**

```solidity
// Line 26: nonce incremented unconditionally on success, before any signature check
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// Lines 28–31: digest computed over attacker-controlled struct
bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
...
address signer = ecrecover(digest, signature.v, signature.r, signature.s);

// Lines 33–36: accepts ANY existing authorized agent of the authorizer
require(
    signer == authorization.authorizer ||
    IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// Line 46–47: no-op if `authorized` is already authorized
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

**Root cause:** The `Unauthorized` check at line 34 is the same delegation check used throughout Midnight for on-chain actions (`src/Midnight.sol` line 732: `require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized())`). Applied here to a nonce-consuming signature flow, it allows any authorized agent to craft and sign a valid `Authorization` struct naming themselves as signer. The nonce of `authorization.authorizer` is incremented at line 26 and is not rolled back as long as the full transaction succeeds — including when the final `Midnight.setIsAuthorized` call is a no-op.

**Exploit flow:**

Preconditions:
- `isAuthorized[victim][attacker] == true` on Midnight (attacker is an existing authorized agent of victim — a normal user-level grant, not a protocol privilege).
- Victim has distributed off-chain a pre-signed `Authorization(authorizer=victim, authorized=X, isAuthorized=true, nonce=N, deadline=T)`.
- Any address `Y` already authorized by victim on Midnight exists (e.g., `dummyRatifier`, `ecrecoverAuthorizer`, any router/relayer).

Attack steps:
1. Attacker constructs `Authorization(authorizer=victim, authorized=Y, isAuthorized=true, nonce=N, deadline=future)` where `isAuthorized[victim][Y]` is already `true`.
2. Attacker signs this struct with their own private key, producing a valid ECDSA signature.
3. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(auth, attackerSig)`.

Execution trace:
- Line 25: deadline passes.
- Line 26: `N == nonce[victim]` → passes; `nonce[victim]` becomes `N+1`.
- Lines 28–31: digest computed; `signer = ecrecover(...)` = attacker.
- Line 32: `signer != address(0)` → passes.
- Line 34: `signer == victim` → false; `isAuthorized[victim][attacker]` → **true** → passes.
- Line 47: `Midnight.setIsAuthorized(Y, true, victim)` → no-op (already `true`).

Result: `nonce[victim]` is now `N+1`. Any relayer or counterparty submitting victim's pre-signed authorization with nonce `N` receives `InvalidNonce()` and the authorization is permanently invalidated.

**Why existing checks fail:** The `Unauthorized` check is designed to allow delegation for on-chain actions, but it does not distinguish between an agent acting to help the authorizer versus one acting to grief them. There is no guard preventing a no-op re-authorization from consuming the nonce, and no requirement that the signer be the authorizer themselves when the action is nonce-sensitive.

## Impact Explanation
Any pending pre-signed `Authorization` distributed by the victim — to a relayer, counterparty, or embedded in a protocol flow for `take`, `repay`, `withdraw`, `liquidate`, or `claimFee` — is permanently invalidated at the cost of one transaction. The victim must re-sign and re-distribute a new authorization with the incremented nonce. The attacker can repeat this indefinitely as long as they remain an authorized agent, creating a sustained DoS on all signature-based authorization flows for the victim. This constitutes service unavailability and severe degradation of the off-chain UX pattern that `EcrecoverAuthorizer` is specifically designed to support.

## Likelihood Explanation
Preconditions are realistic and common: users routinely authorize agents (routers, relayers, bots) on Midnight, and pre-signed authorizations are the primary off-chain UX pattern for `EcrecoverAuthorizer`. The attacker need only be one of the victim's existing authorized agents — a role that is common and expected in normal protocol usage. The attack costs one transaction, requires no oracle manipulation, no admin access, and no leaked keys. It is repeatable every time the victim issues a new pre-signed authorization, enabling sustained griefing.

## Recommendation
Remove the delegation branch from the `Unauthorized` check in `EcrecoverAuthorizer.setIsAuthorized`. The contract's purpose is to allow the authorizer to sign off-chain; only the authorizer themselves should be able to consume their own nonce:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegation is intentionally desired, add an explicit no-op guard before incrementing the nonce — check whether the resulting state change on Midnight would be a no-op and revert if so, or move the nonce increment to after all checks and only increment when the underlying call produces a real state change.

## Proof of Concept

Minimal Foundry test (extend `SetIsAuthorizedWithSigTest.sol`):

```solidity
function testAgentGriefsVictimNonce() public {
    // Setup: borrower authorizes ecrecoverAuthorizer and attacker on Midnight
    vm.prank(borrower);
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
    address attacker = makeAddr("attacker");
    vm.prank(borrower);
    midnight.setIsAuthorized(attacker, true, borrower);

    // Victim pre-signs authorization for lender at nonce 0
    Authorization memory victimAuth = makeAuthorization(borrower, lender, true);
    Signature memory victimSig = signAuthorization(victimAuth, borrower);

    // Attacker constructs no-op auth (ecrecoverAuthorizer already authorized) at same nonce
    Authorization memory attackAuth = Authorization({
        authorizer: borrower,
        authorized: address(ecrecoverAuthorizer), // already true → no-op
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 days
    });
    // Attacker signs with their own key (not borrower's)
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(
        uint256(keccak256(abi.encodePacked("attacker"))), // attacker's private key
        keccak256(bytes.concat("\x19\x01",
            keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(ecrecoverAuthorizer))),
            keccak256(abi.encode(AUTHORIZATION_TYPEHASH, attackAuth))
        ))
    );
    // Use makeAddrAndKey for proper key management in real test

    vm.prank(attacker);
    ecrecoverAuthorizer.setIsAuthorized(attackAuth, Signature({v: v, r: r, s: s}));

    // Nonce is now 1; victim's pre-signed auth at nonce 0 is invalidated
    assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

    // Victim's pre-signed authorization now fails
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}
```
