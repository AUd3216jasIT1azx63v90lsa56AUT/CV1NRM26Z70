Audit Report

## Title
Dust credit position (`credit=1`) fully wiped by any `lossFactor` increase due to `mulDivDown` rounding to zero - (File: src/Midnight.sol)

## Summary
In `updatePositionView`, the slashing formula `credit.mulDivDown(type(uint128).max - marketState[id].lossFactor, type(uint128).max - _lastLossFactor)` with `credit=1` always evaluates to `0` after any bad debt event, because the numerator is strictly less than the denominator and integer floor division truncates to zero. This destroys the entire 1-unit credit position when the proportional loss should round to 0 slash (lender retains 1 unit). The trigger function `updatePosition` is fully permissionless, allowing any caller to force this outcome on any victim.

## Finding Description
**Root cause:** `mulDivDown(a, b, c) = floor(a * b / c)` in `src/libraries/UtilsLib.sol:29-31`. With `credit=1`, the formula reduces to `floor((MAX - currentLossFactor) / (MAX - lastLossFactor))`. Since `lossFactor` is monotonically non-decreasing, after any bad debt event `currentLossFactor > lastLossFactor`, making the ratio strictly less than 1, which floors to 0.

**Code path:**
- `src/Midnight.sol:805-806`: `postSlashCredit = credit.mulDivDown(type(uint128).max - marketState[id].lossFactor, type(uint128).max - _lastLossFactor)`
- `src/Midnight.sol:842`: `_position.credit = newCredit` ŌĆö stores 0, destroying the position
- `src/Midnight.sol:823-827`: `updatePosition` ŌĆö no authorization check, callable by any address

**Exploit flow:**
1. Lender holds `credit=1` (reachable via small initial take or accumulated slashing/fee accrual reducing credit to 1 over time)
2. Any bad debt event occurs, increasing `marketState[id].lossFactor` above the lender's `position[id][lender].lastLossFactor`
3. Any address calls `updatePosition(market, victim)` ŌĆö no `msg.sender` restriction
4. `_updatePosition` ŌåÆ `updatePositionView` computes `postSlashCredit=0`, sets `_position.credit=0`

**Concrete example:**
- `totalUnits=1e18`, `badDebt=1`, `credit=1`, `lastLossFactor=0`
- New `lossFactor = MAX - floor(MAX * (1e18-1) / 1e18) Ōēł MAX/1e18`
- `postSlashCredit = floor(1 * (MAX - MAX/1e18) / MAX) = floor((1e18-1)/1e18) = 0`
- Correct proportional slash: `floor(1 * 1/1e18) = 0` ŌåÆ lender should retain 1 unit
- Actual result: lender retains 0 units

**Why existing checks fail:**
- Certora `updatePositionEffects` only asserts `creditOf(id, user) <= creditBefore` ŌĆö does not verify proportionality
- `LossFactor.spec:updatePositionDoesNotRevert` checks non-reversion only, not slash correctness
- No minimum-credit guard or floor exists in `_updatePosition` or `updatePositionView`
- `EcrecoverAuthorizer` is irrelevant ŌĆö `updatePosition` is unconditionally permissionless

## Impact Explanation
Any lender whose credit has been reduced to exactly 1 unit loses that unit entirely upon the next `updatePosition` call after any bad debt event, regardless of how small the bad debt is relative to `totalUnits`. This violates the protocol's proportional slashing invariant for dust positions. Maximum loss per victim is 1 unit of credit; no value is transferred to the attacker.

## Likelihood Explanation
Both preconditions (a lender holding `credit=1`, and at least one bad debt event since their last update) are organic and require no special privileges. Credit can reach 1 through accumulated slashing and continuous fee accrual over time. Bad debt events are a normal part of the protocol lifecycle. The trigger is permissionless and callable by any address, making the scenario repeatable across any market with bad debt history.

## Recommendation
Add a minimum-credit preservation guard in `updatePositionView`: if `credit > 0` and `postSlashCredit == 0` but the computed proportional slash is less than the full credit (i.e., `currentLossFactor < type(uint128).max`), preserve at least 1 unit of credit. Alternatively, apply `mulDivUp` for the slash amount rather than `mulDivDown` for the remaining credit, ensuring dust positions are not over-slashed. A concrete fix:

```solidity
uint256 postSlashCredit = _lastLossFactor < type(uint128).max
    ? credit.mulDivDown(type(uint128).max - marketState[id].lossFactor, type(uint128).max - _lastLossFactor)
    : 0;
// Preserve dust: if credit was nonzero and not fully slashed, retain at least 1
if (credit > 0 && postSlashCredit == 0 && marketState[id].lossFactor < type(uint128).max) {
    postSlashCredit = 1;
}
```
