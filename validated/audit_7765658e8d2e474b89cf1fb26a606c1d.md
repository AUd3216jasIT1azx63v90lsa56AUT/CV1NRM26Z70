Audit Report

## Title
Authorized Operator Can Front-Run and Permanently Invalidate Authorizer's Pending Signed Authorization — (`src/periphery/EcrecoverAuthorizer.sol`)

## Summary

`EcrecoverAuthorizer.setIsAuthorized` increments `nonce[authorization.authorizer]` at line 26 before signer identity is verified at lines 33–36. Because the signer check at line 34 accepts any address for which `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns `true`, any previously-authorized operator can construct a fresh `Authorization` struct naming the victim as `authorizer`, sign it with their own key, and submit it with higher gas to consume the victim's current nonce. The victim's pending signed message is permanently invalidated, and the attack is indefinitely repeatable.

## Finding Description

**Root cause — nonce bound to `authorization.authorizer`, not to the recovered signer, combined with the operator-signer branch:** [1](#0-0) 

The nonce for `authorization.authorizer` is incremented as soon as the nonce equality check passes. Signer identity is not resolved until lines 31–36: [2](#0-1) 

The second branch of the `||` at line 34 accepts any `signer` for which `isAuthorized[authorization.authorizer][signer] == true` — i.e., any operator the authorizer has previously authorized via `Midnight.setIsAuthorized`.

**Attacker-controlled inputs:**

The `Authorization` struct exposes `authorizer` as a plain, unconstrained field: [3](#0-2) 

Nothing in `setIsAuthorized` binds `authorization.authorizer` to `msg.sender` or to the recovered signer. The attacker freely sets `authorizer = victim` and signs the struct with their own key.

**Exploit flow:**

1. Victim signs `Authorization{authorizer=victim, authorized=X, nonce=N, deadline=T}` and broadcasts.
2. Attacker (holding `isAuthorized[victim][attacker] == true`) observes the pending tx, reads `N = nonce[victim]`.
3. Attacker constructs `Authorization{authorizer=victim, authorized=Y, nonce=N, deadline=T2}`, signs with attacker's key, submits with higher gas.
4. Attacker's tx: nonce check `N == N` passes → `nonce[victim]` becomes `N+1`; `ecrecover` returns attacker's address; `isAuthorized[victim][attacker] == true` → `Unauthorized()` check passes; `isAuthorized[victim][Y]` is set.
5. Victim's tx: nonce check `N == N+1` fails → `InvalidNonce()` revert.
6. Victim must re-sign with nonce `N+1`; attacker repeats indefinitely.

**Why existing checks fail:**

The `Unauthorized()` guard is designed to prevent strangers from acting on behalf of the authorizer, but it explicitly permits any previously-authorized operator. There is no requirement that the signer must be the authorizer themselves when consuming the authorizer's nonce. The `Midnight.setIsAuthorized` direct-call path confirms this operator permission is by design: [4](#0-3) 

The `EcrecoverAuthorizer` mirrors this permission but introduces the nonce-griefing side-effect: an operator can sign a *different* `Authorization` payload (with `authorizer=victim`) and consume the victim's nonce without the victim's knowledge or consent for that specific message.

## Impact Explanation

Any authorized operator can permanently invalidate any pending signed authorization message from the authorizer by front-running with a higher-gas transaction that consumes the same nonce. The attacker can repeat this for every new nonce the victim signs, creating an indefinite DoS on the `EcrecoverAuthorizer` signed-message delegation path. If the victim needs to establish or revoke an authorization via a signed message (e.g., for a time-sensitive gate-restricted operation with a deadline), they are locked out of that path. The victim retains the ability to call `Midnight.setIsAuthorized` directly as `msg.sender`, but the signed-message delegation path is fully griefable. [5](#0-4) 

## Likelihood Explanation

**Preconditions:** `isAuthorized[victim][attacker] == true` — the attacker must have been previously authorized by the victim. This is a normal protocol state (e.g., victim authorized a keeper, relayer, or another user). `Midnight.setIsAuthorized` explicitly supports authorized operators acting on behalf of users, making this a common and expected state. [6](#0-5) 

**Feasibility:** Mempool front-running is standard on non-private mempools. The attacker only needs to observe the victim's pending tx and submit with higher gas. **Repeatability:** The attack is repeatable for every nonce the victim signs, making it a persistent DoS with no natural termination. The only escape for the victim is to revoke the attacker's authorization via a direct `Midnight.setIsAuthorized` call (not via `EcrecoverAuthorizer`), then re-sign.

## Recommendation

Remove the `isAuthorized` branch from the signer check in `EcrecoverAuthorizer.setIsAuthorized`. Require that the recovered signer is exactly `authorization.authorizer`:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

Authorized operators who need to relay a signed authorization can already do so: they submit the *authorizer's own signature* (where `ecrecover` returns the authorizer's address), satisfying `signer == authorization.authorizer` without any change. The `isAuthorized` branch is only needed if operators are expected to sign with their *own* key on behalf of the authorizer — a capability that introduces the nonce-griefing vector and is not required for the relay use-case.

Alternatively, if operator-signed authorizations are intentionally supported, bind the nonce to the *signer* rather than to `authorization.authorizer`, so an operator's submission consumes only the operator's own nonce and cannot invalidate the authorizer's pending messages.

## Proof of Concept

Minimal Foundry test (extend `EcrecoverAuthorizerTest` / `BaseTest`):

```solidity
function testOperatorFrontRunsVictimNonce() public {
    // Setup: victim authorizes attacker via direct call
    vm.prank(borrower);          // borrower = victim
    midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, borrower);
    vm.prank(borrower);
    midnight.setIsAuthorized(lender, true, borrower); // lender = attacker

    // Victim prepares a signed authorization (nonce = 0)
    Authorization memory victimAuth = makeAuthorization(borrower, otherBorrower, true);
    Signature memory victimSig = signAuthorization(victimAuth, borrower);

    // Attacker front-runs: constructs a different Authorization with same nonce,
    // signs with attacker's own key, submits first
    Authorization memory attackerAuth = makeAuthorization(borrower, otherLender, true);
    // nonce is still 0 — same as victim's
    Signature memory attackerSig = signAuthorization(attackerAuth, lender);

    // Attacker's tx succeeds: lender is authorized by borrower → passes isAuthorized check
    ecrecoverAuthorizer.setIsAuthorized(attackerAuth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(borrower), 1); // nonce consumed

    // Victim's tx now fails: nonce 0 != 1
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(victimAuth, victimSig);
}
``` [7](#0-6) 

The existing `testEcrecoverAuthorizerInvalidSignature` test (line 88–97) shows the nonce is *not* consumed on a failed `Unauthorized` revert (nonce stays 0), confirming that only a *successful* attacker transaction — one where the attacker is a legitimate authorized operator — permanently consumes the nonce.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
