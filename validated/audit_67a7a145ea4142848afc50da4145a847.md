Audit Report

## Title
Unconditional zero-value `safeTransferFrom` on zero settlement fee permanently DoS's `take()` for non-standard loan tokens - (File: `src/Midnight.sol`)

## Summary
When all settlement fee breakpoints are zero (the default for any token not explicitly configured), `buyerAssets - sellerAssets` evaluates to exactly `0`. `take()` unconditionally calls `SafeTransferLib.safeTransferFrom(loanToken, payer, address(this), 0)` at line 455 with no zero-value guard. Loan tokens that revert on zero-value `transferFrom` calls will cause every `take()` in that market to permanently revert, freezing the market's core function.

## Finding Description

**Root cause:** `SafeTransferLib.safeTransferFrom` has no zero-value guard and always issues the low-level `transferFrom` call: [1](#0-0) 

In `take()`, prices are computed at lines 360–364: [2](#0-1) 

When `_settlementFee == 0`, `buyerPrice == sellerPrice`, and since both `buyerAssets` and `sellerAssets` use the same rounding direction per offer side, `buyerAssets - sellerAssets == 0` exactly.

Line 418 silently adds 0 to `claimableSettlementFee` — no revert there: [3](#0-2) 

Then, unconditionally at line 455, the zero-value pull is issued: [4](#0-3) 

The `payer` is determined at line 422 — even without a `buyerCallback`, `payer` resolves to `buyer` or `msg.sender`, so the zero-value transfer is always attempted: [5](#0-4) 

**Default state:** `touchMarket` copies `defaultSettlementFeeCbp[market.loanToken]` for all 7 breakpoints. For any token not explicitly configured by the fee setter, all breakpoints are 0: [6](#0-5) 

**Exploit flow:**
1. Deploy or use a loan token that reverts on zero-value `transferFrom` (known real-world pattern: LEND, some rebasing/fee-on-transfer tokens).
2. Create a market using that token — `touchMarket` copies `defaultSettlementFeeCbp` which is 0 for all indices unless the fee setter has explicitly configured it.
3. Call `take()` — execution reaches line 455, `safeTransferFrom(..., 0)` is issued, the token reverts, the entire `take()` reverts.
4. Every subsequent `take()` call in that market also reverts — the DoS is permanent until the fee setter raises the settlement fee above zero.

**Why existing checks fail:** There is no `if (buyerAssets > sellerAssets)` guard before line 455. The `claimableSettlementFee` accounting at line 418 already handles the zero case silently, but the transfer is not similarly guarded.

## Impact Explanation
Any market whose loan token reverts on zero-value transfers is permanently DoS'd for `take()` whenever the settlement fee is zero. All state changes made before line 455 (position updates, consumed tracking, credit/debt accounting) are reverted, so no funds are lost — but valid takes are completely blocked. This is a concrete fund-freeze / liveness failure for affected markets, matching the "Permanent lock, freeze, or unrecoverable corruption of user/project state" impact class in RESEARCHER.md.

## Likelihood Explanation
- Zero settlement fee is the **default** state: `touchMarket` copies `defaultSettlementFeeCbp[loanToken]`, which is `0` for any token not explicitly configured by the fee setter.
- Tokens that revert on zero-value transfers exist in production (e.g., LEND). The protocol places no restriction on which ERC-20 can be a loan token.
- The condition is fully repeatable: every `take()` call in the affected market reverts until the fee setter intervenes.
- No privileged action by the attacker is required; any taker triggers it.

## Recommendation
Add a zero-value guard before the settlement fee transfer at line 455:

```solidity
uint256 settlementFeeAmount = buyerAssets - sellerAssets;
if (settlementFeeAmount > 0) {
    SafeTransferLib.safeTransferFrom(
        offer.market.loanToken, payer, address(this), settlementFeeAmount
    );
}
```

This mirrors the silent handling already applied to `claimableSettlementFee` at line 418 and is consistent with how zero-value edge cases are handled elsewhere in the codebase.

## Proof of Concept
1. Deploy a mock ERC-20 loan token whose `transferFrom` reverts when `amount == 0`.
2. Call `touchMarket` with that token (do **not** call `setDefaultSettlementFee` first — leave all breakpoints at 0).
3. Create a valid offer and call `take()` with any non-zero `units`.
4. Observe revert at line 455 due to the zero-value `transferFrom`.
5. Confirm no state is mutated (all changes rolled back).
6. Confirm the revert is reproducible on every subsequent `take()` call in the same market.

Minimal Foundry test sketch:
```solidity
function testZeroFeeDoS() public {
    // loanToken reverts on transferFrom(_, _, 0)
    MockRevertOnZeroToken token = new MockRevertOnZeroToken();
```

```text
    function safeTransferFrom(address token, address from, address to, uint256 value) internal {
        require(token.code.length > 0, NoCode());

        (bool success, bytes memory returndata) = token.call(abi.encodeCall(IERC20.transferFrom, (from, to, value)));
```
