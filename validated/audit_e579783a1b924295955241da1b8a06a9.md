After thoroughly analyzing the Morpho Midnight codebase against the Y2K vulnerability class (missing state check allowing wrong resolution path, causing fund loss), I mapped every relevant state transition in `src/Midnight.sol`:

**Vulnerability class**: State transition — a function callable in a state where it shouldn't be, causing incorrect fund movement.

**Mapping attempt to Midnight**:

The Y2K analog requires:
1. Two resolution paths for a state
2. A missing guard allowing the wrong path
3. Incorrect fund transfer as a result

Midnight has no epoch-resolution functions. The closest analogs I examined:

- **`liquidate` with `badDebt > 0` when `totalUnits == 0`**: Cannot occur. The invariant `totalUnits = total_debt + withdrawable` is maintained through all code paths in `take`, `repay`, `withdraw`, and `liquidate`. Since `badDebt ≤ _position.debt ≤ total_debt ≤ totalUnits`, `totalUnits == 0` implies `badDebt == 0`. [1](#0-0) 

- **`liquidate` with `postMaturityMode = false` post-maturity**: Allowed by design; normal mode is more restrictive (RCF applies), not less. No fund loss path. [2](#0-1) 

- **`withdraw` when `withdrawable == 0`**: Reverts via underflow in Solidity 0.8+. [3](#0-2) 

- **`take` post-maturity with `sellerDebtIncrease > 0`**: Blocked by `CannotIncreaseDebtPostMaturity`. [4](#0-3) 

- **`_updatePosition` skipped when `hasCredit == false`**: Correct — no credit means no slashing or fee accrual needed. <cite repo="Jakeconolly/midnight--001" path="src/Midnight.sol"
