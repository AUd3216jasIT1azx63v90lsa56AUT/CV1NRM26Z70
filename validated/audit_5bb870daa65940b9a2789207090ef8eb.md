Audit Report

## Title
Malicious delegate front-runs maker's EcrecoverAuthorizer revocation to permanently grief offer groups - (File: `src/periphery/EcrecoverAuthorizer.sol`)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` increments the maker's nonce before verifying the signature, and accepts signatures from any existing delegate of the maker. A malicious delegate can exploit this ordering to consume the maker's current nonce with a crafted authorization, permanently invalidating the maker's pending revocation. The delegate then uses the newly authorized attacker address to call `Midnight.setConsumed(group, type(uint256).max, maker)`, irreversibly cancelling all of the maker's offers in that group.

## Finding Description

**Root cause 1 — Nonce consumed before signature verification:**

In `EcrecoverAuthorizer.setIsAuthorized`, line 26 increments `nonce[authorization.authorizer]` atomically before the signature is checked:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
``` [1](#0-0) 

Once any transaction consuming nonce N lands, the maker's signed message using nonce N is permanently invalid.

**Root cause 2 — Delegates can sign authorizations on behalf of the maker:**

Lines 33–36 accept a signature from any address that `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns true for:

```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [2](#0-1) 

This means a delegate can construct and sign an `Authorization` struct with `authorization.authorizer = maker` and arbitrary `authorized`/`isAuthorized` values, and the contract will accept it as if the maker signed it.

**Root cause 3 — `setConsumed` with `type(uint256).max` is irreversible:**

The `setConsumed` function enforces strict monotonicity via `AlreadyConsumed`, and `consumed` can never decrease. Setting it to `type(uint256).max` permanently blocks every subsequent `take` on any offer in that group (overflow past `maxUnits`/`maxAssets`). [3](#0-2) 

**Full exploit path:**

| Step | Actor | Action | Effect |
|------|-------|--------|--------|
| 0 | maker | `Midnight.setIsAuthorized(ecrecoverAuthorizer, true, maker)` | EcrecoverAuthorizer becomes a delegate |
| 1 | maker | `Midnight.setIsAuthorized(delegate, true, maker)` | delegate is authorized |
| 2 | maker | Signs `Authorization{maker, delegate, false, N, deadline}`, broadcasts | Pending in mempool |
| 3 | delegate | Signs `Authorization{maker, attacker, true, N, deadline2}` | Off-chain |
| 4 | delegate | `EcrecoverAuthorizer.setIsAuthorized(auth3, sig3)` (front-run) | `nonce[maker]→N+1`, `isAuthorized[maker][attacker]=true` |
| 5 | maker | Maker's tx lands | Reverts `InvalidNonce` |
| 6 | attacker | `Midnight.setConsumed(group, type(uint256).max, maker)` | `consumed[maker][group]=type(uint256).max` — permanent |

The maker can still call `Midnight.setIsAuthorized(delegate, false, maker)` directly to revoke the delegate, but `consumed[maker][group]` is already `type(uint256).max` and cannot be lowered.

## Impact Explanation

All of the maker's offers sharing the griefed `group` are permanently cancelled — every `take` on them reverts with `ConsumedUnits` or `ConsumedAssets`. The maker cannot recover the group budget because `setConsumed` enforces strict monotonicity. The maker must redeploy offers under a new group identifier, losing any off-chain routing or aggregator integrations that referenced the old group. This constitutes permanent, irreversible corruption of user state — a concrete in-scope impact class per RESEARCHER.md ("Permanent lock, freeze, or unrecoverable corruption of user/project state"). [4](#0-3) 

## Likelihood Explanation

**Preconditions:**
1. Maker has opted into `EcrecoverAuthorizer` — required for any signature-based authorization flow, common in production setups.
2. Maker has at least one active delegate — standard for automated market-making.
3. Maker attempts to revoke the delegate via a signed message rather than a direct `Midnight.setIsAuthorized` call — plausible when the maker is a smart contract or uses a meta-transaction relay.

The front-run requires no special knowledge beyond the delegate's own private key and the current nonce (publicly readable). The attack is repeatable — every time the maker tries to revoke via `EcrecoverAuthorizer`, the delegate can repeat the nonce-burn. Only one successful front-run is needed to cause permanent damage via `setConsumed`.

## Recommendation

1. **Move nonce increment after signature verification** in `EcrecoverAuthorizer.setIsAuthorized` — read the nonce, verify the signature, then increment. This prevents nonce-burning by any party before the signature is validated.
2. **Restrict who can sign authorizations**: Only allow `authorization.authorizer` themselves (not delegates) to sign `Authorization` structs in `EcrecoverAuthorizer`. Delegates already have the ability to call `Midnight.setIsAuthorized` directly; allowing them to also sign gasless authorizations on behalf of the maker creates an asymmetric griefing vector.
3. **Consider making `setConsumed` reversible downward by the maker directly**, or add a separate "cancel group" mechanism that does not use the same monotonic counter, to limit the blast radius of any authorization abuse.

## Proof of Concept

**Minimal Foundry test outline:**

```solidity
// 1. Deploy Midnight + EcrecoverAuthorizer
// 2. maker calls midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, maker)
// 3. maker calls midnight.setIsAuthorized(delegate, true, maker)
// 4. maker signs Authorization{maker, delegate, false, nonce=0, deadline=far}
// 5. delegate signs Authorization{maker, attacker, true, nonce=0, deadline=far}
// 6. delegate calls ecrecoverAuthorizer.setIsAuthorized(auth5, sig5) — front-run
//    Assert: nonce[maker] == 1, isAuthorized[maker][attacker] == true
// 7. maker's tx (step 4) reverts with InvalidNonce
// 8. attacker calls midnight.setConsumed(group, type(uint256).max, maker)
//    Assert: consumed[maker][group] == type(uint256).max
// 9. taker attempts take() on any offer in group — reverts ConsumedUnits
//    Assert: offer group is permanently dead
```

The existing test file `test/SetIsAuthorizedWithSigTest.sol` provides the scaffolding for steps 2–7. [5](#0-4)
