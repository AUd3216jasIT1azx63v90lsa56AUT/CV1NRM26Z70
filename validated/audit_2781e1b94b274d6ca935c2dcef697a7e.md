Audit Report

## Title
Unvalidated Non-Contract `liquidatorGate` Permanently Bricks Liquidations - (File: src/Midnight.sol)

## Summary

`touchMarket` performs no validation that `market.liquidatorGate` is `address(0)` or a deployed contract. Any unprivileged caller can create a market with `liquidatorGate` set to an arbitrary non-contract address. Every subsequent call to `liquidate()` unconditionally reverts due to Solidity ABI-decoding failure when calling `ILiquidatorGate.canLiquidate` on a codeless address, permanently disabling liquidations for that market and enabling unbounded bad-debt accumulation at lenders' expense.

## Finding Description

**Root cause — missing contract check in `touchMarket` (src/Midnight.sol lines 755–791):**

`touchMarket` validates `maturity`, collateral count, token ordering, `lltv`, and `maxLif`, but performs no check on `market.liquidatorGate`:

```solidity
// src/Midnight.sol lines 757–773 — no liquidatorGate validation
require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
require(market.collateralParams.length > 0, NoCollateralParams());
// ... collateral checks only ...
// liquidatorGate is never inspected
```

The `Market` struct (`src/interfaces/IMidnight.sol` line 11) accepts any `address` for `liquidatorGate` with no constraint.

**Trigger — `liquidate()` calls the interface on the codeless address (src/Midnight.sol lines 597–600):**

```solidity
require(
    market.liquidatorGate == address(0) || ILiquidatorGate(market.liquidatorGate).canLiquidate(msg.sender),
    LiquidatorGatedFromLiquidating()
);
```

When `liquidatorGate = address(0xdead)`:
1. `market.liquidatorGate == address(0)` → `false`; short-circuit does not fire.
2. `ILiquidatorGate(address(0xdead)).canLiquidate(msg.sender)` is evaluated.
3. The EVM `CALL` to a codeless address returns `(success=true, returndata=<empty>)`.
4. Solidity 0.8.34's ABI decoder expects ≥ 32 bytes for the `bool` return value; it receives 0 bytes → **unconditional revert** (ABI decoding error), not `LiquidatorGatedFromLiquidating`.
5. The `require` guard cannot catch this: a revert thrown *inside* the condition expression propagates unconditionally; `require` only catches a `false` return.

`ILiquidatorGate.canLiquidate` is declared to return `bool` (`src/interfaces/IGate.sol` line 11), confirming the 32-byte expectation.

**Exploit flow:**
1. Attacker calls `touchMarket(market)` with `market.liquidatorGate = address(0xdead)` and otherwise valid parameters. Market is created successfully.
2. Attacker seeds the market with attractive offers; lenders and borrowers interact normally (`take`, `repay`, `supplyCollateral` are unaffected).
3. Borrower's position becomes unhealthy (e.g., oracle price drop).
4. Any call to `liquidate(market, ...)` hits the `canLiquidate` call → ABI-decode revert → liquidation permanently impossible.
5. Bad debt accumulates; lenders cannot recover funds.

**Existing protections are insufficient:** The LIVENESS comment at line 152 ("If `liquidatorGate.canLiquidate` reverts or returns false, `liquidate` reverts") documents the behavior of a *legitimate* gate that may transiently revert. It does not constitute acceptance of a permanently broken non-contract gate created by an adversary. No `extcodesize` or equivalent guard exists anywhere in the creation or liquidation path.

## Impact Explanation

Unhealthy borrowers in any market created with a non-contract, non-zero `liquidatorGate` can never be liquidated. Bad debt accumulates without bound. Lenders suffer permanent, unrecoverable fund loss. The core protocol invariant — "unhealthy positions remain liquidatable" — is permanently violated for the affected market. This constitutes a direct, concrete loss of lender funds and falls squarely within the in-scope impact class of permanent freeze/loss of user assets.

## Likelihood Explanation

- **Privilege required:** None. `touchMarket` is public and permissionless.
- **Feasibility:** Trivial. One transaction suffices to create the malformed market.
- **Repeatability:** The market ID is deterministic from its parameters; once created, `liquidatorGate` is immutable (encoded in the market struct via `IdLib.storeInCode`). The DoS is permanent.
- **Victim attraction:** The attacker can seed the market with competitive offers to draw in lenders and borrowers before positions become unhealthy.

## Recommendation

In `touchMarket`, add an `extcodesize` check for `liquidatorGate` (and symmetrically for `enterGate`):

```solidity
require(
    market.liquidatorGate == address(0) || market.liquidatorGate.code.length > 0,
    InvalidLiquidatorGate()
);
require(
    market.enterGate == address(0) || market.enterGate.code.length > 0,
    InvalidEnterGate()
);
```

This ensures that any non-zero gate address is a deployed contract capable of returning a valid ABI-encoded `bool`, preventing the permanent liquidation DoS.

## Proof of Concept

**Minimal Foundry test outline:**

```solidity
function testNonContractLiquidatorGateBricksLiquidations() public {
    // 1. Create market with non-contract liquidatorGate
    market.liquidatorGate = address(0xdead);
    midnight.touchMarket(market); // succeeds

    // 2. Borrower supplies collateral and takes a loan
    collateralize(market, borrower, units);
    take(units, lender, borrowerOffer);

    // 3. Drop oracle price to make borrower unhealthy
    oracle.setPrice(ORACLE_PRICE_SCALE / 10);

    // 4. Attempt liquidation — reverts with ABI decoding error, not LiquidatorGatedFromLiquidating
    vm.expectRevert(); // ABI decode revert, not a named error
    midnight.liquidate(market, 0, 1, 0, borrower, false, address(this), address(0), "");
}
```

The test confirms that `liquidate` reverts unconditionally (not with `LiquidatorGatedFromLiquidating`) for every caller, every time, permanently bricking liquidations for the market.
