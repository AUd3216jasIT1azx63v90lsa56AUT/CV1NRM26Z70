Audit Report

## Title
Missing `loanToken != address(0)` validation in `touchMarket` allows creation of permanently broken, undeletable markets - (File: src/Midnight.sol)

## Summary
`touchMarket` is a public, permissionless function that validates collateral tokens, LLTV tiers, maxLif, and maturity but contains no guard against `market.loanToken == address(0)`. A market created with `loanToken = address(0)` is permanently stored in protocol state and cannot be deleted. Every subsequent `take()`, `withdraw()`, and `repay()` call on that market reverts with `SafeTransferLib.NoCode()` because `address(0).code.length == 0`, and `TakeAmountsLib.buyerAssetsToUnits` returns nonzero values for offers on this market, falsely signaling to integrators that takes are executable.

## Finding Description

**Root cause — `touchMarket` (`src/Midnight.sol`, lines 755–791):**

The creation guard checks maturity, collateral count, sort order, LLTV tier, and maxLif, but contains no `require(market.loanToken != address(0))`: [1](#0-0) 

The collateral sort check (`collateralToken > previousCollateralToken`, where `previousCollateralToken` starts at `address(0)`) only prevents collateral tokens from being `address(0)` — it does not protect the loan token field: [2](#0-1) 

When `market.loanToken = address(0)`, the market is created with all-zero settlement fees (since `defaultSettlementFeeCbp[address(0)]` returns a zero-initialized array) and is permanently stored: [3](#0-2) 

**Trigger — `take()` (`src/Midnight.sol`, lines 455–456):**

Every `take()` call reaches the token transfer step and calls `SafeTransferLib.safeTransferFrom` with `offer.market.loanToken = address(0)`, which reverts because `address(0).code.length == 0`: [4](#0-3) 

The same revert occurs in `withdraw()` (line 499) and `repay()` (line 520): [5](#0-4) [6](#0-5) 

**Market immutability confirmed by Certora spec (`certora/specs/CreatedMarkets.spec`, lines 82–86):** [7](#0-6) 

**Misleading helper — `TakeAmountsLib.buyerAssetsToUnits` (`src/periphery/TakeAmountsLib.sol`, lines 17–30):**

With `settlementFee = 0` (from the zero-initialized market) and `buyerPrice = offerPrice` (nonzero for any valid tick), the function returns a nonzero unit count — signaling to callers that a take is feasible when it will always revert: [8](#0-7) 

**Confirmed reachable — `test/SettersTest.sol`, lines 267–276:**

The test suite explicitly creates a market with `loanToken: address(0)` and calls `midnight.touchMarket(market)` successfully, confirming the path is exercised without error: [9](#0-8) 

## Impact Explanation

Any market created with `loanToken = address(0)` is permanently broken and unrecoverable: all `take()`, `withdraw()`, and `repay()` calls revert with `NoCode()`, the market state is immutable (no deletion mechanism exists per the Certora invariant), and `TakeAmountsLib.buyerAssetsToUnits` returns nonzero values that falsely indicate takes are executable. Because debt can only be created via `take()` (which always reverts), no debt positions can form, so collateral deposited to such a market remains withdrawable. The primary concrete impacts are (1) permanent unrecoverable corruption of protocol state — a broken market ID is permanently occupied and cannot be reclaimed — and (2) integrators relying on `TakeAmountsLib` will receive misleading feasibility signals, potentially causing failed transactions and degraded service. This matches the "permanent lock/freeze or unrecoverable corruption of user/project state" and "service unavailability or severe degradation" impact classes in RESEARCHER.md.

## Likelihood Explanation

`touchMarket` is a public, permissionless function — no role, signature, or special permission is required. The precondition (supply `loanToken = address(0)` with any valid collateral/LLTV/maturity) is trivially satisfiable in a single transaction. The attack is repeatable for any `(address(0), maturity)` pair not yet created, and the broken market is immutable once deployed. The test suite already exercises this exact path without error, confirming zero friction for an attacker.

## Recommendation

Add a zero-address check for `market.loanToken` at the top of the market creation block in `touchMarket`:

```solidity
require(market.loanToken != address(0), InvalidLoanToken());
```

This should be placed alongside the existing input validation checks (e.g., after the maturity check at line 758) to prevent any market with a zero loan token from being permanently stored.

## Proof of Concept

The existing test at `test/SettersTest.sol` lines 267–276 already demonstrates the reachable path. A minimal PoC extending it:

1. Call `midnight.touchMarket(market)` with `loanToken = address(0)`, valid collateral, LLTV, and maturity — succeeds silently.
2. Attempt `midnight.take(offer, ...)` referencing this market — reverts with `NoCode()`.
3. Attempt `midnight.withdraw(market, ...)` — reverts with `NoCode()`.
4. Attempt `midnight.repay(market, ...)` — reverts with `NoCode()`.
5. Call `TakeAmountsLib.buyerAssetsToUnits` for an offer on this market — returns nonzero, falsely indicating the take is executable.
6. Confirm via `tickSpacing(id) > 0` that the market is permanently stored and cannot be removed.

    function touchMarket(Market memory market) public returns (bytes32) {
        bytes32 id = toId(market);
        if (marketState[id].tickSpacing == 0) {
            require(market.maturity <= block.timestamp + 100 * 365 days, MaturityTooFar());
            require(market.collateralParams.length > 0, NoCollateralParams());
            require(market.collateralParams.length <= MAX_COLLATERALS, TooManyCollateralParams());
            address previousCollateralToken;
            for (uint256 i = 0; i < market.collateralParams.length; i++) {
                address collateralToken = market.collateralParams[i].token;
                require(collateralToken > previousCollateralToken, CollateralParamsNotSorted());
                uint256 lltv = market.collateralParams[i].lltv;
                require(isLltvAllowed(lltv), LltvNotAllowed());
                require(
                    market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_LOW)
                        || market.collateralParams[i].maxLif == maxLif(lltv, LIQUIDATION_CURSOR_HIGH),
                    InvalidMaxLif()
                );
                previousCollateralToken = collateralToken;
            }

            MarketState storage _marketState = marketState[id];
            _marketState.tickSpacing = DEFAULT_TICK_SPACING;
            uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
            _marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
            _marketState.settlementFeeCbp1 = _defaultSettlementFeeCbp[1];
            _marketState.settlementFeeCbp2 = _defaultSettlementFeeCbp[2];
            _marketState.settlementFeeCbp3 = _defaultSettlementFeeCbp[3];
            _marketState.settlementFeeCbp4 = _defaultSettlementFeeCbp[4];
            _marketState.settlementFeeCbp5 = _defaultSettlementFeeCbp[5];
            _marketState.settlementFeeCbp6 = _defaultSettlementFeeCbp[6];
            _marketState.continuousFee = defaultContinuousFee[market.loanToken];
            IdLib.storeInCode(market, INITIAL_CHAIN_ID);

            emit EventsLib.MarketCreated(market, id);
        }
        return id;
    }
