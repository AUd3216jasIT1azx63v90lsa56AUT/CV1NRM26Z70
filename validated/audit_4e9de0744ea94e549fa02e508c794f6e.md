### Title
Sponsor Can Redirect All CORRUPTED Sweep Funds by Changing `recoveryAddress` After Stakers Deposit — (File: src/ConfidencePool.sol)

### Summary
`setRecoveryAddress` imposes no post-stake lock, unlike `setExpiry` which is permanently frozen by `expiryLocked` after the first deposit. A malicious sponsor can silently redirect the entire pool — stakers' principal plus bonus — to an address they control by updating `recoveryAddress` at any point before `claimCorrupted` or `sweepUnclaimedCorrupted` executes.

### Finding Description
`setExpiry` is guarded by the one-way `expiryLocked` latch that trips on the first `stake` call, explicitly to protect staker reliance on a fixed deadline. [1](#0-0) 

`setRecoveryAddress` carries no equivalent guard:

```solidity
function setRecoveryAddress(address newRecoveryAddress) external onlyOwner {
    if (newRecoveryAddress == address(0)) revert InvalidRecoveryAddress();
    address oldRecoveryAddress = recoveryAddress;
    recoveryAddress = newRecoveryAddress;
    emit RecoveryAddressUpdated(oldRecoveryAddress, newRecoveryAddress);
}
``` [2](#0-1) 

`claimCorrupted` reads `recoveryAddress` at call time and transfers the entire contract balance to it: [3](#0-2) 

`sweepUnclaimedCorrupted` does the same after the attacker-claim window: [4](#0-3) 

`sweepUnclaimedBonus` also transfers to the live `recoveryAddress`, so even SURVIVED/EXPIRED bonus sweeps are redirectable: [5](#0-4) 

DESIGN.md §10 explicitly justifies the `expiryLocked` protection with "staker reliance" but makes no equivalent statement for `recoveryAddress`, and does not document its post-stake mutability as an accepted trade-off. [6](#0-5) 

### Impact Explanation
Under bad-faith CORRUPTED, `corruptedReserve` is set to `snapshotTotalStaked + snapshotTotalBonus` — the full pool — and `claimCorrupted` sweeps the entire contract balance to `recoveryAddress`. [7](#0-6)  If the sponsor has replaced `recoveryAddress` with their own address before that call, every staker loses their entire principal and any accrued bonus with no recourse.

### Likelihood Explanation
The sponsor is the agreement owner (`IAgreement(agreement).owner() != msg.sender` is enforced at pool creation). [8](#0-7)  The sponsor is explicitly listed as a semi-trusted, non-DAO actor. A CORRUPTED registry state is an external event the sponsor cannot unilaterally manufacture, but the sponsor can opportunistically update `recoveryAddress` the moment the registry enters `CORRUPTED` and before the moderator's `flagOutcome` transaction is mined — a straightforward mempool front-run. Alternatively, the sponsor can change `recoveryAddress` at any earlier point with no on-chain signal, since there is no lock and no timelock delay. The moderator acts honestly throughout; the defect is purely in the missing post-stake immutability of `recoveryAddress`.

### Recommendation
Apply the same `expiryLocked` pattern to `recoveryAddress`: flip a one-way `recoveryAddressLocked` boolean on the first `stake` call and revert in `setRecoveryAddress` when it is set. Alternatively, add a two-step commit-delay (timelock) to `setRecoveryAddress` so stakers have an observable window to exit before a pending change takes effect. Either approach mirrors the stated rationale in DESIGN.md §10 for protecting `expiry`.

### Proof of Concept
1. Sponsor deploys a pool via `createPool` with `recoveryAddress = legitimateMultisig`.
2. Stakers call `stake(...)`, depositing principal. `expiryLocked` flips; `recoveryAddress` remains freely mutable.
3. The underlying agreement is attacked; the registry transitions to `CORRUPTED`.
4. Sponsor observes the moderator's pending `flagOutcome(CORRUPTED, false, address(0))` in the mempool and front-runs it with `setRecoveryAddress(sponsorEOA)`. No guard prevents this.
5. Moderator's `flagOutcome` executes honestly, setting `outcome = CORRUPTED`, `goodFaith = false`, and snapshotting the full pool into `corruptedReserve`.
6. Sponsor (or anyone) calls `claimCorrupted()`. The function reads the now-updated `recoveryAddress = sponsorEOA` and transfers `stakeToken.balanceOf(address(this))` — the entire pool — to the sponsor. [9](#0-8) 
7. All stakers lose their principal and bonus; the sponsor profits the full pool value.

    function claimCorrupted() external nonReentrant {
        if (outcome != PoolStates.Outcome.CORRUPTED) revert OutcomeNotSet();
        if (goodFaith && bountyClaimed < bountyEntitlement) revert MustClaimBountyFirst();

        // aderyn-fp-next-line(reentrancy-state-change)
        uint256 toSweep = stakeToken.balanceOf(address(this));
        if (toSweep == 0) revert NothingToSweep();

        // donations have inflated the balance.

        stakeToken.safeTransfer(recoveryAddress, toSweep);

        emit ClaimCorrupted(msg.sender, recoveryAddress, toSweep);
    }
