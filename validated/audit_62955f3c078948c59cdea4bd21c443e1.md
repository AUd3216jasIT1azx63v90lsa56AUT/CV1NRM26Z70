Audit Report

## Title
Midnight-Authorized Delegate Can Exhaust Victim's EcrecoverAuthorizer Nonces to Invalidate Pre-Signed Authorizations - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` keys its nonce on `authorization.authorizer` and increments it unconditionally on every successful call, while simultaneously permitting any Midnight-authorized delegate of the authorizer to satisfy the signer check. A delegate can therefore self-sign `Authorization` structs that name the victim as `authorizer`, submit them in a loop, and permanently consume `nonce[victim]`, rendering every pre-signed authorization the victim distributed to other agents permanently invalid with `InvalidNonce`.

## Finding Description

**Exact code path — `EcrecoverAuthorizer.setIsAuthorized` (`src/periphery/EcrecoverAuthorizer.sol:24-48`):**

```solidity
// Line 26 — nonce keyed on authorization.authorizer, incremented unconditionally
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// Lines 33-36 — signer may be authorizer OR any Midnight-authorized delegate
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);

// Line 47 — downstream state write uses authorization.authorizer as onBehalf
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

**Root cause:** The nonce increment on line 26 fires against `nonce[authorization.authorizer]` regardless of whether the signer is the authorizer or a delegate. The `Unauthorized` check on lines 33-36 does not restrict who may consume the authorizer's nonces — it only requires the signer to be either the authorizer or a Midnight-authorized delegate. A delegate can therefore sign arbitrary `Authorization` structs naming the victim as `authorizer`, satisfy the `isAuthorized` branch, and burn `nonce[victim]` without the victim's participation.

**Exploit flow:**
1. Precondition: `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` — victim previously authorized attacker on Midnight (e.g., as a trading bot or router).
2. Victim pre-signs N authorizations with nonces 0…N-1 and distributes them to N agents.
3. Attacker constructs `Authorization(authorizer=victim, authorized=attacker, isAuthorized=true, nonce=0, deadline=far_future)`, signs it with their own private key, and calls `EcrecoverAuthorizer.setIsAuthorized`.
4. Line 26 passes (`0 == nonce[victim]`), increments `nonce[victim]` to 1.
5. Lines 33-36 pass because `isAuthorized(victim, attacker) == true`.
6. Line 47 calls `IMidnight.setIsAuthorized(attacker, true, victim)` — a no-op since the authorization already exists.
7. Attacker repeats with nonce=1, 2, …, N-1.
8. All N pre-signed authorizations from the victim now revert with `InvalidNonce` when any agent tries to submit them.

**Why existing checks fail:**
- The `InvalidNonce` check (line 26) enforces sequential ordering only; it does not require that the authorizer themselves signed this particular nonce consumption.
- The `Unauthorized` check (lines 33-36) does not restrict who may consume the authorizer's nonces — it only requires the signer to be either the authorizer or a Midnight-authorized delegate.
- There is no `msg.sender == authorization.authorizer` guard or requirement that the signature originate from the authorizer's own key when consuming nonces.
- The downstream call on line 47 produces no meaningful state change when the authorization already exists, meaning the attacker achieves a pure nonce-burn with zero protocol effect.

## Impact Explanation
Any agent holding a Midnight authorization from the victim can unilaterally burn the victim's sequential nonces in `EcrecoverAuthorizer`. This permanently invalidates any batch of pre-signed `Authorization` structs the victim distributed off-chain. Every protocol action that relies on those pre-signed authorizations — `take`, `repay`, `withdraw`, `liquidate`, `claimFee` — reverts with `InvalidNonce` when submitted by the intended agents, causing a sustained and unrecoverable DoS on all those core market actions for the victim. The victim cannot un-burn consumed nonces; they must re-sign and redistribute all authorizations, which the attacker can immediately burn again as long as the Midnight authorization persists.

## Likelihood Explanation
The precondition — `isAuthorized[victim][attacker] == true` on Midnight — is a normal operational state. Users routinely authorize bots, routers, or agents to act on their behalf. No tokens, collateral, or special role beyond this existing authorization is required. The attack requires only repeated calls to `EcrecoverAuthorizer.setIsAuthorized` with self-signed structs; each call costs one ECDSA signature (computed off-chain) and one transaction. On any L2 or low-fee chain, the cost per nonce burned is negligible. The attacker can front-run or race any legitimate submission. The attack is indefinitely repeatable as long as the attacker remains authorized on Midnight, and any nonces already burned are permanently consumed even after revocation.

## Recommendation
Require that the signer is always `authorization.authorizer` when consuming that authorizer's nonces. Delegates should only be permitted to act when the authorizer themselves has signed the struct, or use a separate nonce namespace per signer rather than per authorizer. A minimal fix:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegate signing must be supported, key the nonce on the signer rather than the authorizer, or require an additional signature from the authorizer authorizing the delegate to consume a specific nonce range.

## Proof of Concept
**Minimal Foundry test plan:**
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. Have `victim` call `Midnight.setIsAuthorized(attacker, true, victim)`.
3. Have `victim` sign 3 `Authorization` structs with nonces 0, 1, 2 (using victim's private key) and record them as `sig0`, `sig1`, `sig2`.
4. Have `attacker` construct `Authorization(authorizer=victim, authorized=attacker, isAuthorized=true, nonce=0, deadline=far_future)`, sign with attacker's key, and call `EcrecoverAuthorizer.setIsAuthorized`. Assert success and `nonce[victim] == 1`.
5. Repeat for nonces 1 and 2. Assert `nonce[victim] == 3`.
6. Attempt to submit victim's pre-signed `sig0`, `sig1`, `sig2` — all three revert with `InvalidNonce`.
7. Assert that `Midnight.isAuthorized(victim, attacker)` is still `true` (no meaningful state change occurred).
