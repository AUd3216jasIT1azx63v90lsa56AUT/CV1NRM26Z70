Audit Report

## Title
Authorized delegate can front-run and consume victim's EcrecoverAuthorizer nonce with arbitrary payload - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` increments the authorizer's nonce before verifying the signer, and accepts any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for — not only the authorizer's own key. A malicious Midnight-authorized delegate can front-run a victim's pending `setIsAuthorized` transaction, consume nonce N with an attacker-chosen payload, permanently invalidate the victim's signature, and silently substitute an attacker-controlled `authorized` address in the victim's Midnight `isAuthorized` mapping.

## Finding Description
In `src/periphery/EcrecoverAuthorizer.sol`, `setIsAuthorized` (lines 24–48):

**Line 26** increments the nonce unconditionally before the signer is verified:
```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

**Lines 33–36** accept the signer if they are either the authorizer themselves or any address the authorizer has previously authorized in Midnight:
```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

**Line 47** then executes the state change using fully attacker-controlled fields from the `Authorization` struct:
```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

The `Authorization` struct (`src/periphery/interfaces/IEcrecoverAuthorizer.sol`, lines 11–17) contains `authorized` and `isAuthorized` as caller-supplied inputs with no binding to the authorizer's intent.

**Exploit flow:**
1. Victim (Alice) has previously called `midnight.setIsAuthorized(bob, true, alice)` — a routine operator authorization (as shown in `test/AuthorizationTest.sol` lines 100–103, 237–239).
2. Alice signs `Authorization{authorizer: alice, authorized: charlie, isAuthorized: true, nonce: N, deadline: T}` and broadcasts it to the mempool.
3. Bob (the attacker) observes the pending transaction, constructs `Authorization{authorizer: alice, authorized: <attacker_address>, isAuthorized: <any>, nonce: N, deadline: <valid>}`, signs it with his own key, and front-runs Alice's transaction.
4. The nonce check passes (N matches current nonce) and the nonce is incremented to N+1.
5. The signer check passes because `IMidnight.isAuthorized(alice, bob) == true`.
6. `IMidnight.setIsAuthorized(attacker_address, attacker_bool, alice)` executes, modifying Alice's authorization mapping with attacker-chosen values.
7. Alice's original transaction reverts with `InvalidNonce`.

The existing nonce check does not prevent this because the nonce is consumed before the signer is verified. The signer check does not prevent this because it was designed to allow delegates to act on behalf of the authorizer, but it does not restrict which payload the delegate may sign.

## Impact Explanation
**DoS on EcrecoverAuthorizer flow**: The victim's signed-and-broadcast transaction is permanently invalidated. The attacker can repeat this for every subsequent nonce (N+1, N+2, …), permanently blocking the victim from using the EcrecoverAuthorizer to manage their authorizations via signature.

**Unauthorized state change / privilege escalation**: The attacker substitutes an arbitrary `authorized` address in the victim's Midnight `isAuthorized` mapping. This allows the attacker to grant or revoke authorization for any address in the victim's name, bypassing the victim's intent and any gate logic that depends on `isAuthorized`. This is a concrete, in-scope unauthorized state change and privilege escalation.

## Likelihood Explanation
Preconditions: (1) the attacker has previously been authorized by the victim in Midnight for any purpose — a routine pattern for operators and bundlers, as demonstrated in `test/AuthorizationTest.sol` lines 100–103 and 237–239; (2) the victim has a pending `EcrecoverAuthorizer.setIsAuthorized` transaction visible in the mempool. Both conditions are normal in protocol usage. The attack requires no privileged protocol access, no admin keys, and no leaked credentials — only a standard user-level authorization relationship. It is repeatable indefinitely at zero additional cost per nonce.

## Recommendation
Restrict the signer check in `EcrecoverAuthorizer.setIsAuthorized` to accept **only** `authorization.authorizer` as the valid signer. Remove the `IMidnight.isAuthorized` branch entirely from this function. The purpose of `EcrecoverAuthorizer` is to allow the authorizer to delegate execution of their own signed intent to any caller via `msg.sender` (which is already unconstrained); it is not to allow Midnight-authorized delegates to sign arbitrary payloads on the authorizer's behalf. If delegate-signed authorization is desired, it must be scoped to a separate nonce namespace per (authorizer, delegate) pair and must bind the delegate's signature to the exact payload the authorizer approved.

## Proof of Concept
Minimal Foundry fork test outline:

```solidity
function testDelegateFrontRunsNonce() public {
    address alice = makeAddr("alice");
    address bob = makeAddr("bob");   // attacker, holds bob_key
    address charlie = makeAddr("charlie"); // alice's intended target
    address mallory = makeAddr("mallory"); // attacker's chosen target

    // Step 1: Alice authorizes Bob in Midnight (routine operator setup)
    vm.prank(alice);
    midnight.setIsAuthorized(bob, true, alice);
    assertTrue(midnight.isAuthorized(alice, bob));

    // Step 2: Alice signs Authorization{authorizer:alice, authorized:charlie, isAuthorized:true, nonce:0}
    // (construct EIP-712 digest, sign with alice_key)

    // Step 3: Bob front-runs with Authorization{authorizer:alice, authorized:mallory, isAuthorized:true, nonce:0}
    // signed with bob_key, submitted before alice's tx
    Authorization memory bobAuth = Authorization({
        authorizer: alice,
        authorized: mallory,
        isAuthorized: true,
        nonce: 0,
        deadline: block.timestamp + 1 hours
    });
    // sign bobAuth with bob_key -> bobSig
    ecrecoverAuthorizer.setIsAuthorized(bobAuth, bobSig);

    // Assert: mallory is now authorized in alice's name
    assertTrue(midnight.isAuthorized(alice, mallory));
    // Assert: charlie is NOT authorized
    assertFalse(midnight.isAuthorized(alice, charlie));
    // Assert: alice's nonce is now 1
    assertEq(ecrecoverAuthorizer.nonce(alice), 1);

    // Step 4: Alice's original tx now reverts with InvalidNonce
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(aliceAuth, aliceSig);
}
```
