### Title
Replay of Failed Signed Authorizations Due to Nonce Not Persisting on Revert — (`src/periphery/EcrecoverAuthorizer.sol`)

---

### Summary

`EcrecoverAuthorizer.setIsAuthorized()` increments `nonce[authorization.authorizer]` before calling `IMidnight(MIDNIGHT).setIsAuthorized(...)`. If the downstream Midnight call reverts (e.g., because `EcrecoverAuthorizer` is not yet authorized in Midnight by the authorizer), the entire transaction reverts and the nonce increment is rolled back. The original signed `Authorization` remains replayable with the same nonce, allowing an attacker to execute it later under changed conditions without the authorizer's current consent.

---

### Finding Description

In `EcrecoverAuthorizer.setIsAuthorized()`, the nonce is incremented on line 26 via the post-increment in the `require` check:

```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
``` [1](#0-0) 

This increment is a storage write. However, the function then calls:

```solidity
IMidnight(MIDNIGHT)
    .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
``` [2](#0-1) 

`Midnight.setIsAuthorized()` enforces:

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [3](#0-2) 

Here `msg.sender` is `address(EcrecoverAuthorizer)` and `onBehalf` is `authorization.authorizer`. If the authorizer has not yet called `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, authorizer)`, this call reverts with `Unauthorized()`. The revert unwinds the entire transaction, including the nonce increment in `EcrecoverAuthorizer`. The signed `Authorization` is now replayable with the same nonce indefinitely (until the deadline expires).

**Root cause:** The nonce increment is not isolated from the downstream call. A revert in the downstream call rolls back the nonce, violating the invariant that a consumed signature cannot be reused.

---

### Impact Explanation

An attacker who holds a previously-failed signed `Authorization` can replay it once the authorizer later grants `EcrecoverAuthorizer` access in Midnight. The `isAuthorized` mapping in Midnight controls all privileged operations on a user's positions:

- `withdraw()` — drain lender credit (loan tokens)
- `withdrawCollateral()` — seize borrower collateral
- `take()` — execute trades on behalf of the victim
- `setIsAuthorized()` — grant further authorizations to arbitrary addresses [4](#0-3) [5](#0-4) 

A successful replay grants the `authorized` address full control over the victim's Midnight positions, enabling direct theft of funds.

---

### Likelihood Explanation

The preconditions are realistic:

1. A user signs an `Authorization` (e.g., for a relayer or a smart-contract operator) before having authorized `EcrecoverAuthorizer` in Midnight — a natural ordering mistake during onboarding.
2. The submission fails silently (reverts on-chain).
3. The user later authorizes `EcrecoverAuthorizer` in Midnight for a legitimate purpose, unaware the old signature is still valid.
4. An attacker (e.g., a mempool observer who saw the failed transaction) replays the original calldata.

The `deadline` field provides a time-bound mitigation, but users commonly sign with long deadlines (hours to days). The attack requires no privileged access — any address can call `EcrecoverAuthorizer.setIsAuthorized()`. [6](#0-5) 

---

### Recommendation

Increment the nonce unconditionally before any external call, and use a `try/catch` or restructure so the nonce persists even if the downstream call fails. Alternatively, mirror the pattern used in `EcrecoverAuthorizer` itself for the nonce check — validate all preconditions first, then perform the state mutation in a way that cannot be reverted by a downstream failure:

```solidity
// Increment nonce first, unconditionally
uint256 currentNonce = nonce[authorization.authorizer]++;
require(authorization.nonce == currentNonce, InvalidNonce());

// ... signature verification ...

// Downstream call — if this reverts, nonce is already consumed
IMidnight(MIDNIGHT).setIsAuthorized(...);
```

Since Solidity reverts roll back all storage writes atomically, the only safe approach is to ensure the nonce increment cannot be undone. One option is to use a separate low-level call and handle the return value without reverting on failure, or to require that `EcrecoverAuthorizer` is authorized in Midnight as a precondition check (before the nonce increment) so the transaction fails early with the nonce intact.

### Proof of Concept

**Setup:**
- Alice (`authorizer`) has not yet called `midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice)`.
- Alice signs `Authorization { authorizer: alice, authorized: bob, isAuthorized: true, nonce: 0, deadline: block.timestamp + 7 days }`.

**Step 1 — Failed submission:**
```
ecrecoverAuthorizer.setIsAuthorized(auth, sig)
// Reverts: IMidnight.setIsAuthorized → Unauthorized (EcrecoverAuthorizer not authorized)
// nonce[alice] remains 0
```

**Step 2 — Alice authorizes EcrecoverAuthorizer (for a different purpose):**
```
midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, alice)
// Alice now trusts EcrecoverAuthorizer to act on her behalf
```

**Step 3 — Attacker replays the original calldata:**
```
ecrecoverAuthorizer.setIsAuthorized(auth, sig)  // same auth, same sig
// Succeeds: nonce[alice] == 0 == auth.nonce ✓
// midnight.isAuthorized[alice][bob] = true
```

**Step 4 — Bob drains Alice:**
```
midnight.withdraw(market, aliceCredit, alice, bob)  // authorized by isAuthorized[alice][bob]
// Alice's loan tokens transferred to Bob
``` [7](#0-6) [8](#0-7)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
