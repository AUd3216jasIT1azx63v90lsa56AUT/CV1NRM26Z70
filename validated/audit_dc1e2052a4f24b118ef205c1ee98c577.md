Audit Report

## Title
Zero-price tick enables free credit and debt creation with no loan token transfer - (`src/libraries/TickLib.sol`, `src/Midnight.sol`)

## Summary
`TickLib.tickToPrice(0)` returns `0`, confirmed by the protocol's own test suite. Tick 0 is always accessible since `0 % tickSpacing == 0` for any spacing. The `take` function contains no guard requiring `offerPrice > 0`. When a sell offer is placed at tick 0 in a market with zero settlement fees (the default), `buyerAssets = 0` and `sellerAssets = 0`, yet the buyer receives real credit units and the seller accumulates real debt units with zero loan tokens transferred. The attacker's free credit can drain `withdrawable` funded by legitimate borrowers' repayments, stealing from legitimate lenders.

## Finding Description

**Root cause — `tickToPrice` returns 0 for ticks 0 and 1:**

`TickLib.tickToPrice` computes `uint256(1e36).divHalfDownUnchecked(...).divHalfDownUnchecked(PRICE_ROUNDING_STEP) * PRICE_ROUNDING_STEP`. For tick 0, the intermediate value before the final multiply is less than 1, so integer division yields 0, and `0 * 1e12 = 0`. The protocol's own test confirms this:

```
assertEq(TickLib.tickToPrice(0), 0, "tick 0");
assertEq(TickLib.tickToPrice(2), 1e12, "first non-zero tick");
```

Ticks 0 and 1 both return price 0; tick 2 is the first non-zero price.

**Tick 0 is always accessible:**

The accessibility check is `require(offer.tick % _marketState.tickSpacing == 0)`. Since `0 % n == 0` for any `n`, tick 0 always passes regardless of `tickSpacing`.

**Settlement fee defaults to zero:**

Settlement fees are stored as `uint16` cbp values initialized from `defaultSettlementFeeCbp[market.loanToken]`. Storage is zero-initialized, so unless the fee setter explicitly configures fees, `_settlementFee = 0` for all markets.

**Price and asset computation with no guard:**

With `offerPrice = 0` and `_settlementFee = 0`:
- `sellerPrice = offerPrice = 0` (sell offer: `offer.buy = false`)
- `buyerPrice = sellerPrice + _settlementFee = 0`
- `buyerAssets = units.mulDivUp(0, WAD) = 0`
- `sellerAssets = units.mulDivUp(0, WAD) = 0`

**Debt and credit are updated regardless:**

```solidity
buyerPos.credit += UtilsLib.toUint128(buyerCreditIncrease);   // += units
sellerPos.debt  += UtilsLib.toUint128(sellerDebtIncrease);    // += units
_marketState.totalUnits += buyerCreditIncrease - sellerCreditDecrease;
```

**Zero tokens are transferred:**

```solidity
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, address(this), buyerAssets - sellerAssets); // 0
SafeTransferLib.safeTransferFrom(offer.market.loanToken, payer, receiver, sellerAssets);                   // 0
```

**Health check does not prevent the invariant violation:**

`isHealthy` checks only that `maxDebt >= debt` (collateral covers debt at oracle price). It does not verify that the protocol's loan token balance covers the newly issued credit. A seller with sufficient collateral passes the check even though zero loan tokens entered the contract.

**Unlimited offer consumption with `maxAssets > 0`:**

For a sell offer with `maxAssets > 0`, consumed capacity is tracked as `sellerAssets = 0`. Since `0 <= maxAssets` always holds, the same offer can be taken an unlimited number of times, allowing unbounded free credit accumulation.

**Exploit flow (two-account attack):**

1. Attacker account A (maker/seller) supplies sufficient collateral and creates a sell offer at tick 0 with `maxAssets > 0`.
2. Attacker account B (taker/buyer) calls `take` with `units > 0`. Self-take is blocked, so two accounts are required.
3. B pays 0 loan tokens, receives `units` of credit. A accumulates `units` of debt, receives 0 loan tokens.
4. `_marketState.totalUnits` increases by `units` with no corresponding loan token deposit.
5. Step 2 can be repeated arbitrarily (since `sellerAssets = 0` never consumes `maxAssets` capacity).
6. When legitimate borrowers repay, `withdrawable` increases. B calls `withdraw` to drain those tokens, stealing from legitimate lenders.

## Impact Explanation

The buyer (B) holds credit claims unbacked by any deposited loan tokens. When legitimate borrowers repay debt, `withdrawable` increases and B can drain it before legitimate lenders, directly stealing their deposited loan tokens. The `totalUnitsEqualsSumNegativeDebtPlusWithdrawable` invariant (`totalUnits == sumDebt + withdrawable`) is maintained in accounting terms (A's debt equals B's credit), but the solvency invariant is violated: the contract's loan token balance does not cover all outstanding credit redemptions. This is a direct, concrete theft from legitimate lenders.

## Likelihood Explanation

All preconditions are trivially satisfied: tick 0 is always accessible, zero settlement fees are the default configuration, and the two-account requirement is trivial for any attacker. No privileged access is required. The exploit is repeatable without bound when `maxAssets > 0` is used. Any market with default (zero) settlement fees is vulnerable from the moment of creation.

## Recommendation

Add a zero-price guard in `take` immediately after computing `offerPrice`:

```solidity
uint256 offerPrice = TickLib.tickToPrice(offer.tick);
require(offerPrice > 0, ZeroPriceTick());
```

Alternatively, restrict the valid tick range so that `tickToPrice` never returns 0 (i.e., enforce `offer.tick >= 2`), or adjust `tickToPrice` to return a minimum of `PRICE_ROUNDING_STEP` for any valid tick. The guard in `take` is the most surgical fix and does not affect the tick/price mapping used elsewhere.

## Proof of Concept

```solidity
// Setup: market with zero settlement fees (default), DEFAULT_TICK_SPACING = 4
// Account A: seller/maker with sufficient collateral
// Account B: buyer/taker

// 1. A supplies collateral
midnight.supplyCollateral(market, 0, COLLATERAL_AMOUNT, accountA);

// 2. A creates sell offer at tick 0 (price = 0), maxAssets = type(uint128).max
Offer memory offer = Offer({
    market: market, tick: 0, buy: false,
    maxAssets: type(uint128).max, maxUnits: 0, ...
});

// 3. B takes the offer — zero loan tokens transferred, B gets free credit
uint256 units = 1e18;
(uint256 buyerAssets, uint256 sellerAssets) = midnight.take(offer, ..., units, accountB, ...);
assert(buyerAssets == 0 && sellerAssets == 0);
assert(midnight.creditOf(id, accountB) == units);
assert(midnight.debtOf(id, accountA) == units);
assert(loanToken.balanceOf(address(midnight)) == 0);

// 4. Repeat step 3 arbitrarily (maxAssets never consumed)

// 5. Legitimate borrower repays, increasing withdrawable
midnight.repay(market, units, legitimateBorrower, address(0), "");
assert(midnight.withdrawable(id) == units);

// 6. B drains withdrawable, stealing from legitimate lenders
midnight.withdraw(market, units, accountB, accountB);
assert(loanToken.balanceOf(accountB) == units); // stolen
```
