Audit Report

## Title
Co-authorized agent can deauthorize arbitrary agents via `EcrecoverAuthorizer.setIsAuthorized`, enabling indefinitely repeatable DoS on all bundled operations - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` permits any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for to sign and submit an `Authorization` struct with an arbitrary `authorized` target and `isAuthorized` value. A co-authorized agent of the victim can therefore revoke `MidnightBundles`' authorization on behalf of the victim, causing all five `MidnightBundles` entry points to revert with `Unauthorized` for that victim. The attack costs only gas and can be repeated every time the victim re-authorizes.

## Finding Description

**Root cause — `EcrecoverAuthorizer.setIsAuthorized` lines 33–36:**

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The second branch of the `||` grants any already-authorized agent of `authorization.authorizer` full authority to call `IMidnight.setIsAuthorized` on behalf of that authorizer with no restriction on the `authorized` target or the `isAuthorized` boolean. The signer is not constrained to only modify their own entry.

The call at lines 46–47 then executes unconditionally with attacker-controlled fields:

```solidity
IMidnight(MIDNIGHT)
    .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

**Exploit flow:**

1. Victim calls `Midnight.setIsAuthorized(MidnightBundles, true, victim)` and `Midnight.setIsAuthorized(attacker, true, victim)` — both are now authorized agents of victim.
2. Attacker reads `nonce[victim]` from `EcrecoverAuthorizer`.
3. Attacker constructs and signs `Authorization { authorizer: victim, authorized: MidnightBundles, isAuthorized: false, nonce: currentNonce, deadline: far_future }` with the attacker's own private key.
4. Attacker calls `EcrecoverAuthorizer.setIsAuthorized(authorization, attackerSig)`.
5. `ecrecover` returns `attacker`; `IMidnight(MIDNIGHT).isAuthorized(victim, attacker) == true` passes.
6. `IMidnight(MIDNIGHT).setIsAuthorized(MidnightBundles, false, victim)` executes.

**Why existing checks fail:**

- The nonce check (line 26) only prevents replay of the same signed message; it does not restrict which `authorized` address or `isAuthorized` value the signer may choose.
- The deadline check (line 25) is irrelevant to authorization scope.
- There is no guard requiring `authorization.authorized == signer` when the signer is not the authorizer itself.

**Downstream revert in `MidnightBundles`:**

All five entry points open with:

```solidity
require(taker == msg.sender || IMidnight(MIDNIGHT).isAuthorized(taker, msg.sender), Unauthorized());
```

After step 6, `isAuthorized(victim, MidnightBundles) == false`, so every call with `taker=victim` reverts immediately.

## Impact Explanation
Any victim who has authorized `MidnightBundles` and at least one other address has all five `MidnightBundles` entry points permanently blocked until they re-authorize — which can be griefed again immediately at the cost of a single transaction. This constitutes a concrete, indefinitely repeatable denial-of-service against the victim's use of the bundler, including repayment and collateral withdrawal operations, which can have downstream financial consequences (e.g., inability to repay debt or withdraw collateral in time).

## Likelihood Explanation
- **Precondition:** Victim has authorized both `MidnightBundles` and at least one other address. This is the normal operating state for any user of the bundler who also uses a keeper, UI relayer, or second wallet.
- **Attacker cost:** Gas for one `setIsAuthorized` call. No capital required.
- **Repeatability:** The attacker can re-execute every time the victim re-authorizes `MidnightBundles`, making the DoS indefinitely sustainable.
- **Detection:** The `SetIsAuthorized` event is emitted but there is no on-chain protection against it.

## Recommendation
When the signer is not the authorizer themselves, restrict the signer to only modifying their own authorization entry. Add the following check immediately after the existing `require` block:

```solidity
if (signer != authorization.authorizer) {
    require(authorization.authorized == signer, Unauthorized());
}
```

This ensures that a delegated signer can only revoke or re-grant their own authorization, not that of any other agent.

## Proof of Concept

**Minimal manual steps:**

1. Deploy `Midnight` and `EcrecoverAuthorizer` on a local fork.
2. As `victim`, call `Midnight.setIsAuthorized(MidnightBundles, true, victim)` and `Midnight.setIsAuthorized(attacker, true, victim)`.
3. As `attacker`, read `EcrecoverAuthorizer.nonce(victim)`.
4. Construct `Authorization { authorizer: victim, authorized: MidnightBundles, isAuthorized: false, nonce: <current>, deadline: type(uint256).max }`.
5. Sign the EIP-712 digest with `attacker`'s private key.
6. Call `EcrecoverAuthorizer.setIsAuthorized(authorization, sig)` as `attacker`.
7. Assert `Midnight.isAuthorized(victim, MidnightBundles) == false`.
8. Attempt any `MidnightBundles` entry point with `taker=victim`; confirm it reverts with `Unauthorized`.
9. Repeat from step 3 after victim re-authorizes to confirm indefinite repeatability.
