### Title
Protocol Fees Bypassed via Zero-Default for Unregistered Loan Tokens — (File: src/Midnight.sol)

---

### Summary

The `defaultSettlementFeeCbp` and `defaultContinuousFee` mappings in `Midnight.sol` return zero for any `loanToken` address that has never had a default set. Because `touchMarket()` is permissionless and copies these zero defaults into every new market's `MarketState`, an attacker can deploy a 1-for-1 ERC-20 wrapper around any loan token and create a market with that wrapper, obtaining zero settlement fees and zero continuous fees for the lifetime of that market.

---

### Finding Description

**Root cause — storage defaults to zero:** [1](#0-0) 

```solidity
mapping(address loanToken => uint16[7]) public defaultSettlementFeeCbp;
mapping(address loanToken => uint32)   public defaultContinuousFee;
```

Both mappings are uninitialized for any token the `feeSetter` has not explicitly configured. Solidity initializes unset mapping values to zero.

**Fee copy at market creation — zero propagates into `MarketState`:** [2](#0-1) 

```solidity
uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
_marketState.settlementFeeCbp0 = _defaultSettlementFeeCbp[0];
// ... indices 1-6 similarly
_marketState.continuousFee = defaultContinuousFee[market.loanToken];
```

If `market.loanToken` has no default set, all seven settlement-fee breakpoints and the continuous fee are stored as `0` in the market's `MarketState`.

**Settlement fee collection at trade time:** [3](#0-2) 

```solidity
uint256 _settlementFee = settlementFee(id, timeToMaturity);
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice;
uint256 buyerPrice  = sellerPrice + _settlementFee;
``` [4](#0-3) 

```solidity
claimableSettlementFee[offer.market.loanToken] += buyerAssets - sellerAssets;
```

When `_settlementFee == 0`, `buyerAssets == sellerAssets` and `claimableSettlementFee` receives nothing.

**Continuous fee accrual:** [5](#0-4) 

```solidity
uint128 buyerPendingFeeIncrease =
    UtilsLib.toUint128(buyerCreditIncrease.mulDivDown(_marketState.continuousFee * timeToMaturity, WAD));
```

When `continuousFee == 0`, `buyerPendingFeeIncrease == 0` — no continuous fee accrues for any lender in the market.

Any caller can create a market with any `loanToken`. There is no allowlist or fee-existence check on the loan token.

**Exploit path:**

1. `feeSetter` calls `setDefaultSettlementFee(USDC, i, nonZeroFee)` for all 7 breakpoints and `setDefaultContinuousFee(USDC, nonZeroFee)`.
2. Attacker deploys `wUSDC` — a minimal ERC-20 that wraps USDC 1:1 (deposit/withdraw).
3. Attacker calls `touchMarket(Market{loanToken: wUSDC, ...})`. Because `defaultSettlementFeeCbp[wUSDC]` and `defaultContinuousFee[wUSDC]` are both zero, the new market's `MarketState` is initialized with all-zero fees.
4. Makers publish offers in the wUSDC market. Takers execute them. Zero settlement fee is collected; zero continuous fee accrues.
5. Users wrap USDC → trade in wUSDC market → unwrap. Protocol collects no fees.

**Additional nuance — continuous fee is crystallized at take time:** [7](#0-6) 

> If the market's continuous fee changes, the pending fee of existing lenders is not updated (=> their fee is fixed).

Even if `feeSetter` later calls `setMarketContinuousFee` on the wrapped-token market, lenders who already hold credit positions will never pay the continuous fee — their `pendingFee` was crystallized at zero when they entered.

---

### Impact Explanation

The protocol collects zero settlement fees and zero continuous fees on any market whose `loanToken` has no pre-configured default. An attacker can systematically route all volume through wrapped-token markets, permanently eliminating protocol fee revenue. The `claimableSettlementFee` and `continuousFeeCredit` accumulators for those markets remain at zero regardless of trading volume.

---

### Likelihood Explanation

The attack requires only:
- Deploying a trivial 1:1 ERC-20 wrapper (no privileged access, ~50 lines of Solidity).
- Calling the public `touchMarket()` function once.

No admin keys, oracle access, or capital at risk is needed. The incentive is clear: users save the full settlement fee (up to 50 bps annualized) and continuous fee (up to 1% annualized) on every trade. Gas cost of wrapping/unwrapping is negligible compared to fee savings on large notional amounts.

---

### Recommendation

Implement a non-zero fallback fee for tokens without a configured default, analogous to the recommendation in the external report:

```solidity
// In touchMarket(), replace:
uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];

// With a check that reverts or applies a global minimum if all breakpoints are zero:
uint16[7] memory _defaultSettlementFeeCbp = defaultSettlementFeeCbp[market.loanToken];
bool allZero = true;
for (uint256 i = 0; i < 7; i++) {
    if (_defaultSettlementFeeCbp[i] != 0) { allZero = false; break; }
}
require(!allZero || defaultContinuousFee[market.loanToken] != 0 || feeExempt[market.loanToken],
    "No fee configured for loan token");
```

Alternatively, maintain an explicit allowlist of approved loan tokens, or store a global minimum default fee applied when no per-token default exists.
