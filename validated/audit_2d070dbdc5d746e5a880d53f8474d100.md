### Title
Missing Zero-Address Validation for `_midnight` in Peripheral Contract Constructors — (`src/periphery/EcrecoverAuthorizer.sol`, `src/ratifiers/EcrecoverRatifier.sol`, `src/ratifiers/SetterRatifier.sol`, `src/periphery/MidnightBundles.sol`)

---

### Summary

Four peripheral contracts store the core `Midnight` protocol address as an immutable via a constructor parameter `address _midnight`, with no zero-address guard. If any of these contracts is deployed with `address(0)`, the immutable `MIDNIGHT` is permanently set to the null address and the contract becomes irreversibly non-functional. The most impactful case is `EcrecoverAuthorizer`, where the failure is **silent**: user nonces are consumed and the transaction succeeds, but no authorization is ever recorded on the real `Midnight` contract.

---

### Finding Description

All four contracts share the same pattern:

**`EcrecoverAuthorizer`** [1](#0-0) 

```solidity
constructor(address _midnight) {
    MIDNIGHT = _midnight;   // no require(_midnight != address(0))
}
```

**`EcrecoverRatifier`** [2](#0-1) 

**`SetterRatifier`** [3](#0-2) 

**`MidnightBundles`** [4](#0-3) 

`MIDNIGHT` is declared `immutable` in every case, so it can never be corrected post-deployment. [5](#0-4) 

**Concrete failure modes per contract:**

| Contract | Failure mode |
|---|---|
| `EcrecoverRatifier` | `isRatified` always reverts (`NotMidnight()`); every offer using this ratifier is permanently unexecutable. |
| `SetterRatifier` | Same — `msg.sender == MIDNIGHT` check always fails. |
| `EcrecoverAuthorizer` | **Silent failure** — nonce is consumed, call to `address(0)` succeeds as a no-op, but authorization is never written to the real `Midnight` contract. |
| `MidnightBundles` | All bundle entry-points revert or silently no-op; token operations fail. |

The `EcrecoverAuthorizer` case is the most dangerous. In Solidity 0.8.x, a high-level call to `address(0)` (no code) returns `(true, "")`. `isAuthorized` decodes the empty return as `false`, and `setIsAuthorized` (void return) completes without revert. The sequence in `setIsAuthorized` is: [6](#0-5) 

1. Deadline and nonce checks pass — **nonce is incremented** (irreversible).
2. Signature is verified against the authorizer.
3. `IMidnight(address(0)).setIsAuthorized(...)` — no-op call to null address, succeeds silently.
4. Function returns without error.

The caller has now spent their nonce and believes authorization is set, but the real `Midnight` contract was never touched.

---

### Impact Explanation

- **`EcrecoverRatifier` / `SetterRatifier`**: All offers signed against this ratifier are permanently unexecutable. Makers cannot trade; any capital flow depending on those offers is frozen.
- **`EcrecoverAuthorizer`**: Users' EIP-712 nonces are consumed silently. The intended authorization (e.g., delegating a keeper or a bundler) is never recorded on `Midnight`. Any subsequent action by the "authorized" address on behalf of the user will revert with `Unauthorized`. Users may not notice until a time-sensitive operation (liquidation avoidance, collateral withdrawal) fails.
- **`MidnightBundles`**: All atomic bundle operations (buy/sell with collateral, repay-and-withdraw) are permanently broken for all users of that deployment.

In all cases the contract is **permanently** broken — `immutable` cannot be updated.

---

### Likelihood Explanation

Deployment with `address(0)` is a realistic human error (copy-paste mistake, scripting bug, wrong argument order). The probability is low for a careful team but non-zero, and the consequence is irreversible. The external report documents exactly this class of error being made in production (`MerchantSubscription`). Because these are peripheral helper contracts that integrators and front-ends point to by address, a broken deployment may go undetected until users report failures.

---

### Recommendation

Add a zero-address guard in every constructor that accepts `_midnight`:

```solidity
constructor(address _midnight) {
    require(_midnight != address(0), "zero address");
    MIDNIGHT = _midnight;
}
```

Apply identically to `EcrecoverAuthorizer`, `EcrecoverRatifier`, `SetterRatifier`, and `MidnightBundles`. This follows the "fail early and loudly" principle cited in the external report and prevents a permanently broken deployment from being silently accepted.

---

### Proof of Concept

**Scenario: `EcrecoverAuthorizer` deployed with `address(0)`**

1. Deployer calls `new EcrecoverAuthorizer(address(0))`. No revert. `MIDNIGHT = address(0)`.
2. User constructs a valid `Authorization` struct and EIP-712 signature, then calls `setIsAuthorized(authorization, sig)`.
3. Deadline check passes. Nonce `nonce[authorization.authorizer]` is incremented from 0 → 1 (irreversible).
4. Signature verifies correctly; `signer == authorization.authorizer`.
5. `IMidnight(address(0)).setIsAuthorized(...)` — EVM call to null address returns `(true, "")`. No revert.
6. Transaction succeeds. User believes `authorization.authorized` is now authorized on Midnight.
7. `authorization.authorized` calls any Midnight function on behalf of the user → reverts `Unauthorized()`.
8. User attempts to re-authorize with a fresh signature → nonce mismatch (`InvalidNonce()`), because nonce was already consumed in step 3.
9. User is permanently locked out of gasless authorization via this contract. [7](#0-6) [8](#0-7) [9](#0-8) [4](#0-3)

```text
    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```
