### Title
Division-by-Zero / Arithmetic Underflow in `liquidate` Bad-Debt Path When `totalUnits == 0` ‚Äî (`File: src/Midnight.sol`)

---

### Summary

In `Midnight.sol`, the `liquidate` function's bad-debt socialization block computes `_totalUnits - badDebt` and then divides by `_totalUnits`. When a prior liquidation has already wiped all credit from the market (`totalUnits == 0`), any subsequent call to `liquidate` on a remaining borrower with `badDebt > 0` causes an arithmetic underflow revert in Solidity 0.8+'s checked arithmetic. This permanently bricks the `liquidate` function for all remaining borrowers in that market, preventing collateral seizure and leaving assets locked.

---

### Finding Description

**Root cause ‚Äî exact location:** [1](#0-0) 

```solidity
if (badDebt > 0) {
    _position.debt -= uint128(badDebt);
    uint256 _totalUnits = _marketState.totalUnits;   // line 629
    uint256 _lossFactor = _marketState.lossFactor;
    _marketState.lossFactor = UtilsLib.toUint128(
        type(uint128).max - (type(uint128).max - _lossFactor)
            .mulDivDown(_totalUnits - badDebt, _totalUnits)  // line 632 ‚Üê CRASH
    );
    _marketState.totalUnits -= UtilsLib.toUint128(badDebt);  // line 634
```

`mulDivDown` is a plain unchecked-style wrapper: [2](#0-1) 

```solidity
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;
}
```

Two failure modes on line 632 when `_totalUnits == 0` and `badDebt > 0`:

1. **Arithmetic underflow** ‚Äî `_totalUnits - badDebt` = `0 - badDebt` reverts under Solidity 0.8 checked arithmetic before the division is even reached.
2. **Division by zero** ‚Äî even if the subtraction were unchecked, `/ _totalUnits` = `/ 0` would revert.

There is **no guard** checking `_totalUnits > 0` before entering this block.

**How `_totalUnits` reaches zero while a borrower still has debt:**

`totalUnits` tracks net credit in the market. Each bad-debt realization decrements it by `badDebt`: [3](#0-2) 

When a liquidation where `badDebt == _totalUnits` completes, `totalUnits` becomes exactly 0 and `lossFactor` is set to `type(uint128).max`. The `take` function then gates on: [4](#0-3) 

```solidity
require(_marketState.lossFactor < type(uint128).max, MarketLossFactorMaxedOut());
```

‚Ä¶so no new positions can be opened. However, **existing borrowers who entered before this point still carry debt**. The `liquidate` function has **no equivalent guard**. Calling it on any such borrower triggers the bad-debt path with `_totalUnits == 0`, causing an unconditional revert.

**Concrete exploit path:**

1. Market is created with two borrowers, Alice and Bob, and lender Carol.
2. Carol lends 100 units ‚Üí `totalUnits = 100`.
3. Alice borrows 100 units (debt = 100, no collateral). Bob borrows 50 units (debt = 50, collateral worth 20).
4. Alice is liquidated: `badDebt = 100 = _totalUnits`. After liquidation: `totalUnits = 0`, `lossFactor = type(uint128).max`.
5. Bob is now liquidatable (post-maturity or unhealthy). Liquidator calls `liquidate(..., borrower=Bob, ...)`.
6. `badDebt` for Bob is computed as `50 - collateral_value_at_maxLif > 0`.
7. Execution enters `if (badDebt > 0)` block. `_totalUnits = 0`. `_totalUnits - badDebt` underflows ‚Üí **revert**.
8. Bob's collateral (worth 20) is permanently locked; no liquidator can ever seize it.

---

### Impact Explanation

- **Liquidation permanently disabled** for all remaining borrowers in a market once `totalUnits` has been zeroed by prior bad-debt realizations.
- **Collateral locked**: borrowers whose debt exceeds their collateral value have no economic incentive to repay voluntarily. Their collateral remains frozen in the contract with no mechanism to recover it.
- The `repay` function is unaffected, so a borrower *could* repay and then call `withdrawCollateral`, but rational actors with `debt > collateral_value` will not do so.
- The market is already in a degraded state (`lossFactor = max`, `take` reverts), but the liquidation DoS compounds the damage by preventing any cleanup of remaining insolvent positions.

---

### Likelihood Explanation

- Requires a market with **at least two borrowers** where one borrower's bad debt is large enough to wipe all remaining credit. This is a realistic condition in any market that experiences cascading defaults (e.g., collateral price crash affecting multiple borrowers simultaneously).
- No privileged access is required. Any permissionless liquidator triggers the revert simply by attempting a legitimate liquidation.
- The triggering condition (`totalUnits == 0` after a prior liquidation) is a **deterministic, permanent state** ‚Äî once reached, every future `liquidate` call on any remaining borrower with bad debt will revert forever.

---

### Recommendation

Add a guard before the bad-debt socialization block to skip it when `_totalUnits == 0` (there are no lenders left to socialize the loss onto):

```solidity
if (badDebt > 0) {
    _position.debt -= uint128(badDebt);
    uint256 _totalUnits = _marketState.totalUnits;
    if (_totalUnits > 0) {                          // ‚Üê add this guard
        uint256 _lossFactor = _marketState.lossFactor;
        _marketState.lossFactor = UtilsLib.toUint128(
            type(uint128).max - (type(uint128).max - _lossFactor)
                .mulDivDown(_totalUnits - badDebt, _totalUnits)
        );
        _marketState.totalUnits -= UtilsLib.toUint128(badDebt);
        _marketState.continuousFeeCredit = ...;
    }
}
```

When `_totalUnits == 0`, all credit is already gone; skipping the socialization is semantically correct and allows the liquidation to proceed to collateral seizure.

### Proof of Concept

**State setup (no privileged keys needed):**

1. Deploy `Midnight`. Create a market with two collateral-less borrowers (LLTV = 1e18 allowed by `isLltvAllowed`).
2. Lender calls `take` to supply 100 units of credit ‚Üí `totalUnits = 100`.
3. Borrower A calls `take` (as seller) for 100 units ‚Üí `A.debt = 100`, `totalUnits = 100`.
4. Borrower B calls `take` (as seller) for 50 units from a second lender ‚Üí `B.debt = 50`, `totalUnits = 150`. B supplies 5 units of collateral.
5. Prices crash; both A and B are liquidatable.

**Trigger:**

6. Liquidate A: `badDebt = 100`. Since `_totalUnits = 150`, `_totalUnits - badDebt = 50`. Succeeds. `totalUnits = 50`, `lossFactor` updated.
7. Liquidate A again or another borrower whose `badDebt = 50` (the remaining `totalUnits`). `totalUnits ‚Üí 0`, `lossFactor ‚Üí type(uint128).max`.
8. Call `liquidate(..., borrower = B, seizedAssets = 5, ...)`. B is liquidatable (post-maturity). `badDebt > 0` (B's debt exceeds collateral value at maxLif).
9. **Result**: transaction reverts with arithmetic underflow at `_totalUnits - badDebt` = `0 - badDebt`.

**Expected correct behavior**: liquidation should succeed, seizing B's 5 units of collateral. [5](#0-4) [2](#0-1)
