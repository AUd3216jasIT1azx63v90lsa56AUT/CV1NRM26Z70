Audit Report

## Title
Authorized Midnight Delegate Can Consume Authorizer's EcrecoverAuthorizer Nonce, Permanently Invalidating Pre-Signed Authorization Messages - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` increments `nonce[authorization.authorizer]` at line 26 before the signer is recovered and validated. The signer check at lines 33–36 explicitly accepts any address for which `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` returns true — i.e., any Midnight-level delegate of the authorizer. Any such delegate can therefore sign and submit an `Authorization` struct naming the authorizer as `authorization.authorizer`, consuming the authorizer's current nonce and permanently invalidating any pre-signed messages carrying that nonce value. The attack is repeatable indefinitely as long as the delegate relationship persists.

## Finding Description

**Root cause:** In `src/periphery/EcrecoverAuthorizer.sol`, `setIsAuthorized` (lines 24–48):

```solidity
// Line 26: nonce consumed BEFORE signer is checked
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

// Lines 31–36: signer check explicitly permits any Midnight delegate
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
require(signer != address(0), InvalidSignature());
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

The nonce for `authorization.authorizer` is incremented at line 26 regardless of who the signer turns out to be. The subsequent signer check at lines 33–36 accepts not only the authorizer themselves but any address `signer` for which `Midnight.isAuthorized(authorization.authorizer, signer)` is true. Because the nonce is already consumed before this check runs, any delegate of Alice in Midnight can consume Alice's current `EcrecoverAuthorizer` nonce.

**Exploit flow:**
1. Alice calls `Midnight.setIsAuthorized(Bob, true, Alice)` — Bob is now a Midnight-level delegate of Alice (normal, expected operation).
2. Alice off-chain pre-signs `Authorization(authorizer=Alice, authorized=Dave, isAuthorized=true, nonce=0, deadline=T)` and distributes or holds it for later submission.
3. Bob constructs `Authorization(authorizer=Alice, authorized=Charlie, isAuthorized=true, nonce=0, deadline=T')` and signs it with his own key.
4. Bob (or any relayer) calls `ecrecoverAuthorizer.setIsAuthorized(auth_bob, sig_bob)`.
5. Line 26: `nonce[Alice]` is incremented 0 → 1. The require passes because `authorization.nonce (0) == nonce[Alice] (0)`.
6. Lines 33–36: `signer == Bob`, and `Midnight.isAuthorized(Alice, Bob) == true`. The require passes.
7. Alice's pre-signed message `(authorizer=Alice, nonce=0)` now reverts with `InvalidNonce()` on any future submission attempt.
8. Bob can immediately repeat with nonce=1, 2, … as long as he remains a delegate.

**Why existing checks fail:**
- The nonce check at line 26 only verifies the value matches and increments it; it does not restrict *who* may consume it.
- The signer check at lines 33–36 is evaluated after the nonce is already consumed; even if evaluated first, it still explicitly permits delegates.
- The Certora `effects` rule in `certora/specs/EcrecoverAuthorizer.spec` (lines 12–21) only asserts that `nonce(authorization.authorizer)` increments by 1 on success and that other nonces are unchanged. It does not constrain that only the authorizer themselves (not a delegate) may trigger that increment, leaving this attack path unverified.

## Impact Explanation
Any address Alice has authorized in Midnight can consume Alice's current `EcrecoverAuthorizer` nonce by signing and submitting an `Authorization` struct with `authorizer=Alice` and the current nonce value. This permanently invalidates all of Alice's pre-signed authorization messages carrying that nonce. No funds are directly at risk, but the DoS is persistent and repeatable: after each nonce consumption, the attacker can immediately consume the next nonce, rendering Alice's use of `EcrecoverAuthorizer` for pre-signed delegations completely non-functional as long as the attacker remains a delegate.

## Likelihood Explanation
**Preconditions:** Alice must have authorized Bob in Midnight — a normal, expected operation explicitly described in the protocol's authorization model. Bob must act adversarially or be compromised. No special privileges beyond being an existing Midnight-level delegate are required. The attack requires no capital, no flash loans, and no oracle manipulation. It is repeatable at negligible cost (gas only) and cannot be stopped without Alice revoking Bob's Midnight authorization.

## Recommendation
Restrict nonce consumption in `EcrecoverAuthorizer` to the authorizer themselves. The simplest fix is to recover and validate the signer *before* incrementing the nonce, and to require `signer == authorization.authorizer` (removing the delegate path entirely from `EcrecoverAuthorizer`). If delegate signing is intentionally desired, use a per-signer nonce (`nonce[signer]`) rather than `nonce[authorization.authorizer]`, so a delegate's submission consumes only the delegate's own nonce and cannot invalidate the authorizer's pre-signed messages.

## Proof of Concept
**Minimal Foundry test outline:**
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. Alice calls `midnight.setIsAuthorized(Bob, true, Alice)`.
3. Alice off-chain signs `Authorization(authorizer=Alice, authorized=Dave, isAuthorized=true, nonce=0, deadline=block.timestamp+1 days)` → `sigAlice`.
4. Bob signs `Authorization(authorizer=Alice, authorized=Charlie, isAuthorized=true, nonce=0, deadline=block.timestamp+1 days)` → `sigBob`.
5. Call `ecrecoverAuthorizer.setIsAuthorized(authBob, sigBob)` — succeeds; `nonce[Alice]` becomes 1.
6. Call `ecrecoverAuthorizer.setIsAuthorized(authAlice, sigAlice)` — reverts with `InvalidNonce()`.
7. Assert `midnight.isAuthorized(Alice, Charlie) == true` (Bob's authorization went through) and `midnight.isAuthorized(Alice, Dave) == false` (Alice's pre-signed message was invalidated).
