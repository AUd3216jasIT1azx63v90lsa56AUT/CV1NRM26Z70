Audit Report

## Title
Zero-price oracle enables free collateral seizure with full bad-debt loss to lenders - (`src/Midnight.sol`)

## Summary
When an oracle returns `0` for a collateral, the `liquidate` function computes `repaidUnits = 0` via integer arithmetic, allowing a liquidator to seize the borrower's collateral while transferring zero loan tokens. Simultaneously, the entire borrower debt is written off as bad debt, socializing the loss to lenders. The liquidator receives the collateral for free with no corresponding loan token repayment.

## Finding Description

**Root cause — `src/Midnight.sol` line 650:**

```solidity
repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
```

When `liquidatedCollatPrice = 0` (oracle returns 0):
- Step 1: `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE)` = `(0 + ORACLE_PRICE_SCALE − 1) / ORACLE_PRICE_SCALE` = `0`
- Step 2: `mulDivUp(0, WAD, lif)` = `(lif − 1) / lif` = `0`

This is explicitly axiomatized in `certora/specs/Reverts.spec` lines 65–68 as "needed for oracle returns zero case": [1](#0-0) 

**Liquidatability is trivially satisfied:** At line 613, each collateral contributes `_collateral.mulDivDown(0, ORACLE_PRICE_SCALE)... = 0` to `maxDebt`, making `originalDebt > maxDebt` always true: [2](#0-1) 

**Bad debt path runs first (lines 626–641):** Since all collaterals contribute 0 to the `zeroFloorSub` computation when `price = 0`, `badDebt = originalDebt`. The entire debt is written off, `_position.debt` is reduced to 0, and `lossFactor` is updated to socialize the loss to lenders: [3](#0-2) 

**Collateral seizure path then runs (lines 643–677):** With `repaidUnits = 0`:
- `_position.collateral[collateralIndex] -= seizedAssets` — collateral removed
- `_marketState.withdrawable += 0` — pool not compensated
- `SafeTransferLib.safeTransfer(collateralToken, receiver, seizedAssets)` — collateral sent to attacker
- `SafeTransferLib.safeTransferFrom(loanToken, payer, address(this), 0)` — zero loan tokens paid [4](#0-3) [5](#0-4) 

**Existing checks are insufficient:**
- `atMostOneNonZero(0, seizedAssets)` at line 595 → passes (one is zero)
- `_position.debt > 0` at line 596 → passes (borrower has debt)
- `NotLiquidatable` at lines 620–624 → passes (oracle=0 makes `originalDebt > maxDebt` trivially true)
- `maxRepaid` RCF check at lines 655–668 → `repaidUnits = 0 ≤ maxRepaid` always passes
- No `require(price > 0)` guard exists anywhere in the liquidate function [6](#0-5) [7](#0-6) 

**Formal proof gap acknowledged:** `certora/specs/NoDivisionByZero.spec` line 124 explicitly requires `ghostPrice(...) > 0` as a precondition, meaning the formal proofs do not cover the `price = 0` scenario: [8](#0-7) 

## Impact Explanation
Lenders suffer a complete loss on the borrower's debt (socialized via `lossFactor` write-down) while the liquidator extracts the borrower's collateral for free. The protocol's `withdrawable` pool receives zero compensation. This is a direct, irreversible theft of collateral value from the lending pool, constituting insolvency of the affected market.

## Likelihood Explanation
The preconditions are: (1) an oracle transiently returns `0` — explicitly modeled as reachable in the Certora specs; (2) the borrower has `debt > 0` and non-zero collateral at the targeted index. No special privilege is required unless `liquidatorGate` is configured. Oracles can return `0` due to uninitialized price feeds, stale/gap conditions, or error-handling paths that return 0 instead of reverting. The exploit is repeatable for any such market.

## Recommendation
Add a `require(liquidatedCollatPrice > 0)` guard after `liquidatedCollatPrice` is assigned at line 611, reverting if the oracle returns zero for the collateral being liquidated. Alternatively, skip or revert the entire liquidation when any active collateral oracle returns `0`, since a zero price makes health accounting unreliable. The oracle interface should also be documented to require non-zero returns, and integration tests should verify oracle revert behavior on zero prices.

## Proof of Concept
1. Deploy a market with a collateral whose oracle can be set to return `0`.
2. Have a borrower supply that collateral and borrow loan tokens (`debt > 0`).
3. Set the oracle to return `0`.
4. Call `liquidate` with `seizedAssets = borrower.collateral[collateralIndex]`, `repaidUnits = 0`, `postMaturityMode = false`.
5. Observe: attacker receives full collateral, transfers 0 loan tokens, borrower's debt is written off as bad debt (lenders lose), `withdrawable` unchanged.

A Foundry fuzz test asserting `repaidUnits > 0` whenever `seizedAssets > 0` and `liquidatedCollatPrice == 0` would immediately falsify the invariant.
