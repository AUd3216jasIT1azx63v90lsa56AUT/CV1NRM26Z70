Audit Report

## Title
Signature Malleability in `setIsAuthorized` Enables Front-Running to Consume Victim's Nonce - (File: `src/periphery/EcrecoverAuthorizer.sol`)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` passes raw `(v, r, s)` to `ecrecover` with no low-`s` guard. Because the function is permissionless, an attacker can observe a pending `setIsAuthorized` transaction, compute the secp256k1-malleable signature `(55-v, r, n-s)`, and front-run the victim to consume their nonce — causing the victim's transaction to revert with `InvalidNonce()` and wasting their gas. The authorization content is unchanged, but the victim loses gas and timing control over every submission.

## Finding Description
**Root cause:** No low-`s` constraint exists in `setIsAuthorized`. At line 31 of `src/periphery/EcrecoverAuthorizer.sol`:
```solidity
address signer = ecrecover(digest, signature.v, signature.r, signature.s);
```
The `Signature` struct (`src/periphery/interfaces/IEcrecoverAuthorizer.sol` lines 5–9) is a plain `(uint8 v, bytes32 r, bytes32 s)` with no range constraints on `s`. The secp256k1 curve allows two valid `(v, s)` pairs per `(r, message)`: `(v, s)` and `(55-v, n-s)`, both recovering to the same address.

**Permissionless entry:** `setIsAuthorized` performs no `msg.sender` validation against `authorization.authorizer`. Any caller may submit any structurally valid authorization with any valid signature form.

**Exploit flow:**
1. Victim signs `Authorization{authorizer, authorized, isAuthorized, nonce=N, deadline}` producing `(v, r, s)` and broadcasts `setIsAuthorized(auth, {v, r, s})`.
2. Attacker observes the mempool, computes `v' = 55 - v`, `s' = secp256k1_order - s`.
3. Attacker front-runs with `setIsAuthorized(auth, {v', r, s'})`.
4. Attacker's tx: nonce check at line 26 passes (`nonce[authorizer] == N`), nonce incremented to N+1; `ecrecover(digest, v', r, s')` returns the same `authorizer` address; `InvalidSignature()` check at line 32 passes; authorization is written to Midnight.
5. Victim's tx: line 26 fails (`nonce[authorizer]` is now N+1 ≠ N) → reverts with `InvalidNonce()`.

**Why existing checks fail:** The `InvalidNonce()` check at line 26 and `InvalidSignature()` check at line 32 both pass for the malleable form — the nonce is valid at the time of the attacker's front-run, and `ecrecover` returns a non-zero address identical to the original signer.

## Impact Explanation
The victim's `setIsAuthorized` transaction is griefed: it reverts with `InvalidNonce()` and the victim wastes gas. The authorization content is fixed by the victim's signed struct, so the attacker cannot alter what is written to Midnight — the same authorization is applied. Concrete harms: (1) victim's tx reverts and gas is lost on every submission; (2) the victim loses control over the timing and submission of their own signed messages. The attack is repeatable for every new signature the victim produces, enabling sustained griefing. This constitutes service degradation against a specific user's ability to self-submit authorizations.

## Likelihood Explanation
Preconditions are minimal: the attacker only needs to monitor the public mempool and perform one modular subtraction (`s' = n - s`) and one subtraction (`v' = 55 - v`). No special role, capital, or privileged access is required. The attack costs only the gas of a single transaction per victim submission and is repeatable indefinitely. On any chain without private mempools, this is trivially executable by any external actor.

## Recommendation
Add a low-`s` guard in `setIsAuthorized` before the `ecrecover` call:
```solidity
require(
    uint256(signature.s) <= 0x7FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF5D576E7357A4501DDFE92F46681B20A0,
    InvalidSignature()
);
```
This enforces canonical signature form, making the malleable counterpart always invalid. Alternatively, use OpenZeppelin's `ECDSA.recover`, which includes this check. Apply the same fix to `EcrecoverRatifier.isRatified` at `src/ratifiers/EcrecoverRatifier.sol` line 42, which has the same pattern.

// Foundry test sketch
function testMalleableFrontRun() public {
    // 1. Victim signs authorization with nonce 0
    Authorization memory auth = makeAuthorization(borrower, lender, true);
    Signature memory victimSig = signAuthorization(auth, borrower);

    // 2. Attacker computes malleable signature
    uint256 n = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141;
    Signature memory attackerSig = Signature({
        v: 55 - victimSig.v,
        r: victimSig.r,
        s: bytes32(n - uint256(victimSig.s))
    });

    // 3. Attacker front-runs — succeeds, nonce advances to 1
    ecrecoverAuthorizer.setIsAuthorized(auth, attackerSig);
    assertEq(ecrecoverAuthorizer.nonce(borrower), 1);

    // 4. Victim's original tx reverts with InvalidNonce
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(auth, victimSig);
}
