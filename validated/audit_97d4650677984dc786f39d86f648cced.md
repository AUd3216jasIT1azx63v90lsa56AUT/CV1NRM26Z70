Audit Report

## Title
Unconditional `safeTransfer(token, receiver, 0)` in `liquidate` permanently blocks bad-debt realization for tokens that revert on zero-value transfers - (`src/Midnight.sol` / `src/libraries/SafeTransferLib.sol`)

## Summary
In `Midnight.liquidate`, the call `SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets)` at line 696 is executed unconditionally, including when `seizedAssets == 0`. For collateral tokens that revert on zero-value `transfer` calls, this causes the entire liquidation transaction to revert, atomically rolling back all bad-debt state changes (lossFactor, totalUnits, position.debt) computed earlier in the same call. The protocol's own NatSpec at line 577 explicitly documents `seizedAssets=0, repaidUnits=0` as the intended path for pure bad-debt realization with no token movement, but the unconditional transfer violates this guarantee.

## Finding Description

**Root cause:** Missing guard around `safeTransfer` at line 696.

**Code path in `src/Midnight.sol`:**

1. **Lines 626–641**: Bad debt is realized — `_position.debt` is decremented, `_marketState.lossFactor` is updated, `_marketState.totalUnits` is decremented, `_marketState.continuousFeeCredit` is adjusted. All writes go to storage. [1](#0-0) 

2. **Lines 643–677**: The collateral-seizure and debt-repayment block is correctly guarded by `if (repaidUnits > 0 || seizedAssets > 0)`. When both are 0, this block is skipped and `seizedAssets` remains 0. [2](#0-1) 

3. **Line 696**: `SafeTransferLib.safeTransfer(market.collateralParams[collateralIndex].token, receiver, seizedAssets)` — called **unconditionally** with `seizedAssets == 0`. [3](#0-2) 

**In `src/libraries/SafeTransferLib.sol` lines 15–20**: The call `token.call(abi.encodeCall(IERC20.transfer, (to, value)))` is made. If the token reverts on `transfer(receiver, 0)`, `success == false` and the assembly block propagates the revert, unwinding all storage writes from step 1. [4](#0-3) 

**Why existing checks fail:** The `if (repaidUnits > 0 || seizedAssets > 0)` guard at line 643 correctly skips collateral accounting but there is no equivalent guard around the `safeTransfer` at line 696. The Certora formal verification rule `liquidateLossFactorDoesNotRevert` in `certora/specs/LossFactor.spec` explicitly summarizes `SafeTransferLib.safeTransfer` as `NONDET` (non-reverting), so the revert path is invisible to the formal proofs. [5](#0-4) [6](#0-5) 

The NatSpec at line 577 explicitly documents the intended behavior that is broken: [7](#0-6) 

## Impact Explanation

For any market whose collateral token reverts on zero-value `transfer` calls:
- Bad debt can never be socialized: `lossFactor` is never updated, so lenders are never slashed proportionally.
- `totalUnits` is never decremented to reflect the loss, breaking the invariant that every credit unit has matching debt or valid settled/loss state.
- The borrower's `position.debt` is never reduced by the bad-debt amount, leaving the position permanently inconsistent.
- The market becomes permanently insolvent: no liquidator can call `liquidate(..., 0, 0, ...)` to realize bad debt, and the unhealthy position remains unliquidatable via the pure bad-debt path.

## Likelihood Explanation

**Preconditions:**
1. A market is created with a collateral token that reverts on zero-value `transfer` calls (a known behavior of certain ERC20 tokens with strict input validation).
2. A borrower accumulates bad debt (collateral value falls below debt value) — a normal market condition.

Any liquidator calling `liquidate(market, idx, 0, 0, borrower, false, receiver, address(0), "")` triggers the revert. The condition is repeatable on every block as long as the token behavior and bad-debt state persist. No special privilege is required.

## Recommendation

Guard the `safeTransfer` call at line 696 with a check on `seizedAssets`, mirroring the existing guard at line 643:

```solidity
if (seizedAssets > 0) {
    SafeTransferLib.safeTransfer(
        market.collateralParams[collateralIndex].token,
        receiver,
        seizedAssets
    );
}
```

This ensures that pure bad-debt realization (`seizedAssets == 0, repaidUnits == 0`) never attempts a zero-value token transfer, consistent with the documented NatSpec guarantee at line 577.

## Proof of Concept

1. Deploy a collateral token that reverts on `transfer(to, 0)`.
2. Create a market using that token as collateral.
3. Open a borrow position and let the collateral value fall below the debt value (bad debt condition).
4. Call `liquidate(market, 0, 0, 0, borrower, false, receiver, address(0), "")`.
5. Observe: the call reverts due to `token.transfer(receiver, 0)` reverting inside `SafeTransferLib.safeTransfer` at line 696.
6. Confirm: `marketState[id].lossFactor`, `marketState[id].totalUnits`, and `position[id][borrower].debt` are all unchanged — bad debt was never realized.
