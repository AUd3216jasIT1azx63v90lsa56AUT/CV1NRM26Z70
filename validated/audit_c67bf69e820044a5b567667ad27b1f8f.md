Audit Report

## Title
Division-by-Zero in `sellerAssetsToUnits` When `sellerPrice == 0` Causes Permanent DoS of `supplyCollateralAndSellWithAssetsTarget` - (File: src/periphery/TakeAmountsLib.sol)

## Summary
`TakeAmountsLib.sellerAssetsToUnits` calls `mulDivUp(targetSellerAssets, WAD, sellerPrice)` where `sellerPrice = offerPrice - settlementFee` for buy offers. When `offerPrice == settlementFee`, `sellerPrice` is zero and `mulDivUp` reverts via arithmetic underflow on `d - 1`. The core `Midnight.take()` handles this identically-computed zero `sellerPrice` without reverting (producing `sellerAssets = 0` via `mulDivDown`). Because `sellerAssetsToUnits` is called outside the try/catch block in `supplyCollateralAndSellWithAssetsTarget`, the revert propagates and permanently blocks any bundler call that includes the crafted offer.

## Finding Description

**Root cause — `mulDivUp` with zero denominator (`TakeAmountsLib.sol` lines 44–46):**

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
return
    offer.buy ? targetSellerAssets.mulDivUp(WAD, sellerPrice) : ...;
```

When `offer.buy == true` and `offerPrice == settlementFee`, `sellerPrice = 0`. `UtilsLib.mulDivUp` is implemented as:

```solidity
return (x * y + (d - 1)) / d;   // d = 0 → (d - 1) underflows → arithmetic revert
``` [1](#0-0) 

**Contrast with core `Midnight.take()` (lines 361–364):**

```solidity
uint256 sellerPrice = offer.buy ? offerPrice - _settlementFee : offerPrice;  // = 0
uint256 sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : ...;  // = 0, no revert
```

`mulDivDown(units, 0, WAD) = (units * 0) / WAD = 0` — the core succeeds. [2](#0-1) 

**Call site in `supplyCollateralAndSellWithAssetsTarget` (lines 285–300):**

```solidity
uint256 unitsToTake = min(
    TakeAmountsLib.sellerAssetsToUnits(   // ← NOT inside try/catch; reverts here
        MIDNIGHT, id, takes[i].offer, targetFilledSellerAssets - filledSellerAssets
    ),
    takes[i].units,
    ConsumableUnitsLib.consumableUnits(MIDNIGHT, id, takes[i].offer)
);
try IMidnight(MIDNIGHT).take(...) { ... } catch {}
``` [3](#0-2) 

The `sellerAssetsToUnits` call is outside the try/catch. Any revert propagates to the caller and reverts the entire bundler transaction.

**Precondition feasibility:**

- `settlementFee(id, 0) = settlementFeeCbp0 * CBP` where `CBP = 1e12` and `MAX_SETTLEMENT_FEE_0_DAYS = 14e12`, so `settlementFeeCbp0 ∈ [0, 14]`. [4](#0-3) 
- `tickToPrice` returns multiples of `PRICE_ROUNDING_STEP = 1e12`. Confirmed: `tickToPrice(2) = 1e12` (first non-zero price). [5](#0-4) 
- Tick prices are monotonically non-decreasing, so prices `1e12, 2e12, …, 14e12` are all reachable at some tick.
- Additionally, tick 0 always has `tickToPrice(0) = 0`. If `settlementFeeCbp0 = 0`, placing a buy offer at tick 0 also triggers the bug (both `offerPrice` and `settlementFee` are 0, giving `sellerPrice = 0`). [6](#0-5) 

**Exploit flow:**

1. Attacker reads `settlementFeeCbp0` for the target market.
2. Attacker finds an accessible tick `t` (multiple of `tickSpacing`) with `tickToPrice(t) == settlementFeeCbp0 * 1e12`.
3. Attacker signs and publishes a buy offer at tick `t` with a valid ratifier and expiry past maturity.
4. Market passes maturity; `zeroFloorSub(maturity, block.timestamp) = 0`, so `settlementFee(id, 0)` is used.
5. Victim calls `supplyCollateralAndSellWithAssetsTarget` with a `takes` array that includes the attacker's offer.
6. `sellerAssetsToUnits` computes `sellerPrice = 0`, calls `mulDivUp(target, WAD, 0)`, reverts.
7. Entire bundler call reverts; victim cannot complete the sell-via-periphery.

**Existing guards are insufficient:** The `sellerAssetsToUnits` NatSpec comment acknowledges it "Reverts if offerPrice < settlementFee in case of a buy offer (midnight reverts too)" but does not handle the `offerPrice == settlementFee` edge case, where the core succeeds but the periphery reverts. [7](#0-6) 

## Impact Explanation
Any call to `MidnightBundles.supplyCollateralAndSellWithAssetsTarget` that includes the attacker's offer in its `takes` array reverts unconditionally, regardless of `targetSellerAssets`. Automated aggregators or UI flows that enumerate all available buy offers are blocked from executing the bundler path for that market post-maturity. The core market is unaffected; only the periphery path is DoS'd.

## Likelihood Explanation
- **Preconditions**: Post-maturity market (time passes naturally); settlement fee at TTM=0 equals the price at some accessible tick (both are multiples of `1e12`; with `settlementFeeCbp0 ∈ [0, 14]` and tick prices covering that range, a matching tick exists for any fee value including zero).
- **Attacker cost**: Zero on-chain cost — only a signed offer message is needed.
- **Repeatability**: The attacker can re-sign the offer with a new expiry if the old one expires, maintaining the DoS indefinitely.
- **No admin dependency**: The attacker only reads the current fee and picks a matching tick; no privileged action is required.

## Recommendation
In `sellerAssetsToUnits`, add an early return of `type(uint256).max` (or `0`) when `sellerPrice == 0`, mirroring the core's behavior of producing zero `sellerAssets`. Alternatively, guard the `mulDivUp` call:

```solidity
if (sellerPrice == 0) return type(uint256).max; // no finite units can yield a positive sellerAssets
```

This matches the core's semantics: when `sellerPrice = 0`, any number of units yields `sellerAssets = 0`, so no finite unit count can reach a positive `targetSellerAssets`. Returning `type(uint256).max` causes the `min(...)` in the bundler to select the offer's own `units` cap, and the subsequent `take()` call (inside the try/catch) will either succeed with `sellerAssets = 0` or be skipped.

## Proof of Concept
```solidity
// Minimal Foundry test sketch
function testSellerAssetsToUnitsRevertWhenSellerPriceZero() public {
    // Set settlementFeeCbp0 = 1 (fee = 1e12 at TTM=0)
    midnight.setMarketSettlementFee(id, 0, 1e12);

    // Warp past maturity
    vm.warp(market.maturity + 1);

    // Attacker's buy offer at tick 2 (tickToPrice(2) = 1e12 == settlementFee)
    Offer memory attackOffer = offers[0]; // buy offer
    attackOffer.tick = 2; // tickToPrice(2) = 1e12
    attackOffer.expiry = block.timestamp + 1 days;
    attackOffer.maxUnits = type(uint256).max;

    Take[] memory takes = new Take[](1);
    takes[0] = Take({offer: attackOffer, units: type(uint256).max, ratifierData: hex""});

    // Victim's call reverts due to division-by-zero in sellerAssetsToUnits
    vm.prank(borrower);
    vm.expectRevert(); // arithmetic underflow in mulDivUp(target, WAD, 0)
    midnightBundles.supplyCollateralAndSellWithAssetsTarget(
        1e18, type(uint256).max, borrower, borrower,
        new CollateralSupply[](0), takes, 0, address(0)
    );
}
```

    /// @dev Forward: sellerAssets = offer.buy ? units.mulDivDown(sellerPrice, WAD) : units.mulDivUp(sellerPrice, WAD).
    /// @dev Assumes that id and offer.market match.
    /// @dev Reverts if offerPrice < settlementFee in case of a buy offer (midnight reverts too).
    /// @dev Returns a number of units for which take yields exactly targetSellerAssets (not necessarily the smallest).
    function sellerAssetsToUnits(address midnight, bytes32 id, Offer memory offer, uint256 targetSellerAssets)
        internal
        view
        returns (uint256)
    {
        uint256 offerPrice = TickLib.tickToPrice(offer.tick);
        uint256 settlementFee =
            IMidnight(midnight).settlementFee(id, UtilsLib.zeroFloorSub(offer.market.maturity, block.timestamp));
        uint256 sellerPrice = offer.buy ? offerPrice - settlementFee : offerPrice;
        return
            offer.buy ? targetSellerAssets.mulDivUp(WAD, sellerPrice) : targetSellerAssets.mulDivDown(WAD, sellerPrice);
    }
