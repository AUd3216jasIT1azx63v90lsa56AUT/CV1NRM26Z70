Audit Report

## Title
Rounding in `lossFactor` update prematurely saturates market when `_lossFactor = type(uint128).max - 1`, permanently blocking all future `take()` calls via `MarketLossFactorMaxedOut` - (File: src/Midnight.sol)

## Summary
When `_marketState.lossFactor` equals `type(uint128).max - 1`, the inner `mulDivDown` in the lossFactor update formula evaluates to `0` for any `badDebt >= 1`, regardless of how small `badDebt` is relative to `totalUnits`. This causes `lossFactor` to be set to `type(uint128).max`, permanently blocking all future `take()` calls on the affected market via `MarketLossFactorMaxedOut`, even though the market still has outstanding debt and credit.

## Finding Description

**Code path — `src/Midnight.sol` lines 626–633:**

```solidity
if (badDebt > 0) {
    _position.debt -= uint128(badDebt);
    uint256 _totalUnits = _marketState.totalUnits;
    uint256 _lossFactor = _marketState.lossFactor;
    _marketState.lossFactor = UtilsLib.toUint128(
        type(uint128).max - (type(uint128).max - _lossFactor).mulDivDown(_totalUnits - badDebt, _totalUnits)
    );
``` [1](#0-0) 

**`mulDivDown` — `src/libraries/UtilsLib.sol` lines 29–31:**

```solidity
function mulDivDown(uint256 x, uint256 y, uint256 d) internal pure returns (uint256) {
    return (x * y) / d;
}
``` [2](#0-1) 

**Root cause arithmetic:**

When `_lossFactor = type(uint128).max - 1`:
- `type(uint128).max - _lossFactor = 1`
- The call becomes `mulDivDown(1, _totalUnits - badDebt, _totalUnits)` = `(1 * (_totalUnits - badDebt)) / _totalUnits`
- For any `badDebt >= 1`, `_totalUnits - badDebt < _totalUnits`, so the result is `< 1`, which truncates to `0`
- Therefore `newLossFactor = type(uint128).max - 0 = type(uint128).max`

This holds even when `badDebt = 1` and `totalUnits = 1_000_000` (0.0001% bad debt fraction).

**Reachability of `_lossFactor = type(uint128).max - 1`:**

The state is reachable through normal market operation. Concrete example: `oldLossFactor = type(uint128).max - 2`, `badDebt = 1`, `totalUnits = 3`:
- `mulDivDown(2, 2, 3) = 4/3 = 1`
- `newLossFactor = type(uint128).max - 1` ✓

The `lossFactor` update formula is monotonically non-decreasing, so any market that has experienced sufficient bad debt events over its lifetime can reach this state.

**Exploit flow:**
1. Market accumulates bad debt over its lifetime, driving `_lossFactor` to `type(uint128).max - 1`
2. An unprivileged liquidator calls `liquidate(market, 0, 0, 0, borrower, false, receiver, address(0), "")` on any borrower with `badDebt >= 1`
3. `mulDivDown(1, totalUnits - 1, totalUnits) = 0`
4. `_marketState.lossFactor` is set to `type(uint128).max`
5. All subsequent calls to `take()` revert with `MarketLossFactorMaxedOut`

**`take()` guard — `src/Midnight.sol` line 349:**

```solidity
require(_marketState.lossFactor < type(uint128).max, MarketLossFactorMaxedOut());
``` [3](#0-2) 

**Why existing checks fail:**

There is no guard preventing `lossFactor` from reaching `type(uint128).max` when `badDebt < totalUnits`. The `liquidate()` function only requires `badDebt > 0` to enter the update block. [4](#0-3) 

The Certora formal verification rule `lossFactorChangesIffBadDebt` explicitly preconditions on `lossFactorBefore < max_uint128` at line 53, meaning it does not cover the near-max regime and provides no protection here. [5](#0-4) 

The `liquidateLossFactorDoesNotRevert` rule verifies non-reversion but does not verify that `lossFactor` does not prematurely saturate. [6](#0-5) 

## Impact Explanation
Once `lossFactor = type(uint128).max`, `take()` permanently reverts with `MarketLossFactorMaxedOut` for the affected market. Any remaining lenders with non-zero credit cannot take new positions or reduce their exposure through the normal take mechanism. The market is frozen by a single liquidation call that realizes as little as 1 unit of bad debt, even when the market still has `totalUnits - 1` units of outstanding debt. This constitutes a permanent, irreversible freeze of a market's core functionality triggered by an unprivileged actor.

## Likelihood Explanation
**Preconditions:** `_lossFactor` must reach `type(uint128).max - 1`. This requires the market to have experienced many bad debt events, which is realistic for distressed markets over their lifetime. The state is reachable through normal liquidation activity without any privileged action.

**Feasibility:** Once the precondition is met, any unprivileged liquidator can trigger the saturation with a single zero-cost call (`seizedAssets = 0`, `repaidUnits = 0`). No tokens need to be transferred. The call is permissionless as long as the borrower is liquidatable (unhealthy or post-maturity) and `market.liquidatorGate == address(0)`. [7](#0-6) 

**Repeatability:** The trigger is a single transaction. The state change is permanent and irreversible.

## Recommendation
Before setting `_marketState.lossFactor`, clamp the result to `type(uint128).max` only when `badDebt >= _totalUnits` (i.e., total loss). For partial bad debt, use `mulDivUp` instead of `mulDivDown` for the inner multiplication to avoid rounding the complement to zero:

```solidity
_marketState.lossFactor = UtilsLib.toUint128(
    type(uint128).max - (type(uint128).max - _lossFactor).mulDivUp(_totalUnits - badDebt, _totalUnits)
);
```

Using `mulDivUp` ensures that when `badDebt < totalUnits`, the complement `(type(uint128).max - _lossFactor)` is scaled down by a factor strictly less than 1 but rounds up to at least 1, preventing premature saturation. Additionally, add an explicit invariant check or Certora rule covering the near-max regime (`lossFactorBefore == max_uint128 - 1`) to close the formal verification gap. [8](#0-7) 

## Proof of Concept
**Minimal manual steps:**

1. Deploy a market and drive `_marketState.lossFactor` to `type(uint128).max - 1` via repeated bad debt liquidations (e.g., using the concrete path: `oldLossFactor = type(uint128).max - 2`, `badDebt = 1`, `totalUnits = 3` → `newLossFactor = type(uint128).max - 1`).
2. Ensure a borrower exists with `debt > 0` and no collateral (so `badDebt = position.debt >= 1`).
3. Call `liquidate(market, 0, 0, 0, borrower, false, receiver, address(0), "")`.
4. Assert `marketState[id].lossFactor == type(uint128).max`.
5. Attempt any `take()` call on the market — it reverts with `MarketLossFactorMaxedOut`.

**Fuzz test plan:** Fuzz `_lossFactor` in `[type(uint128).max - 100, type(uint128).max - 1]` with `badDebt` in `[1, totalUnits - 1]` and assert `newLossFactor < type(uint128).max` whenever `badDebt < totalUnits`. The current code will fail this assertion for all inputs where `type(uint128).max - _lossFactor < totalUnits`. [9](#0-8)

rule lossFactorChangesIffBadDebt(env e, Midnight.Market market, uint256 collateralIndex, uint256 seizedAssets, uint256 repaidUnits, address borrower, address receiver, address callback, bytes data, bool postMaturityMode) {
    bytes32 id = summaryToId(market);
    uint128 lossFactorBefore = currentContract.marketState[id].lossFactor;
    uint256 totalUnitsBefore = totalUnits(id);

    require lossFactorBefore < max_uint128, "market lossFactor must not be saturated";

    liquidate(e, market, collateralIndex, seizedAssets, repaidUnits, borrower, postMaturityMode, receiver, callback, data);

    bool lossFactorChanged = currentContract.marketState[id].lossFactor != lossFactorBefore;
    bool badDebtOccurred = totalUnits(id) < totalUnitsBefore;

    assert lossFactorChanged <=> badDebtOccurred;
}
