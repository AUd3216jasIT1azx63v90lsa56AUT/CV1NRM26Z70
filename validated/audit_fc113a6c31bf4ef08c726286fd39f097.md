Audit Report

## Title
Zero-price oracle enables collateral seizure with zero loan token repayment via `seizedAssets` input path - (File: src/Midnight.sol)

## Summary
When the oracle for the liquidated collateral returns `0`, calling `liquidate` with `seizedAssets > 0` and `repaidUnits = 0` causes `repaidUnits` to be computed as `0` via `mulDivUp` rounding. The borrower's collateral is fully transferred to the liquidator while their debt is decremented by zero, violating the core accounting invariant that collateral cannot be seized without repaying corresponding debt value.

## Finding Description

**Root cause — `src/Midnight.sol` lines 607–618:**

The while loop iterates `collateralBitmap`, calls `IOracle(_collateralParam.oracle).price()` for each active collateral, and assigns the result to `liquidatedCollatPrice` when `i == collateralIndex`. When the oracle returns `0`, `liquidatedCollatPrice = 0`. That collateral also contributes `0` to `maxDebt` (line 613), making `originalDebt > maxDebt` trivially true for any borrower with debt.

**`mulDivUp` — `src/libraries/UtilsLib.sol` line 35:**
```solidity
return (x * y + (d - 1)) / d;
```

**Computation at line 650 when `seizedAssets > 0` and `liquidatedCollatPrice = 0`:**
```solidity
repaidUnits = seizedAssets.mulDivUp(liquidatedCollatPrice, ORACLE_PRICE_SCALE).mulDivUp(WAD, lif);
```
- Step 1: `mulDivUp(seizedAssets, 0, ORACLE_PRICE_SCALE)` = `(0 + ORACLE_PRICE_SCALE − 1) / ORACLE_PRICE_SCALE` = `0` (integer division)
- Step 2: `mulDivUp(0, WAD, lif)` = `(0 + lif − 1) / lif` = `0`

So `repaidUnits = 0`.

**Why existing checks fail:**
- `atMostOneNonZero(0, seizedAssets)` at line 595 → `true` (attacker passes `repaidUnits = 0`, satisfying the constraint)
- `_position.debt > 0` at line 596 → `true` (borrower has debt)
- `NotLiquidatable` at lines 620–624: oracle returning `0` zeroes the collateral's `maxDebt` contribution, making `originalDebt > maxDebt` trivially true
- `maxRepaid` RCF check at lines 655–668: `repaidUnits = 0 ≤ maxRepaid` always passes

**Critical asymmetry:** The `repaidUnits` input path (line 652) divides by `liquidatedCollatPrice` directly (`mulDivDown(ORACLE_PRICE_SCALE, liquidatedCollatPrice)`), which reverts with division-by-zero when price is `0` — providing accidental protection. The `seizedAssets` input path (line 650) never divides by `liquidatedCollatPrice`, so it silently computes `repaidUnits = 0` with no revert.

**Resulting state mutations (lines 670–677, 696, 717):**
- `_position.collateral[collateralIndex] -= seizedAssets` — collateral fully removed
- `_position.collateralBitmap` bit cleared if `newCollateral == 0`
- `_marketState.withdrawable += 0` — pool unchanged
- `_position.debt -= 0` — debt unchanged
- `safeTransfer(collateralToken, receiver, seizedAssets)` — collateral sent to attacker
- `safeTransferFrom(loanToken, payer, address(this), 0)` — zero loan tokens paid

**Formal verification gap — `certora/specs/NoDivisionByZero.spec` line 124:**
The `noDivisionByZeroLiquidate` rule requires `ghostPrice(...) > 0` as a precondition, explicitly excluding the `price = 0` case. `certora/README.md` line 98 states: *"except for `liquidate` if the liquidated collateral is not activated or has a price of 0."*

**Coverage gap — `certora/specs/Reverts.spec` lines 246–253:**
`oracleZeroCausesLiquidateWithRepaidRevert` only proves that `liquidate` reverts when `repaidUnits > 0` is passed with a zero-price oracle. There is no corresponding rule proving that `liquidate` reverts when `seizedAssets > 0` is passed with a zero-price oracle — the uncovered path.

## Impact Explanation
A liquidator seizes the borrower's entire collateral at `collateralIndex` while repaying zero loan tokens. The borrower's debt is not reduced, yet their collateral balance drops to zero and the collateral bitmap bit is cleared. The protocol's `withdrawable` pool is not increased. This constitutes direct, irreversible theft of borrower collateral with no corresponding debt relief — a critical accounting invariant violation causing direct loss of user funds. Severity: Critical.

## Likelihood Explanation
**Required preconditions:**
1. The oracle for the targeted collateral returns `0` — explicitly modeled as reachable in the Certora specs (`Reverts.spec` ghost `singleZeroOracle`, `forceOracleReturnZero`). Reachable via: uninitialized price feed, price feed gap, oracle contract returning `0` on error, or transiently stale feed.
2. The borrower has `debt > 0` and a non-zero balance at `collateralIndex` with the bitmap bit set.
3. No special privilege required — any address can call `liquidate`.

The exploit is repeatable for any market whose oracle can transiently return `0`. The protocol's own formal specs explicitly model this scenario, confirming it is considered a reachable state.

## Recommendation
Add an explicit check after computing `liquidatedCollatPrice` to require it is non-zero before proceeding with a non-zero `seizedAssets` or `repaidUnits` liquidation:

```solidity
if (repaidUnits > 0 || seizedAssets > 0) {
    require(liquidatedCollatPrice > 0, ZeroPriceOracle());
    // ... rest of liquidation logic
}
```

Alternatively, after computing `repaidUnits` from `seizedAssets`, add:
```solidity
require(repaidUnits > 0, ZeroRepaidUnits());
```

This mirrors the accidental protection already present in the `repaidUnits` input path (which reverts via division by zero at line 652) and closes the asymmetry between the two input paths.

## Proof of Concept
1. Deploy a market with a collateral oracle that can be set to return `0`.
2. Have a borrower supply collateral and borrow loan tokens (establishing `debt > 0`).
3. Set the oracle to return `0`.
4. Call `liquidate(market, collateralIndex, seizedAssets=borrowerCollateralBalance, repaidUnits=0, borrower, false, attacker, address(0), "")`.
5. Observe: attacker receives `seizedAssets` collateral tokens, pays `0` loan tokens, borrower's debt is unchanged, borrower's collateral is zeroed.

Minimal Foundry test:
```solidity
function testZeroPriceLiquidation() public {
    // Setup: borrower has collateral and debt
    oracle.setPrice(0); // oracle returns 0
    uint256 collateralBefore = midnight.collateral(id, borrower, collateralIndex);
    uint256 debtBefore = midnight.debtOf(id, borrower);
    
    vm.prank(attacker);
    (uint256 seized, uint256 repaid) = midnight.liquidate(
        market, collateralIndex, collateralBefore, 0, borrower, false, attacker, address(0), ""
    );
    
    assertEq(seized, collateralBefore);
    assertEq(repaid, 0); // zero repayment
    assertEq(midnight.debtOf(id, borrower), debtBefore); // debt unchanged
    assertEq(midnight.collateral(id, borrower, collateralIndex), 0); // collateral gone
}
```
