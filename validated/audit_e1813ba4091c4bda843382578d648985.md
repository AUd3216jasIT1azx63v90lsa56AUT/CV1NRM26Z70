### Title
No Minimum Position Size Allows Dust Debt Positions That Are Unprofitable to Liquidate, Causing Socialized Bad Debt - (File: src/Midnight.sol)

---

### Summary

The `take()` function in `Midnight.sol` imposes no minimum on the `units` parameter when a seller increases their debt. This allows any borrower to open a position with arbitrarily small debt (as low as 1 unit / 1 wei of loan token). When such a dust position becomes undercollateralized, the gas cost of calling `liquidate()` exceeds the liquidation incentive profit, so no rational liquidator will act. The resulting bad debt is socialized among all lenders in the market via the `lossFactor` mechanism.

---

### Finding Description

**Vulnerability class**: Economic incentive failure / bad debt accumulation (analogous to the external report's "lack of incentives to liquidate small positions").

**Root cause — no minimum debt size in `take()`**

When a taker acts as a seller (borrower), `take()` computes `sellerDebtIncrease` and adds it directly to the position's debt with no lower-bound check:

```solidity
uint256 sellerDebtIncrease = units - sellerCreditDecrease;   // line 384
...
sellerPos.debt += UtilsLib.toUint128(sellerDebtIncrease);    // line 414
```

`units` is a caller-supplied `uint256` with no minimum enforced anywhere in `take()`. A borrower can pass `units = 1`, resulting in a debt position of 1 unit (≈ 1 wei of loan token at a price near WAD). [1](#0-0) 

**`touchMarket()` does not validate `rcfThreshold`**

The `Market` struct carries a `rcfThreshold` field intended to deactivate the Recovery Close Factor (RCF) for small positions:

> "The RCF is deactivated for small collateral amount, essentially to mitigate issues with liquidations that are too small compared to the gas cost."

However, `touchMarket()` validates `maturity`, `collateralParams` length, LLTV tiers, and `maxLif`, but performs **no validation of `rcfThreshold`**. It can be set to `0` by any market creator. [2](#0-1) [3](#0-2) 

When `rcfThreshold = 0`, the RCF bypass condition:

```solidity
_position.collateral[collateralIndex].mulDivDown(liquidatedCollatPrice, ORACLE_PRICE_SCALE)
    .mulDivDown(WAD, lif).zeroFloorSub(maxRepaid) < market.rcfThreshold   // line 664-665
```

is **never satisfied** (left side ≥ 0, right side = 0), so the RCF is always active and liquidators are further constrained to only repay up to `maxRepaid` units — making dust liquidations even less profitable. [4](#0-3) 

**`liquidate()` only checks `debt > 0`**

The liquidation entry point has no minimum position size guard:

```solidity
require(_position.debt > 0, NotBorrower());   // line 596
```

A position with 1 unit of debt passes this check. The liquidator must pay gas (tens of thousands of gas units on Ethereum) to seize collateral worth ≈ 1 wei × LIF, which is economically irrational. [5](#0-4) 

**Bad debt socialization path**

When a dust position is undercollateralized and no liquidator acts, the bad debt is never cleared. If eventually triggered (e.g., post-maturity), the `lossFactor` is updated, slashing all lenders proportionally:

```solidity
_marketState.lossFactor = UtilsLib.toUint128(
    type(uint128).max - (type(uint128).max - _lossFactor).mulDivDown(_totalUnits - badDebt, _totalUnits)
);
``` [6](#0-5) 

---

### Impact Explanation

Any undercollateralized dust position that goes unliquidated contributes to `badDebt`, which is socialized among all lenders in the market via `lossFactor`. At each lender's next interaction, their `credit` is slashed proportionally via `updatePosition`. An attacker who deliberately opens many dust positions across permissionlessly created markets (or within a single market by repeatedly borrowing and repaying to fragment state) can cause measurable, permanent loss to lenders with no recovery path. The protocol's own comment acknowledges this class of risk but the mitigation (`rcfThreshold`) is incomplete: it only relaxes the RCF cap, it does not ensure the liquidation is economically viable. [7](#0-6) 

---

### Likelihood Explanation

- **Attacker preconditions**: None beyond being a normal protocol user. No privileged keys required.
- **Market creation**: Permissionless via `touchMarket()`. An attacker can create a market with `rcfThreshold = 0` to maximize the constraint on liquidators.
- **Position opening**: Any call to `take()` with `units = 1` as a seller creates a 1-unit debt position. The attacker only needs to supply the minimum collateral to pass the health check at borrow time.
- **Trigger**: A small oracle price drop (or natural market movement) makes the position undercollateralized. Gas cost on Ethereum mainnet (~50k–100k gas × gas price) will always exceed the LIF profit on a 1-wei position.
- **Realistic scenario**: An attacker opens dust positions in a legitimate market (not a custom one) to harm existing lenders, at the cost of losing their own dust collateral — a low-cost griefing attack.

---

### Recommendation

1. **Enforce a minimum debt size in `take()`**: When `sellerDebtIncrease > 0`, require `sellerDebtIncrease >= MIN_POSITION_SIZE` where `MIN_POSITION_SIZE` is a protocol constant set high enough to ensure liquidation is always profitable (e.g., covering worst-case gas cost at a reasonable gas price).

2. **Validate `rcfThreshold` in `touchMarket()`**: Require `market.rcfThreshold >= MIN_POSITION_SIZE` so that the RCF bypass is always reachable for positions near the minimum size, ensuring full liquidation is possible.

3. **Alternatively, enforce a minimum collateral value at borrow time**: In `take()`, after computing `sellerDebtIncrease`, verify that the seller's total collateral value (in loan token terms) exceeds a protocol-defined floor.

---

### Proof of Concept

**Setup**: Ethereum mainnet fork. Gas price = 10 gwei. `liquidate()` costs ~80,000 gas = 800,000 gwei = 0.0008 ETH. LIF for LLTV=0.965 with low cursor ≈ 1.0143 (i.e., ~1.43% incentive).

**Steps**:

1. Deploy/use a market with `loanToken = USDC`, `lltv = 0.965e18`, `rcfThreshold = 0`, `maxLif = maxLif(0.965e18, LIQUIDATION_CURSOR_LOW)`.

2. Attacker calls `supplyCollateral()` with `assets = 2` (2 wei of collateral token).

3. Attacker calls `take()` with `units = 1` as seller (borrower), creating a debt position of 1 unit.
   - `sellerDebtIncrease = 1` — no revert, no minimum check.
   - `sellerPos.debt = 1`. [8](#0-7) 

4. Oracle price drops so `collateral * price / ORACLE_PRICE_SCALE * lltv / WAD < 1` (position undercollateralized).

5. A liquidator evaluates the call:
   - `seizedAssets ≈ 1 * LIF / price ≈ 1 wei of collateral`
   - Profit = 1 wei of collateral − 1 unit of loan token repaid ≈ 0 (or negative after gas)
   - Gas cost = ~0.0008 ETH >> profit
   - Liquidator does not act.

6. Bad debt = 1 unit remains in the position. If triggered post-maturity, `lossFactor` increases, slashing all lenders. [9](#0-8) 

7. Scaled to 10,000 such dust positions across the same market (each opened by a different address or the same address across different markets), the aggregate bad debt becomes material while the attacker's total cost is only the dust collateral deposited.
