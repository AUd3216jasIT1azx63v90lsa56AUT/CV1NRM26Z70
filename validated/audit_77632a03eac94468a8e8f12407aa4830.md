Audit Report

## Title
Zero-unit take on fully-consumed `maxAssets` offer dispatches maker callback indefinitely — (`src/Midnight.sol`)

## Summary
When `offer.maxAssets > 0` and `consumed[offer.maker][offer.group] == offer.maxAssets`, calling `take` with `units=0` adds zero to `consumed`, causing the cap check to pass trivially every time. Because no guard prevents callback dispatch on a zero-unit take, the maker's `onBuy`/`onSell` callback is invoked with all-zero arguments an unlimited number of times by any unprivileged taker.

## Finding Description

**Root cause — no `units > 0` guard before callback dispatch in `take()`.**

**Step 1 — asset computation always yields zero when `units=0`:**

`buyerAssets` and `sellerAssets` are computed as `units.mulDivDown/Up(price, WAD)`. [1](#0-0) 
With `units=0`, both values are `0` regardless of price.

**Step 2 — consumed cap check passes trivially:**

```solidity
newConsumed = consumed[offer.maker][offer.group] += 0;  // unchanged
require(newConsumed <= offer.maxAssets);                 // maxAssets <= maxAssets → always passes
``` [2](#0-1) 

**Step 3 — position deltas are all zero:**

`buyerCreditIncrease = zeroFloorSub(0, debt) = 0`, `sellerCreditDecrease = min(0, credit) = 0`, `sellerDebtIncrease = 0`. All position writes are no-ops. [3](#0-2) 

**Step 4 — callbacks are dispatched unconditionally:**

```solidity
if (buyerCallback != address(0)) {
    IBuyCallback(buyerCallback).onBuy(id, market, 0, 0, 0, buyer, callbackData);
}
``` [4](#0-3) 
```solidity
if (sellerCallback != address(0)) {
    ISellCallback(sellerCallback).onSell(id, market, 0, 0, 0, seller, receiver, callbackData);
}
``` [5](#0-4) 
There is no `require(units > 0)` or `require(newConsumed > consumedBefore)` guard anywhere before these dispatches.

**Step 5 — token transfers of zero succeed:**

`safeTransferFrom(..., 0)` succeeds for all standard ERC20 tokens, so the call completes without revert. [6](#0-5) 

**Why existing protections fail:**

- `ConsumedAssets` only fires when `newConsumed > maxAssets`; adding 0 never triggers it.
- The Certora rule `fullyConsumedOfferRevertsOnNonTrivialTake` is scoped to `offer.maxAssets == 0` (units mode) only — the assets-mode equivalent is absent. [7](#0-6) 
- `takeConsumedAtMaxUnchangedAssets` only asserts `consumed` doesn't change, not that callbacks are suppressed. [8](#0-7) 
- The existing `testBugBuyMaxAssetsBypass` test already documents the related zero-assets bypass with `units=1` at a low tick, confirming the consumed-check gap for the assets branch is a known code path. The `units=0` variant is price-independent and always reachable. [9](#0-8) 

## Impact Explanation
Any unprivileged taker can invoke the maker's `onBuy` or `onSell` callback contract an unlimited number of times after the offer is fully consumed, passing all-zero amounts. If the callback performs state mutations on zero-amount calls (e.g., collateral deposits, approval grants, counter increments, token pulls from an allowance), those side effects are triggered without any real fill. At minimum this constitutes unbounded gas grief; at maximum it causes state corruption of the maker's callback contract.

## Likelihood Explanation
Preconditions are trivially reachable: any offer with `maxAssets > 0` that has been fully consumed (a normal lifecycle event) and has a non-zero `offer.callback` is permanently vulnerable. The attacker requires no special role — only the ability to call `take` as `msg.sender == taker`. The attack is repeatable in a single transaction via `multicall` or across multiple transactions at no cost beyond gas.

## Recommendation
Add a guard at the top of `take()` (or immediately before callback dispatch) that rejects zero-unit takes:

```solidity
require(units > 0, ZeroUnits());
```

Alternatively, gate callback dispatch on `units > 0`:

```solidity
if (buyerCallback != address(0) && units > 0) { ... }
if (sellerCallback != address(0) && units > 0) { ... }
```

Additionally, add a Certora rule analogous to `fullyConsumedOfferRevertsOnNonTrivialTake` for the `maxAssets > 0` branch to close the formal-verification gap. [7](#0-6) 

## Proof of Concept
Extend `testBugBuyMaxAssetsBypass` with `units=0`:

```solidity
function testZeroUnitCallbackReplay() public {
    // Deploy a callback contract that counts onBuy invocations
    CountingCallback cb = new CountingCallback();
    lenderOffer.callback = address(cb);
    lenderOffer.maxUnits = 0;
    lenderOffer.maxAssets = 1e18;

    // Fully consume the offer
    vm.prank(lender);
    midnight.setConsumed(lenderOffer.group, lenderOffer.maxAssets, lender);

    // Replay zero-unit takes N times
    for (uint i = 0; i < 10; i++) {
        take(0, borrower, lenderOffer);
    }

    // Callback was invoked 10 times with zero amounts
    assertEq(cb.count(), 10);
}
```

Each iteration passes the consumed cap check (adding 0), transfers 0 tokens, and dispatches `onBuy` with all-zero arguments, confirming unbounded callback replay.

/// If consumed is already at or above maxAssets before a take in assets mode, it remains unchanged.
rule takeConsumedAtMaxUnchangedAssets(env e, Midnight.Offer offer, bytes ratifierData, uint256 units, address taker, address receiver, address takerCallback, bytes takerCallbackData) {
    require offer.maxUnits == 0;

    uint256 consumedBefore = consumed(offer.maker, offer.group);

    take(e, offer, ratifierData, units, taker, receiver, takerCallback, takerCallbackData);

    assert consumedBefore >= offer.maxAssets => consumed(offer.maker, offer.group) == consumedBefore;
}

**File:** certora/specs/Consume.spec (L99-111)
```text
/// A fully-consumed offer in units mode only allows no-op takes.
rule fullyConsumedOfferRevertsOnNonTrivialTake(env e, Midnight.Offer offer, bytes ratifierData, uint256 units, address taker, address receiver, address takerCallback, bytes takerCallbackData) {
    require offer.maxAssets == 0;

    uint256 consumedBefore = consumed(offer.maker, offer.group);

    require offer.maxUnits > 0 && consumedBefore >= offer.maxUnits, "assume the offer is fully consumed";

    take(e, offer, ratifierData, units, taker, receiver, takerCallback, takerCallbackData);

    // If take does not revert, its input has to be zero.
    assert units == 0;
}
```
