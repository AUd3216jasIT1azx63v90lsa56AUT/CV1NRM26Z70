Audit Report

## Title
Missing Bytecode Check in `touchMarket` Allows Permanent Creation of Unusable Markets - (File: src/Midnight.sol)

## Summary
`touchMarket` enforces collateral ordering, LLTV tier, and `maxLif` validity but never checks that the collateral token address has deployed bytecode. Any unprivileged caller can permanently register a market with a codeless collateral token. Because markets are provably undeletable (Certora rule `marketCannotBeDeleted`), every such market persists on-chain indefinitely, and every `supplyCollateral` call for it unconditionally reverts with `SafeTransferLib.NoCode()`.

## Finding Description
`touchMarket` at `src/Midnight.sol:762-773` enforces exactly three properties per `collateralParams` entry — ordering, LLTV allowance, and `maxLif` validity — with no `require(collateralToken.code.length > 0)` guard. [1](#0-0) 

On success, market state is permanently written at lines 775-788 (`tickSpacing`, settlement fees, continuous fee, `storeInCode`) and `MarketCreated` is emitted. [2](#0-1) 

When `supplyCollateral` is subsequently called, it calls `touchMarket` (line 528, no-op for existing market), mutates position storage at lines 532-540, then calls `SafeTransferLib.safeTransferFrom(collateralToken, ...)` at line 545. [3](#0-2) 

`SafeTransferLib.safeTransferFrom` opens with `require(token.code.length > 0, NoCode())` at line 25, which always reverts for a codeless token. [4](#0-3) 

The `supplyCollateral` transaction reverts entirely (rolling back position mutations), but the market created by the prior `touchMarket` call is unaffected. The Certora rule `marketCannotBeDeleted` at `certora/specs/CreatedMarkets.spec:82-86` formally proves no function can ever remove a created market. [5](#0-4) 

**Exploit flow:**
1. Attacker constructs a `Market` with `collateralParams[0].token = address(type(uint160).max)`, a valid LLTV (e.g. `0.77e18`), the corresponding valid `maxLif`, and any valid maturity.
2. Attacker calls `touchMarket(market)` — succeeds, `tickSpacing > 0`, market permanently registered.
3. Any subsequent `supplyCollateral(market, 0, assets, onBehalf)` always reverts with `NoCode()`.
4. No borrower can ever post collateral; no borrow is ever possible in this market.

## Impact Explanation
A permanently registered market with a codeless collateral token is an unrecoverable corruption of protocol state. The market exists on-chain, consumes storage, emits a `MarketCreated` event, and will be indexed by integrators and front-ends as a valid market — but no borrower can ever interact with it. Because markets are immutable after creation and provably undeletable, the state cannot be repaired. No user funds are directly frozen (lenders cannot be trapped because debt creation requires passing the health check, which requires collateral), but the protocol accumulates permanently dead market entries that cannot be cleaned up. This matches the `RESEARCHER.md` impact class of "Permanent lock, freeze, or unrecoverable corruption of user/project state."

## Likelihood Explanation
`touchMarket` is fully permissionless — any EOA or contract can call it with arbitrary parameters. The only preconditions are a valid LLTV tier and a matching `maxLif`, both of which are publicly enumerable constants. The attack costs only gas, is trivially repeatable across any number of distinct market parameter combinations (different loan tokens, maturities, or collateral index orderings), and requires no privileged access, no capital, and no victim interaction.

## Recommendation
Add a bytecode existence check inside the `touchMarket` loop, immediately after the collateral token address is read:

```solidity
require(collateralToken.code.length > 0, NoCode());
```

This should be placed after line 763 (`address collateralToken = market.collateralParams[i].token;`) and before the ordering check, consistent with the guard already present in `SafeTransferLib`.

## Proof of Concept
1. Deploy a test environment with the `Midnight` contract.
2. Construct a `Market` struct with one `CollateralParams` entry where `token = address(type(uint160).max)`, `lltv` is any value passing `isLltvAllowed`, and `maxLif` is the corresponding value from `maxLif(lltv, LIQUIDATION_CURSOR_LOW)`.
3. Call `touchMarket(market)` — assert it succeeds and `marketState[id].tickSpacing > 0`.
4. Call `supplyCollateral(market, 0, 1e18, address(this))` — assert it reverts with `NoCode()`.
5. Call `touchMarket(market)` again — assert it still returns the same `id` (market was not deleted).
6. Confirm no path exists to set `marketState[id].tickSpacing = 0`.
