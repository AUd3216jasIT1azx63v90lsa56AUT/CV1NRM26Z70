### Title
`ConfidencePool.initialize()` Callable by Anyone, Bypassing Factory Token Allowlist and Agreement-Owner Validation — (File: src/ConfidencePool.sol)

---

### Summary

`ConfidencePool.initialize()` is `external` with only the `initializer` guard, which prevents re-initialization but places no restriction on *who* may call it. `ConfidencePoolFactory.createPool()` is the intended and only documented entry point; it enforces two validations that `initialize()` itself does not: the stake-token allowlist check and the agreement-owner check. Because `poolImplementation` is a public state variable, any actor can read it, deploy their own clone of the implementation, and call `initialize()` directly — bypassing both factory-level guards.

---

### Finding Description

`ConfidencePoolFactory.createPool()` enforces two guards before delegating to `initialize()`:

```solidity
// ConfidencePoolFactory.sol line 77
if (!allowedStakeToken[stakeToken]) revert StakeTokenNotAllowed();
// line 82
if (IAgreement(agreement).owner() != msg.sender) revert UnauthorizedCreator();
```

`ConfidencePool.initialize()` carries neither check:

```solidity
// ConfidencePool.sol line 179
function initialize(
    address agreement_,
    address stakeToken_,
    ...
) external initializer {
```

The `initializer` modifier (OpenZeppelin `Initializable`) only prevents a second call on the same proxy instance; it does not restrict the caller. Because `poolImplementation` is `public` on the factory, any actor can:

1. Read `factory.poolImplementation()`.
2. Deploy a fresh clone via `Clones.clone(poolImplementation)` or `Clones.cloneDeterministic(poolImplementation, anySalt)` from their own address (a different CREATE2 deployer, so a different address than the factory would produce).
3. Call `initialize()` on that clone with an arbitrary `stakeToken_` — including a fee-on-transfer or rebasing token that the factory allowlist explicitly excludes — and with themselves as `owner_` and `recoveryAddress_`, regardless of whether they own the referenced agreement.

The factory's `_poolsByAgreement` registry will not contain this clone, but nothing in the protocol prevents stakers from interacting with any address that exposes the `IConfidencePool` interface.

---

### Impact Explanation

A rogue pool initialized with a fee-on-transfer token silently under-credits every `stake()` and `contributeBonus()` call (the pool records `amount` but receives `amount − fee`), and under-pays every `claimSurvived()`, `claimExpired()`, and `withdraw()` (the transfer out also incurs the fee). Stakers who deposit into such a pool cannot recover their full principal. The `README.md` explicitly states fee-on-transfer tokens are unsupported precisely because of this accounting mismatch; the factory allowlist is the sole enforcement mechanism, and it is bypassable here.

Additionally, bypassing the `IAgreement(agreement).owner() == msg.sender` check allows any actor to create a pool attributed to any live agreement, with an arbitrary `recoveryAddress` (e.g., the attacker's own address). Under a bad-faith CORRUPTED resolution, the entire pool — stakers' principal plus bonus — sweeps to that address.

---

### Likelihood Explanation

Exploiting this requires:
- Reading the public `poolImplementation` address (trivial).
- Deploying a clone and calling `initialize()` (one transaction, no special privilege).
- Deceiving stakers into depositing into the rogue pool rather than a factory-registered one.

The social-engineering step moderates likelihood, but the on-chain preconditions are zero-cost and permissionless. A sophisticated attacker can make the rogue pool indistinguishable from a legitimate one at the contract level (same bytecode, same interface, a real agreement address, a real-looking token address).

---

### Recommendation

Add an `onlyFactory` access control check to `initialize()`. The simplest approach is to store the factory address at construction time (before `_disableInitializers()`) and enforce it in `initialize()`:

```solidity
address public immutable factory;

constructor(address factory_) Ownable(msg.sender) {
    factory = factory_;
    _disableInitializers();
}

function initialize(...) external initializer {
    if (msg.sender != factory) revert NotFactory();
    ...
}
```

Alternatively, move the `allowedStakeToken` and `agreement.owner()` checks into `initialize()` itself (accepting a reference to the factory's allowlist), so the validation is enforced regardless of the call path.

---

### Proof of Concept

```solidity
// Attacker reads the implementation address from the factory
address impl = factory.poolImplementation();

// Deploys their own clone (different deployer → different address than factory would produce)
