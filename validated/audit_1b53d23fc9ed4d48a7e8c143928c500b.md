Audit Report

## Title
Unconditional zero-value `safeTransfer` on collateral token blocks bad-debt realization when token reverts on zero transfers - (File: src/Midnight.sol)

## Summary
`Midnight.liquidate` explicitly documents via NatSpec that passing `seizedAssets=0, repaidUnits=0` realizes bad debt "with 0 token transferred." However, `SafeTransferLib.safeTransfer(collateralToken, receiver, seizedAssets)` is called unconditionally at line 696 regardless of whether `seizedAssets` is zero. For any market whose collateral token reverts on zero-value `transfer` calls — a documented behavior of several deployed ERC20s — every bad-debt realization attempt reverts, permanently preventing `lossFactor` and `totalUnits` from being updated.

## Finding Description

The code path is confirmed exactly as claimed:

**NatSpec promise:** Lines 577–579 explicitly state that passing both `seizedAssets=0` and `repaidUnits=0` is a valid call that realizes bad debt with no token movement. [1](#0-0) 

**Both-zero is permitted:** Line 595 uses `atMostOneNonZero(repaidUnits, seizedAssets)` — this passes when both are zero, it is not a guard against the zero-zero path. [2](#0-1) 

**Bad-debt accounting executes correctly:** When `badDebt > 0`, lines 626–641 update `_position.debt`, `_marketState.lossFactor`, `_marketState.totalUnits`, and `_marketState.continuousFeeCredit`. [3](#0-2) 

**Accounting block correctly skipped:** Line 643's guard `if (repaidUnits > 0 || seizedAssets > 0)` is false, so `seizedAssets` remains `0` after this block. [4](#0-3) 

**Unconditional transfer at line 696:** `SafeTransferLib.safeTransfer` is called with `seizedAssets = 0` and no guard exists. [5](#0-4) 

**`SafeTransferLib` propagates the revert:** If the token's `transfer(addr, 0)` call returns `success == false`, the assembly block re-reverts with the token's revert data, rolling back all state mutations from the bad-debt block. [6](#0-5) 

**Secondary instance:** Line 717 has the same pattern for `safeTransferFrom(loanToken, payer, address(this), 0)`, but line 696 is reached first. [7](#0-6) 

## Impact Explanation
For any market whose collateral token reverts on `transfer(addr, 0)`, bad-debt realization is permanently and completely blocked. The `lossFactor` and `totalUnits` are never updated, so lenders' credit is never written down to reflect the loss. The borrower's position remains technically liquidatable but every realization attempt reverts. This is a permanent freeze of a core protocol invariant: unhealthy positions must remain liquidatable. Because the collateral token is immutable after market creation, the block is irreversible for the affected market.

## Likelihood Explanation
Two preconditions are required: (1) a market is created with a collateral token that reverts on `transfer(addr, 0)` — this is a documented, deployed behavior (e.g., LEND, some cTokens); (2) a borrower accumulates bad debt in that market. Market creation is permissionless; bad debt accumulates naturally from price movements. Neither condition requires privileged access. Once the collateral token is set, it cannot be changed, making the block permanent and repeatable on every subsequent liquidation attempt.

## Recommendation
Add a zero-value guard before line 696:

```solidity
if (seizedAssets > 0) {
    SafeTransferLib.safeTransfer(
        market.collateralParams[collateralIndex].token, receiver, seizedAssets
    );
}
```

Apply the same guard to line 717:

```solidity
if (repaidUnits > 0) {
    SafeTransferLib.safeTransferFrom(market.loanToken, payer, address(this), repaidUnits);
}
```

This directly implements the NatSpec promise at lines 577–579 and is consistent with the existing guard at line 643.

## Proof of Concept

1. Deploy a mock ERC20 collateral token that reverts on `transfer(addr, 0)`.
2. Create a permissionless market using that token as collateral.
3. Have a borrower supply collateral and borrow such that the position becomes undercollateralized with `badDebt > 0` (e.g., drop the oracle price to zero after borrowing).
4. Call `liquidate(market, collateralIndex, 0, 0, borrower, false, receiver, address(0), "")`.
5. Observe: the call reverts at line 696 despite `seizedAssets = 0`, and `_marketState.lossFactor` / `_marketState.totalUnits` are never updated.
6. Confirm: every subsequent call with `(0, 0)` also reverts — bad debt is permanently unrealizable.
