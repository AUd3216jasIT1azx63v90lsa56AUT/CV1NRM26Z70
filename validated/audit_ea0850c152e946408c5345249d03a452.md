Audit Report

## Title
Co-Authorized Agent Can Deauthorize Peer Agents via Signature Delegation - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` accepts a valid signature from any agent already authorized by the victim in Midnight, not exclusively from the victim themselves. This allows any co-authorized attacker to sign and submit an `Authorization{authorizer=victim, authorized=takeAgent, isAuthorized=false}` message, revoking a peer agent's authorization without the victim's consent. The attack is repeatable and breaks the invariant that only the authorizer can revoke their own delegations.

## Finding Description
**Root cause — `EcrecoverAuthorizer.setIsAuthorized` lines 33–36:**

```solidity
require(
    signer == authorization.authorizer
        || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
``` [1](#0-0) 

The signer check accepts any address for which `isAuthorized[authorization.authorizer][signer] == true`. This means any agent the victim has ever authorized can sign an `Authorization` struct on the victim's behalf, including one that sets `isAuthorized=false` for a different agent.

**Supporting core path — `Midnight.setIsAuthorized` line 732:**

```solidity
require(onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender], Unauthorized());
``` [2](#0-1) 

The core contract enforces the same permissive check: any authorized agent can call `setIsAuthorized` on behalf of the authorizer, so the deauthorization executes successfully once the `EcrecoverAuthorizer` check passes.

**Exploit flow:**
1. Victim calls `midnight.setIsAuthorized(ecrecoverAuthorizer, true, victim)` (standard sig-based setup).
2. Victim authorizes `takeAgent`: `isAuthorized[victim][takeAgent] = true`.
3. Victim authorizes `attacker`: `isAuthorized[victim][attacker] = true`.
4. Attacker reads `ecrecoverAuthorizer.nonce(victim)` → `N` (public storage). [3](#0-2) 

5. Attacker signs `Authorization{authorizer=victim, authorized=takeAgent, isAuthorized=false, nonce=N, deadline=future}` with attacker's own key.
6. Attacker calls `ecrecoverAuthorizer.setIsAuthorized(auth, sig)`.
7. `ecrecover` returns attacker's address; `isAuthorized(victim, attacker) == true` → authorization check passes. [4](#0-3) 

8. `Midnight.setIsAuthorized(takeAgent, false, victim)` executes → `isAuthorized[victim][takeAgent] = false`.
9. `takeAgent`'s subsequent `take` call reverts `TakerUnauthorized`.

No existing guard prevents this. The nonce is sequential and public; the deadline is attacker-chosen; the signer check explicitly permits delegates.

## Impact Explanation
A victim relying on an authorized take-agent for time-sensitive `take` execution (e.g., an offer near expiry) will have that agent silently deauthorized by any other agent the victim has trusted. The `take` call reverts, the offer expires, and the victim loses the trade. The attack is repeatable: every time the victim re-authorizes `takeAgent`, the attacker can front-run with a new deauthorization consuming the next nonce. This constitutes a concrete, repeatable griefing/DoS of authorized agents with direct financial loss potential.

## Likelihood Explanation
All preconditions are realistic in any multi-agent wallet or DeFi integration:
1. Victim has authorized `EcrecoverAuthorizer` (standard for sig-based flows).
2. Victim has authorized both a `takeAgent` and at least one other agent.
3. Attacker knows `nonce[victim]` (public storage, zero cost to read).

The attack requires only one transaction per deauthorization, costs only gas, and is front-runnable on every re-authorization attempt by the victim.

## Recommendation
Restrict the signer check in `EcrecoverAuthorizer.setIsAuthorized` to only accept the authorizer themselves:

```solidity
require(signer == authorization.authorizer, Unauthorized());
``` [1](#0-0) 

The delegate-signer path provides no legitimate use case that cannot be served by the authorizer signing directly. If delegation is intentionally desired, it must be scoped to prevent delegates from modifying other delegates' permissions (e.g., only allow `authorized == signer` when signer is a delegate).

## Proof of Concept
```solidity
// 1. victim authorizes ecrecoverAuthorizer, takeAgent, attacker
midnight.setIsAuthorized(address(ecrecoverAuthorizer), true, victim);  // via vm.prank(victim)
midnight.setIsAuthorized(takeAgent, true, victim);
midnight.setIsAuthorized(attacker, true, victim);

// 2. attacker reads nonce
uint256 n = ecrecoverAuthorizer.nonce(victim);

// 3. attacker signs deauthorization of takeAgent
Authorization memory auth = Authorization({
    authorizer: victim,
    authorized: takeAgent,
    isAuthorized: false,
    nonce: n,
    deadline: block.timestamp + 1 hours
});
bytes32 digest = ...; // EIP-712 digest over auth
(uint8 v, bytes32 r, bytes32 s) = vm.sign(attackerKey, digest);

// 4. attacker submits
vm.prank(attacker);
ecrecoverAuthorizer.setIsAuthorized(auth, Signature(v, r, s));

// 5. assert takeAgent is deauthorized
assertFalse(midnight.isAuthorized(victim, takeAgent));

// 6. takeAgent's take now reverts TakerUnauthorized
vm.prank(takeAgent);
vm.expectRevert(TakerUnauthorized.selector);
midnight.take(offer, ..., taker=victim, ...);
``` [5](#0-4)

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
```
