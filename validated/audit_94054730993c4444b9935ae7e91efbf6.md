### Title
Permissionless Market Creation With Unvalidated Spot-Price Oracle Enables Flash-Loan Collateral Inflation and Bad Debt — (File: src/Midnight.sol)

### Summary

`touchMarket` creates markets permissionlessly and accepts any address as a collateral oracle without validation. An attacker can create a market whose oracle reads a Uniswap V3 spot price from a low-TVL pool they control, attract lenders, then within a single transaction flash-loan-manipulate the pool price upward, borrow the maximum amount against the inflated collateral value, and restore the price. The position is left undercollateralized at the true price, and the resulting bad debt is socialized across all lenders in the market.

### Finding Description

**Root cause — no oracle validation in `touchMarket`**

`touchMarket` enforces several market-parameter constraints: maturity range, non-empty sorted collateral list, allowed LLTV tiers, and valid `maxLif` values. It does not validate the oracle address at all. [1](#0-0) 

The `CollateralParams` struct stores an arbitrary `oracle` address per collateral: [2](#0-1) 

The health check in `isHealthy` (called at the end of every `take` that increases the seller's debt) blindly calls `IOracle(collateralParam.oracle).price()` on whatever address was registered at market creation: [3](#0-2) 

The same raw spot-price call is made inside `liquidate`: [4](#0-3) 

**Exploit flow**

1. Attacker deploys a thin oracle wrapper that reads the instantaneous `sqrtPriceX96` from a Uniswap V3 WETH/USDC pool they seeded with minimal liquidity (e.g., $500).
2. Attacker calls `touchMarket` with `collateralParams[0].oracle = <manipulable oracle>`, `lltv = 0.98e18`, and `loanToken = USDC`. Market is created; configuration is now immutable.
3. Attacker posts a sell offer (borrower side) and waits for legitimate lenders to fill buy offers, or seeds the market themselves with a second address to establish withdrawable USDC.
4. In a single atomic transaction:
   - Flash-loan a large amount of USDC from Midnight's own `flashLoan` or an external source.
   - Swap USDC → WETH in the low-TVL Uniswap V3 pool, driving the spot price of WETH up by 10–100×.
   - Call `take` as the seller (borrower): `isHealthy` reads the inflated oracle price, `maxDebt` is computed as `collateral × inflatedPrice × lltv / WAD`, and the health check passes.
   - Swap WETH → USDC in the same pool to restore the price.
   - Repay the flash loan.
5. The position now has `debt >> collateral × truePrice × lltv`. The attacker walks away with borrowed USDC; lenders are left with bad debt socialized via `lossFactor`.

Midnight's own `flashLoan` function can be used as the flash-loan source, making the entire attack self-contained within the protocol: [5](#0-4) 

### Impact Explanation

Lenders in the manipulated market lose funds proportional to the bad debt realized. The `lossFactor` mechanism socializes the loss across all credit holders in the market: [6](#0-5) 

With a high-LLTV market (e.g., `lltv = 0.98e18`) and a 10× price manipulation, the attacker can borrow ~9.8× the true collateral value. The entire lender-side liquidity of the market can be drained in a single transaction.

### Likelihood Explanation

- No privileged access is required. `touchMarket` is fully permissionless.
- Uniswap V3 pools with low TVL exist for virtually any token pair; the attacker can also seed a new pool with minimal capital.
- The attack is atomic (single transaction), so there is no arb-window exposure for the attacker.
- Lenders have no on-chain mechanism to verify the oracle type before participating; the market struct only exposes an opaque `address oracle`.
- Midnight's own `flashLoan` removes the need for any external flash-loan provider.

### Recommendation

1. **Require TWAP-based oracles**: Add an interface method (e.g., `isManipulationResistant()`) that oracles must implement, or maintain a protocol-level whitelist of approved oracle addresses.
2. **Minimum observation window**: If Uniswap V3 TWAP oracles are permitted, enforce a minimum `secondsAgo` parameter (e.g., ≥ 30 minutes) so single-block manipulation cannot affect the price used in `isHealthy`.
3. **Document the trust model explicitly**: If the design intent is that lenders bear oracle-selection risk, document this prominently and consider an on-chain oracle-type registry so lenders can verify before depositing.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;

            uint128 _collateralBitmap = _position.collateralBitmap;
