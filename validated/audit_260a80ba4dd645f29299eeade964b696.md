Audit Report

## Title
Fee-on-Transfer Collateral Inflates `position.collateral` Accounting, Enabling Undercollateralized Borrowing - (File: src/Midnight.sol)

## Summary
`supplyCollateral` writes the caller-supplied `assets` value into `_position.collateral[collateralIndex]` at line 533 before executing the transfer at line 545, with no balance-delta verification. For fee-on-transfer collateral tokens, the protocol records more collateral than it actually receives. `isHealthy` reads the inflated storage value, allowing a borrower's position to pass the health check while being undercollateralized from inception.

## Finding Description
In `supplyCollateral` (`src/Midnight.sol:533`), the accounting write unconditionally trusts the `assets` argument:

```solidity
// line 533 — records full `assets` before transfer
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + assets);
// ...
// line 545 — only delivers assets*(1-fee) for fee-on-transfer tokens
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
``` [1](#0-0) 

`SafeTransferLib.safeTransferFrom` (`src/libraries/SafeTransferLib.sol:24-34`) only checks the boolean return value; it performs no balance-before/after delta check. [2](#0-1) 

`isHealthy` (`src/Midnight.sol:954-955`) reads `_position.collateral[i]` directly from storage:

```solidity
maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
    .mulDivDown(collateralParam.lltv, WAD);
``` [3](#0-2) 

`touchMarket` (`src/Midnight.sol:755-791`) validates only LLTV tiers, sorted collateral addresses, and `maxLif` — no token-type restriction prevents a fee-on-transfer ERC20 from being used as collateral. [4](#0-3) 

**Exploit path via `take` → `onSell` → `supplyCollateral`:**

1. Attacker deploys a fee-on-transfer ERC20 (e.g., 1% fee) and calls `touchMarket` to create a market with it as collateral. No admin action required.
2. A lender creates a buy offer ratified by `SetterRatifier`. The ratifier only verifies Merkle proof and root membership — no collateral-token checks.
3. Attacker deploys a callback contract implementing `ISellCallback.onSell` that calls `supplyCollateral(market, index, amount, attacker)`. Attacker calls `setIsAuthorized(callbackContract, true, attacker)` so the authorization check at `src/Midnight.sol:527` passes.
4. Attacker calls `take(lenderOffer, ...)`. Inside `take`:
   - Debt state is updated: `sellerPos.debt += sellerDebtIncrease` (`src/Midnight.sol:414`).
   - Loan tokens are transferred to the attacker (`src/Midnight.sol:455-456`).
   - `onSell` is invoked on the callback contract (`src/Midnight.sol:458-473`).
   - Inside `onSell`, `supplyCollateral` is called: `_position.collateral[index] += amount` (records `amount`), but `safeTransferFrom` only delivers `amount * 0.99` to Midnight.
   - After the callback, `isHealthy` is called (`src/Midnight.sol:476`) using the inflated `_position.collateral[index] = amount`, not the actual `amount * 0.99` held.
5. Health check passes. The attacker now holds `units` of loan tokens backed by only `amount * 0.99` actual collateral, while the protocol believes it holds `amount`. [5](#0-4) 

No existing check stops this: `supplyCollateral` has no balance-before/after guard; `SafeTransferLib` only checks the return boolean; `isHealthy` reads storage directly; `touchMarket` imposes no token-type constraints.

## Impact Explanation
The protocol's core solvency invariant — contract token balances must cover all withdrawable assets, collateral claims, credit redemptions, and accrued fees — is violated immediately upon the first `supplyCollateral` call with a fee-on-transfer token. The borrower's position is undercollateralized from inception. When the position is eventually liquidated, the seized collateral is less than the recorded amount, leaving unpayable bad debt socialized among lenders. This constitutes direct theft of lender funds via protocol accounting manipulation.

## Likelihood Explanation
Any permissionless market creator can deploy a market with a fee-on-transfer collateral token — no admin action is required. The borrower only needs to authorize their own callback contract, which is standard self-authorization. `SECURITY.md` contains no exclusion for fee-on-transfer tokens. The exploit is repeatable on every `supplyCollateral` call with a fee-bearing token and scales linearly with the fee rate and borrowed amount.

## Recommendation
Record collateral based on the actual received amount by snapshotting the balance before and after the transfer:

```solidity
uint256 balanceBefore = IERC20(collateralToken).balanceOf(address(this));
SafeTransferLib.safeTransferFrom(collateralToken, msg.sender, address(this), assets);
uint256 received = IERC20(collateralToken).balanceOf(address(this)) - balanceBefore;
_position.collateral[collateralIndex] = UtilsLib.toUint128(oldCollateral + received);
```

Alternatively, restrict collateral tokens to a protocol-controlled allowlist that excludes fee-on-transfer tokens.

## Proof of Concept
1. Deploy a fee-on-transfer ERC20 token with a 1% transfer fee.
2. Call `touchMarket` with the fee token as collateral — succeeds with no restriction.
3. Call `supplyCollateral(market, 0, 1000e18, attacker)`:
   - Storage records `_position.collateral[0] = 1000e18`.
   - Contract receives only `990e18`.
4. Call `take` on a lender offer to borrow loan tokens up to the inflated collateral value.
5. After `onSell` callback completes, `isHealthy` reads `1000e18` from storage and passes.
6. Assert: `IERC20(collateralToken).balanceOf(address(midnight)) == 990e18` while `position[id][attacker].collateral[0] == 1000e18` — a 10e18 discrepancy confirming undercollateralization.

    function isHealthy(Market memory market, bytes32 id, address borrower) public view returns (bool) {
        Position storage _position = position[id][borrower];
        uint256 debt = _position.debt;
        uint256 maxDebt;
        if (debt > 0) {
            uint128 _collateralBitmap = _position.collateralBitmap;
            while (_collateralBitmap != 0) {
                uint256 i = UtilsLib.msb(_collateralBitmap);
                CollateralParams memory collateralParam = market.collateralParams[i];
                uint256 price = IOracle(collateralParam.oracle).price();
                maxDebt += _position.collateral[i].mulDivDown(price, ORACLE_PRICE_SCALE)
                    .mulDivDown(collateralParam.lltv, WAD);
                _collateralBitmap = _collateralBitmap.clearBit(i);
            }
        }
        return maxDebt >= debt;

        require(token.code.length > 0, NoCode());
