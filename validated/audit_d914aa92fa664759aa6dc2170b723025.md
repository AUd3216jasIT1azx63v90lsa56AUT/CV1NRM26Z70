Audit Report

## Title
Authorized Agent Can Grief Victim's Nonce via Self-Deauthorization - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` allows any address that `Midnight.isAuthorized(authorizer, signer)` returns `true` for to sign and submit an `Authorization` struct on behalf of the authorizer. An attacker who is already an authorized agent of the victim can craft `Authorization(authorizer=victim, authorized=attacker, isAuthorized=false, nonce=N)`, sign it with their own key, and submit it. The call passes all checks, increments `nonce[victim]` from N to N+1, and removes the attacker's own authorization — permanently invalidating any pending off-chain signed authorization the victim holds at nonce N.

## Finding Description

**Root cause:** `EcrecoverAuthorizer.setIsAuthorized` permits any currently-authorized agent to sign and submit an `Authorization` struct on behalf of the authorizer, including one that targets themselves as `authorized` with `isAuthorized=false`. No check prevents a delegated signer from self-deauthorizing via this path.

**Exact code path:**

`EcrecoverAuthorizer.sol` line 26 — nonce is incremented unconditionally as part of the `require` evaluation, before any signature verification: [1](#0-0) 

`EcrecoverAuthorizer.sol` lines 31–36 — the signer is recovered and checked. The attacker signs with their own key (`signer = attacker`). The first branch `signer == authorization.authorizer` is `attacker == victim` → false. The second branch `isAuthorized(victim, attacker)` → true by precondition. The check passes: [2](#0-1) 

`EcrecoverAuthorizer.sol` lines 46–47 — the downstream call executes `midnight.setIsAuthorized(attacker, false, victim)`: [3](#0-2) 

`Midnight.sol` line 732 — `Midnight.setIsAuthorized` checks `onBehalf == msg.sender || isAuthorized[onBehalf][msg.sender]`. Here `msg.sender` is `EcrecoverAuthorizer` and `onBehalf` is `victim`. For the EcrecoverAuthorizer flow to function at all, `isAuthorized[victim][EcrecoverAuthorizer]` must be true, so this check passes: [4](#0-3) 

**Net state change after the attack:**
- `nonce[victim]` = N+1 (was N)
- `isAuthorized[victim][attacker]` = false (was true)
- Any off-chain signed `Authorization(authorizer=victim, ..., nonce=N)` is permanently invalid (`InvalidNonce()` on submission)

**Why existing checks fail:** The `Unauthorized()` guard at lines 33–36 is the only gate, and it explicitly allows authorized agents through by design. There is no restriction preventing a delegated signer from naming themselves as `authorized` with `isAuthorized=false`. The nonce increment at line 26 occurs before signature verification, so even if the signature check were to fail after, the nonce would already be consumed — though in this attack it does not fail. [5](#0-4) 

## Impact Explanation
Any pending signed authorization the victim has distributed off-chain (e.g., to a relayer or bundler) with `nonce=N` is rendered permanently invalid. The victim must re-sign with `nonce=N+1` and redistribute. If the attacker is re-authorized (e.g., a trusted relayer the victim re-grants), the pattern can be repeated indefinitely, causing sustained DoS on any gasless authorization flow that relies on `EcrecoverAuthorizer`. The `AUTHORIZATIONS` section of `Midnight.sol` explicitly notes that authorized accounts can authorize other accounts on behalf of the user, confirming this path is reachable by design. [6](#0-5) 

## Likelihood Explanation
**Preconditions:** Attacker must hold `isAuthorized[victim][attacker] == true` and `isAuthorized[victim][EcrecoverAuthorizer] == true`. Both are normal operational states — users authorize relayers, bundlers, and smart contracts routinely, and the EcrecoverAuthorizer must be authorized for the gasless flow to work at all. **Feasibility:** One transaction, no capital required, no oracle dependency. **Repeatability:** One-shot per authorization grant (attacker loses their own authorization), but if the victim re-authorizes the attacker (e.g., a trusted relayer), the attack can be repeated. The attacker can also be any of the many addresses a user might authorize (ratifiers, bundlers, callback contracts).

## Recommendation
Add a check in `EcrecoverAuthorizer.setIsAuthorized` that prevents a delegated signer (i.e., `signer != authorization.authorizer`) from submitting an authorization where `authorization.authorized == signer` and `authorization.isAuthorized == false`. Alternatively, restrict delegated signers so they can only sign authorizations where `authorization.authorizer == signer` (i.e., only the authorizer themselves can use the EcrecoverAuthorizer path for their own nonce), removing the ability for agents to consume the authorizer's nonce entirely. A simpler mitigation is to move the nonce increment to after the signature and authorization checks, so a failed authorization attempt does not consume the nonce — though this alone does not prevent the self-deauthorization grief. [5](#0-4) 

## Proof of Concept
1. Deploy `Midnight` and `EcrecoverAuthorizer`.
2. Victim calls `midnight.setIsAuthorized(EcrecoverAuthorizer, true, victim)` — required for gasless flow.
3. Victim calls `midnight.setIsAuthorized(attacker, true, victim)` — grants attacker authorization.
4. Victim signs off-chain: `Authorization(authorizer=victim, authorized=someRelayer, isAuthorized=true, nonce=0, deadline=...)` and distributes to a relayer.
5. Attacker (before the relayer submits) calls `EcrecoverAuthorizer.setIsAuthorized` with `Authorization(authorizer=victim, authorized=attacker, isAuthorized=false, nonce=0, deadline=...)` signed by attacker's key.
6. Transaction succeeds: `nonce[victim]` = 1, `isAuthorized[victim][attacker]` = false.
7. Relayer attempts to submit victim's signed authorization at nonce=0 → reverts with `InvalidNonce()`.
8. Victim must re-sign at nonce=1 and redistribute. If attacker is re-authorized, repeat from step 5.

    function setIsAuthorized(Authorization memory authorization, Signature calldata signature) external {
        require(block.timestamp <= authorization.deadline, Expired());
        require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());

        bytes32 hashStruct = keccak256(abi.encode(AUTHORIZATION_TYPEHASH, authorization));
        bytes32 domainSeparator = keccak256(abi.encode(EIP712_DOMAIN_TYPEHASH, block.chainid, address(this)));
        bytes32 digest = keccak256(bytes.concat("\x19\x01", domainSeparator, hashStruct));
        address signer = ecrecover(digest, signature.v, signature.r, signature.s);
        require(signer != address(0), InvalidSignature());
        require(
            signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
            Unauthorized()
        );

        emit SetIsAuthorized(
            msg.sender,
            authorization.authorizer,
            authorization.authorized,
            authorization.isAuthorized,
            authorization.nonce
        );

        IMidnight(MIDNIGHT)
            .setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
    }
