Audit Report

## Title
Authorized delegate can front-run and consume victim's EcrecoverAuthorizer nonce with arbitrary payload - (File: src/periphery/EcrecoverAuthorizer.sol)

## Summary
`EcrecoverAuthorizer.setIsAuthorized` increments the authorizer's nonce before verifying the signer, and accepts any address that `IMidnight.isAuthorized(authorizer, signer)` returns `true` for as a valid signer. A Midnight-authorized delegate can front-run a victim's pending `setIsAuthorized` transaction, consume nonce N with an attacker-chosen `Authorization` payload, permanently invalidate the victim's signature, and silently substitute an attacker-controlled `authorized` address in the victim's Midnight `isAuthorized` mapping.

## Finding Description
In `src/periphery/EcrecoverAuthorizer.sol`, `setIsAuthorized` (lines 24–48):

**Line 26** increments the nonce unconditionally before the signer is verified:
```solidity
require(authorization.nonce == nonce[authorization.authorizer]++, InvalidNonce());
```

**Lines 33–36** accept the signer if they are either the authorizer themselves or any address the authorizer has previously authorized in Midnight:
```solidity
require(
    signer == authorization.authorizer || IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer),
    Unauthorized()
);
```

**Line 47** then executes the state change using fully attacker-controlled fields from the `Authorization` struct:
```solidity
IMidnight(MIDNIGHT).setIsAuthorized(authorization.authorized, authorization.isAuthorized, authorization.authorizer);
```

The `Authorization` struct (`src/periphery/interfaces/IEcrecoverAuthorizer.sol`, lines 11–17) contains `authorized` and `isAuthorized` as caller-supplied inputs with no binding to the authorizer's intent.

The EIP-712 digest is computed over the full `Authorization` struct at lines 28–30. When the attacker (Bob) constructs a malicious `Authorization{authorizer: alice, authorized: <attacker_address>, isAuthorized: <any>, nonce: N, deadline: <valid>}` and signs it with his own key, the digest is over Bob's chosen payload. The signer recovered is Bob. The signer check passes because `IMidnight.isAuthorized(alice, bob) == true`. The nonce was already consumed at line 26. Alice's original transaction reverts with `InvalidNonce`.

**Exploit flow:**
1. Alice has previously called `midnight.setIsAuthorized(bob, true, alice)` — a routine operator authorization (as shown in `test/AuthorizationTest.sol` lines 100–103, 237–239).
2. Alice signs `Authorization{authorizer: alice, authorized: charlie, isAuthorized: true, nonce: N, deadline: T}` and broadcasts it to the mempool.
3. Bob observes the pending transaction, constructs `Authorization{authorizer: alice, authorized: <attacker_address>, isAuthorized: <any>, nonce: N, deadline: <valid>}`, signs it with his own key, and front-runs Alice's transaction.
4. The nonce check passes (N matches current nonce) and the nonce is incremented to N+1.
5. The signer check passes because `IMidnight.isAuthorized(alice, bob) == true`.
6. `IMidnight.setIsAuthorized(attacker_address, attacker_bool, alice)` executes, modifying Alice's authorization mapping with attacker-chosen values.
7. Alice's original transaction reverts with `InvalidNonce`.

The existing nonce check does not prevent this because the nonce is consumed before the signer is verified. The signer check does not prevent this because it allows any delegate to sign any payload — it does not restrict which `authorized` address or `isAuthorized` value the delegate may embed.

## Impact Explanation
**Unauthorized state change / privilege escalation**: The attacker substitutes an arbitrary `authorized` address in the victim's Midnight `isAuthorized` mapping. This allows the attacker to grant or revoke authorization for any address in the victim's name, bypassing the victim's intent and any gate logic that depends on `isAuthorized`. This is a concrete, in-scope unauthorized state change and privilege escalation.

**DoS on EcrecoverAuthorizer flow**: The victim's signed-and-broadcast transaction is permanently invalidated. The attacker can repeat this for every subsequent nonce (N+1, N+2, …), permanently blocking the victim from using the EcrecoverAuthorizer to manage their authorizations via signature.

## Likelihood Explanation
Preconditions: (1) the attacker has previously been authorized by the victim in Midnight for any purpose — a routine pattern for operators and bundlers, as demonstrated in `test/AuthorizationTest.sol` lines 100–103 and 237–239; (2) the victim has a pending `EcrecoverAuthorizer.setIsAuthorized` transaction visible in the mempool. Both conditions are normal in protocol usage. The attack requires no privileged protocol access, no admin keys, and no leaked credentials — only a standard user-level authorization relationship. It is repeatable indefinitely at zero additional cost per nonce.

## Recommendation
Remove the delegate signer path from `EcrecoverAuthorizer`. The function should only accept signatures from the authorizer themselves:

```solidity
require(signer == authorization.authorizer, Unauthorized());
```

If delegate submission is desired (i.e., a delegate relays the authorizer's own signature), the current check already achieves that — the authorizer signs the payload and anyone submits it. The second branch `IMidnight(MIDNIGHT).isAuthorized(authorization.authorizer, signer)` is what enables the attack: it allows the delegate to sign an arbitrary payload rather than merely relay the authorizer's signature. Removing this branch eliminates the vulnerability while preserving the gasless authorization use case.

Additionally, move the nonce increment after signer verification to follow the checks-effects-interactions pattern and prevent nonce consumption on failed authorization:

```solidity
require(authorization.nonce == nonce[authorization.authorizer], InvalidNonce());
// ... verify signer ...
nonce[authorization.authorizer]++;
```

## Proof of Concept
Minimal Foundry test outline:

```solidity
function testDelegateFrontRunNonce() public {
    // Setup: alice authorizes bob as operator in Midnight
    vm.prank(alice);
    midnight.setIsAuthorized(bob, true, alice);

    // Alice signs Authorization{authorizer: alice, authorized: charlie, isAuthorized: true, nonce: 0, deadline: ...}
    Authorization memory aliceAuth = Authorization(alice, charlie, true, 0, block.timestamp + 1 hours);
    bytes32 digest = computeDigest(aliceAuth);
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(alicePrivKey, digest);

    // Bob front-runs: constructs malicious payload, signs with bob's key
    Authorization memory bobAuth = Authorization(alice, attacker, true, 0, block.timestamp + 1 hours);
    bytes32 bobDigest = computeDigest(bobAuth);
    (uint8 bv, bytes32 br, bytes32 bs) = vm.sign(bobPrivKey, bobDigest);

    // Bob submits first — nonce 0 consumed, attacker address granted
    vm.prank(bob);
    ecrecoverAuthorizer.setIsAuthorized(bobAuth, Signature(bv, br, bs));
    assertTrue(midnight.isAuthorized(alice, attacker));

    // Alice's transaction reverts
    vm.expectRevert(IEcrecoverAuthorizer.InvalidNonce.selector);
    ecrecoverAuthorizer.setIsAuthorized(aliceAuth, Signature(v, r, s));
}
```
